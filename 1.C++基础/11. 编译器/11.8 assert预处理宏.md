## 1. 简介

在 C++ 中，`assert` 是一个**预处理宏 (macro)**，主要用作调试工具，用于在代码中检查程序运行时的假设（断言）是否为真。

---

## 2. 工作原理

- **检查条件**：`assert` 接受一个表达式作为参数，这个表达式通常代表了程序员认为在程序执行到该点时**必须为真**的一个条件
- **条件为真**：如果表达式的计算结果为 `true`（非零），`assert` 什么也不做，程序继续正常执行
- **条件为假**：如果表达式的计算结果为 `false`（零），`assert` 会触发一个**断言失败 (assertion failure)**，这通常会导致：
  - 向标准错误流 (`stderr`) 输出一条诊断消息，这条消息一般会包含失败的表达式文本、源代码文件名 ([[11.7 预定义宏|`__FILE__`]]) 以及代码行号 (`__LINE__`)
  - 调用 `std::abort()` 函数，**立即终止**程序的执行

---

## 3. 实例

```cpp
#include <cassert>  // 或者 #include <assert.h>
#include <iostream>
#include <vector>

void process_data(int* ptr, int size) {
    // 断言：指针不应为空
    assert(ptr != nullptr && "Error: Input pointer cannot be null.");

    // 断言：大小必须为正数
    assert(size > 0 && "Error: Size must be positive.");

    std::cout << "Processing data at address " << ptr << " with size " << size << std::endl;

    // ... 进行数据处理 ...
    for (int i = 0; i < size; ++i) {
        // 假设 ptr 指向的数组至少有 size 个元素
        // 断言：访问索引必须在有效范围内
        assert(i >= 0 && i < size && "Error: Array index out of bounds.");
        ptr[i] = i * 10;  // 示例操作
        std::cout << "Processed element " << i << ": " << ptr[i] << std::endl;
    }
}

int main() {
    int data[] = { 1, 2, 3 };
    int* null_ptr = nullptr;

    std::cout << "--- Calling process_data with valid arguments ---" << std::endl;
    process_data(data, 3);

    std::cout << "\n--- Calling process_data with null pointer (will likely trigger assert) ---" << std::endl;
    // 下面的调用如果 assert 生效，程序会在这里终止
    process_data(null_ptr, 5);

    // 如果程序能执行到这里，说明上面的 assert 可能被禁用了
    std::cout << "\n--- This line might not be reached if asserts are enabled ---" << std::endl;

    return 0;
}
```

## 4. NDEBUG 宏

`assert` 的行为受到一个名为 `NDEBUG` (No Debug) 的宏的控制：

| 状态 | 行为 |
|------|------|
| **NDEBUG 未定义**（默认情况，通常在调试构建中） | `assert` 宏会展开为执行检查的代码。如果断言失败，程序终止 |
| **NDEBUG 已定义**（通常在发布/优化构建中） | `assert` 宏会展开为空操作（通常是 `((void)0)`）。这意味着所有的断言检查都会在编译时被**完全移除**，不会产生任何运行时代码或开销 |

### 如何定义 NDEBUG

```bash
# 通常在编译器的命令行选项中定义，例如：
# GCC/Clang:
g++ main.cpp -o main -DNDEBUG

# MSVC:
cl main.cpp /DNDEBUG
```

构建系统（如 CMake、Make）或 IDE 通常会自动为 Release 构建配置定义 `NDEBUG`。

## 5. 使用 assert 的注意事项和最佳实践

### 5.1 仅用于调试

`assert` 主要用于在**开发阶段**捕捉程序员的**逻辑错误**或**不应发生的情况**，它假设的是程序**内部状态**的不一致，而不是预期的**运行时错误**（如用户输入错误、文件未找到等）。

### 5.2 不要用于错误处理

对于那些在发布版本中也需要处理的预期错误（例如，检查函数返回值、验证用户输入），应该使用异常处理 (`try-catch`)、返回错误码或其他适当的错误处理机制，而不是 `assert`。

```cpp
// 错误示例：用 assert 处理用户输入
// assert(user_age >= 0);  // 在 Release 版本中不会执行！

// 正确示例：用条件检查处理
if (user_age < 0) {
    std::cerr << "Invalid age input" << std::endl;
    return -1;  // 或抛出异常
}
```

### 5.3 避免副作用

**绝对不要**在 `assert` 的表达式中包含有**副作用**的代码（例如修改变量、调用会改变状态的函数）。因为当 `NDEBUG` 被定义时，这些代码**将不会被执行**，可能导致程序在调试和发布版本中的行为不一致。

```cpp
// 错误示例：副作用在 assert 中
int counter = 0;
assert(++counter > 0);  // 在发布版本中，++counter 不会执行！

// 正确示例：先执行副作用，再 assert
int counter = 0;
++counter;
assert(counter > 0);
```

### 5.4 清晰的条件

断言的条件应该是**明确且易于理解**的，它代表了代码正确运行所依赖的一个前提。可以添加可选的字符串字面量作为第二个参数（需要编译器支持 C++11 或更高标准，或者使用特定库或编译器扩展支持的功能），在断言失败时提供更具体的错误信息。

```cpp
// 带消息的断言
assert(ptr != nullptr && "Input pointer cannot be null");
assert(size > 0 && "Size must be positive");
```

## 6. 自定义断言宏

可以创建自定义断言宏以提供更多功能：

```cpp
#ifdef NDEBUG
    #define MY_ASSERT(condition, message) ((void)0)
#else
    #define MY_ASSERT(condition, message) \
        do { \
            if (!(condition)) { \
                std::cerr << "Assertion failed: " << #condition \
                          << "\nMessage: " << message \
                          << "\nFile: " << __FILE__ \
                          << "\nLine: " << __LINE__ \
                          << "\nFunction: " << __func__ << std::endl; \
                std::abort(); \
            } \
        } while(0)
#endif

// 使用
void process(int* ptr) {
    MY_ASSERT(ptr != nullptr, "Pointer must not be null");
}
```

## 7. static_assert（编译时断言）

C++11 引入了 `static_assert`，用于**编译时**检查：

```cpp
// 编译时检查
static_assert(sizeof(int) == 4, "int must be 4 bytes");
static_assert(sizeof(void*) == 8, "This code requires 64-bit pointers");

template<typename T>
void process() {
    static_assert(std::is_integral<T>::value, "T must be an integral type");
}
```

| 特性 | `assert` | `static_assert` |
|------|----------|-----------------|
| 检查时机 | 运行时 | 编译时 |
| 可被禁用 | 是（NDEBUG） | 否 |
| 用途 | 运行时条件检查 | 类型检查、常量检查 |

---

## 8. 关键要点

1. **调试工具**：`assert` 用于开发阶段检查程序假设
2. **NDEBUG 控制**：定义 `NDEBUG` 会完全禁用 `assert`
3. **不用于错误处理**：运行时错误应使用异常或错误码
4. **避免副作用**：assert 表达式中不要有副作用代码
5. **提供消息**：使用 `&& "message"` 技巧添加错误描述
6. **static_assert**：C++11 编译时断言，不受 NDEBUG 影响
