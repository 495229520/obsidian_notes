# 11.6 预处理 (Preprocessor)

## 1. 简介

**预处理器 (Preprocessor)** 是编译的第一个阶段，在实际编译之前处理源代码。预处理器处理以 `#` 开头的**预处理指令**，进行文本操作如文件包含、[[11.4 define|宏替换]]、条件编译等。

---

## 2. 预处理指令概览

| 指令 | 用途 |
|------|------|
| `#include` | 文件包含 |
| `#define` / `#undef` | [[11.4 define\|宏定义]] / 取消定义 |
| `#if` / `#ifdef` / `#ifndef` | 条件编译 |
| `#elif` / `#else` / `#endif` | 条件编译分支 |
| `#error` | 生成编译错误 |
| `#warning` | 生成编译警告（非标准） |
| `#pragma` | 编译器指令 |
| `#line` | 设置行号和文件名 |

---

## 3. #include 文件包含

### 3.1 两种形式

```cpp
#include <header>    // 在系统/标准库目录中搜索
#include "header"    // 先在当前目录搜索，然后在系统目录搜索
```

### 3.2 搜索顺序

| 形式 | 搜索顺序 |
|------|----------|
| `<header>` | 标准库路径 → 编译器指定的包含路径 |
| `"header"` | 当前文件目录 → 标准库路径 → 包含路径 |

### 3.3 示例

```cpp
#include <iostream>       // 标准库
#include <vector>         // 标准库
#include "my_header.h"    // 项目头文件
#include "../utils.h"     // 相对路径
```

## 4. 条件编译

### 4.1 #ifdef / #ifndef

```cpp
#ifdef DEBUG           // 如果 DEBUG 已定义
    std::cout << "Debug mode" << std::endl;
#endif

#ifndef RELEASE        // 如果 RELEASE 未定义
    // 代码...
#endif
```

### 4.2 #if / #elif / #else

```cpp
#if VERSION >= 2
    // 版本 2 及以上的代码
#elif VERSION == 1
    // 版本 1 的代码
#else
    // 其他版本的代码
#endif
```

### 4.3 defined 操作符

```cpp
#if defined(DEBUG) && !defined(NDEBUG)
    // DEBUG 已定义且 NDEBUG 未定义
#endif

// 等价于
#if defined DEBUG && !defined NDEBUG
#endif
```

### 4.4 头文件保护

防止头文件被多次包含：

```cpp
// 方式一：传统头文件保护
#ifndef MY_HEADER_H
#define MY_HEADER_H

// 头文件内容...

#endif // MY_HEADER_H

// 方式二：#pragma once（被大多数编译器支持）
#pragma once

// 头文件内容...
```

## 5. #error 和 #warning

### #error

生成编译错误并终止编译：

```cpp
#if !defined(__cplusplus)
    #error "This header requires a C++ compiler"
#endif

#if __cplusplus < 201103L
    #error "C++11 or later is required"
#endif
```

### #warning（非标准，但广泛支持）

生成编译警告：

```cpp
#ifdef DEPRECATED_API
    #warning "This API is deprecated, please update your code"
#endif
```

## 6. #pragma 指令

`#pragma` 是编译器特定的指令，行为因编译器而异。

### 6.1 常见 #pragma 指令

```cpp
// 头文件保护
#pragma once

// 禁用特定警告（MSVC）
#pragma warning(disable: 4996)

// 禁用特定警告（GCC/Clang）
#pragma GCC diagnostic ignored "-Wunused-variable"

// 结构体对齐
#pragma pack(push, 1)
struct PackedStruct {
    char a;
    int b;
};
#pragma pack(pop)

// 链接库（MSVC）
#pragma comment(lib, "user32.lib")
```

### 6.2 #pragma message

在编译时输出消息：

```cpp
#pragma message("Compiling with DEBUG enabled")

#ifdef _WIN64
    #pragma message("Building for 64-bit Windows")
#endif
```

---

## 7. #line 指令

修改编译器报告的行号和文件名（用于代码生成器）：

```cpp
#line 100 "generated_code.cpp"
// 后续代码的行号从 100 开始，文件名显示为 generated_code.cpp
```

## 8. 平台和编译器检测

### 8.1 平台检测

```cpp
#ifdef _WIN32
    // Windows（32位或64位）
    #ifdef _WIN64
        // 64位 Windows
    #else
        // 32位 Windows
    #endif
#elif defined(__linux__)
    // Linux
#elif defined(__APPLE__)
    #include <TargetConditionals.h>
    #if TARGET_OS_MAC
        // macOS
    #endif
#elif defined(__unix__)
    // Unix-like 系统
#endif
```

### 8.2 编译器检测

```cpp
#if defined(_MSC_VER)
    // Microsoft Visual C++
    #if _MSC_VER >= 1920
        // VS 2019 或更高
    #endif
#elif defined(__GNUC__)
    // GCC
    #if __GNUC__ >= 9
        // GCC 9 或更高
    #endif
#elif defined(__clang__)
    // Clang
#endif
```

### 8.3 C++ 标准版本检测

```cpp
#if __cplusplus >= 202002L
    // C++20 或更高
#elif __cplusplus >= 201703L
    // C++17
#elif __cplusplus >= 201402L
    // C++14
#elif __cplusplus >= 201103L
    // C++11
#else
    // C++03 或更早
#endif
```

## 9. 预处理运算符

### 9.1 字符串化 `#`

将宏参数转换为字符串：

```cpp
#define STRINGIFY(x) #x
#define TO_STRING(x) STRINGIFY(x)

const char* s = STRINGIFY(Hello);  // "Hello"
const char* v = TO_STRING(__LINE__);  // 行号的字符串形式
```

### 9.2 连接 `##`

连接两个标记：

```cpp
#define CONCAT(a, b) a##b

int xy = 10;
int value = CONCAT(x, y);  // 展开为 xy
```

## 10. 查看预处理结果

```bash
# GCC/Clang
g++ -E source.cpp -o source.i
clang++ -E source.cpp -o source.i

# MSVC
cl /E source.cpp > source.i
cl /P source.cpp  # 生成 source.i 文件
```

## 11. 预处理最佳实践

### 推荐做法

1. **使用 `#pragma once`**：比传统头文件保护更简洁
2. **使用 `constexpr` 替代宏常量**：类型安全
3. **使用 `inline` 函数替代函数宏**：避免副作用
4. **限制条件编译范围**：保持代码可读性

### 避免做法

1. **在头文件中使用 `using namespace`**
2. **复杂的嵌套条件编译**
3. **过度使用宏**

---

## 12. 关键要点

1. **预处理阶段**：在编译之前进行文本处理
2. **#include**：文件包含，`<>` 和 `""` 搜索路径不同
3. **条件编译**：根据条件选择性编译代码
4. **#pragma**：编译器特定指令，行为因编译器而异
5. **头文件保护**：防止重复包含，推荐 `#pragma once`
6. **平台检测**：使用[[11.7 预定义宏|预定义宏]]检测平台和编译器
7. **调试预处理**：使用 `-E` 选项查看预处理结果
