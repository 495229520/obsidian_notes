# 11.7 预定义宏 (Predefined Macros)

## 1. 简介

**预定义宏**是由 C++ 编译器自动定义的一些标识符，无需用户使用 [[11.4 define|#define]] 指令定义，可以直接在代码中使用。这些宏提供了关于编译环境、编译时间、源代码文件、当前行号等有用的信息。

预定义宏通常以**双下划线 `__` 开头和结尾**（如 `__FILE__`），或者以**单下划线加大写字母**开头（尽管后者更常见于标准宏）。

---

## 2. 标准预定义宏

以下宏由 ISO C99 和 ISO C++17 标准指定，所有符合标准的编译器都支持：

| 宏 | 说明 |
|-----|------|
| `__DATE__` | 源文件的编译日期，格式为 `"Mmm dd yyyy"`（如 `"Dec 17 2025"`） |
| `__TIME__` | 当前翻译单元的编译时间，格式为 `"hh:mm:ss"` |
| `__FILE__` | 当前源文件的名称（字符串） |
| `__LINE__` | 当前代码行号（整数） |
| `__cplusplus` | 当编译为 C++ 时定义为整数文本，表示 C++ 标准版本 |

### 2.1 `__cplusplus` 的值

| 值 | C++ 标准版本 |
|----|-------------|
| `199711L` | C++98 或 C++03 |
| `201103L` | C++11 |
| `201402L` | C++14 |
| `201703L` | C++17 |
| `202002L` | C++20 |
| `202302L` | C++23（预期值） |

> **注意**：如果编译器不完全符合某个标准，它可能会定义一个较低的值，或者是一个实验性的值。如果以 C 语言模式编译，此宏通常未定义。

### 2.2 `__func__` / `__FUNCTION__`

- **`__func__`**：C++11 标准规定的，包含当前代码所在的函数名
- **类型**：字符串字面量（在 C++11 及以后是标准规定的一个具有静态存储期的 `const char[]` 变量，行为类似宏）

```cpp
void myFunction() {
    std::cout << "Function name: " << __func__ << std::endl;
    // 输出: Function name: myFunction
}
```

## 3. `__DEBUG` 宏

| 条件 | 定义状态 |
|------|---------|
| 调试阶段（在浏览器中设置调试模式） | 定义为 1 |
| 发布阶段 | 未定义 |

通过这种方式加上 `#ifdef` 就可以避免在发布阶段，用户也可以看到调试的 debug 代码了：

```cpp
#ifdef __DEBUG
    std::cout << "Debug info: x = " << x << std::endl;
#endif
```

## 4. MSVC 预定义宏

Microsoft Visual C++ 编译器定义了一些额外的预定义宏：

| 宏 | 说明 |
|----|------|
| `_CHAR_UNSIGNED` | 如果 `char` 类型为无符号，该宏定义为 1，否则未定义 |
| `__COUNTER__` | 从 0 开始，每次使用都会递增 1 |
| `_DEBUG` | 如果设置了 `/LDd`、`/MDd`、`/MTd`，该宏定义为 1，否则未定义 |
| `__FUNCTION__` | 函数名称，不含修饰名 |
| `__FUNCDNAME__` | 函数名称，包含修饰名 |
| `__FUNCSIG__` | 包含了函数签名的函数名 |
| `_WIN32` | 当编译为 32 位 ARM、64 位 ARM、x68 或 X64 定义为 1，否则未定义 |
| `_WIN64` | 当编译为 64 位 ARM 或 x64 定义为 1，否则未定义 |
| `__TIMESTAMP__` | 最后一次源代码修改的时间和日期 |
| `_MSC_VER` | MSVC 编译器版本号 |

### MSVC 版本号示例

| `_MSC_VER` 值 | Visual Studio 版本 |
|---------------|-------------------|
| 1920 | VS 2019 (16.0) |
| 1930 | VS 2022 (17.0) |
| 1940 | VS 2022 (17.10) |

## 5. 使用示例

### 5.1 调试日志

```cpp
#include <iostream>

#define LOG(msg) \
    std::cout << "[" << __FILE__ << ":" << __LINE__ << "] " \
              << __func__ << "(): " << msg << std::endl

void processData(int value) {
    LOG("Processing value: " << value);
    // 输出: [main.cpp:10] processData(): Processing value: 42
}

int main() {
    processData(42);
    return 0;
}
```

### 5.2 编译时间戳

```cpp
#include <iostream>

int main() {
    std::cout << "Build date: " << __DATE__ << std::endl;
    std::cout << "Build time: " << __TIME__ << std::endl;
    // 输出:
    // Build date: Dec 17 2025
    // Build time: 14:30:45
    return 0;
}
```

### 5.3 C++ 版本检测

```cpp
#include <iostream>

int main() {
    std::cout << "C++ Standard: ";

    #if __cplusplus >= 202002L
        std::cout << "C++20 or later" << std::endl;
    #elif __cplusplus >= 201703L
        std::cout << "C++17" << std::endl;
    #elif __cplusplus >= 201402L
        std::cout << "C++14" << std::endl;
    #elif __cplusplus >= 201103L
        std::cout << "C++11" << std::endl;
    #else
        std::cout << "C++03 or earlier" << std::endl;
    #endif

    return 0;
}
```

### 5.4 平台检测

```cpp
#ifdef _WIN32
    std::cout << "Running on Windows" << std::endl;
    #ifdef _WIN64
        std::cout << "64-bit" << std::endl;
    #else
        std::cout << "32-bit" << std::endl;
    #endif
#elif defined(__linux__)
    std::cout << "Running on Linux" << std::endl;
#elif defined(__APPLE__)
    std::cout << "Running on macOS" << std::endl;
#endif
```

### 5.5 生成唯一标识符

```cpp
// 使用 __COUNTER__ 生成唯一名称
#define CONCAT_IMPL(a, b) a##b
#define CONCAT(a, b) CONCAT_IMPL(a, b)
#define UNIQUE_NAME(prefix) CONCAT(prefix, __COUNTER__)

int UNIQUE_NAME(temp_) = 10;  // temp_0
int UNIQUE_NAME(temp_) = 20;  // temp_1
int UNIQUE_NAME(temp_) = 30;  // temp_2
```

## 6. [[11.8 assert预处理宏|assert]] 中的应用

预定义宏常用于断言和错误报告：

```cpp
#include <cassert>

void validate(int* ptr) {
    if (ptr == nullptr) {
        std::cerr << "Error at " << __FILE__ << ":" << __LINE__
                  << " in " << __func__ << ": null pointer" << std::endl;
        std::abort();
    }
}
```

---

## 7. 关键要点

1. **标准宏**：`__FILE__`、`__LINE__`、`__DATE__`、`__TIME__`、`__cplusplus` 是标准定义的
2. **`__cplusplus`**：用于检测 C++ 标准版本
3. **`__func__`**：C++11 标准化的函数名宏
4. **编译器特定**：`_MSC_VER`、`__GNUC__`、`__clang__` 用于检测编译器
5. **平台检测**：`_WIN32`、`_WIN64`、`__linux__`、`__APPLE__` 用于检测平台
6. **调试辅助**：结合使用这些宏可以创建强大的调试和日志功能
