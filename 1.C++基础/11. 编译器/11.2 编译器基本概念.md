# 11.2 编译器基本概念

## 1. 简介

理解 C++ 编译器的工作原理对于编写高效、正确的代码至关重要。本节介绍编译过程中的核心概念。

---

## 2. 翻译单元 (Translation Unit)

### 定义

**翻译单元**是编译器处理的基本单位，它由以下内容组成：

- 一个源文件（`.cpp`）
- 该源文件包含的所有头文件（经过[[11.6 预处理|预处理]]展开后）

```
翻译单元 = 源文件 + 所有 #include 的头文件（递归展开）
```

### 示例

```cpp
// main.cpp
#include <iostream>      // 标准库头文件
#include "my_header.h"   // 自定义头文件

int main() {
    std::cout << "Hello" << std::endl;
    return 0;
}
```

预处理后，`main.cpp` 会展开成一个包含数万行代码的翻译单元。

### 重要性

- 每个翻译单元**独立编译**
- [[11.1 ODR唯一定义规则|ODR]] 规则在翻译单元级别和程序级别都有约束
- 符号的可见性和链接性与翻译单元相关

---

## 3. 编译阶段

C++ 程序从源代码到可执行文件经历以下阶段：

```
源代码 (.cpp/.h)
    ↓ 预处理
预处理后的代码
    ↓ 编译
汇编代码 (.s)
    ↓ 汇编
目标文件 (.o/.obj)
    ↓ 链接
可执行文件 (.exe/无后缀)
```

### 3.1 预处理 (Preprocessing)

处理以 `#` 开头的[[11.6 预处理|预处理指令]]：

- `#include`：文件包含
- `#define`：[[11.4 define|宏定义]]
- `#ifdef` / `#ifndef`：条件编译
- `#pragma`：编译器指令

```bash
# 查看预处理结果
g++ -E main.cpp -o main.i
```

### 3.2 编译 (Compilation)

将预处理后的代码转换为汇编代码：

- 词法分析
- 语法分析
- 语义分析
- 中间代码生成
- 优化
- 目标代码生成

```bash
# 生成汇编代码
g++ -S main.cpp -o main.s
```

### 3.3 汇编 (Assembly)

将汇编代码转换为目标文件（机器码）：

```bash
# 生成目标文件
g++ -c main.cpp -o main.o
```

### 3.4 链接 (Linking)

将多个目标文件和库文件组合成可执行文件：

- 符号解析：将符号引用与符号定义关联
- 重定位：调整地址

```bash
# 链接生成可执行文件
g++ main.o utils.o -o program
```

## 4. 名称修饰 (Name Mangling)

### 定义

**名称修饰**是 C++ 编译器为了支持[[9.2.1 函数重载|函数重载]]，对函数名进行编码的机制。

### 为什么需要名称修饰？

C++ 允许同名但参数不同的函数共存：

```cpp
void print(int x);
void print(double x);
void print(const char* str);
```

编译器需要为每个函数生成唯一的符号名。

### 修饰后的名称示例

| 原始函数 | GCC/Clang 修饰名 | MSVC 修饰名 |
|----------|------------------|-------------|
| `void print(int)` | `_Z5printi` | `?print@@YAXH@Z` |
| `void print(double)` | `_Z5printd` | `?print@@YAXN@Z` |
| `int MyClass::getValue()` | `_ZN7MyClass8getValueEv` | `?getValue@MyClass@@QEAAHXZ` |

### 查看符号名

```bash
# Linux/macOS
nm program.o
c++filt _Z5printi  # 输出: print(int)

# Windows
dumpbin /symbols program.obj
undname ?print@@YAXH@Z
```

### extern "C" 禁用名称修饰

使用 [[10.1 C和C++联调#`extern "C"` 关键字|extern "C"]] 可以禁用名称修饰：

```cpp
extern "C" void c_function();  // 符号名保持为 c_function
```

## 5. 链接性 (Linkage)

链接性决定了一个名称在不同翻译单元中的可见性：

### 5.1 外部链接 (External Linkage)

名称可以在**其他翻译单元**中被引用：

```cpp
// file1.cpp
int global_var = 10;           // 外部链接
void global_func() { }         // 外部链接

// file2.cpp
extern int global_var;         // 引用 file1.cpp 中的定义
void global_func();            // 引用 file1.cpp 中的定义
```

### 5.2 内部链接 (Internal Linkage)

名称只在**当前翻译单元**中可见：

```cpp
// file1.cpp
static int internal_var = 10;  // 内部链接
namespace {
    int anonymous_var = 20;    // 内部链接
}
const int CONST_VAR = 30;      // 内部链接（const 默认）

// file2.cpp
// 无法访问 file1.cpp 中的 internal_var、anonymous_var、CONST_VAR
```

### 5.3 无链接 (No Linkage)

名称只在**当前作用域**中可见：

```cpp
void func() {
    int local_var = 10;  // 无链接，只在 func 内可见
}
```

### 链接性总结

| 实体 | 默认链接性 |
|------|-----------|
| 全局变量 | 外部链接 |
| 全局函数 | 外部链接 |
| `static` 全局变量/函数 | 内部链接 |
| `const` 全局变量 | 内部链接 |
| 匿名命名空间成员 | 内部链接 |
| 局部变量 | 无链接 |
| 类成员 | 外部链接（通过类名访问） |

## 6. 存储期 (Storage Duration)

存储期决定了对象的生命周期：

### 6.1 自动存储期

局部变量，在作用域结束时销毁：

```cpp
void func() {
    int x = 10;  // 自动存储期，func 返回时销毁
}
```

### 6.2 静态存储期

全局变量、`static` 局部变量，程序运行期间一直存在：

```cpp
int global = 10;           // 静态存储期

void func() {
    static int count = 0;  // 静态存储期，只初始化一次
    count++;
}
```

### 6.3 动态存储期

通过 `new` / `malloc` 分配的内存：

```cpp
int* p = new int(10);  // 动态存储期
delete p;              // 手动释放
```

### 6.4 线程存储期 (C++11)

每个线程有独立的副本：

```cpp
thread_local int tls_var = 0;  // 每个线程有自己的 tls_var
```

## 7. 编译器优化

### 常见优化级别

| 选项 | 说明 |
|------|------|
| `-O0` | 无优化（调试用） |
| `-O1` | 基本优化 |
| `-O2` | 标准优化（推荐） |
| `-O3` | 激进优化 |
| `-Os` | 优化代码大小 |
| `-Ofast` | 最快速度（可能违反标准） |

### 常见优化技术

- **内联展开**：将函数调用替换为函数体
- **常量折叠**：编译时计算常量表达式
- **死代码消除**：移除不可达代码
- **循环优化**：展开、向量化等
- **尾调用优化**：优化[[9.1.11 递归函数#尾递归|尾递归]]

---

## 8. 关键要点

1. **翻译单元**：源文件 + 所有包含的头文件，是编译的基本单位
2. **编译阶段**：预处理 → 编译 → 汇编 → 链接
3. **名称修饰**：C++ 编译器为支持重载对函数名进行编码
4. **链接性**：外部链接（跨文件可见）、内部链接（当前文件）、无链接（当前作用域）
5. **存储期**：自动、静态、动态、线程
6. **优化级别**：`-O0` 到 `-O3`，权衡编译时间和运行效率
