# 11.1 ODR 唯一定义规则 (One Definition Rule)

## 1. 简介

**ODR（One Definition Rule，唯一定义规则）** 是 C++ 中一个非常重要的规则，它规定了程序中各种实体（变量、函数、类、模板等）可以被定义的次数和方式。

违反 ODR 通常会导致：
- 链接错误（如 "multiple definition"）
- 未定义行为（更危险，程序可能看起来正常但行为不可预测）

---

## 2. ODR 的核心内容

### 2.1 整个程序范围

对于以下实体，**整个程序中只能有一个定义**：

- 非 [[11.3 inline|inline]] 函数
- 非 inline 变量（C++17）
- 具有外部链接的变量

```cpp
// file1.cpp
int global_var = 10;  // 定义

// file2.cpp
int global_var = 20;  // 错误！ODR 违规，重复定义
```

### 2.2 每个翻译单元范围

对于以下实体，**每个[[11.2 编译器基本概念#翻译单元|翻译单元]]中只能有一个定义**：

- 类型（class、struct、union、enum）
- inline 函数和变量
- 模板

```cpp
// 同一个 .cpp 文件中
class MyClass { };
class MyClass { };  // 错误！同一翻译单元中重复定义
```

---

## 3. 声明 vs 定义

理解 ODR 的关键是区分**声明**和**定义**：

| 实体 | 声明 | 定义 |
|------|------|------|
| **变量** | `extern int x;` | `int x;` 或 `int x = 10;` |
| **函数** | `void func();` | `void func() { }` |
| **类** | `class MyClass;` | `class MyClass { };` |
| **typedef** | — | `typedef int Int;`（同时是声明和定义） |

### 示例

```cpp
// 头文件 my_header.h
#ifndef MY_HEADER_H
#define MY_HEADER_H

extern int global_var;  // 声明（可以多次）

void print_message(const char* msg);  // 声明（可以多次）

class MyClass;  // 前向声明（可以多次）

#endif
```

```cpp
// 源文件 my_source.cpp
#include "my_header.h"

int global_var = 42;  // 定义（整个程序只能一次）

void print_message(const char* msg) {  // 定义（整个程序只能一次）
    std::cout << msg << std::endl;
}
```

---

## 4. 多翻译单元中的定义

某些实体**可以**在多个翻译单元中定义，但所有定义必须**完全相同**：

### 4.1 类定义

```cpp
// header.h
#ifndef HEADER_H
#define HEADER_H

class Point {
public:
    int x, y;
    int getX() const { return x; }
};

#endif
```

每个包含 `header.h` 的 `.cpp` 文件都会有一份 `Point` 类的定义，这是**允许的**，只要所有定义完全相同。

### 4.2 inline 函数

```cpp
// header.h
inline int add(int a, int b) {
    return a + b;
}
```

[[11.3 inline|inline 函数]]可以在多个翻译单元中定义，但所有定义必须相同。

### 4.3 模板

```cpp
// header.h
template<typename T>
T max(T a, T b) {
    return (a > b) ? a : b;
}
```

[[9.2.2 函数模版|模板]]定义通常放在头文件中，会被多个翻译单元包含。

## 5. ODR 违规示例

### 5.1 明显的重复定义

```cpp
// file1.cpp
void display_message() {
    std::cout << "Message from file1" << std::endl;
}

// file2.cpp
void display_message() {  // ODR 违规！
    std::cout << "Message from file2" << std::endl;
}
```

**链接错误**：`multiple definition of 'display_message()'`

### 5.2 隐蔽的 ODR 违规

```cpp
// config.h
#ifdef DEBUG_MODE
struct Config {
    int value;
    int debug_level;  // DEBUG_MODE 时多一个成员
};
#else
struct Config {
    int value;
};
#endif
```

```cpp
// file1.cpp
#define DEBUG_MODE
#include "config.h"
// Config 有 2 个成员

// file2.cpp
#include "config.h"
// Config 有 1 个成员
// ODR 违规！同一个类的定义不同
```

这种 ODR 违规**不会产生链接错误**，但会导致**未定义行为**。

### 5.3 头文件中定义变量

```cpp
// helper.h - 错误示例
#ifndef HELPER_H
#define HELPER_H

int helper_count = 0;  // 定义！每个包含此头文件的 .cpp 都会有一份

#endif
```

```cpp
// file1.cpp
#include "helper.h"  // 定义了 helper_count

// file2.cpp
#include "helper.h"  // 又定义了 helper_count
// 链接时：multiple definition of 'helper_count'
```

**正确做法**：

```cpp
// helper.h
#ifndef HELPER_H
#define HELPER_H

extern int helper_count;  // 声明

// 或者使用 inline（C++17）
inline int helper_count = 0;

#endif

// helper.cpp
#include "helper.h"
int helper_count = 0;  // 定义（仅此一处）
```

## 6. 如何避免 ODR 违规

### 6.1 头文件保护

使用 `#ifndef` / `#define` / `#endif` 或 `#pragma once`：

```cpp
// 方式一：传统头文件保护
#ifndef MY_HEADER_H
#define MY_HEADER_H
// 内容...
#endif

// 方式二：#pragma once（大多数编译器支持）
#pragma once
// 内容...
```

### 6.2 正确使用 extern

```cpp
// header.h
extern int global_var;  // 声明

// source.cpp
int global_var = 0;     // 定义
```

### 6.3 使用 inline（C++17）

```cpp
// header.h
inline int global_var = 0;  // C++17: inline 变量可以在头文件中定义
```

### 6.4 使用匿名命名空间或 static

```cpp
// file.cpp
namespace {
    int internal_var = 0;  // 内部链接，不会与其他文件冲突
}

// 或者
static int internal_var2 = 0;  // 同样是内部链接
```

## 7. 特殊情况

### 7.1 const 变量

`const` 变量默认具有**内部链接**（仅在当前翻译单元可见）：

```cpp
// header.h
const int MAX_SIZE = 100;  // 内部链接，每个翻译单元有自己的副本
```

如果需要外部链接：

```cpp
// header.h
extern const int MAX_SIZE;  // 声明

// source.cpp
extern const int MAX_SIZE = 100;  // 定义，需要 extern
```

### 7.2 constexpr 变量

C++17 起，`constexpr` 静态成员变量隐式为 `inline`：

```cpp
struct Config {
    static constexpr int MAX_SIZE = 100;  // C++17: 隐式 inline
};
```

---

## 8. 关键要点

1. **非 inline 实体**：整个程序中只能定义一次
2. **inline/模板/类**：可以在多个翻译单元定义，但必须完全相同
3. **声明 vs 定义**：声明可以多次，定义只能一次
4. **头文件保护**：防止同一翻译单元内重复定义
5. **隐蔽违规**：条件编译导致的定义不一致会引发未定义行为
6. **inline 变量**：C++17 允许在头文件中定义 inline 变量
