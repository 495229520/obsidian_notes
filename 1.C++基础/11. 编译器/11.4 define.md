# 11.4 define 宏定义

## 1. 简介

`#define` 是[[11.6 预处理|预处理]]指令，用于定义**宏 (Macro)**。宏在预处理阶段进行**文本替换**，不涉及类型检查。

---

## 2. 基本语法

### 2.1 对象宏（常量宏）

```cpp
#define 宏名 替换文本

// 示例
#define PI 3.14159
#define MAX_SIZE 100
#define VERSION "1.0.0"
```

### 2.2 函数宏（带参数的宏）

```cpp
#define 宏名(参数列表) 替换文本

// 示例
#define SQUARE(x) ((x) * (x))
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define PRINT_VAR(var) std::cout << #var << " = " << var << std::endl
```

### 2.3 取消宏定义

```cpp
#undef PI  // 取消 PI 的定义
```

---

## 3. 宏展开示例

```cpp
#define SQUARE(x) ((x) * (x))
#define PI 3.14159

int main() {
    double area = PI * SQUARE(5);
    // 预处理后变为：
    // double area = 3.14159 * ((5) * (5));
}
```

## 4. 特殊操作符

### 4.1 字符串化操作符 `#`

将宏参数转换为字符串：

```cpp
#define STRINGIFY(x) #x
#define PRINT_VAR(var) std::cout << #var << " = " << var << std::endl

int value = 42;
PRINT_VAR(value);  // 输出: value = 42
// 展开为: std::cout << "value" << " = " << value << std::endl;
```

### 4.2 连接操作符 `##`

连接两个标记（Token）：

```cpp
#define CONCAT(a, b) a##b
#define MAKE_VAR(n) var##n

int MAKE_VAR(1) = 10;   // 展开为: int var1 = 10;
int MAKE_VAR(2) = 20;   // 展开为: int var2 = 20;

int CONCAT(my, Var) = 30;  // 展开为: int myVar = 30;
```

---

## 5. 多行宏

使用反斜杠 `\` 续行：

```cpp
#define SWAP(a, b) \
    do { \
        auto temp = (a); \
        (a) = (b); \
        (b) = temp; \
    } while(0)

// 使用
int x = 1, y = 2;
SWAP(x, y);  // 交换 x 和 y
```

> **为什么用 `do { } while(0)`？** 确保宏在任何上下文中都能正确使用，避免悬空 else 等问题。

## 6. 条件编译宏

### 6.1 基本条件编译

```cpp
#define DEBUG

#ifdef DEBUG
    std::cout << "Debug mode" << std::endl;
#endif

#ifndef RELEASE
    // RELEASE 未定义时执行
#endif

#if defined(DEBUG) && !defined(NDEBUG)
    // 复杂条件
#endif
```

### 6.2 头文件保护

```cpp
// my_header.h
#ifndef MY_HEADER_H
#define MY_HEADER_H

// 头文件内容...

#endif  // MY_HEADER_H
```

### 6.3 平台检测

```cpp
#ifdef _WIN32
    #include <windows.h>
#elif defined(__linux__)
    #include <unistd.h>
#elif defined(__APPLE__)
    #include <TargetConditionals.h>
#endif
```

## 7. 宏的陷阱

### 7.1 括号问题

```cpp
// 错误示例
#define SQUARE(x) x * x
int result = SQUARE(1 + 2);  // 展开为: 1 + 2 * 1 + 2 = 5（错误！）

// 正确示例
#define SQUARE(x) ((x) * (x))
int result = SQUARE(1 + 2);  // 展开为: ((1 + 2) * (1 + 2)) = 9
```

### 7.2 多次求值

```cpp
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int x = 5;
int result = MAX(x++, 3);
// 展开为: ((x++) > (3) ? (x++) : (3))
// x 可能被递增两次！
```

### 7.3 分号问题

```cpp
// 错误示例
#define LOG(msg) std::cout << msg << std::endl;

if (error)
    LOG("Error occurred");  // 分号导致 else 匹配问题
else
    LOG("Success");

// 正确示例：使用 do-while(0)
#define LOG(msg) \
    do { std::cout << msg << std::endl; } while(0)
```

### 7.4 作用域问题

宏没有作用域，定义后全局有效：

```cpp
#define value 100

namespace MySpace {
    int x = value;  // 宏替换，不受命名空间限制
}
```

## 8. 宏 vs 现代 C++ 替代方案

| 宏用途 | 现代 C++ 替代 |
|--------|---------------|
| 常量定义 | `constexpr` / `const` |
| 函数宏 | [[11.3 inline\|inline 函数]] / [[9.2.2 函数模版\|模板]] |
| 类型别名 | `using` / `typedef` |
| 条件编译 | `if constexpr` (C++17) |

### 示例对比

```cpp
// 旧方式：宏
#define PI 3.14159
#define SQUARE(x) ((x) * (x))

// 现代方式：constexpr 和 inline
constexpr double PI = 3.14159;
inline int square(int x) { return x * x; }

// 模板版本（类型安全）
template<typename T>
constexpr T square(T x) { return x * x; }
```

## 9. 宏的合理使用场景

尽管现代 C++ 提供了更好的替代方案，但宏在某些场景仍然有用：

### 9.1 头文件保护

```cpp
#ifndef MY_HEADER_H
#define MY_HEADER_H
// ...
#endif
```

### 9.2 平台/编译器检测

```cpp
#ifdef _MSC_VER
    // MSVC 特定代码
#endif
```

### 9.3 调试日志

```cpp
#ifdef DEBUG
    #define LOG(msg) std::cout << __FILE__ << ":" << __LINE__ << " " << msg << std::endl
#else
    #define LOG(msg)  // 空宏，Release 版本不输出
#endif
```

### 9.4 代码生成

```cpp
// 批量生成相似代码
#define DECLARE_GETTER(type, name) \
    type get_##name() const { return m_##name; }

class Person {
private:
    std::string m_name;
    int m_age;
public:
    DECLARE_GETTER(std::string, name)
    DECLARE_GETTER(int, age)
};
```

### 9.5 [[11.8 assert预处理宏|断言]]

```cpp
#define ASSERT(condition) \
    do { \
        if (!(condition)) { \
            std::cerr << "Assertion failed: " #condition \
                      << " at " << __FILE__ << ":" << __LINE__ << std::endl; \
            std::abort(); \
        } \
    } while(0)
```

## 10. 关键要点

1. **文本替换**：宏在预处理阶段进行纯文本替换
2. **无类型检查**：宏不进行任何类型检查
3. **括号保护**：函数宏的参数和整体都要加括号
4. **多次求值**：宏参数可能被求值多次
5. **无作用域**：宏定义全局有效，直到 `#undef`
6. **现代替代**：优先使用 `constexpr`、`inline`、模板等
7. **合理使用**：条件编译、头文件保护、调试日志等场景仍适合用宏
