## 1. 简介

**命名空间 (Namespace)** 是 C++ 用于组织代码和**避免名称冲突**的机制。它将相关的声明和定义组织在一个逻辑单元中。

---

## 2. 基本语法

### 2.1 定义命名空间

```cpp
namespace MyNamespace {
    int value = 10;

    void print() {
        std::cout << "MyNamespace::print()" << std::endl;
    }

    class MyClass {
        // 类定义...
    };
}
```

### 2.2 访问命名空间成员

```cpp
// 方式一：使用作用域解析运算符 ::
MyNamespace::value = 20;
MyNamespace::print();
MyNamespace::MyClass obj;

// 方式二：using 声明（引入特定成员）
using MyNamespace::print;
print();  // 直接调用

// 方式三：using 指令（引入整个命名空间）
using namespace MyNamespace;
value = 30;  // 直接访问
print();
```

---

## 3. 命名空间的特性

### 3.1 可以分散定义

同一个命名空间可以在**多个位置**定义，编译器会合并它们：

```cpp
// file1.cpp
namespace Utils {
    void funcA() { }
}

// file2.cpp
namespace Utils {
    void funcB() { }
}

// 两个函数都属于 Utils 命名空间
```

### 3.2 嵌套命名空间

```cpp
namespace Outer {
    namespace Inner {
        void func() { }
    }
}

// 访问
Outer::Inner::func();

// C++17 简化语法
namespace Outer::Inner {
    void func() { }
}
```

### 3.3 命名空间别名

```cpp
namespace VeryLongNamespaceName {
    void func() { }
}

// 创建别名
namespace VLN = VeryLongNamespaceName;
VLN::func();  // 使用别名访问
```

## 4. 匿名命名空间

**匿名命名空间**中的成员具有**内部链接**，只在当前[[11.2 编译器基本概念#翻译单元|翻译单元]]中可见。

```cpp
// file.cpp
namespace {
    int internal_var = 10;    // 内部链接
    void internal_func() { }  // 内部链接
}

// 等价于 static（但匿名命名空间更推荐）
static int internal_var2 = 20;
```

### 匿名命名空间 vs static

| 特性 | 匿名命名空间 | `static` |
|------|-------------|----------|
| 可用于类型 | ✓ | ✗ |
| 可用于模板 | ✓ | ✗ |
| C++ 推荐 | ✓ | 旧式写法 |

```cpp
namespace {
    class InternalClass { };  // 可以用匿名命名空间
}

// static class InternalClass { };  // 错误！static 不能用于类
```

## 5. using 声明 vs using 指令

### 5.1 using 声明

引入**特定成员**到当前作用域：

```cpp
using std::cout;
using std::endl;

cout << "Hello" << endl;  // 只有 cout 和 endl 可直接使用
std::vector<int> v;       // vector 仍需完整限定
```

### 5.2 using 指令

引入**整个命名空间**：

```cpp
using namespace std;

cout << "Hello" << endl;
vector<int> v;  // 所有 std 成员都可直接使用
```

### 最佳实践

```cpp
// ❌ 避免在头文件中使用 using namespace
// header.h
using namespace std;  // 会污染所有包含此头文件的文件

// ✓ 推荐在源文件的局部作用域使用
void func() {
    using namespace std;  // 只在函数内有效
    cout << "Hello" << endl;
}

// ✓ 推荐使用 using 声明而非 using 指令
using std::cout;
using std::vector;
```

## 6. 全局命名空间

没有显式命名空间的声明位于**全局命名空间**：

```cpp
int global_var = 10;  // 全局命名空间

namespace MySpace {
    int var = 20;

    void func() {
        int var = 30;        // 局部变量
        ::global_var = 100;  // 显式访问全局命名空间
        ::var;               // 错误：全局没有 var
    }
}
```

`::` 前缀表示从全局命名空间开始查找。

## 7. inline 命名空间（C++11）

**inline 命名空间**的成员可以直接通过外层命名空间访问，常用于版本控制：

```cpp
namespace Library {
    inline namespace v2 {
        void func() {
            std::cout << "v2" << std::endl;
        }
    }

    namespace v1 {
        void func() {
            std::cout << "v1" << std::endl;
        }
    }
}

// 使用
Library::func();     // 调用 v2 版本（inline）
Library::v1::func(); // 显式调用 v1 版本
Library::v2::func(); // 显式调用 v2 版本
```

### 版本控制示例

```cpp
namespace MyLib {
    #if MYLIB_VERSION >= 2
        inline namespace v2 {
            // 新版本 API
        }
    #else
        inline namespace v1 {
            // 旧版本 API
        }
    #endif
}
```

## 8. 命名空间与 ADL

**ADL (Argument-Dependent Lookup)**，也叫 Koenig 查找，允许在函数参数类型所在的命名空间中查找函数：

```cpp
namespace Graphics {
    class Point { };
    void draw(const Point& p) { }
}

int main() {
    Graphics::Point p;
    draw(p);  // ADL：在 Graphics 命名空间中找到 draw
    // 不需要写 Graphics::draw(p)
}
```

### ADL 与 swap

```cpp
namespace MyLib {
    class MyClass { };

    void swap(MyClass& a, MyClass& b) {
        // 自定义 swap 实现
    }
}

template<typename T>
void generic_func(T& a, T& b) {
    using std::swap;  // 引入 std::swap 作为候选
    swap(a, b);       // ADL 会找到 MyLib::swap（如果 T 是 MyClass）
}
```

## 9. std 命名空间

C++ 标准库的所有内容都在 `std` 命名空间中：

```cpp
std::cout
std::vector
std::string
std::move
```

> **注意**：不要向 `std` 命名空间添加内容（除了模板特化）。

---

## 10. 关键要点

1. **避免冲突**：命名空间将名称组织在独立的作用域中
2. **可分散定义**：同一命名空间可以在多个文件中定义
3. **匿名命名空间**：成员具有内部链接，替代 `static`
4. **using 声明 vs 指令**：using 声明更安全，避免名称污染
5. **inline 命名空间**：用于 API 版本控制
6. **ADL**：根据参数类型自动在其命名空间中查找函数
7. **最佳实践**：不要在头文件中使用 `using namespace`
