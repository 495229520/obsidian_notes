# 12.3.1 静态多态与动态多态

## 多态的概念

**多态**（Polymorphism）是[[12.1.1 面向对象概念|面向对象编程]]的三大核心特性之一，它允许同一个操作作用于不同的对象，可以有不同的解释，从而产生不同的执行结果。

多态的本质是**"一个接口，多种实现"**。它使得代码具有更好的通用性和可扩展性。

---

## 多态的分类

C++ 中的多态分为两种：

| 类型 | 别名 | 绑定时机 | 实现方式 |
|------|------|----------|----------|
| **静态多态** | 编译时多态 | 编译阶段 | [[9.2.1 函数重载\|函数重载]]、[[9.2.2 函数模版\|模板]]、运算符重载 |
| **动态多态** | 运行时多态 | 运行阶段 | [[12.3.2 虚函数\|虚函数]]、继承 |

---

## 静态多态（编译时多态）

### 定义

静态多态在**编译阶段**就确定了要调用哪个函数，由编译器根据参数类型或模板参数完成函数的选择。

### 实现方式

#### 1. 函数重载

[[9.2.1 函数重载|函数重载]]允许在同一作用域内定义多个同名函数，只要参数列表不同：

```cpp
#include <iostream>
#include <cstring>

class Printer {
public:
    void print(int value) {
        std::cout << "Printing int: " << value << std::endl;
    }

    void print(double value) {
        std::cout << "Printing double: " << value << std::endl;
    }

    void print(const char* value) {
        std::cout << "Printing string: " << value << std::endl;
    }
};

int main() {
    Printer p;
    p.print(42);        // 调用 print(int)
    p.print(3.14);      // 调用 print(double)
    p.print("Hello");   // 调用 print(const char*)
}
```

#### 2. 函数模板

[[9.2.2 函数模版|函数模板]]允许编写泛型代码，编译器根据调用时的参数类型自动生成对应版本：

```cpp
template <typename T>
T compare(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    std::cout << compare(10, 20) << std::endl;        // 实例化 compare<int>
    std::cout << compare(3.14, 2.71) << std::endl;    // 实例化 compare<double>
}
```

#### 3. 运算符重载

[[9.3.1 运算符重载|运算符重载]]也是静态多态的一种形式：

```cpp
class Complex {
public:
    double real, imag;

    Complex operator+(const Complex& other) const {
        return {real + other.real, imag + other.imag};
    }
};
```

### 静态多态的特点

| 特点 | 说明 |
|------|------|
| **高效** | 编译时确定，无运行时开销 |
| **类型安全** | 编译器进行类型检查 |
| **灵活性有限** | 必须在编译时知道所有类型 |

---

## 动态多态（运行时多态）

### 定义

动态多态在**运行阶段**才确定要调用哪个函数。它依赖于[[12.2.1 类的继承|继承]]关系和[[12.3.2 虚函数|虚函数]]机制，通过基类指针或引用调用派生类的重写函数。

### 实现条件

实现动态多态需要满足以下条件：
1. 存在**继承关系**
2. 基类中有**虚函数**（`virtual`）
3. 派生类**重写**（override）基类的虚函数
4. 通过**基类指针或引用**调用虚函数

### 代码示例

```cpp
#include <iostream>

// 基类
class Animal {
public:
    // 虚函数 - 允许派生类重写
    virtual void speak() {
        std::cout << "Animal makes a sound." << std::endl;
    }

    // 虚析构函数 - 确保正确释放派生类资源
    virtual ~Animal() {
        std::cout << "Animal destructor called." << std::endl;
    }
};

// 派生类 Dog
class Dog : public Animal {
public:
    void speak() override {  // override 关键字明确表示重写
        std::cout << "Dog barks: Woof!" << std::endl;
    }

    ~Dog() override {
        std::cout << "Dog destructor called." << std::endl;
    }
};

// 派生类 Cat
class Cat : public Animal {
public:
    void speak() override {
        std::cout << "Cat meows: Meow!" << std::endl;
    }

    ~Cat() override {
        std::cout << "Cat destructor called." << std::endl;
    }
};

int main() {
    // 使用基类指针指向不同派生类对象
    Animal* dog = new Dog();    // 基类指针指向派生类对象
    Animal* cat = new Cat();

    // 动态多态：运行时根据实际对象类型调用对应函数
    dog->speak();  // 输出: Dog barks: Woof!
    cat->speak();  // 输出: Cat meows: Meow!

    // 多态数组示例
    Animal* animals[3];
    animals[0] = new Animal();
    animals[1] = new Dog();
    animals[2] = new Cat();

    std::cout << "\n--- Polymorphic array ---" << std::endl;
    for (int i = 0; i < 3; ++i) {
        animals[i]->speak();  // 根据实际类型调用对应的 speak()
    }

    // 释放内存
    delete dog;
    delete cat;
    for (int i = 0; i < 3; ++i) {
        delete animals[i];
    }

    return 0;
}
```

### 动态多态的特点

| 特点 | 说明 |
|------|------|
| **灵活性高** | 运行时决定调用哪个函数 |
| **可扩展性好** | 新增派生类无需修改现有代码 |
| **有运行时开销** | 需要通过虚函数表（vtable）间接调用 |

---

## 静态多态 vs 动态多态

| 对比项 | 静态多态 | 动态多态 |
|--------|---------|----------|
| **绑定时机** | 编译时（早绑定） | 运行时（晚绑定） |
| **实现机制** | 函数重载、模板 | 虚函数 + 继承 |
| **性能** | 更高效，无间接调用 | 有虚函数表查找开销 |
| **灵活性** | 类型必须编译时确定 | 运行时可处理不同类型 |
| **代码复用** | 通过模板实现 | 通过继承实现 |

---

## 关键要点

1. **多态的本质**：一个接口，多种实现
2. **静态多态**：编译时确定，通过函数重载和模板实现，效率高
3. **动态多态**：运行时确定，通过虚函数和继承实现，灵活性高
4. **虚函数**：实现动态多态的核心机制
5. **使用场景**：需要扩展性时用动态多态，追求性能时用静态多态
