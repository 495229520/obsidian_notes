# 12.3.6 从内存角度理解

## 概述

理解 C++ 类的内存布局对于深入理解[[12.3.1 静态多态与动态多态|多态]]、[[12.2.1 类的继承|继承]]和对象模型至关重要。本节从内存角度分析类对象在内存中的实际存储方式。

---

## 基类的内存分布

### 简单类的内存布局

```cpp
class Object {
    int data_0 = 0;
    int data_1 = 1;
public:
    int data_2 = 2;
    int data_3 = 3;
};

int main() {
    Object obj;
    int* nRead = (int*)&obj;
    std::cout << "OBJ内存地址: " << &obj << std::endl;
    for (int i = 0; i < sizeof(obj) / 4; i++) {
        std::cout << "内存地址: " << &nRead[i] << " 值: " << nRead[i] << std::endl;
    }
}
```

**输出示例**：
```
OBJ内存地址: 00000009D5EB4FA88
内存地址: 00000009D5EB4FA88值: 0
内存地址: 00000009D5EB4FA8C值: 1
内存地址: 00000009D5EB4FA90值: 2
内存地址: 00000009D5EB4FA94值: 3
```

### 内存布局图

```
Object 对象内存布局：
┌─────────────────┐  低地址
│ data_0 = 0      │  偏移 +0
├─────────────────┤
│ data_1 = 1      │  偏移 +4
├─────────────────┤
│ data_2 = 2      │  偏移 +8
├─────────────────┤
│ data_3 = 3      │  偏移 +12
└─────────────────┘  高地址
```

> **注意**：成员变量按**声明顺序**排列，与访问修饰符（public/private）无关。

## 包含虚函数的内存分布

当类包含[[12.3.2 虚函数|虚函数]]时，对象内存中会增加一个**虚函数指针（vptr）**：

```cpp
class Object {
    int data_0 = 0;
    int data_1 = 1;
public:
    int data_2 = 2;
    int data_3 = 3;

    virtual void Move() {
        std::cout << "Object Moving" << std::endl;
    }
};
```

### 内存布局（64位系统）

```
Object 对象内存布局（有虚函数）：
┌─────────────────┐  低地址
│     vptr        │  8 字节（指向虚函数表）
├─────────────────┤
│ data_0 = 0      │  4 字节
├─────────────────┤
│ data_1 = 1      │  4 字节
├─────────────────┤
│ data_2 = 2      │  4 字节
├─────────────────┤
│ data_3 = 3      │  4 字节
├─────────────────┤
│   (padding)     │  4 字节（内存对齐填充）
└─────────────────┘  高地址

总大小：8 + 16 + 4 = 28 → 对齐到 32 字节
```

> **注意**：vptr 通常位于对象的**最开始位置**（部分编译器可能不同）。

## 继承时虚函数指针的变化

### 派生类继承虚函数

```cpp
class Object {
public:
    int data_0 = 0;
    virtual void Move() {
        std::cout << "Object Moving" << std::endl;
    }
};

class MoveObject : public Object {
public:
    int Mdata_0 = 20;
    void Move() override {
        std::cout << "MoveObject Moving" << std::endl;
    }
};
```

### 内存布局

```
MoveObject 对象内存布局：
┌─────────────────┐  低地址
│     vptr        │  → 指向 MoveObject 的 vtable
├─────────────────┤
│ data_0 = 0      │  （继承自 Object）
├─────────────────┤
│ Mdata_0 = 20    │  （MoveObject 自己的成员）
└─────────────────┘  高地址

vtable（虚函数表）内容：
┌─────────────────┐
│ &MoveObject::Move │  → 指向派生类重写的函数
└─────────────────┘
```

## 多态调用的内存视角

```cpp
int main() {
    Object* ptr = new MoveObject();

    // 内存中的实际过程：
    // 1. ptr 指向 MoveObject 对象
    // 2. 通过 ptr 找到对象的 vptr
    // 3. vptr 指向 MoveObject 的 vtable
    // 4. 在 vtable 中找到 Move 函数地址
    // 5. 调用 MoveObject::Move()

    ptr->Move();  // 输出: MoveObject Moving

    delete ptr;
}
```

```
调用流程：
ptr ──→ MoveObject 对象
            │
            │ vptr
            ↓
        MoveObject::vtable
            │
            │ [0]
            ↓
        MoveObject::Move()
```

## 内存布局的关键规则

| 规则 | 说明 |
|------|------|
| **成员按声明顺序排列** | 与访问修饰符无关 |
| **基类成员在前** | 派生类成员紧随其后 |
| **vptr 通常在最前** | 包含虚函数时自动添加 |
| **内存对齐** | 可能存在填充字节 |
| **继承链越深，对象越大** | 每层都会增加成员 |

---

## 关键要点

1. **内存连续**：类对象的成员在内存中连续存储
2. **继承叠加**：派生类内存 = 基类部分 + 自身部分
3. **虚函数指针**：有虚函数的类会增加一个 vptr
4. **访问修饰符**：不影响内存布局，只影响访问权限
5. **理解多态**：通过 vptr → vtable → 函数地址 实现动态绑定
