# 12.3.3 对象多态

## 多态的实现方式

实现[[12.3.1 静态多态与动态多态|动态多态]]需要通过**基类指针**或**基类引用**来调用[[12.3.2 虚函数|虚函数]]。直接使用对象（值语义）无法实现多态。

---

## 对象切片问题（Object Slicing）

### 什么是对象切片

当把派生类对象赋值给基类对象时，派生类特有的部分会被"切掉"，只保留基类部分。这被称为**对象切片**（Object Slicing）。

### 示例

```cpp
#include <iostream>

class BaseObject {
public:
    int x;

    // 虚函数
    virtual void Move() {
        std::cout << "BaseObject Moving" << std::endl;
    }

    void test() {
        std::cout << "BaseObject::test()" << std::endl;
    }
};

class DFKObject : public BaseObject {
public:
    void Move() override {
        std::cout << "DFKObject Moving" << std::endl;
    }
};

class MasterObject : public BaseObject {
public:
    // 没有重写 Move，使用基类版本
    // 新增成员函数
    void test() {  // 隐藏了基类的 test
        std::cout << "MasterObject::test()" << std::endl;
    }
};

int main() {
    // 对象切片：派生类对象赋值给基类对象
    MasterObject master;
    BaseObject base = master;  // 切片！MasterObject 特有部分丢失

    base.Move();  // 调用 BaseObject::Move()，不是多态！
    base.test();  // 调用 BaseObject::test()

    return 0;
}
```

**输出**：
```
BaseObject Moving
BaseObject::test()
```

### 对象切片的问题

| 问题 | 说明 |
|------|------|
| **数据丢失** | 派生类特有的成员变量被丢弃 |
| **多态失效** | 虚函数调用的是基类版本，而非派生类版本 |
| **意外行为** | 程序行为与预期不符 |

## 通过指针实现多态

要实现真正的多态，必须使用**指针**或**引用**：

```cpp
int main() {
    BaseObject* ptr = new MasterObject();

    ptr->Move();  // 动态绑定，调用实际类型的 Move()

    delete ptr;
    return 0;
}
```

## 多态调用示例

### 基类指针调用虚函数

```cpp
#include <iostream>

class BaseObject {
public:
    int x;

    // 虚函数
    virtual void Move() {
        std::cout << "BaseObject Moving" << std::endl;
    }

    virtual void test() {
        std::cout << "BaseObject::test()" << std::endl;
    }
};

class DFKObject : public BaseObject {
public:
    void Move() override {
        std::cout << "DFKObject Moving" << std::endl;
    }
};

class MasterObject : public BaseObject {
public:
    void test() override {
        std::cout << "MasterObject::test()" << std::endl;
    }
};

int main() {
    // 使用指针实现多态
    BaseObject* objs[3];
    objs[0] = new BaseObject();
    objs[1] = new DFKObject();
    objs[2] = new MasterObject();

    for (int i = 0; i < 3; ++i) {
        objs[i]->Move();  // 动态绑定
        objs[i]->test();  // 动态绑定
        std::cout << std::endl;
    }

    // 释放内存
    for (int i = 0; i < 3; ++i) {
        delete objs[i];
    }

    return 0;
}
```

**输出**：
```
BaseObject Moving
BaseObject::test()

DFKObject Moving
BaseObject::test()

BaseObject Moving
MasterObject::test()
```

## AutoMove 与多态调用链

### 多态函数中调用其他虚函数

当基类的非虚函数内部调用虚函数时，仍然会发生动态绑定：

```cpp
#include <iostream>

class BaseObject {
public:
    int x;

    virtual void Move() {
        std::cout << "BaseObject Moving" << std::endl;
    }

    // 非虚函数，但内部调用虚函数
    virtual void autoMove(int step = 2) {
        for (int i = 0; i < step; ++i) {
            Move();  // 动态绑定！
        }
    }
};

class DFKObject : public BaseObject {
public:
    void Move() override {
        std::cout << "DFKObject Moving" << std::endl;
    }
};

int main() {
    BaseObject* obj = new DFKObject();

    // 调用 autoMove，内部的 Move() 会动态绑定到 DFKObject::Move()
    obj->autoMove(3);

    delete obj;
    return 0;
}
```

**输出**：
```
DFKObject Moving
DFKObject Moving
DFKObject Moving
```

### 重写 autoMove

```cpp
class MasterObject : public BaseObject {
public:
    void autoMove(int step = 2) override {
        std::cout << "===auto move===" << std::endl;
        for (int i = 0; i < step; ++i) {
            Move();
        }
    }
};

int main() {
    BaseObject* obj = new MasterObject();
    obj->autoMove();  // 调用 MasterObject::autoMove()
    delete obj;
    return 0;
}
```

## static_cast 与多态

### 向下转型（Downcasting）

有时需要将基类指针转换为派生类指针，以访问派生类特有的成员：

```cpp
#include <iostream>

class BaseObject {
public:
    int x = 0;
    virtual void Move() {
        std::cout << "BaseObject Moving" << std::endl;
    }
};

class MasterObject : public BaseObject {
public:
    int y = 100;  // 派生类特有成员

    void Move() override {
        std::cout << "MasterObject Moving" << std::endl;
    }

    void specialFunc() {
        std::cout << "MasterObject special function, y = " << y << std::endl;
    }
};

int main() {
    BaseObject* base = new MasterObject();

    base->Move();  // 多态调用

    // 向下转型：基类指针 -> 派生类指针
    MasterObject* master = static_cast<MasterObject*>(base);
    master->specialFunc();  // 现在可以访问派生类特有成员

    delete base;
    return 0;
}
```

### static_cast 的风险

`static_cast` 在编译时不检查实际类型，如果类型不匹配会导致未定义行为：

```cpp
int main() {
    BaseObject* base = new BaseObject();  // 实际是基类对象

    // 危险！base 实际不是 MasterObject
    MasterObject* master = static_cast<MasterObject*>(base);
    master->specialFunc();  // 未定义行为！

    delete base;
    return 0;
}
```

### 安全的向下转型：dynamic_cast

`dynamic_cast` 在运行时检查类型，失败时返回 `nullptr`：

```cpp
int main() {
    BaseObject* base = new BaseObject();

    // 安全的向下转型
    MasterObject* master = dynamic_cast<MasterObject*>(base);
    if (master != nullptr) {
        master->specialFunc();
    } else {
        std::cout << "转型失败：base 不是 MasterObject" << std::endl;
    }

    delete base;
    return 0;
}
```

> **注意**：`dynamic_cast` 要求基类必须有虚函数（即必须是多态类型）。

## 引用与多态

引用同样可以实现多态：

```cpp
void printMove(BaseObject& obj) {
    obj.Move();  // 动态绑定
}

int main() {
    MasterObject master;
    DFKObject dfk;

    printMove(master);  // 输出 MasterObject Moving（如果重写了）
    printMove(dfk);     // 输出 DFKObject Moving

    return 0;
}
```

---

## 关键要点

1. **对象切片**：值语义会丢失派生类特有部分，导致多态失效
2. **指针/引用**：实现多态必须使用基类指针或引用
3. **虚函数链**：非虚函数内部调用虚函数仍然是动态绑定
4. **向下转型**：使用 `dynamic_cast` 更安全，`static_cast` 有风险
5. **多态条件**：继承 + 虚函数 + 指针/引用
