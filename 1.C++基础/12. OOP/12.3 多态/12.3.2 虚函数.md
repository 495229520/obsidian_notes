# 12.3.2 虚函数

## 虚函数的概念

**虚函数**（Virtual Function）是在基类中使用 `virtual` 关键字声明的成员函数。它允许派生类**重写**（override）基类的实现，从而实现[[12.3.1 静态多态与动态多态|动态多态]]。

当通过**基类指针或引用**调用虚函数时，实际调用的是**对象真正类型**对应的函数版本，而不是指针/引用声明类型的版本。

---

## 虚函数的声明

```cpp
class Base {
public:
    // 使用 virtual 关键字声明虚函数
    virtual void speak() {
        std::cout << "Base speaks." << std::endl;
    }
};

class Derived : public Base {
public:
    // 派生类重写虚函数（virtual 可省略，但建议使用 override）
    void speak() override {
        std::cout << "Derived speaks." << std::endl;
    }
};
```

---

## 虚函数表（vtable）与虚函数指针（vptr）

### 底层机制

C++ 通过**虚函数表**（Virtual Table，简称 vtable）实现动态多态。每个包含虚函数的类都有一个 vtable，存储了该类所有虚函数的地址。

### 虚函数指针（vptr）

每个包含虚函数的对象内部都有一个隐藏的指针，称为**虚函数指针**（vptr），指向其所属类的 vtable。

```
对象内存布局：
┌─────────────┐
│   vptr      │ ──→ vtable
├─────────────┤     ┌──────────────┐
│ 成员变量... │     │ &虚函数1     │
└─────────────┘     │ &虚函数2     │
                    │ ...          │
                    └──────────────┘
```

### 调用过程

```cpp
Base* ptr = new Derived();
ptr->speak();  // 实际调用 Derived::speak()
```

1. 编译器通过 `ptr` 找到对象
2. 从对象中读取 `vptr`
3. 通过 `vptr` 找到 vtable
4. 在 vtable 中查找 `speak` 函数的地址
5. 调用该地址处的函数

## 虚函数的使用规则

### 1. 重写条件

派生类重写基类虚函数必须满足：

| 条件 | 说明 |
|------|------|
| **函数名相同** | 必须与基类虚函数同名 |
| **参数列表相同** | 参数类型、个数、顺序完全一致 |
| **返回类型兼容** | 相同，或为协变返回类型（派生类指针/引用） |
| **const 属性相同** | 基类是 const，派生类也必须是 const |

### 2. 虚函数的继承

```cpp
class Base {
public:
    virtual void func() {
        std::cout << "Base::func()" << std::endl;
    }
};

class Derived : public Base {
public:
    // 即使不写 virtual，func 仍然是虚函数
    void func() override {
        std::cout << "Derived::func()" << std::endl;
    }
};

class GrandChild : public Derived {
public:
    // 继续重写，仍然是虚函数
    void func() override {
        std::cout << "GrandChild::func()" << std::endl;
    }
};
```

### 3. 构造函数不能是虚函数

构造函数在对象创建时调用，此时 vtable 还未完全建立，因此构造函数不能是虚函数。

### 4. 析构函数应该是虚函数

当通过基类指针删除派生类对象时，如果析构函数不是虚函数，只会调用基类的[[12.1.7 析构函数|析构函数]]，导致资源泄漏：

```cpp
class Base {
public:
    // 非虚析构函数 - 危险！
    ~Base() {
        std::cout << "Base destructor" << std::endl;
    }
};

class Derived : public Base {
    int* data;
public:
    Derived() { data = new int[100]; }
    ~Derived() {
        delete[] data;  // 不会被调用！
        std::cout << "Derived destructor" << std::endl;
    }
};

int main() {
    Base* ptr = new Derived();
    delete ptr;  // 只调用 Base::~Base()，内存泄漏！
}
```

**解决方案**：将基类析构函数声明为虚函数：

```cpp
class Base {
public:
    virtual ~Base() {
        std::cout << "Base destructor" << std::endl;
    }
};
```

> **规则**：如果一个类有虚函数，它的析构函数也应该是虚函数。

## override 关键字（C++11）

`override` 关键字明确表示函数是对基类虚函数的重写。如果签名不匹配，编译器会报错：

```cpp
class Base {
public:
    virtual void func(int x) {}
};

class Derived : public Base {
public:
    // 编译错误：没有找到可重写的基类虚函数
    void func(double x) override {}  // 参数类型不匹配！

    // 正确
    void func(int x) override {}
};
```

### override 的好处

| 好处 | 说明 |
|------|------|
| **防止拼写错误** | 函数名写错时编译器报错 |
| **防止签名错误** | 参数类型不匹配时编译器报错 |
| **提高可读性** | 明确表明重写意图 |

---

## final 关键字（C++11）

### 1. 禁止重写虚函数

```cpp
class Base {
public:
    virtual void func() final {  // 禁止派生类重写
        std::cout << "Base::func()" << std::endl;
    }
};

class Derived : public Base {
public:
    // 编译错误：不能重写 final 函数
    void func() override {}
};
```

### 2. 禁止继承类

```cpp
class FinalClass final {  // 禁止被继承
    // ...
};

// 编译错误：不能继承 final 类
class Derived : public FinalClass {};
```

## 虚函数的性能开销

| 开销来源 | 说明 |
|---------|------|
| **内存开销** | 每个对象增加一个 vptr（通常 4 或 8 字节） |
| **调用开销** | 需要通过 vtable 间接调用，无法内联 |
| **缓存影响** | vtable 查找可能导致缓存未命中 |

---

## 普通函数 vs 虚函数 vs 纯虚函数

| 特性 | 普通函数 | 虚函数 (Virtual Function) | 纯虚函数 (Pure Virtual Function) |
|------|---------|--------------------------|--------------------------------|
| **声明方式** | 普通声明 | `virtual` 关键字 | `virtual ... = 0` |
| **是否有实现** | 必须有 | 可以有 | 可以有（但通常没有） |
| **是否可重写** | 隐藏基类版本 | 可以重写 | **必须**重写（除非派生类也是抽象类） |
| **调用机制** | 编译时绑定 | 运行时绑定 | 运行时绑定 |
| **类的实例化** | 可以 | 可以 | 使类成为[[12.3.4 抽象类\|抽象类]]，不能实例化 |

## 代码示例：综合演示

```cpp
#include <iostream>
#include <string>

class Shape {
public:
    // 普通虚函数 - 提供默认实现
    virtual void draw() {
        std::cout << "Drawing a generic shape." << std::endl;
    }

    // 纯虚函数 - 派生类必须实现
    virtual double getArea() = 0;

    // 虚析构函数 - 多态删除时正确调用派生类析构
    virtual ~Shape() {
        std::cout << "Shape destructor called." << std::endl;
    }
};

class Circle : public Shape {
    double radius;
public:
    Circle(double r) : radius(r) {}

    void draw() override {
        std::cout << "Drawing a circle with radius " << radius << std::endl;
    }

    double getArea() override {
        return 3.14159 * radius * radius;
    }

    ~Circle() override {
        std::cout << "Circle destructor called." << std::endl;
    }
};

class Rectangle : public Shape {
    double width, height;
public:
    Rectangle(double w, double h) : width(w), height(h) {}

    // 不重写 draw()，使用基类默认实现

    double getArea() override {
        return width * height;
    }

    ~Rectangle() override {
        std::cout << "Rectangle destructor called." << std::endl;
    }
};

int main() {
    Shape* shapes[2];
    shapes[0] = new Circle(5.0);
    shapes[1] = new Rectangle(4.0, 6.0);

    for (int i = 0; i < 2; ++i) {
        shapes[i]->draw();
        std::cout << "Area: " << shapes[i]->getArea() << std::endl;
        std::cout << std::endl;
    }

    // 正确释放内存（虚析构函数确保派生类析构被调用）
    for (int i = 0; i < 2; ++i) {
        delete shapes[i];
    }

    return 0;
}
```

---

## 关键要点

1. **虚函数**：使用 `virtual` 声明，允许运行时多态
2. **vtable/vptr**：C++ 实现虚函数的底层机制
3. **override**：明确重写意图，提高代码安全性
4. **final**：禁止进一步重写或继承
5. **虚析构函数**：多态基类必须使用虚析构函数
6. **性能**：虚函数有额外开销，但通常可以忽略
