# 12.3.5 接口类

## 接口类的定义

在 C++ 中，**接口类**通常是指一种特殊的[[12.3.4 抽象类|抽象类]]，它的**所有函数都是纯虚函数**，并且**没有数据成员**（除了可能有的虚函数表指针），它完全只定义行为规范，而不提供任何实现。

---

## 核心特征

| 特征 | 说明 |
|------|------|
| **所有函数都是纯虚函数** | 接口类中的所有成员函数都声明为纯虚函数 |
| **没有数据成员** | 接口类通常不包含非静态数据成员，但这不被视为它"拥有"数据 |
| **纯粹定义行为** | 它只关注"能做什么"，不关注"如何做"或"有什么状态" |
| **多重继承** | 接口类尤其适用于多重[[12.2.1 类的继承|继承]]的场景，允许一个类实现多个不相关的功能契约，从而模拟 Java 或 C# 中的接口概念 |

---

## 接口类的声明

```cpp
// 接口类 IPrintable
class IPrintable {
public:
    // 纯虚函数：打印信息，强制派生类实现
    virtual void print() const = 0;

    // 虚析构函数是接口的常见实践，确保多态删除时的正确性
    virtual ~IPrintable() {
        std::cout << "IPrintable destructor called." << std::endl;
    }
};

// 接口类 IClickable
class IClickable {
public:
    // 纯虚函数：处理点击事件
    virtual void onClick() = 0;

    virtual ~IClickable() {
        std::cout << "IClickable destructor called." << std::endl;
    }
};
```

## 实现接口

### 实现单个接口

```cpp
#include <iostream>
#include <string>

// 接口类
class IPrintable {
public:
    virtual void print() const = 0;
    virtual ~IPrintable() = default;
};

// 类 Document 只实现了 IPrintable 接口
class Document : public IPrintable {
public:
    Document(const std::string& title) : docTitle(title) {}

    void print() const override {
        std::cout << "Document Title: " << docTitle << std::endl;
    }

    ~Document() override {
        std::cout << "Document destructor called." << std::endl;
    }

private:
    std::string docTitle;
};

int main() {
    Document myDoc("Report 2024");
    myDoc.print();

    // 通过接口指针使用
    IPrintable* printable = &myDoc;
    printable->print();

    return 0;
}
```

### 实现多个接口（多重继承）

一个类可以同时实现多个接口，这是接口类最强大的特性：

```cpp
#include <iostream>
#include <string>

// 接口类 IPrintable
class IPrintable {
public:
    virtual void print() const = 0;
    virtual ~IPrintable() = default;
};

// 接口类 IClickable
class IClickable {
public:
    virtual void onClick() = 0;
    virtual ~IClickable() = default;
};

// 类 Button 同时继承并实现了 IPrintable 和 IClickable 接口
class Button : public IPrintable, public IClickable {
public:
    Button(const std::string& label) : buttonLabel(label) {}

    void print() const override {
        std::cout << "Button Label: " << buttonLabel << std::endl;
    }

    void onClick() override {
        std::cout << "Button '" << buttonLabel << "' was clicked!" << std::endl;
    }

    ~Button() override {
        std::cout << "Button destructor called." << std::endl;
    }

private:
    std::string buttonLabel;
};

// 类 Document 只实现了 IPrintable 接口
class Document : public IPrintable {
public:
    Document(const std::string& title) : docTitle(title) {}

    void print() const override {
        std::cout << "Document Title: " << docTitle << std::endl;
    }

    ~Document() override {
        std::cout << "Document destructor called." << std::endl;
    }

private:
    std::string docTitle;
};

int main() {
    Button myButton("Submit");
    Document myDoc("Report 2024");

    // 使用 IPrintable 接口
    IPrintable* printableItems[2];
    printableItems[0] = &myButton;
    printableItems[1] = &myDoc;

    std::cout << "--- Printing Items ---" << std::endl;
    for (int i = 0; i < 2; ++i) {
        printableItems[i]->print();
    }

    // 使用 IClickable 接口
    IClickable* clickableItem = &myButton;
    std::cout << "\n--- Clicking Button ---" << std::endl;
    clickableItem->onClick();

    // 释放内存（如果是在堆上创建的）
    // delete new Button("Temp");  // 示例

    return 0;
}
```

**输出**：
```
--- Printing Items ---
Button Label: Submit
Document Title: Report 2024

--- Clicking Button ---
Button 'Submit' was clicked!
```

## 接口类 vs 抽象类

| 特性 | 接口类 | 抽象类 |
|------|--------|--------|
| **纯虚函数** | 所有成员函数都是纯虚函数 | 至少有一个纯虚函数 |
| **数据成员** | 通常没有 | 可以有 |
| **函数实现** | 不提供任何实现 | 可以提供部分实现 |
| **设计目的** | 纯粹定义契约/行为 | 提供部分共享实现 |
| **多重继承** | 非常适合 | 可能导致复杂性 |

---

## 接口命名约定

在 C++ 中，接口类通常使用以下命名约定：

| 约定 | 示例 |
|------|------|
| **I 前缀** | `IPrintable`、`IClickable`、`ISerializable` |
| **able 后缀** | `Printable`、`Clickable`、`Comparable` |

## 使用场景

### 1. 定义功能契约

```cpp
// 所有可序列化的对象必须实现此接口
class ISerializable {
public:
    virtual std::string serialize() const = 0;
    virtual void deserialize(const std::string& data) = 0;
    virtual ~ISerializable() = default;
};
```

### 2. 实现回调机制

```cpp
// 事件监听器接口
class IEventListener {
public:
    virtual void onEvent(int eventId) = 0;
    virtual ~IEventListener() = default;
};

class EventManager {
public:
    void addListener(IEventListener* listener) {
        listeners.push_back(listener);
    }

    void fireEvent(int eventId) {
        for (auto* listener : listeners) {
            listener->onEvent(eventId);
        }
    }

private:
    std::vector<IEventListener*> listeners;
};
```

### 3. 依赖注入

```cpp
// 日志接口
class ILogger {
public:
    virtual void log(const std::string& message) = 0;
    virtual ~ILogger() = default;
};

// 具体实现
class ConsoleLogger : public ILogger {
public:
    void log(const std::string& message) override {
        std::cout << "[Console] " << message << std::endl;
    }
};

class FileLogger : public ILogger {
public:
    void log(const std::string& message) override {
        // 写入文件...
    }
};

// 使用接口，不依赖具体实现
class Application {
    ILogger* logger;
public:
    Application(ILogger* l) : logger(l) {}
    void run() {
        logger->log("Application started");
    }
};
```

---

## 关键要点

1. **接口类**：所有函数都是纯虚函数，没有数据成员
2. **纯粹契约**：只定义"能做什么"，不关心"如何做"
3. **多重继承**：接口类非常适合多重继承场景
4. **虚析构函数**：接口类应该有虚析构函数
5. **设计模式**：接口是"依赖倒置原则"和"接口隔离原则"的基础
