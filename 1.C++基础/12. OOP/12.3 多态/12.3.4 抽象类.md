# 12.3.4 抽象类

## 抽象类的定义

**抽象类**是不能直接实例化的类，它的主要目的是作为**基类**，为派生类提供一个共同的接口（和可选的）一些共享的实现。

### 核心特征

- **包含纯虚函数**：抽象类至少包含一个[[12.3.2 虚函数|纯虚函数]]
- **不能直接实例化**：因为纯虚函数没有实现，编译器不允许直接创建抽象类的对象。你只能通过派生类来创建对象
- **提供接口和部分实现**：抽象类可以包含：
  - **纯虚函数**：强制派生类实现这些函数，从而定义了一个接口
  - **普通虚函数**：提供了默认的实现，派生类可以选择重写或直接使用
  - **非虚函数**：提供所有派生类共享的固定行为
  - **数据成员**：存储所有派生类共享的状态
- **继承**：派生类如果要实例化，**必须实现**所有继承来的纯虚函数，否则派生类本身也会成为一个抽象类

## 纯虚函数

### 声明语法

纯虚函数是在基类中声明但没有实现的虚函数，它的声明格式为 `virtual 返回类型 函数名(参数) = 0;`

```cpp
class Shape {
public:
    // 纯虚函数：计算面积，强制派生类实现
    virtual double getArea() = 0;
};
```

### 纯虚函数的特点

| 特点 | 说明 |
|------|------|
| **使类变为抽象类** | 包含纯虚函数的类不能实例化 |
| **强制派生类实现** | 派生类必须重写纯虚函数才能实例化 |
| **可以有实现** | 纯虚函数可以在类外提供实现（但很少使用） |

### 纯虚函数的实现（少见情况）

```cpp
class Base {
public:
    virtual void func() = 0;  // 纯虚函数
};

// 纯虚函数也可以有实现（在类外）
void Base::func() {
    std::cout << "Base::func() default implementation" << std::endl;
}

class Derived : public Base {
public:
    void func() override {
        Base::func();  // 可以调用基类的实现
        std::cout << "Derived::func()" << std::endl;
    }
};
```

## 代码示例：Shape 抽象类

```cpp
#include <iostream>
#include <string>

// 抽象类 Shape
class Shape {
public:
    // 纯虚函数：计算面积，强制派生类实现
    virtual double getArea() = 0;

    // 普通虚函数：绘制，提供默认实现，派生类可重写
    virtual void draw() {
        std::cout << "Drawing a generic shape." << std::endl;
    }

    // 非虚函数：颜色，所有派生类共享
    void setColor(std::string c) {
        color = c;
    }

    // 虚析构函数：重要！确保多态删除时的正确析构
    virtual ~Shape() {
        std::cout << "Shape destructor called." << std::endl;
    }

protected:
    std::string color;
};

// 派生类 Circle
class Circle : public Shape {
public:
    Circle(double r) : radius(r) {}

    // 实现纯虚函数 getArea
    double getArea() override {
        return 3.14159 * radius * radius;
    }

    // 重写虚函数 draw
    void draw() override {
        std::cout << "Drawing a circle with radius " << radius << std::endl;
    }

    ~Circle() override {
        std::cout << "Circle destructor called." << std::endl;
    }

private:
    double radius;
};

// 派生类 Rectangle
class Rectangle : public Shape {
public:
    Rectangle(double w, double h) : width(w), height(h) {}

    // 实现纯虚函数 getArea
    double getArea() override {
        return width * height;
    }

    ~Rectangle() override {
        std::cout << "Rectangle destructor called." << std::endl;
    }

private:
    double width, height;
};

int main() {
    // Shape s;  // 错误：不能实例化抽象类

    Circle c(5.0);
    c.setColor("Red");
    std::cout << "Circle area: " << c.getArea() << std::endl;
    c.draw();

    Rectangle r(4.0, 6.0);
    r.setColor("Blue");
    std::cout << "Rectangle area: " << r.getArea() << std::endl;
    r.draw();  // 使用基类默认的 draw()，因为 Rectangle 没有重写它

    // 多态使用
    Shape* shapes[2];
    shapes[0] = &c;
    shapes[1] = &r;

    std::cout << "\nUsing polymorphic pointers:" << std::endl;
    for (int i = 0; i < 2; ++i) {
        std::cout << "Area: " << shapes[i]->getArea() << std::endl;
        shapes[i]->draw();
    }

    return 0;
}
```

## 抽象类的继承规则

### 派生类必须实现所有纯虚函数

```cpp
class Shape {
public:
    virtual double getArea() = 0;
    virtual void draw() = 0;
};

// 只实现了 getArea，没有实现 draw
// PartialShape 仍然是抽象类！
class PartialShape : public Shape {
public:
    double getArea() override {
        return 0.0;
    }
    // draw() 没有实现
};

// PartialShape p;  // 错误：PartialShape 是抽象类

// 完整实现所有纯虚函数
class ConcreteShape : public PartialShape {
public:
    void draw() override {
        std::cout << "Drawing ConcreteShape" << std::endl;
    }
};

// ConcreteShape cs;  // 正确：可以实例化
```

### 抽象类可以有构造函数

虽然抽象类不能直接实例化，但它可以有构造函数，用于初始化基类部分：

```cpp
class Animal {
protected:
    std::string name;
    int age;

public:
    // 抽象类的构造函数
    Animal(const std::string& n, int a) : name(n), age(a) {}

    virtual void speak() = 0;  // 纯虚函数

    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    // 调用基类构造函数
    Dog(const std::string& n, int a) : Animal(n, a) {}

    void speak() override {
        std::cout << name << " says: Woof!" << std::endl;
    }
};
```

## 抽象类 vs 具体类

| 特性 | 抽象类 | 具体类 |
|------|--------|--------|
| **能否实例化** | 不能 | 可以 |
| **纯虚函数** | 至少有一个 | 没有 |
| **作用** | 定义接口，作为基类 | 提供完整实现 |
| **派生类要求** | 必须实现所有纯虚函数 | 无特殊要求 |

---

## 使用抽象类的好处

| 好处 | 说明 |
|------|------|
| **定义接口** | 强制派生类实现特定函数 |
| **代码复用** | 共享代码可以放在抽象基类中 |
| **多态支持** | 通过基类指针/引用操作不同派生类 |
| **设计清晰** | 明确表达"这个类不应该被直接使用" |

---

## 关键要点

1. **抽象类**：包含纯虚函数的类，不能直接实例化
2. **纯虚函数**：`virtual ... = 0`，强制派生类实现
3. **派生类**：必须实现所有纯虚函数才能实例化
4. **虚析构函数**：抽象类应该有虚析构函数
5. **设计模式**：抽象类是"面向接口编程"的基础
