# 12.12 malloc 与 new

## 简介

`malloc`/`free` 是 C 语言的内存分配函数，而 `new`/`delete` 是 C++ 的运算符。两者都用于在[[6.6 堆与栈的区别|堆]]上分配内存，但有重要区别。

---

## 基本对比

| 特性 | malloc/free | new/delete |
|------|-------------|------------|
| **语言** | C 语言函数 | C++ 运算符 |
| **返回类型** | `void*`（需要强制转换） | 直接返回正确类型的指针 |
| **大小计算** | 需要手动计算 `sizeof` | 自动计算 |
| **构造函数** | **不调用** | **调用** |
| **析构函数** | **不调用** | **调用** |
| **失败处理** | 返回 `NULL` | 抛出 `std::bad_alloc` 异常 |
| **可重载** | 不可以 | 可以 |

---

## 使用方式对比

### malloc/free

```cpp
#include <cstdlib>

// 分配单个 int
int* p1 = (int*)malloc(sizeof(int));
if (p1 == NULL) {
    // 处理分配失败
}
*p1 = 10;
free(p1);

// 分配 int 数组
int* arr = (int*)malloc(sizeof(int) * 10);
free(arr);
```

### new/delete

```cpp
// 分配单个 int
int* p1 = new int;      // 不初始化
int* p2 = new int(10);  // 初始化为 10
int* p3 = new int{10};  // C++11 列表初始化
delete p1;
delete p2;
delete p3;

// 分配 int 数组
int* arr = new int[10];
int* arr2 = new int[10]();  // 零初始化
delete[] arr;
delete[] arr2;
```

## 关键区别：构造函数和析构函数

这是 `malloc`/`free` 和 `new`/`delete` 最重要的区别。

```cpp
#include <iostream>
#include <cstdlib>

class MyClass {
public:
    int value;

    MyClass() : value(42) {
        std::cout << "Constructor called, value = " << value << std::endl;
    }

    ~MyClass() {
        std::cout << "Destructor called" << std::endl;
    }
};

int main() {
    std::cout << "=== Using malloc/free ===" << std::endl;
    MyClass* p1 = (MyClass*)malloc(sizeof(MyClass));
    // 构造函数不会被调用！
    std::cout << "p1->value = " << p1->value << std::endl;  // 未定义值
    free(p1);
    // 析构函数不会被调用！

    std::cout << "\n=== Using new/delete ===" << std::endl;
    MyClass* p2 = new MyClass();
    // 输出：Constructor called, value = 42
    std::cout << "p2->value = " << p2->value << std::endl;  // 42
    delete p2;
    // 输出：Destructor called

    return 0;
}
```

**输出**：
```
=== Using malloc/free ===
p1->value = <垃圾值>

=== Using new/delete ===
Constructor called, value = 42
p2->value = 42
Destructor called
```

## 为什么 malloc 不调用构造函数？

`malloc` 是 C 语言的函数，它只负责**分配内存**，不了解 C++ 的对象模型。它只是返回一块原始的、未初始化的内存。

```cpp
// malloc 的行为等价于：
void* malloc(size_t size) {
    // 1. 向操作系统请求 size 字节的内存
    // 2. 返回内存地址（或 NULL）
    // 就这样，没有其他操作
}
```

而 `new` 运算符做了更多工作：

```cpp
// new 的行为等价于：
T* new_operator() {
    // 1. 调用 operator new 分配内存
    void* memory = operator new(sizeof(T));

    // 2. 在分配的内存上调用构造函数
    T* object = new (memory) T();  // placement new

    // 3. 返回对象指针
    return object;
}
```

---

## placement new

如果必须使用 `malloc` 分配内存但又需要调用构造函数，可以使用 **[[12.1.8 对象放置构造|placement new]]**：

```cpp
#include <new>  // 需要包含这个头文件

MyClass* p = (MyClass*)malloc(sizeof(MyClass));
new (p) MyClass();  // 在 p 指向的内存上调用构造函数

// 使用完毕后，需要手动调用析构函数
p->~MyClass();
free(p);
```

## 数组的分配与释放

| 分配方式 | 释放方式 |
|----------|----------|
| `malloc` | `free` |
| `new` | `delete` |
| `new[]` | `delete[]` |

**重要**：`new[]` 必须用 `delete[]` 释放，否则只有第一个元素的析构函数被调用。

```cpp
class Item {
public:
    Item() { std::cout << "Item constructed" << std::endl; }
    ~Item() { std::cout << "Item destroyed" << std::endl; }
};

int main() {
    Item* arr = new Item[3];
    // 输出 3 次 "Item constructed"

    delete[] arr;  // 正确：输出 3 次 "Item destroyed"
    // delete arr;  // 错误！只调用一次析构函数，内存泄漏
}
```

---

## 何时使用 malloc？

在 C++ 中，**几乎总是应该使用 `new`/`delete`**。但以下情况可能使用 `malloc`：

1. **与 C 代码交互**：调用 C 库函数时
2. **自定义内存分配器**：实现自己的 `operator new`
3. **需要 `realloc`**：`new` 没有对应的重新分配功能

---

## 推荐做法

在现代 C++ 中，推荐使用**智能指针**而不是裸指针：

```cpp
#include <memory>

// 在栈上分配类
MyQueue Que;  // 正确：在栈上直接创建一个 MyQueue 对象

// 在堆上分配类
// 如果使用裸指针
// 正确，但需要手动 delete
MyQueue* Que = new MyQueue();

// 如果使用智能指针（更推荐）
std::unique_ptr<MyQueue> Que = std::make_unique<MyQueue>();
```

---

## 关键要点

1. **new 调用构造函数，malloc 不调用**
2. **delete 调用析构函数，free 不调用**
3. **不要混用**：`malloc` 配 `free`，`new` 配 `delete`，`new[]` 配 `delete[]`
4. **优先使用 new**：在 C++ 中几乎总是使用 `new`/`delete`
5. **更推荐智能指针**：`std::unique_ptr` 和 `std::shared_ptr`
