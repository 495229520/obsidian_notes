# 12.13 类的底层

## 内存布局

### 数据成员存储

类对象在内存中的布局遵循以下规则：

- 非静态数据成员按声明顺序存储在对象内部
- 编译器可能会插入**填充字节（Padding）** 以满足对齐要求
- 虚函数表指针（vptr）通常放在对象的开头或结尾

```cpp
class MyClass {
    int a;      // 4 字节
    char b;     // 1 字节
    int c;      // 4 字节
};
```

**内存布局**（假设 4 字节对齐）：

```
偏移量    成员      大小
0         a        4 字节
4         b        1 字节
5-7       填充     3 字节
8         c        4 字节
总计               12 字节
```

## 成员函数的调用机制

### this 指针传递

当调用成员函数时，编译器会隐式地将对象地址作为第一个参数传递，这就是 `this` 指针。

```cpp
class Role {
    int hp;
public:
    void SetHp(int value) {
        hp = value;
    }
};

Role r;
r.SetHp(100);
// 编译器转换为类似：SetHp(&r, 100);
```

### 不同调用约定

在 x86 架构上，成员函数通常使用 **thiscall** 调用约定：

| 调用约定 | this 指针位置 | 参数清理 |
|----------|---------------|----------|
| thiscall (MSVC) | ECX 寄存器 | 被调用者清理 |
| thiscall (GCC) | 栈上第一个参数 | 调用者清理 |

## 虚函数表（vtable）

当类包含虚函数时，编译器会为类生成一个**虚函数表（vtable）**，并在每个对象中添加一个**虚函数指针（vptr）**。

```cpp
class Base {
public:
    virtual void func1() { }
    virtual void func2() { }
};

class Derived : public Base {
public:
    void func1() override { }  // 重写
    virtual void func3() { }   // 新增
};
```

**内存布局**：

```
Base 对象：
┌─────────────┐
│ vptr ──────────────────────► Base vtable
├─────────────┤                 ┌─────────────┐
│ 其他成员    │                 │ &Base::func1│
└─────────────┘                 ├─────────────┤
                                │ &Base::func2│
                                └─────────────┘

Derived 对象：
┌─────────────┐
│ vptr ──────────────────────► Derived vtable
├─────────────┤                 ┌───────────────┐
│ 其他成员    │                 │ &Derived::func1│ (重写)
└─────────────┘                 ├───────────────┤
                                │ &Base::func2   │ (继承)
                                ├───────────────┤
                                │ &Derived::func3│ (新增)
                                └───────────────┘
```

### 虚函数调用过程

```cpp
Base* p = new Derived();
p->func1();

// 实际执行：
// 1. 从 p 指向的对象读取 vptr
// 2. 在 vtable 中查找 func1 的地址（索引 0）
// 3. 调用该地址的函数
```

## 对象构造过程

### 构造顺序

1. **分配内存**：为对象分配所需的内存空间
2. **基类构造**：调用基类的构造函数（如果有继承）
3. **成员初始化**：按声明顺序初始化成员变量
4. **vptr 设置**：设置虚函数表指针（如果有虚函数）
5. **构造函数体**：执行构造函数体内的代码

```cpp
class Base {
public:
    Base() {
        std::cout << "Base constructor" << std::endl;
    }
};

class Derived : public Base {
    int data;
public:
    Derived() : data(0) {
        std::cout << "Derived constructor" << std::endl;
    }
};

// 创建 Derived 对象时：
// 1. 分配内存
// 2. 调用 Base()
// 3. 初始化 data
// 4. 设置 vptr 指向 Derived 的 vtable
// 5. 执行 Derived() 函数体
```

### 析构顺序（与构造相反）

1. 执行析构函数体
2. 销毁成员对象（按声明逆序）
3. 调用基类析构函数

## 继承的内存布局

### 单继承

```cpp
class Base {
    int a;
};

class Derived : public Base {
    int b;
};
```

```
Derived 对象内存布局：
┌─────────────┐
│ Base::a     │  ← Base 部分
├─────────────┤
│ Derived::b  │  ← Derived 新增
└─────────────┘
```

### 多重继承

```cpp
class A { int a; };
class B { int b; };
class C : public A, public B { int c; };
```

```
C 对象内存布局：
┌─────────────┐
│ A::a        │  ← A 部分
├─────────────┤
│ B::b        │  ← B 部分
├─────────────┤
│ C::c        │  ← C 新增
└─────────────┘
```

## 空类的大小

即使类没有任何成员，其大小也不为 0：

```cpp
class Empty {};
std::cout << sizeof(Empty);  // 输出 1
```

**原因**：C++ 标准要求每个对象必须有唯一的地址。如果空类大小为 0，两个空类对象可能有相同的地址。

## 关键要点

1. **内存对齐**：编译器会插入填充字节以优化访问效率
2. **this 指针**：成员函数调用时隐式传递对象地址
3. **vtable**：虚函数通过虚函数表实现动态绑定
4. **vptr 开销**：每个含虚函数的对象额外占用一个指针大小
5. **构造/析构顺序**：基类先构造后析构，成员按声明顺序构造
