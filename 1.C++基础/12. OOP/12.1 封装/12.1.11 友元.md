# 12.10 友元

## 简介

在 C++ 中，**友元（Friend）** 是一种机制，允许一个函数或另一个类访问当前类的 `private` 或 `protected` 成员。友元打破了封装的界限，但在某些场景下非常有用。

---

## 友元的特性

| 特性 | 说明 |
|------|------|
| **访问私有成员** | 友元可以访问类的私有和保护成员 |
| **不是成员** | 友元函数不是类的成员函数，友元类不是成员类 |
| **单向性** | 友元关系是单向的，不具有传递性 |
| **破坏封装** | 应谨慎使用，过度使用会破坏封装性 |

---

## 友元函数

**友元函数**是在类中用 `friend` 关键字声明的非成员函数，它可以访问该类的私有成员。

```cpp
class Role {
private:
    int money = 0;
    int diamond = 0;

public:
    // 声明友元函数
    friend void cheat(Role& r);

    void show() {
        std::cout << "money: " << money << ", diamond: " << diamond << std::endl;
    }
};

// 友元函数定义（不是成员函数，不需要 Role::）
void cheat(Role& r) {
    r.money = 999999;      // 可以访问私有成员
    r.diamond = 999999;
}

int main() {
    Role player;
    player.show();         // money: 0, diamond: 0
    cheat(player);         // 调用友元函数
    player.show();         // money: 999999, diamond: 999999
}
```

---

## 友元类

**友元类**是被声明为友元的另一个类，该类的所有成员函数都可以访问当前类的私有成员。

```cpp
class Secret {
private:
    int secretData = 42;

    // 声明 Accessor 为友元类
    friend class Accessor;
};

class Accessor {
public:
    void reveal(Secret& s) {
        // 可以访问 Secret 的私有成员
        std::cout << "Secret data: " << s.secretData << std::endl;
    }

    void modify(Secret& s, int newValue) {
        s.secretData = newValue;
    }
};

int main() {
    Secret s;
    Accessor a;
    a.reveal(s);       // Secret data: 42
    a.modify(s, 100);
    a.reveal(s);       // Secret data: 100
}
```

---

## 友元成员函数

可以只将另一个类的特定成员函数声明为友元，而不是整个类：

```cpp
class Secret;  // 前向声明

class Accessor {
public:
    void reveal(Secret& s);  // 声明，定义在后面
};

class Secret {
private:
    int secretData = 42;

    // 只有 Accessor::reveal 是友元
    friend void Accessor::reveal(Secret& s);
};

// 定义必须在 Secret 完整定义之后
void Accessor::reveal(Secret& s) {
    std::cout << "Secret data: " << s.secretData << std::endl;
}
```

## 友元关系的特点

### 单向性

友元关系是**单向的**。如果 A 是 B 的友元，B 不会自动成为 A 的友元。

```cpp
class B;

class A {
    friend class B;  // B 是 A 的友元
private:
    int dataA = 1;
};

class B {
    // 没有声明 A 为友元
private:
    int dataB = 2;

public:
    void accessA(A& a) {
        std::cout << a.dataA;  // OK：B 可以访问 A 的私有成员
    }
};

// A 不能访问 B 的私有成员（除非 B 也声明 A 为友元）
```

### 不具有传递性

如果 A 是 B 的友元，B 是 C 的友元，A **不会**自动成为 C 的友元。

### 不能继承

友元关系**不能被继承**。如果基类有一个友元，派生类不会自动拥有这个友元。

---

## 使用场景

1. **运算符重载**：某些运算符（如 `<<`、`>>`）通常需要声明为友元
2. **紧密协作的类**：两个类需要紧密协作，频繁访问彼此的私有成员
3. **测试类**：测试类需要访问被测类的私有成员
4. **工厂类**：工厂类需要访问产品类的私有构造函数

```cpp
class MyClass {
private:
    int value;

public:
    MyClass(int v) : value(v) {}

    // 声明 << 运算符为友元
    friend std::ostream& operator<<(std::ostream& os, const MyClass& obj);
};

// 友元函数实现
std::ostream& operator<<(std::ostream& os, const MyClass& obj) {
    os << "MyClass(" << obj.value << ")";
    return os;
}

int main() {
    MyClass m(42);
    std::cout << m << std::endl;  // MyClass(42)
}
```

---

## 注意事项

1. **谨慎使用**：友元破坏了封装性，应该仅在必要时使用
2. **优先使用公有接口**：如果可以通过公有方法实现，就不要使用友元
3. **友元声明位置**：友元声明可以放在类的任何位置（public/private/protected 区域都可以）

---

## 关键要点

1. **friend 关键字**：用于声明友元函数或友元类
2. **访问私有成员**：友元可以访问类的私有和保护成员
3. **单向、不传递、不继承**：友元关系的三个特点
4. **破坏封装**：应谨慎使用，避免过度依赖友元
