# 12.7 析构函数

## 简介

**析构函数**是一种特殊的成员函数，在对象生命周期结束时被**自动调用**。它的主要职责是**清理资源**（如释放动态分配的内存、关闭文件句柄、释放网络连接等）。

---

## 析构函数的特性

| 特性 | 说明 |
|------|------|
| **名称** | 波浪号 `~` 加类名，如 `~ClassName()` |
| **无参数** | 析构函数不接受任何参数 |
| **无返回类型** | 连 `void` 也没有 |
| **不能重载** | 每个类只能有一个析构函数 |
| **自动调用** | 对象销毁时自动调用，不需要手动调用 |

---

## 析构函数的调用时机

| 场景 | 说明 |
|------|------|
| **栈上对象离开作用域** | 局部对象在其所在的代码块（如函数、`if` 语句块、循环体等）结束时被销毁 |
| **delete 堆上对象** | 使用 `new` 创建的对象，在调用 `delete` 时析构函数被调用 |
| **程序结束** | 全局对象和静态对象在程序结束时被销毁 |

---

## 示例

```cpp
#include <iostream>

class Resource {
private:
    int* data;
    std::string name;

public:
    // 构造函数：分配资源
    Resource(const std::string& n) : name(n) {
        data = new int[100];
        std::cout << "Resource " << name << " constructed" << std::endl;
    }

    // 析构函数：释放资源
    ~Resource() {
        delete[] data;
        std::cout << "Resource " << name << " destroyed" << std::endl;
    }
};

int main() {
    Resource r1("Object 1");  // 构造 Object 1

    {
        Resource r2("Object 2");  // 构造 Object 2
        Resource r3("Object 3");  // 构造 Object 3
    }  // r3 和 r2 按相反顺序析构（先 r3，后 r2）

    Resource* r4 = new Resource("Object 4");  // 构造 Object 4
    delete r4;  // 析构 Object 4

    return 0;
}  // r1 析构
```

**输出**：
```
Resource Object 1 constructed
Resource Object 2 constructed
Resource Object 3 constructed
Resource Object 3 destroyed
Resource Object 2 destroyed
Resource Object 4 constructed
Resource Object 4 destroyed
Resource Object 1 destroyed
```

## 析构顺序

**重要规则**：析构顺序与构造顺序**相反**。

### 局部对象

```cpp
void func() {
    MyClass a;  // 先构造
    MyClass b;  // 后构造
}  // 先析构 b，再析构 a
```

### 成员对象

```cpp
class Outer {
    Inner member1;  // 先构造
    Inner member2;  // 后构造
public:
    ~Outer() {
        // 析构函数体执行完后
        // 先析构 member2，再析构 member1
    }
};
```

### [[12.2.3 继承析构函数|继承关系]]

```cpp
class Derived : public Base {
    // 构造：先 Base，后 Derived
    // 析构：先 Derived，后 Base
};
```

## 默认析构函数

如果你不定义析构函数，编译器会生成一个**默认析构函数**。

- 默认析构函数会自动调用：
  - 所有非静态成员对象的析构函数
  - 基类的析构函数
- 默认析构函数**不会**释放你用 `new` 分配的内存

```cpp
class NeedsDestructor {
    int* ptr;
public:
    NeedsDestructor() {
        ptr = new int[100];  // 分配内存
    }
    // 如果没有析构函数，ptr 指向的内存会泄漏！
    ~NeedsDestructor() {
        delete[] ptr;  // 必须手动释放
    }
};
```

## 虚析构函数

当使用基类指针指向派生类对象时，如果基类的析构函数不是虚函数，删除时只会调用基类的析构函数，导致派生类资源无法释放。

```cpp
class Base {
public:
    ~Base() {  // 非虚析构函数
        std::cout << "Base destructor" << std::endl;
    }
};

class Derived : public Base {
    int* data;
public:
    Derived() { data = new int[100]; }
    ~Derived() {
        delete[] data;
        std::cout << "Derived destructor" << std::endl;
    }
};

int main() {
    Base* ptr = new Derived();
    delete ptr;  // 只调用 Base::~Base()，内存泄漏！
}
```

**解决方案**：将基类析构函数声明为 `virtual`：

```cpp
class Base {
public:
    virtual ~Base() {
        std::cout << "Base destructor" << std::endl;
    }
};
```

> **规则**：如果一个类可能被继承，并且可能通过基类指针删除派生类对象，那么它的析构函数应该是虚函数。

## RAII 模式

**RAII（Resource Acquisition Is Initialization）** 是 C++ 中一种重要的资源管理模式：

- **构造函数**：获取资源
- **析构函数**：释放资源

```cpp
class FileHandler {
    FILE* file;
public:
    FileHandler(const char* filename) {
        file = fopen(filename, "r");
    }
    ~FileHandler() {
        if (file) fclose(file);
    }
    // 禁止拷贝，避免重复释放
    FileHandler(const FileHandler&) = delete;
    FileHandler& operator=(const FileHandler&) = delete;
};

void readFile() {
    FileHandler fh("data.txt");
    // 使用文件...
}  // 函数结束，fh 析构，文件自动关闭
```

智能指针（如 `std::unique_ptr`、`std::shared_ptr`）就是 RAII 模式的典型应用。

---

## 关键要点

1. **析构函数名**：`~ClassName()`，无参数，无返回类型
2. **自动调用**：对象销毁时自动调用
3. **析构顺序**：与构造顺序相反
4. **资源管理**：在析构函数中释放 `new` 分配的内存
5. **虚析构函数**：多态基类必须使用虚析构函数
6. **RAII**：利用构造/析构管理资源生命周期
