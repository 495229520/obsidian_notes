# 12.3 类的空间

## 简介

计算类的大小，方法是 `sizeof(ClassName)`。

---

## 计算规则

### 影响类大小的因素

| 因素 | 说明 |
|------|------|
| **非静态数据成员** | 类对象的大小至少是其所有非静态数据成员大小的总和，每个对象都有自己的一份非静态成员拷贝 |
| **内存对齐** | 为了提高访问效率，编译器通常会按特定规则对齐成员变量。这意味着在成员之间或在最后一个成员之后可能会插入一些填充字节（Padding）。例如，一个 `int`（通常4字节）可能要求起始地址是4的倍数，如果它前面是一个 `char`（1字节），编译器可能会在 `char` 后面填充3个字节。因此，类的大小往往**大于**其成员大小的简单相加。对齐要求取决于平台架构和编译器 |
| **虚函数指针** | 如果类包含任何虚函数（直接声明或从基类继承），或者它继承自一个有虚函数的基类，那么编译器通常会为该类的每个对象添加一个隐藏的指针成员，称为虚函数指针（vptr）。这个指针指向该类的虚函数表（vtbl），用于实现多态。vptr 的大小就是指针的大小（在32位系统上通常是4字节，64位系统上是8字节） |
| **基类的大小** | 如果类有继承关系，那么派生类的大小需要包含其所有基类（包括直接和间接基类）的大小（同样要考虑基类的成员、对齐和可能的 vptr）。多重继承和虚继承会使计算更复杂 |
| **空基类优化** | 如果一个基类没有任何非静态数据成员（且没有 vptr，或者其 vptr 可以与派生类的 vptr 合并），编译器有时可以优化，使得这个空基类不占用派生类对象的额外空间 |
| **空类的大小** | 即使一个类没有任何成员（没有数据成员，没有虚函数），它的大小也**不为0**。C++ 标准规定，任何不同对象的地址必须是不同的，所以空类的大小通常是 **1 字节**（或编译器选择的某个最小尺寸），以作为占位符确保对象拥有唯一的地址 |

### 不影响类大小的因素

| 因素 | 说明 |
|------|------|
| **成员函数** | 成员函数的代码存储在代码区，只有一份，被所有对象共享，不存储在对象内部，因此不计入对象大小 |
| **静态数据成员** | 静态数据成员属于类本身，不属于任何特定对象。它们存储在全局/静态存储区，只有一份，不计入对象大小 |
| **类型定义** | 这些只是别名，不占用空间 |
| **枚举** | 枚举类型本身不占空间，除非你定义了一个该枚举类型的非静态数据成员 |
| **嵌套类** | 嵌套类的定义本身不增加外部类的大小，只有当外部类包含嵌套类的非静态数据成员对象时，才会增加大小 |

---

## 总结

- 成员函数（静态或非静态）的代码是**共享的**，只有一份
- 非静态数据成员是**每个对象独有的**，每个对象都有一份拷贝，占用独立的空间
- 静态数据成员是**类级别的**，只有一份，被所有对象共享

---

## 示例

```cpp
class MyClass {
    int a;
    int b;
    char c;
};

// sizeof(MyClass) = ?
// 理论上：4 + 4 + 1 = 9 字节
// 实际上：由于内存对齐，通常是 12 字节
```

### 内存布局分析

```
偏移量    成员     大小
0         a       4 字节
4         b       4 字节
8         c       1 字节
9-11      填充    3 字节（对齐到 4 字节边界）
总计              12 字节
```

### 空类的大小

```cpp
class Empty {};

int main() {
    std::cout << sizeof(Empty);  // 输出 1
}
```

### 含虚函数的类

```cpp
class WithVirtual {
    int data;
    virtual void func() {}
};

// 32位系统：sizeof = 8 (4字节vptr + 4字节int)
// 64位系统：sizeof = 16 (8字节vptr + 4字节int + 4字节padding)
```

---

## 关键要点

1. **非静态成员**：每个对象独有，占用空间
2. **静态成员和成员函数**：共享，不占对象空间
3. **内存对齐**：实际大小通常大于成员大小之和
4. **虚函数**：会增加 vptr 指针（4或8字节）
5. **空类**：大小为 1 字节，确保对象地址唯一
