# 12.8 对象放置构造（Placement New）

## 简介

**对象放置构造（Placement New）** 是一种特殊的 `new` 操作符的重载形式，它允许你在**已分配好的内存**上直接构造对象，而不是让 `new` 自动分配内存。

### 常规 new vs Placement New

| 操作 | 常规 `new` | Placement New |
|------|-----------|---------------|
| **内存分配** | ✅ 在[[6.6 堆与栈的区别\|堆]]上分配内存 | ❌ 不分配内存 |
| **调用构造函数** | ✅ 调用 | ✅ 调用 |

- **常规 `new`**：做两件事
  1. **分配内存**：在堆上找一块足够大的内存区域
  2. **调用[[12.1.6 构造函数\|构造函数]]**：在刚分配好的内存上构造对象

- **Placement New**：只做一件事
  1. **调用构造函数**：在你已经指定好的、预先分配的内存地址上构造对象

换而言之，Placement New **不负责分配内存**，它只是在一个已存在的、已经在别处分配好的内存上"放置"并构造一个对象。它的核心作用是**将内存分配和对象构造这两个步骤分离开来**。

---

## 语法

```cpp
#include <new>  // 必须包含 <new> 头文件

void* buffer = ...;  // 一块已经分配好的原始内存缓冲区
MyClass* pObject = new (buffer) MyClass(arg1, arg2);
```

- **`new (buffer)`**：这就是 Placement New 的语法。括号里的 `buffer` 是一个 `void*` 指针，指向一块预分配的、足够大的内存
- **`MyClass(arg1, arg2)`**：这部分和常规 `new` 一样，是调用 `MyClass` 的构造函数
- **`pObject`**：操作完成后，`pObject` 将指向 `buffer`，并指向一个在 `buffer` 这块内存上完整构造好的 `MyClass` 对象

---

## 注意事项

> **重要**：对于由 Placement New 创建的对象，**绝对不能使用 `delete pObject`**。

### 为什么不能用 delete？

因为 `delete` 会做两件事：
1. 调用对象的[[12.1.7 析构函数|析构函数]]
2. 释放内存

由于 `pObject` 所指向的内存（即 `buffer`）是你自行安排管理的（可能来自栈、全局区、或者是别的已经配置好的内存池），而不是由 `new` 从堆上分配的。如果用 `delete`，它会试图去释放一个不属于它的内存，这会导致**未定义行为**，通常是程序崩溃。

---

## 正确的销毁方式

由于不能使用 `delete`，需要手动完成两个步骤：

### 1. 显式调用析构函数

手动调用对象的析构函数来清理对象内部的资源：

```cpp
pObject->~MyClass();  // 显式调用析构函数
```

### 2. 手动释放内存（如果需要）

根据这块内存的来源，决定如何释放它：

```cpp
// 1. 显式调用析构函数
pObject->~MyClass();

// 2. 如果 buffer 是在堆上分配的，现在可以释放它了
// 例如：
delete[] static_cast<char*>(buffer);
```

## 使用场景

Placement New 主要用于以下场景：

| 场景 | 说明 |
|------|------|
| **内存池** | 预分配大块内存，避免频繁的堆分配开销 |
| **自定义内存分配器** | 实现高性能的内存管理 |
| **对象重用** | 在同一块内存上反复构造/销毁对象 |
| **特殊内存位置** | 在共享内存、内存映射文件等特定位置构造对象 |

## 实战示例：内存池

假设我们有一个游戏，需要频繁创建和销毁大量的 `Bullet`（子弹）对象。为了性能，我们使用内存池来管理内存。

### 1. 定义 Bullet 类和内存池

```cpp
#include <iostream>
#include <new>

// 简单的子弹类
class Bullet {
private:
    int id_;
    float x_, y_, z_;

public:
    Bullet(int id, float x, float y, float z)
        : id_(id), x_(x), y_(y), z_(z) {
        std::cout << "Bullet " << id_ << " constructed at address "
                  << this << std::endl;
    }

    ~Bullet() {
        std::cout << "Bullet " << id_ << " destructed." << std::endl;
    }

    void update() {
        // ... 更新子弹位置等逻辑
    }
};

// 一个非常基础的内存池类，用于分配固定大小的内存块
class BulletMemoryPool {
private:
    static const int POOL_SIZE = 100;
    char* pool_buffer_;
    bool* is_in_use_;  // 标记每个内存块是否正在被占用

public:
    BulletMemoryPool() {
        pool_buffer_ = new char[sizeof(Bullet) * POOL_SIZE];
        is_in_use_ = new bool[POOL_SIZE]{false};
        std::cout << "Memory Pool created. Buffer starts at "
                  << static_cast<void*>(pool_buffer_) << std::endl;
    }

    ~BulletMemoryPool() {
        delete[] is_in_use_;
        delete[] pool_buffer_;
    }

    // 从池中分配一块原始内存
    void* allocate() {
        for (int i = 0; i < POOL_SIZE; ++i) {
            if (!is_in_use_[i]) {
                is_in_use_[i] = true;
                char* block_address = pool_buffer_ + i * sizeof(Bullet);
                std::cout << "Pool: Allocated block " << i
                          << " at address " << static_cast<void*>(block_address)
                          << std::endl;
                return block_address;
            }
        }
        std::cout << "Pool: No free blocks left!" << std::endl;
        return nullptr;  // 内存池已满
    }

    // 将一块内存归还给池
    void deallocate(void* block_to_free) {
        ptrdiff_t offset = static_cast<char*>(block_to_free) - pool_buffer_;
        int index = offset / sizeof(Bullet);

        if (index >= 0 && index < POOL_SIZE) {
            is_in_use_[index] = false;
            std::cout << "Pool: Deallocated block " << index << std::endl;
        }
    }
};
```

### 2. 使用 Placement New 在内存池上创建和销毁对象

```cpp
int main() {
    BulletMemoryPool pool;

    std::cout << "\n--- Creating first bullet ---" << std::endl;
    // 1. 从内存池分配一块原始内存（内存分配）
    void* memory_for_bullet1 = pool.allocate();
    Bullet* bullet1 = nullptr;
    if (memory_for_bullet1) {
        // 2. 使用 Placement New 在这块内存上构造对象（对象构造）
        bullet1 = new (memory_for_bullet1) Bullet(1, 10.0f, 20.0f, 0.0f);
    }

    std::cout << "\n--- Creating second bullet ---" << std::endl;
    void* memory_for_bullet2 = pool.allocate();
    Bullet* bullet2 = nullptr;
    if (memory_for_bullet2) {
        bullet2 = new (memory_for_bullet2) Bullet(2, 30.0f, 40.0f, 0.0f);
    }

    // 使用子弹对象
    if (bullet1) bullet1->update();
    if (bullet2) bullet2->update();

    std::cout << "\n--- Destroying first bullet ---" << std::endl;
    if (bullet1) {
        // 1. 显式调用析构函数（对象销毁）
        bullet1->~Bullet();
        // 2. 将内存归还给内存池（内存释放）
        pool.deallocate(bullet1);
    }

    std::cout << "\n--- Destroying second bullet ---" << std::endl;
    if (bullet2) {
        bullet2->~Bullet();
        pool.deallocate(bullet2);
    }

    std::cout << "\n--- Reusing memory for a third bullet ---" << std::endl;
    // 它会复用刚刚被释放的内存
    void* memory_for_bullet3 = pool.allocate();
    Bullet* bullet3 = nullptr;
    if (memory_for_bullet3) {
        bullet3 = new (memory_for_bullet3) Bullet(3, 50.0f, 60.0f, 0.0f);
        bullet3->~Bullet();
        pool.deallocate(bullet3);
    }

    return 0;
}
```

## 与 malloc/new 的关系

Placement New 实际上是 `new` 运算符底层实现的一部分。当你使用普通的 `new` 时，编译器大致执行以下步骤：

```cpp
// new T(args) 的底层实现类似于：
T* ptr = (T*) operator new(sizeof(T));  // 1. 分配内存
new (ptr) T(args);                       // 2. Placement New 构造对象
return ptr;
```

更多关于 `malloc`、`new` 以及 Placement New 的对比，参见 [[12.1.13 malloc与new]]。

---

## 关键要点

1. **Placement New 不分配内存**：只在已有内存上调用构造函数
2. **必须包含 `<new>` 头文件**：使用 Placement New 需要该头文件
3. **不能用 delete 释放**：必须显式调用析构函数，然后手动管理内存
4. **主要用于性能优化**：内存池、自定义分配器等场景
5. **显式析构函数调用**：`pObject->~ClassName()` 是合法且必要的语法
