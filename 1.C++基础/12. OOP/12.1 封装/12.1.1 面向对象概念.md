# 12.1 面向对象概念

## 简介

**面向对象编程**（Object-Oriented Programming，OOP）是一种编程范式，它使用"对象"（objects）来设计软件和应用程序。这种范式的核心思想是将数据和操作数据的方法（函数）捆绑在一起，形成一个独立的对象。

在 C++ 中，OOP 主要通过**类**（Class）和**对象**（Object）来实现：

1. **类 (Class)**：类是一个蓝图或模板，用于创建对象。它定义了一组属性（数据成员，即变量）和行为（成员函数，即方法）。例如，你可以定义一个 `Car` 类，包含颜色、品牌等属性，以及启动、刹车等行为。

2. **对象 (Object)**：对象是类的一个实例（instance）。根据 `Car` 类的蓝图，你可以创建具体的 `Car` 对象，比如一辆"红色的法拉利"或一辆"蓝色的丰田"。每个对象都有类定义的属性和行为，但属性值（如颜色）可能不同。

## 不使用面向对象的例子

```cpp
struct NPC {
    int hp;
    int mp;
    char* name;
};

struct MONSTER {
    int hp;
    int mp;
    char* name;
    int damage;
    unsigned price;
};

struct ROLE {
    int hp;
    int mp;
    char* name;
    int damage;
    unsigned diamond;
    char* id;
};

bool Act(ROLE& acter, ROLE& beacter) {
    beacter.hp -= acter.damage;
    return beacter.hp > 0;
}
```

上述代码存在大量重复的数据成员定义，维护困难。

## OOP 三大特性

### 1. 封装 (Encapsulation)

**定义**：封装是将数据和操作数据的方法绑定在一起的机制，同时隐藏对象的内部实现细节，只对外暴露必要的接口（公有方法）。

**如何实现（C++）**：主要通过访问修饰符（access specifiers）来实现：

| 访问修饰符 | 说明 |
|-----------|------|
| `public` | 类外部可以访问，通常用于定义类的接口（成员函数） |
| `private` | 只能被类的成员函数访问，通常用于隐藏内部数据（成员变量）和实现细节，这是默认的访问级别（如果使用 `class` 关键字定义） |
| `protected` | 可以被类的成员函数以及其派生类（子类）的成员函数访问 |

**目的/好处**：

- **数据隐藏 (Data Hiding)**：保护对象内部状态不被外部随意修改，保证数据的完整性和安全性
- **降低耦合度**：对象的内部实现可以自由修改，只要公共接口不变，就不会影响使用该对象的其他代码
- **提高模块化**：将相关的数据和行为组织在一起

**例子**：就像一个胶囊把药粉包起来，使用者只需要知道如何服用（公共接口），不需要关心里面具体的成分和比例（私有数据和实现）。

### 2. [[12.2.1 类的继承|继承]] (Inheritance)

**定义**：继承是一种机制，允许一个类（称为派生类或子类）获取另一个类（称为基类或父类）的属性和方法。派生类可以重用基类的代码，并可以添加自己的新属性和方法，或者覆盖（override）基类的方法。

**如何实现（C++）**：使用冒号在派生类名后指定基类。例如：
```cpp
class DerivedClass : public BaseClass { ... };
```
`public` 是继承访问说明符，还有 `protected` 和 `private` 继承。

**目的/好处**：

- **代码重用**：避免在多个类中编写重复的代码
- **建立 "is-a" 关系**：表示一个类是另一个类的特殊类型（例如，Dog is a Animal）
- **创建类层次结构**：组织相关的类，使其更易于理解和管理
- **可扩展性**：可以在不修改基类的情况下扩展其功能

**例子**：汽车是一个基类，具有轮子、引擎等属性和启动、停止等方法。轿车和卡车可以继承汽车类，它们都拥有汽车的基本属性和方法，但轿车可能有独特的开天窗方法，卡车可能有独特的装载货物方法。

### 3. 多态 (Polymorphism)

**定义**：多态（源于希腊语，意为 "many forms"，多种形态）是指允许不同类的对象对相同的消息（方法调用）做出不同的响应。简单来说，就是"一个接口，多种实现"。

**如何实现（C++）**：主要有两种形式：

| 类型 | 说明 |
|------|------|
| **编译时多态 (Compile-time/Static Polymorphism)** | 通过[[9.2.1 函数重载\|函数重载]]（function overloading）和运算符重载（operator overloading）实现。编译器在编译阶段就能确定调用哪个函数 |
| **运行时多态 (Run-time/Dynamic Polymorphism)** | 通过虚函数（virtual functions）和继承来实现。当通过基类指针或引用调用派生类对象的虚函数时，实际执行哪个版本的函数是在程序运行时确定的，这是 OOP 中通常意义上的多态 |

**目的/好处**：

- **灵活性和可扩展性**：可以编写更通用的代码，处理不同类型的对象，而无需知道它们的具体类型。添加新的派生类通常不需要修改使用基类接口的代码
- **简化接口**：提供统一的接口来操作不同类的对象

**例子**：假设有一个基类 `Shape` 和一个虚函数 `draw()`，派生类 `Circle`、`Square`、`Triangle` 都重写了 `draw()` 方法。你可以创建一个 `Shape*` 指针数组，存储不同形状的对象。当遍历数组并调用每个对象的 `draw()` 方法时，程序会在运行时判断对象的实际类型（是 `Circle`、`Square` 还是 `Triangle`），并调用相应类中实现的 `draw()` 方法，从而绘制出不同的形状。

---

## 关键要点

1. **类是蓝图**：定义了对象的属性和行为
2. **对象是实例**：根据类创建的具体实体
3. **封装**：隐藏实现细节，提供公共接口
4. **继承**：代码重用，建立类层次结构
5. **多态**：同一接口，不同实现
