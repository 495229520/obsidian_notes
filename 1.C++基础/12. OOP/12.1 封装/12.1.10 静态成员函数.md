# 12.9 静态成员函数

## 简介

**静态成员函数**是使用 `static` 关键字声明的成员函数。它与普通成员函数的最大区别是：**它不与任何特定的对象实例绑定**，因此它**没有 `this` 指针**。

```cpp
static ReturnType FunctionName(parameters);
```

---

## 静态成员函数的特点

| 特点 | 说明 |
|------|------|
| **无 this 指针** | 不能访问非静态成员（变量和函数） |
| **类级别调用** | 可以通过 `ClassName::staticFunctionName()` 直接调用 |
| **可访问静态成员** | 只能直接访问静态成员变量和其他静态成员函数 |
| **不依赖对象** | 即使没有创建对象，也可以调用 |

---

## 可以访问的内容

静态成员函数**可以**访问：
- 静态成员变量
- 其他静态成员函数
- 函数参数
- 全局变量和函数

由于静态成员函数没有**this指针**，因此**不可以**直接访问：
- 非静态成员变量
- 非静态成员函数

```cpp
class SimpleClass {
private:
    int non_static_data;
    static int static_data;

public:
    // 非静态成员函数
    void non_static_func() {
        non_static_data = 10;  // OK：非静态函数可以访问非静态成员
        static_data = 20;      // OK：非静态函数也可以访问静态成员
    }

    // 静态成员函数
    static void static_func() {
        // non_static_data = 10;  // 错误！静态函数不能访问非静态成员
        static_data = 20;          // OK：静态函数可以访问静态成员
    }
};

int SimpleClass::static_data = 0;
```

---

## 调用方式

```cpp
class MyClass {
public:
    static void staticFunc() {
        std::cout << "Static function called" << std::endl;
    }
};

int main() {
    // 方式1：通过类名调用（推荐）
    MyClass::staticFunc();

    // 方式2：通过对象调用（不推荐，但合法）
    MyClass obj;
    obj.staticFunc();  // 实际上还是 MyClass::staticFunc()
}
```

---

## 示例：使用静态成员函数访问私有静态成员

```cpp
// Counter.h
#ifndef COUNTER_H
#define COUNTER_H

class Counter {
private:
    static int object_count;  // 声明静态成员变量

public:
    Counter() {
        object_count++;
        std::cout << "Counter 对象创建，当前数量：" << object_count << std::endl;
    }

    ~Counter() {
        object_count--;
        std::cout << "Counter 对象销毁，当前数量：" << object_count << std::endl;
    }

    // 静态成员函数：获取对象数量
    static int get_count() {
        return object_count;
    }
};

#endif // COUNTER_H
```

```cpp
// Counter.cpp
#include "Counter.h"

int Counter::object_count = 0;  // 定义静态成员变量
```

```cpp
// main.cpp
#include <iostream>
#include "Counter.h"

int main() {
    // 在创建对象之前就可以调用静态成员函数
    std::cout << "初始数量：" << Counter::get_count() << std::endl;

    Counter c1;
    Counter c2;
    std::cout << "当前数量：" << Counter::get_count() << std::endl;

    {
        Counter c3;
        std::cout << "当前数量：" << Counter::get_count() << std::endl;
    }  // c3 销毁

    std::cout << "当前数量：" << Counter::get_count() << std::endl;
    return 0;
}
```

## 为什么静态成员函数不能访问非静态成员？

**根本原因**：静态成员函数**没有 `this` 指针**。

- 非静态成员函数有一个隐式的 `this` 指针，指向调用该函数的对象
- 通过 `this` 指针，非静态成员函数可以访问该对象的非静态成员
- 静态成员函数不与任何对象关联，所以没有 `this` 指针
- 没有 `this` 指针，就无法知道要访问哪个对象的非静态成员

```cpp
class Example {
    int data;  // 非静态成员

    void normalFunc() {
        data = 10;        // 等价于 this->data = 10;
    }

    static void staticFunc() {
        // data = 10;     // 错误！没有 this 指针，不知道是哪个对象的 data
    }
};
```

## 静态成员变量的初始化

### 为什么需要在类外定义？

静态成员变量在类内的声明**只是声明，不是定义**。类的定义可能被多个源文件包含，如果在类内定义静态成员变量，会导致**重复定义**（违反 ODR，One Definition Rule）。

因此，静态成员变量必须在**类外**进行定义，且只能定义一次（通常放在 `.cpp` 文件中）。

```cpp
// MyClass.h
class MyClass {
private:
    static int count;          // 声明（不分配内存）
    static std::string name;   // 声明
public:
    static int getCount() { return count; }
};

// MyClass.cpp
int MyClass::count = 0;                   // 定义并初始化
std::string MyClass::name = "default";    // 定义并初始化
```

---

### 初始化语法

```cpp
// 语法：类型 类名::变量名 = 初始值;
int MyClass::count = 0;

// 也可以不显式初始化（使用默认值）
int MyClass::count;  // 默认初始化为 0（静态存储期变量的特性）
```

> [!note] 静态存储期
> 静态成员变量具有**静态存储期**，存储在全局/静态数据区，程序启动时初始化，程序结束时销毁。未显式初始化的基本类型会被**零初始化**。

---

### 特殊情况：const static 整型的类内初始化

对于 `const static` 的**整型或枚举类型**，C++ 允许在类内直接初始化：

```cpp
class Config {
public:
    // ✅ const static 整型可以在类内初始化
    const static int MAX_SIZE = 100;
    const static char FLAG = 'A';

    // ❌ 非整型不能在类内初始化（C++11 之前）
    // const static double PI = 3.14;  // 错误
};

// 如果该变量被 ODR-use（如取地址），仍需类外定义
const int Config::MAX_SIZE;  // 定义（不再初始化）
```

---

### Modern C++：inline static（C++17）

C++17 引入了 `inline static`，允许在类内**定义并初始化**静态成员变量，无需类外定义：

```cpp
class Counter {
public:
    // ✅ C++17: inline static 可以在类内定义
    inline static int count = 0;
    inline static std::string name = "Counter";

    // constexpr static 隐含 inline（C++17）
    constexpr static double PI = 3.14159;
};

// 不再需要类外定义！
```

> [!tip] 最佳实践
> 在 C++17 及以上版本中，**优先使用 `inline static`**，它更简洁且避免了类外定义的问题。

---

### 初始化顺序问题

> [!warning] 静态初始化顺序陷阱
> 不同编译单元中的静态变量初始化顺序是**未定义的**。如果一个静态成员变量依赖另一个编译单元的静态变量，可能导致未初始化就被使用。

**解决方案**：使用"局部静态变量"惯用法（Construct On First Use Idiom）

```cpp
class Database {
public:
    // ❌ 危险：初始化顺序不确定
    // static Connection conn;

    // ✅ 安全：首次调用时才初始化
    static Connection& getConnection() {
        static Connection conn("localhost", 3306);
        return conn;
    }
};
```

---

### 静态成员变量初始化总结

| 情况 | 初始化位置 | 示例 |
|------|-----------|------|
| 普通静态成员 | 类外 `.cpp` 文件 | `int MyClass::count = 0;` |
| `const static` 整型 | 类内（可选类外定义） | `const static int N = 10;` |
| `inline static`（C++17） | 类内 | `inline static int x = 0;` |
| `constexpr static`（C++17） | 类内 | `constexpr static double PI = 3.14;` |

## 静态成员函数的使用场景

1. **工厂函数**：创建对象的静态方法
2. **工具函数**：不需要对象状态的辅助函数
3. **访问静态成员**：提供对私有静态成员的访问接口
4. **回调函数**：作为 C 风格回调函数使用

```cpp
class Logger {
private:
    static std::string logFile;
    static int logLevel;

public:
    // 静态配置函数
    static void setLogFile(const std::string& file) {
        logFile = file;
    }

    static void setLogLevel(int level) {
        logLevel = level;
    }

    // 静态日志函数
    static void log(const std::string& message) {
        if (logLevel > 0) {
            std::cout << "[LOG] " << message << std::endl;
        }
    }
};

std::string Logger::logFile = "app.log";
int Logger::logLevel = 1;

int main() {
    // 不需要创建 Logger 对象
    Logger::setLogLevel(2);
    Logger::log("Application started");
}
```

---

## 关键要点

1. **无 this 指针**：静态成员函数不与特定对象绑定
2. **只能访问静态成员**：不能直接访问非静态成员
3. **类名调用**：推荐使用 `ClassName::func()` 方式调用
4. **不依赖对象**：可以在没有对象的情况下调用
5. **常用于工具函数和工厂方法**
