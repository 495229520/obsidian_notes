# 12.8 静态成员变量

## 简介

静态成员变量（`static` 成员变量）是**属于类本身**而非类的任何特定对象的变量。它在所有该类的对象之间**共享同一份数据**，而不是每个对象都有自己的一份副本。

---

## 静态成员变量的特点

| 特点 | 说明 |
|------|------|
| **类级别** | 属于类，不属于任何对象 |
| **共享** | 所有对象共享同一个静态成员 |
| **独立于对象** | 即使没有创建对象，静态成员也存在 |
| **存储位置** | 存储在全局/静态存储区，不在对象内部 |
| **生命周期** | 程序运行期间一直存在 |

---

## 声明与定义

### 传统方式（C++17 之前）

静态成员变量需要在**类内声明，类外定义**：

```cpp
// MyClass.h
class MyClass {
public:
    static int myStaticVar;  // 声明
};

// MyClass.cpp
int MyClass::myStaticVar = 0;  // 定义和初始化
```

**为什么要在类外定义？**

这种方式是为了遵循[[11.1 ODR唯一定义规则|一次定义规则（ODR）]]。如果一个头文件被多个源文件包含，而头文件中又定义了静态成员变量，那么在链接的时候就会出现重复定义的错误。

当你在头文件中定义一个**非 const** 的 `static` 全局变量或一个 `static` 函数时，每个包含该头文件的 .cpp 文件都会：
1. **独立地包含**该头文件中的 `static` 定义
2. **独立地编译**生成该 `static` 变量或函数的副本
3. 最终，在链接阶段，链接器会发现多个目标文件都定义了同一个 `static` 成员变量，从而导致 ODR 冲突

### C++17 inline 静态成员变量

C++17 引入的 `inline` 关键字用于静态成员变量，改变了这一规则。当一个静态成员变量被声明为 `inline` 时，它可以在头文件中声明并同时定义和初始化，而不会违反 ODR。编译器和链接器会确保在整个程序中，这个内联静态成员变量只有一个实例，并且所有对它的引用都指向同一个内存地址。

```cpp
// MyClass.h
class MyClass {
public:
    static inline int myStaticVar = 0;  // 声明、定义和初始化都在头文件中
};

// 可以在其他任何 .cpp 文件中直接使用 MyClass::myStaticVar
// 而不需要在某个 .cpp 文件中额外定义它
```

---

## 关键点总结

1. **C++17 及更高版本**：`inline static` 成员变量是 C++17 引入的特性
2. **ODR 例外**：`inline` 关键字允许变量在多个翻译单元（即 .cpp 文件）中被定义，但链接器会确保最终只有一个实例
3. **直接在类定义中初始化**：`inline static` 成员变量可以在类定义中直接初始化，这在 C++17 之前通常只对 `static const` 整型或枚举成员变量（且被用作常量表达式）是允许的
4. **constexpr 隐式 inline**：如果一个静态成员变量被声明为 `constexpr`，那么它会隐式地被认为是 `inline` 的，因此也可以在类定义中直接初始化，而无需显式使用 `inline` 关键字

## 示例

```cpp
class T {
    inline static int count{};
public:
    int hp;

    T() {
        count++;
    }

    int GetCount() {
        return count;
    }

    ~T() {
        count--;
    }
};

void test();
T t4;

int main() {
    test();
    T t1, t2, t3;
    T t5;
    std::cout << t1.GetCount();  // 输出：5
}

void test() {
    T t1, t2, t3;  // 函数结束后这3个对象被销毁
}
```

## 访问方式

静态成员变量可以通过以下两种方式访问：

```cpp
class Counter {
public:
    static int count;
};
int Counter::count = 0;

int main() {
    // 方式1：通过类名访问（推荐）
    Counter::count = 10;

    // 方式2：通过对象访问（不推荐）
    Counter c;
    c.count = 20;  // 实际上访问的还是 Counter::count
}
```

## 使用场景

1. **对象计数**：跟踪类创建了多少个对象
2. **共享配置**：所有对象共享的配置信息
3. **单例模式**：存储唯一实例
4. **缓存数据**：类级别的缓存

```cpp
class Widget {
private:
    static int instanceCount;
    int id;

public:
    Widget() : id(++instanceCount) {
        std::cout << "Widget " << id << " created" << std::endl;
    }

    ~Widget() {
        std::cout << "Widget " << id << " destroyed" << std::endl;
        instanceCount--;
    }

    static int getInstanceCount() {
        return instanceCount;
    }
};

int Widget::instanceCount = 0;
```

---

## 关键要点

1. **类级别共享**：静态成员变量被所有对象共享
2. **独立于对象**：不需要创建对象即可访问
3. **类外定义**：传统方式需要在 .cpp 文件中定义
4. **inline 变量**：C++17 允许在头文件中直接定义
5. **不占对象空间**：不计入 `sizeof(ClassName)`
