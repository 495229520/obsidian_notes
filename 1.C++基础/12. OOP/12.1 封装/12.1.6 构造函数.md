# 12.6 构造函数

## 简介

在 C++ 中，**构造函数**是一种特殊的成员函数，它的主要职责是在创建对象时进行**初始化**。这是一种对对象生命周期的承诺：分配资源、初始化成员变量、开始进行一些初始化操作。

---

## 构造函数的特点

- **名称与类名相同**：和所有构造函数一样
- **参数**：它通常接受一个参数，这个参数是对同类对象的常量引用（const reference），标准形式如下：
  ```cpp
  ClassName(const ClassName& other);
  ```
- **const**：表示在拷贝过程中，不应修改被拷贝的原对象（other）
- **&（引用）**：这是必须的。如果参数是按值传递（ClassName other），那么在调用副本构造函数时，又会需要调用副本构造函数来创建这个参数的副本，这会导致无限递归调用
- **目的/好处**：分配资源（如 `new` 分配内存）、初始化成员变量、开始进行初始化
- **没有返回类型**：构造函数没有返回类型，连 `void` 也没有
- **可以重载（Overload）**：一个类可以有多个构造函数，只要它们的参数列表不同（参数个数、类型或顺序不同）。这使得对象可以用多种方式初始化
- **不是 static**：构造函数是非静态成员函数，为特定对象分配和初始化资源。而 static 成员函数不与任何对象关联
- **volatile**：构造函数不能被声明为 `const` 或 `volatile`。这是因为对象在构造函数执行期间正处于"正在被创建"的状态，`const` 对象的"不可修改性"以及 `volatile` 对象的"易变性"语义在对象完全构造之前是不适用的

---

## 构造函数的种类

### 普通构造函数

```cpp
// 普通的构造函数
Role(int _lv, int _damage) {
    lv = _lv;
    damage = _damage;
    std::cout << "lv:" << lv << "damage:" << damage;
}
```

**过程**：
1. 为成员变量分配内存空间，并设置默认值
2. 进入构造函数，开始进行赋值

**缺点**：
1. 效率较低
2. 不能初始化 const 或引用成员

### 成员列表构造函数

```cpp
// 成员列表构造函数
Role(int _lv, int _damage) : lv(_lv), damage(_damage) {
    std::cout << "lv:" << lv << " " << "damage:" << damage;
}
```

这里是把 Role 类的 `lv` 成员变量和 `damage` 成员变量初始化为 `_lv` 和 `_damage`。

**优势**：
1. 效率高，直接进行初始化，而不是先初始化再赋值
2. 当成员变量中有结构体或类类型的时候，只能使用成员列表构造函数
3. 成员变量列表在函数执行之前就已经初始化完成了

**缺点**：
1. 需要注意顺序：成员变量列表的顺序要与定义的顺序一致

```cpp
Role(int _lv, int _damage) : lv(_lv), damage(_damage + lv * 100) {
    // ...
}
```

它会先执行 `lv` 的初始化，再执行 `damage` 的初始化。

## 委托构造函数

**委托构造函数**的主要目的是**减少代码重复**。当一个类有多个构造函数，并且它们之间存在一些共同的初始化逻辑时，可以将这部分公共逻辑放在一个目标构造函数中，然后让其他构造函数委托给它，而不是在每个构造函数中都编写相同的代码。

```cpp
class MyClass {
public:
    int a;
    std::string b;

    // 目标构造函数
    MyClass(int x, std::string y) : a(x), b(y) {
        // 共同的初始化逻辑
        std::cout << "Target constructor called with a = " << a << ", b = " << b << std::endl;
    }

    // 委托构造函数 1
    MyClass(int x) : MyClass(x, "default") {
        // 可以执行额外的特定初始化
        std::cout << "Delegating constructor 1 called with a = " << x << std::endl;
    }

    // 委托构造函数 2
    MyClass() : MyClass(0, "empty") {
        std::cout << "Delegating constructor 2 called" << std::endl;
    }
};
```

**执行顺序**：

1. 当创建一个 `MyClass` 对象并调用委托构造函数（例如 `MyClass(5)`）时，委托构造函数会在其成员初始化列表中调用目标构造函数 `MyClass(5, "default")`
2. 目标构造函数首先执行其初始化列表中的成员初始化（`a(x)` 和 `b(y)`），然后执行其函数体内的代码
3. 目标构造函数执行完毕后，控制权返回给委托构造函数，委托构造函数会继续执行其自身函数体内的代码
4. 最终，对象 `MyClass` 被完全构造

## 副本构造函数

**副本构造函数**（也叫拷贝构造函数）的作用是使用一个**已存在的同类型对象**来初始化一个**新的对象**。当需要创建一个现有对象的副本时，副本构造函数会被调用。

```cpp
class Role {
private:
    int lv;
    int hp;
    mutable int GetDamageCount;

public:
    // 默认构造函数
    Role(int _lv = 500) : lv(_lv) {
        std::cout << "Role 类的构造函数被调用了" << std::endl;
    }

    // 副本构造函数
    Role(const Role& other) : lv(other.lv), hp(other.hp) {
        std::cout << "Role 类的副本构造函数被调用了" << std::endl;
    }
};
```

### 调用副本构造函数的情况

| 场景 | 说明 |
|------|------|
| **使用一个已存在的对象来初始化一个新对象** | `ClassName obj1; ClassName obj2 = obj1;` |
| **按值传递对象给函数** | 当函数的参数是类的对象（按值传递）时，会调用副本构造函数来创建参数的副本 |
| **函数按值返回对象** | 当函数返回一个类的对象（按值返回）时，可能会调用副本构造函数来创建返回值的临时副本（尽管现代编译器通常会使用返回值优化 RVO/NRVO 来避免这种拷贝） |

### 使用示例

```cpp
int main() {
    Role role;
    Role user;
    user.SetLv(200);
    user.SetLv(500);

    // 调用副本构造函数
    ClassName obj1;
    ClassName obj2 = obj1;      // 调用副本构造函数
    ClassName obj3(obj1);       // 调用副本构造函数

    MyClass obj2; obj2 = obj1;  // 这行代码是赋值操作，调用的是赋值运算符 (operator=)，
                                // 而不是副本构造函数。副本构造函数只在对象创建时被调用
}
```

### explicit 关键字

在 C++ 中，如果一个构造函数只有一个参数（或者所有参数都有默认值，使得它可以用一个参数调用），它可能会被编译器用于**隐式类型转换**。

```cpp
class Role {
public:
    int lv;

    Role(int _lv = 500) : lv(_lv) {
        std::cout << "Role 类的构造函数被调用了" << std::endl;
    }

    bool IsBig(Role r1) {
        return r1.GetLv() > lv;
    }
};

int main() {
    Role role;
    Role user;
    user.SetLv(300);
    user.SetLv(200);

    // 隐式转换：50 被隐式转换为 Role(50)
    std::cout << user.IsBig(50) << std::endl;
}
```

它会把 `IsBig` 参数的第一个 Role 对象与 500 进行比较，为什么可以这样进行？

这里的 `IsBig` 接收一个 `Role` 类型，但实际上 `Role` 的构造函数有默认值，创建了一个参数的构造函数，这会导致**隐式类型转换**——会用 50 去初始化一个 `Role` 对象。

如果想要防止这种隐式类型转换，可以使用 **`explicit`** 关键字：

```cpp
explicit Role(int _lv = 500) {
    lv = _lv;
}
```

使用 `explicit` 后，`user.IsBig(50)` 会编译错误，必须显式构造：`user.IsBig(Role(50))`。

---

## 关键要点

1. **构造函数名与类名相同**，无返回类型
2. **成员初始化列表**：效率更高，对 const 和引用成员是必须的
3. **委托构造函数**：减少代码重复，委托给其他构造函数
4. **副本构造函数**：用已有对象初始化新对象
5. **explicit**：防止单参数构造函数的隐式类型转换
6. **继承中的构造函数**：参见 [[12.2.2 继承构造函数]]
