# 12.2 类

## 简介

- **结构 (struct)**：以数据成员为主（字段），可以包含方法，默认访问权限是 `public`
- **类 (class)**：以成员函数（方法）和成员变量为主，用于创建对象并封装属性和行为，默认访问权限是 `private`

> 在 C++ 中，`struct` 和 `class` 的唯一本质区别是**默认访问权限**不同。

---

## 类的定义与声明

```cpp
class ClassName {
private:    // 私有成员（默认）：只能在类内部访问
    int a;                // 数据成员
    void foo();           // 成员函数（可以在类内外定义）

protected:  // 保护成员：可以被派生类访问，不能被其他外部代码访问
    int b;
    void bar();           // 同样可以在类内外定义

public:     // 公有成员：允许在类外通过对象访问
    int c;                // 公有数据成员（通常不推荐，尽量私有化）
    ClassName();          // 构造函数 (Constructor)：用于初始化新建对象
    ~ClassName();         // 析构函数 (Destructor)：用于销毁对象前清理
    void publicFunc();    // 公有成员函数

    // 成员函数也可直接在类声明内定义
    void inlineFunc() {
        // 这是一个隐式 inline 函数
    }
};
```

## 类的使用

```cpp
class Role {
public:
    int damage;
    int lv;
    int hp;
};

int main() {
    Role user;
    user.hp = 100;
    user.lv = 200;
}
```

### 隐私数据的特性与类型安全

将成员变量声明为 `private`，使用**公有方法**来读写数据是推荐的做法：

```cpp
class Role {
private:
    int hp;

public:
    void SetHp(int _hp) {
        hp = _hp;
    }
};
```

### 顺序问题

类的成员的先后顺序**不影响类的正确性**，同位置的成员可以互相调用，而在定义顺序方面，函数内的调用会在**类定义完全完成后**才解析。

```cpp
class MyClass {
public:
    void method1() {
        method2();  // OK：可以调用在后面定义的 method2
    }
    void method2() { }
};
```

## 默认访问权限

### 类的默认访问权限 (Default Access Specifier)

- 用 `class` 关键字定义的类，如果没有声明访问权限（如 `public`、`private`、`protected`），其成员默认是 `private`
- 用 `struct` 关键字定义的结构体，默认访问权限是 `public`。它更常用于向下兼容 C 语言或表示纯数据结构
- **两者的默认访问权限**：`class` 默认 `private`

### 默认继承类型 (Default Inheritance Type)

- 对于 `class`，如果在继承时不指定访问类型（`public`、`protected`、`private`），默认使用 `private` 继承
- 对于 `struct`，默认使用 `public` 继承

```cpp
class Derived : Base { };  // 等同于 class Derived : private Base { }
struct Derived : Base { }; // 等同于 struct Derived : public Base { }
```

### 补充：私有继承和公有继承的区别

- `public` 继承后，基类的 `public` 成员在派生类中仍然是 `public`，`protected` 成员仍然是 `protected`
- `private` 继承后，基类的 `public` 和 `protected` 成员在派生类中都会变成 `private`
- 无论哪种继承方式，基类的 `private` 成员在派生类中都不能直接访问
- `public` 继承强调的是“is-a”关系，也就是“派生类是一种基类”
- `private` 继承更像“借用基类实现”，表示“派生类内部用到了基类能力”，但不想把基类接口暴露给外部
- 更完整的继承访问规则可以参考 [[12.2.1 类的继承]]

| 对比项 | `public` 继承 | `private` 继承 |
|--------|----------------|-----------------|
| 基类 `public` 成员 | 在派生类中仍是 `public` | 在派生类中变成 `private` |
| 基类 `protected` 成员 | 在派生类中仍是 `protected` | 在派生类中变成 `private` |
| 外部是否能通过派生类对象访问基类 `public` 成员 | 可以 | 不可以 |
| 是否能把派生类对象当作基类对象使用 | 可以 | 不可以 |

```cpp
class Base {
public:
    void foo() {}
protected:
    int x;
};

class A : public Base { };
class B : private Base { };

int main() {
    A a;
    a.foo();        // OK，public 继承后仍可对外访问

    B b;
    // b.foo();     // 错，private 继承后对外不可访问

    Base* p1 = &a;  // OK，A 是一种 Base
    // Base* p2 = &b; // 错，B 不能对外当成 Base 使用
}
```

## 示例：完整的类定义

```cpp
#include <iostream>
#include <string>

class Person {
private:
    std::string name;  // 私有成员变量
    int age;

public:
    // 构造函数
    Person(std::string _name, int _age) : name(_name), age(_age) {}

    // Getter 方法
    std::string getName() const { return name; }
    int getAge() const { return age; }

    // Setter 方法
    void setName(std::string _name) { name = _name; }
    void setAge(int _age) {
        if (_age >= 0) {
            age = _age;
        }
    }
};

int main() {
    Person p("Alice", 25);
    std::cout << "Name: " << p.getName() << ", Age: " << p.getAge() << std::endl;

    p.setAge(30);
    std::cout << "Updated Age: " << p.getAge() << std::endl;

    return 0;
}
```

## 类的访问控制

| 访问修饰符 | 类内部 | 派生类 | 类外部 |
|-----------|--------|--------|--------|
| `public` | ✓ | ✓ | ✓ |
| `protected` | ✓ | ✓ | ✗ |
| `private` | ✓ | ✗ | ✗ |

---

## 关键要点

1. **class vs struct**：唯一区别是默认访问权限（`class` 默认 `private`，`struct` 默认 `public`）
2. **封装原则**：数据成员通常设为 `private`，通过公有方法访问
3. **成员顺序无关紧要**：类内成员可以互相调用，不受定义顺序影响
4. **继承默认类型**：`class` 默认 `private` 继承，`struct` 默认 `public` 继承
