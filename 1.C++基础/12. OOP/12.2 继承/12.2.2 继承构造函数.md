# 12.2.2 继承构造函数

## 简介

在[[12.2.1 类的继承|继承]]关系中，[[7. 构造函数与析构函数|构造函数]]的调用有特殊的规则。理解这些规则对于正确初始化派生类对象至关重要。

---

## 构造函数的调用顺序

**核心规则**：初始化派生类时，派生类会**层层调用基类的构造函数**。

构造顺序是**从最顶层的基类开始，逐层向下**，最后才执行派生类自己的构造函数。

### 示例：三层继承

```cpp
class object {
public:
    int x;
    int y;

    object() {
        std::cout << "object was created\n" << std::endl;
    }

    object(const object& obj) {
        std::cout << "object was created by copy\n" << std::endl;
    }
};

class mapObject : public object {
public:
    int mapId;

    mapObject() {
        std::cout << "mapObject was created\n" << std::endl;
    }

    mapObject(const mapObject& obj) {
        std::cout << "mapObject was created by copy\n" << std::endl;
    }
};

class actObject : public mapObject {
public:
    int damage;

    actObject() {
        std::cout << "actObject was created\n" << std::endl;
    }

    actObject(const actObject& obj) {
        std::cout << "actObject was created by copy\n" << std::endl;
    }
};

int main() {
    /*
    object was created
    mapObject was created
    actObject was created
    */
    actObject obj;

    /*
    object was created
    mapObject was created
    actObject was created by copy
    */
    actObject obj2 = obj;
}
```

**输出分析**：

1. 创建 `actObject obj` 时：
   - 先调用 `object` 的构造函数
   - 再调用 `mapObject` 的构造函数
   - 最后调用 `actObject` 的构造函数

2. 创建 `actObject obj2 = obj` 时：
   - 先调用 `object` 的**默认**构造函数
   - 再调用 `mapObject` 的**默认**构造函数
   - 最后调用 `actObject` 的**拷贝**构造函数

---

## 自定义选择基类构造函数

派生类可以初始化基类，也可以**自定义选择调用基类的哪一个构造函数**。

### 示例：选择特定的基类构造函数

```cpp
class mapObject : public object {
public:
    int mapId;

    mapObject() {
        std::cout << "mapObject was created\n" << std::endl;
    }

    mapObject(const mapObject& obj) {
        std::cout << "mapObject was created by copy\n" << std::endl;
    }

    mapObject(int id) : mapId(id) {
        std::cout << "mapObject was created by mapId\n" << std::endl;
    }
};

class actObject : public mapObject {
public:
    int damage;

    // 派生类可以初始化基类
    actObject() : mapObject(10), damage(100) {
        std::cout << "actObject was created\n" << std::endl;
    }

    actObject(const actObject& obj) {
        std::cout << "actObject was created by copy\n" << std::endl;
    }
};

int main() {
    /*
    object was created
    mapObject was created
    actObject was created
    */
    actObject obj;

    /*
    object was created
    mapObject was created
    actObject was created by copy
    */
    actObject obj2 = obj;
}
```

通过在派生类构造函数的**成员初始化列表**中指定基类构造函数，可以选择调用哪个版本的基类构造函数。

## 构造函数的继承（using 声明）

C++11 引入了 `using` 声明，允许派生类**继承基类的构造函数**。

### 语法

```cpp
class actObject : public mapObject {
public:
    // 构造函数可以通过 using 强行继承
    // 此时 actObject 可以访问 mapObject 中的构造函数
    using mapObject::mapObject;

    int damage;

    // 派生类可以初始化基类
    actObject() : mapObject(10), damage(100) {
        std::cout << "actObject was created\n" << std::endl;
    }

    actObject(const actObject& obj) {
        std::cout << "actObject was created by copy\n" << std::endl;
    }
};
```

### 使用继承的构造函数

使用 `using mapObject::mapObject;` 后，`actObject` 可以直接使用 `mapObject` 中定义的构造函数：

```cpp
int main() {
    // 使用继承的构造函数
    actObject obj(10);  // 调用从 mapObject 继承来的 mapObject(int id) 构造函数
}
```

### 注意事项

| 要点 | 说明 |
|------|------|
| **继承所有构造函数** | `using Base::Base;` 会继承基类的所有构造函数 |
| **不继承默认/拷贝/移动构造函数** | 这些特殊构造函数不会被继承 |
| **成员初始化** | 继承的构造函数只会初始化基类成员，派生类新增的成员需要使用默认值或类内初始化 |

## 构造函数调用规则总结

```
派生类对象创建
    │
    ▼
┌─────────────────────────────┐
│  1. 调用最顶层基类构造函数    │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  2. 逐层调用中间基类构造函数  │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  3. 调用派生类自己的构造函数  │
└─────────────────────────────┘
```

---

## 关键要点

1. **构造顺序**：基类先于派生类构造（从顶向下）
2. **默认行为**：如果不显式指定，派生类会调用基类的默认构造函数
3. **显式指定**：通过成员初始化列表可以选择调用基类的特定构造函数
4. **using 继承**：C++11 允许使用 `using Base::Base;` 继承基类的构造函数
5. **拷贝构造**：即使调用派生类的拷贝构造函数，基类部分仍会调用默认构造函数（除非显式调用基类的拷贝构造函数）
