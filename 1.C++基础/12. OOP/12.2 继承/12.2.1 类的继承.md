# 12.2.1 类的继承

## 简介

**继承**（Inheritance）是[[12.1.1 面向对象概念|面向对象编程]]三大特性之一，允许一个类（派生类/子类）获取另一个类（基类/父类）的属性和方法。

---

## 继承的优点

| 优点 | 说明 |
|------|------|
| **代码重用 (Code Reusability)** | 子类可以直接使用父类的代码，避免了重复编写相同的代码，继承减少了冗余代码 |
| **可扩展性 (Extensibility)** | 你可以在不修改现有父类代码的情况下，通过创建子类来扩展它的功能。形成清晰的继承层次关系，使得代码更容易扩展和修改 |
| **多态支持 (Polymorphism)** | 继承是实现多态的基础。程序可以针对父类编写通用的代码，而在运行时根据实际的子类对象执行不同的操作 |

---

## 继承语法

```cpp
class DerivedClassName : access_specifier BaseClassName {
    // ... members of the derived class ...
};
```

- **DerivedClassName**：你要创建的新类（子类）的名称
- 冒号 `:` 表示继承关系的开始，后面是 `DerivedClassName` 所继承自的 `BaseClassName`
- **access_specifier**：访问修饰符，可以是 `public`、`protected` 或 `private`，它决定了继承来的成员在派生类中的可见性，最常用的是 `public` 继承
- **BaseClassName**：你要继承的现有类（父类）的名称

---

## 访问修饰符与继承方式

**基类中的成员**（变量和函数）有三种访问方式：`public`、`protected` 和 `private`。它决定了继承来的成员在派生类中的可见性，以及派生类对象是否可以访问继承的成员。

### 我们首先需要知道：

- 基类的 `public` 成员在基类中可被外界访问，是 `public`
- 基类的 `protected` 成员在派生类中仍然是 `protected`
- 基类的 `private` 成员虽然被继承下来了，但是在派生类中是不可访问的

| 基类成员访问权限 | 派生类内部访问 | 派生类对象访问 |
|-----------------|---------------|---------------|
| `public` | 可访问 | 可访问 |
| `protected` | 可访问 | 不可访问 |
| `private` | 不可访问 (inaccessible) | 不可访问 |

---

## 三种继承方式

### public 继承

- 基类的 `public` 成员在派生类中还是 `public`
- 基类的 `protected` 成员在派生类中仍然是 `protected`
- 基类的 `private` 成员在派生类中是 `private`（不可访问）

### protected 继承

- 基类的 `public` 成员在派生类中变成 `protected`
- 基类的 `protected` 成员在派生类中仍然是 `protected`
- 基类的 `private` 成员在派生类中是 `private`

### private 继承

- 基类的 `public`、`protected` 成员在派生类中全变成 `private`

---

## 继承后的成员属性

| 基类成员 \ 继承方式 | public 继承 | protected 继承 | private 继承 |
|-------------------|------------|----------------|--------------|
| **public** | public | protected | private |
| **protected** | protected | protected | private |
| **private** | 不可访问 | 不可访问 | 不可访问 |

> **注意**：无论采用哪种继承方式，基类的 `private` 成员在派生类中都是不可访问的，但它们仍然存在于派生类对象的内存中。

## 代码示例

### 示例 1：访问权限演示

```cpp
class subObject : public object {
public:
    int x;
    int y;
    // 在派生类 protected 区域直接访问
    // 基类的 private 私有成员是无法访问的
private:
    int z;
};

class MoveObject : private subObject {
private:
    int lx;
    int ly;
protected:
    int lz;
public:
    // ...
};
```

### 示例 2：继承中的访问控制

```cpp
class ArtObject : public MoveObject {
public:
    // private 继承将 public 属性转变成了 private 属性继承
    // 在 MoveObject 中，无法访问父类 subObject 的 x 和 y 属性了
};
```

## 完整示例：Animal 类继承

```cpp
#include <iostream>
#include <string>

// 基类（父类）
class Animal {
public:
    // 构造函数
    Animal(int age) : age(age) {
        std::cout << "一个动物诞生了。" << std::endl;
    }

    // 公有成员变量
    int age;

    // 公有成员函数
    void eat() {
        std::cout << "动物正在吃东西..." << std::endl;
    }
};

// 第二步：定义派生类 Dog
// Dog 类继承自 Animal，包括有 Animal 的所有 public 和 protected 成员
// 同时，Dog 还有自己独特的行为，比如 "汪汪" 叫
class Dog : public Animal {
public:
    // 派生类自己的成员变量
    std::string name;

    // 派生类需要调用基类的构造函数来初始化基类中的成员变量
    Dog(int age, std::string name) : Animal(age), name(name) {
        std::cout << "一只狗诞生了。" << std::endl;
    }

    // 派生类自己的成员函数
    void bark() {
        std::cout << name << " 正在汪汪叫！" << std::endl;
    }
};

// 第三步：在 main 函数中使用
int main() {
    // 创建一个 Dog 对象
    Dog myDog(3, "旺财");

    // 调用派生类自己的方法
    myDog.bark();     // 输出: 旺财 正在汪汪叫！

    // 调用从基类继承来的方法
    myDog.eat();      // 输出: 动物正在吃东西...

    // 访问从基类继承来的成员变量
    std::cout << "今年 " << myDog.name << " 今年 " << myDog.age << " 岁了。" << std::endl;

    return 0;
}
```

**程序输出**：
```
一个动物诞生了。
一只狗诞生了。
旺财 正在汪汪叫！
动物正在吃东西...
今年 旺财 今年 3 岁了。
```

---

## 关键要点

1. **继承语法**：`class Derived : access_specifier Base { };`
2. **三种继承方式**：`public`（最常用）、`protected`、`private`
3. **访问控制**：基类的 `private` 成员在派生类中永远不可直接访问
4. **代码重用**：派生类自动获得基类的 `public` 和 `protected` 成员
5. **构造顺序**：先调用基类[[7. 构造函数与析构函数]]，再调用派生类构造函数
