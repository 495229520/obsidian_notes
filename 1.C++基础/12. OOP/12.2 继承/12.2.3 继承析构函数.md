# 12.2.3 继承析构函数

## 简介

与[[12.2.2 继承构造函数|构造函数]]的调用顺序相反，[[12.1.7 析构函数|析构函数]]在继承中遵循**先派生类，后基类**的调用顺序。

---

## 析构函数的调用顺序

**核心规则**：与构造函数相反，析构函数是**先调用派生类再调用基类**的。

```
对象销毁
    │
    ▼
┌─────────────────────────────┐
│  1. 调用派生类的析构函数      │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  2. 逐层调用中间基类析构函数  │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  3. 调用最顶层基类的析构函数  │
└─────────────────────────────┘
```

### 示例代码

```cpp
class object {
public:
    int x;
    int y;

    object() {
        std::cout << "object was created\n" << std::endl;
    }

    object(const object& obj) {
        std::cout << "object was created by copy\n" << std::endl;
    }

    ~object() {
        std::cout << "object was delete" << std::endl;
    }
};

class mapObject : public object {
public:
    int mapId;

    mapObject() {
        std::cout << "mapObject was created\n" << std::endl;
    }

    mapObject(const mapObject& obj) {
        std::cout << "mapObject was created by copy\n" << std::endl;
    }

    mapObject(int id) : mapId(id) {
        std::cout << "mapObject was created by mapId\n" << std::endl;
    }

    ~mapObject() {
        std::cout << "mapObject was delete\n" << std::endl;
    }
};

class actObject : public mapObject {
public:
    // 构造函数可以通过 using 强行继承
    // 此时 actObject 可以访问 mapObject 中的构造函数
    using mapObject::mapObject;

    int damage;

    // 派生类可以初始化基类
    actObject() : mapObject(10), damage(100) {
        std::cout << "actObject was created\n" << std::endl;
    }

    actObject(const actObject& obj) {
        std::cout << "actObject was created by copy\n" << std::endl;
    }

    ~actObject() {
        std::cout << "actObject was delete\n" << std::endl;
    }
};

int main() {
    /*
    object was created
    mapObject was created
    mapObject was delete
    object was delete
    */
    mapObject Map1;
}
```

**输出分析**：
- 构造时：先 `object`，后 `mapObject`
- 析构时：先 `mapObject`，后 `object`

## 重命名问题（名称隐藏）

在派生类运行变量名与基类一样，在派生类中可以调用基类的同名变量。具体方法如下：

### 使用作用域解析运算符访问基类成员

```cpp
class mapObject : public object {
public:
    int x;        // 与基类 object 中的 x 同名
    int mapId;

    mapObject() {
        x = 2500;
        object::x = 3500;  // 显式访问基类的 x
        std::cout << "mapObject was created\n" << std::endl;
    }
};

int main() {
    mapObject Map;
    Map.x = 1;           // 访问派生类的 x
    Map.object::x = 3500; // 访问基类的 x
}
```

### 访问同名函数

同样的方法也可以用于访问基类中有同名的函数：

```cpp
actObject obj;

mapObject MAP;

MAP.x = 250;
MAP.object::x = 3500;

MAP.ShowX(1500);           // 调用 mapObject::ShowX
MAP.object::ShowX(2500);   // 调用 object::ShowX

obj.ShowX(22);             // 调用 actObject::ShowX（如果有）
obj.mapObject::ShowX(33);  // 调用 mapObject::ShowX
obj.mapObject::object::ShowX(44);  // 调用 object::ShowX
obj.object::ShowX(55);     // 直接调用 object::ShowX
```

## 名称隐藏规则

| 情况 | 说明 |
|------|------|
| **同名变量** | 派生类的同名变量会**隐藏**基类的同名变量 |
| **同名函数** | 派生类的同名函数会**隐藏**基类的所有同名函数（即使参数不同） |
| **访问基类成员** | 使用 `Base::member` 语法显式访问 |

> **注意**：名称隐藏与函数重写（override）不同。隐藏是编译时的名称查找规则，而重写是运行时多态的机制。

---

## 构造与析构顺序对比

| 操作 | 顺序 |
|------|------|
| **构造** | 基类 → 派生类（从顶向下） |
| **析构** | 派生类 → 基类（从底向上） |

这种设计确保了：
- 构造时，派生类可以安全地使用基类已初始化的成员
- 析构时，派生类先清理自己的资源，然后基类再清理基类的资源

---

## 关键要点

1. **析构顺序**：与构造顺序**相反**，先派生类后基类
2. **名称隐藏**：派生类的同名成员会隐藏基类的同名成员
3. **显式访问**：使用 `BaseClass::member` 语法访问被隐藏的基类成员
4. **资源管理**：析构顺序确保派生类资源先于基类资源被释放
5. **虚析构函数**：如果通过基类指针删除派生类对象，基类应声明[[12.1.7 析构函数]]
