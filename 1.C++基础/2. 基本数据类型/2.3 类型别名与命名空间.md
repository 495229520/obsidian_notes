## 类型别名

C++ 提供了多种为类型创建别名的方式，便于代码的可读性和维护性。

### 方式对比

| 方式 | 语法 | 示例 | 说明 |
|------|------|------|------|
| `#define` | `#define 别名 原类型` | `#define xxx int`<br>`xxx a[100];` | 预处理器宏替换，**不推荐用于类型别名** |
| `typedef` | `typedef 原类型 别名` | `typedef int eInt;`<br>`eInt b{};` | C风格类型别名，仍然广泛使用 |
| `using` | `using 别名 = 原类型` | `using int64 = long long;`<br>`int64 c{};` | **C++11推荐方式**，语法更清晰 |

> **推荐**：现代C++优先使用 `using`，语法更直观，且支持模板别名。

## 命名空间（namespace）

### 什么是命名空间

命名空间用于解决**名称冲突**问题，将标识符（变量、函数、类等）组织在一个逻辑分组中。

```cpp
// 标准库的命名空间
std::cout << "这里是命名空间";

// 使用 using namespace 后，后面的代码都不需要加 std:: 前缀
using namespace std;

// 也可以只引入特定的标识符（更安全的做法）
using std::cout;
using std::cin;

cout << "不需要std! ";
int a;
cin >> hex >> a;  // hex 是输入格式控制符
```

### 自定义命名空间

```cpp
namespace IGame
{
    int HP{ 1000 };
    int MP{ 1000 };
}

// 访问方式1：使用完全限定名
cout << IGame::HP;

// 访问方式2：使用 using 声明
using IGame::HP;
cout << HP;
```

### 嵌套命名空间

命名空间可以嵌套定义：

```cpp
namespace IGame
{
    int HP{ 1000 };
    int MP{ 1000 };

    namespace Weapon
    {
        int damage{ 3000 };
    }
}

// 访问嵌套命名空间的成员
cout << IGame::Weapon::damage;  // 输出: 3000
```

#### 多层嵌套示例

```cpp
namespace IGame
{
    int HP{ 1000 };
    int MP{ 1000 };
    int lv{ 1314 };

    namespace Weapon
    {
        int damage{ 3000 };
        int lv{ 1200 };

        namespace WeaponInfo
        {
            int lv = IGame::lv;       // 1314 - 访问外层命名空间
            int lv1 = Weapon::lv;     // 1200 - 访问父级命名空间
        }
    }
}

cout << IGame::Weapon::WeaponInfo::lv;   // 1314
cout << IGame::Weapon::WeaponInfo::lv1;  // 1200
```

### 注意事项

1. **命名空间不可以放在函数体内**
2. 命名空间可以在一个命名空间内嵌套另一个命名空间
3. 子级命名空间调用父级命名空间依然需要限定符

---
## 关键要点

- **命名空间**：用于组织代码，避免名称冲突，推荐使用 `using` 声明而非 `using namespace`
- **类型别名**：优先使用 `using`（C++11），比 `typedef` 更直观
