## 枚举类型（enum）

### 声明语法

```cpp
enum class 类型名称 : 基本类型  // 基本类型可选，默认为 int
{
    选项1,
    选项2,
    选项3
};
```

### 基本概念

| 概念 | 说明 |
|------|------|
| **命名整数常量** | 枚举中的每个成员（称为"枚举器"或"枚举常量"）都代表一个整数值 |
| **默认值** | 默认情况下，枚举中的第一个成员被赋值为 0，后续成员的值依次递增 1 |
| **手动赋值** | 可以手动为枚举成员指定整数值。如果没有为所有成员赋值，则未赋值的成员将从前一个已赋值成员的值开始递增 |

### 实例

```cpp
enum class EquipLv : int
{
    normal,   // 0
    hight,    // 1
    rare,     // 2
    epic,     // 3
    legend,   // 4
    myth      // 5
};

// 如果不设置初始值，则从 0 开始递增
// 这里 normal 的值是 0；hight 的值是 1

// 使用枚举变量
// 语法：类型名称 变量名称{类型名称::选项名称};
EquipLv weaponCLV{ EquipLv::normal };
EquipLv weaponDLv{ EquipLv::myth };
```

### 为什么使用枚举？

- **提高可读性**：使用有意义的名称比使用原始整数值更能清楚地表达代码的意图。例如，`Color::Red` 比 `0` 更容易理解
- **增强可维护性**：如果需要更改某个常量的值，只需修改枚举定义即可，而不需要在代码中查找和替换所有出现该值的地方
- **类型安全**（特别是 `enum class`）：传统的 C++ 枚举存在一些类型安全问题，但 C++11 引入的 `enum class` 解决了这些问题

### 普通枚举 vs 枚举类（enum class）

#### 普通枚举（enum）

```cpp
enum Color {
    Red,      // 默认值为 0
    Green,    // 默认值为 1
    Blue = 5, // 手动赋值为 5
    Yellow    // 默认值为 6（从 Blue 的值递增）
};

// 使用方式
Color myColor = Red;
int colorValue = Green;  // 隐式转换为 int
```

#### 枚举类（enum class）- C++11 推荐

```cpp
enum class Day {
    Sunday,
    Monday,
    Tuesday,
    Wednesday,
    Thursday,
    Friday,
    Saturday
};

// 使用方式
Day today = Day::Tuesday;

// int dayValue = today;  // 错误: 无法隐式转换
int dayValue = static_cast<int>(today);  // 正确: 显式转换

if (today == Day::Tuesday) {
    // ...
}
```

**`enum class` 的优势**：
- 枚举器的名称仅在其枚举类型的作用域内可见，**解决了命名冲突问题**
- 需要使用 `EnumTypeName::EnumeratorName` 的形式来访问它们
- **不能隐式转换为整数类型**，也不能隐式地将整数类型转换为枚举类型，提供了更好的类型安全性
- 需要进行显式类型转换（`static_cast`）

### 注意事项

1. 枚举的类型必须是**整数类型**：`int`、`short`、`long` 等，**不可是浮点数类型**（`short`、`double` 等）
2. 对枚举类型进行操作的时候需要进行强制转换，否则无法使用：
   ```cpp
   std::cout << "这件装备的品质为: " << (int)weaponCLV;
   ```

## 结构体（struct）

### 为什么需要自定义数据类型？

当我们需要描述一个复杂对象（如一辆车）时，使用多个独立变量会很麻烦：

```cpp
// 不使用结构体 - 代码冗余
int wheel{ 19 };
unsigned price{ 1000000 };
char logo{ 'B' };
unsigned gls{ 100 };

std::cout << "车辆商标" << logo << "车轮大小" << wheel << "车辆售价" << price << std::endl;

// 如果要创建另一辆车，需要重复定义所有变量
int wheel2{ 19 };
unsigned price2{ 1000000 };
char logo2{ 'B' };
unsigned gls2{ 100 };

std::cout << "车辆商标" << logo2 << "车轮大小" << wheel2 << "车辆售价" << price2 << std::endl;
```

像这样，每创建一台新的车都需要重复的进行上述操作十分的麻烦，可以引用结构体来解决。

### 定义结构体

```cpp
// 自定义结构体
struct CAR
{
    char logo{ 'A' };
    unsigned short wheel{ 19 };
    unsigned price{ 90000 };
};
```

### 使用结构体

```cpp
// 创建结构体变量并初始化
CAR carA{ 'C', 25, 5000000 };
CAR carB{ 'B', 22, 6000000 };
CAR carC{ 'A', 22, 16000000 };

// 访问结构体成员
std::cout << "车辆商标" << carA.logo << "车轮大小" << carA.wheel << "车辆售价" << carA.price << std::endl;
std::cout << "车辆商标" << carB.logo << "车轮大小" << carB.wheel << "车辆售价" << carB.price << std::endl;
std::cout << "车辆商标" << carC.logo << "车轮大小" << carC.wheel << "车辆售价" << carC.price << std::endl;
```

### 结构体的本质

1. **结构的本质是按照我们自己的方式定义一块连续的内存的结构**
2. 声明一个结构变量的本质是向计算机申请一块内存，而这块内存的大小，至少是我们定义的结构成员需要占用的内存之和
3. 使用结构则是按照我们定义好的方式从这块内存存取和写入数据
