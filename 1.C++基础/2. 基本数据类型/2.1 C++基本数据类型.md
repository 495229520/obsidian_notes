## 1. 变量与常量

### 1.1 变量定义的方式

```cpp
类型 名字 {初始值};
类型 名字 = 初始值;

// 示例
int _hp_{ 900 };
int _hp1_ = 900;
```

### 1.2 常量定义的方式

```cpp
// 方式一：直接在输入输出的地方使用值
cout << 100;

// 方式二：const 关键字
const 类型 名字 {初始值};
类型 const 名字 {初始值};

// 方式三：#define 宏定义
#define 名字 值

// 示例
#define hp 100
const int hp = 900;
// 推荐写法：
const int hp {900};
int const hp {900};
```

### 1.3 变量与常量的区别

| 类型 | 内存特性 |
|------|----------|
| **变量** | 在内存中申请一块区域，用来存放数据，该内存区域**允许修改** |
| **常量** | 在内存中申请一块区域，用来存放数据，该内存区域**不允许修改** |

### 1.4 输入和输出

```cpp
// 输出
std::cout << 输出的内容;

// 输入
// printf 输出的字符串，int 是不可以的
```

## 4. 浮点型 (Floating-Point Types)

| **类型** | **内存占用字节** | **取值范围** |
|----------|-----------------|--------------|
| `float` | 4 | 大约 7 位精度，±3.4E±38 |
| `double` | 8 | 大于 15 位精度，±1.7E±308 |
| `long double` | 8 | 大于 15 位精度，±1.7E±308 |

### 4.1 浮点数字面量表达

| **写法** | **示例** | **说明** |
|----------|----------|----------|
| 单精度浮点数 `float`，小数后加 `f` 或 `F` | `200.0f` / `200.0F` | 注意必须要有小数点 |
| 不带后缀的浮点数一律视为 `double` 类型 | `200.0` | 默认类型 |
| 后缀为 `l/L` 的浮点数视为 `long double` 类型 | `200.0l` / `200.0L` | 注意必须要有小数点 |

### 4.2 注意事项

> **浮点数不仅可以表达小数，也可以表达整数**
>
> 例如：`1E2 = 100.0 = 1.0 × 10²`

> **浮点数运算精度需要注意**
>
> ```cpp
> float a {65923600.}, b {0.000001}, c {a + b};  // c = ?
> ```
>
> `float` 浮点数可以存储 ±3.4E±38 的内容，但是它的**精度只有 7 位**，很有可能会造成精度损失。
>
> 这里 `c` 会等于 `a`，就是因为精度的问题。

| 65 | ASCII | A |
| 97 | ASCII | a |

字符在内存中存储的是数字，通过编码表（如 ASCII）映射为对应的字符显示。

### 5.2 字符数据类型

|----------|-------------|----------|

| `char32_t` | 4 字节 | UTF-32 字符 |

```cpp
// 可以使用字符和数字进行初始化，同时可以进行运算操作
char charA{ 'A' };

charA++;  // charA 变为 'B'
```

```cpp
char32_t d{ U'A' };  // UTF-32
```

## 6. 类型转换

### 6.1 类型转换顺序表

虽然具体的转换规则比较复杂，但通常遵循以下顺序（精度从低到高）：

```
bool -> char -> short -> int -> long -> long long
float -> double -> long double
```

### 6.2 隐式转换

当两个不同的数据类型进行计算的时候，**系统会使优先级低的类型转换为优先级高的类型**。

```cpp
int a {500};
unsigned int b {1000};
a - b;  // 当遇到这种情况的时候，系统就会将 a 转化为 unsigned int 了
```

### 6.3 显式转换

```cpp
类型(要转换的内容)

int a = int(b);
```

此时 `a - b` 的结果就变成了 `int` 的类型，**不会因为类型转换导致计算出错**。

### 6.4 静态转换 (static_cast)

#### 语法

```cpp
static_cast<目标类型>(要转换的内容);

int a = static_cast<int>(b);
```

#### 什么是 static_cast

`static_cast` 是 C++ 提供的**类型安全的转换操作符**之一。它在编译期进行类型检查，只允许**有意义的、相关联的类型之间的转换**。

#### 与显式转换的区别

| 特性 | 显式转换 `(type)value` | 静态转换 `static_cast<type>(value)` |
|------|----------------------|-----------------------------------|
| **语法** | C 风格：`int(b)` 或 `(int)b` | C++ 风格：`static_cast<int>(b)` |
| **编译期检查** | **弱检查**，几乎允许任何转换 | **强检查**，只允许安全的转换 |
| **类型安全** | 不安全，可能导致未定义行为 | 相对安全，拒绝危险转换 |
| **可读性** | 不明显，容易被忽略 | 显眼，代码意图清晰 |
| **推荐度** | 不推荐（C++ 中应避免） | **推荐**（Modern C++ 最佳实践） |

#### 为什么要用 static_cast

1. **编译期类型检查**：防止无意义的转换

```cpp
int a = 100;
double* pd1 = (double*)&a;              // C 风格：编译通过（危险！）
double* pd2 = static_cast<double*>(&a); // static_cast：编译错误！类型不兼容
```

2. **代码意图清晰**：一眼看出这是类型转换操作

```cpp
int x = (int)3.14;              // 不太显眼
int y = static_cast<int>(3.14); // 清晰明确
```

3. **符合 Modern C++ 最佳实践**：根据《Effective C++》Item 27，优先使用 C++ 风格的类型转换

#### 什么是"相关类型"

`static_cast` 只允许在**存在合理转换关系**的类型之间进行转换。具体包括：

| 相关类型 | 示例 | 说明 |
|---------|------|------|
| **数值类型之间** | `int` ↔ `double` | 隐式转换的显式版本 |
| **指针与 `void*`** | `int*` → `void*` → `int*` | 必须转回原类型 |
| **继承层次中的指针/引用** | `Derived*` → `Base*` | 向上转型安全，向下转型需确保实际类型 |
| **枚举与整型** | `enum` ↔ `int` | C++11 枚举类需显式转换 |

**不相关的类型**（static_cast 会拒绝）：
```cpp
int* pi = nullptr;
double* pd = static_cast<double*>(pi);  // ❌ 编译错误！int* 与 double* 不相关
char* pc = static_cast<char*>(pi);      // ❌ 编译错误！
```

> **核心原则**：如果两种类型之间不存在隐式转换（或其逆转换），则它们是"不相关的"，static_cast 会拒绝转换。

#### 适用场景

- **基本数据类型之间的转换**（如 `int` → `double`）
- **指针类型的向上转换**（派生类指针 → 基类指针）
- **`void*` 转回原始指针类型**
- **消除隐式转换的歧义**

#### 示例：对比三种转换方式

```cpp
double pi = 3.14159;

// 隐式转换（自动）
int a = pi;  // 编译器自动转换，可能丢失精度

// 显式转换（C 风格，不推荐）
int b = (int)pi;
int c = int(pi);

// 静态转换（C++ 风格，推荐）
int d = static_cast<int>(pi);  // 明确表达转换意图，编译期检查
```

#### 关键要点

1. `static_cast` 提供**编译期类型安全检查**
2. 拒绝无关联类型之间的转换（如 `int*` → `double*`）
3. 在 C++ 中应**优先使用 `static_cast`**，避免 C 风格转换
4. 是 Modern C++ 中最常用的类型转换方式

> **深入学习**：更多关于 C++ 四种类型转换（`static_cast`、`const_cast`、`dynamic_cast`、`reinterpret_cast`）的详细说明，请参阅 [[13. 类型转换方式]]

### 6.5 重新解释转换 (reinterpret_cast)

#### 语法

```cpp
reinterpret_cast<目标类型>(要转换的内容);
```

#### 什么是 reinterpret_cast

`reinterpret_cast` 是 C++ 中**最危险**的类型转换操作符。它执行的是**底层位模式的重新解释**，不进行任何类型检查或数据转换，仅仅是告诉编译器"把这块内存当作另一种类型来看待"。

#### 与 static_cast 的本质区别

| 特性 | static_cast | reinterpret_cast |
|------|-------------|------------------|
| **转换对象** | 相关类型之间 | **任意**指针/引用类型之间 |
| **类型检查** | 编译期检查类型关联性 | **几乎不检查**，仅验证语法 |
| **数据转换** | 可能产生实际的值转换代码 | **无转换**，仅重新解释位模式 |
| **安全性** | 相对安全 | **极度危险**，易导致未定义行为 |
| **典型用途** | 数值转换、继承层次转换 | 底层内存操作、硬件编程 |

#### 使用场景

`reinterpret_cast` 的合法使用场景非常有限：

```cpp
// 1. 指针与足够大的整数类型之间转换（用于打印地址或存储指针值）
int* ptr = new int(42);
uintptr_t addr = reinterpret_cast<uintptr_t>(ptr);  // 指针 → 整数
int* ptr2 = reinterpret_cast<int*>(addr);           // 整数 → 指针

// 2. 不同指针类型之间转换（底层内存操作）
char buffer[sizeof(int)];
int* pInt = reinterpret_cast<int*>(buffer);  // 将 char 数组当作 int 访问

// 3. 函数指针转换（极少使用）
using FuncPtr = void(*)();
void* vp = reinterpret_cast<void*>(&someFunction);
```

#### 危险示例

```cpp
// ❌ 危险：类型双关（type punning）可能导致未定义行为
float f = 3.14f;
int* pi = reinterpret_cast<int*>(&f);
int i = *pi;  // 未定义行为！违反严格别名规则（strict aliasing）

// ❌ 危险：对象布局不兼容
struct A { int x; };
struct B { double y; };
A a{42};
B* pb = reinterpret_cast<B*>(&a);  // 可以编译，但访问 pb->y 是未定义行为
```

> [!warning] 严格别名规则
> C++ 标准规定，通过指向不兼容类型的指针访问对象是**未定义行为**。
> `reinterpret_cast` 不会绕过这个规则，滥用会导致优化器产生错误代码。

#### 合法的类型双关方式

如果需要查看浮点数的位表示，应使用 `std::memcpy` 或 C++20 的 `std::bit_cast`：

```cpp
// ✅ 安全：使用 memcpy
float f = 3.14f;
int i;
std::memcpy(&i, &f, sizeof(f));  // 安全地复制位模式

// ✅ C++20：使用 bit_cast
#include <bit>
int i = std::bit_cast<int>(f);  // 编译期安全转换
```

#### 关键要点

1. `reinterpret_cast` **不进行任何值转换**，仅重新解释内存位模式
2. 适用于**不相关类型**之间的转换（static_cast 无法完成的场景）
3. **极易导致未定义行为**，仅在底层系统编程中谨慎使用
4. 优先考虑 `std::memcpy` 或 `std::bit_cast` 替代类型双关
5. 遵循《Effective C++》Item 27：**尽量少做转型，尤其是 reinterpret_cast**

## 8. 进制转换

### 8.1 十进制转换为其他进制

**方法**：将要转换的十进制数除以要转换的目标进制数，得到商和余数，商如果大于目标进制数，那么继续用商除以目标进制数，重复直到商小于目标进制数，将得到的最后的商，和历次相除的余数连接起来，就是转换后的目标进制数。

| **983 转 2 进制** | **983 转 8 进制** | **983 转 16 进制** |
|-------------------|-------------------|---------------------|
| 983/2=491..1 | 983/8=122..7 | 983/16=61..7 |
| 491/2=245..1 | 122/8=15..2 | 61/16=3..13(D) |
| 245/2=122..1 | 15/8=1..7 | **结果：3D7** |
| 122/2=61..0 | **结果：1727** | |
| 61/2=30..1 | | |
| 30/2=15..0 | | |
| 15/2=7..1 | | |
| 7/2=3..1 | | |
| 3/2=1..1 | | |
| **结果：1111010111** | | |

### 8.2 其他进制转换为十进制

**方法**：从最高位依次计算每一位数字与当前进制数与该位序减一次方的乘积，最后将各位的乘积相加。

| **0b1111010111 转 10 进制** | **01727 转 10 进制** | **0x3D7 转 10 进制** |
|-----------------------------|----------------------|----------------------|
| 1×2⁹ = 512 | 1×8³ = 512 | 3×16² = 768 |
| 1×2⁸ = 256 | 7×8² = 448 | D×16¹ = 208 |
| 1×2⁷ = 128 | 2×8¹ = 16 | 7×16⁰ = 7 |
| 1×2⁶ = 64 | 7×8⁰ = 7 | **结果：983** |
| 0×2⁵ = 0 | **结果：983** | |
| 1×2⁴ = 16 | | |
| 0×2³ = 0 | | |
| 1×2² = 4 | | |
| 1×2¹ = 2 | | |
| 1×2⁰ = 1 | | |
| **结果：983** | | |

### 8.3 二进制、八进制、十六进制互转

| **2进制** | **8进制** | **2进制** | **16进制** |
|-----------|-----------|-----------|------------|
| 000 | 0 | 0000 | 0 |
| 001 | 1 | 0001 | 1 |
| 010 | 2 | 0010 | 2 |
| 011 | 3 | 0011 | 3 |
| 100 | 4 | 0100 | 4 |
| 101 | 5 | 0101 | 5 |
| 110 | 6 | 0110 | 6 |
| 111 | 7 | 0111 | 7 |
| | | 1000 | 8 |
| | | 1001 | 9 |
| | | 1010 | A |
| | | 1011 | B |
| | | 1100 | C |
| | | 1101 | D |
| | | 1110 | E |
| | | 1111 | F |

**转换示例**：

- `0x3D7` 转 2 进制：`3→0011`, `D→1101`, `7→0111` → `0b001111010111`
- `01727` 转 2 进制：`1→001`, `7→111`, `2→010`, `7→111` → `0b1111010111`
- `0b1111010111` 转 16 进制：`0011→3`, `1101→D`, `0111→7` → `0x3D7`

## 10. 变量的生命周期与作用域

### 10.1 原则

1. **变量赋值遵循最近原则**
2. **变量赋值要注意生命周期**：一个变量的生命周期从 `{` 开始，从 `}` 结束
3. **在 `main` 外面的变量声明贯穿程序**：通过 `::变量名` 调用全局变量

### 10.2 代码块中的变量

- 代码块中的变量的生命从声明开始，直到这个代码块结束
- 声明在代码开始之前的变量叫做全局变量，全局变量的生命从程序运行开始，直到程序退出
- 在变量名冲突的情况下，采用就近原则
- 要访问名称冲突的全局变量，可以使用限定符 `::` 来访问

### 10.3 示例

```cpp
// 声明全局变量
int a{ 100 };

int main()
{
    using std::cout;

    int a{ 160 };

    {
        int a{ 350 };
        {
            char a = 'A';
            // ::变量名，调用全局变量
            cout << ::a << char(10);    // 打印 A，遵循最近原则
            cout << a << char(10);
        }

        // 打印 350，遵循生命周期
        cout << a;
    }
}
```
