## 示例代码
int sum(int a, int b) {
    int temp = 0;
    temp = a + b;
    return temp;
}

int main() {
    int a = 10;
    int b = 20;
    
    int ret = sum(a, b);
    cout << "ret:" << ret << endl;

# 3. 函数调用完整流程与栈帧演变

这一节把“汇编动作”和“栈空间变化”放到同一条时间线上来理解，这样更容易看清楚 `esp` 和 `ebp` 在每一步到底发生了什么。

> [!note]
> 这里的图示采用“上方 = 低地址，下方 = 高地址”的显示方式，所以页面上看起来像是“往上扩展”，但**地址本身仍然是从高地址向低地址变化**，这和“栈向下生长”并不矛盾。

### 3.1 调用前（在 `main` 中）

在调用 `sum(a, b)` 之前，`main` 自己已经有一套栈帧，此时的 `esp` 和 `ebp` 大致关系如下：

```
┌──────────────┐
│ 0xCCCCCCCC   │ ← esp (栈顶)
├──────────────┤
│ 0x00D18F140  │ ← ebp (main 的栈底)
├──────────────┤
│ ...          │
└──────────────┘
```

### 3.2 第一步：准备参数（从右向左）

遵循 `__cdecl` 调用约定，参数从右向左依次入栈。

代码段

```
int ret = sum(a, b);    ; 准备调用 sum
                        ; mov eax, dword ptr[ebp-8]  ; 取出 b 的值 (20)
                        ; push eax                   ; b 入栈
                        ; mov ecx, dword ptr[ebp-4]  ; 取出 a 的值 (10)
                        ; push ecx                   ; a 入栈
```

参数压栈后的栈状态如下：

```
┌──────────────┐
│ 10           │ ← esp (参数 a)
├──────────────┤
│ 20           │ (参数 b)
├──────────────┤
│ ...          │ ← ebp
└──────────────┘
```

### 3.3 第二步：执行 `call` 指令

代码段

```
call sum                ; 执行 call 指令
                        ; 核心动作:
                        ; 1. 将下一条指令的地址(即返回地址)压入栈中
                        ;    (以便函数执行完后能跳回来)
                        ; 2. 跳转到 sum 函数的入口地址
```

这一步的核心是：`call` 不只是“跳转”，它还会先把**返回地址**压栈，所以函数将来才能靠 `ret` 回到调用点。

### 3.4 第三步：保存旧 `ebp` 并创建新栈帧（序言）

进入 `sum` 函数后的头几行指令：

代码段

```
push ebp                ; 1. 将调用者(main)的 ebp 压栈保存
                        ;    (为了以后能恢复 main 的栈底)
mov ebp, esp            ; 2. 将当前栈顶(esp)赋值给 ebp
                        ;    (此时 ebp 指向当前函数的栈底)
sub esp, 4Ch            ; 3. 抬高栈顶，为局部变量开辟空间
```

**此时栈帧建立了三项关键内容（从下往上）：**

1. **Arguments**：参数（位于更高地址处）。
2. **Return Address**：返回地址（由 `call` 指令压入）。
3. **Saved EBP**：保存的旧栈底指针。

如果把局部变量空间也算进来，此时 `sum` 内部的完整栈帧大致如下：
![[1_1_1_2_1.png|503]]

### 3.5 第四步：执行函数体

代码段

```
int temp = 0           ; mov dword ptr[ebp-4], 0      ; 初始化 temp
temp = a + b           ; mov eax, dword ptr[ebp+8]    ; 取出参数 a (注意是 ebp+8)
                        ; add eax, dword ptr[ebp+0Ch] ; 加上参数 b (注意是 ebp+12)
                        ; mov dword ptr[ebp-4], eax   ; 结果存回 temp
return temp            ; mov eax, dword ptr[ebp-4]    ; 将返回值放入 eax 寄存器
```

这一阶段最关键的观察点是：

- 局部变量通过 `ebp - 偏移量` 访问。
- 参数通过 `ebp + 偏移量` 访问。
- 返回值通常通过 `eax` 带回调用者。

### 3.6 第五步：恢复栈顶（准备返回）

代码段

```
mov esp, ebp            ; 将 esp 重置回 ebp 的位置
                        ; 这一步瞬间释放了所有局部变量占用的栈空间
```

这一步执行完以后，局部变量区已经逻辑上被回收，`esp` 回到了保存旧 `ebp` 的位置附近。

### 3.7 第六步：恢复旧栈帧（尾声）

代码段

```
pop ebp                 ; 弹出栈顶的值给 ebp
                        ; ebp 变回了 main 函数时的值
                        ; 此时栈顶指向了“返回地址”
```

到这里，`sum` 自己的栈帧骨架已经拆掉了，只剩返回地址等待 `ret` 使用。

### 3.8 第七步：返回调用者

代码段

```
ret                     ; 弹出栈顶的“返回地址”给指令寄存器(EIP)
                        ; 程序跳转回 main 函数继续执行
```

# 4. 核心要点总结

1. **参数入栈顺序**：从右向左压栈（这是 **cdecl** 调用约定的标准）。
    
2. **返回地址**：由 `call` 指令自动压入栈中，紧挨着参数。
    
3. **旧 EBP 保存**：`push ebp` 用于保存上层函数的环境，形成“调用链”。
    
4. **局部变量寻址**：通过 `ebp` 减去偏移量访问（如 `ebp - 4`）。
    
5. **参数寻址**：通过 `ebp` 加上偏移量访问（如 `ebp + 8`，跳过旧 ebp 和返回地址）。
    
6. **返回值传递**：返回值通常存放在 **eax** 寄存器中带回。
    
7. **栈清理**：在 cdecl 约定中，由**调用者 (Caller)** 负责在函数返回后清理参数栈空间（`add esp, 8`）。
    

# 5. 内存布局简图

```
低地址 (Low Address)
├─────────────────
│ 局部变量 (Local Variables)
│ 保存的旧 ebp (Saved ebp)
│ 返回地址 (Return Address)
│ 参数 a (最左侧参数)
│ 参数 b (最右侧参数)
├─────────────────
高地址 (High Address)
```

# 6. 重要提示

- 栈是**向下生长**的（从高地址向低地址扩展）。本笔记为了直观展示 `esp` 与 `ebp` 的位置，统一采用“上低下高”的图示，所以视觉上会像“向上长”，但地址方向仍然是向下。详见[[1.3 进程的虚拟地址空间#5.3 堆和栈为什么相向生长？|堆栈相向生长的设计原因]]。
    
- **ebp** 非常重要，它提供了一个稳定的锚点来访问参数和变量，因为 **esp** 在函数运行过程中可能会频繁变动。
    
- 这种结构解释了为什么缓冲区溢出（覆盖局部变量）可能导致程序跳转到恶意地址（覆盖了下方的返回地址）。


# 7. 重点问题与避坑指南

### 7.1 为什么不要返回局部变量的地址？

笔记中特别框出了一个经典错误：

C++

```
int* func() {
    int data = 10;
    return &data; // 危险！
}
```

原因解析：

当 func 执行 mov esp, ebp 和 pop ebp 后，data 所在的栈内存就被“释放”了（标记为可覆盖）。虽然数据可能暂时还在，但下一次函数调用或者中断发生时，这块内存会被立即覆盖。在外部使用这个悬空指针会导致程序崩溃或逻辑错误。

### 7.2 缓冲区溢出原理

观察栈结构，局部变量上面紧挨着 旧EBP 和 返回地址。

如果我们在局部变量（比如一个数组）中写入了超过其长度的数据，数据就会向高地址“溢出”，覆盖掉 返回地址。

- **后果**：当函数执行 `ret` 时，会跳转到一个错误的地址（可能是黑客构造的恶意代码地址），这就是经典的栈溢出攻击。
    

### 7.3 汇编指令速查表

| **指令**         | **作用** | **伪代码描述**                   |
| -------------- | ------ | --------------------------- |
| `push X`       | 压栈     | `esp = esp - 4`; `*esp = X` |
| `pop X`        | 出栈     | `X = *esp`; `esp = esp + 4` |
| `call ADDR`    | 函数调用   | `push eip`; `eip = ADDR`    |
| `ret`          | 函数返回   | `pop eip`                   |
| `mov ebp, esp` | 建立帧    | `ebp = esp` (快照当前栈顶)        |
