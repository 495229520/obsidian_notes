# 9.2.1 函数重载 (Function Overloading)

## 1. 什么是函数重载

C++ 允许在**同一作用域**内定义多个**同名函数**，只要它们的**参数列表不同**即可。这种机制被称为函数重载。

> **核心思想**：相同的函数名，根据传入参数的不同，自动选择调用合适的版本。

编译器会根据函数调用时传递的实参来决定调用哪个重载版本，这个过程称为**重载决议 (Overload Resolution)**。

```cpp
#include <iostream>
#include <cstring>

// 重载版本 1：比较两个 int
int compare(int a, int b) {
    return (a > b) ? a : b;
}

// 重载版本 2：比较两个 double
double compare(double a, double b) {
    return (a > b) ? a : b;
}

// 重载版本 3：比较两个 C 字符串
bool compare(const char* a, const char* b) {
    return std::strcmp(a, b) > 0;
}

int main() {
    std::cout << compare(3, 5) << std::endl;           // 调用版本1，输出: 5
    std::cout << compare(3.14, 2.71) << std::endl;     // 调用版本2，输出: 3.14
    std::cout << compare("hello", "world") << std::endl; // 调用版本3
}
```

## 2. 函数重载的条件

函数重载必须满足以下条件：

| 条件 | 说明 | 示例 |
|------|------|------|
| **函数名相同** | 必须使用完全相同的函数名 | - |
| **参数个数不同** | 参数数量不同 | `void func(int)` vs `void func(int, int)` |
| **参数类型不同** | 参数类型不同 | `void func(int)` vs `void func(double)` |
| **参数顺序不同** | 类型顺序不同 | `void func(int, double)` vs `void func(double, int)` |
| **同一作用域** | 必须在同一个作用域内声明 | - |

### 不构成重载的情况

**仅返回值不同**不能构成函数重载：

```cpp
int func(int a);      // 声明1
double func(int a);   // 错误！仅返回值不同，不构成重载
```

原因：编译器在[[1.1 程序编译与链接原理#1. 编译全过程 (The Build Process)|编译阶段]]进行函数匹配时，只看函数名和参数列表，不考虑返回值。

## 3. const 修饰符与函数重载

### 3.1 顶层 const vs 底层 const

| 类型 | 说明 | 示例 |
|------|------|------|
| **顶层 const** | 修饰变量本身，表示变量不可修改 | `const int x`、`int* const p` |
| **底层 const** | 修饰指针/引用所指向的内容 | `const int*`、`const int&` |

### 3.2 重载规则

- **顶层 const 不构成重载**：因为传值时会忽略顶层 const
- **底层 const 构成重载**：因为它影响了参数的语义

```cpp
// 顶层 const - 不构成重载
void func(int x);        // 版本1
void func(const int x);  // 错误！与版本1冲突（顶层const被忽略）

// 底层 const - 构成重载
void func(int* p);        // 版本1：接受非const指针
void func(const int* p);  // 版本2：接受指向const的指针 ✓

void func(int& r);        // 版本3：接受非const引用
void func(const int& r);  // 版本4：接受const引用 ✓
```

### 3.3 const 引用的特殊行为

`const` 引用可以绑定到临时对象（右值），而非 `const` 引用不行：

```cpp
void process(int& x);        // 只接受左值
void process(const int& x);  // 可以接受左值和右值

int main() {
    int a = 10;
    process(a);      // 调用 process(int&)
    process(10);     // 调用 process(const int&)，10是右值
    process(a + 5);  // 调用 process(const int&)，表达式结果是右值
}
```

### 3.4 成员函数的 const 重载

对于成员函数，`const` 修饰符可以用于重载：

```cpp
class Container {
public:
    // 非 const 版本：允许修改
    int& at(size_t index) {
        return data_[index];
    }

    // const 版本：只读访问
    const int& at(size_t index) const {
        return data_[index];
    }

private:
    std::vector<int> data_;
};

Container c;
const Container& cc = c;

c.at(0) = 10;      // 调用非 const 版本，可以修改
int val = cc.at(0); // 调用 const 版本，只读
```

## 4. 引用类型重载

C++11 后，可以根据左值引用和右值引用进行重载：

```cpp
void process(int& x) {
    std::cout << "左值引用版本" << std::endl;
}

void process(int&& x) {
    std::cout << "右值引用版本" << std::endl;
}

int main() {
    int a = 10;
    process(a);             // 调用左值引用版本
    process(10);            // 调用右值引用版本
    process(std::move(a));  // 调用右值引用版本
}
```

这是实现**移动语义**的基础，详见 [[9.1.4 引用]]。

## 5. 重载决议 (Overload Resolution)

当调用重载函数时，编译器按以下优先级选择最佳匹配：

### 5.1 匹配优先级

| 优先级 | 匹配类型 | 说明 | 示例 |
|--------|---------|------|------|
| 1 | **精确匹配** | 类型完全相同，或仅需数组→指针、函数→指针转换 | `int` 匹配 `int` |
| 2 | **const 转换** | 非 const 到 const 的转换 | `int*` → `const int*` |
| 3 | **类型提升** | 小类型到大类型的安全提升 | `char`→`int`、`float`→`double` |
| 4 | **标准转换** | 其他内置类型转换 | `int`→`double`、指针转换 |
| 5 | **用户定义转换** | 通过构造函数或转换运算符 | 类类型转换 |

### 5.2 示例分析

```cpp
void func(int x);           // 版本1
void func(double x);        // 版本2
void func(int* p);          // 版本3
void func(const int* p);    // 版本4

int main() {
    int a = 10;
    double b = 3.14;

    func(a);        // 精确匹配 → 版本1
    func(b);        // 精确匹配 → 版本2
    func('c');      // 类型提升 char→int → 版本1
    func(3.14f);    // 类型提升 float→double → 版本2

    int* ptr = &a;
    const int* cptr = &a;
    func(ptr);      // 精确匹配 → 版本3
    func(cptr);     // 精确匹配 → 版本4
}
```

### 5.3 二义性错误

当存在多个同等优先级的匹配时，编译器会报二义性错误：

```cpp
void func(int a, double b);
void func(double a, int b);

func(1, 1);  // 错误！二义性：两个版本同样匹配
```

**解决方法**：使用显式类型转换

```cpp
func(1, static_cast<double>(1));  // 明确调用第一个版本
func(static_cast<double>(1), 1);  // 明确调用第二个版本
```

## 6. 默认参数与函数重载

默认参数可能与重载产生冲突，需要谨慎使用：

```cpp
void print(int a, int b = 10);
void print(int a);

print(5);  // 错误！二义性：两个版本都可以匹配
```

**最佳实践**：避免默认参数和重载函数的参数数量形成重叠。

## 7. 名称修饰 (Name Mangling)

C++ 编译器通过**名称修饰**技术来区分重载函数。编译器会将函数名和参数类型信息编码成唯一的符号名。

### 7.1 名称修饰示例

```cpp
int func(int a);           // GCC: _Z4funci
int func(double a);        // GCC: _Z4funcd
int func(int a, int b);    // GCC: _Z4funcii
```

> **注意**：不同编译器的名称修饰规则不同（如 MSVC vs GCC）。

### 7.2 为什么 C 不支持重载

C 编译器不进行名称修饰，函数名就是[[1.1 程序编译与链接原理#3.2 符号表 (Symbol Table)|符号表]]中的唯一标识。因此 C 语言无法区分同名函数。

### 7.3 extern "C" 与重载

使用 `extern "C"` 声明的函数不能重载，因为它禁用了名称修饰：

```cpp
extern "C" {
    void cfunc(int x);      // OK
    void cfunc(double x);   // 错误！C 链接不支持重载
}
```

`extern "C"` 主要用于 C++ 代码调用 C 库，或让 C 代码调用 C++ 函数。

## 8. 继承中的函数隐藏

派生类中定义同名函数会**隐藏**基类的所有同名重载版本：

```cpp
class Base {
public:
    void func(int x) { std::cout << "Base::func(int)" << std::endl; }
    void func(double x) { std::cout << "Base::func(double)" << std::endl; }
};

class Derived : public Base {
public:
    void func(int x) { std::cout << "Derived::func(int)" << std::endl; }
    // 注意：Base::func(double) 被隐藏了！
};

int main() {
    Derived d;
    d.func(10);     // OK：调用 Derived::func(int)
    d.func(3.14);   // 调用 Derived::func(int)，double 被隐式转换！
    // d.Base::func(3.14);  // 显式调用基类版本
}
```

**解决方法**：使用 `using` 声明引入基类函数

```cpp
class Derived : public Base {
public:
    using Base::func;  // 引入基类所有 func 重载
    void func(int x) { std::cout << "Derived::func(int)" << std::endl; }
};
```

## 9. 函数重载 vs 函数模板

| 特性 | 函数重载 | 函数模板 |
|------|----------|----------|
| **代码复用** | 每个版本需单独编写 | 一份代码处理多种类型 |
| **灵活性** | 可针对特定类型优化 | 通用但难以特殊处理 |
| **编译时间** | 较短 | 较长（模板实例化） |
| **适用场景** | 不同类型有不同逻辑 | 相同逻辑处理不同类型 |

```cpp
// 函数重载：不同类型有不同处理逻辑
void print(int x) {
    std::cout << "Integer: " << x << std::endl;
}

void print(const std::string& s) {
    std::cout << "String: " << s << std::endl;
}

// 函数模板：相同逻辑处理不同类型
template<typename T>
void display(const T& x) {
    std::cout << "Value: " << x << std::endl;
}
```

详见 [[9.2.2 函数模版]] 和 [[9.2.3 重载函数模版]]。

## 10. 运算符重载

运算符重载是函数重载的特殊形式，C++ 允许重载大多数运算符：

```cpp
class Vector {
public:
    int x, y;

    Vector(int x = 0, int y = 0) : x(x), y(y) {}

    // 重载 + 运算符（成员函数形式）
    Vector operator+(const Vector& other) const {
        return {x + other.x, y + other.y};
    }

    // 重载 == 运算符
    bool operator==(const Vector& other) const {
        return x == other.x && y == other.y;
    }
};

// 重载 << 运算符（必须是非成员函数）
std::ostream& operator<<(std::ostream& os, const Vector& v) {
    return os << "(" << v.x << ", " << v.y << ")";
}

int main() {
    Vector a{1, 2}, b{3, 4};
    Vector c = a + b;  // 调用 operator+
    std::cout << c << std::endl;  // 输出: (4, 6)
}
```

## 11. 实际应用场景

### 11.1 统一接口

标准库中的 `std::to_string` 就是函数重载的典型应用：

```cpp
std::string s1 = std::to_string(42);      // int 版本
std::string s2 = std::to_string(3.14);    // double 版本
std::string s3 = std::to_string(123L);    // long 版本
```

### 11.2 构造函数重载

类的构造函数是函数重载的常见应用：

```cpp
class Date {
public:
    Date() : year_(1970), month_(1), day_(1) {}           // 默认构造
    Date(int year, int month, int day)                     // 三参数构造
        : year_(year), month_(month), day_(day) {}
    Date(const std::string& dateStr) { /* 从字符串解析 */ } // 字符串构造

private:
    int year_, month_, day_;
};
```

## 12. 关键要点

| 要点 | 说明 |
|------|------|
| **重载条件** | 同名、同作用域、参数列表不同 |
| **返回值** | 仅返回值不同不构成重载 |
| **const 规则** | 底层 const 构成重载，顶层 const 不构成 |
| **匹配优先级** | 精确匹配 > const转换 > 类型提升 > 标准转换 > 用户定义转换 |
| **名称修饰** | C++ 通过编码函数签名实现重载，C 不支持 |
| **继承隐藏** | 派生类同名函数会隐藏基类所有重载，用 `using` 解决 |
| **extern "C"** | C 链接的函数不能重载 |
