# 9.2.5 auto 类型推导

## 1. 简介

将[[9.2.2 函数模版|函数模板]]与 `decltype` 结合使用，可以解决一些仅使用函数模板无法很好解决的问题，主要体现在**泛型编程**和**完美转发**方面。

---

## 2. 推导返回类型的问题

### 2.1 问题描述

在使用函数模板的时候如果传入的多个参数类型并不相同，那么就需要使用[[9.2.4 特化函数模版|特化]]，这样确认返回的类型就变得十分的困难，稍有不慎就会导致**大类型转换为小类型导致数据丢失**。

```cpp
// 问题：当 T1 和 T2 类型不同时，返回类型该如何确定？
template <typename T>
auto add(T x, T y) {
    return (x + y) / 2;
}

int main() {
    int a = 10;
    float b = 10.5;
    add(10, 10.5f);  // 错误！T 无法同时推导为 int 和 float
    return 0;
}
```

### 2.2 使用两个类型参数

```cpp
template <typename T1, typename T2>
T1 add(T1 x, T2 y) {
    return (x + y) / 2;
}

int main() {
    int a = 10;
    float b = 10.5;
    add(10, 10.5f);  // 返回 T1 = int，结果会丢失精度
    return 0;
}
```

**问题**：这种解决办法会导致精度的损失，同时也失去了函数模版的便捷性。

---

## 3. 解决方案

### 3.1 显示指定返回类型

```cpp
// 普通模版
template <typename T1, typename T2>
T1 swapValues(T1 a, T2 b) {
    return (a + b) / 2;
}

// 显示指定返回类型模版
template <typename TR, typename T1, typename T2>
TR swapValues(T1 a, T2 b) {
    return (a + b) / 2;
}

template <typename T1, typename T2>
T1 bigger(T1 a, T2 b) {
    return a > b ? a : b;
}

int main() {
    char a = 50;
    int b = 5000000;

    // 没有显示指定导致精度损失
    std::cout << swapValues(100, 100.5f) << std::endl;  // 100

    // 显示指定返回类型是 double
    std::cout << swapValues<double>(100, 100.5f) << std::endl;  // 100.25

    // 没有显示指定导致大类型转换小类型，数据溢出
    std::cout << bigger(a, b) << std::endl;  // @ (错误结果)

    return 0;
}
```

### 3.2 使用 `auto` + `decltype` 后置返回类型

```cpp
template <typename T1, typename T2>
auto bigger(T1 a, T2 b) -> decltype(a > b ? a : b) {
    return a > b ? a : b;
}

int main() {
    char a = 50;
    int b = 5000000;
    std::cout << bigger(a, b) << std::endl;  // 5000000（正确）
    return 0;
}
```

**说明**：使用 `auto -> decltype` 函数返回类型自动推导。

### 3.3 C++14 简化写法

```cpp
template <typename T1, typename T2>
decltype(auto) bigger(T1& a, T2& b) {
    return a > b ? a : b;
}

int main() {
    int a = 50;
    int b = 5000000;

    bigger(a, b) = 200;  // 可以修改返回的引用
    std::cout << bigger(a, b) << std::endl;
    return 0;
}
```

## 4. `auto` 关键字

### 4.1 基本用法

`auto` 让编译器根据初始化表达式自动推导变量类型：

```cpp
auto x = 10;        // int
auto y = 3.14;      // double
auto z = "hello";   // const char*

auto ptr = new int(5);  // int*
```

### 4.2 注意事项

| 规则 | 说明 |
|------|------|
| 必须初始化 | `auto x;` 是错误的 |
| 忽略顶层 const | `const int a = 10; auto b = a;` → `b` 是 `int` |
| 保留底层 const | `const int* p; auto q = p;` → `q` 是 `const int*` |
| 引用需显式声明 | `auto& r = x;` 才是引用 |

---

## 5. `decltype` 关键字

### 5.1 基本用法

`decltype` 推导表达式的类型，**不会实际计算表达式**：

```cpp
int x = 10;
decltype(x) y = 20;      // y 的类型是 int

int& rx = x;
decltype(rx) ry = x;     // ry 的类型是 int&

decltype(x + 1.0) z;     // z 的类型是 double
```

### 5.2 `decltype` vs `auto`

| 特性 | `auto` | `decltype` |
|------|--------|------------|
| 推导时机 | 根据初始值 | 根据表达式 |
| 保留引用 | 不保留 | 保留 |
| 保留 const | 部分保留 | 完全保留 |
| 是否需要初始化 | 是 | 否 |

```cpp
int x = 10;
int& rx = x;

auto a = rx;       // a 是 int（丢失引用）
decltype(rx) b = x;  // b 是 int&（保留引用）
```

## 6. 后置返回类型 (Trailing Return Type)

### 语法

```cpp
auto 函数名(参数列表) -> 返回类型 {
    // 函数体
}
```

### 应用场景

当返回类型依赖于参数类型时特别有用：

```cpp
// 传统写法（无法编译，a 和 b 在返回类型位置还未声明）
// decltype(a + b) add(T1 a, T2 b);  // 错误！

// 后置返回类型
template <typename T1, typename T2>
auto add(T1 a, T2 b) -> decltype(a + b) {
    return a + b;
}
```

---

## 7. `decltype(auto)` (C++14)

`decltype(auto)` 结合了两者的优点：

- 像 `auto` 一样自动推导
- 像 `decltype` 一样保留引用和 const

```cpp
int x = 10;
int& rx = x;

auto a = rx;            // int（丢失引用）
decltype(auto) b = rx;  // int&（保留引用）

// 用于函数返回
template <typename T1, typename T2>
decltype(auto) bigger(T1& a, T2& b) {
    return a > b ? a : b;  // 返回引用
}
```

---

## 8. 关键要点

1. **`auto`**：根据初始值推导类型，忽略顶层 const 和引用
2. **`decltype`**：根据表达式推导类型，保留所有类型信息
3. **后置返回类型**：解决返回类型依赖参数的问题
4. **`decltype(auto)`**：C++14 结合两者优点的简化写法
5. **精度问题**：不同类型运算时需注意返回类型的选择
