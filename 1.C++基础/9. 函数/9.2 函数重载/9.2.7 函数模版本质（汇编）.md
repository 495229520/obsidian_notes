# 9.2.7 函数模板本质（汇编）

## 1. 模板实例化的本质

[[9.2.2 函数模版|函数模板]]本身**不是函数**，而是编译器用来生成函数的"蓝图"。当我们使用不同的类型调用模板时，编译器会为每种类型生成一个**独立的函数实例**。

---

## 2. 代码示例

```cpp
template <typename T>
T ave(T a, T b) {
    return a + b;
}

int main() {
    ave(100, 200);                       // 调用 ave<int>
    ave(short(100), short(200));         // 调用 ave<short>
    ave(short(300), short(400));         // 调用 ave<short>（复用已生成的）
    return 0;
}
```

---

## 3. 汇编分析

### 调用代码的汇编

```asm

00007FF6F7651AA0  mov     ecx, 64h            ; 第一个参数 100

00007FF6F7651AAE  mov     cx, 64h             ; 第一个参数 100 (short)
00007FF6F7651AB2  call    ave<short> (07FF6F765153Ch)  ; 调用 ave<short>

    ave(short(300), short(400));
00007FF6F7651AB7  mov     dx, 190h            ; 第二个参数 400 (short)
00007FF6F7651ABB  mov     cx, 12Ch            ; 第一个参数 300 (short)
```

### 关键观察

| 调用 | 生成的函数 | 函数地址 |
|------|-----------|---------|
| `ave(100, 200)` | `ave<int>` | `07FF6F7651537h` |
| `ave(short(100), short(200))` | `ave<short>` | `07FF6F765153Ch` |
| `ave(short(300), short(400))` | `ave<short>` | `07FF6F765153Ch` |

> **唯一的区别就是 call，同样的函数调用的内存地址是不一样的**

---

## 4. 结论

### 4.1 不同类型生成不同函数

当传入参数的类型不同时，编译器会**生成两个不同的函数**。这个就是代价，会造成内存的浪费。

```
ave<int>   → 地址 07FF6F7651537h
ave<short> → 地址 07FF6F765153Ch
```

### 4.2 相同类型复用同一函数

但是当传入参数的类型相同时，会生成相同的函数，无论参数的数值如何变化。

```
ave(short(100), short(200)) → 调用 07FF6F765153Ch
ave(short(300), short(400)) → 调用 07FF6F765153Ch  // 同一个函数
```

**结论**：函数模版会根据传入参数的类型生成不同的函数，当生成的函数可以解决后面传入的参数时就不会再次生成另一个函数了。

## 5. 非类型参数的影响

[[9.2.6 函数模版参数#2. 非类型模板参数 (Non-type Template Parameters)|非类型模板参数]]也会影响函数的生成：

```cpp
template <typename T, T max = 2000, T min = 1000>
bool AddHp(T& hp, T damage) {
    hp -= damage;
    if (hp > max) {
        hp = max;
    }
    return hp < min;
}

int main() {
    int hp = 500;
    AddHp(hp, 200);                    // 使用默认参数 max=2000, min=1000
    AddHp<int, 200, 300>(hp, 200);     // 显式指定 max=200, min=300
    return 0;
}
```

### 汇编分析

```asm
    AddHp(hp, 200);
00007FF717371C54  mov     edx, 0C8h
00007FF717371C59  lea     rcx, [hp]
00007FF717371C5D  call    AddHp<int, 2000, 1000> (07FF717371546h)

    AddHp<int, 200, 300>(hp, 200);
00007FF717371C62  mov     edx, 0C8h
00007FF717371C67  lea     rcx, [hp]
00007FF717371C6B  call    AddHp<int, 200, 300> (07FF7173714Bh)
```

> 这里函数参数和参数都是一样的但是为什么生成的函数地址不一样？因为前一个生成的函数无法解决后面的参数，**同一个函数模版不同的非类型参数最后生成的函数是不一样的**。

---

## 6. 内存代价与优化

### 代价

- 每种类型组合都会生成一份独立的函数代码
- 可能导致**代码膨胀 (Code Bloat)**

### 优化

- 编译器会尝试合并相同的实例化
- 链接器可以移除重复的模板实例（COMDAT 折叠）
- 使用显式实例化控制生成的版本

---

## 7. 关键要点

1. **模板不是函数**：它是生成函数的蓝图
2. **类型决定实例**：不同类型参数生成不同的函数
3. **复用机制**：相同类型参数复用已生成的函数
4. **非类型参数**：不同的非类型参数值也会生成不同的函数
5. **内存代价**：模板可能导致代码膨胀，但编译器/链接器会优化
