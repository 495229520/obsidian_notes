# 9.2.2 函数模板 (Function Templates)

## 1. 为什么需要函数模板

在没有模板的情况下，如果我们需要对不同类型执行相同的操作，就必须为每种类型编写[[9.2.1 函数重载|重载函数]]：

```cpp
int max(int a, int b) { return a > b ? a : b; }
double max(double a, double b) { return a > b ? a : b; }
float max(float a, float b) { return a > b ? a : b; }
// ... 每种类型都要写一遍
```

**函数模板**允许我们编写一个通用的函数定义，让编译器根据调用时的参数类型自动生成对应的函数版本。

---

## 2. 函数模板的语法

### 基本语法

```cpp
template <typename T>  // 或 template <class T>
返回类型 函数名(参数列表) {
    // 函数体
}
```

### 关键字说明

| 关键字 | 说明 |
|--------|------|
| `template` | 声明这是一个模板 |
| `typename` / `class` | 声明类型参数（两者在此处等价） |
| `T` | 类型参数名（可以是任意合法标识符） |

### 示例

```cpp
template <typename T>
T bigger(T a, T b) {
    return a > b ? a : b;
}

int main() {
    int x = bigger(10, 20);           // T 被推导为 int
    double y = bigger(3.14, 2.71);    // T 被推导为 double
    char c = bigger('a', 'z');        // T 被推导为 char
}
```

---

## 3. 模板实例化 (Template Instantiation)

### 3.1 隐式实例化

编译器根据函数调用时的参数类型**自动推导**模板参数：

```cpp
template <typename T>
T add(T a, T b) {
    return a + b;
}

int main() {
    add(1, 2);       // 隐式实例化 add<int>
    add(1.5, 2.5);   // 隐式实例化 add<double>
}
```

### 3.2 显式实例化

手动指定模板参数类型：

```cpp
int main() {
    add<int>(1, 2);           // 显式指定 T = int
    add<double>(1, 2);        // 显式指定 T = double（参数会被转换）
    add<float>(1.5f, 2.5f);   // 显式指定 T = float
}
```

### 3.3 实例化的本质

函数模板本身**不是函数**，而是生成函数的"蓝图"。只有在模板被使用时，编译器才会根据具体类型生成真正的函数代码（详见[[9.2.7 函数模版本质（汇编）]]）。

```cpp
// 模板定义（不生成代码）
template <typename T>
T square(T x) { return x * x; }

// 使用时生成代码
square(5);      // 生成 int square(int)
square(3.14);   // 生成 double square(double)
```

---

## 4. 多模板参数

模板可以有多个类型参数：

```cpp
template <typename T1, typename T2>
auto add(T1 a, T2 b) -> decltype(a + b) {
    return a + b;
}

int main() {
    auto result = add(10, 3.14);  // T1=int, T2=double, 返回 double
}
```

---

## 5. 类型推导规则

### 5.1 基本推导

编译器根据实参类型推导模板参数：

```cpp
template <typename T>
void func(T x);

func(42);       // T = int
func(3.14);     // T = double
func("hello");  // T = const char*
```

### 5.2 引用和 const 的处理

```cpp
template <typename T>
void func(T& x);      // 引用参数

int a = 10;
const int b = 20;

func(a);   // T = int
func(b);   // T = const int（保留 const）
```

### 5.3 类型不一致的问题

当两个参数类型不同时，编译器无法推导：

```cpp
template <typename T>
T max(T a, T b) { return a > b ? a : b; }

max(1, 3.14);  // 错误！T 无法同时是 int 和 double
```

**解决方案**：

```cpp
// 方案1：显式指定类型
max<double>(1, 3.14);

// 方案2：使用多个模板参数
template <typename T1, typename T2>
auto max(T1 a, T2 b) -> decltype(a > b ? a : b);
```

---

## 6. 模板参数的默认值

C++11 起，函数模板也可以有默认模板参数：

```cpp
template <typename T = int>
T getValue() {
    return T();
}

int main() {
    auto a = getValue();        // 使用默认类型 int
    auto b = getValue<double>();  // 显式指定 double
}
```

---

## 7. 函数模板与普通函数的选择

当同时存在函数模板和[[9.2.1 函数重载|普通重载函数]]时，编译器优先选择**精确匹配的普通函数**：

```cpp
template <typename T>
void print(T x) {
    std::cout << "Template: " << x << std::endl;
}

void print(int x) {
    std::cout << "Non-template: " << x << std::endl;
}

int main() {
    print(10);     // 调用普通函数（精确匹配优先）
    print(3.14);   // 调用模板函数
    print<>(10);   // 强制调用模板函数
}
```

---

## 8. 模板的编译模型

函数模板通常需要在**头文件中定义**（而不仅仅是声明），因为：

1. 模板在[[1.1 程序编译与链接原理#1. 编译全过程 (The Build Process)|编译阶段]]实例化，编译器需要看到完整定义
2. 每个使用模板的翻译单元都需要能够生成实例化代码

```cpp
// math_utils.h
template <typename T>
T square(T x) {
    return x * x;  // 定义必须在头文件中
}
```

---

## 9. 关键要点

1. **模板不是函数**：它是生成函数的蓝图，使用时才实例化
2. **类型推导**：编译器根据实参自动推导模板参数类型
3. **显式实例化**：可以手动指定模板参数类型
4. **头文件定义**：模板通常需要在头文件中完整定义
5. **优先级**：精确匹配的普通函数优先于模板函数
