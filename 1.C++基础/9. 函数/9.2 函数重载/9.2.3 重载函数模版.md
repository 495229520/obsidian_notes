# 9.2.3 重载函数模板 (Overloading Function Templates)

## 1. 函数模板的重载

与[[9.2.1 函数重载|普通函数重载]]类似，[[9.2.2 函数模版|函数模板]]也可以被重载。我们可以定义多个同名的函数模板，只要它们的**参数列表不同**即可。

```cpp
#include <iostream>
#include <vector>
#include <string>

// 第一个函数模板：接受一个参数，打印其值
template <typename T>
void print(T value) {
    std::cout << "Single value: " << value << std::endl;
}

// 第二个函数模板：接受两个相同类型的参数，打印它们的和
template <typename T>
void print(T a, T b) {
    std::cout << "Sum of two values: " << a + b << std::endl;
}

// 第三个函数模板：接受一个 vector 并打印其元素
template <typename T>
void print(const std::vector<T>& vec) {
    std::cout << "Vector elements: ";
    for (const T& element : vec) {
        std::cout << element << " ";
    }
    std::cout << std::endl;
}
```

---

## 2. 使用示例

```cpp
int main() {
    int num = 10;
    double pi = 3.14;
    std::string text = "Hello";
    std::vector<int> numbers = {1, 2, 3, 4, 5};

    print(num);             // 调用第一个模板：print<int>(int)
    print(pi, pi);          // 调用第二个模板：print<double>(double, double)
    print(text);            // 调用第一个模板：print<std::string>(std::string)
    print(10, 20);          // 调用第二个模板：print<int>(int, int)
    print(numbers);         // 调用第三个模板：print<int>(const std::vector<int>&)

    return 0;
}
```

---

## 3. 重载解析

在调用重载的函数模板时，编译器会根据我们提供的**参数类型和数量**来选择最合适的函数模板进行实例化：

| 调用 | 匹配的模板 | 推导的类型 |
|------|-----------|-----------|
| `print(num)` | 匹配第一个模板 | `T` 被推导为 `int` |
| `print(pi, pi)` | 匹配第二个模板 | `T` 被推导为 `double` |
| `print(text)` | 匹配第一个模板 | `T` 被推导为 `std::string` |
| `print(10, 20)` | 匹配第二个模板 | `T` 被推导为 `int` |
| `print(numbers)` | 匹配第三个模板 | `T` 被推导为 `int` |

## 4. 函数模板与普通函数的重载

函数模板可以与**普通函数**共存并构成重载关系。

```cpp
// 通用模板
template <typename T>
T bigger(T a, T b) {
    std::cout << "template <typename T> int* bigger" << std::endl;
    return a > b ? a : b;
}

// 特化模板（比较传入的引用指向值的大小并返回最大值的指针）
template <>
int* bigger(int* a, int* b) {
    std::cout << "template <> int* bigger" << std::endl;
    return *a > *b ? a : b;
}

// 普通函数重载
int* bigger(int* a, int* b) {
    std::cout << "int* bigger" << std::endl;
    return *a > *b ? a : b;
}
```

### 调用示例

```cpp
int main() {
    int x = 10;
    int y = 20;
    int result = *bigger(&x, &y);
    std::cout << result << std::endl;
}
```

### 输出

```
int* bigger
20
```

## 5. 匹配优先级

> **完全匹配的普通函数优先级最高**

当同时存在模板和普通函数时，编译器按以下顺序选择：

| 优先级 | 类型 | 说明 |
|--------|------|------|
| 1 | **完全匹配的普通函数** | 参数类型完全相同的非模板函数 |
| 2 | **完全匹配的函数模板[[9.2.4 特化函数模版\|特化]]版本** | 针对特定类型的模板特化 |
| 3 | **可以通过模板参数推导实例化的函数模板** | 通用模板 |
| 4 | **可以通过类型转换匹配的普通函数** | 需要隐式转换的非模板函数 |

---

## 6. 关键要点

- 函数模板的重载与普通函数的重载类似，都是通过参数列表的不同来区分
- 编译器在选择要实例化的函数模板时，会尝试找到一个与函数调用参数类型和数量**最匹配**的模板
- 可以基于参数的**数量**、**类型**或者参数是否为**模板类型**（如 `std::vector`）进行函数模板的重载
