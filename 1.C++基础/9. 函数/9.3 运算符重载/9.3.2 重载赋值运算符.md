# 9.3.2 重载赋值运算符 (Assignment Operator)

## 1. 简介

赋值运算符 `=` 用于将一个对象的值赋给另一个**已存在**的对象。与拷贝构造函数不同，赋值运算符在对象已经被创建后调用。

---

## 2. 语法

```cpp
class MyClass {
public:
    // 返回类型通常是类型的一个新对象
    // 参数是右操作数
    // 参数是否使用引用，如需要避免拷贝可以使用引用
    MyClass& operator=(const MyClass& other);
};
```

---

## 3. 完整示例：hstring 类

```cpp
class hstring {
private:
    unsigned short uslen;           // 字符串当前长度
    char* cstr;                     // 指向动态内存
    unsigned short nMaxStrLen;      // 分配内存大小
    char cflag[3];                  // 特殊标志
    hstring(unsigned short usmaxlen);  // 一定要以为置

public:
    hstring();
    ~hstring();
    hstring(const hstring& hstr);   // 拷贝构造函数
    hstring& operator=(const hstring& hstr);  // 赋值运算符

    unsigned short length();
};
```

### 赋值运算符实现

```cpp
hstring& hstring::operator=(const hstring& hstr) {
    // 1. 自赋值检查
    if (this == &hstr) {
        return *this;
    }

    // 2. 释放原有资源
    delete[] cstr;

    // 3. 分配新内存
    uslen = hstr.uslen;
    nMaxStrLen = hstr.nMaxStrLen;
    cstr = new char[nMaxStrLen];

    // 4. 复制数据
    memcpy(cstr, hstr.cstr, uslen + 1);

    // 5. 返回当前对象引用（支持链式赋值）
    return *this;
}
```

---

## 4. 关键实现要点

### 4.1 自赋值检查

```cpp
if (this == &hstr) {
    return *this;
}
```

防止 `a = a` 这种情况导致的问题。如果不检查，先 `delete` 再 `new` 会导致读取已释放的内存。

### 4.2 释放原有资源

```cpp
delete[] cstr;
```

赋值前必须释放当前对象持有的动态资源，否则会造成内存泄漏。

### 4.3 深拷贝

```cpp
cstr = new char[nMaxStrLen];
memcpy(cstr, hstr.cstr, uslen + 1);
```

重新分配内存并复制内容，而不是简单地复制指针（浅拷贝）。

### 4.4 返回引用

```cpp
return *this;
```

返回当前对象的引用以支持**链式赋值**：

```cpp
a = b = c;  // 等价于 a = (b = c)
```

---

## 5. 深拷贝 vs 浅拷贝

| 类型 | 说明 | 问题 |
|------|------|------|
| **浅拷贝** | 只复制指针值，两个对象指向同一块内存 | 析构时重复释放（double free） |
| **深拷贝** | 分配新内存并复制内容 | 安全但开销较大 |

```cpp
// 浅拷贝（错误）
cstr = hstr.cstr;  // 两个对象指向同一内存！

// 深拷贝（正确）
cstr = new char[nMaxStrLen];
memcpy(cstr, hstr.cstr, uslen + 1);
```

---

## 6. 赋值运算符 vs 拷贝构造函数

| 特性 | 拷贝构造函数 | 赋值运算符 |
|------|-------------|-----------|
| **调用时机** | 创建新对象时 | 对象已存在时 |
| **语法** | `MyClass(const MyClass&)` | `MyClass& operator=(const MyClass&)` |
| **需要释放资源** | 否（新对象无资源） | 是（覆盖旧资源） |
| **自赋值检查** | 不需要 | 需要 |

```cpp
hstring a;
hstring b = a;   // 拷贝构造函数（初始化）
hstring c;
c = a;           // 赋值运算符（赋值）
```

---

## 7. 编译器默认生成

如果不自定义赋值运算符，编译器会生成默认版本，执行**逐成员浅拷贝**。对于包含指针成员的类，必须自定义以实现深拷贝。

---

## 8. 关键要点

1. **自赋值检查**：`if (this == &other) return *this;`
2. **释放原资源**：避免内存泄漏
3. **深拷贝**：重新分配内存并复制内容
4. **返回引用**：支持链式赋值 `a = b = c`
5. **与拷贝构造的区别**：赋值运算符用于已存在的对象
