# 9.3.1 运算符重载 (Operator Overloading)

## 1. 简介

**运算符重载**的概念是指允许你为**用户自定义的数据类型**赋予 C++ 内置运算符（如 `+`、`-`、`*`、`/`、`==`、`<<`、`>>` 等）**新的含义**。

它的核心思想是：

1. **扩展现有符号的功能**：你可以让你定义的类对象也能像内置类型（如 `int` 或 `double`）一样，直接使用这些运算符与运算
2. **增强代码的可读性和直观性**：通过使用符合习惯的运算符来表达类的操作，使代码更接近于自然语言或数学表达式，提高了代码的易于理解性

> 本质上，运算符重载是一种语法糖 (syntactic sugar)，它并没有引入新的运算符，也没有改变运算符的优先级、结合性或操作数个数。它只是允许你为现有的运算符号定义在你的类对象上的具体行为，使得代码看起来更自然，更富有表现力。

---

## 2. 实现方式

运算符重载可以通过两种方式实现：

| 实现方式 | 说明 |
|---------|------|
| **成员函数** | 作为类的成员函数重载，左操作数是调用该函数的对象（即 `*this`） |
| **友元函数** | 作为类的[[9.2.1 函数重载\|友元函数]]重载，通常需要访问类的私有成员 |

---

## 3. 成员函数实现

### 语法

```cpp
class MyClass {
public:
    // 返回类型通常是类型的一个新对象
    // 参数是右操作数
    MyClass operator+(const MyClass& other) const;
};
```

### 示例

```cpp
class Person {
    // 在私有作用域中使用友元类
    friend bool operator<(Person& psa, Person& psb);
    unsigned short Age;
public:
    Person(unsigned short _Age) : Age(_Age) {}
};

// Man<Woman == operator<(Man, Woman)
bool operator<(Person& psa, Person& psb) {
    return psa.Age < psb.Age;
}

int main() {
    Person Man(20);
    Person Woman(50);
    if (Man < Woman)
        std::cout << "找到富婆";
}
```

### `operator` 有什么用？

```cpp
// 以下两种写法等价
if (operator<(Man, Woman))
    std::cout << "找到富婆";

if (Man < Woman)
    std::cout << "找到富婆";
```

上面的 `Man<Woman` 等价于 `operator<(Man, Woman)`，这样实现了运算符的重载。

## 4. 友元函数实现

```cpp
class Person {
    // 在私有作用域中使用友元类
    friend bool operator<(Person& psa, Person& psb);
    friend bool operator<(Person& psa, unsigned short _Age);
    unsigned short Age;
public:
    Person(unsigned short _Age) : Age(_Age) {}
    unsigned short GetAge() { return Age; }

    // 成员函数重载 operator>
    bool operator>(Person& person) {
        return Age > person.Age;
    }
};

// Man<Woman == operator<(Man, Woman)
bool operator<(Person& psa, Person& psb) {
    return psa.Age < psb.Age;
}

bool operator<(Person& psa, unsigned short _Age) {
    return psa.Age < _Age;
}

int main() {
    Person Man(120);
    Person Woman(50);
    if (Man > Woman)
        std::cout << "找到富婆";
}
```

> **注意**：如果写的是小于那么就有两个 `operator` 了，程序无法根据参数判断，因此这里使用的是 `operator>`

---

## 5. 成员函数 vs 友元函数

| 特性 | 成员函数 | 友元函数 |
|------|---------|---------|
| **左操作数** | 必须是类对象 (`*this`) | 可以是任意类型 |
| **访问权限** | 可直接访问私有成员 | 需要声明为 `friend` |
| **推荐场景** | 单目运算符、`=`、`[]`、`()`、`->` | 双目运算符（如 `+`、`-`、`<<`） |

---

## 6. 不能重载的运算符

以下运算符**不能被重载**：

| 运算符 | 说明 |
|--------|------|
| `::` | 作用域解析运算符 |
| `.` | 成员访问运算符 |
| `.*` | 成员指针访问运算符 |
| `?:` | 三元条件运算符 |
| `sizeof` | 大小运算符 |
| `typeid` | 类型信息运算符 |

---

## 7. 关键要点

1. **本质**：运算符重载是语法糖，不改变运算符的优先级和结合性
2. **实现方式**：成员函数或友元函数
3. **选择依据**：需要类型转换时用友元，否则用成员函数
4. **限制**：某些运算符不能重载
5. **语义一致**：重载后的运算符行为应与原始语义相近
