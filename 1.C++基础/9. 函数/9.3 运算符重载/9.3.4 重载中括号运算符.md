# 9.3.4 重载中括号运算符 (Subscript Operator)

## 1. 简介

重载 `operator[]` 的主要目的是为你的类提供直观的、基于**索引的元素访问**功能，想象一下 `std::string` 或 `std::vector`：你可以通过 `myString[0]` 或 `myVector[i]` 来获取或设置它们的元素。通过重载这个运算符，你的自定义类也可以拥有这种便利性。

---

## 2. 语法

`operator[]` **必须**是一个非静态成员函数，它通常需要接收一个索引参数，并返回对应元素的引用。

```cpp
// 非 const 版本：用于对对象进行读写操作
ReturnType& operator[](ParameterType index);

// const 版本：用于对 const 对象或通过 const 引用访问的对象进行只读操作
const ReturnType& operator[](ParameterType index) const;
```

> 这个 `const` 关键字很重要，它表明这个函数不会修改对象的状态。如果你有一个 `const MyClass obj;`，你就只能调用其 `const` 成员函数，包括 `const` 版本的 `operator[]`。

---

## 3. 完整示例：MyArray 类

```cpp
#include <iostream>
#include <stdexcept>  // 用于抛出 std::out_of_range 异常

class MyArray {
private:
    int* data;       // 动态分配的数组
    size_t size;     // 数组的当前大小

public:
    // 构造函数
    MyArray(size_t initialSize) : size(initialSize) {
        if (size == 0) {
            data = nullptr;
        } else {
            data = new int[size];
            // 初始化数组元素，避免垃圾值
            for (size_t i = 0; i < size; ++i) {
                data[i] = 0;
            }
        }
    }

    // 析构函数
    ~MyArray() {
        delete[] data;
    }

    // --- 重载 operator[] 的非 const 版本（读写访问）---
    // 返回一个 int 的引用，这样就可以修改数组中的元素
    int& operator[](size_t index) {
        if (index >= size) {
            // 越界检查，抛出异常示错误
            throw std::out_of_range("MyArray: Index out of bounds in non-const operator[]");
        }
        return data[index];  // 返回对元素的引用
    }

    // --- 重载 operator[] 的 const 版本（只读访问）---
    // 注意函数签名末尾的第二个 'const' 关键字
    // 返回一个 const int 的引用，表示不能通过此引用修改元素
    const int& operator[](size_t index) const {
        if (index >= size) {
            // 越界检查，抛出异常
            throw std::out_of_range("MyArray: Index out of bounds in const operator[]");
        }
        return data[index];  // 返回对 const 元素的引用
    }

    // 获取数组大小的边数
    size_t getSize() const {
        return size;
    }
};
```

---

## 4. 使用示例

```cpp
int main() {
    // 1. 使用非 const 版本的 operator[]（读写）
    MyArray arr(5);  // 创建一个大小为 5 的 MyArray 对象

    std::cout << "--- 读写操作 (非 const operator[]) ---" << std::endl;
    std::cout << "初始值:" << std::endl;
    for (size_t i = 0; i < arr.getSize(); ++i) {
        std::cout << "arr[" << i << "] = " << arr[i] << std::endl;
    }

    // 写入操作: 通过 operator[] 返回的引用修改元素
    arr[0] = 10;
    arr[1] = 20;
    arr[4] = 50;

    std::cout << "\n修改后的值:" << std::endl;
    for (size_t i = 0; i < arr.getSize(); ++i) {
        std::cout << "arr[" << i << "] = " << arr[i] << std::endl;
    }

    // 2. 使用 const 版本的 operator[]（只读）
    // 注意: 我们将 arr_const 声明为 const MyArray
    const MyArray arr_const(3);
    // arr_const[0] = 99;  // 编译错误！不能修改 const 对象

    std::cout << "\n--- 只读操作 (const operator[]) ---" << std::endl;
    try {
        std::cout << "arr_const[0] = " << arr_const[0] << std::endl;  // 调用 const
        std::cout << "arr_const[1] = " << arr_const[1] << std::endl;
        // 尝试访问越界索引，会抛出异常
        std::cout << "arr_const[5] = " << arr_const[5] << std::endl;
    } catch (const std::out_of_range& e) {
        std::cerr << "捕获到异常: " << e.what() << std::endl;
    }

    // 3. 越界访问 (非 const 版本)
    std::cout << "\n--- 越界访问测试 (非 const operator[]) ---" << std::endl;
    try {
        arr[10] = 100;  // 越界，会抛出异常
    } catch (const std::out_of_range& e) {
        std::cerr << "捕获到异常: " << e.what() << std::endl;
    }

    return 0;
}
```

---

## 5. const 版本 vs 非 const 版本

| 版本 | 用途 | 返回类型 |
|------|------|---------|
| **非 const** | 用于读写操作 | `T&` |
| **const** | 用于只读操作 | `const T&` |

```cpp
MyArray arr(5);
arr[0] = 10;           // 调用非 const 版本

const MyArray carr(3);
int x = carr[0];       // 调用 const 版本
// carr[0] = 10;       // 错误！const 版本返回 const 引用
```

---

## 6. 为什么需要两个版本？

1. **非 const 版本**：允许通过 `[]` 修改元素
2. **const 版本**：确保 `const` 对象只能读取，不能修改

如果只有非 const 版本，以下代码将无法编译：

```cpp
void printFirst(const MyArray& arr) {
    std::cout << arr[0];  // 需要 const 版本
}
```

---

## 7. 边界检查

良好的实现应该包含边界检查：

```cpp
int& operator[](size_t index) {
    if (index >= size) {
        throw std::out_of_range("MyArray: Index out of bounds");
    }
    return data[index];
}
```

> **注意**：`std::vector::operator[]` 不进行边界检查（为了性能），而 `std::vector::at()` 会检查。

---

## 8. 关键要点

1. **必须是成员函数**：`operator[]` 不能是友元或普通函数
2. **两个版本**：提供 `const` 和非 `const` 版本以支持不同场景
3. **返回引用**：返回引用允许通过 `[]` 进行赋值操作
4. **边界检查**：建议添加越界检查并抛出异常
5. **索引类型**：通常使用 `size_t` 或 `int` 作为索引类型
