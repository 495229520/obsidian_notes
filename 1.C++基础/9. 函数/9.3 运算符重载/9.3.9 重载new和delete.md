# 9.3.9 重载 new 和 delete (Memory Management Operators)

## 1. 简介

C++ 允许你重载 `new` 和 `delete` 运算符，以自定义类对象的内存分配和释放行为。这在以下场景中非常有用：

- 追踪内存分配（调试内存泄漏）
- 实现内存池
- 对齐内存
- 统计内存使用情况

---

## 2. `new` 和 `delete` 的工作流程

### `new` 表达式

```cpp
MyClass* ptr = new MyClass(args);
```

实际上执行两步：

1. 调用 `operator new` 分配原始内存
2. 在分配的内存上调用构造函数

### `delete` 表达式

```cpp
delete ptr;
```

实际上执行两步：

1. 调用析构函数
2. 调用 `operator delete` 释放内存

---

## 3. 语法

### 类特定的重载（推荐）

```cpp
class MyClass {
public:
    // 重载 new 运算符
    static void* operator new(size_t size);

    // 重载 delete 运算符
    static void operator delete(void* ptr);

    // 重载数组版本
    static void* operator new[](size_t size);
    static void operator delete[](void* ptr);
};
```

### 全局重载（谨慎使用）

```cpp
// 全局 new
void* operator new(size_t size);

// 全局 delete
void operator delete(void* ptr) noexcept;
```

> **注意**：全局重载会影响整个程序的内存分配，应谨慎使用。

---

## 4. 示例：Particle 类

```cpp
#include <iostream>
#include <cstdlib>

class Particle {
public:
    double x, y, z;     // 位置
    double vx, vy, vz;  // 速度

    Particle() : x(0), y(0), z(0), vx(0), vy(0), vz(0) {}

    // 重载类特定的 operator new
    static void* operator new(size_t size) {
        std::cout << "Customized new for Particle called. Size = " << size << " bytes.\n";
        void* ptr = std::malloc(size);
        if (!ptr) {
            throw std::bad_alloc();
        }
        return ptr;
    }

    // 重载类特定的 operator delete
    static void operator delete(void* ptr) {
        std::cout << "Customized delete for Particle called.\n";
        std::free(ptr);
    }

    // 重载数组版本的 operator new[]
    static void* operator new[](size_t size) {
        std::cout << "Customized new[] for Particle called. Size = " << size << " bytes.\n";
        void* ptr = std::malloc(size);
        if (!ptr) {
            throw std::bad_alloc();
        }
        return ptr;
    }

    // 重载数组版本的 operator delete[]
    static void operator delete[](void* ptr) {
        std::cout << "Customized delete[] for Particle called.\n";
        std::free(ptr);
    }
};
```

---

## 5. 使用示例

```cpp
int main() {
    std::cout << "--- 单个对象 ---\n";
    Particle* p1 = new Particle();
    delete p1;

    std::cout << "\n--- 对象数组 ---\n";
    Particle* particles = new Particle[5];
    delete[] particles;

    return 0;
}
```

### 输出

```
--- 单个对象 ---
Customized new for Particle called. Size = 48 bytes.
Customized delete for Particle called.

--- 对象数组 ---
Customized new[] for Particle called. Size = 248 bytes.
Customized delete[] for Particle called.
```

## 6. placement new

**placement new** 允许在**已分配的内存**上构造对象：

```cpp
#include <new>  // 需要包含此头文件

class Demo {
public:
    int value;
    Demo(int v) : value(v) {
        std::cout << "Constructor called, value = " << value << std::endl;
    }
    ~Demo() {
        std::cout << "Destructor called" << std::endl;
    }
};

int main() {
    // 分配原始内存
    char* buffer = new char[sizeof(Demo)];

    // 在已分配的内存上构造对象（placement new）
    Demo* obj = new (buffer) Demo(42);

    // 使用对象
    std::cout << "obj->value = " << obj->value << std::endl;

    // 手动调用析构函数（必须！）
    obj->~Demo();

    // 释放原始内存
    delete[] buffer;

    return 0;
}
```

> **注意**：使用 placement new 时，必须**手动调用析构函数**，然后再释放原始内存。

## 7. 重载注意事项

| 要点 | 说明 |
|------|------|
| **必须是静态的** | `operator new` 和 `operator delete` 隐式为静态成员函数 |
| **`new` 可能抛出异常** | 分配失败时应抛出 `std::bad_alloc` |
| **`delete` 应为 `noexcept`** | 析构函数不应抛出异常 |
| **处理空指针** | `delete nullptr` 应该是安全的 |
| **匹配使用** | `new` 配 `delete`，`new[]` 配 `delete[]` |

## 8. 应用场景

### 内存池

```cpp
class PoolAllocated {
    static std::vector<void*> pool;
public:
    static void* operator new(size_t size) {
        if (!pool.empty()) {
            void* ptr = pool.back();
            pool.pop_back();
            return ptr;
        }
        return std::malloc(size);
    }

    static void operator delete(void* ptr) {
        pool.push_back(ptr);  // 归还到池中而不是释放
    }
};
```

### 内存追踪

```cpp
static void* operator new(size_t size) {
    std::cout << "Allocating " << size << " bytes\n";
    return std::malloc(size);
}
```

---

## 9. 关键要点

1. **两阶段过程**：`new` = 分配内存 + 构造，`delete` = 析构 + 释放
2. **隐式静态**：`operator new/delete` 自动是静态成员函数
3. **全局谨慎**：全局重载影响整个程序
4. **placement new**：在指定内存位置构造对象
5. **匹配使用**：`new/delete` 和 `new[]/delete[]` 必须配对
