## 简介

**栈 (Stack)** 是程序运行时用于存储函数调用信息的内存区域，这些信息包括局部变量、函数参数和返回地址。这种数据结构遵循后进先出 (LIFO, Last In First Out) 的原则。

**栈溢出攻击 (Stack Overflow Attack)** 是一种常见的安全漏洞利用方式，攻击者通过向程序栈中写入过多数据，覆盖关键信息（如返回地址），从而控制程序的执行流程。

---

## 核心概念

### 1. 栈帧 (Stack Frame)

每当调用一个函数，系统会为其在栈上分配一块内存，称为**栈帧**。栈帧包含：

| 组成部分 | 说明 |
|----------|------|
| **局部变量** | 函数内定义的变量 |
| **参数** | 传递给函数的参数 |
| **返回地址** | 函数执行完后返回到调用者的地址 |
| **保存的寄存器** | 如保存的 EBP |

### 2. 缓冲区溢出 (Buffer Overflow)

当程序向缓冲区（如数组）写入的数据超出其预定大小时，就会发生缓冲区溢出。溢出的数据会覆盖相邻内存区域的数据，这可能包括：
- 其他局部变量
- 保存的帧指针 (EBP)
- **返回地址 (Return Address)**

---

## 栈帧结构

```
高地址
├─────────────────────────────────────┤
│           参数 (Arguments)           │  [ebp + 0xC], [ebp + 0x8]
├─────────────────────────────────────┤
│         返回地址 (Return Address)     │  ← call 指令自动压入
├─────────────────────────────────────┤
│          保存的 EBP (Saved EBP)       │  ← push ebp
├─────────────────────────────────────┤
│         局部变量 (Local Variables)    │  [ebp - 0x4], [ebp - 0x8] ...
├─────────────────────────────────────┤
│              缓冲区 (Buffer)          │  ← 可能被溢出
├─────────────────────────────────────┤
低地址 (ESP 栈顶)
```

---

## 栈溢出攻击原理

### 攻击流程

1. **输入超长数据**：向缓冲区写入超过其容量的数据
2. **覆盖返回地址**：溢出的数据覆盖栈帧中的返回地址
3. **控制执行流**：当函数返回时，CPU 会跳转到被篡改的地址
4. **执行恶意代码**：攻击者可以让程序跳转到注入的恶意代码

### 图解

```
正常情况:                       溢出后:
┌────────────┐                ┌────────────┐
│ 返回地址    │ ← 正常地址      │ 返回地址    │ ← 被篡改为恶意地址
├────────────┤                ├────────────┤
│ Saved EBP  │                │ AAAA...    │ ← 被覆盖
├────────────┤                ├────────────┤
│ buffer[64] │ ← 正常数据      │ AAAA...    │ ← 溢出数据
└────────────┘                └────────────┘
```

## 漏洞代码示例

```cpp
#include <cstdio>
#include <cstring>

void vulnerable_function(char* input) {
    char buffer[100];
    strcpy(buffer, input);  // 危险！没有边界检查
    printf("Buffer content: %s\n", buffer);
}

int main() {
    char user_input[200];
    std::cin.getline(user_input, 200);
    vulnerable_function(user_input);
    std::cout << "Program continues..." << std::endl;
    return 0;
}
```

### 漏洞分析

1. `buffer` 在栈上分配了 100 字节
2. `strcpy` 没有边界检查，会复制整个 `input`
3. 如果 `input` 超过 100 字节，就会覆盖 `buffer` 后面的内存
4. 最终可能覆盖返回地址

## 汇编层面分析

### 漏洞函数的栈帧

```cpp
void vulnerable_function(char* input) {
    char buffer[100];
    strcpy(buffer, input);
}
```

```asm
vulnerable_function:
    push ebp                    ; 保存调用者的 EBP
    mov ebp, esp                ; 设置新的栈帧基址
    sub esp, 0x6C               ; 为局部变量分配空间 (buffer 等)

    ; strcpy(buffer, input)
    lea eax, [ebp-64h]          ; buffer 的地址 ([ebp-100])
    mov ecx, [ebp+8]            ; input 参数
    push ecx                    ; 第二个参数
    push eax                    ; 第一个参数
    call strcpy

    mov esp, ebp                ; 恢复栈指针
    pop ebp                     ; 恢复 EBP
    ret                         ; 返回 → 跳转到 [esp] 指向的地址
```

### 攻击过程

当 `strcpy` 复制超过 100 字节的数据时：

| 偏移 | 内容 | 说明 |
|------|------|------|
| `[ebp-100]` ~ `[ebp-1]` | buffer | 正常缓冲区 |
| `[ebp]` | Saved EBP | 被 `AAAA` 覆盖 |
| `[ebp+4]` | **Return Address** | 被恶意地址覆盖 |

当函数执行 `ret` 指令时，会将 `[esp]`（即被篡改的返回地址）弹出到 EIP，程序跳转到攻击者指定的地址。

## 防护机制

### 1. 栈保护 (Stack Canary / Stack Protector)

在返回地址和局部变量之间插入一个随机值（金丝雀值），函数返回前检查该值是否被修改。

```
┌────────────┐
│ 返回地址    │
├────────────┤
│ Canary 值  │ ← 随机值，函数返回前检查
├────────────┤
│ 局部变量    │
└────────────┘
```

### 2. 地址空间布局随机化 (ASLR)

随机化程序的内存布局，使攻击者难以预测目标地址。

### 3. 数据执行保护 (DEP / NX)

将栈标记为不可执行，即使攻击者注入了代码，也无法执行。

### 4. 安全编程实践

使用安全的函数替代不安全的函数：

| 不安全函数 | 安全替代 |
|-----------|----------|
| `strcpy` | `strncpy`, `strcpy_s` |
| `gets` | `fgets` |
| `sprintf` | `snprintf` |
| `scanf("%s", ...)` | `scanf("%99s", ...)` |

```cpp
// 不安全
strcpy(buffer, input);

// 安全
strncpy(buffer, input, sizeof(buffer) - 1);
buffer[sizeof(buffer) - 1] = '\0';

// C11 安全
strcpy_s(buffer, sizeof(buffer), input);
```

---

## 关键要点

1. **栈溢出本质**：向缓冲区写入过多数据，覆盖返回地址
2. **攻击目标**：篡改返回地址，劫持程序控制流
3. **危险函数**：`strcpy`、`gets`、`sprintf` 等无边界检查的函数
4. **防护手段**：Stack Canary、ASLR、DEP/NX、安全编码
5. **安全意识**：永远不要信任用户输入，始终进行边界检查
