# 9.1.11 递归函数 (Recursive Functions)

> 递归函数是一种通过调用自身来解决问题的函数，它会将问题分解为更小的、相似的子问题。每次递归调用都会在[[1.2 函数栈帧建立与销毁原理|栈]]上创建新的栈帧。

---

## 1. 简介

递归函数是一种通过**调用自身**来解决问题的函数，它会将问题分解为更小的、相似的子问题。

### 递归的核心思想

- **分治**：将大问题分解为小问题
- **自相似**：小问题与大问题具有相同的结构
- **终止条件**：必须有明确的退出条件

---

## 2. 执行步骤

### 2.1 检查基本情况 (Base Case)

函数首先会检查是否满足某个或某些特定的**终止条件**，这些条件被称为基本情况或递归出口。如果满足基本情况，函数将直接返回一个结果，而不再进行递归调用。

### 2.2 递归调用 (Recursive Step)

如果不满足基本情况，函数会将问题分解为更小的、与原问题相似的子问题，并**使用相同的函数自身**来解决这些子问题。这个过程就是递归调用。

### 2.3 组合结果

在递归调用返回结果后，原始函数可能会对这些结果进行组合或处理，最终得到整个问题的解。

---

## 3. 基本情况的重要性

> **基本情况是递归函数中至关重要的部分**

如果没有定义明确的基本情况，或者基本情况永远无法被满足，那么函数将会无限地调用自身，导致**无限递归 (Infinite Recursion)**，最终会耗尽计算机的内存资源并导致程序崩溃（通常会抛出[[9.1.6 函数栈溢出攻击|栈溢出]]错误）。

---

## 4. 例子：计算阶乘

计算一个非负整数 n 的阶乘（表示为 n!）的定义是：

- `0! = 1`
- `n! = n * (n-1)!` 对于 n > 0

### 代码实现

```cpp
int factorial(int n) {
    // 基本情况：当 n 为 0 时，阶乘为 1
    if (n == 0) {
        return 1;
    }
    // 递归调用：当 n 大于 0 时，n! = n * (n-1)!
    else {
        return n * factorial(n - 1);
    }
}

int main() {
    int num = 5;
    int result = factorial(num);  // 调用递归函数计算 5 的阶乘
    // 输出结果: 5! = 120
    return 0;
}
```

### 执行过程图解

```
factorial(5)
    │
    └─→ 5 * factorial(4)
              │
              └─→ 4 * factorial(3)
                        │
                        └─→ 3 * factorial(2)
                                  │
                                  └─→ 2 * factorial(1)
                                            │
                                            └─→ 1 * factorial(0)
                                                      │
                                                      └─→ return 1  ← 基本情况
                                            │
                                            └─→ return 1 * 1 = 1
                                  │
                                  └─→ return 2 * 1 = 2
                        │
                        └─→ return 3 * 2 = 6
              │
              └─→ return 4 * 6 = 24
    │
    └─→ return 5 * 24 = 120
```

## 5. 递归与栈

每次递归调用都会在栈上创建一个新的[[1.2 函数栈帧建立与销毁原理|栈帧]]：

```
栈的增长（factorial(5) 调用过程）

高地址
├────────────────────┤
│ factorial(0) 栈帧  │ ← 最后调用，首先返回
├────────────────────┤
│ factorial(1) 栈帧  │
├────────────────────┤
│ factorial(2) 栈帧  │
├────────────────────┤
│ factorial(3) 栈帧  │
├────────────────────┤
│ factorial(4) 栈帧  │
├────────────────────┤
│ factorial(5) 栈帧  │ ← 首先调用
├────────────────────┤
│    main() 栈帧     │
└────────────────────┘
低地址
```

## 6. 优缺点

### 优点

- **简洁直观**：对于某些问题，递归的定义和实现非常简洁、直观，更容易理解和编写（例如，树的遍历、分治算法等）
- **代码优雅**：可以使代码看起来更优雅

### 缺点

- **栈空间开销**：每次递归调用都需要在内存中创建一个新的函数调用栈帧，如果递归深度过大，可能会导致**栈溢出**
- **性能问题**：在某些情况下，递归的效率可能不如迭代（循环），因为递归涉及到额外的函数调用开销

---

## 7. 递归优化：尾递归

**尾递归**是一种特殊的递归形式，递归调用是函数的最后一个操作。某些编译器可以将尾递归优化为迭代，避免栈溢出。

### 普通递归

```cpp
int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);  // 递归后还需要乘法运算
}
```

### 尾递归

```cpp
int factorial_tail(int n, int accumulator = 1) {
    if (n == 0) return accumulator;
    return factorial_tail(n - 1, n * accumulator);  // 递归是最后的操作
}
```

---

## 8. 何时使用递归

### 适合使用递归的场景

- 问题可以自然地分解为相似的子问题
- 树、图的遍历
- 分治算法（如快速排序、归并排序）
- 回溯算法

### 不适合使用递归的场景

- 递归深度可能非常大
- 有简单的迭代解法
- 性能要求极高的场景

---

## 9. 关键要点

1. **基本情况**：必须定义明确的终止条件，否则会无限递归
2. **递归调用**：函数调用自身，处理更小的子问题
3. **栈帧**：每次调用创建新栈帧，深度过大会导致栈溢出
4. **时间换空间**：递归可能更简洁，但可能有性能开销
5. **尾递归优化**：如果递归是最后的操作，编译器可能优化为迭代
