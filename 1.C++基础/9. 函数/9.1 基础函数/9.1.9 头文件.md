# 9.1.9 头文件 (Header Files)

> 头文件是 C/C++ 中用于**共享声明**的机制，在[[1.1 程序编译与链接原理#1. 编译全过程 (The Build Process)|预处理阶段]]被 `#include` 指令插入到源文件中。

---

## 1. 多源文件编译流程

```
程序中多个源文件同时代码的编译
a.cpp    编译    a.obj
b.cpp  ──────→  b.obj  ──→  目标程序
c.cpp    编译    c.obj
d.cpp    编译    d.obj
e.cpp    编译    e.obj
f.cpp    编译    f.obj
```

每个 `.cpp` 文件独立编译成 `.obj`（目标文件），然后链接成最终的可执行程序。

---

## 2. 头文件的作用

头文件主要用于**存放声明**，以便在多个源文件之间共享接口信息：

| 内容类型 | 示例 |
|----------|------|
| **函数声明** | `int add(int a, int b);`<br>`void print_message(const char* message);` |
| **结构体** | `struct Point { int x; int y; };` |
| **枚举** | `enum Color { RED, GREEN, BLUE };` |
| **宏定义** | `#define PI 3.14159`<br>`#define MAX(a, b) ((a) > (b) ? (a) : (b))` |
| **外部变量声明** | `extern int global_counter;`<br>（声明另一源文件中定义的变量，`extern` 关键字表示变量的存储空间在其他地方分配） |

---

## 3. 头文件的包含保护

### 问题：重复包含

当有两个头文件（例如 `header1.h` 包含 `header2.h`）直接或间接地互相包含时，可能会产生生循环依赖问题。

```
包含保护：当有两个头文件（例如header1.h和header2.h）直接或间接地互相包含时
            ┌───────────────┐
            │   header1.h   │ ←──┐
            │  包括header2.h│    │
            └───────┬───────┘    │
                    │            │
                    ↓            │
            ┌───────────────┐    │
            │   header2.h   │    │
            │  包括header1.h│ ───┘
            └───────────────┘

- header1.h 包括 header2.h
- header2.h 包括 header1.h
- header1.h 包括 header2.h，等等

这种情况可能导致编译器陷入无限循环，从而导致编译错误
```

如果没有包含保护且头文件被同一个编译单元多次包含，最常见的错误是**重定义错误**，例如：
- `error: redefinition of 'struct Point'`（结构体重定义）
- `error: 'enum Color' has already been declared`（枚举重定义）

### 解决方案一：`#pragma once`

```cpp
#pragma once

// 头文件内容
int add(int a, int b);
```

- 告诉编译器该文件只包含一次
- 已成为事实标准，被大多数现代编译器支持
- 比传统宏保护稍快（编译器可以跳过文件的 `#include` 指令）

### 解决方案二：传统宏保护 (`#ifndef`)

```cpp
#ifndef MY_HEADER_H
#define MY_HEADER_H

// Header file contents go here

#endif // MY_HEADER_H
```

- 使用宏来检测头文件是否已被包含
- 如果没有被定义过，则定义该宏并包含内容
- 第二次包含时，宏已定义，跳过内容

### 两种方式的对比

| 特性 | `#pragma once` | `#ifndef` 宏保护 |
|------|---------------|-----------------|
| 标准 | 非标准（但广泛支持） | 标准 C/C++ |
| 可移植性 | 可能在某些老旧编译器上不支持 | 完全可移植 |
| 简洁性 | 更简洁 | 需要三行代码 |
| 编译速度 | 通常更快 | 略慢 |

## 4. 源文件与头文件的分工

### 头文件 (.h)

为头文件中声明的函数提供实际代码：

```cpp
#include "my_header.h"

int add(int a, int b) {
    return a + b;
}
```

### 源文件 (.cpp/.c)

```cpp
// 函数定义：为头文件中声明的函数提供实际代码
#include "my_header.h"

void print_message(const char* message) {
    printf("Message: %s\n", message);
}

// 全局变量定义：为具有全局范围的变量分配存储空间
int global_counter = 0;
```

### 使用头文件

```cpp
// main.cpp
#include "my_header.h"

int main() {
    print_message("Hello!");
    return 0;
}
```

---

## 5. `main()` 函数

每个可执行程序必须有一个 `main()` 函数，它是程序的入口点：

```cpp
int main() {
    // 程序从这里开始执行
    return 0;
}
```

- 编译器不会主动引入头文件，除非源文件 `.cpp` 引入了这个头文件
- 只要在一个工程中，相同的函数名在不同的源文件中也会被调用

---

## 6. 头文件使用的注意事项

### 静态函数和内联函数

静态函数和内联函数可以写在头文件中：

```cpp
// 在头文件中
static int helper() {
    return 42;
}

inline int square(int x) {
    return x * x;
}
```

### 在多个 `.h` 头文件中

如果某相互引用了会编译错误，但是当 `#pragma` 的默认下不会编译错误：

```cpp
// 在多个.h头文件中，如果某相互引用了
// 会编译错误，但是当#pragma的默认下
// 不会编译错误

#define

#endif
```

---

## 7. 关键要点

1. **头文件作用**：存放声明，供多个源文件共享
2. **包含保护**：使用 `#pragma once` 或 `#ifndef` 防止重复包含
3. **分工**：头文件放声明，源文件放定义
4. **预处理**：`#include` 在预处理阶段将头文件内容插入源文件
5. **编译单元**：每个 `.cpp` 文件是独立的编译单元
