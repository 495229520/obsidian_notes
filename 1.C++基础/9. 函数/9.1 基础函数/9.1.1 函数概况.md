## 简介

函数是 C++ 中代码复用的基本单元，用于封装特定的功能逻辑。

```cpp
#include <iostream>

// 函数定义：计算两个整数的和
int add(int a, int b) {
    int sum = a + b;
    return sum;
}

int main() {
    int x = 5;
    int y = 3;
    int result = add(x, y);  // 函数调用
    std::cout << "The sum is: " << result << std::endl;
    return 0;
}
```

## 函数的本质

经过分析，我们知道**函数的本质是一段内存里的二进制数据**。我们写下的 C++ 代码会翻译成对应的二进制数据，程序运行的时候会通过某种规则来加载到我们的内存中。一旦程序一旦编译(生成)，这个程序的二进制数据就不会再发生变化。

```
程序的生成
C++ 代码 => 二进制数据 => 程序文件(硬盘)

程序的运行
程序文件(硬盘) => 加载到内存中
```

在 C++ 中，直接输出一个函数名会发生以下情况：

### 1. 函数指针

在 C++ 中，函数名在大多数情况下会被隐式转换为指向该函数的指针。因此，直接输出函数名实际上是输出该函数在内存中的地址。这个地址表示该函数代码在内存中的起始位置，也就是函数指令存放在内存中的地址。

### 2. 输出结果

输出的结果将是该函数的地址（Function address: 0x401160），其中 0x401160 是一个示例地址。

### 3. 代码示例

```cpp
#include <iostream>

void myFunction() {
    std::cout << "Hello, World!" << std::endl;
}

int main() {
    std::cout << "Function address: " << myFunction << std::endl;
    return 0;
}
```

在这个例子中，`myFunction` 函数的地址会被输出。

### 4. 函数指针的用途

函数指针可以用于将函数作为参数传递给其他函数，实现回调机制，允许在运行时动态选择要执行的函数。它们还可以用于创建函数指针数组，以便根据需要调用不同的函数。

## 函数栈帧（汇编层面）

详见 [[1.2 函数栈帧建立与销毁原理]] 了解函数调用时内存的变化过程。

| 寄存器 | 作用 |
|--------|------|
| **EBP** | 基址指针：通常用作栈帧的基址，用于访问函数参数和局部变量 |
| **ESP** | 堆栈指针：始终指向当前栈帧的顶部位置 |

### 汇编指令分析

```asm
push 2          ; 在栈里放一个 2
push 1          ; 在栈里放一个 1
call 00CF1000   ; 跳转到 00CF1000
push ebp        ; 将 ebp 压入栈中，保存了调用当前函数的栈底指针
mov ebp, esp    ; 将 ESP 寄存器的值复制到 EBP 寄存器，将当前堆栈的顶部设置为当前函数的栈帧基址
```

### 参数获取

```asm
mov eax, dword ptr [ebp+8]    ; 将数据移动到 EAX 寄存器
                              ; dword ptr [...] 访问一个双字（4字节）大小的内存单元
                              ; [ebp+8] 内存单元的地址是 EBP 寄存器的值加上偏移量 8
```

这条指令从栈帧中第二个参数位置的内存中读取一个双字，并将读取的值存储到 EAX 寄存器中的第一个参数。

### 返回值处理

```asm
add eax, dword ptr [ebp+0Ch]  ; 将后面的作数加到 EAX 寄存器中
                              ; EAX 寄存器通常用于存储函数的返回值或中间计算结果

dword ptr     ; 访问一个双字（4字节）大小的内存单元
[ebp+0Ch]     ; 内存单元的地址是 EBP 寄存器的值加上偏移量 0Ch

              ; 将存储在栈帧中第二个参数位置的值加到 EAX 寄存器的第一个参数

pop ebp       ; 指令从栈顶弹出一个值，并将其存储到 EBP 寄存器中
              ; 恢复调用者的栈帧基址

ret           ; 从栈顶弹出一个值，并将其赋给作用程序的下一条指令入栈中
              ; 在函数调用时，调用者的返回地址被压入栈中
              ; ret 指令的作用是跳转回调用者的代码，可理解为
              ; 将跳转到 myFunction
```

## 指针参数

通过[[6.1基本指针|指针]]传递参数，函数可以直接访问并修改调用者传递的变量，这使得函数能够返回多个结果或者改变输入参数的值。

与值传递不同，值传递是在函数内部创建参数的副本，而指针传递则允许函数操作原始变量。

### 1. 修改函数外部的变量

```cpp
#include <iostream>

void increment(int* num) {
    (*num)++;  // 使用解引用运算符修改指针指向的值
}

int main() {
    int value = 10;
    increment(&value);  // 传递变量的地址
    std::cout << "Value: " << value << std::endl;  // 输出：Value: 11
    return 0;
}
```

### 2. 传递大型数据结构

传递大型数据结构（如数组、结构体或对象）时，使用指针可以避免复制整个数据结构，从而提高效率。传递副本可能导致性能问题，尤其是当数据结构非常大时。

```cpp
#include <iostream>

void printArray(int* arr, int size) {
    for (int i = 0; i < size; i++) {
        std::cout << arr[i] << " ";
    }
    std::cout << std::endl;
}

int main() {
    int myArray[] = {1, 2, 3, 4, 5};
    printArray(myArray, 5);  // 传递数组的地址和大小
    return 0;
}
```

### 3. 动态内存分配

指针对于处理[[6.2 动态内存分配与释放|动态分配的内存]]至关重要。函数可以使用指针来接收和操作动态创建的数据。

```cpp
#include <iostream>

void createArray(int** arr, int size) {
    *arr = new int[size];  // 动态分配内存
    for (int i = 0; i < size; i++) {
        (*arr)[i] = i + 1;
    }
}

int main() {
    int* myArray;
    createArray(&myArray, 3);
    for (int i = 0; i < 3; i++) {
        std::cout << myArray[i] << " ";
    }
    std::cout << std::endl;
    delete[] myArray;
    return 0;
}
```

### 4. 实现回调函数

函数指针可以作为参数传递给其他函数，实现回调机制，允许在运行时动态选择要执行的函数。

## 关键要点

1. **函数本质**：函数是一段存储在内存中的二进制指令，函数名代表该段代码的起始地址
2. **栈帧机制**：函数调用时会创建栈帧来存储参数、局部变量和返回地址
3. **指针参数**：通过指针传参可以修改外部变量、避免大对象拷贝、支持动态内存操作
4. **返回值**：通常通过 EAX 寄存器传递返回值
