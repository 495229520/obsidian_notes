> 静态变量使用 `static` 关键字声明，它们在程序的整个生命周期内存在，存储在[[1.3 进程的虚拟地址空间|虚拟地址空间]]的 `.data` 或 `.bss` 段中。

---

## 1. 静态变量的基本特性

| 特性 | 说明 |
|------|------|
| **内存分配** | 在编译期分配内存，存储在静态存储区 |
| **生命周期** | 整个程序运行期间（与程序同生共死） |
| **初始化** | 只初始化一次（首次执行到声明语句时） |
| **默认值** | 未初始化时默认为 0（与全局变量相同） |
| **存储位置** | 存放在 `.data`（已初始化）或 `.bss`（未初始化/初始化为0）段 |

---

## 2. 函数内静态变量

函数内的静态变量（静态局部变量）在函数调用之间保持其值。

### 示例代码

```cpp
void counter() {
    static int count = 0;  // 只初始化一次
    count++;
    std::cout << "Count: " << count << std::endl;
}

int main() {
    counter();  // 输出 Count: 1
    counter();  // 输出 Count: 2
    counter();  // 输出 Count: 3
    return 0;
}
```

### 与普通局部变量的对比

| 特性 | 静态局部变量 | 普通局部变量 |
|------|-------------|-------------|
| 存储位置 | `.data` / `.bss` 段 | 栈 |
| 生命周期 | 整个程序运行期间 | 函数调用期间 |
| 初始化次数 | 一次 | 每次函数调用 |
| 默认值 | 0 | 未定义（垃圾值） |
| 是否产生符号 | 是（局部符号） | 否 |

## 3. 静态变量的存储位置详解

根据初始化状态，静态变量存储在不同的内存段：

```cpp
static int gdata1 = 10;   // .data 段（已初始化且非零）
static int gdata2 = 0;    // .bss 段（初始化为零）
static int gdata3;        // .bss 段（未初始化，默认为0）

void func() {
    static int a = 5;     // .data 段
    static int b = 0;     // .bss 段
    static int c;         // .bss 段
}
```

> **注意**：静态变量不存储在栈上，因此不会随函数调用而创建销毁。

## 4. 静态全局变量与链接属性

### 外部链接 (External Linkage)

普通全局变量具有**外部链接**属性，可以在其他源文件中通过 `extern` 声明访问：

```cpp
// file_a.cpp
int global_var = 10;       // 外部链接，其他文件可访问

// file_b.cpp
extern int global_var;     // 声明，引用 file_a.cpp 中的定义
```

### 内部链接 (Internal Linkage)

静态全局变量具有**内部链接**属性，只能在定义它的源文件内访问：

```cpp
// file_a.cpp
static int file_static_var = 10;  // 内部链接，仅本文件可见

// file_b.cpp
extern int file_static_var;        // 错误！无法访问其他文件的静态变量
```

### 多文件示例

```cpp
// a.cpp
static int file_static_var = 10;   // 内部链接，a.cpp 私有
int global_var = 20;               // 外部链接，全局可见

void print_a_vars() {
    std::cout << "a.cpp: file_static_var = " << file_static_var << std::endl;
    std::cout << "a.cpp: global_var = " << global_var << std::endl;
}

// main.cpp
extern int global_var;             // 正确：访问 a.cpp 的全局变量
// extern int file_static_var;     // 错误：无法访问 static 变量

void print_a_vars();

int main() {
    print_a_vars();
    std::cout << "main.cpp: global_var = " << global_var << std::endl;
    return 0;
}
```

## 5. 静态成员变量（类作用域）

在 C++ 类中，静态成员变量被所有对象共享：

```cpp
class ClassName {
public:
    static int static_member;  // 声明
};

// 类外定义和初始化（必须）
int ClassName::static_member = 0;
```

### 访问方式

```cpp
// 通过类名访问（推荐）
ClassName::static_member = 10;

// 通过对象访问（不推荐）
ClassName obj;
obj.static_member = 10;
```

## 6. 内联函数与静态变量

> 在 C++17 之前，内联函数中的静态变量可能在每个翻译单元中有独立副本，导致意外行为。

```cpp
// 在头文件中定义的内联函数
inline int getValue() {
    static int value = 0;  // C++17 前可能有多个副本
    return ++value;
}
```

**解决方案**：

- **C++17**：使用 `inline` 变量确保单一定义
- **C++17 前**：将静态变量定义移到 `.cpp` 文件中

---

## 7. 关键要点

1. **生命周期**：静态变量从程序开始到结束，不随作用域销毁
2. **初始化**：只执行一次，发生在首次执行到声明时
3. **存储位置**：`.data`（非零初始化）或 `.bss`（零/未初始化）
4. **链接属性**：静态全局变量为内部链接，仅本文件可见
5. **类成员**：静态成员被所有对象共享，必须在类外定义
