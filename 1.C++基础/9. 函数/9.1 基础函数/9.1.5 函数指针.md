## 简介

**函数指针**是 C/C++ 中一种特殊的[[6.1基本指针|指针]]，用来存储函数的内存地址。它可以让我们在运行时动态选择要调用的函数，实现回调机制和策略模式。

### 函数指针的声明

函数指针的声明方式看起来比较复杂：

```cpp
// 声明一个指向 "返回int，接受两个int参数" 的函数的指针
int (*pFunc)(int, int);
```

---

## 函数指针的作用

1. **回调函数**：将函数作为参数传递给其他函数
2. **事件处理**：注册事件处理函数
3. **策略模式**：运行时选择不同的算法实现
4. **函数表**：实现类似虚函数表的机制

---

## 回调函数 (Callback Functions)

**概念**：回调函数（`CallBack Function`）是通过函数指针调用的函数。只要把一个函数的指针（即地址）作为参数传递给其他函数，当这个指针被用来调用它所指向的函数时，这种机制就叫做回调函数。

**重点理解**：
- 回调函数就像是在调用函数前做的"准备工作"，就好比做饭时的"定好火候"，等到要炒菜时才调用
- 实现了控制反转的设计模式，提供了函数调用的灵活性，尤其在异步编程中广泛使用

```cpp
#include <iostream>

// 定义一个返回 void，接受两个 int 参数的函数类型的指针
using Callback = void (*)(int, int);

// 定义回调函数
void myCallback(int a, int b) {
    std::cout << "Callback called with: " << a << " and " << b << std::endl;
}

// 接受回调函数的函数
void performOperation(Callback callback, int x, int y) {
    // 执行一些操作...
    callback(x, y);  // 调用回调函数
}

int main() {
    performOperation(myCallback, 10, 20);
    return 0;
}
```

## 函数指针数组

可以创建函数指针的数组来存储多个函数指针：

```cpp
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int mul(int a, int b) { return a * b; }

int main() {
    // 函数指针数组
    int (*operations[3])(int, int) = { add, sub, mul };

    std::cout << operations[0](5, 3) << std::endl;  // 输出: 8
    std::cout << operations[1](5, 3) << std::endl;  // 输出: 2
    std::cout << operations[2](5, 3) << std::endl;  // 输出: 15
    return 0;
}
```

## 排序回调函数示例

以下是使用回调函数实现排序的完整示例：

```cpp
#include <iostream>
#include <algorithm>
#include <vector>

// 1. 默认排序：使用标准库 sort
void defaultSort(std::vector<int>& numbers) {
    std::sort(numbers.begin(), numbers.end());
}

// 2. 自定义排序：传入比较函数
void printVector(const std::vector<int>& numbers, int N) {
    for (int i = 0; i < N; ++i) {
        std::cout << numbers[i] << " ";
    }
    std::cout << std::endl;
}

void printVectorShuffle(std::vector<int>& arr) {
    // 打乱数组
    std::cout << "{ ";
    for (int i = 0; i < arr.size(); ++i) {
        std::cout << arr[i] << ", ";
    }
    std::cout << "}" << std::endl;
}

int main() {
    int seed[] = {
        1, 2, 3, 5, 6, 1, 9, 8
    };

    std::vector<int> numbers(std::begin(seed), std::end(seed));
    printVectorShuffle(numbers);

    // 3. 使用比较函数
    bool ascendingOrder(int a, int b) {
        return a < b;
    }

    // 4. 传递比较函数 - 这就是 sort 的回调函数
    std::sort(numbers.begin(), numbers.end(), ascendingOrder);
    printVectorShuffle(numbers);  // 升序

    // 下面的 descendingOrder 是一个函数对象
    // CompareFunction 继承的 std::binary_function 是降序

    return 0;
}
```

### 函数对象 (Functor)

`std::sort` 的第三个参数可以是：
1. **函数指针**：普通函数
2. **函数对象**：重载了 `operator()` 的类
3. **Lambda 表达式**：匿名函数

```cpp
// 使用 Lambda 表达式作为排序回调
std::sort(numbers.begin(), numbers.end(), [](int a, int b) {
    return a > b;  // 降序排序
});
```

## typedef 和 using 的区别

| 特性 | typedef (C-Style) | using (C++11 别名) |
|------|-------------------|-------------------|
| **基本语法** | 语法比较"绕"，名称在中间 | 语法更直观（名称 = 类型） |
| **示例** | `typedef int MyInt;` | `using MyInt = int;` |
| **复杂类型** | 尤其在函数指针时，可读性差 | 语法结构保持一致，可读性高 |
| **函数指针示例** | `typedef void (*FuncPtr)(int);` | `using FuncPtr = void (*)(int);` |
| **模板别名** | 不支持 | 支持（核心优势） |

### using 的模板别名优势

```cpp
// typedef 不支持模板别名
// typedef std::vector<T> Vec<T>;  // 错误！

// using 支持模板别名
template <typename T>
using Vec = std::vector<T>;

Vec<int> numbers;  // 等价于 std::vector<int>
```

## typeid 运算符

`typeid` 运算符用于获取表达式或类型的**运行时类型信息 (RTTI)**。

```cpp
#include <iostream>
#include <typeinfo>

int main() {
    int a = 10;
    double b = 3.14;

    std::cout << "Type of a: " << typeid(a).name() << std::endl;  // i (int)
    std::cout << "Type of b: " << typeid(b).name() << std::endl;  // d (double)

    // 比较类型
    if (typeid(a) == typeid(int)) {
        std::cout << "a is an int" << std::endl;
    }

    return 0;
}
```

### typeid 的用途

1. **调试**：查看变量的实际类型
2. **多态**：确定多态对象的真实类型
3. **类型比较**：在运行时比较两个类型是否相同

## 函数对象 (Functor)

函数对象是重载了 `operator()` 的类的实例，可以像函数一样被调用：

```cpp
#include <iostream>

// 函数对象类
class Adder {
private:
    int value;
public:
    Adder(int v) : value(v) {}

    int operator()(int x) const {
        return x + value;
    }
};

int main() {
    Adder add5(5);
    std::cout << add5(10) << std::endl;  // 输出: 15

    return 0;
}
```

### 函数对象的优势

1. **可以携带状态**：成员变量可以存储状态
2. **内联优化**：编译器更容易进行内联优化
3. **类型安全**：比函数指针更类型安全

## std::function

C++11 引入的 `std::function` 是一个通用的函数包装器，可以存储任何可调用对象：

```cpp
#include <functional>
#include <iostream>

int add(int a, int b) { return a + b; }

int main() {
    // 可以存储函数指针
    std::function<int(int, int)> func = add;
    std::cout << func(3, 4) << std::endl;  // 7

    // 可以存储 Lambda
    func = [](int a, int b) { return a * b; };
    std::cout << func(3, 4) << std::endl;  // 12

    return 0;
}
```

## 关键要点

1. **函数指针**：存储函数地址，允许运行时动态调用
2. **回调函数**：通过函数指针传递函数，实现控制反转
3. **using vs typedef**：`using` 语法更清晰，支持模板别名
4. **函数对象**：重载 `operator()` 的类，可携带状态
5. **std::function**：C++11 通用函数包装器
