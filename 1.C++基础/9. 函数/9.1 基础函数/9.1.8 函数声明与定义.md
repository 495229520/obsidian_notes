# 9.1.8 函数声明与定义

> C++ 编译器需要在使用函数之前知道函数的存在。函数声明告诉编译器函数的"接口"，而函数定义提供实际的实现。

---

## 1. 函数声明 (Declaration / Prototype)

### 目的

告诉编译器某个函数存在，以及它的**接口信息**：
- 返回类型
- 函数名
- 参数类型列表

### 语法

```cpp
返回类型 函数名(参数类型列表);
```

### 示例

```cpp
int add(int a, int b);                    // 声明一个名为 add 的函数
void printMessage(std::string msg);       // 声明一个名为 printMessage 的函数
double getRandomNumber();                 // 声明一个名为 getRandomNumber 的函数
```

### 特点

| 特性 | 说明 |
|------|------|
| **包含内容** | 返回类型、函数名、参数类型列表 |
| **结尾** | 以分号 `;` 结尾 |
| **参数名** | 可选（可以省略，只保留类型） |
| **规则** | 可多次声明（需一致） |
| **作用阶段** | [[1.1 程序编译与链接原理#1. 编译全过程 (The Build Process)|编译阶段]]（语法检查） |
| **常见位置** | 头文件 (`.h`/`.hpp`) 或 `.cpp` 文件开头 |

---

## 2. 函数定义 (Definition)

### 目的

提供函数的**实际代码实现**，是编译器生成机器代码的依据。

### 语法

```cpp
返回类型 函数名(参数类型列表) {
    // 函数体
}
```

### 示例

```cpp
int add(int a, int b) {
    return a + b;
}
```

### 特点

| 特性 | 说明 |
|------|------|
| **包含内容** | 返回类型、函数名、参数类型列表、**函数体** `{}` |
| **结尾** | 参数列表后无分号，包含 `{}` 代码块 |
| **规则** | **只能定义一次** (One Definition Rule, ODR) |
| **作用阶段** | [[1.1 程序编译与链接原理#4. 静态链接与动态链接|链接阶段]]（找到实现代码）、运行阶段（执行） |
| **常见位置** | 源文件 (`.cpp`) |

---

## 3. 声明与定义的对比

| 特性 | 函数声明 (Declaration / Prototype) | 函数定义 (Definition) |
|------|-----------------------------------|----------------------|
| **目的** | 告知编译器函数的存在和接口 | 提供函数的实际代码实现 |
| **包含内容** | 返回类型、函数名、参数类型列表 | 返回类型、函数名、参数列表、**函数体** `{}` |
| **结尾** | 以分号 `;` 结尾 | 参数列表后无分号，包含 `{}` 代码块 |
| **规则** | 可多次声明（需一致） | **只能定义一次** (ODR) |
| **作用阶段** | 编译阶段（语法检查） | 链接阶段（找到实现代码）、运行阶段（执行） |
| **常见位置** | 头文件 (`.h`/`.hpp`) 或 `.cpp` 文件开头 | 源文件 (`.cpp`) |

## 4. 内存分配与地址

> **定义涉及内存分配，声明不涉及内存分配**
> 函数名是内存地址的本质

### C++ 元素的内存区域

| C++ 元素 | 主要内存区域 | 说明 |
|----------|-------------|------|
| 函数代码（指令） | 代码区 (`.text`) | 只读、共享 |
| 全局变量（已初始化） | 全局静态区 (`.data`) | 程序生命周期，编译时分配 |
| 全局变量（未初始化/零） | 全局静态区 (`.bss`) | 程序生命周期，加载时清零 |
| 静态变量（全局/局部） | 全局静态区 (`.bss`) | 程序生命周期，作用域可能局部 |
| 字符串字面量（`"abc"`） | 常量存储区 (`.rodata`) 或 `.data` | 只读，程序生命周期 |
| const 全局静态常量 | 常量存储区 (`.rodata`) 或 `.data` | 通常只读 |
| 函数内的局部变量 | 栈区 | 函数/块生命周期，自动管理 |
| 函数参数 | 栈区（或寄存器） | 函数调用生命周期，自动管理 |
| `new` / `malloc` 分配的内存 | 堆区 | 手动管理生命周期（从分配到释放） |
| 类结构体对象 | 取决于对象如何创建 | 全局区→全局区；局部对象→栈；new 对象→堆 |
| 类的成员函数代码 | 代码区 | 与普通函数代码一样 |

### 内存地址示意图

```
                      理解函数的定义
                            ↓
┌───────────────────────────────────────────────────────────┐
│ 函数的定义是我们通过编译器把我们想要                      │
│ 计算机执行的C/C++命令翻译计算出地址                       │  x86-32位windows操作系统下内存分配图
│ 就是我们执行的C/C++命令翻译计算出地址                     │
│                                                           │  0-0xFFFF
│ 函数这个计算机器也是要放在内存中的因为                    │  ┌──────────┐
│ 计算机只识别0和1                                          │  │ 受保护针区 │
│                                                           │  ├──────────┤
│                                                           │  │ 代码区    │
└───────────────────────────────────────────────────────────┘  ├──────────┤
                                                               │ 全局数据区 │
int add(int a, int b);                                         ├──────────┤
int main()                                                     │ 堆区      │
{                          000010000-7FFFFFFF                  ├──────────┤
    int c=250;               用户内存区                        │ 堆区      │
    int* b=new int[100];                                       ├──────────┤
    std::cout<<add(c,300);                                     │ 栈区      │
}                                                              └──────────┘

int add(int a, int b)
{
    static int x;
    return a+b;
}
```

## 5. 为什么需要分离声明和定义？

### 5.1 支持多文件项目

在大型项目中，代码分布在多个源文件中。只要在一个工程中，相同的函数名在不同的源文件中也会被调用（不是函数重载，但是 C++ 支持函数重载），声明有多次，定义只有一次。

```cpp
// math_utils.h - 头文件（声明）
int add(int a, int b);

// math_utils.cpp - 源文件（定义）
#include "math_utils.h"
int add(int a, int b) {
    return a + b;
}

// main.cpp - 使用
#include "math_utils.h"
int main() {
    int result = add(3, 5);  // 正确：编译器知道 add 的接口
    return 0;
}
```

### 5.2 编译效率

- 修改函数实现时，只需重新编译包含定义的 `.cpp` 文件
- 其他使用该函数的文件只需重新链接，无需重新编译

## 6. 一定义规则 (One Definition Rule, ODR)

> 一个函数在整个程序中**只能被定义一次**。

### 违反 ODR 的错误

```cpp
// file1.cpp
int add(int a, int b) { return a + b; }

// file2.cpp
int add(int a, int b) { return a + b; }  // 错误！重复定义
```

链接时会报错：`multiple definition of 'add'`

### 例外：内联函数

内联函数（`inline`）可以在多个翻译单元中定义，但每个定义必须完全相同：

```cpp
// 在头文件中
inline int square(int x) {
    return x * x;
}
```

---

## 7. 关键要点

1. **声明**：告诉编译器函数的接口（返回类型、名称、参数类型）
2. **定义**：提供函数的实际实现代码
3. **ODR**：一个函数只能定义一次，但可以声明多次
4. **头文件**：通常放声明，源文件放定义
5. **内存分配**：定义时分配内存，声明不分配
