# 10.1 C 和 C++ 联调

## 1. 简介

在实际项目中，经常需要在 C++ 代码中调用 C 语言编写的库，或者在 C 代码中调用 C++ 编写的函数。由于 C 和 C++ 的[[11.2 编译器基本概念#名称修饰|名称修饰 (Name Mangling)]]机制不同，直接混合编译会导致链接错误。

---

## 2. 名称修饰的差异

### C 语言

C 编译器**不进行名称修饰**，函数名在目标文件中保持原样：

```c
// C 语言
void print(int x);   // 符号名: _print 或 print
```

### C++ 语言

C++ 编译器为了支持[[9.2.1 函数重载|函数重载]]，会对函数名进行**名称修饰**：

```cpp
// C++ 语言
void print(int x);      // 符号名可能是: _Z5printi
void print(double x);   // 符号名可能是: _Z5printd
```

---

## 3. `extern "C"` 关键字

`extern "C"` 告诉 C++ 编译器使用 C 语言的链接规范（不进行名称修饰）。

### 语法

```cpp
// 单个函数
extern "C" void c_function();

// 多个函数
extern "C" {
    void func1();
    void func2(int x);
    int func3(const char* str);
}
```

---

## 4. C++ 调用 C 代码

### 场景

在 C++ 项目中使用 C 语言编写的库（如系统库、第三方 C 库）。

### 方法

```cpp
// my_c_lib.h - C 语言头文件
#ifndef MY_C_LIB_H
#define MY_C_LIB_H

#ifdef __cplusplus
extern "C" {
#endif

void c_print(const char* msg);
int c_add(int a, int b);

#ifdef __cplusplus
}
#endif

#endif
```

```c
// my_c_lib.c - C 语言实现
#include "my_c_lib.h"
#include <stdio.h>

void c_print(const char* msg) {
    printf("%s\n", msg);
}

int c_add(int a, int b) {
    return a + b;
}
```

```cpp
// main.cpp - C++ 调用
#include "my_c_lib.h"

int main() {
    c_print("Hello from C!");
    int result = c_add(3, 5);
    return 0;
}
```

### `__cplusplus` 宏

- `__cplusplus` 是 C++ 编译器[[11.7 预定义宏|预定义的宏]]
- C 编译器不定义此宏
- 用于创建 C/C++ 兼容的头文件

## 5. C 调用 C++ 代码

### 场景

在 C 项目中使用 C++ 编写的功能。

### 方法

需要为 C++ 函数创建 C 兼容的包装接口：

```cpp
// cpp_lib.h - C 兼容的头文件
#ifndef CPP_LIB_H
#define CPP_LIB_H

#ifdef __cplusplus
extern "C" {
#endif

// C 兼容的接口
void* create_object();
void destroy_object(void* obj);
void object_do_something(void* obj, int value);

#ifdef __cplusplus
}
#endif

#endif
```

```cpp
// cpp_lib.cpp - C++ 实现
#include "cpp_lib.h"

class MyClass {
public:
    void doSomething(int value) {
        // 实现...
    }
};

extern "C" {
    void* create_object() {
        return new MyClass();
    }

    void destroy_object(void* obj) {
        delete static_cast<MyClass*>(obj);
    }

    void object_do_something(void* obj, int value) {
        static_cast<MyClass*>(obj)->doSomething(value);
    }
}
```

```c
// main.c - C 调用
#include "cpp_lib.h"

int main() {
    void* obj = create_object();
    object_do_something(obj, 42);
    destroy_object(obj);
    return 0;
}
```

## 6. 编译命令

### 混合编译示例

```bash
# 编译 C 文件
gcc -c my_c_lib.c -o my_c_lib.o

# 编译 C++ 文件
g++ -c main.cpp -o main.o

# 链接（使用 g++ 链接以包含 C++ 运行时库）
g++ my_c_lib.o main.o -o program
```

---

## 7. 注意事项

| 注意点 | 说明 |
|--------|------|
| **链接器选择** | 混合编译时，使用 `g++` 进行链接，以包含 C++ 标准库 |
| **异常处理** | C 代码不支持 C++ 异常，需要在边界处捕获 |
| **内存管理** | `new/delete` 和 `malloc/free` 不要混用 |
| **全局对象** | C++ 全局对象的构造/析构顺序需要注意 |
| **头文件保护** | 使用 `#ifdef __cplusplus` 保护 `extern "C"` |

---

## 8. 关键要点

1. **名称修饰差异**：C 不修饰，C++ 修饰函数名以支持重载
2. **`extern "C"`**：告诉 C++ 编译器使用 C 链接规范
3. **`__cplusplus` 宏**：用于检测是否为 C++ 编译环境
4. **C++ 调用 C**：在 C 头文件中添加 `extern "C"` 包装
5. **C 调用 C++**：为 C++ 代码创建 C 兼容的包装函数
