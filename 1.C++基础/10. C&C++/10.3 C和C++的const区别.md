# 10.3 C 和 C++ 的 const 区别

## 1. 概述

`const` 关键字在 C 和 C++ 中有着本质的区别：

| 特性 | C 语言 | C++ |
|------|--------|-----|
| **名称** | 常变量 | 常量 |
| **是否必须初始化** | 否 | 是 |
| **能否作为左值** | 可以 | 不可以 |
| **存储方式** | 存储在内存中 | 编译期常量折叠 |

---

## 2. 案例对比

### 2.1 C 语言中的 const

```c
void main()
{
    const int a;    // C 允许不初始化
    a = 20;         // C 允许后续赋值（虽然不推荐）
}
```

**特点**：C 中的 `const` 修饰的变量**可以不初始化**，本质上仍然是一个变量，只是被标记为"不应该修改"，编译器可能只给出警告而非错误。

### 2.2 C++ 中的 const

```cpp
int main()
{
    int a = 10;
    a = 20;         // OK：普通变量可以修改

    const int b = 20;       // C++ 必须初始化
    // b = 30;              // 错误！const 修饰的变量不能作为左值
}
```

**特点**：C++ 中 `const` 修饰的变量**必须初始化**，且**不能作为左值**被修改。

---

## 3. 指针绕过 const 的问题

### 3.1 通过[[6.1基本指针#常量指针 (Pointer to Constant)|指针]]修改 const 变量

虽然 C++ 的 `const` 变量不能直接修改，但可以通过指针绕过编译器检查：

```cpp
const int a = 10;

int* p = (int*)&a;   // 强制类型转换获取地址
*p = 30;             // 通过指针修改内存中的值
```

> **警告**：不要将常量的地址泄漏给普通的[[6.1基本指针|指针]]或普通的[[6.4 引用|引用]]，因为这样可以间接修改常量的值，导致未定义行为。

---

## 4. 内存本质的差异

这是 C 和 C++ 中 `const` 最本质的区别。

### 4.1 C 语言：按变量处理

在 C 中，`const` 就是被**当做一个变量**来编译生成指令：

```c
void main()
{
    const int a = 20;

    int* p = (int*)&a;
    *p = 30;

    // 结果都是 30
    printf("%d %d %d \n", a, *p, *(&a));  // 输出: 30 30 30
}
```

**解释**：C 编译器将 `a` 视为普通变量存储在内存中，所有对 `a` 的访问都是从内存读取。当通过指针修改后，`a`、`*p`、`*(&a)` 都读取的是同一块内存，所以都是 30。

### 4.2 C++ 语言：常量折叠优化

在 C++ 中，编译器会进行**常量折叠（Constant Folding）**优化：所有出现 `const` 常量名字的地方，都被常量的初始化值替换了。

```cpp
int main()
{
    const int a = 20;

    int* p = (int*)&a;
    *p = 30;

    // 被编译器替换为: printf("%d %d %d \n", 20, *p, 20);
    // 因此输出的是 20 30 20 !
    printf("%d %d %d \n", a, *p, *(&a));
}
```

**解释**：
- `a` → 被替换为字面量 `20`（常量折叠）
- `*p` → 从内存读取，得到修改后的值 `30`
- `*(&a)` → 编译器优化为 `20`

> 这说明 C++ 编译器在编译期就将 `const` 常量的值"内联"到了使用它的地方，类似于[[11.4 define|宏替换]]的效果，但更加类型安全。

---

## 5. 常量折叠的底层原理

常量折叠是[[11.2 编译器基本概念#7. 编译器优化|编译器优化]]的一种：

```
源代码                    编译后等效代码
------------------------  ------------------------
const int a = 20;         // a 的值在编译期已知
printf("%d", a);    →     printf("%d", 20);  // 直接替换为字面量
```

### 什么时候不会折叠？

当 `const` 变量的值在编译期无法确定时，不会进行常量折叠：

```cpp
int getValue() { return 20; }

int main() {
    const int a = getValue();  // 运行时才能确定值
    // 此时 a 不会被常量折叠，行为更接近 C 的 const
}
```

---

## 6. 对比总结

| 方面 | C 语言 (常变量) | C++ (常量) |
|------|-----------------|------------|
| **语义** | "请不要修改"的变量 | 真正的常量 |
| **初始化** | 可选 | 必须 |
| **作为左值** | 允许（可能警告） | 禁止（编译错误） |
| **内存存储** | 一定存储在内存 | 可能只存在于编译期 |
| **运行时修改** | 可以通过指针修改 | 修改导致未定义行为 |
| **优化** | 无特殊优化 | 常量折叠 |

---

## 7. 最佳实践

1. **始终初始化 const 变量**：即使在 C 中也应该这样做
2. **不要用指针绕过 const**：这会导致未定义行为
3. **理解常量折叠**：在 C++ 中，不要期望 `const` 变量一定有内存地址
4. **使用 `constexpr`（C++11）**：如果需要编译期常量，使用 `constexpr` 更明确

```cpp
constexpr int SIZE = 100;  // C++11: 保证编译期常量
```

---

## 8. 关键要点

1. **C 的 const 是"常变量"**：本质是变量，只是被标记为不应修改
2. **C++ 的 const 是"常量"**：编译器会进行常量折叠优化
3. **常量折叠**：C++ 编译器在编译期将 const 常量替换为字面值
4. **指针绕过的风险**：虽然语法上可以，但会导致 C++ 中的未定义行为
5. **初始化要求**：C++ 强制要求 const 必须初始化
