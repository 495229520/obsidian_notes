# 6.2 动态内存分配与释放

> 本节内容与 [[6.1基本指针]] 中的动态内存分配部分相关，并为 [[6.3 动态内存分配的风险]] 提供基础。

## C 语言风格：malloc 与 free

### malloc 函数

```c
void* malloc(size_t size);
```

- `malloc` 在**堆**上分配指定字节数的内存
- 返回 `void*` 类型指针，需要强制类型转换
- 分配失败返回 `NULL`

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int* ptr = (int*)malloc(sizeof(int));
    // 使用完毕后调用 free(ptr) 释放内存
    return 0;
}
```

### free 函数

```c
void free(void* _Block);
```

- `free` 释放由 `malloc`/`calloc`/`realloc` 分配的内存
- 释放后指针变成**悬空指针**，建议置为 `NULL`

## C++ 风格：new 与 delete

### new 运算符

```cpp
void* operator new(size_t size);        // 分配单个对象
void* operator new[](size_t size);      // 分配数组
```

- `new` 在堆上分配内存并调用构造函数
- 分配失败抛出 `std::bad_alloc` 异常（而非返回 NULL）

```cpp
#include <iostream>
using namespace std;

int main() {
    // 分配单个 int
    int* ptr = new int;
    *ptr = 10;
    delete ptr;

    // 分配 int 数组
    int* arr = new int[100];
    arr[0] = 250;
    delete[] arr;

    return 0;
}
```

### delete 运算符

```cpp
void operator delete(void* _Block);      // 释放单个对象
void operator delete[](void* _Block);    // 释放数组
```

- `delete` 先调用析构函数，再释放内存
- **数组必须用 `delete[]` 释放**

---

## 内存释放规则

| 分配方式 | 释放方式 | 示例 |
|----------|----------|------|
| `malloc` | `free` | `free(p);` |
| `new` (单对象) | `delete` | `delete p;` |
| `new[]` (数组) | `delete[]` | `delete[] p;` |

```cpp
// 单个对象
int* p = new int;
delete p;

// 数组
int* p = new int[100];
delete[] p;
```

> **重要**：`malloc` 分配的内存不可用 `delete` 释放，`new` 分配的内存不可用 `free` 释放。混用会导致未定义行为。

## memcpy：内存拷贝

```cpp
void* memcpy(void* Dst, const void* Src, size_t Size);
```

- 从 `Src` 复制 `Size` 字节到 `Dst`
- **不检查重叠**，重叠时应使用 `memmove`

```cpp
int a[5] = {1001, 1002, 1003, 1004, 1005};
int* pnew = new int[5];
memcpy(pnew, a, sizeof(int) * 5);

// 老方法：使用 for 循环进行复制
int a5[5] = {1001, 1002, 1003, 1004, 1005};
int* b = new int[5];
for (int i = 0; i < 5; i++) {
    b[i] = a[i];
}

// 新方法：使用 memcpy 进行复制
int a5[5] = {1001, 1002, 1003, 1004, 1005};
int* b = new int[5];
memcpy(b, a, 5 * sizeof(int));
```

## memset：内存初始化

```cpp
void* memset(void* s, int c, size_t n);
```

- 将内存块的前 `n` 个字节设置为值 `c`
- 通常用于将内存清零或初始化

> **注意**：`memset` 是按字节进行填充的，当用于初始化非字符类型数据时要特别小心。

### memset 的陷阱

```cpp
#include <cstdio.h>
#include <cstring.h>

int main() {
    char str[10];
    // 将 str 数组前 10 个字节都设置为字符 'A'
    memset(str, 'A', 10);
    for (int i = 0; i < 10; i++) {
        printf("%c ", str[i]);
    }
    printf("\n");
    return 0;
}
```

**整型数组的问题**：

```cpp
#include <cstdio.h>
#include <cstring.h>

int main() {
    int arr[5];
    // 将 arr 数组前 5 * sizeof(int) 个字节都设置为 0
    memset(arr, 0, 5 * sizeof(int));
    for (int i = 0; i < 5; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
    return 0;
}
```

**为什么用 `memset` 将整数数组初始化为 1 会得到错误结果？**

由于 `memset` 是按字节填充的，每个 `int` 类型占 4 个字节，结果会是：

```
0000 0001 0000 0001 0000 0001 0000 0001
= 16,843,009 (而不是 1)
```

## 关键要点

1. **配对使用**：`malloc`/`free`，`new`/`delete`，`new[]`/`delete[]`
2. **不要混用**：C 和 C++ 的内存分配/释放函数不能混用
3. **数组释放**：使用 `new[]` 分配的数组必须用 `delete[]` 释放
4. **memset 陷阱**：按字节填充，对非字符类型要小心
5. **释放后置 NULL**：避免产生悬空指针
