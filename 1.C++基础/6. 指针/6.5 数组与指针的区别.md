# 6.5 数组与指针的区别

> 本节深入探讨数组与 [[6.1基本指针|指针]] 的关系与区别，补充 [[6.1基本指针]] 中关于数组指针的内容。

---

## 数组 vs 指针

| 特性 | 数组 | 指针 |
|------|------|------|
| **定义** | 一段**连续的内存空间**，用于存储相同类型的数据 | 一个**变量**，存储的是内存地址，可以指向任何类型的数据包括数组 |
| **内存分配** | 声明时**静态分配**内存空间 | 可以**动态分配**内存空间，如 `int* p = new int` |
| **赋值** | 数组名**不能被重新赋值**，始终代表数组的起始地址 | 指针**可以被重新赋值**，指向不同的内存空间 |
| **sizeof** | 返回整个数组的大小 | 返回指针本身的大小（4或8字节） |

---

## 联系

数组和指针有着密切的关系：

1. **数组名本质上是指针**：数组名可以看作是一个指向数组第一个元素的**常量指针**

2. **指针可以访问数组元素**：可以通过指针的算术运算来访问数组中的元素
   - 例如，`p` 是一个指向数组 `a` 的指针，`p + i` 指向 `a[i]`

3. **数组作为函数参数会退化为指针**：当数组作为函数参数传递时，会退化为指向数组第一个元素的指针

---

## 示例代码

### sizeof 的区别

```cpp
#include <iostream>

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int* ptr = arr;

    std::cout << "sizeof(arr) = " << sizeof(arr) << std::endl;  // 20 (5 * 4)
    std::cout << "sizeof(ptr) = " << sizeof(ptr) << std::endl;  // 4 或 8

    return 0;
}
```

### 数组名不能重新赋值

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int brr[5] = {6, 7, 8, 9, 10};

// arr = brr;  // 错误！数组名是常量，不能重新赋值

int* ptr = arr;
ptr = brr;     // 正确，指针可以重新赋值
```

### 指针访问数组元素

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int* ptr = arr;

// 以下两种方式等价
std::cout << arr[2] << std::endl;   // 3
std::cout << *(ptr + 2) << std::endl; // 3
std::cout << ptr[2] << std::endl;   // 3
```

---

## 函数参数中的退化

```cpp
// 以下三种声明等价
void func(int arr[]);
void func(int arr[10]);
void func(int* arr);

// 数组传入函数后退化为指针，无法获取原数组大小
void printSize(int arr[]) {
    // sizeof(arr) 返回指针大小，不是数组大小
    std::cout << sizeof(arr) << std::endl;  // 4 或 8
}
```

---

## 关键要点

1. **数组是静态的**：大小在编译时确定，不能改变
2. **指针是动态的**：可以指向不同的内存地址
3. **数组名是常量指针**：指向首元素，不能修改
4. **函数参数退化**：数组作为参数传递时会退化为指针
5. **sizeof 行为不同**：数组返回总大小，指针返回指针大小
