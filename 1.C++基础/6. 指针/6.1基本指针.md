# 6.1 基本指针

## 内存地址与指针

C/C++ 通过了指针直接操作内存的方式，让程序运算更加高效，但这种操作有危险性，已逐渐被其他安全指针/引用等手段替代。

### 指针的本质

- **内存地址**：内存中每个字节都有唯一的地址编号
- **指针变量**：存储内存地址的变量，通过指针可以间接访问该地址上的数据

### 指针的声明方式

```cpp
// 数据类型*是一个整体，代表这是一个指向该类型的指针
int *pSomething;

// 推荐声明方式：类型和*放在一起
int* a;
int* b;

// 不建议的方式（容易误解）
int *a;
int *b;
```

## 指针数组

```cpp
// 普通数组
int arr[5] = {1, 2, 3, 4, 5};
int* ptr = arr;
int* ptr2 = &arr[0];  // 等价于上面

// 指针数组：存储指针的数组
int* ptrArray[5] = { ptr, ptr2, arr, &arr[1] };
int* p = ptrArray[0];
std::cout << "值：" << *p << "，&ptrArray[0] = " << &ptrArray[0] << std::endl;
```

## 动态内存分配

### 使用 `new` 和 `delete`

```cpp
// 动态分配单个对象
int* ptrArray2 = new int(5);  // 分配并初始化为5
// ...使用后需要释放
delete ptrArray2;

// 动态分配数组
int studentId[5] = { 1001, 1002, 1003, 1004, 1005 };
int* ptrArray3 = new int[5];  // 分配5个int的数组

// 释放动态分配的内存
delete ptrArray3;     // 错误！数组应该用 delete[]
delete[] ptrArray3;   // 正确：释放数组内存
```

> **注意**：`new int[5]` 分配的内存必须用 `delete[]` 释放，而不是 `delete`。

### `new` 的原理

- `new` 本质调用 `operator new`，底层调用 `malloc` 函数
- `delete` 本质调用 `operator delete`，底层调用 `free` 函数

```cpp
int* ptrArray4 = new int[5];
ptrArray4 = new int[7];  // 内存泄漏！原来的5个int空间未释放
```

### 数组遍历

```cpp
for (int i = 0; i < 5; i++) {
    std::cout << ptrArray4[i] << std::endl;
}
```

## 指针大小

在 32 位系统中，指针占用 4 字节；在 64 位系统中，指针占用 8 字节。

- 每个系统的指针占用空间是固定的
- 无论 `char*`、`int*`、`float*`、`double*` 等，指针本身大小相同
- 因为指针存储的是内存地址，只需存储足够表示系统最大内存的位数

```cpp
int* ptrExample;
std::cout << sizeof(ptrExample) << std::endl;  // 32位：4，64位：8
```

## 常量指针与指针常量

### 区分方法

**记忆技巧**：`const` 修饰其右边紧邻的内容

| 代码 | 多级指针 | 内存地址 |
|------|----------|----------|
| `int a[500]` | 500 | 0x50000 |
| `int* ptr[&a]` | 0x50000 | 0x50100 |
| `int** pptr[&ptr]` | 0x50100 | 0x50200 |
| `int*** ppptr[&pptr]` | 0x50200 | 0x50300 |

### 常量指针 (Pointer to Constant)

**指向常量的指针**：不能通过指针修改所指向的值，但指针本身可以改变指向。

```cpp
const int* ptr;
// 或
int const* ptr;

// 示例
const int a = 1000;
const int* ptr1 = &a;
// *ptr1 = 2000;  // 错误！不能通过ptr1修改值
int b = 2000;
ptr1 = &b;        // 正确，可以改变指向
```

### 指针常量 (Constant Pointer)

**常量的指针**：指针本身是常量，不能改变指向，但可以通过指针修改所指向的值。

```cpp
int* const ptr;

// 示例
int a = 1000;
int* const ptr2 = &a;
*ptr2 = 2000;     // 正确，可以修改值
// ptr2 = &b;    // 错误！不能改变指向
```

### 常量指针常量

**既是常量指针又是指针常量**：既不能改变指向，也不能通过指针修改值。

```cpp
const int* const ptr;

// 示例
const int a = 1000;
const int* const ptr3 = &a;
// *ptr3 = 2000;  // 错误！
// ptr3 = &b;     // 错误！
```

### 总结表格

| 声明方式 | 能否修改指向 | 能否通过指针修改值 |
|----------|--------------|-------------------|
| `int* ptr` | ✓ | ✓ |
| `const int* ptr` | ✓ | ✗ |
| `int* const ptr` | ✗ | ✓ |
| `const int* const ptr` | ✗ | ✗ |

## 常量指针的应用

以下代码展示将普通变量地址赋给常量指针后的行为：

```cpp
int a = 1000;
int b = 3000;
const int* ptrA = (int*)&a;  // 将a的地址赋给常量指针
*ptrA = 9999;  // 通过类型转换后可以修改
std::cout << "ptrA = " << ptrA << std::endl;
std::cout << "a = " << a << std::endl;
```

## 指针与数组

### 数组跟指针一样，底层是一个连续的内存地址

```cpp
int* ptrA = &a[0];

// 指针运算
int* ptrA = &a[0];
ptr[xxx]  // 就是地址为xxx的指针
&a[0]     // a[0]的地址
edx = &ptr;  // ptr的地址为 edp+ecx-1c
```

### 汇编层面的数组访问

```asm
int* ptrA = { &a[0] };
mov     eax, 4
imul    ecx, eax, 0
lea     edx, [ebp+ecx-1Ch]    ; edx = a[0]的地址
mov     dword ptr [ebp-28h], edx  ; ptrA = a[0]的地址

*ptrA = 5;
mov     eax, dword ptr [ebp-28h]  ; 指针指向的a[0]地址放给eax寄存器
mov     dword ptr [eax], 5        ; 地址为a[0]地址的指针存放5这个值
```

**总结**：`[ebp-28]` 是 `ptrA` 指针的地址，这个指针存放了 `edx`，也就是 `a[0]` 的地址，然后通过 `ptrA` 指针将 5 存入 `a[0]` 的地址中。

### 数组元素访问的汇编分析

```cpp
a[0] = 5;
// 汇编：直接看最后，就是把 5 存入 a[0] 的地址中去，a[0] 的地址为 ebp+ecx-1c
mov     eax, 4
imul    ecx, eax, 0
mov     dword ptr [ebp+ecx-1Ch], 5

a[1] = 5;
// shl eax,0 是左位移，这里是左位移 2的0次方，也就是 1，此时 eax 为 4
// a[1] 的地址就是 [ebp+eax-1c]，这里就是 a[0] 的地址加 4
mov     eax, 4
shl     eax, 0
mov     dword ptr [ebp+eax-1Ch], 5

a[2] = 5;
// shl eax,1 就是 eax 向左移动一位，0100变为1000 (8)
// a[2] 的地址就是 [ebp+eax-1c]，这里就是 a[0] 的地址加 8
mov     eax, 4
shl     eax, 1
mov     dword ptr [ebp+eax-1Ch], 5

a[3] = 5;
// 这里就明显不一样了，ecx 的地址是 eax*3，也就是 12
mov     eax, 4
imul    ecx, eax, 3
mov     dword ptr [ecx], 5
```

**结论**：从这里可以看出数组实现不同 index 的搜索和存放数据，本质也是通过修改地址来实现的。

## 数组指针

**数组跟指针一样，底层是一个连续的内存地址。**

```cpp
int a[5];
int* ptrA = { a };
a[0] = 5;
a[1] = 5 + 1;

std::cout << &a[0] << std::endl;
// 输出: 5
std::cout << *a << std::endl;
// 输出: 6
std::cout << ptrA[1] << std::endl;
// 输出: 20，因为 a 初始化为 5，5 * 4 = 20
std::cout << sizeof(a) << std::endl;
// 输出: 4，因为 ptrA 指针在 32 位的系统里面，占据 4 个字节
std::cout << sizeof(ptrA) << std::endl;
```

## 多维数组

### 数组的本质是连续的内存区域

```cpp
int test[2][5] = {
    {1001, 1002, 1003, 1004, 1005},
    {2001, 2002, 2003, 2004, 2005}
};

int* ptestA[5];           // 五个int类型的指针
// 指向包含5个整数的数组的指针，这里就是二维数组的第一维
// 二维数组的第一维其实就是指向二维的指针
// test[1] = ptest[1]
int (*ptest)[5]{test};

std::cout << test[1][4] << std::endl;
std::cout << ptest[0][4] << std::endl;
```

### 多维数组的内存布局

从这里就可以看出来，多维数组在底层还是连续的内存空间。

**一维数组 `int a[10]`：**

| a[0] | a[1] | a[2] | a[3] | a[4] | a[5] | a[6] | a[7] | a[8] | a[9] | a[10] |
|------|------|------|------|------|------|------|------|------|------|-------|

**二维数组 `int a[2][5]`：**

| a[0,0] | a[0,1] | a[0,2] | a[0,3] | a[0,4] |
|--------|--------|--------|--------|--------|
| a[1,0] | a[1,1] | a[1,2] | a[1,3] | a[1,4] |

**二维数组 `int a[5][2]`：**

| a[0][0] | a[0][1] |
|---------|---------|
| a[1][0] | a[1][1] |
| a[2][0] | a[2][1] |
| a[3][0] | a[3][1] |
| a[4][0] | a[4][1] |

> **要点**：在这里使用指针 `ptest` 指向多维数组时，多维数组在底层还是连续的内存空间。

## 关键要点

1. **指针本质**：存储内存地址的变量
2. **动态内存**：`new` 分配，`delete`/`delete[]` 释放，注意配对使用
3. **常量指针 vs 指针常量**：`const` 修饰谁，谁就不能变
4. **数组与指针**：数组名本质是指向首元素的指针，底层是连续内存
5. **多维数组**：在内存中仍然是连续存储的
