# 6.6 堆与栈的区别

> 本节解释 [[6.2 动态内存分配与释放|动态内存分配]] 时涉及的堆内存，以及局部变量使用的栈内存。

---

## 栈 vs 堆 对比

| 特性 | 栈 (Stack) | 堆 (Heap) |
|------|------------|-----------|
| **分配方式** | 编译器**自动分配和释放** | 程序员**手动分配和释放** |
| **存储对象** | 局部变量、函数参数、函数调用信息等 | 动态创建的对象、大型数据结构等 |
| **生存周期** | 作用域结束时**自动释放** | 由程序员决定 |
| **访问速度** | **快速** | 较慢 |
| **大小限制** | **有限**（通常几 MB） | 较大，可动态调整 |
| **内存管理** | **自动** | **手动** |

---

## 内存分配位置

- **静态申请的内存**（例如：局部变量、全局变量）通常存储在**栈**中
- **动态申请的内存**（例如：使用 `new` 运算符创建的对象）通常存储在**堆**中

### 示例

```cpp
int* d = new int[5];
```

| 变量/内存区域 | 存储位置 | 说明 |
|---------------|----------|------|
| `d` | 栈 | 指针变量，存储动态分配内存的首地址 |
| `new int[5]` | 堆 | 动态分配的内存块，用于存储 5 个整数 |
| `&d` | 栈 | 指针变量 `d` 的地址，它的值指向存储 `d` 变量的内存位置 |

> **关键理解**：`d` 就是存放动态分配内存的首地址，而 `&d` 存放的是 `d` 这个变量的地址。

## 栈指针：esp 和 ebp

栈空间的分配和访问是通过 **ebp**（基址指针）和 **esp**（栈指针）寄存器以及减法操作来实现的。

### 寄存器说明

- **esp (Stack Pointer - 栈指针)**：寄存器**始终指向栈顶**的位置
  - 它会随着压栈和出栈操作动态变化
  - esp 的值越小，栈就越大（因为栈向低地址增长）

- **ebp (Base Pointer / Frame Pointer - 基址指针)**：寄存器通常用于**指向当前函数栈帧的底部**（或者说基地址）
  - 在函数调用时，ebp 会被设置为一个固定的位置，通常是函数栈帧的起始位置
  - ebp 的主要目的是提供一个稳定的参考点，方便在函数内部访问局部变量、函数参数以及返回地址等数据

### 栈操作

- **压栈 (Push)**：只需要将数据写入 esp 指向的地址，然后将 esp 的值减小（栈向低地址增长）
- **出栈 (Pop)**：只需要将 esp 指向的地址的数据读取出来，然后将 esp 的值增加

### 栈结构图

```
         栈顶
    ┌─────────┐
    │         │  ← esp (小地址)
    │         │
    │  局部   │
    │  变量   │
    │         │
    │         │
    ├─────────┤
    │         │  ← ebp (大地址)
    │  栈帧   │
    │  底部   │
    └─────────┘
         栈底

    esp ↑ 小
        │
        │ 栈增长方向
        │
    ebp ↓ 大
```

## 代码示例

### 栈上分配

```cpp
void func() {
    int a = 10;      // 栈上分配
    int arr[5];      // 栈上分配
}  // 函数结束，a 和 arr 自动释放
```

### 堆上分配

```cpp
void func() {
    int* p = new int(10);    // p 在栈上，*p 指向的内存在堆上
    int* arr = new int[5];   // arr 在栈上，数组内容在堆上

    delete p;       // 手动释放
    delete[] arr;   // 手动释放
}
```

---

## 关键要点

1. **栈是自动管理的**：编译器负责分配和释放
2. **堆是手动管理的**：程序员负责 `new`/`delete`
3. **栈速度快但空间有限**：适合小型、短生命周期的数据
4. **堆空间大但需要管理**：适合大型、长生命周期的数据
5. **指针变量在栈上**：但它指向的动态内存在堆上
