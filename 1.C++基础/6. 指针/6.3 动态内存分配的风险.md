# 6.3 动态内存分配的风险

> 本节是 [[6.2 动态内存分配与释放]] 的延续，讨论动态内存管理中常见的风险和错误。

动态内存分配虽然灵活，但如果管理不当会导致严重的程序问题。以下是常见的风险类型。

---

## 1. 内存泄漏 (Memory Leak)

**定义**：分配内存后，如果没有及时释放内存，就会导致内存泄漏。此时该指针没有指向正确地址或置 `NULL`，就会导致无法访问内存。

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int* ptr = (int*)malloc(sizeof(int));
    // 没有调用 free(ptr) 释放内存
    return 0;
}
```

### 后果

- 程序占用内存持续增长
- 长时间运行可能耗尽系统内存
- 导致程序崩溃或系统变慢

---

## 2. 悬空指针 (Dangling Pointer)

**定义**：当释放了一块动态分配的内存后，指向该内存的指针仍然存在，但指向的是无效地址。

```cpp
#include <iostream>
using namespace std;

int main() {
    int* ptr = new int;
    *ptr = 10;
    delete ptr;
    // 这里设计代码是想引用 ptr 指向的值，但此时 ptr 指向的是空指针
    // 下面这行代码会导致未定义行为
    // cout << *ptr << endl;  // 危险！

    // 正确做法：释放后置 NULL
    ptr = nullptr;

    return 0;
}
```

### 如何避免

1. 释放内存后立即将指针置为 `nullptr`
2. 使用智能指针（C++11 及以后）

---

## 3. 重复释放 (Double Free)

**定义**：意思是需要使用 `delete` 释放的，但是却使用了 `delete`，还没有释放。

```cpp
int main() {
    int* p = new int[100];
    int* pold = p;
    p[0] = 250;
    delete[] p;
    delete[] pold;  // 错误！pold 和 p 指向同一块内存，重复释放
}
```

### 正确示例

```cpp
int main() {
    int* p = new int[100];
    p[0] = 250;
    delete[] p;
    p = nullptr;  // 释放后置空
}
```

---

## 4. 错误的释放

**定义**：释放了内存，但使用了错误的释放方式。

```cpp
int main() {
    // 错误：用 delete 释放 new[] 分配的内存
    int* p = new int[100];
    delete p;  // 应该用 delete[] p;
}
```

### 释放规则

| 分配方式 | 正确释放方式 |
|----------|--------------|
| `malloc` | `free` |
| `new` | `delete` |
| `new[]` | `delete[]` |

> **重要**：`malloc` 分配的内存不可用 `delete` 释放，`new` 分配的内存不可用 `free` 释放。

---

## 5. 内存拷贝风险

使用 `memcpy` 拷贝内存时需要注意：

```cpp
int a[5] = {1001, 1002, 1003, 1004, 1005};
int* pnew = new int[5];
memcpy(pnew, a, sizeof(int) * 5);
```

### 风险点

- 源和目标内存区域重叠
- 拷贝大小计算错误
- 目标缓冲区太小（缓冲区溢出）

---

## 6. 内存初始化风险

使用 `memset` 初始化内存时的注意事项：

```cpp
int arr[5];
// 正确：初始化为 0
memset(arr, 0, 5 * sizeof(int));

// 错误：试图初始化为 1
memset(arr, 1, 5 * sizeof(int));  // 每个字节都是 1，结果不是 1
```

> 由于 `memset` 是按字节设置，将 `int` 数组每个字节设为 1 后，每个 `int` 的值是 `0x01010101` = 16843009。

---

## 最佳实践

1. **及时释放**：分配的内存使用完毕后立即释放
2. **释放后置空**：`delete` 后将指针设为 `nullptr`
3. **配对使用**：`new`/`delete`，`new[]`/`delete[]`，`malloc`/`free`
4. **使用智能指针**：[[独享智能指针|std::unique_ptr]]、[[共享智能指针|std::shared_ptr]]（参见现代 C++）
5. **RAII 原则**：资源获取即初始化，利用对象生命周期管理资源
