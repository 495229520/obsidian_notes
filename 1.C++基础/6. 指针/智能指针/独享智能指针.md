# 独享智能指针 (unique_ptr)

> `unique_ptr` 是 C++11 引入的智能指针，用于解决 [[6.8 指针安全|原始指针的安全问题]]，实现**独占所有权**的内存管理。

---

## 为什么要使用 unique_ptr

原始指针（裸指针）从 C++ 诞生起就一直存在，使用便捷但有以下问题：

1. **声明中并没有指出，裸指针指向的是单个对象还是一个数组**
2. **声明没有指出是否在用完后需要销毁裸指针所指向的对象**（即指针是否拥有所有权）
3. **销毁时，无法确定使用 `delete` 还是其他机制**（如专门的释放函数）
4. **无法确定是使用 `delete` 还是 `delete[]`**
5. **析构路径单一，难以保证在所有路径上只调用一次释放函数**

智能指针可以解决以上全部问题，它已经成为 C++11 以后的标准。C++ 设置了 3 种类型的智能指针来分别应对不同场景。

---

## unique_ptr 的特点

- **本地局部变量**：通常用于局部作用域
- **无法复制**：只能移动 (move)，不能复制 (copy)
- **自动释放**：作用域结束时自动释放内存，无需手动 `delete`
- **零开销抽象**：默认情况下，`unique_ptr` 与原始指针大小相同

---

## unique_ptr 的大小

`unique_ptr` 的大小取决于删除器的类型：

| 删除器类型 | sizeof(unique_ptr) | 说明 |
|-----------|-------------------|------|
| 默认删除器 | `sizeof(T*)` | 与原始指针相同 |
| 无状态 lambda | `sizeof(T*)` | 空类优化 |
| 函数指针 | `2 * sizeof(T*)` | 需存储函数地址 |
| 有状态 lambda | `sizeof(T*) + sizeof(state)` | 需存储捕获状态 |

```cpp
auto stateless = [](int* p) { delete p; };
auto stateful = [captured = 42](int* p) { delete p; };

std::unique_ptr<int> p1;                                    // 8 bytes
std::unique_ptr<int, decltype(stateless)> p2(nullptr, stateless);  // 8 bytes
std::unique_ptr<int, void(*)(int*)> p3(nullptr, [](int* p){ delete p; }); // 16 bytes
std::unique_ptr<int, decltype(stateful)> p4(nullptr, stateful);    // 16 bytes
```

> **最佳实践**：优先使用无状态 lambda 作为自定义删除器，保持零开销。

## 1. 管理单个对象

```cpp
#include <iostream>

int main() {
    std::unique_ptr<int> ptr(new int(10));  // ptr 拥有 int(10) 的所有权

    std::cout << *ptr;  // 输出 10

    // ptr 不需要显式销毁，作用域结束时自动销毁
}
```

---

## 2. 管理数组对象

```cpp
#include <iostream>

int main() {
    // 使用 unique_ptr 管理一个 int 数组（大小为 5）
    std::unique_ptr<int[]> ptrArray(new int[5]);

    // 手动初始化数组元素
    for (int i = 0; i < 5; ++i) {
        ptrArray[i] = i * 2;  // 0, 2, 4, 6, 8
    }

    // 访问和输出数组元素
    for (int i = 0; i < 5; ++i) {
        std::cout << "ptrArray[" << i << "] = " << ptrArray[i] << std::endl;
    }

    // 作用域结束时自动调用 delete[]
}
```

---

## 3. 使用 make_unique（推荐）

`std::make_unique` 是 C++14 引入的工厂函数，用于更安全地创建 `unique_ptr`。

```cpp
#include <iostream>

int main() {
    // 推荐写法：使用 make_unique
    std::unique_ptr<int> ptr = std::make_unique<int>(5);

    std::cout << *ptr;  // 输出 5

    // 管理数组
    auto arr = std::make_unique<int[]>(10);  // 10 个 int 的数组
}
```

**make_unique 的优点**：
- 更简洁的语法
- 异常安全
- 避免裸 `new` 表达式

---

## 4. 自定义删除器

可以为 `unique_ptr` 提供自定义的删除器：

```cpp
#include <iostream>

int main() {
    auto deleter = [](int* ptr) {
        std::cout << "Custom deleter called" << std::endl;
        delete ptr;
    };

    std::unique_ptr<int, decltype(deleter)> ptr(new int(5), deleter);
    // std::make_unique 不支持自定义删除器

    // 打印地址和指针
    std::cout << "addr ptr: " << ptr.get() << std::endl;
    std::cout << "val *ptr: " << *ptr << std::endl;
}
```

## 获取原始指针

### get() 方法

使用 `get()` 获取原始指针，但**不会转移所有权**：

```cpp
// 1. unique_ptr 和原始指针的使用
std::unique_ptr<int> unique_p(new int(5));

int* raw_ptr = unique_p.get();  // 获取原始指针
std::cout << *raw_ptr;          // 输出 5

// unique_p 仍然拥有所有权，作用域结束时会自动释放
```

> **警告**：不要对 `get()` 返回的指针调用 `delete`，否则会导致双重释放！

---

## 所有权转移

### reset() 方法

`reset()` 用于重置 `unique_ptr`：

```cpp
std::unique_ptr<int> ptr1 = std::make_unique<int>(10);

// 释放原有对象，接管新对象
ptr1.reset(new int(20));

// 释放对象，变为空指针
ptr1.reset();  // 或 ptr1.reset(nullptr);
```

### release() 方法

`release()` 释放所有权并返回原始指针：

```cpp
std::unique_ptr<int> ptr = std::make_unique<int>(10);

int* raw = ptr.release();  // ptr 放弃所有权，变为 nullptr
// 现在需要手动管理 raw
delete raw;  // 必须手动释放
```

### std::move() 转移所有权

使用 `std::move()` 将所有权从一个 `unique_ptr` 转移到另一个：

```cpp
std::unique_ptr<int> ptr1 = std::make_unique<int>(10);
std::unique_ptr<int> ptr2 = std::move(ptr1);  // 所有权转移

// ptr1 现在为 nullptr
// ptr2 拥有对象的所有权
```

**关于 swap**：
- 使用 `std::swap(ptr1, ptr2)` 或 `ptr1.swap(ptr2)` 交换两个指针
- 交换后原指针的所有权也随之交换

## 工厂函数模式

`unique_ptr` 非常适合作为工厂函数的返回类型：

```cpp
class Widget {
public:
    Widget(int id) : id_(id) {}
    int getId() const { return id_; }
private:
    int id_;
};

// 工厂函数返回 unique_ptr
std::unique_ptr<Widget> createWidget(int id) {
    return std::make_unique<Widget>(id);
}

int main() {
    auto widget = createWidget(42);  // 调用方获得所有权
    std::cout << widget->getId();    // 输出 42
}
```

**优点**：
- 明确表达所有权转移语义
- 调用方自动获得对象所有权
- 异常安全：若抛出异常，资源自动释放

## unique_ptr 转换为 shared_ptr

`unique_ptr` 可以隐式转换为 `shared_ptr`，反之则不行：

```cpp
// unique_ptr → shared_ptr（允许）
std::unique_ptr<int> unique = std::make_unique<int>(42);
std::shared_ptr<int> shared = std::move(unique);  // unique 变为 nullptr

// shared_ptr → unique_ptr（不允许）
// std::unique_ptr<int> back = std::move(shared);  // 编译错误！
```

**应用场景**：工厂函数返回 `unique_ptr`，调用方可根据需要转换为 `shared_ptr`：

```cpp
std::unique_ptr<Widget> createWidget(int id);

// 场景1：独占使用
auto exclusive = createWidget(1);

// 场景2：需要共享时转换
std::shared_ptr<Widget> shared = createWidget(2);
```

## 复制语义 vs 移动语义

### 复制语义 (Copy Semantics)

- 当对象被复制（拷贝）时，会调用**拷贝构造函数**或**拷贝赋值运算符**
- 复制后，两个对象独立存在，各自拥有独立的数据副本
- `unique_ptr` **禁止复制**，因为会导致多个指针指向同一内存

### 移动语义 (Move Semantics)

- 当对象被移动时，会调用**移动构造函数**或**移动赋值运算符**
- 移动后，原对象的资源被转移到新对象，原对象进入有效但未定义的状态
- 移动语义避免了不必要的深拷贝，提高性能

```cpp
// unique_ptr 只支持移动，不支持复制
std::unique_ptr<int> p1 = std::make_unique<int>(10);
// std::unique_ptr<int> p2 = p1;        // 编译错误！不能复制
std::unique_ptr<int> p2 = std::move(p1); // 正确，移动语义
```

### 移动语义的原理

移动构造函数签名：`ClassName(ClassName&& other) noexcept`

```cpp
class MyClass {
    int* data;
public:
    // 移动构造函数
    MyClass(MyClass&& other) noexcept : data(other.data) {
        other.data = nullptr;  // 将源对象的指针置空
    }
};
```

- 右值引用 `&&` 表示可以绑定到临时对象
- `noexcept` 表示不会抛出异常（重要！STL 容器依赖此特性）
- 移动后源对象的指针被置空，避免双重释放

---

## 关键要点

1. **独占所有权**：`unique_ptr` 独占资源，不能复制只能移动
2. **自动释放**：作用域结束自动调用析构函数释放资源
3. **推荐 make_unique**：比直接 `new` 更安全
4. **get() vs release()**：`get()` 不转移所有权，`release()` 转移所有权
5. **移动语义**：使用 `std::move()` 转移所有权
