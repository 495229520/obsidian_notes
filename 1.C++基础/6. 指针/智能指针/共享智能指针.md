# 共享智能指针 (shared_ptr)

> `shared_ptr` 是 C++11 引入的智能指针，用于管理 [[6.2 动态内存分配与释放|动态分配]] 的对象的生命周期。与 [[独享智能指针|unique_ptr]] 独占所有权不同，`shared_ptr` 允许多个指针**共享**同一对象的**所有权**。

---

## 简介

`shared_ptr` 的底层实现依赖于**引用计数**来跟踪共享对象的使用情况。它使用一个**控制块**（Control Block）来存储引用计数和其他元数据。控制块包含两个引用计数：

1. **共享计数**（shared count）：表示有多少个 `shared_ptr` 实例共享同一个对象
2. **弱计数**（weak count）：表示有多少个 [[weak_ptr]] 实例引用该对象

### 主要特点

- **共享所有权**：多个 `shared_ptr` 可以指向同一个对象，共同拥有该对象的管理权
- **引用计数**：`shared_ptr` 内部维护一个引用计数器，记录有多少个 `shared_ptr` 指向同一个对象
- **自动释放**：当最后一个指向对象的 `shared_ptr` 被销毁（`reset()`）时，即引用计数归零时，会自动删除该对象，释放内存

### 使用场景

- **多个对象共享资源**：当需要在多个对象之间共享同一个动态分配的资源时，可以使用 `shared_ptr` 进行管理
- **循环依赖**：在某些情况下，对象之间可能存在循环依赖关系，导致内存泄漏。`shared_ptr` 可以帮助打破这种循环依赖，实现内存的自动释放（配合 [[weak_ptr]] 使用）

---

## 基本用法

```cpp
#include <iostream>
#include <memory>

int main() {
    // 使用 std::make_shared 创建一个 shared_ptr，指向一个整数
    std::shared_ptr<int> ptr = std::make_shared<int>(5);

    // 可以像使用普通指针一样使用 shared_ptr
    std::cout << *ptr << std::endl;  // 输出 5

    // 多个 shared_ptr 可以指向同一个整数
    std::shared_ptr<int> another_ptr = ptr;

    // 引用计数为 2
    std::cout << ptr.use_count() << std::endl;  // 输出 2

    // 当所有指向整数的 shared_ptr 都被销毁时，整数会被自动删除
    return 0;
}
```

## 为什么优先使用 make_shared

> 📚 参考《Effective Modern C++》Item 21

### 性能优势

| 创建方式 | 内存分配次数 | 说明 |
|---------|-------------|------|
| `std::shared_ptr<T>(new T)` | 2次 | 对象 + 控制块分开分配 |
| `std::make_shared<T>()` | 1次 | 对象和控制块一起分配 |

```cpp
// ❌ 效率较低：两次内存分配
std::shared_ptr<Widget> sp1(new Widget);

// ✅ 推荐：一次内存分配
auto sp2 = std::make_shared<Widget>();
```

### 异常安全

```cpp
// ❌ 潜在内存泄漏
processWidget(std::shared_ptr<Widget>(new Widget), computePriority());
// 如果 computePriority() 抛出异常，new Widget 可能泄漏

// ✅ 异常安全
processWidget(std::make_shared<Widget>(), computePriority());
```

### make_shared 的局限

- 无法指定自定义删除器（需要管理数组或自定义资源时）
- 控制块和对象内存绑定，当存在 `weak_ptr` 时，即使对象已销毁，内存也不会释放（因为控制块还在）

## 从原始指针构造

```cpp
#include <iostream>
#include <memory>

int main() {
    // 1. 创建一个原始指针
    int* raw_ptr = new int(10);

    // 2. 使用 shared_ptr 管理原始指针
    std::shared_ptr<int> shared_ptr1(raw_ptr);

    // 3. 多个 shared_ptr 可以指向同一个原始指针
    std::shared_ptr<int> shared_ptr2 = shared_ptr1;

    // 4. 使用 shared_ptr 访问和操作数据
    std::cout << *shared_ptr1 << std::endl;  // 输出 10
    *shared_ptr2 = 20;
    std::cout << *shared_ptr1 << std::endl;  // 输出 20

    // 5. 当所有 shared_ptr 都被销毁时，原始指针会被自动 delete
    return 0;
}
```

## 指向数组

`shared_ptr` 可以指向数组，但需要注意一些细节。

### 默认行为

默认情况下，`shared_ptr` 使用 `delete` 来释放它所指向的对象。由于数组需要使用 `delete[]` 来释放，因此直接将 `shared_ptr` 指向数组会导致程序崩溃或内存泄漏。

### 自定义删除器

为了使 `shared_ptr` 正确管理数组，需要提供一个自定义删除器，告诉 `shared_ptr` 使用 `delete[]` 而不是 `delete`：

```cpp
#include <iostream>
#include <memory>

int main() {
    // 1. 创建一个指向数组的原始指针
    int* array_ptr = new int[5]{1, 2, 3, 4, 5};

    // 2. 使用 shared_ptr 管理数组指针，并提供自定义删除器
    std::shared_ptr<int> shared_array_ptr(array_ptr, [](int* arr) {
        delete[] arr;
    });

    // 3. 使用 shared_ptr 访问和操作数组元素
    for (int i = 0; i < 5; ++i) {
        std::cout << shared_array_ptr.get()[i] << " ";  // 注意：这里使用 get() 获取原始指针
    }
    std::cout << std::endl;

    // 4. 当 shared_array_ptr 被销毁时，数组会被自动 delete[]
    return 0;
}
```

## 内部结构

`shared_ptr` 的内部结构通常如下：

- **指向对象的指针**：指向被管理的对象
- **控制块的指针**：指向包含引用计数和其他元数据的控制块

### 控制块的典型结构

```cpp
struct ControlBlock {
    std::atomic<long> shared_count;  // shared_ptr 的引用计数
    std::atomic<long> weak_count;    // weak_ptr 的引用计数
    // 其他元数据（如自定义删除器、分配器等）
};
```

### 引用计数的工作原理

1. **创建时**：当创建 `shared_ptr` 时，会初始化控制块并将 `shared_count` 设为 1
2. **复制/赋值时**：当 `shared_ptr` 被复制或赋值时，`shared_count` 增加
3. **销毁时**：当 `shared_ptr` 被销毁时，`shared_count` 减少
4. **对象释放**：如果 `shared_count` 减至 0，被管理的对象会被销毁
5. **控制块释放**：弱计数（`weak_count`）的目的是跟踪 [[weak_ptr]] 的数量。当 `weak_count` 和 `shared_count` 都为 0 时，控制块自身也会被销毁

---

## 线程安全

`std::shared_ptr` 的引用计数操作是**线程安全**的，这意味着多个线程可以安全地拷贝和销毁 `shared_ptr` 对象，而不需要额外的同步机制。这是通过使用 `std::atomic` 类型来实现的，保证了对共享计数和弱计数的原子操作。

### 线程安全的引用计数

- `shared_ptr` 的**引用计数操作是线程安全的**，即引用计数的增加和减少是原子的，不会发生竞争条件
- 当你在多线程环境下对同一个 `shared_ptr` 实例进行复制（增加引用计数）或销毁（减少引用计数）时，这些操作是原子性的
- 这意味着内核会保证这些操作是不可分割的，不会因为多个线程同时读写而导致竞态条件

### 非线程安全的对象访问

- `shared_ptr` **所管理的对象的访问并不是线程安全的**
- 如果多个线程需要同时访问共享对象，仍然需要额外的同步机制（如互斥锁）来保证线程安全
- 如果你在多个线程中同时对同一个对象进行读写操作，而该对象没有内置的同步机制，那么仍然会发生竞态条件

### 总结

| 操作 | 线程安全性 |
|------|------------|
| 引用计数的增减 | ✓ 线程安全（原子操作） |
| 所管理对象的读写 | ✗ 不是线程安全的，需要额外同步 |

## enable_shared_from_this

> 📚 参考《Effective Modern C++》Item 19

当对象需要在成员函数中获取指向自身的 `shared_ptr` 时，**不能直接用 `this` 构造**，否则会创建两个独立的控制块，导致双重删除。

### 错误做法

```cpp
class Widget {
public:
    std::shared_ptr<Widget> getPtr() {
        return std::shared_ptr<Widget>(this);  // ❌ 危险！双重删除
    }
};

auto sp1 = std::make_shared<Widget>();
auto sp2 = sp1->getPtr();  // sp1 和 sp2 有独立的控制块，都会 delete this
```

### 正确做法

```cpp
class Widget : public std::enable_shared_from_this<Widget> {
public:
    std::shared_ptr<Widget> getPtr() {
        return shared_from_this();  // ✅ 安全获取 shared_ptr
    }
};

auto sp1 = std::make_shared<Widget>();
auto sp2 = sp1->getPtr();  // sp1 和 sp2 共享同一个控制块
```

> [!warning] 注意
> 调用 `shared_from_this()` 前，对象**必须已被某个 `shared_ptr` 管理**，否则会抛出 `std::bad_weak_ptr` 异常。
>
> ```cpp
> Widget w;
> w.getPtr();  // ❌ 抛出异常，w 不是由 shared_ptr 管理的
> ```

## shared_ptr 与 weak_ptr 的交互

当将 `shared_ptr` 赋值给 [[weak_ptr]] 时，会进行以下操作：

1. **引用控制块**：`weak_ptr` 会指向与 `shared_ptr` 相同的控制块
2. **增加弱计数**：控制块中的 `weak_count` 会增加 1，以表明有一个新的 `weak_ptr` 引用了该对象

### 示例代码

```cpp
#include <iostream>
#include <memory>

int main() {
    std::shared_ptr<int> sp = std::make_shared<int>(42);
    std::weak_ptr<int> wp(sp);  // shared_ptr 赋值给 weak_ptr

    // 打印引用计数
    std::cout << "Shared count: " << sp.use_count() << std::endl;  // 输出 1
    std::cout << "Weak count: " << wp.use_count() << std::endl;    // 输出 1

    return 0;
}
```

### 赋值操作的内部过程

1. **获取控制块**：从 `shared_ptr` 中获取控制块指针
2. **增加弱计数**：对控制块的 `weak_count` 进行原子增加操作
3. **初始化 weak_ptr**：将 `weak_ptr` 的内部指针指向该控制块

> 在上面的例子中，`std::weak_ptr<int> wp(sp);` 语句执行时，`weak_ptr` 的构造函数会增加控制块的 `weak_count`，表示有一个新的 `weak_ptr` 引用了该对象。

## 性能开销

> 📚 参考《Effective Modern C++》Item 19

| 对比项 | `unique_ptr` | `shared_ptr` |
|-------|-------------|--------------|
| 大小 | 1个指针（与裸指针相同） | 2个指针（对象 + 控制块） |
| 拷贝开销 | 不可拷贝 | 原子操作增减引用计数 |
| 解引用开销 | 与裸指针相同 | 与裸指针相同 |
| 适用场景 | 独占所有权 | 共享所有权 |

> [!tip] 选择建议
> 当不需要共享所有权时，优先使用 [[独享智能指针|unique_ptr]]。它更轻量，且可以轻松转换为 `shared_ptr`（反之则不行）。

## 循环引用问题

两个 `shared_ptr` 互相引用会导致引用计数永远不为 0，造成 [[6.3 动态内存分配的风险#1. 内存泄漏 (Memory Leak)|内存泄漏]]。

### 问题示例

```cpp
#include <memory>
#include <iostream>

struct Node {
    std::shared_ptr<Node> next;
    ~Node() { std::cout << "Node destroyed\n"; }
};

int main() {
    auto a = std::make_shared<Node>();
    auto b = std::make_shared<Node>();

    a->next = b;  // a 引用 b
    b->next = a;  // b 引用 a（循环引用！）

    // main 结束时：
    // a 的引用计数 = 1 (b->next)
    // b 的引用计数 = 1 (a->next)
    // 两者都不为 0，内存泄漏！
    return 0;
}
// 输出：无（析构函数没有被调用）
```

### 解决方案

使用 [[weak_ptr]] 打破循环：

```cpp
struct Node {
    std::weak_ptr<Node> next;  // ✅ 弱引用不增加 shared_count
    ~Node() { std::cout << "Node destroyed\n"; }
};
// 输出：Node destroyed（两次）
```

## 关键要点

1. **共享所有权**：多个 `shared_ptr` 可以共同管理同一个对象
2. **引用计数**：自动追踪有多少个指针指向对象
3. **控制块**：存储共享计数和弱计数，使用 `std::atomic` 实现线程安全
4. **自动释放**：最后一个 `shared_ptr` 销毁时自动释放内存
5. **优先 make_shared**：一次内存分配，异常安全
6. **enable_shared_from_this**：在成员函数中安全获取 `shared_ptr<this>`
7. **数组需要自定义删除器**：使用 `delete[]` 替代默认的 `delete`
8. **线程安全有限**：引用计数操作是线程安全的，但对象本身不是
9. **循环引用问题**：用 [[weak_ptr]] 打破循环
