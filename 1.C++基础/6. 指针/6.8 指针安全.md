# 6.8 指针安全

> 本节讨论原始指针的安全问题，以及如何使用[[独享智能指针|智能指针]]来解决这些问题。与 [[6.3 动态内存分配的风险]] 密切相关。

---

## 问题一：作用域外访问已释放内存

**指针没有了，但是其指向的内存空间没释放**

```cpp
int* p;

{
    int* a = new int[50];
    p = a;
    a[2] = 255;
}

std::cout << p[2];  // 危险！a 已超出作用域，但内存未释放
```

**问题分析**：
- 在作用域内，`a` 指向动态分配的内存
- 作用域结束后，`a` 被销毁，但内存未释放
- `p` 仍然指向该内存，可以访问但这是不安全的
- 这会导致**内存泄漏**

---

## 问题二：访问已释放的内存

**指针指向的内存空间释放了，当时指针仍存在**

```cpp
int* p;

{
    int* a = new int[5];
    p = a;
    std::cout << p[0] << std::endl;
    delete[] a;
}

std::cout << p[0];  // 危险！内存已释放，p 成为悬空指针
```

**问题分析**：
- 内存被正确释放了
- 但 `p` 仍然保存着已释放内存的地址
- `p` 成为[[6.3 动态内存分配的风险#2. 悬空指针 (Dangling Pointer)|悬空指针]]
- 访问已释放内存导致**未定义行为**

---

## 解决办法：使用智能指针

```cpp
int* p;

{
    std::unique_ptr<int[]> a(std::make_unique<int[]>(50));
    a[2] = 250;

    p = a.get();
    std::cout << p[2];
}

std::cout << p[2];  // 这里 p 指向的内存已被 unique_ptr 自动释放
```

**智能指针的优势**：
- 自动管理内存生命周期
- 作用域结束时自动释放内存
- 避免内存泄漏和悬空指针

---

## 智能指针的缺点

虽然智能指针极大地简化了内存管理，但它们也带来了一些问题：

| 问题 | 说明 |
|------|------|
| **性能开销** | 引用计数等操作有运行时成本 |
| **循环引用** | `shared_ptr` 互相引用会导致内存泄漏 |
| **代码复杂性** | 需要理解不同智能指针的语义 |

### 循环引用问题

**循环引用 (Circular Reference)** 是指两个或多个智能指针对象互相持有对方的引用，导致它们都无法被正确释放的内存泄漏问题。

```cpp
struct A {
    std::shared_ptr<B> b_ptr;
};

struct B {
    std::shared_ptr<A> a_ptr;  // 循环引用！
};
```

> **解决方案**：使用 [[weak_ptr|weak_ptr]] 打破循环引用。

---

## 智能指针类型总结

| 类型 | 特点 | 使用场景 |
|------|------|----------|
| [[独享智能指针\|unique_ptr]] | 独占所有权，不可复制 | 单一所有者场景 |
| [[共享智能指针\|shared_ptr]] | 共享所有权，引用计数 | 多个所有者共享资源 |
| [[weak_ptr\|weak_ptr]] | 弱引用，不增加引用计数 | 打破循环引用 |

---

## 关键要点

1. **原始指针风险**：容易造成内存泄漏和悬空指针
2. **智能指针优势**：自动内存管理，RAII 原则
3. **选择正确的智能指针**：
   - 独占所有权 → `unique_ptr`
   - 共享所有权 → `shared_ptr`
   - 避免循环引用 → `weak_ptr`
