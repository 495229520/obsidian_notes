# 7.2 联合体

联合体（union）是一种特殊的数据类型，它允许在相同的内存位置存储不同的数据类型。联合体是 C++ 最古老的特性之一，旨在一种节省内存的方式。

---

## union 概述

### 核心概念

- 联合体（union）的**所有成员共享同一块内存空间**
- 联合体的大小等于其**最大成员的大小**（而 [[7.1 结构体|结构体]] 至少是所有成员大小之和）
- 在任何时候，联合体中只有**一个成员**是"活动的"

### 内存对比

```
struct MyStruct          union MyUnion
┌─────────────┐          ┌─────────────┐
│   short s   │          │             │
├─────────────┤          │  共享内存   │
│   int i     │          │  (4 bytes)  │
├─────────────┤          │             │
│  double d   │          └─────────────┘
└─────────────┘
  (16 bytes)               (8 bytes)
```

---

## 基本用法

```cpp
#include <iostream>

union Value {
    int intValue;
    double doubleValue;
    char charValue;
};

int main() {
    Value myValue;

    // 存储一个整数
    myValue.intValue = 42;
    std::cout << "Integer value: " << myValue.intValue << std::endl;

    // 存储一个浮点数（覆盖整数）
    myValue.doubleValue = 3.14159;
    std::cout << "Double value: " << myValue.doubleValue << std::endl;

    // 存储一个字符（覆盖浮点数）
    myValue.charValue = 'A';
    std::cout << "Character value: " << myValue.charValue << std::endl;

    return 0;
}
```

> **注意**：存储新值后，读取之前的成员是**未定义行为**（除了用于类型双关的情况）。

---

## 主要用途

### 1. 节省内存

当你确定在任何给定时间只需要存储一种类型的数据时，`union` 可以节省内存：

```cpp
union Data {
    int i;        // 4 bytes
    float f;      // 4 bytes
    char str[20]; // 20 bytes
};
// sizeof(Data) = 20，而不是 28
```

### 2. 类型转换（类型双关）

`union` 可以用于查看同一数据的不同表示形式：

```cpp
union FloatInt {
    float f;
    int i;
};

int main() {
    FloatInt fi;
    fi.f = 3.14f;

    // 查看 float 的二进制表示作为 int
    std::cout << "Float: " << fi.f << std::endl;
    std::cout << "As int (hex): " << std::hex << fi.i << std::endl;

    return 0;
}
```

> 这种技术常用于底层编程、网络协议解析等场景。

---

## 注意事项

### 1. 活动成员

- 联合体的**活动成员**是最后被写入的成员
- 在任何时刻，只有一个成员是有效的
- 读取非活动成员是**未定义行为**（除了某些特定情况，如类型双关）

### 2. 初始化规则

- 已初始化 `union` 的情况：`union` 内的所有成员共享同一内存，因此只需要初始化一个成员
- 只能初始化**第一个成员**，或者使用指定初始化器（C++20）

```cpp
union MyUnion {
    int i;
    float f;
};

MyUnion u1 = {10};       // 初始化 i
MyUnion u2 = {.f = 3.14f}; // C++20: 初始化 f
```

### 3. 访问规则

- 向联合体中写入一个值会使先前写入的值变得不可预测
- 对 C++ 而言，确保只读取最后写入的成员是程序员的责任

---

## 内存访问注意事项

```cpp
union Test {
    int i;
    char c;
};

Test t;
t.i = 0x12345678;

// c 的值取决于系统的字节序
// 小端系统：c = 0x78
// 大端系统：c = 0x12
```

内存布局（小端系统）：

```
地址:  低 ───────────────────> 高
内存:  [78] [56] [34] [12]
        ↑
        c 读取这个字节
```

## 匿名联合体

匿名联合体（anonymous union）是一种特殊的联合体，它**没有名字**，且其成员可以直接在包含它的作用域中访问：

```cpp
struct Variant {
    enum Type { INT, FLOAT, STRING } type;

    union {  // 匿名联合体
        int intValue;
        float floatValue;
        char stringValue[20];
    };
};

int main() {
    Variant v;
    v.type = Variant::INT;
    v.intValue = 42;  // 直接访问，无需 union 名称

    return 0;
}
```

**匿名联合体的规则**：
- 其成员被注入到外围作用域
- 只能包含非静态数据成员
- 不能有成员函数
- 不能有 `private` 或 `protected` 成员

---

## std::variant（C++17 推荐）

在 C++17 中，推荐使用 `std::variant` 替代 `union`：

```cpp
#include <variant>
#include <iostream>

int main() {
    // 定义一个可以存储 int、float 或 double 的 variant
    std::variant<int, float, double> myVariant;

    // 存储一个 int
    myVariant = 10;
    std::cout << "int: " << std::get<int>(myVariant) << std::endl;

    // 存储一个 float
    myVariant = 3.14f;
    std::cout << "float: " << std::get<float>(myVariant) << std::endl;

    // 存储一个 double
    myVariant = 2.71828;
    std::cout << "double: " << std::get<double>(myVariant) << std::endl;

    return 0;
}
```

### std::variant 的优势

- **类型安全**：访问错误类型会抛出 `std::bad_variant_access` 异常
- **可以使用 `std::visit`** 进行模式匹配
- **存储当前活动类型**：知道当前存储的是哪种类型
- 可以使用 `std::holds_alternative<T>(v)` 检查当前类型
- 可以使用 `std::get_if<T>(&v)` 安全地获取值

```cpp
// 检查当前类型
if (std::holds_alternative<int>(myVariant)) {
    std::cout << "存储的是 int" << std::endl;
}

// 安全访问
if (auto* val = std::get_if<float>(&myVariant)) {
    std::cout << "Float value: " << *val << std::endl;
}
```

---

## union vs struct vs variant

| 特性 | union | struct | std::variant |
|------|-------|--------|--------------|
| 内存共享 | ✓ | ✗ | ✓ |
| 类型安全 | ✗ | ✓ | ✓ |
| 知道活动成员 | ✗ | N/A | ✓ |
| C++ 版本要求 | C++98 | C++98 | C++17 |

---

## 关键要点

1. **内存共享**：联合体所有成员共享同一块内存
2. **大小**：联合体大小等于最大成员的大小
3. **活动成员**：同一时刻只有一个成员有效
4. **类型双关**：可用于查看数据的不同表示形式
5. **推荐使用 std::variant**：C++17 提供了类型安全的替代方案
