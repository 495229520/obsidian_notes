## 位运算

### 输出二进制内容

使用 `<bitset>` 头文件可以方便地输出变量的二进制表示。`std::bitset` 是 C++ 标准库提供的一个模板类，用于处理固定大小的位序列。

#### 什么是 bitset？

`std::bitset<N>` 是一个容器，它保存了 `N` 个位（0 或 1）。与 `bool` 数组或 `vector<bool>` 相比，它的优势在于：
1. **内存效率**：它针对空间进行了极致优化，通常每个位只占用 1 bit 的物理内存。
2. **位运算支持**：它直接支持 `&`, `|`, `^`, `~`, `<<`, `>>` 等位运算符，以及丰富的成员函数进行位操作。
3. **方便转换**：可以轻松地与整数、字符串进行相互转换。

#### 基本用法

```cpp
#include <bitset>
#include <iostream>

int main() {
    // 1. 初始化
    std::bitset<8> b1(42);          // 从整数初始化 (00101010)
    std::bitset<8> b2("1100");      // 从字符串初始化 (00001100)

    // 2. 输出二进制
    std::cout << b1 << std::endl;   // 输出: 00101010

    // 3. 常用成员函数
    b1.set(0);           // 将第0位设为1 -> 00101011 (注意：索引从右往左，从0开始)
    b1.reset(1);         // 将第1位设为0 -> 00101001
    b1.flip();           // 全位取反 -> 11010110
    b1.flip(2);          // 翻转第2位 -> 11010010

    std::cout << "Count (1的个数): " << b1.count() << std::endl;
    std::cout << "Size (总位数): " << b1.size() << std::endl;
    std::cout << "Test (第2位是否为1): " << b1.test(2) << std::endl;
    
    // 4. 状态判断
    if (b1.any())  std::cout << "至少有一个位是1" << std::endl;
    if (b1.none()) std::cout << "全位都是0" << std::endl;
    if (b1.all())  std::cout << "全位都是1" << std::endl;

    // 5. 转换回其他类型
    unsigned long long val = b1.to_ullong();
    std::string s = b1.to_string();

    return 0;
}
```

> **注意**：`bitset` 的大小 `N` 必须在编译期确定（模板参数）。索引方向与数学意义上的位序一致：`b[0]` 代表最低位（LSB）。

### 位运算左移 `<<`

```cpp
int a{ (int)0b11111111111111111111111111111111 };
a <<= 8;
std::cout << std::bitset<32>(a);
// 结果: 11111111111111111111111100000000
// 可以发现是用 0 补右边的
```

### 位运算右移 `>>`

```cpp
unsigned b{ (unsigned)0b11111111111111111111111111111111 };
b >>= 7;
std::cout << std::bitset<32>(b);
// 结果: 00000001111111111111111111111111
// 可以发现这里也是用 0 补左边的
```

### 为什么右移的时候使用 `unsigned` 而不是使用 `int`？

1. **计算机为了正负数之间的计算方便通常使用正数取反再加1的方式来表示负数**
   ```
   0000 0011 → +3
   1111 1101 → -3
   两者相加直接就变成0了，十分方便
   ```

2. **向左位移就是做 2 的乘法，向右位移就是做 2 的除法**

3. **从上面的 1 可知，正数的首位是 0，负数的首位是 1，而乘法和除法是不会改变正负的**
   - 因此对应 `int` 而言，它有正负
   - 如果 `int` 是正数，右移的时候就是补 0
   - 如果 `int` 是负数，右移的时候就是补 1

## 位运算详解

### 求反运算 `~`

```
  0000 0000 1010 1111
~ ───────────────────
  1111 1111 0101 0000
```

将所有位取反，0 变 1，1 变 0。

### 与运算 `&`

```
  0010 1000 0011 0011
& 1111 1111 0000 0000
─────────────────────
  0010 1000 0000 0000
```

**与运算规则**：
- 两个 1 进行与运算的时候结果是 1
- 0 与 1 / 0 进行与运算的时候结果是 0

### 或运算 `|`

```
  0010 1000 0011 0011
| 1111 1111 0000 0000
─────────────────────
  1111 1111 0011 0011
```

**或运算规则**：
- 1 与 1 / 0 进行或运算的时候结果是 1
- 只有 0 与 0 运算的时候结果才是 0

### 异或运算 `^`

```
  0010 1000 0011 0011
^ 1111 1111 0000 0000
─────────────────────
  1101 0111 0011 0011
```

**异或运算特性**：
- 异或通常用于**加密和解密**
- `c = a ^ b`
- `a = b ^ c = b ^ a ^ b`（由于 `b ^ b == 0`，`0 ^ a == a`）
- `b = a ^ c = a ^ a ^ b`

> **重要性质**：任何数与自身异或结果为 0，任何数与 0 异或结果为自身。

## 实战案例：防作弊项目设计

### 需求背景

我们的"膘江湖"是一个单机游戏：
- 变量 `diamond` 代表钻石
- 变量 `vip_exp` 代表累计钻石消费数

设计一个算法，使得能够发现用户使用单机修改器修改了钻石或者累计钻石消费数这样的作弊行为。

### 根据异或运算原理

让钻石成为 `a`，钻石消费成为 `b`，然后通过 `a ^ b` 生成一个 `c`：
- 如果 `a` 和 `b` 被改变而 `c` 没有被改变的时候
- `a ^ c` 得不出 `b`
- 通过这样的原理就可以知道玩家是否作弊了

### 代码实现

```cpp
#include <iostream>

int main() {
    unsigned int diamond{ 6000 };
    unsigned int vip_exp{ 80000 };

    // 生成校验码
    unsigned xbase = diamond ^ vip_exp;

    using std::cout;
    using std::cin;

    // 模拟玩家修改钻石数
    cout << "修改钻石数: ";
    cin >> diamond;

    unsigned int realDiamond;
    // 无论如何修改，最后的钻石数都会被修改回来
    realDiamond = xbase ^ vip_exp;
    diamond = realDiamond;

    cout << "真正的钻石数目: " << diamond;

    return 0;
}
```

### 原理说明

```
初始状态：
  diamond = 6000
  vip_exp = 80000
  xbase = diamond ^ vip_exp

如果玩家只修改了 diamond：
  realDiamond = xbase ^ vip_exp
              = (原diamond ^ vip_exp) ^ vip_exp
              = 原diamond

钻石会被还原为原始值！
```

> **注意**：这只是一个简单的防作弊示例。实际游戏中需要更复杂的校验机制，并且 `xbase` 本身也需要保护。
