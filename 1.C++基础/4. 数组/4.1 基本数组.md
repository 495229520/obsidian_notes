# 基本数组

## 为什么需要数组

实际生活中，我们经常会需要处理多个具有相同性质的数据，比如 100 个员工的工资、10 个学员的学号。这种情况下，依次申请多个变量变的不太现实或者太过繁杂，这种情况下我们使用**数组**来解决这样的问题。

## 数组的本质

> 数组的本质是按照所申请的数据类型，向操作系统申请了一段**连续的内存空间**。

例如 `int age[6]` 在内存中是这样的：

| age[0] | age[1] | age[2] | age[3] | age[4] | age[5] |
|--------|--------|--------|--------|--------|--------|

## 关键特性

| 特性 | 说明 |
|------|------|
| **同质性** | 所有元素必须是相同的数据类型 |
| **连续存储** | 元素在内存中紧密排列，知道起始地址即可计算任意元素地址 |
| **固定大小** | 大小在编译时确定，运行时不能改变 |
| **索引访问** | 通过从 `0` 开始的整数索引直接访问元素，时间复杂度 O(1) |

## C 风格数组

### 声明与初始化

```cpp
// 声明格式
数据类型 数组名[元素个数];
数据类型 数组名[元素个数]{初始化列表};

// 声明示例
int numbers[5];              // 未初始化，包含垃圾值
int scores[3]{90, 85, 95};   // 列表初始化（推荐）
int grades[5]{};             // 全部初始化为 0
int data[5]{1, 2};           // 部分初始化，剩余为 0
double prices[]{1.5, 2.0};   // 省略大小，编译器自动推断
```

### 访问与遍历

```cpp
int arr[]{1, 2, 3, 4, 5};

// 索引访问
arr[0] = 10;                        // 修改第一个元素
std::cout << arr[2] << std::endl;   // 输出第三个元素

// 传统 for 循环
for (int i = 0; i < 5; ++i) {
    std::cout << arr[i] << " ";
}

// 范围 for 循环（C++11，推荐）
for (int x : arr) {
    std::cout << x << " ";
}

// 需要修改元素时使用引用
for (int& x : arr) {
    x *= 2;
}
```

### 获取数组长度

```cpp
int arr[]{1, 2, 3, 4, 5};

// C++11/14: sizeof 方式
std::size_t len = sizeof(arr) / sizeof(arr[0]);

// C++17: std::size()（推荐）
#include <iterator>
std::size_t len = std::size(arr);
```

> **注意**：`sizeof` 方式仅对原始数组有效，数组作为函数参数传递后会衰减为指针，此时 `sizeof` 返回的是指针大小。

### 多维数组

```cpp
// 声明与初始化
int matrix[2][3]{
    {1, 2, 3},
    {4, 5, 6}
};

// 访问元素
int val = matrix[0][1];  // val = 2

// 遍历
for (int i = 0; i < 2; ++i) {
    for (int j = 0; j < 3; ++j) {
        std::cout << matrix[i][j] << " ";
    }
    std::cout << "\n";
}
```

**内存布局**：多维数组在内存中仍是连续存储，按**行优先**顺序排列。

`int m[2][3]` 实际内存布局：
```
m[0][0] → m[0][1] → m[0][2] → m[1][0] → m[1][1] → m[1][2]
```

### 数组衰减 (Array Decay)

当数组作为函数参数传递时，会**衰减为指向首元素的指针**，丢失大小信息：

```cpp
void process(int arr[]) {          // 实际接收的是 int*
    // sizeof(arr) 返回指针大小，不是数组大小！
}

void process(int arr[5]) {         // 5 被忽略，仍是 int*
    // 同上
}

// 正确做法：同时传递大小
void process(int arr[], std::size_t size);

// 或使用模板保留大小信息
template<std::size_t N>
void process(int (&arr)[N]);       // N 在编译时可用
```

## std::array（C++11，推荐）

`std::array` 是对 C 风格数组的封装，提供更安全、更方便的接口，**应优先使用**。

### 基本用法

```cpp
#include <array>

std::array<int, 5> arr{1, 2, 3, 4, 5};

arr.size();       // 返回元素数量：5
arr.at(1);        // 边界检查访问，越界抛出 std::out_of_range
arr[1];           // 无边界检查访问（更快）
arr.front();      // 第一个元素
arr.back();       // 最后一个元素
arr.fill(0);      // 用 0 填充所有元素
arr.data();       // 返回底层 C 数组指针
```

### 核心优势

| 特性 | 说明 |
|------|------|
| **类型安全** | 模板类，类型和大小是类型的一部分 |
| **不会衰减** | 传参时保留大小信息 |
| **边界检查** | `at()` 方法提供运行时边界检查 |
| **STL 兼容** | 支持迭代器，可与 `std::sort` 等算法配合 |
| **零开销** | 与 C 数组相同的内存布局和性能 |

### 与 STL 算法配合

```cpp
#include <array>
#include <algorithm>
#include <numeric>

std::array<int, 5> arr{5, 3, 1, 4, 2};

// 排序
std::sort(arr.begin(), arr.end());

// 求和
int sum = std::accumulate(arr.begin(), arr.end(), 0);

// 查找
auto it = std::find(arr.begin(), arr.end(), 3);

// 填充
std::fill(arr.begin(), arr.end(), 0);

// 复制
std::array<int, 5> arr2;
std::copy(arr.begin(), arr.end(), arr2.begin());
```

## 常用算法操作

### C 风格数组 + STL 算法

```cpp
#include <algorithm>
#include <numeric>

int arr[]{5, 3, 1, 4, 2};
constexpr std::size_t len = std::size(arr);  // C++17

// 排序
std::sort(arr, arr + len);

// 求和
int sum = std::accumulate(arr, arr + len, 0);

// 最大/最小值
int maxVal = *std::max_element(arr, arr + len);
int minVal = *std::min_element(arr, arr + len);

// 填充
std::fill(arr, arr + len, 0);
```

## 总结对比

| 类型 | 大小 | 内存位置 | 边界检查 | 衰减 | 推荐度 |
|------|------|---------|---------|------|-------|
| C 风格数组 `int[N]` | 编译时固定 | 栈 | 无 | 会衰减 | 遗留代码 |
| `std::array<T,N>` | 编译时固定 | 栈 | `at()` 有 | 不衰减 | **优先使用** |
| `std::vector<T>` | 运行时可变 | 堆 | `at()` 有 | 不衰减 | 动态场景 |

**最佳实践**：
- 固定大小 → 使用 `std::array`
- 动态大小 → 使用 `std::vector`
- C 风格数组 → 仅在与 C 接口交互或特殊性能场景使用

> 更多动态数组内容请参考 [[4.2 vector数组]]
