# 13.1 左值和右值

> 在 C++ 中，左值 (lvalue) 和右值 (rvalue) 是**表达式的分类**，用于区分表达式所代表的对象的性质和使用方式。理解这两个概念对于掌握 C++ 中的资源管理（尤其是[[13.2 右值引用|移动语义]]）至关重要。

---

## 值类别体系 (Value Categories)

C++11 引入了更完整的值类别体系，将表达式分为三种基本类别：

![[图片/SVG/13_1_1.svg|860]]

| 类别 | 全称 | 含义 |
|------|------|------|
| **lvalue** | left value | 左值，具有持久身份的对象 |
| **prvalue** | pure rvalue | 纯右值，临时值或字面量 |
| **xvalue** | expiring value | 将亡值，即将被移动的对象 |
| **glvalue** | generalized lvalue | 广义左值 = lvalue + xvalue |
| **rvalue** | right value | 右值 = prvalue + xvalue |

---

## 1. 左值 (lvalue)

**定义**：左值是指一个**持久的对象**，它拥有[[1.3 进程的虚拟地址空间|内存地址]]，你可以通过取地址运算符 `&` 获取它的地址。换句话说，左值是可以放在赋值运算符 `=` 左边的表达式。

### 特征

- **具有身份 (identity)**：你可以确定两个左值是否引用同一个对象
- **可以被修改**（除非它是 `const` 的）
- **通常可以多次使用**

### 常见的左值表达式

| 类型 | 示例 | 说明 |
|------|------|------|
| 变量名 | `int x = 10;` | `x` 是一个左值 |
| 数组元素 | `arr[0] = 1;` | `arr[0]` 是一个左值 |
| 解引用[[6.1基本指针\|指针]] | `*ptr = 20;` | `*ptr` 是一个左值 |
| 前置递增/递减 | `++x`, `--x` | 返回对象本身的引用 |
| 赋值表达式 | `x = 5` | 整个表达式也是左值 |
| 字符串字面量 | `"hello"` | 存储在静态区，是 `const char[]` 类型的左值 |

### 返回左值引用的函数

```cpp
int& get_ref(int& value) {
    return value;
}

int main() {
    int y = 5;
    get_ref(y) = 10;  // get_ref(y) 返回左值引用，可以赋值
    // 此时 y == 10
}
```

### 参数和返回值写法对照

围绕 `int& get_ref(int& value)`，最容易混淆的是：**参数写不写 `&`**，以及**返回值写不写 `&`**，到底分别意味着什么。

| 写法 | 参数含义 | 返回含义 | 是否影响原变量 | 是否安全 |
|------|------|------|------|------|
| `int& f(int& x)` | 传入原对象本身 | 返回原对象本身 | 会 | 安全 |
| `int f(int& x)` | 传入原对象本身 | 返回值副本 | 一般不会通过返回值影响原变量 | 安全 |
| `int& f(int x)` | 传入的是副本 | 返回局部副本的引用 | 不会正确影响原变量 | 不安全 |
| `int f(int x)` | 传入的是副本 | 返回值副本 | 不会 | 安全 |

#### 1. `int& f(int& x)`

```cpp
int& f(int& x) {
    return x;
}

int main() {
    int a = 10;
    f(a) = 30;  // 直接改到 a 本身
    // a == 30
}
```

- 参数 `x` 是外部变量的别名
- 返回值也是这个外部变量的别名
- 所以 `f(a)` 是一个左值，可以继续放在赋值号左边

#### 2. `int f(int& x)`

```cpp
int f(int& x) {
    return x;
}

int main() {
    int a = 10;
    int b = f(a);  // 返回的是副本
    // b == 10，a 仍然是 10
}
```

- 参数仍然绑定原对象，所以函数体内部能看到 `a`
- 但返回时会复制出一个普通 `int`
- 因此 `f(a)` 不再是原对象，而是一个临时值（prvalue）

#### 3. `int& f(int x)`

```cpp
int& f(int x) {
    return x;  // 错误：返回局部变量引用
}
```

- `x` 是传值进来的副本
- 它只是函数内部的局部变量
- 函数返回后，`x` 就销毁了
- 这时返回它的引用，会形成**悬空引用（dangling reference）**

这种写法虽然有时能编译，但后续使用属于**未定义行为**。

#### 4. `int f(int x)`

```cpp
int f(int x) {
    return x;
}

int main() {
    int a = 10;
    int b = f(a);  // 整个过程都只是值拷贝
}
```

- 参数是副本
- 返回值也是副本
- 函数和原变量之间没有引用关系

### 一句话记忆

- **参数写 `&`**：函数里拿到的是原对象
- **参数不写 `&`**：函数里拿到的是副本
- **返回写 `&`**：函数返回的是原对象本身
- **返回不写 `&`**：函数返回的是副本

其中最需要避免的是：

> **不要返回局部变量的引用。**

## 2. 右值 (rvalue)

**定义**：右值是指一个**临时的、不具名的对象**，它没有持久的内存地址。通常，右值出现在赋值运算符 `=` 的右边，并且在表达式结束后就会被销毁。

### 特征

- **没有身份**（无法确定两个右值是否相同，因为它们是临时的）
- **通常不能被修改**（虽然可以通过[[13.2 右值引用|右值引用]]修改某些类型的右值）
- **通常只使用一次**（因为它们是临时的）

### 常见的右值表达式

| 类型 | 示例 | 说明 |
|------|------|------|
| 字面量（非字符串） | `10`, `3.14`, `true` | 纯右值 (prvalue) |
| 算术表达式结果 | `x + y`, `a * b` | 临时计算结果 |
| 后置递增/递减 | `x++`, `y--` | 返回修改前的副本 |
| 类型转换 | `static_cast<int>(3.14)` | 产生临时值 |
| 返回非引用的函数 | `get_value()` | 返回临时对象 |
| `std::move()` 结果 | `std::move(s)` | 将亡值 (xvalue) |

### 返回右值的函数

```cpp
int get_value() {
    return 5;
}

int main() {
    int z = get_value();  // get_value() 是一个右值
    // get_value() = 10;  // 错误！不能给右值赋值
}
```

## 3. 将亡值 (xvalue)

将亡值是 C++11 新增的概念，表示**即将被移动的资源**。通过 `std::move()` 可以将左值转换为将亡值：

```cpp
#include <string>
#include <utility>

int main() {
    std::string s = "hello";
    std::string s2 = std::move(s);  // std::move(s) 是一个将亡值
    // s 现在处于"有效但未定义"状态（通常为空）
    // s2 拥有了原来 s 的资源
}
```

## 4. 左值与右值的判断技巧

### 方法一：能否取地址

```cpp
int x = 10;
int* p1 = &x;      // OK，x 是左值
// int* p2 = &10;  // 错误，10 是右值，不能取地址
// int* p3 = &(x + 1);  // 错误，x + 1 是右值
```

### 方法二：能否放在赋值左边

```cpp
int x = 10;
x = 20;        // OK，x 是左值
// 10 = 20;    // 错误，10 是右值
// (x + 1) = 20;  // 错误，x + 1 是右值
```

### 方法三：使用 decltype 推导

```cpp
#include <type_traits>

int x = 10;
// decltype(x) 是 int（变量名是左值）
// decltype((x)) 是 int&（表达式是左值引用）
// decltype(x + 1) 是 int（右值）
```

## 5. 前置与后置递增的区别

这是一个经典的左值/右值区别示例：

```cpp
int x = 5;

// 前置递增：返回递增后的对象本身（左值）
++x = 10;  // OK，++x 是左值
int* p1 = &(++x);  // OK，可以取地址

// 后置递增：返回递增前的副本（右值）
// x++ = 10;  // 错误！x++ 是右值
// int* p2 = &(x++);  // 错误！不能取地址
```

**原因**：
- `++x`：先递增 `x`，然后返回 `x` 的引用
- `x++`：先保存 `x` 的副本，递增 `x`，然后返回那个副本（临时对象）

---

## 关键要点

1. **左值**：有持久内存地址，可以取地址，可以放在赋值左边
2. **右值**：临时对象，无持久地址，表达式结束后销毁
3. **将亡值 (xvalue)**：C++11 新增，通过 `std::move()` 产生，用于移动语义
4. **字符串字面量特殊**：虽然是字面量，但属于左值（存储在静态区）
5. **前置 vs 后置**：前置递增返回左值，后置递增返回右值
