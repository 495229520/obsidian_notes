# 13.3 左值引用

> 左值引用是 C++98 就存在的特性，使用 `&` 声明，用于创建[[13.1 左值和右值#1. 左值 (lvalue)|左值]]的别名。本节在 [[6.4 引用]] 的基础上，从值类别的角度进一步探讨左值引用的特性。

---

## 左值引用的定义

左值引用使用 `&` 符号声明，必须绑定到一个**左值**：

```cpp
int x = 10;
int& ref = x;      // OK，ref 是 x 的别名

// int& ref2 = 10; // 错误！不能绑定到右值（字面量）
// int& ref3 = x + 1; // 错误！不能绑定到右值（临时值）
```

### const 左值引用的特殊性

`const` 左值引用可以绑定到**右值**，这是一个重要的例外：

```cpp
const int& cref = 10;       // OK！const 引用可以绑定右值
const int& cref2 = x + 1;   // OK！

// 为什么这样设计？
// 因为 const 引用不会修改值，绑定右值是安全的
// 编译器会创建一个临时对象来保存右值，引用绑定到这个临时对象
```

**底层机制**：当 `const int& cref = 10;` 时，编译器会：
1. 创建一个临时的 `int` 变量存储 `10`
2. 将 `cref` 绑定到这个临时变量
3. 临时变量的生命周期被延长到与引用相同

---

## 左值引用的绑定规则

### 规则表

| 引用类型 | 绑定左值 | 绑定右值 |
|----------|---------|---------|
| `T&` | ✓ | ✗ |
| `const T&` | ✓ | ✓ |
| `T&&` | ✗ | ✓ |
| `const T&&` | ✗ | ✓（少用）|

### 示例代码

```cpp
void demo() {
    int x = 10;

    // 非 const 左值引用
    int& r1 = x;          // OK
    // int& r2 = 10;      // 错误
    // int& r3 = x + 1;   // 错误

    // const 左值引用
    const int& cr1 = x;      // OK，绑定左值
    const int& cr2 = 10;     // OK，绑定右值
    const int& cr3 = x + 1;  // OK，绑定右值

    // 右值引用
    int&& rr1 = 10;          // OK
    int&& rr2 = x + 1;       // OK
    // int&& rr3 = x;        // 错误
    int&& rr4 = std::move(x); // OK
}
```

## 左值引用作为函数参数

左值引用最常见的用途是作为函数参数，避免[[9.1.2 函数参数|值传递]]的拷贝开销。

### 非 const 引用参数

非 const 引用参数只接受左值：

```cpp
void modify(int& x) {
    x = x * 2;
}

int main() {
    int a = 5;
    modify(a);     // OK，a 是左值
    // modify(10); // 错误！10 是右值

    std::cout << a << std::endl;  // 输出 10
}
```

### const 引用参数

const 引用参数可以接受左值和右值，是最通用的只读参数形式：

```cpp
void print(const std::string& s) {
    std::cout << s << std::endl;
}

int main() {
    std::string str = "Hello";
    print(str);      // OK，传左值
    print("World");  // OK，传右值（字符串字面量转为临时 string）
}
```

### 为什么 const T& 如此重要

`const T&` 是 C++ 中传递只读参数的标准方式，因为它：
1. **避免拷贝**：不会复制整个对象
2. **接受范围广**：既可以传左值也可以传右值
3. **语义明确**：表示函数不会修改参数

```cpp
// 推荐：对于大对象使用 const 引用
void process(const std::vector<int>& vec);

// 不推荐：值传递会拷贝整个 vector
void process(std::vector<int> vec);
```

## 左值引用作为返回值

返回左值引用允许函数调用出现在赋值语句的左边：

```cpp
class Array {
private:
    int data[100];
public:
    // 返回引用，允许修改元素
    int& operator[](int index) {
        return data[index];
    }

    // const 版本，用于 const 对象
    const int& operator[](int index) const {
        return data[index];
    }
};

int main() {
    Array arr;
    arr[0] = 10;   // operator[] 返回引用，可以赋值
    int x = arr[0]; // 也可以读取
}
```

### 危险：返回局部变量的引用

**永远不要**返回局部变量的引用，这会导致悬空引用：

```cpp
int& dangerous() {
    int local = 10;
    return local;  // 危险！local 在函数返回后销毁
}

int main() {
    int& ref = dangerous();  // ref 是悬空引用
    std::cout << ref;        // 未定义行为！
}
```

## 引用折叠规则

当引用的引用出现时（在模板中），遵循**引用折叠**规则：

| 原类型 | 新修饰 | 结果 |
|--------|--------|------|
| `T&` | `&` | `T&` |
| `T&` | `&&` | `T&` |
| `T&&` | `&` | `T&` |
| `T&&` | `&&` | `T&&` |

**规则总结**：只要有 `&`，结果就是 `&`；只有 `&& + &&` 才是 `&&`。

```cpp
template<typename T>
void func(T&& param) {
    // 如果传入左值 int&，T 推导为 int&
    // param 类型：int& && → 折叠为 int&

    // 如果传入右值 int，T 推导为 int
    // param 类型：int&&
}
```

## 左值引用的底层实现

左值引用在底层通常实现为**隐藏的常量指针**（指针本身不可变，但指向的值可变）：

```cpp
int a = 5;
int& ref = a;
```

对应的汇编（简化）：

```asm
mov    dword ptr [ebp-0Ch], 5      ; a = 5
lea    eax, [ebp-0Ch]              ; 取 a 的地址
mov    dword ptr [ebp-18h], eax    ; ref 存储 a 的地址
```

可以看到，引用 `ref` 实际上存储了 `a` 的地址，与[[6.4 引用#本质|指针的实现方式]]相同。

---

## 关键要点

1. **左值引用 (&)**：绑定到左值，创建别名
2. **const 左值引用**：可以绑定左值和右值，最通用的只读参数形式
3. **函数参数**：使用 `const T&` 避免拷贝，使用 `T&` 允许修改
4. **函数返回值**：返回引用允许链式调用，但不要返回局部变量的引用
5. **引用折叠**：`& + &` = `&`，`&& + &&` = `&&`，其他都是 `&`
