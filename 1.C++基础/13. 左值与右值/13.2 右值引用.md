# 13.2 右值引用

> 右值引用是 C++11 引入的新特性，使用 `&&` 声明，主要用于实现**移动语义**和**完美转发**，从而提高程序性能。

---

## 右值引用的定义

右值引用是一种可以绑定到[[13.1 左值和右值#2. 右值 (rvalue)|右值]]的引用类型，使用 `&&` 符号声明：

```cpp
int&& rref = 10;        // OK，绑定到字面量（右值）
int&& rref2 = 5 + 3;    // OK，绑定到表达式结果（右值）

int x = 10;
// int&& rref3 = x;     // 错误！不能直接绑定到左值
int&& rref4 = std::move(x);  // OK，std::move 将左值转为右值
```

### 左值引用 vs 右值引用

| 特性 | 左值引用 (`&`) | 右值引用 (`&&`) |
|------|---------------|----------------|
| **绑定对象** | 左值 | 右值（临时对象） |
| **const 版本** | `const T&` 可绑定右值 | `T&&` 绑定右值 |
| **主要用途** | 避免拷贝、传参 | 移动语义、完美转发 |
| **引入版本** | C++98 | C++11 |

```cpp
int x = 10;
int& lref = x;           // 左值引用绑定左值
// int& lref2 = 10;      // 错误！
const int& clref = 10;   // const 左值引用可绑定右值

int&& rref = 10;         // 右值引用绑定右值
// int&& rref2 = x;      // 错误！右值引用不能绑定左值
```

## 移动语义 (Move Semantics)

移动语义是右值引用最重要的应用，它允许将资源（如[[6.2 动态内存分配与释放|动态分配的内存]]）从一个对象"**移动**"到另一个对象，而不是进行代价昂贵的**深拷贝**。

### 为什么需要移动语义

考虑以下场景：

```cpp
std::vector<int> createVector() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    return v;  // 返回时会发生什么？
}

int main() {
    std::vector<int> result = createVector();
}
```

在 C++11 之前，返回值会触发**拷贝构造函数**，复制整个 vector 的内容。如果 vector 很大，这会非常低效。

有了移动语义，编译器可以识别 `v` 是一个即将销毁的临时对象，直接"窃取"它的资源给 `result`，避免了不必要的拷贝。

## std::move

`std::move` 并不真正"移动"任何东西，它只是将左值**转换**为右值引用，从而允许调用移动构造函数或移动赋值运算符。

```cpp
#include <utility>
#include <string>
#include <iostream>

int main() {
    std::string s1 = "Hello, World!";
    std::cout << "移动前 s1: " << s1 << std::endl;

    std::string s2 = std::move(s1);  // s1 的资源移动到 s2

    std::cout << "移动后 s1: " << s1 << std::endl;  // s1 可能为空
    std::cout << "移动后 s2: " << s2 << std::endl;  // s2 拥有原来的内容
}
```

**输出**：
```
移动前 s1: Hello, World!
移动后 s1:
移动后 s2: Hello, World!
```

> **注意**：移动后的对象处于**有效但未定义**的状态。可以对它重新赋值或销毁，但不应该假设它的值。

## 移动构造函数与移动赋值运算符

要让自定义类支持移动语义，需要实现**移动构造函数**和**移动赋值运算符**。

### 移动构造函数

```cpp
class MyString {
private:
    char* data;
    size_t length;

public:
    // 移动构造函数
    MyString(MyString&& other) noexcept
        : data(other.data), length(other.length) {
        // 将源对象的指针置空，防止析构时重复释放
        other.data = nullptr;
        other.length = 0;
    }

    // 析构函数
    ~MyString() {
        delete[] data;
    }
};
```

#### 这里的 `other` 会变成悬空指针吗？

不会。这里发生的是**所有权转移**，不是“资源失效后还保留旧地址”。

- `data(other.data), length(other.length)` 这一步没有重新分配内存，而是让**新对象直接接管** `other` 原来持有的堆内存
- 接管之后，这块资源的所有权就属于新对象了，不再属于 `other`
- 随后执行 `other.data = nullptr; other.length = 0;`，把源对象主动重置为“空对象”状态

因此：

- **新对象**：`data` 指向原来的那块堆内存，真正拥有资源
- **other**：`data == nullptr`，`length == 0`，仍然是一个**有效对象**，只是变成了“已移动对象”

这里要区分两个概念：

- **空指针 (`nullptr`)**：明确表示“不指向任何对象”
- **悬空指针 (dangling pointer)**：指针里还保存着旧地址，但那块内存已经被释放或失效

所以 `other.data = nullptr;` 之后，`other` **不是悬空指针**，反而正是为了**避免悬空指针和重复释放**。

如果不把 `other` 置空，会发生什么？

- 新对象和 `other` 会同时指向同一块堆内存
- 当其中一个对象先析构时，这块内存会被释放
- 另一个对象中的 `data` 就会变成**悬空指针**
- 之后再析构一次，就可能发生 **double free（重复释放）**

可以把移动构造理解成：

- **拷贝构造**：重新复制一份资源，两边各有一份
- **移动构造**：直接把资源“交接”给新对象，源对象自己变空

对于这个 `MyString` 实现，我们把“移动后的对象”明确定义为：

```cpp
data == nullptr;
length == 0;
```

这意味着它虽然不再拥有原来的字符数组，但仍然可以被安全析构，也可以之后再被赋新值。

### 移动赋值运算符

```cpp
class MyString {
public:
    // 移动赋值运算符
    MyString& operator=(MyString&& other) noexcept {
        if (this != &other) {
            // 1. 释放当前资源
            delete[] data;

            // 2. 窃取源对象的资源
            data = other.data;
            length = other.length;

            // 3. 将源对象置空
            other.data = nullptr;
            other.length = 0;
        }
        return *this;
    }
};
```

### noexcept 的重要性

移动操作通常应该标记为 `noexcept`：

```cpp
MyString(MyString&& other) noexcept;
MyString& operator=(MyString&& other) noexcept;
```

**原因**：STL 容器（如 `std::vector`）在扩容时会检查移动操作是否为 `noexcept`。如果不是，容器会退回使用拷贝操作以保证异常安全性。

## 右值引用的汇编分析

```cpp
int&& e = 320 + 230;
```

对应的汇编代码：

```asm
mov    dword ptr [ebp-30h], 226h    ; 计算结果 550 存入临时位置
lea    eax, [ebp-30h]               ; 取临时位置的地址
mov    dword ptr [ebp-24h], eax     ; 将地址存入 e（右值引用）
```

**分析**：
- 右值 `320 + 230` 的结果 `550`（0x226）被存储在栈上的临时位置 `[ebp-30h]`
- 右值引用 `e` 实际存储的是这个临时位置的**地址** `[ebp-24h]`
- 这说明右值引用在底层也是通过指针实现的，与[[6.4 引用#本质|左值引用]]类似

## 完美转发 (Perfect Forwarding)

完美转发允许函数模板将参数的**值类别**（左值/右值）完整地传递给另一个函数。

### 它到底有什么用？

可以先把它理解成：**模板函数既能接收左值，也能接收右值**。但完美转发真正重要的地方，不只是“都能接住”，而是：

- 接住参数之后
- 再把参数传给下一层函数时
- **仍然保留它原本的左值 / 右值身份**

也就是说，完美转发解决的是“**中间包装层不要把参数的值类别弄丢**”这个问题。

例如我们写一个包装函数：

```cpp
void process(int& x)  { std::cout << "左值版本\n"; }
void process(int&& x) { std::cout << "右值版本\n"; }

template<typename T>
void wrapper(T&& arg) {
    process(arg);  // 注意：这里会有问题
}
```

看起来 `wrapper(20)` 传进来的是右值，但在 `wrapper` 内部，`arg` 是一个**有名字的变量**，因此表达式 `arg` 本身会被当成**左值**。

这就会导致：

- `wrapper(a)` 调用左值版本，正常
- `wrapper(20)` 本来希望调用右值版本，但如果直接写 `process(arg)`，它也会调用左值版本

所以完美转发的核心不是“能传左值和右值”这么简单，而是：

- **传进来是什么值类别**
- **转发出去时还是什么值类别**

这就需要 `std::forward<T>(arg)`。

### 问题：引用折叠

```cpp
template<typename T>
void wrapper(T&& arg) {
    // 这里的 T&& 是"转发引用"（也叫万能引用）
    // 当传入左值时，T 被推导为 T&，T&& 变成 T& && = T&
    // 当传入右值时，T 被推导为 T，T&& 保持 T&&
}
```

### 使用 std::forward

```cpp
#include <utility>

template<typename T>
void wrapper(T&& arg) {
    // std::forward 保持参数的原始值类别
    process(std::forward<T>(arg));
}

void process(int& x)  { std::cout << "左值版本\n"; }
void process(int&& x) { std::cout << "右值版本\n"; }

int main() {
    int a = 10;
    wrapper(a);           // 调用 process(int&)
    wrapper(20);          // 调用 process(int&&)
    wrapper(std::move(a)); // 调用 process(int&&)
}
```

#### 为什么 `std::forward` 必不可少？

因为单独有 `T&&` 还不够：

- `T&&` 让模板参数**既能绑定左值，也能绑定右值**
- `std::forward<T>(arg)` 让参数在**转交给别人时保持原来的值类别**

两者配合起来，才叫“完美转发”。

可以记成一句话：

- `T&&` 负责**接**
- `std::forward<T>(arg)` 负责**按原样转出去**

#### 它和普通右值引用不一样

这里还有一个很容易混淆的点：

- 模板里的 `T&&`（发生类型推导时）叫**转发引用**
- 普通函数参数里的 `MyString&& x` 才是普通的**右值引用**

例如：

```cpp
void f(MyString&& x);  // 只能接右值

template<typename T>
void g(T&& x);         // 既可能接左值，也可能接右值
```

所以不是所有 `&&` 都能“既接左值又接右值”，只有**模板推导场景下的 `T&&`** 才有这个特性。

### 常见用途

完美转发最常见的用途是写“中间层”或“通用接口”：

- **包装函数**：函数先接住参数，再转调真正的实现函数
- **工厂函数**：比如把参数原样转发给构造函数
- **原地构造接口**：如 `emplace_back(...)`、`make_unique(...)`
- **避免不必要的拷贝**：右值继续走移动语义，左值保持普通引用语义

如果没有完美转发，中间层很容易把原本可以移动的右值，错误地变成一次拷贝。

## 实际应用：智能指针中的移动

[[独享智能指针|unique_ptr]] 禁止拷贝，只允许移动，这正是利用了右值引用：

```cpp
#include <memory>

int main() {
    std::unique_ptr<int> p1 = std::make_unique<int>(10);

    // std::unique_ptr<int> p2 = p1;         // 错误！不能拷贝
    std::unique_ptr<int> p2 = std::move(p1); // OK，移动所有权

    // p1 现在为 nullptr
    // p2 拥有资源
}
```

---

## 关键要点

1. **右值引用 (&&)**：绑定到右值的引用，用于捕获临时对象
2. **std::move**：将左值转换为右值引用，不实际移动数据
3. **移动语义**：通过"窃取"资源避免深拷贝，提升性能
4. **noexcept**：移动操作应标记为 noexcept 以获得最佳 STL 兼容性
5. **完美转发**：使用 `std::forward` 保持参数的值类别
