# 12.3 迭代器的失效问题

## 什么是迭代器失效

**迭代器失效（Iterator Invalidation）** 是指迭代器所指向的元素被删除、移动或容器内部发生重新分配后，迭代器不再有效的现象。使用失效的迭代器会导致**未定义行为**（Undefined Behavior），可能引发程序崩溃或数据错误。

### 失效的本质

迭代器本质上是对容器元素的引用（通常是指针或封装的指针）。当容器的内存布局发生变化时，原有的迭代器就会失效：

```
失效前：                    失效后（扩容）：
┌─────────────┐            ┌─────────────────────────┐
│ 1 │ 2 │ 3 │             │ 1 │ 2 │ 3 │ 4 │        │ (新内存)
└─────────────┘            └─────────────────────────┘
     ↑
    it (有效)              ┌─────────────┐
                          │ ??? │ ??? │ ??? │ (已释放)
                          └─────────────┘
                               ↑
                              it (失效！指向已释放的内存)
```

## 不同容器的迭代器失效规则

### 序列容器

| 容器 | 操作 | 失效范围 |
|------|------|----------|
| **vector** | `insert` | 插入点及之后失效；若扩容则全部失效 |
| **vector** | `erase` | 删除点及之后失效 |
| **vector** | `push_back` | 若扩容则全部失效 |
| **deque** | `insert`/`erase` 在首尾 | 迭代器失效，引用不失效 |
| **deque** | `insert`/`erase` 在中间 | 全部失效 |
| **list** | `insert` | 不失效 |
| **list** | `erase` | 仅被删除元素的迭代器失效 |

### 关联容器

| 容器 | 操作 | 失效范围 |
|------|------|----------|
| **set/map** | `insert` | 不失效 |
| **set/map** | `erase` | 仅被删除元素的迭代器失效 |
| **unordered_set/map** | `insert` | 若 rehash 则全部失效 |
| **unordered_set/map** | `erase` | 仅被删除元素的迭代器失效 |

## vector 迭代器失效详解

### 插入操作导致的失效

```cpp
std::vector<int> vec = {1, 2, 3, 4, 5};
auto it = vec.begin() + 2;  // 指向 3

// 情况1：不触发扩容
vec.insert(vec.begin(), 0);
// it 失效！因为插入点在 it 之前，所有元素后移

// 情况2：触发扩容
vec.push_back(6);  // 可能触发扩容
// 若扩容，所有迭代器都失效
```

### 删除操作导致的失效

```cpp
std::vector<int> vec = {1, 2, 3, 4, 5};
auto it = vec.begin() + 2;  // 指向 3

vec.erase(vec.begin());  // 删除 1
// it 失效！删除点之后的所有迭代器都失效
```

## 正确处理迭代器失效

### 错误示例：删除偶数元素

```cpp
// 错误代码！可能崩溃或行为异常
std::vector<int> vec = {1, 2, 3, 4, 5};
for (auto it = vec.begin(); it != vec.end(); ++it) {
    if (*it % 2 == 0) {
        vec.erase(it);  // it 失效！
        // ++it 操作未定义行为
    }
}
```

### 正确示例 1：使用 erase 的返回值

`erase` 返回指向被删除元素下一个元素的有效迭代器：

```cpp
std::vector<int> vec = {1, 2, 3, 4, 5};
for (auto it = vec.begin(); it != vec.end(); ) {
    if (*it % 2 == 0) {
        it = vec.erase(it);  // erase 返回下一个有效迭代器
    } else {
        ++it;
    }
}
// 结果：vec = {1, 3, 5}
```

### 正确示例 2：使用 std::remove_if + erase

```cpp
#include <algorithm>

std::vector<int> vec = {1, 2, 3, 4, 5};
vec.erase(
    std::remove_if(vec.begin(), vec.end(),
                   [](int x) { return x % 2 == 0; }),
    vec.end()
);
// 结果：vec = {1, 3, 5}
```

### 正确示例 3：倒序遍历删除

```cpp
std::vector<int> vec = {1, 2, 3, 4, 5};
for (int i = vec.size() - 1; i >= 0; --i) {
    if (vec[i] % 2 == 0) {
        vec.erase(vec.begin() + i);
    }
}
// 结果：vec = {1, 3, 5}
```

## list 容器的迭代器失效

`list` 是双向链表，删除操作只会使被删除节点的迭代器失效：

```cpp
std::list<int> lst = {1, 2, 3, 4, 5};
for (auto it = lst.begin(); it != lst.end(); ) {
    if (*it % 2 == 0) {
        it = lst.erase(it);  // 推荐写法
    } else {
        ++it;
    }
}
```

或者：

```cpp
std::list<int> lst = {1, 2, 3, 4, 5};
for (auto it = lst.begin(); it != lst.end(); ) {
    if (*it % 2 == 0) {
        lst.erase(it++);  // 先保存 it，再自增，然后删除
    } else {
        ++it;
    }
}
```

## 实现支持正确删除的迭代器

在自定义[[12.2 vector容器的迭代器实现|vector 实现]]中，`erase` 应该返回新的有效迭代器：

```cpp
template<typename T>
class vector {
public:
    using iterator = T*;

    // 返回指向被删除元素下一个位置的迭代器
    iterator erase(iterator pos) {
        // 将 pos 之后的元素前移
        for (iterator it = pos; it < _last - 1; ++it) {
            *it = *(it + 1);
        }
        --_last;
        return pos;  // pos 现在指向原来的下一个元素
    }

    // 范围删除
    iterator erase(iterator first, iterator last) {
        // 计算要保留的尾部元素
        iterator tail = _last;
        int count = last - first;

        // 将 last 之后的元素前移
        for (iterator it = last; it < tail; ++it, ++first) {
            *first = *it;
        }

        _last -= count;
        return first - count + 1;  // 根据实际情况调整
    }

private:
    T* _first;
    T* _last;
    T* _end;
};
```

## 常见陷阱与最佳实践

### 陷阱 1：在循环中同时修改容器

```cpp
// 危险！
for (auto it = vec.begin(); it != vec.end(); ++it) {
    if (condition) {
        vec.push_back(newValue);  // 可能导致扩容，所有迭代器失效
    }
}
```

### 陷阱 2：保存 end() 迭代器

```cpp
// 危险！
auto endIt = vec.end();
for (auto it = vec.begin(); it != endIt; ++it) {
    vec.push_back(*it);  // endIt 可能失效
}
```

### 最佳实践

1. **使用 erase 返回值**：删除元素后获取新的有效迭代器
2. **考虑使用算法**：`std::remove_if` + `erase` 惯用法
3. **选择合适的容器**：频繁插入删除考虑 `list`
4. **预分配空间**：使用 `reserve()` 避免扩容导致的失效
5. **使用索引**：对于 vector，索引比迭代器更安全

---

## 关键要点

1. **迭代器失效原因**：内存重新分配、元素移动或删除
2. **vector 最易失效**：扩容会使所有迭代器失效
3. **list 相对安全**：只有被删除元素的迭代器失效
4. **erase 返回值**：使用返回的新迭代器继续遍历
5. **优先使用算法**：`remove_if` + `erase` 更安全高效
