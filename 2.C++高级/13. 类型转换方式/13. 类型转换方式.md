# 13. 类型转换方式

## 概述

C++ 提供了四种类型转换运算符，用于在不同情况下进行类型转换。与 C 风格的强制类型转换 `(type)` 相比，这些转换更加安全和明确。

| 转换方式 | 用途 | 安全性 |
|----------|------|--------|
| `const_cast` | 去掉常量属性 | 安全（仅去const） |
| `static_cast` | 编译器认为安全的类型转换 | 较安全 |
| `reinterpret_cast` | 类似C风格的强制类型转换 | 不安全 |
| `dynamic_cast` | 运行时类型识别（RTTI） | 最安全 |

---

## const_cast

### 作用

**去掉常量属性的类型转换**。用于移除 `const` 或 `volatile` 限定符。

### 语法

```cpp
const_cast<目标类型>(表达式)
```

### 示例

```cpp
const int a = 10;
int* p1 = (int*)&a;                    // C风格转换
int* p2 = const_cast<int*>(&a);        // C++风格转换
```

### 汇编层面

```cpp
int* p1 = (int*)&a;
// 00007FF769A11865  lea    rax, [a]
// 00007FF769A11869  mov    qword ptr [p1], rax

int* p2 = const_cast<int*>(&a);
// 00007FF769A1186D  lea    rax, [a]
// 00007FF769A11871  mov    qword ptr [p2], rax
```

**结论**：在汇编层次是一样的，但是在编译阶段是不同的。

### 编译期检查

```cpp
const int a = 10;
// 内存上是四个字节有效，但是在内存解引用时为八字节有效
double* p1 = (double*)&a;              // C风格：编译通过（危险！）
// 使用 const_cast 则会更加安全，此时在编译时期就报错了
double* p2 = const_cast<double*>(&a);  // 编译错误！类型不匹配
```

### 注意事项

`const_cast` 只能用于[[4. const、指针和引用|指针或者引用]]类型：

```cpp
const int a = 10;
// const_cast<这里必须放指针或者引用类型如：int* int&>
int b = const_cast<int>(a);   // 错误！不能用于非指针/引用类型
```

## static_cast

### 作用

**提供编译器认为安全的类型转换**（没有任何联系的类型之间的转换就被否定）。这是最常用的类型转换。

### 语法

```cpp
static_cast<目标类型>(表达式)
```

### 适用场景

1. 基本数据类型之间的转换
2. 有继承关系的类之间的指针/引用转换
3. `void*` 与其他指针类型之间的转换

### 示例

```cpp
int* p = nullptr;

// 没有任何联系的类型之间的转换就被否定
// 基类类型和派生类类型之间的转换可以使用 static_cast，因为他们之间有关联
short* p = static_cast<short*>(p);    // 错误！int* 和 short* 无关联
double* b = static_cast<double*>(p);  // 错误！int* 和 double* 无关联
```

### 与继承的关系

在有[[6.2.1 继承|继承关系]]的类之间，`static_cast` 可以进行转换：

```cpp
class Base { };
class Derived : public Base { };

Derived* d = new Derived();
Base* b = static_cast<Base*>(d);       // 向上转换：安全
Derived* d2 = static_cast<Derived*>(b); // 向下转换：编译通过但可能不安全
```

---

## reinterpret_cast

### 作用

**类似于 C 风格的强制类型转换**（不安全）。可以在任意指针类型之间转换。

### 语法

```cpp
reinterpret_cast<目标类型>(表达式)
```

### 示例

```cpp
int* p = nullptr;

// 没有任何联系的类型之间的转换也能通过
// 基类类型和派生类类型之间的转换可以使用 static_cast，因为他们之间有关联
short* ps = reinterpret_cast<short*>(p);   // 编译通过
double* pd = reinterpret_cast<double*>(p); // 编译通过
```

### 使用场景

1. 指针和整数之间的转换
2. 任意指针类型之间的转换
3. 函数指针类型之间的转换

### 危险性

`reinterpret_cast` 不会进行任何检查，只是简单地重新解释内存中的二进制表示，使用不当会导致未定义行为。

## dynamic_cast

### 作用

**运行时类型识别（RTTI）**，用于在继承层次中进行安全的向下转换。只能用于包含[[6.2.3 虚函数|虚函数]]的[[6.3.1 深入理解多态|多态]]类型。

### 语法

```cpp
dynamic_cast<目标类型>(表达式)
```

### 工作原理

`dynamic_cast` 会检查 `p` 指针是否指向的是一个目标类型的对象：
- 通过 `p->vfptr->vftable` 中的 RTTI 信息判断
- 如果是，`dynamic_cast` 转换类型成功，返回目标类型对象的地址
- 如果不是，则返回 `nullptr`

### 示例

```cpp
class Base
{
public:
    virtual void func() = 0;
};

class Derive1 : public Base
{
public:
    void func() { cout << "call Derive1::func" << endl; }
};

class Derive2 : public Base
{
public:
    void func() { cout << "call Derive2::func" << endl; }
    // Derive2 实现新功能的 API 接口函数
    void derive02func()
    {
        cout << "call Derive2::derive02func" << endl;
    }
};
```

### 使用 dynamic_cast 进行安全转换

```cpp
// 该函数根据 p 指向的对象，执行不一样的任务。
// 如果 p 指向了 Derive2 则执行 derive02func()，其他就执行 func() 函数
/*
typeid(*p).name() == "Derive"
*/
void showFunc(Base* p)
{
    // dynamic_cast 会检查 p 指针是否指向的是一个 Derive2 类型的对象?
    // p->vfptr->vftable RTTI信息 如果是, dynamic_cast 转换类型成功,
    // 返回 Derive2 对象的地址, 给 pd2; 否则返回 nullptr
    Derive2* pd2 = dynamic_cast<Derive2*>(p);
    if (pd2 != nullptr)
    {
        pd2->derive02func();
    }
    else
    {
        p->func();  // 动态绑定 *p的类型  Derive2  derive02func
    }
}

int main()
{
    Derive1 d1;
    Derive2 d2;

    showFunc(&d1);  // 输出: call Derive1::func
    showFunc(&d2);  // 输出: call Derive2::derive02func

    return 0;
}
```

## 四种转换的对比

| 特性 | const_cast | static_cast | reinterpret_cast | dynamic_cast |
|------|------------|-------------|------------------|--------------|
| **检查时机** | 编译期 | 编译期 | 编译期 | 运行期 |
| **安全性** | 安全 | 较安全 | 不安全 | 最安全 |
| **用途** | 去const | 相关类型转换 | 任意指针转换 | 多态类型向下转换 |
| **需要虚函数** | 否 | 否 | 否 | 是 |
| **失败返回** | 编译错误 | 编译错误 | - | nullptr |

---

## 关键要点

1. **const_cast**：只用于去除 const/volatile，必须是指针或引用
2. **static_cast**：最常用，编译期检查，用于有关联的类型之间
3. **reinterpret_cast**：类似C风格转换，危险但灵活
4. **dynamic_cast**：运行时检查，需要虚函数，返回 nullptr 表示失败
5. **优先级**：优先使用 static_cast，需要 RTTI 时用 dynamic_cast
