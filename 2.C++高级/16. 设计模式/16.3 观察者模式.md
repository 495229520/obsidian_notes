# 观察者模式

> 观察者模式是一种**行为型设计模式**，定义对象间的一对多依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。

---

## 概念定义

观察者模式（Observer Pattern）也称为**发布-订阅模式（Pub-Sub）**。它包含两个核心角色：

| 角色 | 职责 |
|------|------|
| **Subject（被观察者）** | 维护观察者列表，状态变化时通知所有观察者 |
| **Observer（观察者）** | 定义更新接口，接收被观察者的通知 |

```
         ┌─────────────┐
         │   Subject   │
         │  + attach() │
         │  + detach() │
         │  + notify() │
         └──────┬──────┘
                │ notifies
      ┌─────────┼─────────┐
      ▼         ▼         ▼
┌──────────┐ ┌──────────┐ ┌──────────┐
│ObserverA │ │ObserverB │ │ObserverC │
└──────────┘ └──────────┘ └──────────┘
```

---

## 为什么需要

### 问题：紧耦合的状态同步

假设一个股票系统，股价变化时需要更新多个界面：

```cpp
// ❌ 紧耦合方式
class Stock {
    double price_;
    PriceDisplay* display_;      // 直接依赖具体类
    AlertSystem* alert_;
    LogSystem* logger_;

public:
    void setPrice(double price) {
        price_ = price;
        display_->update(price);  // 修改通知逻辑需要改这里
        alert_->check(price);
        logger_->log(price);
    }
};
```

**问题分析**：
1. **高耦合**：Stock 直接依赖所有消费者
2. **违反开闭原则**：新增消费者需要修改 Stock 类
3. **难以维护**：通知逻辑散落在业务代码中

### 解决：观察者模式解耦

```cpp
// ✅ 使用观察者模式
class Stock : public Subject {
public:
    void setPrice(double price) {
        price_ = price;
        notify();  // 自动通知所有观察者
    }
};

// 新增观察者无需修改 Stock
stock.attach(std::make_shared<PriceDisplay>());
stock.attach(std::make_shared<AlertSystem>());
```

## 基本实现

### 1. 定义观察者接口

观察者需要实现统一的更新接口：

```cpp
// 观察者抽象基类
class Observer {
public:
    virtual ~Observer() = default;
    virtual void update(double price) = 0;  // 接收通知
};
```

**设计要点**：
- 使用虚析构函数支持多态删除
- `update()` 参数可以是具体数据或 Subject 引用

### 2. 实现被观察者

被观察者负责管理观察者列表和发送通知：

```cpp
class Subject {
protected:
    std::vector<std::weak_ptr<Observer>> observers_;  // 使用 weak_ptr 避免循环引用

public:
    virtual ~Subject() = default;

    void attach(std::shared_ptr<Observer> observer) {
        observers_.push_back(observer);
    }

    void detach(std::shared_ptr<Observer> observer) {
        observers_.erase(
            std::remove_if(observers_.begin(), observers_.end(),
                [&observer](const std::weak_ptr<Observer>& wp) {
                    auto sp = wp.lock();
                    return !sp || sp == observer;
                }),
            observers_.end()
        );
    }

    void notify(double price) {
        // 清理已失效的观察者并通知有效的观察者
        for (auto it = observers_.begin(); it != observers_.end(); ) {
            if (auto sp = it->lock()) {
                sp->update(price);
                ++it;
            } else {
                it = observers_.erase(it);  // 自动清理失效观察者
            }
        }
    }
};
```

**关键设计决策**：

| 设计选择 | 原因 |
|---------|------|
| 使用 `weak_ptr` | 避免循环引用导致内存泄漏 |
| notify 时清理失效观察者 | 惰性清理，减少开销 |
| 接受 `shared_ptr` 参数 | 与 `weak_ptr` 配合使用 |

### 3. 实现具体类

```cpp
// 具体被观察者：股票
class Stock : public Subject {
    std::string symbol_;
    double price_ = 0.0;

public:
    explicit Stock(std::string symbol) : symbol_(std::move(symbol)) {}

    void setPrice(double price) {
        if (price_ != price) {
            price_ = price;
            notify(price);  // 价格变化时通知观察者
        }
    }

    double getPrice() const { return price_; }
    const std::string& getSymbol() const { return symbol_; }
};
```

```cpp
// 具体观察者：价格显示
class PriceDisplay : public Observer {
    std::string name_;

public:
    explicit PriceDisplay(std::string name) : name_(std::move(name)) {}

    void update(double price) override {
        std::cout << name_ << ": Price updated to " << price << "\n";
    }
};

// 具体观察者：价格警报
class PriceAlert : public Observer {
    double threshold_;

public:
    explicit PriceAlert(double threshold) : threshold_(threshold) {}

    void update(double price) override {
        if (price > threshold_) {
            std::cout << "ALERT: Price " << price
                      << " exceeds threshold " << threshold_ << "\n";
        }
    }
};
```

### 4. 使用示例

```cpp
int main() {
    Stock stock("AAPL");

    auto display1 = std::make_shared<PriceDisplay>("Display1");
    auto display2 = std::make_shared<PriceDisplay>("Display2");
    auto alert = std::make_shared<PriceAlert>(150.0);

    stock.attach(display1);
    stock.attach(display2);
    stock.attach(alert);

    stock.setPrice(145.0);  // 两个 display 更新，alert 不触发
    std::cout << "---\n";
    stock.setPrice(155.0);  // 两个 display 更新，alert 触发

    stock.detach(display2);
    std::cout << "---\n";
    stock.setPrice(160.0);  // 只有 display1 和 alert
}
```

**输出**：
```
Display1: Price updated to 145
Display2: Price updated to 145
---
Display1: Price updated to 155
Display2: Price updated to 155
ALERT: Price 155 exceeds threshold 150
---
Display1: Price updated to 160
ALERT: Price 160 exceeds threshold 150
```

## 推拉模型

观察者模式有两种信息传递方式：

### 推模型（Push）

Subject 主动将数据推送给 Observer：

```cpp
void update(double price, const std::string& symbol, time_t timestamp);
```

| 优点 | 缺点 |
|-----|------|
| Observer 实现简单 | Subject 需要预知 Observer 需求 |
| 一次调用获取所有数据 | 可能传递冗余数据 |

### 拉模型（Pull）

Observer 主动从 Subject 拉取数据：

```cpp
class Observer {
public:
    virtual void update(Subject& subject) = 0;  // 传递 Subject 引用
};

// Observer 实现中按需获取数据
void PriceDisplay::update(Subject& subject) {
    auto& stock = static_cast<Stock&>(subject);
    double price = stock.getPrice();  // 按需拉取
}
```

| 优点 | 缺点 |
|-----|------|
| Subject 无需知道 Observer 需求 | Observer 需要知道 Subject 接口 |
| 按需获取，避免冗余 | 多次调用可能有开销 |

**实践建议**：简单场景用推模型，复杂场景用拉模型。

## 线程安全实现

在多线程环境中，观察者模式需要考虑线程安全：

```cpp
class ThreadSafeSubject {
    mutable std::mutex mutex_;
    std::vector<std::weak_ptr<Observer>> observers_;

public:
    void attach(std::shared_ptr<Observer> observer) {
        std::lock_guard<std::mutex> lock(mutex_);
        observers_.push_back(observer);
    }

    void detach(std::shared_ptr<Observer> observer) {
        std::lock_guard<std::mutex> lock(mutex_);
        // ... 同上
    }

    void notify(double price) {
        // 复制观察者列表，避免持锁调用回调
        std::vector<std::shared_ptr<Observer>> validObservers;
        {
            std::lock_guard<std::mutex> lock(mutex_);
            for (const auto& wp : observers_) {
                if (auto sp = wp.lock()) {
                    validObservers.push_back(sp);
                }
            }
        }
        // 锁外通知，避免死锁
        for (const auto& observer : validObservers) {
            observer->update(price);
        }
    }
};
```

**关键点**：
- 使用 `std::mutex` 保护观察者列表
- **先复制后通知**：避免持锁调用回调导致死锁
- 回调时持有 `shared_ptr`，确保对象不会被销毁

## Modern C++ 实现

### 使用 `std::function` 简化

无需定义 Observer 基类，直接使用回调函数：

```cpp
class ModernStock {
    using Callback = std::function<void(double)>;
    std::vector<Callback> callbacks_;
    double price_ = 0.0;

public:
    // 返回取消订阅的函数
    std::function<void()> subscribe(Callback callback) {
        callbacks_.push_back(std::move(callback));
        size_t index = callbacks_.size() - 1;

        // 返回取消订阅的闭包
        return [this, index]() {
            if (index < callbacks_.size()) {
                callbacks_[index] = nullptr;
            }
        };
    }

    void setPrice(double price) {
        price_ = price;
        for (const auto& cb : callbacks_) {
            if (cb) cb(price);
        }
    }
};
```

**使用示例**：

```cpp
ModernStock stock;

auto unsubscribe = stock.subscribe([](double price) {
    std::cout << "Lambda observer: " << price << "\n";
});

stock.setPrice(100.0);  // 输出: Lambda observer: 100
unsubscribe();          // 取消订阅
stock.setPrice(110.0);  // 无输出
```

### 使用信号槽库

实际项目中推荐使用成熟的信号槽库，如 Boost.Signals2：

```cpp
#include <boost/signals2.hpp>

class Stock {
    boost::signals2::signal<void(double)> priceChanged;
    double price_ = 0.0;

public:
    // 返回连接对象，用于断开
    boost::signals2::connection onPriceChanged(
        const std::function<void(double)>& slot) {
        return priceChanged.connect(slot);
    }

    void setPrice(double price) {
        price_ = price;
        priceChanged(price);  // 触发信号
    }
};
```

**Boost.Signals2 的优势**：
- 线程安全
- 自动管理连接生命周期
- 支持槽的排序和返回值组合

## 陷阱与误区

> [!warning] 常见问题
> 1. **内存泄漏**：观察者未正确注销
> 2. **循环引用**：Subject 持有 Observer 的 shared_ptr，Observer 又持有 Subject
> 3. **通知风暴**：频繁更新导致大量通知
> 4. **通知顺序依赖**：假设特定的通知顺序

### 避免循环引用

```cpp
// ❌ 循环引用
class BadObserver : public Observer {
    std::shared_ptr<Subject> subject_;  // 持有 Subject 的 shared_ptr
};

// ✅ 使用 weak_ptr 打破循环
class GoodObserver : public Observer {
    std::weak_ptr<Subject> subject_;  // 使用 weak_ptr
};
```

### 避免通知风暴

```cpp
class Stock : public Subject {
public:
    // 批量更新，只通知一次
    void batchUpdate(const std::vector<double>& prices) {
        for (double p : prices) {
            price_ = p;  // 不通知
        }
        notify(price_);  // 最后统一通知
    }
};
```

## 应用场景

| 场景 | 说明 |
|------|------|
| **GUI 事件系统** | 按钮点击通知多个处理器 |
| **数据绑定** | Model 变化自动更新 View |
| **消息队列** | 生产者-消费者模式 |
| **日志系统** | 日志事件通知多个输出端 |
| **配置热更新** | 配置变化通知所有使用者 |

## 与其他模式的关系

| 模式 | 关系 |
|------|------|
| **中介者模式** | 都涉及对象间通信，但中介者集中管理 |
| **发布-订阅模式** | 观察者是简化版，Pub-Sub 引入消息代理 |
| **命令模式** | 可结合使用，将通知封装为命令对象 |
