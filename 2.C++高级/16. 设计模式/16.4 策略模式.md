# 策略模式

> 策略模式是一种**行为型设计模式**，定义一系列算法，将每个算法封装起来，并使它们可以互相替换。策略模式让算法独立于使用它的客户端而变化。

---

## 概念定义

策略模式（Strategy Pattern）将算法的**定义与使用分离**，通过组合而非继承实现行为的动态切换。

核心角色：

| 角色 | 职责 |
|------|------|
| **Strategy（策略接口）** | 定义算法的公共接口 |
| **ConcreteStrategy（具体策略）** | 实现具体的算法 |
| **Context（上下文）** | 持有策略引用，委托策略执行算法 |

```
┌─────────────┐         ┌─────────────────┐
│   Context   │────────▶│    Strategy     │
│             │         │  + execute()    │
└─────────────┘         └────────┬────────┘
                                 │
                    ┌────────────┼────────────┐
                    ▼            ▼            ▼
              ┌──────────┐ ┌──────────┐ ┌──────────┐
              │StrategyA │ │StrategyB │ │StrategyC │
              └──────────┘ └──────────┘ └──────────┘
```

---

## 为什么需要

### 问题：if-else 地狱

假设一个支付系统支持多种支付方式：

```cpp
// ❌ 条件分支地狱
class PaymentProcessor {
public:
    void pay(const std::string& method, double amount) {
        if (method == "credit") {
            // 信用卡支付逻辑，50行代码
        } else if (method == "paypal") {
            // PayPal支付逻辑，40行代码
        } else if (method == "bitcoin") {
            // 比特币支付逻辑，60行代码
        }
        // 新增支付方式需要在此添加 else if
    }
};
```

**问题分析**：
1. **违反开闭原则**：新增支付方式需修改核心类
2. **代码膨胀**：所有算法挤在一个方法中
3. **难以测试**：无法单独测试某种支付方式
4. **职责混乱**：PaymentProcessor 承担了过多职责

### 解决：策略模式

```cpp
// ✅ 使用策略模式
PaymentProcessor processor;
processor.setStrategy(std::make_unique<CreditCardPayment>());
processor.pay(100.0);  // 使用信用卡支付

processor.setStrategy(std::make_unique<PayPalPayment>());
processor.pay(100.0);  // 使用 PayPal 支付
```

## 基本实现

### 1. 定义策略接口

```cpp
// 策略接口
class PaymentStrategy {
public:
    virtual ~PaymentStrategy() = default;
    virtual void pay(double amount) = 0;
    virtual std::string getName() const = 0;
};
```

**设计要点**：
- 接口只定义**必要的**方法
- 使用虚析构函数支持多态

### 2. 实现具体策略

每种支付方式实现为独立的策略类：

```cpp
// 具体策略：信用卡支付
class CreditCardPayment : public PaymentStrategy {
    std::string cardNumber_;
    std::string cvv_;

public:
    CreditCardPayment(std::string cardNumber, std::string cvv)
        : cardNumber_(std::move(cardNumber))
        , cvv_(std::move(cvv)) {}

    void pay(double amount) override {
        std::cout << "Paying $" << amount
                  << " with credit card " << maskCard() << "\n";
        // 实际的支付逻辑...
    }

    std::string getName() const override { return "Credit Card"; }

private:
    std::string maskCard() const {
        // 只显示最后4位
        return "****-****-****-" + cardNumber_.substr(cardNumber_.size() - 4);
    }
};
```

```cpp
// 具体策略：PayPal 支付
class PayPalPayment : public PaymentStrategy {
    std::string email_;

public:
    explicit PayPalPayment(std::string email)
        : email_(std::move(email)) {}

    void pay(double amount) override {
        std::cout << "Paying $" << amount
                  << " via PayPal account: " << email_ << "\n";
    }

    std::string getName() const override { return "PayPal"; }
};
```

```cpp
// 具体策略：比特币支付
class BitcoinPayment : public PaymentStrategy {
    std::string walletAddress_;

public:
    explicit BitcoinPayment(std::string address)
        : walletAddress_(std::move(address)) {}

    void pay(double amount) override {
        double btcAmount = amount / 40000.0;  // 示例汇率
        std::cout << "Paying " << btcAmount << " BTC to "
                  << walletAddress_ << "\n";
    }

    std::string getName() const override { return "Bitcoin"; }
};
```

**设计要点**：
- 每个策略类**只关注自己的算法**
- 不同策略可以有不同的构造参数
- 策略类可以独立测试

### 3. 实现上下文

```cpp
// 上下文：支付处理器
class PaymentProcessor {
    std::unique_ptr<PaymentStrategy> strategy_;

public:
    void setStrategy(std::unique_ptr<PaymentStrategy> strategy) {
        strategy_ = std::move(strategy);
    }

    void processPayment(double amount) {
        if (!strategy_) {
            throw std::runtime_error("Payment strategy not set");
        }
        std::cout << "Processing payment using " << strategy_->getName() << "\n";
        strategy_->pay(amount);
    }
};
```

**关键点**：
- 使用 `unique_ptr` 管理策略对象的生命周期
- Context 不关心具体使用哪种策略
- 策略可以在运行时动态切换

### 4. 使用示例

```cpp
int main() {
    PaymentProcessor processor;

    // 使用信用卡支付
    processor.setStrategy(
        std::make_unique<CreditCardPayment>("1234567890123456", "123")
    );
    processor.processPayment(99.99);

    // 切换到 PayPal
    processor.setStrategy(
        std::make_unique<PayPalPayment>("user@example.com")
    );
    processor.processPayment(49.99);

    // 切换到比特币
    processor.setStrategy(
        std::make_unique<BitcoinPayment>("1A2b3C4d5E6f...")
    );
    processor.processPayment(199.99);
}
```

**输出**：
```
Processing payment using Credit Card
Paying $99.99 with credit card ****-****-****-3456
Processing payment using PayPal
Paying $49.99 via PayPal account: user@example.com
Processing payment using Bitcoin
Paying 0.00499975 BTC to 1A2b3C4d5E6f...
```

## Modern C++ 实现

### 使用 `std::function` 替代继承

对于简单策略，可以用函数对象替代类继承：

```cpp
class ModernPaymentProcessor {
    using PaymentFunc = std::function<void(double)>;
    PaymentFunc paymentStrategy_;

public:
    void setStrategy(PaymentFunc strategy) {
        paymentStrategy_ = std::move(strategy);
    }

    void pay(double amount) {
        if (paymentStrategy_) {
            paymentStrategy_(amount);
        }
    }
};
```

**使用示例**：

```cpp
ModernPaymentProcessor processor;

// 使用 lambda 定义策略
processor.setStrategy([](double amount) {
    std::cout << "Cash payment: $" << amount << "\n";
});
processor.pay(50.0);

// 使用成员函数
class WeChatPay {
public:
    void process(double amount) {
        std::cout << "WeChat Pay: ￥" << amount * 7.2 << "\n";
    }
};

WeChatPay wechat;
processor.setStrategy([&wechat](double amt) { wechat.process(amt); });
processor.pay(100.0);
```

**优点**：无需定义接口类，更加灵活。

### 使用 `std::variant` 实现（C++17）

```cpp
// 定义策略类型（无需共同基类）
struct CashPayment {
    void pay(double amount) const {
        std::cout << "Cash: $" << amount << "\n";
    }
};

struct CardPayment {
    std::string cardNo;
    void pay(double amount) const {
        std::cout << "Card " << cardNo << ": $" << amount << "\n";
    }
};

// 使用 variant 持有任意策略
using PaymentStrategy = std::variant<CashPayment, CardPayment>;

class VariantPaymentProcessor {
    PaymentStrategy strategy_;

public:
    void setStrategy(PaymentStrategy s) { strategy_ = std::move(s); }

    void pay(double amount) {
        std::visit([amount](auto& s) { s.pay(amount); }, strategy_);
    }
};
```

**优点**：
- 无虚函数开销
- 值语义，无堆分配
- 编译期类型检查

### 使用模板策略（编译期策略）

当策略在编译期确定时，可使用模板：

```cpp
template<typename Strategy>
class PaymentProcessor {
    Strategy strategy_;

public:
    explicit PaymentProcessor(Strategy s = Strategy{})
        : strategy_(std::move(s)) {}

    void pay(double amount) {
        strategy_.pay(amount);
    }
};

// 编译期确定策略
PaymentProcessor<CashPayment> cashProcessor;
cashProcessor.pay(100.0);
```

**优点**：零开销抽象，无虚函数调用。
**缺点**：策略无法在运行时切换。

## 策略选择器模式

当策略选择逻辑复杂时，可以引入策略选择器：

```cpp
class PaymentStrategyFactory {
public:
    static std::unique_ptr<PaymentStrategy> create(
        const std::string& type,
        const std::map<std::string, std::string>& config)
    {
        if (type == "credit") {
            return std::make_unique<CreditCardPayment>(
                config.at("card"), config.at("cvv"));
        } else if (type == "paypal") {
            return std::make_unique<PayPalPayment>(config.at("email"));
        } else if (type == "bitcoin") {
            return std::make_unique<BitcoinPayment>(config.at("wallet"));
        }
        throw std::invalid_argument("Unknown payment type: " + type);
    }
};

// 使用
auto strategy = PaymentStrategyFactory::create("credit", {
    {"card", "1234567890123456"},
    {"cvv", "123"}
});
```

**设计要点**：结合[[16.2 工厂模式]]来创建策略对象。

## 与其他模式的对比

### 策略 vs 状态模式

| 策略模式 | 状态模式 |
|---------|---------|
| 客户端选择策略 | 状态自动转换 |
| 策略之间相互独立 | 状态之间有转换关系 |
| 替换算法 | 改变行为（随状态） |

### 策略 vs 模板方法

| 策略模式 | 模板方法 |
|---------|---------|
| 组合实现 | 继承实现 |
| 运行时切换 | 编译期确定 |
| 更灵活 | 更简单 |

## 应用场景

| 场景 | 说明 |
|------|------|
| **排序算法** | 不同场景使用不同排序策略 |
| **压缩算法** | ZIP、GZIP、LZ4 等 |
| **验证规则** | 不同字段使用不同验证策略 |
| **日志格式** | JSON、文本、XML 格式切换 |
| **网络请求** | HTTP、WebSocket、gRPC 切换 |
| **缓存策略** | LRU、LFU、FIFO 切换 |

---

## 最佳实践

| 推荐 | 避免 | 原因 |
|-----|------|------|
| 策略接口保持简洁 | 接口方法过多 | 降低实现复杂度 |
| 使用 `unique_ptr` 持有策略 | 裸指针 | RAII 管理生命周期 |
| 策略无状态或状态独立 | 策略依赖 Context 内部状态 | 保持策略可复用 |
| 简单场景用 `std::function` | 过度设计 | 避免类爆炸 |

## 陷阱与误区

> [!warning] 常见错误
> 过度使用策略模式，为每个简单变化都创建策略类

```cpp
// ❌ 过度设计：简单的加减乘除没必要用策略模式
class AddStrategy { ... };
class SubStrategy { ... };

// ✅ 直接使用 lambda 或函数指针
using BinaryOp = std::function<int(int, int)>;
int calculate(int a, int b, BinaryOp op) {
    return op(a, b);
}
calculate(1, 2, std::plus<int>{});
```

**何时使用策略模式**：
- 算法逻辑复杂（超过 10 行）
- 算法需要独立测试
- 算法需要运行时切换
- 有多个相关算法需要统一管理

## 完整示例：排序策略

```cpp
#include <algorithm>
#include <functional>
#include <vector>

// 排序策略接口
template<typename T>
class SortStrategy {
public:
    virtual ~SortStrategy() = default;
    virtual void sort(std::vector<T>& data) = 0;
    virtual std::string getName() const = 0;
};

// 快速排序策略
template<typename T>
class QuickSort : public SortStrategy<T> {
public:
    void sort(std::vector<T>& data) override {
        std::sort(data.begin(), data.end());
    }
    std::string getName() const override { return "QuickSort"; }
};

// 稳定排序策略
template<typename T>
class StableSort : public SortStrategy<T> {
public:
    void sort(std::vector<T>& data) override {
        std::stable_sort(data.begin(), data.end());
    }
    std::string getName() const override { return "StableSort"; }
};

// 堆排序策略
template<typename T>
class HeapSort : public SortStrategy<T> {
public:
    void sort(std::vector<T>& data) override {
        std::make_heap(data.begin(), data.end());
        std::sort_heap(data.begin(), data.end());
    }
    std::string getName() const override { return "HeapSort"; }
};

// 排序上下文
template<typename T>
class Sorter {
    std::unique_ptr<SortStrategy<T>> strategy_;

public:
    void setStrategy(std::unique_ptr<SortStrategy<T>> s) {
        strategy_ = std::move(s);
    }

    void sort(std::vector<T>& data) {
        if (strategy_) {
            std::cout << "Using " << strategy_->getName() << "\n";
            strategy_->sort(data);
        }
    }
};
```
