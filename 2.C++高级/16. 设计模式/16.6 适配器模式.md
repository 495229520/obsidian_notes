# 适配器模式

> 适配器模式是一种**结构型设计模式**，将一个类的接口转换成客户端期望的另一个接口。适配器让原本接口不兼容的类可以协同工作。

---

## 概念定义

适配器模式（Adapter Pattern）通过**包装对象**来转换接口，使不兼容的类能够一起工作。它就像现实中的电源适配器，将不同规格的插头转换为可用的接口。

核心角色：

| 角色 | 职责 |
|------|------|
| **Target（目标接口）** | 客户端期望的接口 |
| **Adaptee（被适配者）** | 需要适配的现有类 |
| **Adapter（适配器）** | 将 Adaptee 接口转换为 Target 接口 |

两种实现方式：

```
类适配器（继承）                    对象适配器（组合）

┌────────┐    ┌─────────┐         ┌────────┐    ┌─────────┐
│ Target │    │ Adaptee │         │ Target │    │ Adaptee │
└───┬────┘    └────┬────┘         └───┬────┘    └─────────┘
    │              │                  │              ▲
    └──────┬───────┘                  │              │
           │                          ▼              │
    ┌──────┴──────┐            ┌─────────────┐       │
    │   Adapter   │            │   Adapter   │───────┘
    │  (多重继承)  │            │  (持有引用)  │ adaptee_
    └─────────────┘            └─────────────┘
```

## 为什么需要

### 问题：接口不兼容

假设我们有一个绘图系统，使用 `Shape` 接口：

```cpp
// 系统期望的接口
class Shape {
public:
    virtual void draw() = 0;
    virtual ~Shape() = default;
};

// 我们自己的形状类
class Circle : public Shape {
public:
    void draw() override {
        std::cout << "Drawing circle\n";
    }
};
```

现在要集成一个第三方图形库，但它的接口不同：

```cpp
// ❌ 第三方库接口不兼容
class LegacyRectangle {
public:
    void drawRectangle(int x1, int y1, int x2, int y2) {
        std::cout << "Drawing rectangle from (" << x1 << "," << y1
                  << ") to (" << x2 << "," << y2 << ")\n";
    }
};

// 无法直接使用：
std::vector<std::unique_ptr<Shape>> shapes;
shapes.push_back(std::make_unique<Circle>());
// shapes.push_back(std::make_unique<LegacyRectangle>()); // ❌ 编译错误
```

**问题分析**：
1. **接口签名不同**：`draw()` vs `drawRectangle(x1, y1, x2, y2)`
2. **无法修改第三方代码**：源码不可用或不应修改
3. **需要统一调用方式**：客户端不应关心实现细节

### 解决：适配器模式

```cpp
// ✅ 使用适配器
class RectangleAdapter : public Shape {
    LegacyRectangle legacy_;
    int x1_, y1_, x2_, y2_;

public:
    RectangleAdapter(int x1, int y1, int x2, int y2)
        : x1_(x1), y1_(y1), x2_(x2), y2_(y2) {}

    void draw() override {
        legacy_.drawRectangle(x1_, y1_, x2_, y2_);
    }
};

// 现在可以统一使用
shapes.push_back(std::make_unique<RectangleAdapter>(0, 0, 100, 50));
for (auto& shape : shapes) {
    shape->draw();  // 统一接口调用
}
```

## 两种实现方式

### 1. 对象适配器（组合，推荐）

通过**组合**持有被适配对象：

```cpp
// 目标接口
class MediaPlayer {
public:
    virtual ~MediaPlayer() = default;
    virtual void play(const std::string& filename) = 0;
};

// 被适配者：高级播放器（不同接口）
class AdvancedMediaPlayer {
public:
    virtual ~AdvancedMediaPlayer() = default;
    virtual void playVlc(const std::string& filename) = 0;
    virtual void playMp4(const std::string& filename) = 0;
};

class VlcPlayer : public AdvancedMediaPlayer {
public:
    void playVlc(const std::string& filename) override {
        std::cout << "Playing VLC: " << filename << "\n";
    }
    void playMp4(const std::string&) override {}  // 不支持
};

class Mp4Player : public AdvancedMediaPlayer {
public:
    void playVlc(const std::string&) override {}  // 不支持
    void playMp4(const std::string& filename) override {
        std::cout << "Playing MP4: " << filename << "\n";
    }
};
```

**适配器实现**：

```cpp
// 对象适配器
class MediaAdapter : public MediaPlayer {
    std::unique_ptr<AdvancedMediaPlayer> advancedPlayer_;
    std::string audioType_;

public:
    explicit MediaAdapter(const std::string& audioType)
        : audioType_(audioType)
    {
        if (audioType == "vlc") {
            advancedPlayer_ = std::make_unique<VlcPlayer>();
        } else if (audioType == "mp4") {
            advancedPlayer_ = std::make_unique<Mp4Player>();
        }
    }

    void play(const std::string& filename) override {
        if (audioType_ == "vlc") {
            advancedPlayer_->playVlc(filename);
        } else if (audioType_ == "mp4") {
            advancedPlayer_->playMp4(filename);
        }
    }
};
```

**关键点**：
- 适配器通过**组合**持有被适配对象
- 在适配器方法中**委托**给被适配对象
- 使用 `unique_ptr` 管理生命周期

### 2. 类适配器（继承）

通过**多重继承**同时继承目标接口和被适配者：

```cpp
// 被适配者
class LegacyPrinter {
public:
    void printDocument(const std::string& doc) {
        std::cout << "Legacy printing: " << doc << "\n";
    }
};

// 目标接口
class ModernPrinter {
public:
    virtual ~ModernPrinter() = default;
    virtual void print(const std::string& doc) = 0;
};

// 类适配器：多重继承
class PrinterAdapter : public ModernPrinter, private LegacyPrinter {
public:
    void print(const std::string& doc) override {
        // 调用继承的方法
        printDocument(doc);
    }
};
```

**对比**：

| 对象适配器 | 类适配器 |
|-----------|---------|
| 组合关系 | 继承关系 |
| 可适配 Adaptee 的子类 | 只能适配特定类 |
| 运行时指定被适配对象 | 编译期绑定 |
| **推荐使用** | 受限于多重继承 |

## STL 中的适配器

### 容器适配器

STL 的 `stack`、`queue`、`priority_queue` 都是容器适配器：

```cpp
// std::stack 适配器（简化版）
template<typename T, typename Container = std::deque<T>>
class stack {
protected:
    Container c;  // 底层容器

public:
    void push(const T& value) { c.push_back(value); }
    void pop() { c.pop_back(); }
    T& top() { return c.back(); }
    bool empty() const { return c.empty(); }
    size_t size() const { return c.size(); }
};
```

**设计思路**：
- `stack` 将 `deque`/`vector`/`list` 的接口适配为栈接口
- 隐藏了底层容器的 `push_front`、`operator[]` 等方法
- 只暴露栈所需的 `push`、`pop`、`top` 操作

**使用示例**：

```cpp
// 使用不同底层容器
std::stack<int> s1;                              // 默认 deque
std::stack<int, std::vector<int>> s2;            // 使用 vector
std::stack<int, std::list<int>> s3;              // 使用 list

s1.push(1);
s1.push(2);
std::cout << s1.top() << "\n";  // 2
s1.pop();
```

### 迭代器适配器

```cpp
// 反向迭代器适配器
std::vector<int> vec{1, 2, 3, 4, 5};

// 正向迭代
for (auto it = vec.begin(); it != vec.end(); ++it) {
    std::cout << *it << " ";  // 1 2 3 4 5
}

// 使用反向迭代器适配器
for (auto it = vec.rbegin(); it != vec.rend(); ++it) {
    std::cout << *it << " ";  // 5 4 3 2 1
}
```

**常用迭代器适配器**：

| 适配器 | 功能 |
|--------|------|
| `reverse_iterator` | 反向遍历 |
| `back_insert_iterator` | 尾部插入 |
| `front_insert_iterator` | 头部插入 |
| `insert_iterator` | 任意位置插入 |
| `move_iterator` | 移动语义遍历 |

```cpp
std::vector<int> src{1, 2, 3};
std::vector<int> dst;

// 使用 back_inserter 适配器
std::copy(src.begin(), src.end(), std::back_inserter(dst));

// 等价于
for (auto& x : src) {
    dst.push_back(x);
}
```

### 函数适配器

```cpp
// std::bind 适配器
void greet(const std::string& greeting, const std::string& name) {
    std::cout << greeting << ", " << name << "!\n";
}

// 适配为单参数函数
auto sayHello = std::bind(greet, "Hello", std::placeholders::_1);
sayHello("World");  // Hello, World!

// std::mem_fn 适配器
struct Widget {
    void process() { std::cout << "Processing\n"; }
};

std::vector<Widget> widgets(3);
std::for_each(widgets.begin(), widgets.end(), std::mem_fn(&Widget::process));

// std::not_fn (C++17)
auto isOdd = [](int x) { return x % 2 != 0; };
auto isEven = std::not_fn(isOdd);
std::cout << isEven(4) << "\n";  // true
```

## 实际应用：数据库接口适配

假设系统使用统一的数据库接口，但需要支持多种数据库：

```cpp
// 统一数据库接口（Target）
class Database {
public:
    virtual ~Database() = default;
    virtual void connect(const std::string& connectionString) = 0;
    virtual void execute(const std::string& query) = 0;
    virtual std::vector<std::string> fetch() = 0;
    virtual void disconnect() = 0;
};

// MySQL 库（Adaptee）- 假设是第三方库
class MySQLConnection {
public:
    void mysqlConnect(const char* host, int port, const char* user, const char* pass) {
        std::cout << "MySQL: Connecting to " << host << ":" << port << "\n";
    }
    void mysqlQuery(const char* sql) {
        std::cout << "MySQL: Executing " << sql << "\n";
    }
    char** mysqlFetchRows() {
        std::cout << "MySQL: Fetching rows\n";
        return nullptr;
    }
    void mysqlClose() {
        std::cout << "MySQL: Connection closed\n";
    }
};

// PostgreSQL 库（Adaptee）- 假设是第三方库
class PGConnection {
public:
    void pgConnect(const std::string& connInfo) {
        std::cout << "PostgreSQL: Connecting with " << connInfo << "\n";
    }
    void pgExec(const std::string& command) {
        std::cout << "PostgreSQL: Executing " << command << "\n";
    }
    std::vector<std::string> pgGetResult() {
        std::cout << "PostgreSQL: Getting result\n";
        return {};
    }
    void pgFinish() {
        std::cout << "PostgreSQL: Connection finished\n";
    }
};
```

**适配器实现**：

```cpp
// MySQL 适配器
class MySQLAdapter : public Database {
    MySQLConnection mysql_;
    std::string host_;
    int port_;
    std::string user_;
    std::string password_;

public:
    MySQLAdapter(std::string host, int port, std::string user, std::string pass)
        : host_(std::move(host)), port_(port)
        , user_(std::move(user)), password_(std::move(pass)) {}

    void connect(const std::string&) override {
        mysql_.mysqlConnect(host_.c_str(), port_, user_.c_str(), password_.c_str());
    }

    void execute(const std::string& query) override {
        mysql_.mysqlQuery(query.c_str());
    }

    std::vector<std::string> fetch() override {
        mysql_.mysqlFetchRows();
        return {};  // 转换为标准格式
    }

    void disconnect() override {
        mysql_.mysqlClose();
    }
};

// PostgreSQL 适配器
class PostgreSQLAdapter : public Database {
    PGConnection pg_;

public:
    void connect(const std::string& connectionString) override {
        pg_.pgConnect(connectionString);
    }

    void execute(const std::string& query) override {
        pg_.pgExec(query);
    }

    std::vector<std::string> fetch() override {
        return pg_.pgGetResult();
    }

    void disconnect() override {
        pg_.pgFinish();
    }
};
```

**使用示例**：

```cpp
// 工厂函数创建适配器
std::unique_ptr<Database> createDatabase(const std::string& type) {
    if (type == "mysql") {
        return std::make_unique<MySQLAdapter>("localhost", 3306, "root", "password");
    } else if (type == "postgresql") {
        return std::make_unique<PostgreSQLAdapter>();
    }
    throw std::invalid_argument("Unknown database type");
}

int main() {
    // 统一的使用方式
    auto db = createDatabase("mysql");
    db->connect("");
    db->execute("SELECT * FROM users");
    auto results = db->fetch();
    db->disconnect();

    // 切换数据库类型，代码不变
    db = createDatabase("postgresql");
    db->connect("host=localhost dbname=test");
    db->execute("SELECT * FROM users");
    results = db->fetch();
    db->disconnect();
}
```

## 双向适配器

有时需要在两种接口之间相互转换：

```cpp
// 两种不同的日志接口
class OldLogger {
public:
    virtual void writeLog(int level, const char* message) = 0;
    virtual ~OldLogger() = default;
};

class NewLogger {
public:
    virtual void log(const std::string& level, const std::string& message) = 0;
    virtual ~NewLogger() = default;
};

// 双向适配器
class LoggerAdapter : public OldLogger, public NewLogger {
    OldLogger* oldLogger_ = nullptr;
    NewLogger* newLogger_ = nullptr;

public:
    // 适配旧接口
    explicit LoggerAdapter(OldLogger* old) : oldLogger_(old) {}
    // 适配新接口
    explicit LoggerAdapter(NewLogger* newL) : newLogger_(newL) {}

    // 实现旧接口（内部使用新日志器）
    void writeLog(int level, const char* message) override {
        if (newLogger_) {
            std::string levelStr = (level == 0) ? "INFO" : "ERROR";
            newLogger_->log(levelStr, message);
        }
    }

    // 实现新接口（内部使用旧日志器）
    void log(const std::string& level, const std::string& message) override {
        if (oldLogger_) {
            int levelInt = (level == "ERROR") ? 1 : 0;
            oldLogger_->writeLog(levelInt, message.c_str());
        }
    }
};
```

## 与其他模式的对比

### 适配器 vs 装饰器

| 适配器模式 | 装饰器模式 |
|-----------|-----------|
| **转换**接口 | **扩展**功能 |
| 接口不变，实现不同 | 接口相同，增加行为 |
| 让不兼容的类协作 | 动态添加职责 |

### 适配器 vs 外观（Facade）

| 适配器模式 | 外观模式 |
|-----------|---------|
| 转换**一个**类的接口 | 简化**多个**类的接口 |
| 接口不兼容时使用 | 接口过于复杂时使用 |
| 适配已有代码 | 提供简化入口 |

### 适配器 vs 代理

| 适配器模式 | 代理模式 |
|-----------|---------|
| 转换接口 | 接口相同 |
| 改变调用方式 | 控制访问 |

## 最佳实践

| 推荐 | 避免 | 原因 |
|-----|------|------|
| 优先使用对象适配器 | 类适配器（多重继承） | 更灵活，可适配子类 |
| 适配器保持简单 | 在适配器中添加业务逻辑 | 适配器只做接口转换 |
| 结合工厂模式创建 | 客户端直接创建适配器 | 隐藏适配细节 |
| 使用智能指针管理 | 裸指针传递 | RAII 管理生命周期 |

## 陷阱与误区

> [!warning] 常见错误
> 过度使用适配器导致系统复杂化

```cpp
// ❌ 过度适配：多层适配器
class A { };
class AtoB : public B { A a; };  // A 适配为 B
class BtoC : public C { B b; };  // B 适配为 C
class CtoD : public D { C c; };  // C 适配为 D

// 调用链太长，难以维护和调试
```

**注意事项**：
1. **适配器不是万能的**：如果接口差异过大，可能需要重新设计
2. **性能开销**：每次调用都经过适配器转换
3. **避免多层适配**：一般不应超过两层适配器
4. **优先考虑重构**：如果可以修改源码，直接重构优于适配

```cpp
// 何时适合使用适配器？
// ✅ 集成第三方库
// ✅ 兼容遗留代码
// ✅ 统一多种实现的接口

// 何时不适合？
// ❌ 可以直接修改代码
// ❌ 接口差异过大
// ❌ 需要多层适配
```
