# 装饰器模式

> 装饰器模式是一种**结构型设计模式**，允许在不修改原有对象的情况下，动态地给对象添加额外的职责。装饰器比继承更灵活。

---

## 概念定义

装饰器模式（Decorator Pattern）通过**包装对象**的方式扩展功能，而非通过继承。装饰器与被装饰对象实现相同的接口，使得客户端可以透明地使用它们。

核心角色：

| 角色 | 职责 |
|------|------|
| **Component（组件接口）** | 定义对象的公共接口 |
| **ConcreteComponent（具体组件）** | 被装饰的原始对象 |
| **Decorator（装饰器基类）** | 持有组件引用，实现组件接口 |
| **ConcreteDecorator（具体装饰器）** | 添加具体的额外职责 |

```
         ┌────────────────┐
         │   Component    │
         │  + operation() │
         └───────┬────────┘
                 │
        ┌────────┴────────┐
        ▼                 ▼
┌───────────────┐  ┌──────────────┐
│ Concrete      │  │  Decorator   │──────┐
│ Component     │  │              │      │
└───────────────┘  └──────┬───────┘      │
                          │              │ component_
              ┌───────────┴───────────┐  │
              ▼                       ▼  │
       ┌─────────────┐         ┌─────────────┐
       │ DecoratorA  │         │ DecoratorB  │
       │ + addedFunc │         │ + otherFunc │
       └─────────────┘         └─────────────┘
```

## 为什么需要

### 问题：继承的局限性

假设我们要开发一个咖啡店系统，咖啡可以加各种配料：

```cpp
// ❌ 使用继承：类爆炸
class Coffee { virtual double cost() = 0; };
class Espresso : public Coffee { };
class EspressoWithMilk : public Coffee { };
class EspressoWithMilkAndSugar : public Coffee { };
class EspressoWithMilkAndSugarAndWhip : public Coffee { };
// ... 组合爆炸，每种组合都需要一个类
```

**问题分析**：
1. **类数量爆炸**：n 种配料需要 2^n 个类
2. **违反开闭原则**：新增配料需要修改大量类
3. **静态绑定**：功能在编译期确定，无法运行时调整
4. **代码重复**：相似的逻辑分散在多个类中

### 解决：装饰器模式

```cpp
// ✅ 使用装饰器：动态组合
auto coffee = std::make_unique<Espresso>();
auto withMilk = std::make_unique<MilkDecorator>(std::move(coffee));
auto withSugar = std::make_unique<SugarDecorator>(std::move(withMilk));
std::cout << withSugar->cost();  // 动态叠加价格
```

## 基本实现

### 1. 定义组件接口

```cpp
// 抽象组件：饮品
class Beverage {
public:
    virtual ~Beverage() = default;
    virtual std::string getDescription() const = 0;
    virtual double cost() const = 0;
};
```

**设计要点**：
- 接口定义**核心行为**
- 虚析构函数支持多态删除

### 2. 实现具体组件

```cpp
// 具体组件：浓缩咖啡
class Espresso : public Beverage {
public:
    std::string getDescription() const override {
        return "Espresso";
    }

    double cost() const override {
        return 1.99;
    }
};

// 具体组件：综合咖啡
class HouseBlend : public Beverage {
public:
    std::string getDescription() const override {
        return "House Blend Coffee";
    }

    double cost() const override {
        return 0.89;
    }
};
```

**设计要点**：具体组件是被装饰的"核心对象"。

### 3. 定义装饰器基类

```cpp
// 装饰器基类
class CondimentDecorator : public Beverage {
protected:
    std::unique_ptr<Beverage> beverage_;  // 持有被装饰对象

public:
    explicit CondimentDecorator(std::unique_ptr<Beverage> beverage)
        : beverage_(std::move(beverage)) {}

    // 默认委托给被装饰对象
    std::string getDescription() const override {
        return beverage_->getDescription();
    }

    double cost() const override {
        return beverage_->cost();
    }
};
```

**关键点**：
- 装饰器**继承组件接口**（是一个 Beverage）
- 装饰器**持有组件引用**（有一个 Beverage）
- 使用 `unique_ptr` 管理生命周期，体现所有权转移

### 4. 实现具体装饰器

每种配料实现为独立的装饰器：

```cpp
// 具体装饰器：牛奶
class MilkDecorator : public CondimentDecorator {
public:
    explicit MilkDecorator(std::unique_ptr<Beverage> beverage)
        : CondimentDecorator(std::move(beverage)) {}

    std::string getDescription() const override {
        return beverage_->getDescription() + ", Milk";
    }

    double cost() const override {
        return beverage_->cost() + 0.30;  // 牛奶加价
    }
};

// 具体装饰器：糖
class SugarDecorator : public CondimentDecorator {
public:
    explicit SugarDecorator(std::unique_ptr<Beverage> beverage)
        : CondimentDecorator(std::move(beverage)) {}

    std::string getDescription() const override {
        return beverage_->getDescription() + ", Sugar";
    }

    double cost() const override {
        return beverage_->cost() + 0.10;
    }
};

// 具体装饰器：奶泡
class WhipDecorator : public CondimentDecorator {
public:
    explicit WhipDecorator(std::unique_ptr<Beverage> beverage)
        : CondimentDecorator(std::move(beverage)) {}

    std::string getDescription() const override {
        return beverage_->getDescription() + ", Whip";
    }

    double cost() const override {
        return beverage_->cost() + 0.50;
    }
};
```

**设计要点**：
- 每个装饰器**只添加一种职责**
- 装饰器可以**任意组合叠加**
- 新增配料只需添加新的装饰器类

### 5. 使用示例

```cpp
int main() {
    // 点一杯浓缩咖啡
    std::unique_ptr<Beverage> beverage = std::make_unique<Espresso>();
    std::cout << beverage->getDescription()
              << " $" << beverage->cost() << "\n";

    // 加牛奶
    beverage = std::make_unique<MilkDecorator>(std::move(beverage));
    std::cout << beverage->getDescription()
              << " $" << beverage->cost() << "\n";

    // 再加糖
    beverage = std::make_unique<SugarDecorator>(std::move(beverage));
    std::cout << beverage->getDescription()
              << " $" << beverage->cost() << "\n";

    // 再加奶泡
    beverage = std::make_unique<WhipDecorator>(std::move(beverage));
    std::cout << beverage->getDescription()
              << " $" << beverage->cost() << "\n";
}
```

**输出**：
```
Espresso $1.99
Espresso, Milk $2.29
Espresso, Milk, Sugar $2.39
Espresso, Milk, Sugar, Whip $2.89
```

## C++ 标准库中的装饰器

### iostream：经典的装饰器应用

C++ 的 IO 流就是装饰器模式的典型应用：

```cpp
// 基础流 → 缓冲流 → 格式化流
std::ofstream file("log.txt");              // 具体组件
std::ostream& buffered = file;               // 已内置缓冲
// 可以叠加更多功能...
```

**流的装饰器结构**：
```
           ┌─────────┐
           │  ios    │  ← 基础接口
           └────┬────┘
                │
           ┌────┴────┐
           │ ostream │  ← 输出流接口
           └────┬────┘
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
┌────────┐ ┌────────┐ ┌──────────────┐
│ofstream│ │cout    │ │ostringstream │
└────────┘ └────────┘ └──────────────┘
```

### 自定义流装饰器

```cpp
// 自定义装饰器：给输出添加时间戳
class TimestampStream : public std::ostream {
    std::ostream& wrapped_;
    std::streambuf* buf_;

    class TimestampBuf : public std::streambuf {
        std::ostream& out_;
        bool newLine_ = true;

    protected:
        int overflow(int c) override {
            if (newLine_ && c != EOF) {
                auto now = std::chrono::system_clock::now();
                auto time = std::chrono::system_clock::to_time_t(now);
                out_ << "[" << std::put_time(std::localtime(&time), "%H:%M:%S") << "] ";
            }
            newLine_ = (c == '\n');
            return out_.put(static_cast<char>(c)) ? c : EOF;
        }

    public:
        explicit TimestampBuf(std::ostream& out) : out_(out) {}
    };

    TimestampBuf tsBuf_;

public:
    explicit TimestampStream(std::ostream& out)
        : std::ostream(&tsBuf_)
        , wrapped_(out)
        , tsBuf_(out) {}
};

// 使用
TimestampStream ts(std::cout);
ts << "Server started\n";      // [14:32:05] Server started
ts << "Listening on 8080\n";   // [14:32:05] Listening on 8080
```

## Modern C++ 实现

### 使用 `std::function` 简化装饰器

对于简单场景，可以用函数组合替代类继承：

```cpp
#include <functional>

// 基础操作
using Operation = std::function<std::string(const std::string&)>;

// 装饰器函数
Operation withPrefix(Operation op, const std::string& prefix) {
    return [op = std::move(op), prefix](const std::string& input) {
        return prefix + op(input);
    };
}

Operation withSuffix(Operation op, const std::string& suffix) {
    return [op = std::move(op), suffix](const std::string& input) {
        return op(input) + suffix;
    };
}

Operation toUpper(Operation op) {
    return [op = std::move(op)](const std::string& input) {
        std::string result = op(input);
        std::transform(result.begin(), result.end(), result.begin(), ::toupper);
        return result;
    };
}

// 使用
int main() {
    Operation base = [](const std::string& s) { return s; };

    auto decorated = toUpper(
        withSuffix(
            withPrefix(base, "[INFO] "),
            " - END"
        )
    );

    std::cout << decorated("hello world") << "\n";
    // 输出: [INFO] HELLO WORLD - END
}
```

**优点**：无需定义大量类，代码更简洁。

### 模板装饰器

```cpp
// 通用装饰器模板
template<typename T>
class LoggingDecorator {
    T wrapped_;
    std::string name_;

public:
    LoggingDecorator(T obj, std::string name)
        : wrapped_(std::move(obj)), name_(std::move(name)) {}

    template<typename... Args>
    auto operator()(Args&&... args) {
        std::cout << "[" << name_ << "] Called with "
                  << sizeof...(args) << " args\n";
        auto result = wrapped_(std::forward<Args>(args)...);
        std::cout << "[" << name_ << "] Returned\n";
        return result;
    }
};

// 辅助函数
template<typename T>
auto makeLogging(T&& obj, const std::string& name) {
    return LoggingDecorator<std::decay_t<T>>(
        std::forward<T>(obj), name
    );
}

// 使用
auto add = [](int a, int b) { return a + b; };
auto loggingAdd = makeLogging(add, "add");
std::cout << loggingAdd(3, 4) << "\n";
// 输出:
// [add] Called with 2 args
// [add] Returned
// 7
```

## 实际应用：日志系统

```cpp
// 日志接口
class Logger {
public:
    virtual ~Logger() = default;
    virtual void log(const std::string& message) = 0;
};

// 基础日志器：输出到控制台
class ConsoleLogger : public Logger {
public:
    void log(const std::string& message) override {
        std::cout << message << "\n";
    }
};

// 装饰器基类
class LoggerDecorator : public Logger {
protected:
    std::unique_ptr<Logger> wrapped_;

public:
    explicit LoggerDecorator(std::unique_ptr<Logger> logger)
        : wrapped_(std::move(logger)) {}
};

// 时间戳装饰器
class TimestampLogger : public LoggerDecorator {
public:
    using LoggerDecorator::LoggerDecorator;

    void log(const std::string& message) override {
        auto now = std::chrono::system_clock::now();
        auto time = std::chrono::system_clock::to_time_t(now);
        std::ostringstream oss;
        oss << "[" << std::put_time(std::localtime(&time), "%Y-%m-%d %H:%M:%S")
            << "] " << message;
        wrapped_->log(oss.str());
    }
};

// 日志级别装饰器
class LevelLogger : public LoggerDecorator {
    std::string level_;

public:
    LevelLogger(std::unique_ptr<Logger> logger, std::string level)
        : LoggerDecorator(std::move(logger)), level_(std::move(level)) {}

    void log(const std::string& message) override {
        wrapped_->log("[" + level_ + "] " + message);
    }
};

// 文件装饰器
class FileLogger : public LoggerDecorator {
    std::ofstream file_;

public:
    FileLogger(std::unique_ptr<Logger> logger, const std::string& filename)
        : LoggerDecorator(std::move(logger))
        , file_(filename, std::ios::app) {}

    void log(const std::string& message) override {
        wrapped_->log(message);  // 先输出到原始目标
        if (file_) {
            file_ << message << "\n";  // 同时写入文件
        }
    }
};

// 使用：组合多个装饰器
int main() {
    auto logger = std::make_unique<ConsoleLogger>();
    auto withTimestamp = std::make_unique<TimestampLogger>(std::move(logger));
    auto withLevel = std::make_unique<LevelLogger>(std::move(withTimestamp), "INFO");
    auto withFile = std::make_unique<FileLogger>(std::move(withLevel), "app.log");

    withFile->log("Application started");
    // 控制台输出: [INFO] [2024-01-15 14:30:00] Application started
    // 同时写入 app.log
}
```

## 与其他模式的对比

### 装饰器 vs 继承

| 装饰器模式 | 继承 |
|-----------|------|
| 运行时组合 | 编译期绑定 |
| 单一职责，可任意组合 | 功能固定 |
| 对象级别扩展 | 类级别扩展 |
| n 个装饰器 = n 个类 | n 种组合 = 2^n 个类 |

### 装饰器 vs 代理

| 装饰器模式 | 代理模式 |
|-----------|---------|
| **增强**功能 | **控制**访问 |
| 可叠加多层 | 通常单层 |
| 客户端创建装饰链 | 代理隐藏实现 |

### 装饰器 vs 策略

| 装饰器模式 | 策略模式 |
|-----------|---------|
| 改变对象的"皮肤" | 改变对象的"内脏" |
| 叠加新行为 | 替换算法 |
| 包装整个对象 | 替换部分行为 |

---

## 最佳实践

| 推荐 | 避免 | 原因 |
|-----|------|------|
| 保持装饰器轻量 | 装饰器中包含复杂逻辑 | 装饰器应该只做"添加" |
| 装饰器与组件实现相同接口 | 改变接口方法签名 | 保证透明性 |
| 使用 `unique_ptr` 管理 | 裸指针传递所有权 | RAII 管理生命周期 |
| 提供工厂方法简化创建 | 让客户端手动构建长链 | 提高易用性 |

## 陷阱与误区

> [!warning] 常见错误
> 装饰器顺序影响结果

```cpp
// ❌ 顺序敏感
auto a = std::make_unique<EncryptDecorator>(
    std::make_unique<CompressDecorator>(data)
);  // 先压缩后加密

auto b = std::make_unique<CompressDecorator>(
    std::make_unique<EncryptDecorator>(data)
);  // 先加密后压缩 - 结果不同！
```

**注意事项**：
1. **装饰器顺序**：某些场景下顺序很重要（如压缩和加密）
2. **性能开销**：每层装饰器都有虚函数调用开销
3. **调试困难**：深层嵌套的装饰链难以调试
4. **对象身份**：装饰后的对象与原对象是不同的实例

```cpp
// 身份问题
auto original = std::make_unique<Espresso>();
auto* ptr = original.get();
auto decorated = std::make_unique<MilkDecorator>(std::move(original));

// ptr != decorated.get()  ← 不是同一个对象
```
