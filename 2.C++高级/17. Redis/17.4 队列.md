# 队列

> Redis 的 List 和 Stream 数据结构天然适合实现消息队列、任务队列和延迟队列，是分布式系统中异步解耦的重要手段。

---

## 概念定义

**Redis 队列** 利用 List 的双向链表特性或 Stream 的日志流特性，实现生产者-消费者模式。核心价值：

- **异步解耦**：生产者和消费者独立运行，互不阻塞
- **削峰填谷**：突发流量先入队，消费者按能力处理
- **可靠传递**：支持消息确认，防止丢失

---

## 队列类型对比

| 类型 | 数据结构 | 特点 | 适用场景 |
|------|---------|------|---------|
| **消息队列** | List | 简单 FIFO | 简单任务分发 |
| **任务队列** | List + Hash | 支持任务状态 | 后台作业处理 |
| **延迟队列** | ZSet | 按时间排序 | 定时任务、订单超时 |
| **Stream** | Stream | 消费组、ACK | 可靠消息传递 |

---

## 消息队列（List 实现）

### 基本原理

利用 List 的 LPUSH/RPOP（或 RPUSH/LPOP）实现先进先出队列。

![[Pasted image 20251224100617.png]]

### Redis 命令

```bash
# 生产者：左侧入队
LPUSH task:queue "task_001"
LPUSH task:queue "task_002"

# 消费者：右侧出队（非阻塞）
RPOP task:queue

# 消费者：右侧出队（阻塞版本，推荐）
BRPOP task:queue 0    # 0 表示无限等待
BRPOP task:queue 30   # 最多等待 30 秒
```

**为什么用 BRPOP**：非阻塞 RPOP 需要轮询，浪费 CPU 和网络资源；BRPOP 在队列为空时阻塞等待，有消息时立即返回。

### C++ 实现

#### 消息队列封装

```cpp
#include <hiredis/hiredis.h>
#include <string>
#include <optional>
#include <memory>
#include <vector>

class RedisMessageQueue {
public:
    explicit RedisMessageQueue(const std::string& queue_name,
                               const std::string& host = "127.0.0.1",
                               int port = 6379)
        : queue_name_(queue_name)
        , ctx_(redisConnect(host.c_str(), port), redisFree)
    {
        if (!ctx_ || ctx_->err) {
            throw std::runtime_error("Redis 连接失败");
        }
    }

    // 发送消息（生产者）
    bool push(const std::string& message) {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "LPUSH %s %s",
                queue_name_.c_str(), message.c_str())
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER;
        if (reply) freeReplyObject(reply);
        return success;
    }

    // 接收消息（消费者，阻塞）
    std::optional<std::string> pop(int timeout_seconds = 0) {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "BRPOP %s %d",
                queue_name_.c_str(), timeout_seconds)
        );

        std::optional<std::string> result;

        // BRPOP 返回 [队列名, 消息内容] 数组
        if (reply && reply->type == REDIS_REPLY_ARRAY && reply->elements == 2) {
            result = std::string(reply->element[1]->str, reply->element[1]->len);
        }

        if (reply) freeReplyObject(reply);
        return result;
    }

    // 获取队列长度
    size_t size() {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "LLEN %s", queue_name_.c_str())
        );

        size_t len = 0;
        if (reply && reply->type == REDIS_REPLY_INTEGER) {
            len = static_cast<size_t>(reply->integer);
        }

        if (reply) freeReplyObject(reply);
        return len;
    }

private:
    std::string queue_name_;
    std::unique_ptr<redisContext, decltype(&redisFree)> ctx_;
};
```

**设计要点**：
- 使用 RAII 管理 Redis 连接
- `pop` 返回 `std::optional`，超时返回空值
- BRPOP 返回的是数组，第二个元素才是消息内容

#### 使用示例

```cpp
#include <iostream>
#include <thread>

void producer(RedisMessageQueue& queue) {
    for (int i = 0; i < 5; ++i) {
        std::string msg = "task_" + std::to_string(i);
        queue.push(msg);
        std::cout << "[生产者] 发送: " << msg << '\n';
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
}

void consumer(RedisMessageQueue& queue) {
    while (true) {
        auto msg = queue.pop(5);  // 最多等待 5 秒
        if (msg) {
            std::cout << "[消费者] 收到: " << *msg << '\n';
        } else {
            std::cout << "[消费者] 超时，退出\n";
            break;
        }
    }
}

int main() {
    RedisMessageQueue queue("demo:queue");

    std::thread t1(producer, std::ref(queue));
    std::thread t2(consumer, std::ref(queue));

    t1.join();
    t2.join();
    return 0;
}
```

## 任务队列（可靠投递）

### 问题：消息丢失

普通队列的问题：消费者取出消息后崩溃，消息丢失。

```
队列: [A, B, C]
        ↓ RPOP
消费者取出 C，正在处理...
        ↓ 消费者崩溃！
消息 C 丢失，无人知晓
```

### 解决方案：RPOPLPUSH

```bash
# 原子操作：从源队列取出，放入备份队列
RPOPLPUSH task:queue task:processing

# 处理完成后，从备份队列删除
LREM task:processing 1 "task_001"
```

**工作流程**：

![[Pasted image 20251224101150.png]]

### C++ 实现

```cpp
class ReliableTaskQueue {
public:
    ReliableTaskQueue(redisContext* ctx, const std::string& name)
        : ctx_(ctx)
        , queue_name_("task:" + name)
        , processing_name_("processing:" + name)
    {}

    // 获取任务（原子地移入处理队列）
    std::optional<std::string> fetch(int timeout = 0) {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "BRPOPLPUSH %s %s %d",
                queue_name_.c_str(), processing_name_.c_str(), timeout)
        );

        std::optional<std::string> result;
        if (reply && reply->type == REDIS_REPLY_STRING) {
            result = std::string(reply->str, reply->len);
        }

        if (reply) freeReplyObject(reply);
        return result;
    }

    // 确认完成（从处理队列移除）
    bool ack(const std::string& task) {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "LREM %s 1 %s",
                processing_name_.c_str(), task.c_str())
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER
                       && reply->integer > 0;
        if (reply) freeReplyObject(reply);
        return success;
    }

    // 恢复超时任务（定时调用）
    int recoverTimeoutTasks() {
        // 将 processing 队列中的任务放回主队列
        int recovered = 0;

        while (true) {
            auto* reply = static_cast<redisReply*>(
                redisCommand(ctx_, "RPOPLPUSH %s %s",
                    processing_name_.c_str(), queue_name_.c_str())
            );

            if (!reply || reply->type != REDIS_REPLY_STRING) {
                if (reply) freeReplyObject(reply);
                break;
            }

            freeReplyObject(reply);
            ++recovered;
        }

        return recovered;
    }

private:
    redisContext* ctx_;
    std::string queue_name_;
    std::string processing_name_;
};
```

**关键点**：
- `fetch` 使用 BRPOPLPUSH 原子操作，消息不会丢失
- `ack` 确认任务完成，从处理队列移除
- `recoverTimeoutTasks` 定期恢复卡住的任务

## 延迟队列（ZSet 实现）

### 应用场景

- **订单超时取消**：下单后 30 分钟未支付自动取消
- **延迟推送**：定时发送消息
- **重试机制**：失败后延迟重试

### 实现原理

利用 ZSet 的 score 存储执行时间戳，按时间排序。

```bash
# 添加延迟任务：score = 执行时间戳
ZADD delay:queue 1703001600 "order:1001:cancel"

# 获取到期任务：score <= 当前时间
ZRANGEBYSCORE delay:queue 0 1703001500 LIMIT 0 10

# 移除已处理的任务
ZREM delay:queue "order:1001:cancel"
```

### C++ 实现

```cpp
#include <chrono>

class DelayQueue {
public:
    DelayQueue(redisContext* ctx, const std::string& name)
        : ctx_(ctx), queue_name_("delay:" + name)
    {}

    // 添加延迟任务
    bool addTask(const std::string& task, int delay_seconds) {
        auto execute_time = std::chrono::system_clock::now()
                          + std::chrono::seconds(delay_seconds);
        auto timestamp = std::chrono::duration_cast<std::chrono::milliseconds>(
            execute_time.time_since_epoch()
        ).count();

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "ZADD %s %lld %s",
                queue_name_.c_str(), timestamp, task.c_str())
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER;
        if (reply) freeReplyObject(reply);
        return success;
    }

    // 获取到期任务（使用 Lua 脚本保证原子性）
    std::vector<std::string> fetchDueTasks(int limit = 10) {
        auto now = std::chrono::duration_cast<std::chrono::milliseconds>(
            std::chrono::system_clock::now().time_since_epoch()
        ).count();

        // Lua 脚本：获取并删除到期任务
        const char* lua_script = R"(
            local tasks = redis.call('ZRANGEBYSCORE', KEYS[1], 0, ARGV[1], 'LIMIT', 0, ARGV[2])
            if #tasks > 0 then
                redis.call('ZREM', KEYS[1], unpack(tasks))
            end
            return tasks
        )";

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "EVAL %s 1 %s %lld %d",
                lua_script, queue_name_.c_str(), now, limit)
        );

        std::vector<std::string> tasks;
        if (reply && reply->type == REDIS_REPLY_ARRAY) {
            for (size_t i = 0; i < reply->elements; ++i) {
                tasks.emplace_back(reply->element[i]->str, reply->element[i]->len);
            }
        }

        if (reply) freeReplyObject(reply);
        return tasks;
    }

    // 获取队列中的任务数量
    size_t size() {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "ZCARD %s", queue_name_.c_str())
        );

        size_t count = 0;
        if (reply && reply->type == REDIS_REPLY_INTEGER) {
            count = static_cast<size_t>(reply->integer);
        }

        if (reply) freeReplyObject(reply);
        return count;
    }

private:
    redisContext* ctx_;
    std::string queue_name_;
};
```

**设计思路**：
- score 使用毫秒时间戳，精度足够且方便比较
- 使用 Lua 脚本原子地获取并删除到期任务，防止重复处理
- `fetchDueTasks` 返回到期的任务列表

#### 使用示例：订单超时取消

```cpp
void orderTimeoutDemo(redisContext* ctx) {
    DelayQueue delay_queue(ctx, "order:timeout");

    // 模拟下单：30 秒后超时
    delay_queue.addTask("order:1001:cancel", 30);
    delay_queue.addTask("order:1002:cancel", 60);

    std::cout << "已添加 " << delay_queue.size() << " 个超时任务\n";

    // 后台线程：轮询处理到期任务
    while (true) {
        auto tasks = delay_queue.fetchDueTasks();

        for (const auto& task : tasks) {
            std::cout << "处理到期任务: " << task << '\n';
            // 执行取消订单逻辑...
        }

        std::this_thread::sleep_for(std::chrono::seconds(1));
    }
}
```

## Stream 消息队列（Redis 5.0+）

### 为什么需要 Stream

List 队列的局限：
- 不支持消费者组（多消费者竞争）
- 不支持消息确认（ACK）
- 消息消费后即删除，无法回溯

Stream 是 Redis 5.0 引入的专业消息队列数据结构。

### 基本命令

```bash
# 发送消息（返回消息 ID）
XADD mystream * field1 value1 field2 value2

# 创建消费者组
XGROUP CREATE mystream mygroup $ MKSTREAM

# 消费者读取消息
XREADGROUP GROUP mygroup consumer1 COUNT 1 BLOCK 5000 STREAMS mystream >

# 确认消息
XACK mystream mygroup 1609459200000-0
```

### C++ 基础封装

```cpp
class RedisStream {
public:
    RedisStream(redisContext* ctx, const std::string& stream)
        : ctx_(ctx), stream_(stream)
    {}

    // 发送消息
    std::string send(const std::vector<std::pair<std::string, std::string>>& fields) {
        std::string cmd = "XADD " + stream_ + " *";
        for (const auto& [key, value] : fields) {
            cmd += " " + key + " " + value;
        }

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, cmd.c_str())
        );

        std::string msg_id;
        if (reply && reply->type == REDIS_REPLY_STRING) {
            msg_id = std::string(reply->str, reply->len);
        }

        if (reply) freeReplyObject(reply);
        return msg_id;
    }

    // 创建消费者组
    bool createGroup(const std::string& group) {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "XGROUP CREATE %s %s $ MKSTREAM",
                stream_.c_str(), group.c_str())
        );

        // 组已存在也算成功
        bool success = reply != nullptr;
        if (reply) freeReplyObject(reply);
        return success;
    }

    // 确认消息
    bool ack(const std::string& group, const std::string& msg_id) {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "XACK %s %s %s",
                stream_.c_str(), group.c_str(), msg_id.c_str())
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER
                       && reply->integer > 0;
        if (reply) freeReplyObject(reply);
        return success;
    }

private:
    redisContext* ctx_;
    std::string stream_;
};
```

## 队列方案选择

![[Pasted image 20251224101344.png]]

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 简单任务分发 | List + BRPOP | 实现简单，性能高 |
| 可靠消息传递 | Stream | 原生支持 ACK |
| 订单超时/定时任务 | ZSet | 按时间排序 |
| 多消费者竞争 | Stream + 消费组 | 自动负载均衡 |

---

## 最佳实践

| 推荐 | 避免 |
|-----|------|
| 使用 BRPOP 阻塞等待 | RPOP 轮询（浪费资源） |
| 消息体使用 JSON 序列化 | 复杂对象直接存储 |
| 设置队列最大长度 | 无限增长导致内存溢出 |
| 任务幂等设计 | 完全依赖队列保证只执行一次 |
| 记录消息 ID 便于排查 | 消息无法追踪 |
