# 分布式锁

> 分布式锁用于在分布式系统中协调多个进程/服务对共享资源的互斥访问，是保证数据一致性的关键机制。

---

## 概念定义

**分布式锁** 是一种跨进程、跨机器的互斥机制。与单机的 [[12.3 互斥量]] 不同，分布式锁需要借助外部存储（如 Redis）来实现多个独立进程之间的协调。

核心要求：
- **互斥性**：同一时刻只有一个客户端能持有锁
- **无死锁**：即使持锁客户端崩溃，锁也能最终释放
- **容错性**：只要大多数 Redis 节点存活，锁服务可用

## Redis 分布式锁原理

### 基本命令：SETNX + EXPIRE

```bash
# SETNX: SET if Not eXists（原子操作）
SETNX lock:order:1001 "holder_uuid"

# 设置过期时间（防止死锁）
EXPIRE lock:order:1001 30
```

**问题**：SETNX 和 EXPIRE 是两个命令，不是原子的！

### 正确方式：SET NX EX（Redis 2.6.12+）

```bash
# 原子操作：设置值 + 过期时间
SET lock:order:1001 "holder_uuid" NX EX 30

# NX: 仅当 key 不存在时设置
# EX 30: 过期时间 30 秒
```

## C++ 实现

### 基础版本

```cpp
#include <hiredis/hiredis.h>
#include <string>
#include <chrono>
#include <thread>
#include <random>

class RedisDistributedLock {
public:
    RedisDistributedLock(redisContext* ctx, const std::string& key,
                         int ttl_seconds = 30)
        : ctx_(ctx)
        , key_("lock:" + key)
        , ttl_(ttl_seconds)
        , token_(generateToken())
        , locked_(false)
    {}

    ~RedisDistributedLock() {
        unlock();
    }

    // 尝试获取锁
    bool tryLock() {
        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "SET %s %s NX EX %d",
                key_.c_str(), token_.c_str(), ttl_)
        );

        if (reply && reply->type == REDIS_REPLY_STATUS &&
            std::string(reply->str) == "OK") {
            locked_ = true;
            freeReplyObject(reply);
            return true;
        }

        if (reply) freeReplyObject(reply);
        return false;
    }

    // 阻塞获取锁（带超时）
    bool lock(int timeout_ms = 5000) {
        auto start = std::chrono::steady_clock::now();
        while (true) {
            if (tryLock()) {
                return true;
            }

            auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>(
                std::chrono::steady_clock::now() - start
            ).count();

            if (elapsed >= timeout_ms) {
                return false;  // 超时
            }

            std::this_thread::sleep_for(std::chrono::milliseconds(10));
        }
    }

    // 释放锁
    bool unlock() {
        if (!locked_) return false;

        // 使用 Lua 脚本保证原子性：只删除自己的锁
        const char* lua_script = R"(
            if redis.call('GET', KEYS[1]) == ARGV[1] then
                return redis.call('DEL', KEYS[1])
            else
                return 0
            end
        )";

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "EVAL %s 1 %s %s",
                lua_script, key_.c_str(), token_.c_str())
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER
                       && reply->integer == 1;

        if (reply) freeReplyObject(reply);
        locked_ = false;
        return success;
    }

    // 续期（防止业务执行时间超过锁过期时间）
    bool extend(int extra_seconds) {
        if (!locked_) return false;

        const char* lua_script = R"(
            if redis.call('GET', KEYS[1]) == ARGV[1] then
                return redis.call('EXPIRE', KEYS[1], ARGV[2])
            else
                return 0
            end
        )";

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_, "EVAL %s 1 %s %s %d",
                lua_script, key_.c_str(), token_.c_str(), extra_seconds)
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER
                       && reply->integer == 1;

        if (reply) freeReplyObject(reply);
        return success;
    }

private:
    redisContext* ctx_;
    std::string key_;
    int ttl_;
    std::string token_;  // 唯一标识，防止误删他人的锁
    bool locked_;

    static std::string generateToken() {
        static std::mt19937_64 gen(std::random_device{}());
        return std::to_string(gen());
    }
};
```

### 使用示例

```cpp
void processOrder(redisContext* ctx, int order_id) {
    // 创建分布式锁
    RedisDistributedLock lock(ctx, "order:" + std::to_string(order_id), 30);

    // 尝试获取锁（最多等待 5 秒）
    if (!lock.lock(5000)) {
        std::cerr << "获取锁超时，订单 " << order_id << " 正在被其他进程处理\n";
        return;
    }

    // 获取锁成功，执行业务逻辑
    std::cout << "开始处理订单 " << order_id << '\n';

    // 模拟业务处理
    std::this_thread::sleep_for(std::chrono::seconds(2));

    std::cout << "订单 " << order_id << " 处理完成\n";

    // 锁会在析构时自动释放
}

int main() {
    auto* ctx = redisConnect("127.0.0.1", 6379);
    if (!ctx || ctx->err) {
        std::cerr << "Redis 连接失败\n";
        return 1;
    }

    processOrder(ctx, 1001);

    redisFree(ctx);
    return 0;
}
```

## RAII 锁守卫

```cpp
// 类似 std::lock_guard 的 RAII 封装
class DistributedLockGuard {
public:
    DistributedLockGuard(redisContext* ctx, const std::string& key,
                         int ttl = 30, int timeout_ms = 5000)
        : lock_(ctx, key, ttl)
        , acquired_(lock_.lock(timeout_ms))
    {}

    ~DistributedLockGuard() = default;  // lock_ 析构时自动释放

    bool acquired() const { return acquired_; }

    explicit operator bool() const { return acquired_; }

private:
    RedisDistributedLock lock_;
    bool acquired_;
};

// 使用
void safeProcess(redisContext* ctx) {
    DistributedLockGuard guard(ctx, "resource:123");

    if (!guard) {
        std::cerr << "获取锁失败\n";
        return;
    }

    // 安全地操作共享资源
    // ...
}  // 自动释放锁
```

## 常见问题与解决方案

### 1. 锁误删

**问题**：A 获取锁后执行超时，锁自动过期；B 获取锁；A 执行完毕后删除了 B 的锁。

```
时间线：
t0: A 获取锁 (TTL=10s)
t5: A 开始执行业务（耗时 15s）
t10: 锁过期自动释放
t11: B 获取锁
t20: A 执行完毕，删除锁 ← 删掉了 B 的锁！
```

**解决**：使用唯一 token，释放时验证

```cpp
// 释放时检查 token
const char* lua_script = R"(
    if redis.call('GET', KEYS[1]) == ARGV[1] then
        return redis.call('DEL', KEYS[1])
    else
        return 0
    end
)";
```

### 2. 锁续期（看门狗机制）

**问题**：业务执行时间不确定，可能超过锁过期时间。

**解决**：后台线程定期续期

```cpp
class WatchdogLock {
public:
    WatchdogLock(redisContext* ctx, const std::string& key, int ttl = 30)
        : lock_(ctx, key, ttl)
        , running_(false)
    {}

    bool lock(int timeout_ms = 5000) {
        if (!lock_.lock(timeout_ms)) {
            return false;
        }

        // 启动看门狗线程
        running_ = true;
        watchdog_ = std::thread([this]() {
            while (running_) {
                std::this_thread::sleep_for(std::chrono::seconds(10));
                if (running_) {
                    lock_.extend(30);  // 每 10 秒续期 30 秒
                }
            }
        });

        return true;
    }

    ~WatchdogLock() {
        running_ = false;
        if (watchdog_.joinable()) {
            watchdog_.join();
        }
    }

private:
    RedisDistributedLock lock_;
    std::atomic<bool> running_;
    std::thread watchdog_;
};
```

## 与本地锁的对比

| 特性 | std::mutex | Redis 分布式锁 |
|------|-----------|----------------|
| 作用范围 | 单进程内 | 跨进程/跨机器 |
| 性能 | 纳秒级 | 毫秒级（网络IO） |
| 死锁处理 | 需手动处理 | 自动过期释放 |
| 可重入 | 否（需 recursive_mutex） | 需额外实现 |
| 故障恢复 | 进程崩溃锁释放 | 需 TTL 机制 |

---

## 最佳实践

| 推荐 | 避免 |
|-----|------|
| 设置合理的过期时间 | 过期时间过长（影响可用性）或过短（锁失效） |
| 使用唯一 token | 直接 DEL 删除锁 |
| 业务幂等设计 | 完全依赖锁保证正确性 |
| Lua 脚本保证原子性 | 多个 Redis 命令分开执行 |
| RAII 封装自动释放 | 手动调用 unlock |
