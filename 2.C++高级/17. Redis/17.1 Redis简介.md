# Redis 简介

> Redis (Remote Dictionary Server) 是一个开源的、基于内存的高性能键值存储系统，广泛用于缓存、消息队列、分布式锁等场景。

---

## 概念定义

Redis 是一个 **内存数据结构存储**，支持多种数据结构（字符串、哈希、列表、集合、有序集合等），具有以下核心特性：

- **极高性能**：纯内存操作，单机 QPS 可达 10万+
- **丰富的数据结构**：不仅仅是简单的 Key-Value 存储
- **原子性操作**：单个命令是原子的，支持事务
- **持久化**：支持 RDB 快照和 AOF 日志

---

## 为什么需要 Redis

### 传统架构的问题

![[Pasted image 20251224094441.png]]

**问题**：
- 数据库查询涉及磁盘 IO，延迟高（毫秒级）
- 高并发下数据库成为瓶颈
- 热点数据重复查询，浪费资源

### 引入 Redis 后

![[Pasted image 20251224094506.png]]

**解决**：
- 热点数据存入内存，读取延迟降到微秒级
- 减轻数据库压力
- 支持更高的并发量

## Redis 数据结构

| 数据结构 | 说明 | 典型应用 |
|---------|------|---------|
| **String** | 字符串/二进制数据 | 缓存、计数器、分布式锁 |
| **Hash** | 字段-值映射表 | 用户信息、对象存储 |
| **List** | 双向链表 | 消息队列、最新列表 |
| **Set** | 无序集合（去重） | 标签、共同好友 |
| **ZSet** | 有序集合 | 排行榜、延迟队列 |
| **Stream** | 消息流（5.0+） | 消息队列、事件日志 |

## 基本操作示例

### String 操作

```bash
# 设置键值
SET user:1001:name "张三"

# 获取值
GET user:1001:name

# 设置过期时间（秒）
SET session:abc123 "user_data" EX 3600

# 原子递增
INCR page:view:count
```

### Hash 操作

```bash
# 设置哈希字段
HSET user:1001 name "张三" age 25 city "北京"

# 获取单个字段
HGET user:1001 name

# 获取所有字段
HGETALL user:1001
```

### List 操作

```bash
# 左侧插入（队列入口）
LPUSH task:queue "task_001"

# 右侧弹出（队列出口，阻塞版本）
BRPOP task:queue 0
```

### ZSet 操作

```bash
# 添加带分数的元素
ZADD leaderboard 9500 "player_A" 8800 "player_B"

# 获取排名前10（降序）
ZREVRANGE leaderboard 0 9 WITHSCORES
```

## C++ 客户端：hiredis

### 安装

```bash
# Ubuntu/Debian
sudo apt-get install libhiredis-dev

# 或从源码编译
git clone https://github.com/redis/hiredis.git
cd hiredis && make && sudo make install
```

### 同步连接示例

```cpp
#include <hiredis/hiredis.h>
#include <iostream>
#include <string>

int main() {
    // 连接 Redis
    redisContext* ctx = redisConnect("127.0.0.1", 6379);
    if (ctx == nullptr || ctx->err) {
        std::cerr << "连接失败: "
                  << (ctx ? ctx->errstr : "无法分配context") << '\n';
        return 1;
    }

    // SET 操作
    redisReply* reply = static_cast<redisReply*>(
        redisCommand(ctx, "SET %s %s", "greeting", "Hello Redis")
    );
    std::cout << "SET: " << reply->str << '\n';
    freeReplyObject(reply);

    // GET 操作
    reply = static_cast<redisReply*>(
        redisCommand(ctx, "GET %s", "greeting")
    );
    if (reply->type == REDIS_REPLY_STRING) {
        std::cout << "GET: " << reply->str << '\n';
    }
    freeReplyObject(reply);

    // 断开连接
    redisFree(ctx);
    return 0;
}
```

## 关键配置参数

| 参数 | 说明 | 建议值 |
|------|------|-------|
| `maxmemory` | 最大内存限制 | 物理内存的 60-70% |
| `maxmemory-policy` | 内存淘汰策略 | `allkeys-lru` |
| `timeout` | 客户端空闲超时 | 300 |
| `tcp-keepalive` | TCP 保活间隔 | 60 |

---

## 注意事项

> [!warning] 内存管理
> Redis 是内存数据库，必须配置 `maxmemory` 防止内存耗尽导致 OOM。

> [!warning] 持久化取舍
> - **RDB**：定时快照，恢复快但可能丢数据
> - **AOF**：追加日志，数据更安全但文件较大
> - 生产环境建议两者结合使用
