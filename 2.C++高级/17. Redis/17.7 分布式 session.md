# 分布式 Session

> 在集群部署环境中，使用 Redis 存储 Session 数据，实现多服务器间的会话共享，是 Web 应用扩展的基础架构。

---

## 概念定义

**Session** 是服务器端存储的用户会话数据，用于跟踪用户状态（登录信息、购物车等）。

**分布式 Session** 将 Session 数据从单机内存迁移到共享存储（如 Redis），使多台服务器能够访问同一份会话数据。

---

## 为什么需要分布式 Session

### 单机 Session 的问题

```
┌─────────────────────────────────────────────────────────┐
│                    单机架构                              │
│                                                         │
│  用户 ──► 服务器 (Session 存在内存中)                    │
│                                                         │
│  ✓ 简单，无需额外组件                                    │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    集群架构                              │
│                                                         │
│              ┌──► 服务器A (Session A)                   │
│  用户 ──► LB │                                          │
│              └──► 服务器B (Session B)  ← 找不到用户会话！ │
│                                                         │
│  ✗ 请求被负载均衡到不同服务器，Session 不共享            │
└─────────────────────────────────────────────────────────┘
```

**问题**：用户登录后，下一次请求可能被路由到另一台服务器，找不到 Session，导致需要重新登录。

### 解决方案对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **Sticky Session** | LB 将同一用户固定到同一服务器 | 简单 | 服务器宕机会话丢失 |
| **Session 复制** | 服务器间同步 Session | 高可用 | 网络开销大，延迟 |
| **Redis Session** | 集中存储到 Redis | 简单、高效 | 依赖 Redis 可用性 |

---

## Redis Session 架构

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                     客户端请求                           │
│                    (带 Session ID)                      │
│                         │                               │
│                         ▼                               │
│                   ┌──────────┐                          │
│                   │ 负载均衡  │                          │
│                   └────┬─────┘                          │
│              ┌─────────┼─────────┐                      │
│              ▼         ▼         ▼                      │
│         服务器A    服务器B    服务器C                    │
│              │         │         │                      │
│              └─────────┼─────────┘                      │
│                        ▼                                │
│                 ┌────────────┐                          │
│                 │   Redis    │                          │
│                 │  Session   │                          │
│                 │   Store    │                          │
│                 └────────────┘                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**流程**：
1. 用户首次访问，服务器创建 Session，存入 Redis，返回 Session ID（通常通过 Cookie）
2. 后续请求携带 Session ID
3. 任意服务器都能从 Redis 读取 Session 数据

## C++ 实现

### Session 数据结构

```cpp
#include <hiredis/hiredis.h>
#include <string>
#include <unordered_map>
#include <optional>
#include <random>
#include <chrono>

// Session 数据：key-value 形式
using SessionData = std::unordered_map<std::string, std::string>;

class RedisSessionStore {
public:
    explicit RedisSessionStore(const std::string& host = "127.0.0.1",
                               int port = 6379,
                               int default_ttl = 1800)  // 默认 30 分钟过期
        : ctx_(redisConnect(host.c_str(), port), redisFree)
        , default_ttl_(default_ttl)
    {
        if (!ctx_ || ctx_->err) {
            throw std::runtime_error("Redis 连接失败");
        }
    }

    // 创建新 Session
    std::string createSession(const SessionData& data = {}) {
        std::string session_id = generateSessionId();
        std::string key = "session:" + session_id;

        // 存储 Session 数据
        if (!data.empty()) {
            std::string cmd = "HSET " + key;
            for (const auto& [field, value] : data) {
                cmd += " " + field + " " + value;
            }

            auto* reply = static_cast<redisReply*>(
                redisCommand(ctx_.get(), cmd.c_str())
            );
            if (reply) freeReplyObject(reply);
        } else {
            // 空 Session，设置一个占位符
            auto* reply = static_cast<redisReply*>(
                redisCommand(ctx_.get(), "HSET %s __created__ %lld",
                    key.c_str(),
                    std::chrono::system_clock::now().time_since_epoch().count())
            );
            if (reply) freeReplyObject(reply);
        }

        // 设置过期时间
        setExpire(session_id);

        return session_id;
    }

    // 获取 Session 数据
    std::optional<SessionData> getSession(const std::string& session_id) {
        std::string key = "session:" + session_id;

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "HGETALL %s", key.c_str())
        );

        if (!reply || reply->type != REDIS_REPLY_ARRAY || reply->elements == 0) {
            if (reply) freeReplyObject(reply);
            return std::nullopt;
        }

        SessionData data;
        for (size_t i = 0; i < reply->elements; i += 2) {
            std::string field(reply->element[i]->str, reply->element[i]->len);
            std::string value(reply->element[i + 1]->str, reply->element[i + 1]->len);

            if (field != "__created__") {  // 过滤内部字段
                data[field] = value;
            }
        }

        freeReplyObject(reply);

        // 刷新过期时间（活跃续期）
        setExpire(session_id);

        return data;
    }

    // 获取单个字段
    std::optional<std::string> get(const std::string& session_id,
                                    const std::string& field) {
        std::string key = "session:" + session_id;

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "HGET %s %s", key.c_str(), field.c_str())
        );

        std::optional<std::string> result;
        if (reply && reply->type == REDIS_REPLY_STRING) {
            result = std::string(reply->str, reply->len);
        }

        if (reply) freeReplyObject(reply);

        if (result) {
            setExpire(session_id);  // 续期
        }

        return result;
    }

    // 设置单个字段
    bool set(const std::string& session_id,
             const std::string& field,
             const std::string& value) {
        std::string key = "session:" + session_id;

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "HSET %s %s %s",
                key.c_str(), field.c_str(), value.c_str())
        );

        bool success = reply != nullptr;
        if (reply) freeReplyObject(reply);

        if (success) {
            setExpire(session_id);  // 续期
        }

        return success;
    }

    // 删除字段
    bool remove(const std::string& session_id, const std::string& field) {
        std::string key = "session:" + session_id;

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "HDEL %s %s", key.c_str(), field.c_str())
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER
                       && reply->integer > 0;
        if (reply) freeReplyObject(reply);
        return success;
    }

    // 销毁 Session（登出）
    bool destroySession(const std::string& session_id) {
        std::string key = "session:" + session_id;

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "DEL %s", key.c_str())
        );

        bool success = reply && reply->type == REDIS_REPLY_INTEGER
                       && reply->integer > 0;
        if (reply) freeReplyObject(reply);
        return success;
    }

    // 检查 Session 是否存在
    bool exists(const std::string& session_id) {
        std::string key = "session:" + session_id;

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "EXISTS %s", key.c_str())
        );

        bool result = reply && reply->type == REDIS_REPLY_INTEGER
                      && reply->integer == 1;
        if (reply) freeReplyObject(reply);
        return result;
    }

    // 获取 Session 剩余时间
    int getTTL(const std::string& session_id) {
        std::string key = "session:" + session_id;

        auto* reply = static_cast<redisReply*>(
            redisCommand(ctx_.get(), "TTL %s", key.c_str())
        );

        int ttl = -1;
        if (reply && reply->type == REDIS_REPLY_INTEGER) {
            ttl = static_cast<int>(reply->integer);
        }

        if (reply) freeReplyObject(reply);
        return ttl;
    }

private:
    std::unique_ptr<redisContext, decltype(&redisFree)> ctx_;
    int default_ttl_;

    // 生成安全的 Session ID
    std::string generateSessionId() {
        static std::mt19937_64 gen(std::random_device{}());
        static const char charset[] =
            "abcdefghijklmnopqrstuvwxyz"
            "ABCDEFGHIJKLMNOPQRSTUVWXYZ"
            "0123456789";

        std::string session_id;
        session_id.reserve(32);

        std::uniform_int_distribution<> dist(0, sizeof(charset) - 2);

        for (int i = 0; i < 32; ++i) {
            session_id += charset[dist(gen)];
        }

        return session_id;
    }

    // 设置/刷新过期时间
    void setExpire(const std::string& session_id) {
        std::string key = "session:" + session_id;
        redisCommand(ctx_.get(), "EXPIRE %s %d", key.c_str(), default_ttl_);
    }
};
```

**设计要点**：
- 使用 Hash 存储 Session 数据，支持字段级操作
- Session ID 使用 32 位随机字符串，足够安全
- 每次读写操作都刷新过期时间（滑动过期）
- `__created__` 内部字段用于空 Session 的占位

### 使用示例

```cpp
#include <iostream>

void sessionDemo() {
    RedisSessionStore store("127.0.0.1", 6379, 1800);  // 30 分钟过期

    // 1. 用户登录，创建 Session
    std::string session_id = store.createSession({
        {"user_id", "1001"},
        {"username", "zhangsan"},
        {"role", "admin"}
    });
    std::cout << "创建 Session: " << session_id << '\n';

    // 2. 后续请求：验证 Session 并获取用户信息
    if (auto data = store.getSession(session_id)) {
        std::cout << "Session 有效\n";
        for (const auto& [k, v] : *data) {
            std::cout << "  " << k << ": " << v << '\n';
        }
    }

    // 3. 更新 Session 数据（如：添加购物车信息）
    store.set(session_id, "cart_count", "3");

    // 4. 获取单个字段
    if (auto cart = store.get(session_id, "cart_count")) {
        std::cout << "购物车数量: " << *cart << '\n';
    }

    // 5. 查看剩余时间
    std::cout << "Session 剩余时间: " << store.getTTL(session_id) << "秒\n";

    // 6. 用户登出
    store.destroySession(session_id);
    std::cout << "Session 已销毁\n";

    // 7. 验证已销毁
    if (!store.exists(session_id)) {
        std::cout << "Session 不存在\n";
    }
}
```

## Session 管理器（完整封装）

```cpp
class SessionManager {
public:
    SessionManager(RedisSessionStore& store) : store_(store) {}

    // 登录：创建 Session
    std::string login(const std::string& user_id,
                      const std::string& username,
                      const std::string& role = "user") {
        return store_.createSession({
            {"user_id", user_id},
            {"username", username},
            {"role", role},
            {"login_time", std::to_string(std::time(nullptr))}
        });
    }

    // 验证 Session，返回用户 ID
    std::optional<std::string> authenticate(const std::string& session_id) {
        return store_.get(session_id, "user_id");
    }

    // 检查权限
    bool hasRole(const std::string& session_id, const std::string& required_role) {
        auto role = store_.get(session_id, "role");
        return role && *role == required_role;
    }

    // 登出
    void logout(const std::string& session_id) {
        store_.destroySession(session_id);
    }

    // 获取当前登录用户信息
    struct UserInfo {
        std::string user_id;
        std::string username;
        std::string role;
        bool valid = false;
    };

    UserInfo getCurrentUser(const std::string& session_id) {
        UserInfo info;

        if (auto data = store_.getSession(session_id)) {
            info.valid = true;
            if (auto it = data->find("user_id"); it != data->end()) {
                info.user_id = it->second;
            }
            if (auto it = data->find("username"); it != data->end()) {
                info.username = it->second;
            }
            if (auto it = data->find("role"); it != data->end()) {
                info.role = it->second;
            }
        }

        return info;
    }

private:
    RedisSessionStore& store_;
};
```

### 在 HTTP 处理中使用

```cpp
// 伪代码：HTTP 请求处理
void handleRequest(HttpRequest& req, HttpResponse& res,
                   SessionManager& session_mgr) {
    // 从 Cookie 获取 Session ID
    std::string session_id = req.getCookie("SESSION_ID");

    // 需要登录的接口
    if (req.path == "/api/profile") {
        auto user = session_mgr.getCurrentUser(session_id);
        if (!user.valid) {
            res.status(401).json({{"error", "unauthorized"}});
            return;
        }

        res.json({
            {"user_id", user.user_id},
            {"username", user.username}
        });
    }

    // 需要管理员权限的接口
    if (req.path == "/api/admin/users") {
        if (!session_mgr.hasRole(session_id, "admin")) {
            res.status(403).json({{"error", "forbidden"}});
            return;
        }

        // 管理员操作...
    }

    // 登录接口
    if (req.path == "/api/login" && req.method == "POST") {
        // 验证用户名密码...
        auto new_session_id = session_mgr.login("1001", "zhangsan", "user");

        res.setCookie("SESSION_ID", new_session_id, {
            .httpOnly = true,
            .secure = true,
            .sameSite = "Strict",
            .maxAge = 1800
        });

        res.json({{"success", true}});
    }

    // 登出接口
    if (req.path == "/api/logout") {
        session_mgr.logout(session_id);
        res.clearCookie("SESSION_ID");
        res.json({{"success", true}});
    }
}
```

## 安全考虑

### Session ID 安全

```cpp
// ✅ 安全的 Session ID 生成
std::string generateSecureSessionId() {
    // 使用密码学安全的随机数生成器
    std::random_device rd;
    std::mt19937_64 gen(rd());

    std::string id;
    id.reserve(64);

    for (int i = 0; i < 64; ++i) {
        static const char chars[] =
            "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789";
        id += chars[gen() % (sizeof(chars) - 1)];
    }

    return id;
}
```

### Session 固定攻击防护

用户登录后，应该重新生成 Session ID：

```cpp
std::string loginAndRegenerateSession(SessionManager& mgr,
                                      RedisSessionStore& store,
                                      const std::string& old_session_id,
                                      const std::string& user_id,
                                      const std::string& username) {
    // 销毁旧 Session
    if (!old_session_id.empty()) {
        store.destroySession(old_session_id);
    }

    // 创建新 Session
    return mgr.login(user_id, username);
}
```

### Cookie 安全设置

```
Set-Cookie: SESSION_ID=xxx;

            Secure;        // 仅 HTTPS 传输
            SameSite=Strict;  // 防止 CSRF
```

## 高可用方案

### Redis 主从 + 哨兵

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│     应用服务器 ──► Sentinel ──► Redis Master            │
│                       │                                 │
│                       └──► Redis Slave (只读副本)       │
│                                                         │
│     Master 宕机时，Sentinel 自动将 Slave 提升为 Master  │
└─────────────────────────────────────────────────────────┘
```

### Redis Cluster

对于超大规模系统，使用 Redis Cluster 分片存储 Session。

## 与 JWT 的对比

| 特性 | Redis Session | JWT |
|------|--------------|-----|
| 存储位置 | 服务端 (Redis) | 客户端 (Token) |
| 状态 | 有状态 | 无状态 |
| 注销 | 立即生效 | 需要黑名单机制 |
| 存储数据量 | 可以存储大量数据 | 有大小限制 |
| 服务器压力 | 每次请求查 Redis | 仅需验证签名 |
| 适用场景 | 传统 Web 应用 | 微服务、移动端 |

**选择建议**：
- 需要立即踢出用户、存储复杂会话数据 → Redis Session
- 无状态微服务、跨域场景多 → JWT

---

## 最佳实践

| 推荐 | 避免 |
|-----|------|
| 使用 64 位以上随机 Session ID | 短或可预测的 ID |
| 设置合理的过期时间（30分钟） | 过期时间过长（安全风险） |
| 滑动过期（每次访问续期） | 固定过期（用户体验差） |
| Cookie 设置 HttpOnly + Secure | 允许 JS 访问 Cookie |
| 登录后重新生成 Session ID | 复用匿名 Session ID |
| Redis 高可用部署 | 单点 Redis |
