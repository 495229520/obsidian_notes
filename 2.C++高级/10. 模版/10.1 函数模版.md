# 10.1 函数模板

## 知识点

1. **函数模板的意义**：对类型也进行参数化，比如 `int sum(int a,int b) {return a+b;}`
2. **函数模版无法进行编译**：因为无法确定 T 的类型
3. **函数模版仅仅在使用的地方可以被编译，因为在函数模版调用的时候可以根据用户的类型实参来推导出模版类型参数的具体类型**
4. 在函数调用点，编译器会根据用户的实参类型来推导出模版类型参数的具体类型
5. 为什么需要模版特例化？C++ 为什么区分普通元素和数组的内存分配和释放？
   - 通过函数指额，编译器根据传入的实参类型来推导出模版类型参数的类型
   - 对于某些类型来说，依赖编译器默认实例化的模版代码，程序运行会出现问题
6. 模版代码不能在一个文件中定义，在另一个文件中使用
7. 模版代码调用之前，一定要看到模版定义的地方，这样的话，模版才能够进行正常的实例化，产生能够被编译器编译的代码

---

## 函数模板语法

```cpp
template<typename T>    // 定义一个模板参数列表
bool compare(T a, T b)  // compare是一个函数模板
{
    cout << "template compare" << endl;
    return a > b;
}
```

### 函数模板 vs 普通函数 vs 特例化

| 类型 | 语法 | 说明 |
|------|------|------|
| **函数模板** | `template<typename T> bool compare(T a, T b)` | 定义一个模板参数列表 |
| **特例化函数模板** | `template<> bool compare<const char*>(const char* a, const char* b)` | 针对 `const char*` 类型的特例化版本 |
| **普通函数** | `bool compare(const char* a, const char* b)` | 非模板函数 |

---

## 案例

```cpp
// 函数模版
template<typename T>    // 定义一个模板参数列表
bool compare(T a, T b)  // compare是一个函数模板
{
    cout << "template compare" << endl;
    return a > b;
}

// 针对compare函数模版，提供const char*类型的特例化版本
template<>
bool compare<const char*>(const char* a, const char* b)
{
    cout << "compare<const char*>" << endl;
    return strcmp(a, b) > 0;
}

// 非模版函数 - 普通函数
bool compare(const char* a, const char* b)
{
    cout << "normal compare" << endl;
    return strcmp(a, b) > 0;
}

int main()
{
    compare<int>(10, 20);
    // 通过函数指额，编译器根据传入的实参类型来推导出模版类型参数的类型
    compare(10.5, 20.5);

    // 两个参数的类型不相同，编译器无法确定T的类型，因此报错
    // compare(30, 40.5);
    // 指定模版参数类型之后可以编译，此时double会被强转为int类型参数
    compare<int>(30, 40.5);

    // 对于某些类型来说，依赖编译器默认实例化的模版代码，程序运行会出现问题
    // 比较的const char*是地址的大小比较，而不是字符串的值是比较
    // 对于compare函数模版来说，对于const char*类型的比较是存在问题的
    compare("aaa", "bbb");
    compare<const char*>("aaa", "bbb");

    return 0;
}
```

## 模板的非类型参数 (Non-type Template Parameter)

### 简介

模板的**非类型参数** (Non-type Template Parameter) 是一种特殊的模板参数，它允许你除了类型之外，还能将值作为模板的参数。通常我们看到的模板参数是用来指定类型的（例如 `typename T` 或 `class T`），而非类型参数则用来指定一个整数、枚举、指针或引用等常量值。

### 作用

非类型模板参数最常见的用途是在编译时确定数组组的大小，或者在类模板中存储一个固定的、编译时就知道的值。

### 特点

1. **必须是编译期常量**：非类型模板参数的值在编译时就必须是已知的。你不能使用运行时才能确定的变量作为非类型参数。
2. **数据类型限制**：
   - **整型**：最常见的是整数类型（int, long, short, char 等），包括它们的 signed 和 unsigned 版本。
   - **枚举类型**：枚举值也可以作为非类型参数。
   - **指针或引用**：指向对象或函数的指针/引用也可以作为非类型参数，但它们必须在编译时是已知的地址（例如全局变量、静态成员变量或函数地址）。局部变量的地址通常不能作为非类型参数。
3. **不可修改**：在模板内部，非类型参数的值是常量，不能被修改。

### 案例

```cpp
template<typename T, int SIZE>
void sort(T* arr)
{
    for (int i = 0; i < SIZE - 1; ++i)
    {
        for (int j = 0; j < SIZE - 1 - i; ++j)
        {
            if (arr[j] > arr[j + 1])
            {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

int main()
{
    int arr[] = { 12, 5, 7, 89, 32, 21, 35 };
    const int size = sizeof(arr) / sizeof(arr[0]);
    // int SIZE是非类型模板参数，它要求一个编译期常量值
    // SIZE 期望的是一个整数值，而不是一个类型
    sort<int, size>(arr);
    for (int val : arr)
    {
        cout << val << " ";
    }
    return 0;
}
```

## 规范化

函数模版的参数类型需要是 **const引用**。

### 原因

1. **避免不必要的拷贝**：如果参数不是引用类型，那么在函数调用时会创建操作数的一个副本。对于大型对象，这会带来显著的性能开销。使用引用可以直接操作原始对象，避免了拷贝。
2. **效率和性能**：通过引用传递避免了对象的拷贝构造函数和析构函数的调用，从而提高了程序的运行效率。
3. **避免修改原始对象（const 的作用）**：
   - 保持操作符的语义一致性：多数运算符（例如加法 +，比较运算符 ==、!=）通常不应该改变其操作数。将参数声明为 const 引用可以强制编译器检查并防止在运算符函数内部意外地修改操作数。
   - 结合了按值传递的"纯粹性"：对于那些设计为不修改其输入的操作符（例如，从C风格字符串自定义字符串类），使用 const 引用可以有效防止由于不当的隐式类型转换（例如，从 C 风格字符串）而引起的循环递归。
4. **避免循环递归**：使用 const 引用作为重载运算符的参数，可以有效防止由于不当的对象拷贝和更精确的类型匹配规则来达这种意图，并防止误用。

---

## 关键要点

1. **函数模板**：对类型参数化，编译时根据实参类型推导
2. **模板特例化**：为特定类型提供专门实现
3. **非类型参数**：编译期常量，常用于数组大小
4. **模板代码位置**：调用前必须能看到模板定义
5. **const引用参数**：避免拷贝，提高效率
