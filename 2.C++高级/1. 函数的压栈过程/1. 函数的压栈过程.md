## 概述

函数参数的压栈过程是理解[[1.2 函数栈帧建立与销毁原理|函数栈帧]]的重要组成部分。本节主要讨论默认参数在函数调用时的压栈行为。

---

## 注意点

1. **给默认值的时候，从右向左给**
2. **默认值的调用效率会更高**，因为不用 `mov` 汇编将常量保存在 `ptr` 中，然后再压栈。而是直接压入默认值
3. **定义可以给出形参默认值，声明也可以给形参默认值**
4. **形参给默认值的时候，不管是定义处给还是声明处给，形参默认值只能出现一次**

---

## 案例

```cpp
#include <iostream>
using namespace std;

// 压栈是从右向左的，因此默认值也是从右向左
// 如果左边有默认值，右边没有就会报错
int sum(int a = 10, int b = 20) {
    return a + b;
}

int main()
{
    int a = 10;
    int b = 20;

    /*
    mov eax, dword ptr[ebp-8]
    push eax
    mov ecx, dword ptr[ebp-4]
    push ecx
    call sum
    */
    int ret = sum(a, b);
    cout << "ret:" << ret << endl;

    /*
    没有b，因此直接调用默认值:20
    push 14H
    mov ecx, dword ptr[ebp-4]
    push ecx
    call sum
    */
    ret = sum(a);

    /*
    push 14H
    push 0AH
    call sum
    */
    ret = sum();
}
```

---

## 汇编层面分析

### 传入两个参数的情况

```asm
; int ret = sum(a, b);
mov eax, dword ptr[ebp-8]    ; 取出 b 的值
push eax                      ; b 入栈
mov ecx, dword ptr[ebp-4]    ; 取出 a 的值
push ecx                      ; a 入栈
call sum
```

### 只传入一个参数的情况

```asm
; ret = sum(a);
; 没有b，因此直接调用默认值:20 (0x14)
push 14H                      ; 默认值 20 直接压栈
mov ecx, dword ptr[ebp-4]    ; 取出 a 的值
push ecx                      ; a 入栈
call sum
```

### 不传入参数的情况

```asm
; ret = sum();
push 14H    ; 默认值 20 (0x14) 直接压栈
push 0AH    ; 默认值 10 (0x0A) 直接压栈
call sum
```

---

## 效率分析

使用默认参数时，编译器可以直接将常量值压栈，省去了从内存中读取变量值的步骤。这是默认参数调用效率更高的原因。

---

## 关键要点

1. **压栈顺序**：参数从右向左压栈（遵循 `__cdecl` 调用约定）
2. **默认参数顺序**：必须从右向左连续给出默认值
3. **效率优势**：默认参数直接压入常量，无需内存访问
4. **声明与定义**：默认值只能在声明或定义中出现一次，不能重复
