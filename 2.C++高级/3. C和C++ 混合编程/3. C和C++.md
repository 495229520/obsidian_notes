# 3. C/C++ 混合编程

## 概述

C 和 C++ 虽然语法相似，但由于[[11.2 编译器基本概念#名称修饰|名称修饰 (Name Mangling)]]机制的差异，直接混合编译会导致链接错误。本节介绍如何实现 C 和 C++ 代码的互相调用。

---

## C 与 C++ 的主要差异

| 特性 | C | C++ |
|------|---|-----|
| **名称修饰** | 不修饰 | 修饰（支持重载） |
| **函数重载** | 不支持 | 支持 |
| **类和对象** | 不支持 | 支持 |
| **异常处理** | 不支持 | 支持 |
| **默认参数** | 不支持 | 支持 |

---

## extern "C" 关键字

`extern "C"` 告诉 C++ 编译器使用 C 语言的链接规范（不进行名称修饰）。

### 基本语法

```cpp
// 单个函数
extern "C" void c_function();

// 多个函数
extern "C" {
    void func1();
    void func2(int x);
}
```

### 头文件模板

```cpp
// my_lib.h - C/C++ 兼容头文件
#ifndef MY_LIB_H
#define MY_LIB_H

#ifdef __cplusplus
extern "C" {
#endif

void my_function(int x);
int my_add(int a, int b);

#ifdef __cplusplus
}
#endif

#endif
```

**说明**：`__cplusplus` 宏只在 C++ 编译器中定义，因此这种写法可以让同一个头文件同时被 C 和 C++ 代码包含。

---

## C++ 调用 C 代码

当 C++ 代码需要调用 C 语言编写的库时：

```cpp
// main.cpp
extern "C" {
    #include "c_library.h"  // C 语言头文件
}

int main() {
    c_function();  // 调用 C 函数
    return 0;
}
```

---

## C 调用 C++ 代码

当 C 代码需要调用 C++ 编写的功能时，需要创建 C 兼容的包装接口：

```cpp
// cpp_wrapper.h
#ifdef __cplusplus
extern "C" {
#endif

void* create_object();
void destroy_object(void* obj);
void do_something(void* obj);

#ifdef __cplusplus
}
#endif
```

```cpp
// cpp_wrapper.cpp
#include "cpp_wrapper.h"
#include "MyClass.h"

extern "C" {
    void* create_object() {
        return new MyClass();
    }

    void destroy_object(void* obj) {
        delete static_cast<MyClass*>(obj);
    }

    void do_something(void* obj) {
        static_cast<MyClass*>(obj)->doSomething();
    }
}
```

---

## 注意事项

1. **链接器选择**：混合编译时使用 `g++` 链接，以包含 C++ 运行时库
2. **内存管理**：`new/delete` 和 `malloc/free` 不要混用
3. **异常处理**：C 不支持异常，需要在边界处捕获
4. **const 语义**：C 和 C++ 的[[10.3 C和C++的const区别|const 语义有差异]]

---

## 关键要点

1. **名称修饰**是混合编程的核心问题
2. **`extern "C"`** 是解决方案的关键
3. **`__cplusplus`** 宏用于编写兼容头文件
4. C++ 调用 C 相对简单，C 调用 C++ 需要包装函数
