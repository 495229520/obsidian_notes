# 6.3.1 深入理解多态

## 静态多态（编译时期）

### 函数重载

```cpp
bool compare(int, int) {}
bool compare(double, double) {}

compare(10, 5.20);      // call compare_int_int
compare(10.5, 20.5);    // call compare_double_double
// 在编译阶段就确定好调用的函数
```

### 函数模版

```cpp
template<typename T>
bool compare(T a, T b) {}

compare<int>(10, 20);       // => int 实例化一个 compare<int>
compare(10.5, 20.5);        // => double 实例化一个 compare<double>
```

## 动态多态（运行时期）

在继承结构中，**基类指针（引用）指向派生类对象，通过该指针（引用）调用同名覆盖方法（虚函数）**，基类指针指向哪个派生类对象，就会调用哪个派生类对象的覆盖方法，这就是多态。

**多态底层是通过动态绑定来实现的**：
`Base->访问值的vfptr->继续访问值的vftable->最后调用相应的派生类对象的覆盖方法`

## 动态绑定和静态绑定

```
                Animal   virtual void bark() {}
                   │
       ┌───────────┼───────────┐
       ▼           ▼           ▼
      Dog         Cat         Pig
 void bark() {} void bark() {} void bark() {}
```

### 代码示例

```cpp
using namespace std;

class Animal
{
public:
    Animal(string name) : _name(name) {}
    virtual void bark() {}
protected:
    string _name;
};

class Cat : public Animal
{
public:
    Cat(string name) : Animal(name) {}
    void bark()
    {
        cout << _name << " miao miao" << endl;
    }
};

class Dog : public Animal
{
public:
    Dog(string name) : Animal(name) {}
    void bark()
    {
        cout << _name << " wang wang" << endl;
    }
};

class Pig : public Animal
{
public:
    Pig(string name) : Animal(name) {}
    void bark()
    {
        cout << _name << " heng heng" << endl;
    }
};

// 静态绑定的实现方式
void bark(Cat& cat)
{
    cat.bark();
}
void bark(Dog& dog)
{
    dog.bark();
}
void bark(Pig& pig)
{
    pig.bark();
}

int main()
{
    Cat cat("汤姆");
    Dog dog("布莱恩");
    Pig pig("佩奇");

    // 静态绑定，没有涉及到多态
    // 但是这样违背了"开闭原则"：对修改关闭，对扩展开放
    bark(cat);
    bark(dog);
    bark(pig);

    return 0;
}
```

### 动态绑定

```cpp
// 动态绑定实现
void bark(Animal* p)
{
    p->bark();
}

int main()
{
    Cat cat("汤姆");
    Dog dog("布莱恩");
    Pig pig("佩奇");

    // 动态绑定，涉及到多态
    Animal* p = &cat;
    bark(p);
    p = &dog;
    bark(p);
    p = &pig;
    bark(p);

    return 0;
}
```

## 继承的好处

1. **代码的复用**
2. 在基类中提供统一的虚函数接口，在派生类处重写，实现多态

---

## 关键要点

| 多态类型 | 时机 | 实现方式 | 绑定方式 |
|----------|------|----------|----------|
| **静态多态** | 编译时 | 函数重载、模板 | 静态绑定 |
| **动态多态** | 运行时 | 虚函数 + 继承 | 动态绑定 |

### 动态多态的实现条件

1. **继承关系**：必须有基类和派生类
2. **虚函数**：基类中必须有虚函数
3. **覆盖**：派生类必须覆盖基类的虚函数
4. **指针/引用调用**：必须通过基类指针或引用调用虚函数

### 设计原则

- **开闭原则**：对修改关闭，对扩展开放
- 使用动态多态可以在不修改现有代码的情况下扩展新功能
