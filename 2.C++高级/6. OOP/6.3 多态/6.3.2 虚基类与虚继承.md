# 6.3.2 虚基类与虚继承

## 多重继承的缺陷

### 菱形继承问题

```
        A   ma
       / \
      /   \
     B     C   ma、mc
    ma    ma
    mb    mc
     \   /
      \ /
       D   ma、mb
      ma   ma、mc
      mb
      mc
```

当类 D 同时继承 B 和 C，而 B 和 C 又都继承自 A 时，D 中会包含**两份 A 的成员**。这会导致：
- **数据冗余**：A 的成员被重复存储
- **二义性**：访问 A 的成员时编译器不知道该访问哪一份

### 不采用虚继承的代码

```cpp
class A
{
public:
    A(int data) : ma(data) { cout << "A()" << endl; }
    ~A() { cout << "~A()" << endl; }
protected:
    int ma;
};

class B : public A
{
public:
    B(int data) : A(data), mb(data) { cout << "B()" << endl; }
    ~B() { cout << "~B()" << endl; }
protected:
    int mb;
};

class C : public A
{
public:
    C(int data) : A(data), mc(data) { cout << "C()" << endl; }
    ~C() { cout << "~C()" << endl; }
protected:
    int mc;
};

class D : public B, public C
{
public:
    D(int data) : B(data), C(data), md(data) { cout << "C()" << endl; }
    ~D() { cout << "~D()" << endl; }
protected:
    int md;
};

int main()
{
    D d(10);
    return 0;
}
```

### 内存布局（不采用虚继承）

```
┌──────────────┐
│      D       │
├──────────────┤
│    B::       │
│    A::ma     │  ← A 的第一份拷贝
│    mb        │
├──────────────┤
│    C::       │
│    A::ma     │  ← A 的第二份拷贝（冗余）
│    mc        │
├──────────────┤
│    md        │
└──────────────┘

多次调用了 A 的构造函数
```

## 如何解决多重继承的问题？

### 采用虚继承

```cpp
class A
{
public:
    A(int data) : ma(data) { cout << "A()" << endl; }
    ~A() { cout << "~A()" << endl; }
protected:
    int ma;
};

class B : virtual public A  // 虚继承
{
public:
    B(int data) : A(data), mb(data) { cout << "B()" << endl; }
    ~B() { cout << "~B()" << endl; }
protected:
    int mb;
};

class C : virtual public A  // 虚继承
{
public:
    C(int data) : A(data), mc(data) { cout << "C()" << endl; }
    ~C() { cout << "~C()" << endl; }
protected:
    int mc;
};

class D : public B, public C
{
public:
    // 注意：D 必须直接调用 A 的构造函数！
    D(int data) : A(data), B(data), C(data), md(data) { cout << "D()" << endl; }
    ~D() { cout << "~D()" << endl; }
protected:
    int md;
};

int main()
{
    D d(10);
    return 0;
}
```

> **注意**：`ma` 此时在派生类 D 内存空间中，因此**需要 D 自己初始化**，于是多了 `A(data)`

### 内存布局（采用虚继承）

```
┌──────────────┐
│      D       │
├──────────────┤
│    B::       │
│    vbptr  ───┼──► B::vbtable
│    mb        │
├──────────────┤
│    C::       │
│    vbptr  ───┼──► C::vbtable
│    mc        │
├──────────────┤
│    md        │
├──────────────┤
│   A::ma      │  ← A 的成员只有一份！
└──────────────┘

只调用 A 的构造函数一次
```

## 虚基类（Virtual Base Class）

### virtual 关键字的作用

1. **修饰成员方法**时是虚函数
2. **修饰继承方式**时，是虚继承。被虚继承的类，被称为**虚基类**

### 派生类的内存布局变化

1. 不考虑虚继承，考虑普通继承情况下，派生类的内存分配
2. 此时考虑虚继承
3. 把虚基类的数据移动到派生类内存的最下方
4. 在原来基类的位置补充 vbptr 虚基类指针

具体过程如下：

```
┌──────────────┐         ┌──────────────┐
│    vfptr     │         │    vbptr     │
│    ma        │   ──►   │    mb        │
│    mb        │         ├──────────────┤
└──────────────┘         │    vfptr     │
                         │    ma        │
                         └──────────────┘
```

5. **vbptr 指向虚基类表 vbtable**（编译时期生成，运行时期在数据段）

### vbtable 的内容

**vbptr 虚函数指针 距离派生类内存最上方和虚基类的偏移量**

```
┌──────────────┐         B::vbtable:
│    vbptr     │    ──►  ┌──────────────────────────┐
│    mb        │         │ 0 (vbptr距离最上方的偏移量) │
├──────────────┤         │ 8 (vbptr距离虚基类的偏移量) │
│   A::ma      │         └──────────────────────────┘
└──────────────┘
```

- A a: 4 个字节
- B b: ma, mb 8 个字节
- B 虚继承了 A：此时有 8 个字节 + 4（vbptr）= 12 个字节
- `sizeof(B) = 4 vfptr + 8 vbptr + vfptr = 12`

## 为什么会有错误？

```cpp
A* p = new B();
```

```
        ┌──────────────┐
   p ──►│    vbptr     │
        │    mb        │
        ├──────────────┤
        │    vfptr     │
        │    ma        │  ← 虚基类
        └──────────────┘
```

- **基类指针指向派生类对象**
- **永远指向的是派生类基类部分数据的起始地址**

在 Windows VS 编译器中，在 delete 的时候，直接从 vfptr 处进行 free 导致派生类内存空间的释放错误。

在 Linux 下的 GCC 编译器中，则不会有这个错误。

---

## 多重继承总结

- **多重继承的好处**：可以做更多代码的复用
- **多重继承的缺陷**：可能会导致菱形继承，基类构造函数可能会被多次调用，导致派生类有多份间接基类的数据

**解决办法**：使用虚基类，二级基类都需要虚继承一级基类

## 关键要点

| 概念 | 说明 |
|------|------|
| **虚继承** | `class B : virtual public A`，解决菱形继承问题 |
| **虚基类** | 被虚继承的类称为虚基类 |
| **vbptr** | 虚基类指针，指向虚基类表 |
| **vbtable** | 虚基类表，存储偏移量信息 |
