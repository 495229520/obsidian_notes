## 面试题 1：虚函数和多态

### 题目代码

```cpp
using namespace std;

class Base
{
public:
    virtual void show(int i = 10)
    {
        cout << "call Base::show() i:" << i << endl;
    }
};

class Derive : public Base
{
public:
    void show(int i = 20)
    {
        cout << "call Derive::show() i:" << i << endl;
    }
};

int main()
{
    Base* p = new Derive();
    p->show();

    delete p;
    return 0;
}
```

### 输出结果

```
call Derive::show() i:10
```

### 分析

1. **函数调用**：`p->show()` 是通过基类指针调用虚函数
2. **动态绑定**：由于 `show()` 是虚函数，运行时会根据实际对象类型（Derive）调用 `Derive::show()`
3. **参数默认值**：但是参数的默认值是在**编译时期**确定的，编译器只看到 `Base* p`，所以使用 `Base::show()` 的默认值 `i = 10`

### 关键结论

> **虚函数的调用是动态绑定（运行时），但参数的默认值是静态绑定（编译时）！**

**派生类的虚函数应该使用和基类相同的默认参数值**，否则会产生令人困惑的行为。

## 面试题 2：show 函数调用分析

### 题目代码

```cpp
using namespace std;

class Base
{
public:
    virtual void show()
    {
        cout << "call Base::show()" << endl;
    }
};

class Derive : public Base
{
private:  // 注意：这里是 private
    void show()
    {
        cout << "call Derive::show()" << endl;
    }
};

int main()
{
    Base* p = new Derive();
    p->show();  // 能否调用？调用的是谁？

    delete p;
    return 0;
}
```

### 输出结果

```
call Derive::show()
```

### 分析

1. **编译时检查**：编译器看到的是 `Base* p`，检查 `Base::show()` 是 public 的，通过
2. **运行时绑定**：运行时通过 [[6.2.3 虚函数|vfptr]] 找到实际对象（Derive）的虚函数表，调用 `Derive::show()`
3. **访问权限不影响动态绑定**：虽然 `Derive::show()` 是 private，但动态绑定只看 vftable 中的函数地址

### 关键结论

> **虚函数的调用只取决于 vfptr 指向的虚函数表，与访问权限无关！**

但这是**不推荐的设计**。如果派生类想覆盖虚函数，应该保持相同的访问权限。

## 面试题 3：构造函数中调用虚函数

### 题目代码

```cpp
class Base
{
public:
    Base()
    {
        cout << "call Base()" << endl;
        clear();
    }
    void clear() { memset(this, 0, sizeof(*this)); }
    virtual void show()
    {
        cout << "call Base::show()" << endl;
    }
};

class Derive : public Base
{
public:
    Derive()
    {
        cout << "call Derive()" << endl;
    }
    void show()
    {
        cout << "call Derive::show()" << endl;
    }
};

int main()
{
    // 情况1
    Base* pb1 = new Base();
    pb1->show();  // 正常调用
    delete pb1;

    // 情况2
    Base* pb2 = new Derive();
    pb2->show();  // 会发生什么？
    delete pb2;

    return 0;
}
```

### 分析

**情况1**：`new Base()` → 构造 Base 对象
- 构造函数中 `clear()` 把对象清零（包括 vfptr）
- `pb1->show()` 通过 vfptr 找虚函数，但 vfptr 已被清零
- **结果**：程序崩溃或未定义行为

**情况2**：`new Derive()` → 先构造 Base 部分，再构造 Derive 部分
- Base 构造时 vfptr 指向 `Base::vftable`，然后被 `clear()` 清零
- **Derive 构造时会重新设置 vfptr** 指向 `Derive::vftable`
- `pb2->show()` 通过重新设置的 vfptr 找到 `Derive::show()`
- **结果**：正常输出 `call Derive::show()`

### 内存布局变化

```
Base 构造后清零：           Derive 构造后：
┌──────────────┐            ┌──────────────┐
│ vfptr = 0    │    ──►     │ vfptr ───────┼──► Derive::vftable
│ ...          │            │ ...          │
└──────────────┘            └──────────────┘
```

### 关键结论

1. **派生类构造时会重新设置 vfptr**，指向派生类自己的虚函数表
2. **不要在构造函数中调用 memset 清零 this**，这会破坏 vfptr
3. **构造函数中调用虚函数是静态绑定**，不会发生多态

## 面试题总结

| 题目 | 考察点 | 关键结论 |
|------|--------|----------|
| 题目1 | 默认参数与虚函数 | 默认参数是静态绑定，函数调用是动态绑定 |
| 题目2 | 访问权限与虚函数 | 动态绑定不检查访问权限 |
| 题目3 | 构造函数与 vfptr | 派生类构造会重置 vfptr |
