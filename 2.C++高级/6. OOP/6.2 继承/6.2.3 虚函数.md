# 6.2.3 虚函数

## 基础知识

1. **虚函数能产生函数地址**，存储在虚函数表 vftable 中
2. **需要依赖对象**，因为对象中存在虚函数指针，指针指向虚函数表中的虚函数地址（vfptr→vftable→虚函数地址）

### 哪些函数不能实现成虚函数？

| 函数类型 | 能否为虚函数 | 原因 |
|----------|--------------|------|
| **构造函数** | ❌ 不能 | 对象的创建需要先开辟内存空间再调用构造函数，调用构造函数的时候，对象还没有被创建出来 |
| **构造函数中调用的函数** | ❌ 不能 | 构造函数中调用的任何函数都是静态绑定的，因此调用虚函数是不会发生动态绑定的 |
| **static 静态成员方法** | ❌ 不能 | 静态成员不依赖对象 |
| **析构函数** | ✅ **可以** | 因为在调用析构函数时，肯定会存在对象 |

## 虚析构函数

### 案例

```cpp
class Base
{
public:
    Base(int data = 10) : ma(data) { cout << "Base()" << endl; }
    ~Base() { cout << "~Base()" << endl; }
    virtual void show() { cout << "Base::show()" << endl; }
protected:
    int ma;
};

class Derive : public Base
{
public:
    Derive(int data = 20)
        : Base(data), mb(data)
    {
        cout << "Derive()" << endl;
    }
    ~Derive() { cout << "~Derive()" << endl; }
private:
    int mb;
    int* ptr;
};

int main()
{
    Base* pb = new Derive(10);
    pb->show();
    delete pb;
    return 0;
}
```

### 输出

```
Base()
Derive()
Base::show()
~Base()
```

### 问题

派生类对象的析构函数没有被调用！因此在 delete pb 的时候，调用了基类调用析构函数的时候直接把派生类对象给删除了，但是其内部还有 ptr 指针，指向一块堆，此时会导致内存泄漏。

### 内存布局

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Derive d   │     │             │     │             │
│  ┌───────┐  │     │             │     │             │
│  │ vfptr │──┼────►│             │     │             │
│  │  ma   │  │     │             │     │             │
│  │  mb   │  │     │             │     │             │
│  │  ptr  │──┼────►│  堆内存      │     │             │
│  └───────┘  │     │             │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 为什么不会调用派生类的析构函数？

当执行 `delete pb;` 时：
1. `pb` 是一个 Base 类型的指针
2. 因为 `~Base()` **不是虚函数**，delete 操作会进行**静态绑定**，这意味着编译器在编译时就决定了要调用 Base 类的析构函数
3. 因此，只有 `Base::~Base()` 会被调用
4. `Derive::~Derive()` 永远不会被调用到

### 解决办法

**将基类的析构函数变成虚析构函数**

```cpp
public:
    Base(int data = 10) : ma(data) { cout << "Base()" << endl; }
    virtual ~Base() { cout << "~Base()" << endl; }  // 虚析构函数
    virtual void show() { cout << "Base::show()" << endl; }
```

### 什么时候必须把基类的析构函数实现成虚函数？

**基类的指针（引用）指向堆上 new 出来的派生类对象的时候**，delete 基类的指针。

因为，它在调用析构函数的时候，必须发生动态绑定，否则会导致派生类的析构函数无法调用。

## 虚函数表的分配过程

### (1) 编译阶段

**虚函数表的创建**：在编译阶段，编译器会分析类的声明和定义，确定哪些类包含虚函数，并为这些类创建虚函数表。虚函数表中包含指向类中所有虚函数的指针。

### (2) 链接阶段

**虚函数表的分配**：在链接阶段，虚函数表被分配具体的内存地址。链接器会将所有虚函数表放入全局内存空间中，确保每个虚函数表都有唯一的内存地址。

### (3) 运行时

**虚表指针的初始化**：在运行时，当类的对象被实例化时，虚表指针（vptr）被初始化，指向对应类的虚函数表。每个包含虚函数的类的对象都会有一个虚表指针，指向该类的虚函数表。

## 虚函数的动态绑定

**虚函数的调用不是在编译时解析的，而是在运行时解析的，因此知道程序运行时才决定调用哪个函数。**

### 虚函数的调用并不一定全是动态绑定

1. 在类的**构造函数**中，调用虚函数时也是静态绑定（因为构造函数中调用的任何函数都是静态绑定）
2. **使用对象本身调用**虚函数的时候，为静态绑定，因为对象确定了这个函数为对象的成员函数

### 代码示例与汇编分析

```cpp
Base b;
Derive d;

// 用对象本身调用虚函数，是静态绑定
b.show();   // 虚函数
// 00007FF78AA66F49  lea         rcx,[b]
// 00007FF78AA66F4D  call        Base::show (07FF78AA61528h)  ← 直接调用

d.show();   // 虚函数
// 00007FF78AA66F52  nop
// 00007FF78AA66F53  lea         rcx,[d]
// 00007FF78AA66F57  call        Derive::show (07FF78AA61541h)  ← 直接调用

// 动态绑定：必须由指针调用虚函数
Base* pb1 = &b;
pb1->show();
// 00007FF78AA66F65  mov         rax,qword ptr [pb1]
// 00007FF78AA66F69  mov         rax,qword ptr [rax]    ← 取 vfptr
// 00007FF78AA66F6C  mov         rcx,qword ptr [pb1]
// 00007FF78AA66F70  call        qword ptr [rax+8]      ← 从虚函数表调用

Base* pb2 = &d;
pb2->show();
// 00007FF78AA66F7F  mov         rax,qword ptr [pb2]
// 00007FF78AA66F86  mov         rax,qword ptr [rax]    ← 取 vfptr
// 00007FF78AA66F89  mov         rcx,qword ptr [pb2]
// 00007FF78AA66F90  call        qword ptr [rax+8]      ← 从虚函数表调用

// 动态绑定：使用引用底层也是指针调用
Base& rb1 = b;
rb1.show();

Base& rb2 = d;
rb2.show();

// 动态绑定：虚函数通过指针或者引用变量调用时才发生动态绑定
Derive* pd1 = &d;
pd1->show();

Derive& rd1 = d;
rd1.show();
```

---

## 关键要点

1. **虚函数表（vftable）**：存储虚函数地址的表，每个含有虚函数的类都有一个
2. **虚函数指针（vfptr）**：对象中指向虚函数表的指针
3. **动态绑定条件**：通过**指针**或**引用**调用虚函数时才发生
4. **虚析构函数**：当基类指针指向派生类对象时，基类析构函数必须是虚函数
5. **构造函数不能是虚函数**：对象创建时还没有 vfptr
