## 简介

`inline` 是一个关键字，用于建议编译器将函数调用**内联展开**（Inline Expansion），即用函数体直接替换函数调用，从而消除[[1.2 函数栈帧建立与销毁原理|函数调用]]的开销。

> **重要**：`inline` 只是对编译器的**建议**，编译器可以忽略这个建议。

---

## 内联展开的原理

### 普通函数调用

```cpp
int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(3, 5);  // 函数调用开销：压栈、跳转、返回
}
```

### 内联展开后

```cpp
inline int add(int a, int b) {
    return a + b;
}

int main() {
    int result = 3 + 5;  // 直接替换，无函数调用开销
}
```

---

## 注意事项

1. **作用阶段**：`inline` 在编译阶段生效，符号会消失（相当于宏替换但有类型安全）
2. **ODR 豁免**：内联函数不受 [[11.1 ODR唯一定义规则|ODR（One Definition Rule）]] 限制
3. **Debug 模式**：在 Debug 模式下 `inline` 不会生效，方便调试
4. **建议性质**：编译器可以忽略 `inline` 建议，特别是对于复杂函数

## inline 与链接问题

### 问题场景

```cpp
// my_utils.h
#ifndef MY_UTILS_H
#define MY_UTILS_H

void highFrequency() {
    // 高频调用的函数
}

#endif

// 如果不同的 .cpp 文件 #include 了同一个头文件，
// 链接时会报 multiple definition 错误
```

### 问题分析

- 在 C++ 的[[1.1 程序编译与链接原理|编译]]阶段，每个 `.cpp` 文件会被独立编译为一个 `.obj` 文件
- 如果在多个文件中都包含并定义了同一个函数，链接阶段会产生**符号重定义**错误

### 解决方案

#### 方案一：使用 inline

```cpp
// my_utils.h
#ifndef MY_UTILS_H
#define MY_UTILS_H

inline void highFrequency() {
    // 函数体直接在调用处展开
    // 不会产生符号
}

#endif
```

#### 方案二：使用 static

```cpp
// my_utils.h
#ifndef MY_UTILS_H
#define MY_UTILS_H

static void highFrequency() {
    // 每个翻译单元有自己的副本
    // 作用域限制在当前文件
}

#endif
```

## ODR 与 inline

### 唯一定义规则 (One Definition Rule - ODR)

[[11.1 ODR唯一定义规则|ODR]] 的含义：一个函数只能在整个程序中被定义一次（多个翻译单元加起来只能定义一次）。

### inline 豁免 ODR 的条件

1. **同一 `inline` 函数**必须在所有使用它的翻译单元中**定义完全一致**
2. 编译器会在符号表中特别标记 `inline` 函数，链接器会自动合并重复定义

## 类成员函数与 inline

### 类内定义（隐式 inline）

在类的内部定义的成员函数**自动处理为 inline**，不会在类外生成符号。

```cpp
class Counter {
private:
    int count = 0;
public:
    // 类内定义 → 隐式 inline
    int getCount() { return count; }
};
```

### 类外定义

```cpp
// Counter.h
class Counter {
private:
    int count = 0;
public:
    int getCount();  // 声明
};

// 方式一：在头文件中定义（需要 inline）
inline int Counter::getCount() {
    return count;
}

// 方式二：在 .cpp 文件中定义（不需要 inline）
// Counter.cpp
int Counter::getCount() {
    return count;
}
```

## Header-only 类（纯头文件类）

### 概念

C++ 中没有"内联类"这个正式术语，但存在一种常见的设计模式：**Header-only 类**——所有成员函数的实现都放在头文件中，不需要对应的 `.cpp` 文件。

这种类能够只在头文件中实现，正是因为 **inline 机制**：

| 定义位置 | inline 属性 | 是否需要 .cpp |
|---------|-------------|--------------|
| 类内定义 | **隐式 inline** | 不需要 |
| 类外定义（头文件中） | 需要显式 `inline` | 不需要 |
| 类外定义（源文件中） | 不是 inline | 需要 .cpp |

### 示例：典型的 Header-only 类

```cpp
// Vector2D.h - 不需要 Vector2D.cpp
#ifndef VECTOR2D_H
#define VECTOR2D_H

#include <cmath>

class Vector2D {
private:
    double x_, y_;

public:
    // 构造函数 - 类内定义，隐式 inline
    Vector2D(double x = 0, double y = 0) : x_(x), y_(y) {}

    // Getter - 类内定义，隐式 inline
    double x() const { return x_; }
    double y() const { return y_; }

    // 简单运算 - 类内定义，隐式 inline
    Vector2D operator+(const Vector2D& other) const {
        return Vector2D(x_ + other.x_, y_ + other.y_);
    }

    // 较复杂的函数也可以类内定义
    double magnitude() const {
        return std::sqrt(x_ * x_ + y_ * y_);
    }
};

#endif
```

### 类外定义的 Header-only 写法

如果希望保持类定义简洁，可以在头文件中类外定义，但**必须加 `inline`**：

```cpp
// Matrix.h
#ifndef MATRIX_H
#define MATRIX_H

class Matrix {
private:
    double data_[4][4];
public:
    Matrix();
    Matrix multiply(const Matrix& other) const;  // 只声明
};

// 类外定义 - 必须 inline，否则多个 .cpp 包含时会链接错误
inline Matrix::Matrix() {
    for (int i = 0; i < 4; ++i)
        for (int j = 0; j < 4; ++j)
            data_[i][j] = (i == j) ? 1.0 : 0.0;
}

inline Matrix Matrix::multiply(const Matrix& other) const {
    Matrix result;
    // ... 矩阵乘法实现
    return result;
}

#endif
```

### 何时使用 Header-only 类

| 适合场景 | 原因 |
|---------|------|
| **模板类** | 模板必须在头文件中定义，天然 header-only |
| **小型工具类** | 函数体短小，内联展开有性能优势 |
| **接口简单的类** | 减少文件数量，简化项目结构 |
| **库的公开接口** | 用户只需包含头文件，无需链接 |

| 不适合场景 | 原因 |
|-----------|------|
| **函数体庞大** | 内联展开会导致代码膨胀 |
| **频繁修改实现** | 修改头文件导致所有包含者重新编译 |
| **需要隐藏实现** | 头文件会暴露实现细节 |

### 与传统分离式的对比

```
传统分离式                    Header-only
├── MyClass.h  (声明)         ├── MyClass.h  (声明 + 实现)
└── MyClass.cpp (实现)        └── (无需 .cpp)
```

**Header-only 优点**：
- 使用简单，只需 `#include`
- 无需配置编译链接
- 有利于编译器优化（内联展开）

**Header-only 缺点**：
- 头文件修改会触发大量重编译
- 编译时间可能变长
- 实现细节完全暴露

## inline 变量（C++17）

C++17 引入了 `inline` 变量，允许在头文件中定义全局变量或静态成员变量：

```cpp
// settings.h
#ifndef SETTINGS_H
#define SETTINGS_H

#include <string>

struct Settings {
    static inline int MAX_USERS = 100;
    // C++17: 静态成员可以在类内初始化
};

// 或者独立的 inline 变量
inline const std::string APP_VERSION = "1.2.0";

#endif
```

**注意**：`inline` 确保链接器不会抱怨重定义，只保留一个实例。

---

## 示例：头文件中的 inline 使用

```cpp
// my_utils.h
#ifndef MY_UTILS_H
#define MY_UTILS_H

#include <iostream>

inline void printMessage() {
    std::cout << "Program started.\n";
}

// 带有静态局部变量的 inline 函数
inline int& getCount() {
    static int count = 0;  // 所有翻译单元共享同一个 count
    return count;
}

#endif
```

## 关键要点

| 要点 | 说明 |
|------|------|
| **建议性质** | `inline` 是建议，编译器可以忽略 |
| **ODR 豁免** | `inline` 允许在多个翻译单元中定义同一函数 |
| **隐式 inline** | 类内定义的成员函数自动为 inline |
| **Header-only 类** | 所有实现在头文件中，依赖 inline 避免链接错误 |
| **C++17** | `inline` 变量可以在头文件中定义 |
| **Debug 模式** | Debug 模式下 inline 通常不生效 |
