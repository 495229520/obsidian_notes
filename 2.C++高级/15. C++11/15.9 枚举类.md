# 15.9 enum class 枚举类

---

## 简介

`enum class` 是 C++11 引入的一种**枚举类型**，也被称为**强类型枚举（Strongly-typed Enums）**。它旨在解决传统 C++ [[2.4 枚举与结构体|枚举类型]]的一些固有问题，让枚举变得更安全、更易用。

---

## 语法

`enum class` 的语法与传统 enum 非常相似，只是多了一个 `class` 关键字。

```cpp
enum class 枚举名 {
    成员1,
    成员2,
    成员3
};
```

### 示例

```cpp
enum class Color {
    Red,
    Green,
    Blue
};

enum class Animal {
    Dog,
    Cat,
    Bird
};
```

---

## 作用

`enum class` 解决了传统 enum 的三个主要问题，这些问题也正是它的三大核心作用：

### 优势1：强类型和类型安全

- **作用**：`enum class` 的成员不会隐式转换为整数类型
- **传统 enum 的问题**：传统 enum 的成员可以被当成整数使用，这可能导致意外的类型转换

```cpp
enum Color { Red, Green, Blue };
int c = Red;  // 隐式转换，c 等于 0，没有警告
```

- **enum class 的解决方案**：`enum class` 的成员无法直接赋值给整数类型，这强制你进行显式转换，从而提高类型安全性

```cpp
enum class Color { Red, Green, Blue };
// int c = Color::Red;  // 编译错误！不能隐式转换
int c = static_cast<int>(Color::Red);  // 必须显式转换
```

### 优势2：作用域限定

- **作用**：`enum class` 的成员名称只在其枚举名内部可见
- **传统 enum 的问题**：传统 enum 的成员会"泄露"到其所在的作用域中，如果两个枚举有相同的成员名，会引起命名冲突

```cpp
enum Color { Red, Green, Blue };
enum Animal { Dog, Cat, Red };  // 编译错误！'Red' 重定义
```

- **enum class 的解决方案**：`enum class` 成员必须通过枚举名::成员名的方式来访问，这杜绝了命名冲突

```cpp
enum class Color { Red, Green, Blue };
enum class Animal { Dog, Cat, Red };  // 编译通过，因为它们在不同作用域

// 访问方式必须是：
Color my_color = Color::Red;
Animal my_animal = Animal::Dog;
```

### 优势3：可以指定底层类型

- **作用**：你可以显式地指定 `enum class` 使用的底层整数类型，比如 `char` 或 `unsigned int`
- **传统 enum 的问题**：底层类型由编译器决定，通常是 `int`，这可能导致内存浪费
- **enum class 的解决方案**：

```cpp
// 显式指定底层类型为 char，节省内存
enum class TinyState : char {
    Running,
    Paused,
    Stopped
};
```

---

## 总结

| 特性 | 传统 enum | enum class |
|------|-----------|------------|
| 类型安全 | 隐式转换为整数 | 无隐式转换，必须显式转换 |
| 作用域 | 成员名"泄露"到外部作用域 | 成员名作用域限定在枚举内部 |
| 命名冲突 | 容易发生 | 杜绝命名冲突 |
| 底层类型 | 编译器决定（通常 int） | 可显式指定 |
