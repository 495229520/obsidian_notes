# 15.3 std::atomic 原子操作

---

## 简介

`std::atomic` 是 C++ 标准库中一个非常重要的工具，它用于在多线程环境中安全地操作共享数据。

```cpp
std::atomic<类型> 变量名{初始值};
```

- **std::atomic**：这是类模板的名称
- **<类型>**：你想要进行原子操作的数据类型，比如 int、long、long、bool 等

主要作用是提供**原子性 (Atomicity)**——确保对变量的读写操作是不可分割的。在多线程程序中，当一个线程正在对 atomic 变量进行操作时，其他任何线程都无法操作这个变量。这从根本上解决了数据竞争 (Race Condition) 问题，避免了数据混乱和不可预测的行为。

`std::atomic` 不仅仅支持 ++，还提供了一些特殊的自增/自减操作符，这提供了更精确的原子操作：

- **load()**：原子地读取变量的值
  ```cpp
  int current_value = my_atomic_int.load();
  ```
- **store()**：原子地写入变量的值
  ```cpp
  my_atomic_int.store(100);
  ```
- **exchange()**：原子地用新值替换旧值，并返回旧值
  ```cpp
  int old_value = my_atomic_int.exchange(200);
  ```
- **compare_exchange_weak() / compare_exchange_strong()**：这是最复杂也最强大的原子操作，用于实现"无锁 (lock-free)"编程。它会比较变量的当前值是否等于期望值，如果是，则用新值替换它；否则，不做任何事

---

## 作用

在以下场景下，一个简单的操作，比如 `a++`，在底层是分解为多个机器指令：

1. 从内存中读取 `a` 的值
2. 在寄存器中对该值进行加 1
3. 将新值写回内存

在多线程环境中，如果两个线程同时执行 `a++`，就可能发生以下情况：
- 线程 A 读取 a 的值（例如 5）
- 线程 B 也读取 a 的值（也是 5）
- 线程 A 将 a 加 1 得到 6，并写回内存
- 线程 B 也将 a 加 1 得到 6，并写回内存

最终结果是 a 变成了 6，而不是期望的 7。这就是典型的**竞态条件**导致的错误。

而使用 `std::atomic`，`a++` 会被保证为一个**原子操作**，可以安全地在多个线程中同时执行，不会出现数据混乱的问题。

## 案例

```cpp
#include <iostream>
#include <thread>
#include <vector>
#include <atomic>

// 使用 std::atomic<long long> 来保护共享变量
std::atomic<long long> counter{0};

void increment_counter() {
    for (int i = 0; i < 10000; ++i) {
        counter++;
        // 这个操作是原子的，不会出现竞态条件
    }
}

int main() {
    std::vector<std::thread> threads;
    for (int i = 0; i < 10; ++i) {
        threads.push_back(std::thread(increment_counter));
    }

    for (auto& t : threads) {
        t.join();
    }

    std::cout << "最终的计数器值是：" << counter << std::endl;
    // 结果总是 100000，因为 atomic 操作保证了正确性

    return 0;
}
```

### 代码分析

#### `std::atomic<long long> counter{0};`

- 这行代码声明了一个全局变量 counter
- `std::atomic<long long>` 是关键，它告诉编译器，这个 counter 变量是一个原子变量，这意味着任何对它的操作（比如 `++`）都会被保证是原子的，即它在执行时不会被其他线程中断。将 counter 的初始值设为 0

#### `void increment_counter() { ... }`

- 这个函数定义了每个线程的具体任务：一个循环，循环10000次
- 在循环体内，`counter++` 对全局变量 counter 进行自增操作。因为 counter 是 `std::atomic` 类型，所以 `counter++` 是一个原子操作，可以安全地在多个线程中同时执行，而不会导致数据竞争

#### `std::vector<std::thread> threads;`

- 创建一个 vector，用来存储我们创建的10个线程对象

#### `for (int i = 0; i < 10; ++i) { threads.push_back(std::thread(increment_counter)); }`

- 这是一个循环，循环10次
- 在每次循环中，`std::thread(increment_counter)` 都会创建一个新的线程，并让它去执行 `increment_counter` 函数
- `threads.push_back(...)` 将创建的线程对象存入 vector 中
- **注意**：一旦创建就会开始执行其任务，所以这10个线程几乎同时开始计数

#### `for (auto& t : threads) { t.join(); }`

- 这是另一个循环，遍历 threads 容器中的每一个线程
- `t.join()` 是对每个 threads 容器同循环
- 这个循环确保了主线程会等待所有10个线程都执行完 `increment_counter` 任务

#### `std::cout << "最终的计数器值是：" << counter << std::endl;`

- 当上面那个 for 循环执行完毕后，我们就知道所有10个线程都已经完成了它们的任务
- 此时输出 counter 的最终值
- 因为每个线程循环10000次，一共10个线程，所以最终结果应该是 10 * 10000 = 100000

---

## 关键要点

| 概念 | 说明 |
|------|------|
| `std::atomic` | 原子类型模板，保证操作的原子性 |
| 原子性 | 操作不可分割，要么全部完成，要么完全不执行 |
| 竞态条件 | 多线程同时访问共享数据导致的错误 |
| `load()` | 原子读取 |
| `store()` | 原子写入 |
| `exchange()` | 原子交换，返回旧值 |
