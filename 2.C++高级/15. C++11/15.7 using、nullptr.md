# 15.7 using 类型别名与 nullptr

---

## using 类型别名

### 基本语法

```cpp
using alias = existing_type;
```

`using` 是 C++11 引入的创建[[2.3 类型别名与命名空间|类型别名]]的新方式，比传统的 `typedef` 更加直观和强大。

### 使用示例

#### 简单类型别名

```cpp
// 简单类型别名
using Integer = int;
Integer x = 10;
std::cout << x << std::endl;  // 输出 10
```

#### 指针类型别名

```cpp
// 指针类型别名
using IntPtr = int*;
int a = 5;
IntPtr p = &a;
std::cout << *p << std::endl;  // 输出 5
```

#### 模板类型别名

```cpp
// 模板类型别名
template<typename T>
using Vec = std::vector<T>;
Vec<int> numbers = { 1, 2, 3, 4, 5 };
for (const auto& num : numbers) {
    std::cout << num << " ";
}
std::cout << std::endl;  // 输出 1 2 3 4 5
```

#### 复杂类型别名

```cpp
// 复杂类型别名
using StringMap = std::map<std::string, std::string>;
StringMap myMap = { {"name", "Alice"}, {"age", "30"} };
for (const auto& [key, value] : myMap) {
    std::cout << key << ": " << value << std::endl;
}
```

## nullptr 空指针

`nullptr` 是 C++11 引入的一个关键字，用于表示空指针。它解决了传统 NULL 常量在某些情况下的歧义问题，因为 NULL 通常被定义为整数 0，在重载和模板场景中可能会引发问题。

### 为什么使用 nullptr？

在 C++11 之前，我们使用 `NULL` 或 `0` 来表示空指针：

```cpp
int* p = NULL;  // 或 int* p = 0;
```

问题在于 `NULL` 通常被定义为 `#define NULL 0`，这意味着它本质上是一个整数。在[[9.2.1 函数重载|函数重载]]和模板场景中，这可能导致歧义：

```cpp
void func(int x) { std::cout << "int version" << std::endl; }
void func(int* p) { std::cout << "pointer version" << std::endl; }

func(NULL);     // 可能调用 int 版本！因为 NULL 是 0
func(nullptr);  // 明确调用 pointer 版本
```

### nullptr 的优势

- **类型安全**：`nullptr` 是 `std::nullptr_t` 类型，只能转换为指针类型
- **避免歧义**：在重载和模板中不会被误解为整数
- **更清晰**：代码意图更明确

### 示例

```cpp
int* ptr = nullptr;  // 明确表示空指针

if (ptr == nullptr) {
    std::cout << "ptr 是空指针" << std::endl;
}

// 在函数重载中
void process(int* p) {
    if (p != nullptr) {
        std::cout << "处理指针" << std::endl;
    }
}

process(nullptr);  // 明确调用指针版本
```

## 关键要点

| 特性 | 说明 |
|------|------|
| `using` | C++11 类型别名语法，比 typedef 更直观 |
| 模板别名 | `using` 支持模板别名，typedef 不支持 |
| `nullptr` | C++11 空指针关键字，类型安全 |
| vs `NULL` | nullptr 避免了 NULL 作为整数 0 的歧义 |
