# 15.1 auto 与 范围 for 循环

---

## auto 关键字

### 语法

auto 的语法非常简洁：在声明变量时，用 auto 替换具体的类型名。

```cpp
auto 变量名 = 初始化表达式;
```

### 作用

auto 的作用是让[[9.2.5 auto类型推导|编译器根据初始化表达式自动推断变量的类型]]，你不再需要显式地写出变量的类型，这减少了"代码冗余"，尤其是在类型很长或很复杂的时候。

**优点**：

- **代码简洁**：减少了重复输入长类型名的麻烦，比如在遍历容器时
- **避免错误**：确保变量类型与初始化表达式的类型完全匹配，避免了隐式类型转换可能带来的问题
- **与模板配合**：在泛型编程和模板代码中，auto 尤其有用，因为变量的类型在编译时才能确定

### 案例

```cpp
#include <iostream>
#include <vector>
#include <map>

int main() {
    // 编译器推断 i 为 int 类型
    auto i = 10;

    // 编译器推断 pi 为 int* 类型
    auto pi = &i;

    std::vector<int> nums = {1, 2, 3};
    // 编译器推断 it 为 std::vector<int>::iterator 类型
    auto it = nums.begin();

    std::cout << "i 的类型是 int，值是：" << i << std::endl;

    std::map<std::string, int> scores = {{"Alice", 95}, {"Bob", 88}};
    // 编译器推断 kv 为 std::pair<const std::string, int> 类型
    // 这是一个常量引用的例子，用 auto 极大地简化了代码
    for (auto kv : scores) {
        std::cout << kv.first << " : " << kv.second << std::endl;
    }

    return 0;
}
```

## 范围 for 循环 (Range-based for loop)

### 语法

范围 for 循环的语法是为了遍历容器（如 std::vector、std::list、数组等）而设计的，它比传统的 for 循环更简单、更安全。

```cpp
for (元素类型 变量名 : 容器) {
    // 循环体
}
```

### 作用

auto 的作用是让编译器根据初始化表达式自动推断变量的类型，你不再需要显式写出变量的类型，这减少了"代码冗余"，尤其是在类型很长或很复杂的时候。

**优点**：

- **代码简洁**：在泛型编程和模板代码中，auto 尤其有用，因为变量的类型在编译时才能确定
- **避免错误**：确保变量类型与初始化表达式的类型完全匹配，避免了隐式类型转换可能带来的问题
- **与模板配合**：在泛型编程和模板代码中，auto 尤其有用，因为变量的类型在编译时才能确定

### 示例

#### 1. 按值遍历（拷贝）

```cpp
std::vector<int> numbers = {1, 2, 3};
// 这里的 num 是 numbers 中每个元素的拷贝
for (int num : numbers) {
    // 这里的修改不会影响原始容器
    num += 1;
}
```

这种方式在遍历中会创建元素的拷贝，如果你只是想读取数据，这是可以的，但如果元素类型很大，会产生不必要的性能开销。

#### 2. 按引用遍历（无拷贝）

```cpp
std::vector<int> numbers = {1, 2, 3};
// 这里的 num 是 numbers 中每个元素的引用
for (int& num : numbers) {
    // 这里的修改会直接改变原始容器中的元素
    num += 1;
}
```

#### 3. 结合 auto 使用

auto 和范围 for 循环是天作之合，你可以让编译器自动推断元素的类型。

```cpp
std::vector<std::string> names = {"Alice", "Bob"};
// 按值拷贝，让编译器推断类型
for (auto name : names) {
    std::cout << name << std::endl;
}

// 按引用，让编译器推断类型
for (auto& name : names) {
    name += "!";
}

// 按常量引用，只读不写，最安全高效
for (const auto& name : names) {
    std::cout << name << std::endl;
}
```

> **最佳实践**：`for (const auto& variable : container)` 是范围 for 循环的最佳实践，它在大多数只读遍历的场景下，既能保持代码简洁，又能保证性能和安全。

---

## 关键要点

- **auto**：让编译器自动推导变量类型，减少冗余代码
- **范围 for**：简化容器遍历，避免迭代器操作的复杂性
- **const auto&**：只读遍历的最佳选择，避免不必要的拷贝
