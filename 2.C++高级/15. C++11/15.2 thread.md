# 15.2 std::thread 线程

---

## std::thread：开启新的执行流

`std::thread` 是 C++ 标准库中用于管理线程的类，它允许你在程序中创建并运行新的执行线程，以真实现并发处理。

### 核心概念

- **线程 (Thread)**：线程是操作系统调度的最小执行单元。一个[[1.3 进程的虚拟地址空间|进程]]可以包含多个线程，这些线程共享进程的内存空间，但每个线程有自己独立的栈，程序计数器和寄存器状态
- **并发 (Concurrency)**：指在同一时间处理多个任务的能力。不一定是同时执行，在单核 CPU 上，并发通过时间片轮转实现
- **并行 (Parallelism)**：指在同一时间内同时执行多个任务，通常需要多核 CPU

### 如何使用 std::thread?

创建 `std::thread` 对象时，需要传入一个可调用对象（函数、函数对象、[[15.4 lambda|lambda 表达式]]），这个可调用对象是新线程要执行的任务。

### 案例

```cpp
#include <iostream>
#include <thread>

// 1. 使用普通函数作为线程任务
void task_function(int id) {
    std::cout << "Thread " << id << " is running." << std::endl;
}

// 2. 使用 Lambda 表达式
void task_lambda() {
    std::thread t2([]() {
        std::cout << "This is a lambda thread." << std::endl;
    });
    t2.join();
}

int main() {
    // 创建一个新线程，执行 task_function
    std::thread t1(task_function, 1);

    // 等待线程执行完毕
    t1.join();

    task_lambda();

    std::cout << "Main thread finished." << std::endl;

    return 0;
}
```

## std::thread 的生命周期管理：join() 和 detach()

创建线程后，你必须在 `std::thread` 对象被销毁前，对它进行管理，这个管理主要通过 `join()` 和 `detach()` 来完成。

### join()

`join()` 是最常用的方法，它会阻塞当前的线程，直到被调用的线程执行完毕，以保证了在主线程结束前，所有子线程都已经完成任务。

**线程对象名.join();**

- **作用**：阻塞（暂停）调用 join() 的线程（通常是主线程），直到被调用的线程执行完毕
- **副作用**：无
- **注意**：join() 只能调用一次。调用后，`std::thread` 对象将不再代表任何线程

### my_thread.join();

- 会阻塞主线程，让主线程等待下来。直到 my_thread 所代表的线程执行完毕，只有新线程的任务完成后，主线程才会从 join() 的位置继续往下执行
- 当你在 `std::vector` 或 `std::push_back` 一个大对象时，如果这个元素需要容器内部重新分配内存，所有已有的元素都必须移动或复制到新的内存位置

### detach()

将线程与 `std::thread` 对象分离，让线程在后台独立运行，行为系统会负责清理资源。

**线程对象名.detach();**

- **作用**：将线程与 std::thread 对象分离，让线程在后台独立运行，行为系统负责清理资源
- **副作用**：无
- **注意**：detach() 只能调用一次。分离后的线程会变成一个"守护线程"(daemon thread)

```cpp
#include <thread>
#include <iostream>

void background_task() {
    std::cout << "我在后台独立运行..." << std::endl;
    // ...
}

int main() {
    std::thread my_thread(background_task);

    // 在这里，你可以选择：
    // 1. 等待它完成
    // my_thread.join();

    // 2. 或者让它独立运行
    my_thread.detach();

    // 如果这里主线程结束，后台线程可能还没执行完，操作系统负责清理它
    return 0;
}
```

## 1. 创建线程

std::thread 的构造函数可以接受一个可调用对象作为参数，这个可调用对象是新线程要执行的任务。

```cpp
// 传入函数
std::thread 线程对象名(函数名, 参数1, 参数2, ...);

// 传入 Lambda 表达式
std::thread 线程对象名(
    []{参数列表}{ 任务代码; }, 参数1, 参数2, ...);

// 传入函数对象（仿函数）
MyFunctor functor_obj;
std::thread 线程对象名(functor_obj, 参数1, 参数2, ...);

// 传入成员函数
class MyClass { ... };
MyClass obj;
std::thread 线程对象名(&MyClass::成员函数名, &obj, 参数1, 参数2, ...);
```

```cpp
#include <iostream>
#include <thread>

void task_function(int a, int b) {
    std::cout << "函数任务：参数是 " << a << " 和 " << b << std::endl;
}

int main() {
    // 传入普通函数和参数
    std::thread t1(task_function, 10, 20);

    // 传入 Lambda 表达式，可以捕获外部变量
    int capture_var = 100;
    std::thread t2([&capture_var]() {
        std::cout << "Lambda 任务：捕获的变量是 " << capture_var << std::endl;
    });

    t1.join();
    t2.join();

    return 0;
}
```

## 2. 生命周期管理：join() 和 detach()

创建线程后，你必须在 `std::thread` 对象销毁前决定如何管理这个线程。

### join()

- **作用**：阻塞当前线程，直到被调用线程执行完毕
- **典型用法**：确保子线程完成后再继续主线程

```cpp
std::thread t1(task_function);
t1.join();  // 主线程在此等待 t1 完成
```

### detach()

- **作用**：分离线程，让其在后台独立运行
- **注意**：分离后无法再对该线程进行任何控制

```cpp
std::thread t2(background_task);
t2.detach();  // t2 成为守护线程
```

## 关键要点

| 概念 | 说明 |
|------|------|
| `std::thread` | C++11 标准线程类，用于创建和管理线程 |
| `join()` | 阻塞等待线程完成 |
| `detach()` | 分离线程，让其独立运行 |
| 可调用对象 | 函数、Lambda、仿函数都可以作为线程任务 |
