# 5. new 和 delete

## 概述

`new` 和 `delete` 是 C++ 中用于[[6.2 动态内存分配与释放|动态内存管理]]的运算符，相比 C 语言的 `malloc`/`free`，它们提供了类型安全和自动调用构造/析构函数的能力。

---

## C 的库函数 vs C++ 运算符

### C 的 malloc/free

```cpp
int* p = (int*)malloc(sizeof(int));
if (p == nullptr) {
    return -1;  // malloc 开辟内存失败，通过返回值和 nullptr 做比较
}
*p = 20;
free(p);

int* q = (int*)malloc(sizeof(int) * 20);
if (!q) {
    return -1;
}
free(q);
```

**特点**：
1. `malloc` 开辟内存失败，是通过返回值和 `nullptr` 做比较
2. `malloc` 只根据字节开辟内存，返回的是 `void*`，因此需要进行类型转换

### C++ 的 new/delete

```cpp
int* p1 = new int(20);
delete p1;

// 有没有括号的区别就是是否将数组初始化为零
int* q1 = new int[20]();
delete[] q1;
```

**特点**：
1. `new` 不仅可以做内存开辟，还可以做内存初始化操作
2. `new` 开辟内存失败是通过抛出 `bad_alloc` 类型的异常来判断
3. `new` 开辟内存是指定类型，因此不需要进行类型转换

---

## new 的类型

| 类型 | 语法 | 说明 |
|------|------|------|
| **普通 new** | `int* p1 = new int(20);` | 抛出异常 |
| **nothrow new** | `int* p2 = new (std::nothrow) int;` | 不抛出异常，异常时返回值为空 |
| **const new** | `const int* p3 = new const int(40);` | 在堆上开辟一个常量内存，初始化为 40，因此需要使用一个指向常量指针 |
| **定位 new** | `int* p4 = new (&data) int(50);` | 定位在一个内存上，划分指定类型的空间 |

### 定位 new 示例

```cpp
int data = 0;
// 定位在一个内存上，划分指定类型的空间
int* p4 = new (&data) int(50);
// p4 指向 data 的地址，*p4 == data == 50
```

## operator new 和 operator delete

### 底层实现

```cpp
// 先调用 operator new 开辟内存
// 然后调用对象的构造函数进行初始化
void* operator new(size_t size)
{
    // 底层通过 malloc 开辟内存
    void* p = malloc(size);
    // 判断内存是否开辟失败
    if (p == nullptr)
        throw bad_alloc();
    cout << "operator new addr:" << p << endl;
    return p;
}

// delete p；调用 p 指向对象的析构函数
// 再调用 operator delete 释放内存空间
void operator delete(void* ptr)
{
    cout << "operator delete addr:" << ptr << endl;
    free(ptr);
}

// 数组版本
void* operator new[](size_t size)
{
    void* p = malloc(size);
    if (p == nullptr)
        throw bad_alloc();
    cout << "operator new[] addr:" << p << endl;
    return p;
}

void operator delete[](void* ptr)
{
    cout << "operator delete[] addr:" << ptr << endl;
    free(ptr);
}
```

### 类中重载示例

```cpp
class Test
{
public:
    Test(int data = 10) : ptr(new int(data))
    {
        cout << "Test()" << endl;
    }

    ~Test()
    {
        delete ptr;
        cout << "~Test()" << endl;
    }

private:
    int* ptr;
};

int main()
{
    Test* p1 = new Test();
    delete p1;
}
```

**输出**：
```
operator new addr: 000002IDOOEP04330
Test()
~Test()
operator delete addr: 000002IDOOEP04330
```

## new 和 delete 的配对问题

### 问题：为什么区分单个元素和数组的 new 和 delete？

```cpp
int *p = new int;
delete[]p;       // 错误用法

int *q = new int[10];
delete q;        // 错误用法
```

**正常情况下** `new` 就是 `malloc`，而 `delete` 就是 `free`，释放内存空间没问题。

但是遇到下面的情况就会报错：

```cpp
Test* p1 = new Test[5];
delete p1;       // 错误！
```

### 问题分析

```cpp
Test* p1 = new Test[5];
delete[] p1;     // 正确
```

**内存布局**：

        Obj2["Test[1]"]
        Obj3["Test[2]"]

- 在这里 `new` 了 5 个字节的内存空间 `5 * 4 = 20` 个字节
- 注意 `s2` 指针是 `0x104` 开始，此时 `s1` 变成 `0x104` 等于 `p1 - 4` 的地方

### 为什么会这样？

- 从内存的角度分析：`delete[]` 以为在 `p1` 上面还有一块内存存放对象的个数
- 此时会删除 `p1 - 4` 的内存，导致出现问题
- 从这里默认知道为什么 `new` 和 `delete` 不可以混用了

```cpp
Test* p1 = new Test[5];  // delete 不知道有几个对象
delete p1;               // delete 只释放 p1-4 的内存，导致出现问题
```

## 对象池应用

对象池是一种常见的内存优化技术，通过预分配和重用对象来减少频繁的内存分配开销。

### 问题场景

```cpp
template<typename T>
class Queue
{
public:
    Queue<T>()
    {
        _front = _rear = new QueueItem();
    }

    ~Queue<T>()
    {
        QueueItem* cur = _front;
        while (cur != nullptr)
        {
            delete cur;
            cur = _front;
        }
    }

    void push(const T& val)
    {
        QueueItem* item = new QueueItem(val);
        _rear->_next = item;
        _rear = item;
    }

    void pop()
    {
        if (empty())
            return;
        QueueItem* first = _front->_next;
        _front->_next = first->_next;
        if (_front->_next == nullptr)
        {
            _rear = _front;
        }
        delete first;
    }

private:
    struct QueueItem
    {
        QueueItem(T data = T()) : _data(data), _next(nullptr) {}
        T _data;
        QueueItem* _next;
    };

    QueueItem* _front;  // 指向头结点
    QueueItem* _rear;   // 指向队尾
};

int main()
{
    Queue<int> que;
    for (int i = 0; i < 1000000; ++i)
    {
        que.push(i);
        que.pop();
    }
    // 科普引入：在进行一百万次操作的时候
    // 每次 push 都是 new，每次 pop 都是 delete
    // 频繁的内存分配和释放导致了大量的性能消耗
    cout << que.empty() << endl;
}
```

### 解决方案：自定义 QueueItem 的内存管理

```cpp
struct QueueItem
{
    // 在 new 的时候首先会在内存空间，再调用构造函数数进行值的初始化处理
    QueueItem(T data = T()) : _data(data), _next(nullptr) {}

    // 在 QueueItem 自己定义内存空间管理
    void* operator new(size_t size)
    {
        // 首次调用
        if (_itemPool == nullptr)
        {
            // 一次性向内存中请一块内存空间
            _itemPool = (QueueItem*)new char[POOL_ITEM_SIZE * sizeof(QueueItem)];
            // 将这块内存空间中的 next 指向串成一个空闲链表
            QueueItem* p = _itemPool;
            for (; p < _itemPool + POOL_ITEM_SIZE - 1; ++p)
            {
                // 每个节点的 _next 指向内存中的下一个节点
                p->_next = p + 1;
            }
            // 最后一个节点指向空
            p->_next = nullptr;
        }

        // 从空闲链表的头部取出一个节点
        QueueItem* p = _itemPool;
        _itemPool = _itemPool->_next;
        return p;
    }

    void operator delete(void* ptr)
    {
        QueueItem* p = (QueueItem*)ptr;
        p->_next = _itemPool;
        _itemPool = p;
    }

    T _data;
    QueueItem* _next;
    static QueueItem* _itemPool;
    static const int POOL_ITEM_SIZE = 100000;
};

// 指向头结点
QueueItem* _front;
// 指向队尾
QueueItem* _rear;
```

## 关键要点

1. **new vs malloc**：`new` 会调用构造函数，`malloc` 只分配内存
2. **delete vs free**：`delete` 会调用析构函数，`free` 只释放内存
3. **配对使用**：`new` 配 `delete`，`new[]` 配 `delete[]`
4. **异常处理**：`new` 失败抛出 `bad_alloc`，`nothrow new` 返回 `nullptr`
5. **对象池**：减少频繁内存分配的性能开销
