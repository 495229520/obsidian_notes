# 8. 浅拷贝与深拷贝

## 简介

| 概念 | 说明 |
|------|------|
| **对象默认的拷贝构造是内存的数据拷贝** | `data` 数据原型是对象在存进在的数据拷贝的内存 |
| **关键是对象如果占用外部资源，那么浅拷贝就会出现问题了** | 因为对象进行拷贝会拷贝指针内存值 |
| **不要使用 `memcpy`、`realloc` 等操作** | 看到对象就不不可以使用 |

---

## 内存的问题总结

1. **内存问题是 `data` 数据类型是对象会进行拷贝的内存**
2. **浅拷贝问题是对象外部资源的问题**
3. **如果有指向外部内存的拷贝的对象，那么浅拷贝就会出现问题**
4. **深拷贝的解决方案**
5. 使用 `memcpy` 和 `realloc` 等操作，看到对象就不可以使用
6. 在内存创建和实际过程中应将了大量的性能消耗，此时使用对象池

---

## 浅拷贝

### 问题代码

```cpp
class SeqStack
{
private:
    int* _pstack;
    int _top;
    int _size;

public:
    SeqStack(int size = 10)
    {
        _pstack = new int[size];
        _top = -1;
        _size = size;
    }

    ~SeqStack()
    {
        cout << this << " is ~SeqStack()" << endl;
        delete[] _pstack;
        _pstack = nullptr;
    }
};

int main()
{
    SeqStack s;
    SeqStack s1(s);  // 浅拷贝问题！
    // s 和 s1 的 _pstack 指向同一块内存
    // 析构时会 double free
    return 0;
}
```

### 问题分析

**浅拷贝**只复制指针的值，而不复制指针指向的内容。这会导致：

1. **两个对象共享同一块堆内存**
2. **析构时重复释放**（double free）
3. **一个对象修改影响另一个对象**

### 内存布局

```
浅拷贝后的内存状态：

  s               s1
┌─────────┐    ┌─────────┐
│ _pstack ├────┤ _pstack │  → 指向同一块内存
│ _top    │    │ _top    │
│ _size   │    │ _size   │
└─────────┘    └─────────┘
      │              │
      └──────┬───────┘
             ↓
       ┌──────────┐
       │ 堆内存    │  ← 被两个对象共享
       └──────────┘
```

## 深拷贝

### 解决方案

自定义拷贝构造函数和赋值运算符，**为新对象分配独立的内存空间**。

```cpp
class SeqStack
{
private:
    int* _pstack;
    int _top;
    int _size;

public:
    // 构造函数
    SeqStack(int size = 10)
    {
        cout << this << " is SeqStack()" << endl;
        _pstack = new int[size];
        _top = -1;
        _size = size;
    }

    // 自定义拷贝构造函数：深拷贝
    SeqStack(const SeqStack& src)
        : _top(src._top)
        , _size(src._size)
    {
        // 分配新的内存空间
        _pstack = new int[src._size];
        // 拷贝数据
        for (int i = 0; i < src._top; i++)
        {
            _pstack[i] = src._pstack[i];
        }
    }

    // 自定义赋值重载函数：深拷贝
    SeqStack& operator=(const SeqStack& src)
    {
        // 防止自赋值
        if (this == &src)
            return *this;

        // 需要先释放当前对象占用的外部堆内存
        delete[] _pstack;

        // 进行拷贝构造操作
        _pstack = new int[src._size];
        for (int i = 0; i < src._top; i++)
        {
            _pstack[i] = src._pstack[i];
        }

        _top = src._top;
        _size = src._size;

        return *this;
    }

    // 析构函数
    ~SeqStack()
    {
        cout << this << " is ~SeqStack()" << endl;
        delete[] _pstack;
        _pstack = nullptr;
    }
};
```

### 深拷贝后的内存布局

```
深拷贝后的内存状态：

  s               s1
┌─────────┐    ┌─────────┐
│ _pstack │    │ _pstack │
│ _top    │    │ _top    │
│ _size   │    │ _size   │
└────┬────┘    └────┬────┘
     │              │
     ↓              ↓
┌──────────┐  ┌──────────┐
│ 堆内存1   │  │ 堆内存2   │  ← 各自独立的内存
└──────────┘  └──────────┘
```

## 应用案例

```cpp
int main()
{
    // 没有提供任何构造函数的时候，会为你生成默认构造函数
    SeqStack s;
    // s2 拷贝的是 s1 的内存，s2 指向了 s1 指向的内存
    // 在 s2 析构之后，这块内存被释放了，此时 s1 还是指向这块内存
    // 然后 s1 析构的时候，再次释放这块内存，此时会报错
    // 注意 s2 拷贝的时候，s2 先前的内存会直接丢掉，没有释放
    // 而且这个时候，s2 本来指向的内存内存就没有删除
    // 因此自己实现的拷贝构造函数，需要先释放原来的内存

    SeqStack s1(s);       // 使用拷贝构造函数
    // s1 = s;             // 使用赋值运算符

    // 注意以下两种情况的区别：
    // SeqStack s2;
    // s2 = s1;            // 赋值操作，调用 operator=

    // SeqStack s2 = s1;   // 初始化，调用拷贝构造函数

    return 0;
}
```

---

## 关键要点

1. **浅拷贝**：只复制指针值，共享堆内存，导致 double free
2. **深拷贝**：分配新内存，复制内容，各自独立
3. **赋值运算符**：需要先释放原有资源，再分配新资源
4. **自赋值检查**：`if (this == &src) return *this;`
5. **Rule of Three**：如果定义了析构函数，通常也需要定义拷贝构造函数和赋值运算符
