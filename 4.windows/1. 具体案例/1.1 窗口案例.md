# 1.1 Windows 窗口案例

> 本笔记介绍 Windows 窗口程序的基本结构和创建流程，是学习 Windows GUI 编程的入门案例。

---

## 源码

```cpp
#include <Windows.h>

LPCTSTR szClass = "My";
LPCTSTR szTitle = "Welcome crude";

// 窗口消息处理
LRESULT CALLBACK MyWndProc(
    HWND hwnd,
    UINT uMsg,
    WPARAM wParam,
    LPARAM lParam)
{
    // 1. 逻辑1 响应窗口关 2. 注册窗口类 3. 创建窗口 4. 显示以及更新窗口 5. 循环等待接收
    LRESULT lParam;

    // 注意: 窗口会创建失败, 注册窗口类的时候不能把类名写死
    int szClass;    // WNDCLASS:WndProc,
    /*
     * WNDCLASS: 分类窗口的类型, 目的在于制定消息被分配的地方,
     * 一般有: WS_OVERLAPPEDWINDOW(标准窗口), WS_POPUP(弹出窗口),
     * _T("name"), WNDCLASSEX *lpWindInstance,
     * _T(), LPCTSTR *pszInstance,
     * _T(), LPCTSTR *pfnClass,
     * _T(), BOOL *pfn ShowCold,
     * */

    switch (msg) {
        case WM_CLOSE:
            // 销毁窗口
            DestroyWindow(hwnd);
            break;
        case WM_DESTROY:
            // 投递一个 WM_QUIT 消息
            PostQuitMessage(0);
            break;
        default:
            return DefWindowProc(hwnd, msg, wParam, lParam);
    }

    return 0;
}

// WinMain 入口函数 (使用 __stdcall 调用约定)
int WINAPI WinMain(HINSTANCE hInstance,
                   HINSTANCE hPrevInstance,
                   LPSTR lpCmdLine,
                   int nCmdShow)
{
    // 1. 设计窗口类
    WNDCLASS wc;
    wc.cbClsExtra = 0;
    wc.cbWndExtra = 0;
    wc.hbrBackground = (HBRUSH)GetStockObject(WHITE_BRUSH);
    wc.hCursor = LoadCursor(NULL, IDC_ARROW);
    wc.hIcon = LoadIcon(NULL, IDI_APPLICATION);
    wc.hInstance = hInstance;
    wc.lpfnWndProc = MyWndProc;  // 窗口过程函数 (回调函数)
    wc.lpszClassName = szClass;
    wc.lpszMenuName = NULL;
    wc.style = CS_HREDRAW | CS_VREDRAW;

    // 2. 注册窗口类
    RegisterClass(&wc);

    // 3. 创建窗口
    HWND hwnd = CreateWindow(
        szClass,
        szTitle,
        WS_OVERLAPPEDWINDOW,
        CW_USEDEFAULT, CW_USEDEFAULT,
        CW_USEDEFAULT, CW_USEDEFAULT,
        NULL,
        NULL,
        hInstance,
        NULL);

    // 4. 显示及更新窗口
    ShowWindow(hwnd, nCmdShow);
    UpdateWindow(hwnd);

    // 5. 消息循环
    MSG msg;
    while (GetMessage(&msg, NULL, 0, 0)) {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    }

    return msg.wParam;
}
```

## 程序骨架

Windows 窗口程序的 `WinMain` 函数执行流程：

```
int WinMain(...)
│
├── 1. 设计窗口外观及功能属性（注册、申请专利）
│       RegisterClass(...)
│
├── 2. 生产窗口
│       CreateWindow(...)
│
├── 3. 显示窗口
│       ShowWindow(...)
│
├── 4. 更新窗口
│       UpdateWindow(...)
│
└── 5. 进入消息循环
        while (GetMessage(...)) {
            TranslateMessage(...)  // 消息翻译
            DispatchMessage(...)   // 消息分发
        }
```

> **注意**：`WinMain` 使用 [[9.1.10 函数约定调用#2.2 `__stdcall`|__stdcall 调用约定]]，这是 Win32 API 的标准约定。`WINAPI` 宏实际上就是 `__stdcall`。

## 概念分析

### 窗口 (Window)

窗口就是在显示器屏幕上的一片矩形区域，接收用户的输入，显示程序的输出，可以处理鼠标和键盘的输入、工具栏、对话框。

**C++ 模拟定义**：`Cwnd myWnd; m_wnd`

### 客户区 (Client Area)

- 窗口通常分为**非客户区**和**客户区**
- **客户区**是程序员可以自定义使用的区域，如绘图区域、文本显示区域
- **非客户区**包括标题栏、菜单栏、系统按钮等，由操作系统管理

**C++ 窗口对象**：`Cwnd myWnd; m_wnd`

### 句柄 (Handle)

句柄是一个**标识符**，用于标识一个**特定的对象**。它可以理解为一个对象的"名片"或"门牌号"。

| 类型 | 说明 |
|------|------|
| 实质 | 一个指针类型的数据类型 |
| 作用 | 标识/引用一个内核对象或用户对象 |

- **窗口句柄定义**：`HWND m_wnd`
- **窗口通过句柄标识**
- 句柄使得程序可以访问和操作特定的资源，如窗口、文件、设备上下文等
- 可以理解为[[6.1基本指针|指针]]的"指针"，是间接引用的机制

> **关键点**：
> 1. 窗口类数据的定义 `Cwnd myWnd; m_wnd`
> 2. 窗口句柄的定义 `HWND m_wnd`

### 窗口过程函数 (Window Procedure)

窗口过程函数是一个[[9.1.5 函数指针#回调函数 (Callback Functions)|回调函数]]，用于处理发送到窗口的消息。当用户与窗口交互（如点击、键盘输入）或系统需要通知窗口时，系统会调用此函数。

```cpp
LRESULT CALLBACK WndProc(HWND hwnd, UINT msg, WPARAM wParam, LPARAM lParam);
```

- `CALLBACK` 宏等同于 `__stdcall`
- 必须按照特定的函数签名定义
- 由操作系统调用，而非程序员直接调用

## 关键要点

1. **程序结构**：Windows 窗口程序遵循"注册窗口类 → 创建窗口 → 显示窗口 → 消息循环"的固定模式
2. **消息驱动**：Windows 程序是事件驱动的，通过消息循环处理各种事件
3. **回调机制**：窗口过程函数是由系统调用的[[9.1.5 函数指针#回调函数 (Callback Functions)|回调函数]]
4. **调用约定**：Win32 API 使用 [[9.1.10 函数约定调用#2.2 `__stdcall`|__stdcall 调用约定]]
5. **句柄管理**：通过各种句柄（HWND、HINSTANCE 等）来标识和操作系统资源
