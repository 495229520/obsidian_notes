# 互斥量 (Mutex)

## 定义和作用

- **定义**：互斥对象是一个**内核对象**，在任何时刻只允许一个线程拥有
- **作用**：它用于实现互斥访问共享资源的[[3.3 线程同步|互斥锁]] (Mutual Exclusion)，从而保护临界区 (Critical Section) 代码的原子性，让线程能够安全访问此共享资源
- **特点**：一个线程通过等待 (Wait) 操作试图获取互斥量，则成为它的拥有者

---

## 特性

| 特性 | 说明 |
|------|------|
| **所有权 (Ownership)** | 互斥量是信号灯对象的变种之一，已拥有 (Owned) 或未拥有 (Not Owned)。一个线程通过等待 (Wait) 操作试图获取互斥量，则成为它的拥有者 |
| **信号状态 (Signaled State)** | - **已激活 (Signaled)**：当互斥量未被任何线程拥有时，它处于信号状态可以被获取<br>- **未激活 (Non-signaled)**：当互斥量已经被某线程获取后，它会处于非信号状态 |
| **独占资源访问** | Mutex 是一个内核对象，可进程名称创建，因此可用于不同程序之间的同步 |

---

## 核心函数

| 函数 | 描述 |
|------|------|
| `CreateMutex` | 创建一个新的互斥对象（或打开一个已存在的命名互斥量） |
| `OpenMutex` | 打开一个已存在的命名互斥对象 |
| `WaitForSingleObject` | 等待（获或获取）互斥量。如果互斥量未被锁定时，调用此函数时会立即成功并返回 |
| `ReleaseMutex` | 释放互斥对象的所有权，这将使互斥对象进入已激活状态，允许等待的其他线程获取互斥量 |

---

## 函数详解

### 1. 创建互斥量

```cpp
hMutex = CreateMutex(NULL, FALSE, NULL);
```

- **函数**：`CreateMutex`
- **关键参数解析**：
  - `lpMutexAttributes (NULL)`：安全属性，NULL 表示使用默认安全设置，句柄不可继承
  - `bInitialOwner (FALSE)`：初始拥有者标志，FALSE 表示在创建时，互斥量未被拥有（处于"激活"状态），任何线程可以获取它
  - `lpName (NULL)`：互斥量的名称，NULL 表示这是一个**匿名互斥量**（只在当前进程有效）

### 2. 线程中对互斥量的操作

#### A. 加锁 (等待)

```cpp
WaitForSingleObject(hMutex, INFINITE);
```

- **函数**：`WaitForSingleObject`
- **关键参数解析**：
  - `hHandle (hMutex)`：即等待获取的对象（这里是互斥量）的句柄
  - `dwMilliseconds (INFINITE)`：等待超时时间，INFINITE 表示无限等待，直到互斥量可用
  - 当一个线程使用 `WaitForSingleObject` 获得互斥量时，其他调用 `WaitForSingleObject` 的线程将阻塞等待
- **注意**：如果互斥量已被其他线程拥有且未被释放 `ReleaseMutex` 和 `ReleaseNultes` 释放或被互斥量，则其他线程在等待。

#### B. 解锁 (释放)

```cpp
ReleaseMutex(hMutex);
```

- **函数**：`ReleaseMutex`
- **关键参数解析**：
  - `hMutex`：即待释放的互斥量对象
  - 只有拥有互斥量的线程才能调用 `ReleaseMutex`，直到对所有者调用 `ReleaseMutex` 后，互斥量才会变成已激活正在等待的互斥量，触发结果是操作完成地释放互斥量

#### C. 销毁 (关闭)

```cpp
CloseHandle(hMutex);
```

- **函数**：`CloseHandle`
- **关键参数解析**：
  - `hMutex`：即待关闭的互斥量对象的句柄
  - 当互斥量的引用计数降到 0 时，系统会销毁该互斥量

## 示例代码

以下示例展示了如何使用互斥量保护共享变量 `num`：

```cpp
#include <stdio.h>
#include <windows.h>
#include <process.h>

#define NUM_THREAD 50

unsigned WINAPI threadInc(void* arg);
unsigned WINAPI threadDes(void* arg);

long long num = 0;
HANDLE hMutex;

int main(int argc, char* argv[])
{
    HANDLE tHandles[NUM_THREAD];
    int i;

    printf("sizeof long long: %d \n", sizeof(long long));

    // 创建一个互斥量
    hMutex = CreateMutex(NULL, FALSE, NULL);

    for (i = 0; i < NUM_THREAD; i++)
    {
        if (i % 2)
            tHandles[i] = (HANDLE)_beginthreadex(NULL, 0, threadInc, NULL, 0, NULL);
        else
            tHandles[i] = (HANDLE)_beginthreadex(NULL, 0, threadDes, NULL, 0, NULL);
    }

    WaitForMultipleObjects(NUM_THREAD, tHandles, TRUE, INFINITE);

    // 销毁锁
    CloseHandle(hMutex);
    printf("result: %lld \n", num);
    return 0;
}

unsigned WINAPI threadInc(void* arg)
{
    int i;
    // 加锁
    WaitForSingleObject(hMutex, INFINITE);
    for (i = 0; i < 500000; i++)
        num += 1;
    // 解锁
    ReleaseMutex(hMutex);
    return 0;
}

unsigned WINAPI threadDes(void* arg)
{
    int i;
    // 等待解锁才能继续运行
    WaitForSingleObject(hMutex, INFINITE);
    for (i = 0; i < 500000; i++)
        num -= 1;
    ReleaseMutex(hMutex);
    return 0;
}
```

---

## 代码分析

### 预期结果验证

如果程序正确同步运行，由于有 25 个线程执行递增、25 个线程执行递减，每个线程循环 500,000 次，理论上的结果应该是：

$$
\text{Final num} = (25 \times 500,000) - (25 \times 500,000) = 0
$$

### 代码说明

1. **创建互斥量**：`CreateMutex(NULL, FALSE, NULL)` 创建一个匿名、未被拥有的互斥量

2. **线程同步流程**：
   - 在 `threadInc` 和 `threadDes` 函数中，线程首先调用 `WaitForSingleObject` 获取互斥量
   - 如果互斥量已被其他线程持有，当前线程会阻塞等待
   - 只有获取互斥量的线程才能执行临界区代码（对 `num` 的操作）
   - 完成后调用 `ReleaseMutex` 释放互斥量，允许其他等待的线程获取

3. **等待所有线程完成**：`WaitForMultipleObjects` 等待所有 50 个线程执行完毕

## 互斥量 vs 临界区

| 特性 | 互斥量 (Mutex) | [[3.8 临界区\|临界区 (Critical Section)]] |
|------|---------------|------------------------------------------|
| 作用范围 | 跨进程同步 | 仅限同一进程内 |
| 性能 | 较慢（涉及内核态切换） | 较快（用户态操作） |
| 命名 | 支持命名，可跨进程共享 | 不支持命名 |
| 适用场景 | 进程间同步、需要超时机制 | 同一进程内的高性能同步 |

---

## 注意事项

1. **避免死锁**：确保每次 `WaitForSingleObject` 后都有对应的 `ReleaseMutex`，参见[[3.10 线程死锁]]
2. **只有拥有者能释放**：只有成功获取互斥量的线程才能调用 `ReleaseMutex`
3. **资源释放**：程序结束前应调用 `CloseHandle` 关闭互斥量句柄
