# 3.10 线程死锁 (Deadlock)

## 核心概念

死锁的本质是 **"互相等待，永不放手"**。

想象一下：
- **线程 A** 拥有了资源 1，正在等待资源 2
- **线程 B** 拥有了资源 2，正在等待资源 1

这两个线程都无法继续，也无法释放自己已占有的资源，因此它们会永远等待下去，程序表现为挂起。

---

## 四个必要条件

死锁的发生必须同时满足以下四个条件（缺一不可）：

| 条件名称 | 描述 | 示例 |
|----------|------|------|
| **互斥条件** | 资源是非共享的，一次只能被一个线程独占使用（例如 [[3.4 互斥量\|Mutex]] 锁） | 线程 A 正在使用 Mutex 1，线程 B 无法同时使用 Mutex 1 |
| **占有且等待** | 线程已经占有至少一个资源，并且正在等待获取另一个被其他线程占有的资源 | 线程 A 锁住了 Mutex 1，同时阻塞等待 Mutex 2 |
| **不可剥夺条件** | 资源只能由占有它的线程自愿释放，不能被其他线程或操作系统强制剥夺 | 线程 B 不能强制拿走线程 A 已经锁住的 Mutex 1 |
| **循环等待条件** | 存在一个线程资源的循环链，链中的每个线程都在等待下一个线程所占有的资源（这是最关键的条件） | A 等 B 占有的资源，B 等 C 占有的资源，...，Z 等 A 占有的资源 |

---

## 预防死锁

要预防死锁，只需要破坏四个必要条件中的**任意一个**即可。最实用且最容易实现的方法是破坏**循环等待条件**：

### 1. 资源有序分配（破坏循环等待）

- **策略**：为所有共享资源（锁）定义一个全局的、固定的**获取顺序**
  - 例如：总是先锁 Mutex A，再锁 Mutex B
- **修正后的代码**：无论线程 A 还是线程 B，都必须按 A → B 的顺序加锁

```cpp
// 正确：所有线程都按相同顺序获取锁
void thread_function() {
    WaitForSingleObject(hMutexA, INFINITE);  // 先获取 A
    WaitForSingleObject(hMutexB, INFINITE);  // 再获取 B

    // 临界区代码...

    ReleaseMutex(hMutexB);
    ReleaseMutex(hMutexA);
}
```

### 2. 超时尝试加锁（破坏占有且等待）

- **策略**：使用带有超时的锁函数
  - Windows：`WaitForSingleObject` 带有限时间
  - POSIX：`pthread_mutex_trylock`
- **操作**：如果线程无法立即获取锁，它应该释放自己已占有的所有锁，然后等待一段时间后再重新尝试获取所有需要的锁

```cpp
// 尝试获取锁，超时后放弃并重试
DWORD result = WaitForSingleObject(hMutexB, 1000);  // 等待1秒
if (result == WAIT_TIMEOUT) {
    // 获取失败，释放已持有的锁
    ReleaseMutex(hMutexA);
    // 稍后重试...
}
```

### 3. 使用高级 RAII 锁

- **C++ 推荐**：在 C++ 中，使用 `std::lock()` 配合 `std::scoped_lock` 或 `std::unique_lock` 可以原子性地获取多个锁，防止在获取第二个锁失败时造成占有且等待的僵局

```cpp
#include <mutex>

std::mutex mutexA, mutexB;

void safe_function() {
    // std::lock 原子性地获取两个锁，避免死锁
    std::lock(mutexA, mutexB);

    // 使用 adopt_lock 表示锁已被获取
    std::lock_guard<std::mutex> lockA(mutexA, std::adopt_lock);
    std::lock_guard<std::mutex> lockB(mutexB, std::adopt_lock);

    // 临界区代码...
}

// C++17 更简洁的写法
void safe_function_cpp17() {
    std::scoped_lock lock(mutexA, mutexB);  // 自动原子获取
    // 临界区代码...
}
```

## 死锁检测

除了预防死锁，还可以通过检测来发现死锁：

| 方法 | 描述 |
|------|------|
| **超时检测** | 设置合理的等待超时，如果超时则认为可能发生死锁 |
| **资源分配图** | 维护线程-资源的分配图，检测是否存在环 |
| **调试工具** | 使用调试器或专用工具（如 Visual Studio 的并发可视化工具）分析死锁 |

---

## 关键要点

1. **死锁 = 互相等待 + 永不放手**：多个线程各自持有资源，同时等待对方释放
2. **四个必要条件**：互斥、占有且等待、不可剥夺、循环等待
3. **最佳预防策略**：资源有序分配（固定加锁顺序）
4. **C++ 最佳实践**：使用 `std::scoped_lock` 或 `std::lock()` 原子获取多个锁
5. **注意配对**：确保每个 `WaitForSingleObject` 都有对应的 `ReleaseMutex`
