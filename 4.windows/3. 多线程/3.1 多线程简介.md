# 多线程简介

## 1. 进程 (Process)

**定义**：进程是正在运行的程序的一个实例。

**特点**：
- 系统进行资源分配和调度的一个独立单位（在引入线程的操作系统中，进程是资源分配的基本单位，线程是调度的基本单位）
- 每个进程都有自己的[[1.3 进程的虚拟地址空间|地址空间]]

**缺点**：
- 进程在执行的过程中如果阻塞，整个进程就会被挂起，即使进程中有些工作不依赖于等待的资源，仍然不会执行
- 多个进程同时执行时，使用进程间通信（IPC）比较复杂，效率也不是很高

## 2. 线程 (Thread)

**定义**：线程是操作系统能够进行运算调度的最小单位。

**特点**：
- 线程被包含在进程之中，是进程的实际运作单位
- 一个进程可以有多个线程，每个线程并发执行
- 线程有就绪、运行、阻塞三种基本状态

**线程的优势**：
1. 线程比进程更轻量级，创建和销毁的开销更小
2. 同一进程内的线程共享内存空间，通信更方便
3. 上下文切换比进程切换更快

## 创建线程

### `_beginthreadex` (推荐)

`_beginthreadex` 是 Microsoft Visual C++ 运行时库 (CRT) 提供的一个函数，用于在 Windows 环境中创建新的线程。

```cpp
_ACRTIMP uintptr_t __cdecl _beginthreadex(
    _In_opt_  void*                    _Security,
    _In_      unsigned                 _StackSize,
    _In_      _beginthreadex_proc_type _StartAddress,
    _In_opt_  void*                    _ArgList,
    _In_      unsigned                 _InitFlag,
    _Out_opt_ unsigned*                _ThrdAddr
);
```

| 参数            | 方向/类型          | 描述                                                                                                                         |
| --------------- | ------------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| `_Security`     | `_In_opt_` 输入    | 指向 `SECURITY_ATTRIBUTES` 结构的指针，决定返回的句柄是否可以被子进程继承。如果为 NULL，则句柄不能被继承。                   |
| `_StackSize`    | `_In_` 输入        | 新线程栈的大小（以字节为单位）。如果为 0，则使用默认大小（通常为 1MB）。                                                     |
| `_StartAddress` | `_In_` 输入        | 新线程要执行的函数的起始地址（即线程函数）。函数签名必须是 `unsigned __stdcall function_name(void* args)`                    |
| `_ArgList`      | `_In_opt_` 输入    | 传递给线程函数的参数。如果不需要传递参数，可以为 NULL。                                                                      |
| `_InitFlag`     | `_In_` 输入        | 控制新线程初始状态的标志。可以是 0（立即运行）或 `CREATE_SUSPENDED`（挂起状态，需要调用 `ResumeThread` 启动）                |
| `_ThrdAddr`     | `_Out_opt_` 输出   | 指向一个变量的指针，该变量将接收新线程的标识符（Thread ID）。如果不需要此信息，可以为 NULL。                                 |

**返回值**：
- 成功时返回新线程的句柄（转换为 `uintptr_t` 类型）
- 失败时返回 0，并设置 `errno`

**示例代码**：

```cpp
#include <stdio.h>
#include <process.h>
#include <windows.h>

unsigned WINAPI Thread_main_beginThread(void* arg)
{
    int cnt = *(int*)arg;
    for (int i = 0; i < cnt; i++)
    {
        printf("cnt=i\n");
    }
    return 0;
}

unsigned WINAPI Thread_main_beginThread2(void* arg)
{
    int cnt = *(int*)arg;
    for (int i = 0; i < cnt; i++)
    {
        printf("cnt=i\n");
        Sleep(1000);
    }
    return 0;
}

unsigned WINAPI Thread_main_beginThread3(void* arg)
{
    int cnt = *(int*)arg;
    for (int i = 0; i < cnt; i++)
    {
        printf("cnt=i\n");
    }
    return 0;
}

int main()
{
    int counting = 20, timing = 30, looping = 50;

    unsigned int long_id, ming_id, loop_id;
    HANDLE hThread_0 = (HANDLE)_beginthreadex(NULL, 0, Thread_main_beginThread, &counting, 0, &long_id);
    HANDLE hThread_1 = (HANDLE)_beginthreadex(NULL, 0, Thread_main_beginThread2, &timing, 0, &ming_id);
    HANDLE hThread_2 = (HANDLE)_beginthreadex(NULL, 0, Thread_main_beginThread3, &looping, 0, &loop_id);

    // 等待线程结束
    WaitForSingleObject(hThread_0, INFINITE);
    WaitForSingleObject(hThread_1, INFINITE);
    WaitForSingleObject(hThread_2, INFINITE);

    return 0;
}
```

### `CreateThread` (Windows API)

`CreateThread` 是 Windows API (Application Programming Interface)，用于在 Windows 操作系统上创建新的执行线程。

与 `_beginthreadex` 不同的是，`CreateThread` 是操作系统提供的原生函数。

```cpp
WINBASEAPI
_Ret_maybenull_
HANDLE
WINAPI
CreateThread(
    _In_opt_  LPSECURITY_ATTRIBUTES  lpThreadAttributes,
    _In_      SIZE_T                 dwStackSize,
    _In_      LPTHREAD_START_ROUTINE lpStartAddress,
    _In_opt_  __drv_aliasesMem LPVOID lpParameter,
    _In_      DWORD                  dwCreationFlags,
    _Out_opt_ LPDWORD                lpThreadId
);
```

| 参数                 | 参数/类型              | 描述                                                                                                              |
| -------------------- | ---------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `lpThreadAttributes` | `LPSECURITY_ATTRIBUTES` | 指向安全属性结构的指针。如果为 NULL，句柄不能被子进程继承。                                                       |
| `dwStackSize`        | `SIZE_T`               | 线程栈的初始大小（以字节为单位）。如果为 0，使用默认大小。                                                        |
| `lpStartAddress`     | `LPTHREAD_START_ROUTINE` | 指向线程函数的指针。函数签名为 `DWORD WINAPI ThreadProc(LPVOID lpParameter)`                                     |
| `lpParameter`        | `LPVOID`               | 传递给线程函数的参数。                                                                                            |
| `dwCreationFlags`    | `DWORD`                | 控制线程创建的标志。0 表示立即运行，`CREATE_SUSPENDED` 表示挂起状态。                                             |
| `lpThreadId`         | `LPDWORD`              | 接收线程 ID 的变量指针。如果为 NULL，则不返回线程 ID。                                                            |

**返回值**：
- 成功时返回新线程的句柄
- 失败时返回 NULL

**示例代码**：

```cpp
#include <stdio.h>
#include <windows.h>

DWORD WINAPI ThreadFunc(LPVOID lpParam)
{
    int cnt = *(int*)lpParam;
    for (int i = 0; i < cnt; i++)
    {
        printf("loop=i\n");
    }
    return 0;
}

int main()
{
    printf("Main begin:\n");
    DWORD hThread;

    int nData = 10;
    HANDLE h = CreateThread(NULL, 0, ThreadFunc, &nData, 0, &hThread);
    printf("线程已创建, TID = %d\n", GetCurrentThreadId());

    WaitForSingleObject(h, INFINITE);
    CloseHandle(h);

    // 这里主线程等待回车，对应子线程先运行并结束
    getchar();

    return 0;
}
```

## `_beginthreadex` vs `CreateThread` 对比

| 特性             | `_beginthreadex`                                | `CreateThread`                          |
| ---------------- | ----------------------------------------------- | --------------------------------------- |
| 所属             | C 运行时库 (CRT)                                | Windows API                             |
| CRT 安全         | 是，会初始化 CRT 数据结构                       | 否，可能导致 CRT 函数内存泄漏           |
| 推荐使用场景     | 使用 C/C++ 标准库函数的程序                     | 纯 Windows API 程序                     |
| 资源清理         | 使用 `_endthreadex` 或让函数返回                | 使用 `CloseHandle`                      |

**建议**：在 C/C++ 程序中优先使用 `_beginthreadex`，因为它会正确初始化 CRT 的线程本地存储，避免内存泄漏。

## 总结

| 概念   | 描述                                                       |
| ------ | ---------------------------------------------------------- |
| 进程   | 资源分配的基本单位，拥有独立的地址空间                     |
| 线程   | CPU 调度的基本单位，同进程内线程共享地址空间               |
| 句柄   | 用于引用内核对象的不透明值                                 |
| 线程函数 | 线程执行的入口点，必须符合特定的函数签名                 |
