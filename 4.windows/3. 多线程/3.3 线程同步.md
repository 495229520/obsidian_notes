# 线程同步

## 简介

1. **存在位置**：它们存储在 Windows 内核空间内存中，因此只能通过内核提供的 API 函数（例如 `CreateMutex`、`CreateEvent`、`CloseHandle` 等）来访问和操作。

2. **作用**：它们代表了操作系统中的抽象或物理资源，用于执行以下核心任务：
   - **同步 (Synchronization)**：协调多个线程或进程的活动，防止竞态条件（如互斥量、事件、信号量）。
   - **资源管理 (Resource Management)**：代表系统资源（如文件、线程、进程、内存映射）。

---

## 特性

| 特性       | 解释                                                                                                                                                                                                                                                                                         |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 引用计数   | 每个内核对象都有一个引用计数。当一个程序通过 `Create...` 函数获得一个句柄时，计数增加；当程序调用 `CloseHandle` 关闭句柄时，计数减少。只有当引用计数降到零时，内核才会真正销毁该对象并释放其内存。                                                                                            |
| 句柄       | 应用程序不能直接访问内核对象。它们通过一个名为**句柄**的不透明值来引用对象。句柄是进程私有的，一个进程的句柄对另一个进程来说没有意义。Windows 提供了一组函数进行操作内核对象。成功调用一个创建内核对象的函数后，会返回一个句柄，它表示了所创建的内核对象，可由进程中的任何线程使用。在 32 位进程中，句柄是一个 32 位值。在 64 位进程中句柄是一个 64 位值。我们可以使用唯一标识内核对象的句柄，调用内核操作函数对内核对象进行操作。 |
| 资源持久性 | 内核对象通常在创建它们的进程结束后仍能存在，直到所有引用都关闭（引用计数归零）。例如，一个进程创建的互斥量，可以被另一个进程通过其名称打开和使用。                                                                                                                                             |

## 常见的内核对象

| 对象类型            | 代表资源             | 主要用途                       |
| ------------------- | -------------------- | ------------------------------ |
| 进程 (Process)      | 正在运行的程序实例   | 资源隔离和管理                 |
| 线程 (Thread)       | 进程内的执行流       | 并发执行                       |
| 文件 (File)         | 硬盘或网络上的数据流 | I/O 操作                       |
| [[3.4 互斥量\|互斥量 (Mutex)]]      | 独占性锁             | 进程或线程间的互斥访问         |

| [[3.7 信号量\|信号量 (Semaphore)]]  | 计数器锁             | 限制同时访问共享资源的线程/进程数量 |

## 注意事项

1. **判断是否为内核对象**：一个对象是不是内核对象，通常可以看创建此对象 API 的参数中是否需要 `PSECURITY_ATTRIBUTES` 类型的参数。

2. **内核对象的内存结构**：内核对象只是一个内存块，这块内存位于操作系统内核的地址空间。内存块中存放一个**数据结构**（此数据结构的成员有如：安全描述符、使用计数等）。

3. **句柄表**：每个进程中有一个句柄表（handle table），这个句柄表仅供内核对象使用，如下图：

| 索引 | 指向内核对象内存块的指针 | 访问掩码(包含标志位的一个 DWORD) | 标志     |
| ---- | ------------------------ | -------------------------------- | -------- |
| 1    | 0x????????               | 0x????????                       | 0x???????? |
| 2    | 0x????????               | 0x????????                       | 0x???????? |

4. **调用方式**：
   ```cpp
   hThread = CreateThread(..., &threadId);
   ```

   - **内核对象 (Kernel Object)**：这是真正的资源，它是操作系统内核空间中的一块内存，包含了线程的状态、ID、优先级等信息。
   - **句柄 (Handle)**：这是您在用户程序中用来访问该对象的"钥匙"或"门票"。`hThread` 变量就是句柄。

   `CreateThread` 成功后，对应的线程内核对象的引用计数为 2：
   - **引用 1**："内核对象被创建，其引用计数初始为 1"
     - 含义：这代表操作系统内核（特别是调度器）对这个线程的引用。
     - 目的：只要一个线程还在运行，内核就必须持有一个对该线程内核对象的引用，以便管理和调度它。当线程函数执行完毕（即线程退出）时，内核才会释放这个引用。
   - **引用 2**："创建线程的函数打开（访问）了此对象，所以...引用计数加 1"
     - 含义：这代表 `CreateThread` 函数返回给您的那个 HANDLE（句柄）。
     - 目的：操作系统在进程的句柄表中创建了一个条目，指向这个内核对象，并将这个句柄（`hThread`）返回给您的程序。这个句柄是您用来控制（如 `WaitForSingleObject`）或查询这个线程的工具。

   **总结 (CreateThread)**：
   1. 内核创建线程对象（内存块），此时系统引用使其计数为 1。
   2. 内核为调用进程创建句柄 `hThread`，句柄引用使其计数变为 2。
   3. 函数返回 `hThread`。

5. **关闭**

   `CloseHandle(hThread)` 发生了什么？

   当调用 `CloseHandle(hThread)` 时发生这样的事情：系统通过 `hThread` 计算出此句柄在句柄表中的索引，然后把那一项处理后标主为空闲可用的项（**内核对象的引用计数减 1**，即此时此内核对象的引用计数为 1）

   - **句柄表操作**：`CloseHandle` 告诉系统："我（这个进程）不再需要用 `hThread` 这个句柄了。"系统会找到进程句柄表中 `hThread` 对应的条目，并将其销毁（标记为空闲）。
   - **引用计数变化**：销毁句柄的同时，内核会去递减该句柄所指向的内核对象的引用计数。
     - 计数从 2（系统引用 + 句柄引用）
     - 变为 1（仅剩系统引用）
   - **内核对象的状态**：此时，内核对象仍然存活，因为它还在被系统引用（引用计数为 1），所以线程会继续运行。
   - **当程序结束，线程被释放，计数归零时才会真正归零**

## 关键概念补充

### 竞态条件 (Race Condition)

当多个线程同时访问共享资源，且最终结果取决于线程执行的具体时序时，就会发生竞态条件。这是多线程编程中最常见的问题之一。

### [[3.8 临界区|临界区 (Critical Section)]]

临界区是访问共享资源的代码段。在任意时刻，只能有一个线程执行临界区代码，以防止数据竞争。Windows 提供了 [[3.9 关键代码段|CRITICAL_SECTION]] 作为轻量级的临界区保护机制。

### [[3.10 线程死锁|死锁 (Deadlock)]]

当两个或多个线程相互等待对方释放资源时，就会发生[[3.10 线程死锁|死锁]]。例如：
- 线程 A 持有资源 1，等待资源 2
- 线程 B 持有资源 2，等待资源 1

避免死锁的策略（详见[[3.10 线程死锁#预防死锁|预防死锁]]）：
1. 按固定顺序获取锁
2. 使用超时机制
3. 避免嵌套锁
