# 3.7 信号量 (Semaphore)

## 基本概念

信号量维护一个**内部的计数器**（通常称为资源的可用数量）。

| 操作 | 描述 |
|------|------|
| **初始化** | 在创建时，信号量被赋予一个初期计数（例如 5），表示最多允许 5 个线程同时访问某共享资源 |
| **获取资源 (P/Wait 操作)** | 当一个线程想要访问资源时，它必须请求"获取"该信号量：<br/>- 如果计数器 > 0，计数器减 1，线程继续执行<br/>- 如果计数器 = 0，表示所有资源都被占用，线程阻塞，直到计数器 > 0 |
| **释放资源 (V/Signal 操作)** | 当一个线程使用完资源后，它必须释放（递增）信号量：<br/>- 计数器 +1<br/>- 如果此时有其他线程正在等待，会从等待队列中唤醒一个等待线程 |
| **属于[[3.3 线程同步\|内核对象]]** | 信号量存储在内核空间中 |

---

## 核心特性

| 特性 | 描述 |
|------|------|
| **计数机制** | 信号量包含一个计数器，使得可以控制多个线程同时访问同一资源，而不限于互斥量的"零或一"的访问 |
| **资源预留** | 信号量用于限制并发访问的资源，而不是保证独占。当一个信号量初始化为 5，则最多允许 5 个线程同时运行代码段 |
| **跨用性** | 信号量是非常普遍的，在任何操作系统都可以跟踪使用信号量（减少计数器），原理它不是上一个获取（减少计数）的线程 |
| **用途广泛** | 信号量可以用于资源池管理（如数据库连接池、线程池）和让进程同步，作为一种更强大的同步工具 |

## 关键函数

### 创建信号量

```cpp
HANDLE WINAPI
CreateSemaphore(
    _In_opt_ LPSECURITY_ATTRIBUTES lpSemaphoreAttributes, // NULL 安全属性
    _In_     LONG lInitialCount,    // 初始化计数，共有多少个资源可以用。0: 未触发状态
                                    // （无信号状态），表示没有可用资源
    _In_     LONG lMaximumCount,    // 能够处理的最大的资源数量
    _In_opt_ LPCWSTR lpName         // NULL 信号量的名称
);
// semTwo = CreateSemaphore(NULL, 1, 1, NULL);
// semTwo 有可用资源，有信号状态，有信号
```

### 增加信号量

```cpp
WINAPI
ReleaseSemaphore(
    _In_      HANDLE hSemaphore,    // 信号量的句柄
    _In_      LONG lReleaseCount,   // 将 lReleaseCount 值加到信号量的当前资源计数上。> 0，
                                    // 如果加上后资源数超过最大值，返回 FALSE 并设置错误
    _Out_opt_ LPLONG lpPreviousCount // 当前资源计数前的原始值
);
```

### 关闭句柄

```cpp
CloseHandle(
    _In_ _Post_ptr_invalid_ HANDLE hObject
);
```

## 案例：生产者-消费者模式

```cpp
#include <stdio.h>
#include <windows.h>
#include <process.h>

unsigned WINAPI Read(void* arg);
unsigned WINAPI Accu(void* arg);

static HANDLE semOne;
static HANDLE semTwo;
static int num;

int main(int argc, char* argv[])
{
    HANDLE hThread1, hThread2;

    semOne = CreateSemaphore(NULL, 0, 1, NULL);  // semOne 初始无资源（未触发）
    semTwo = CreateSemaphore(NULL, 1, 1, NULL);  // semTwo 初始有资源（已触发）

    hThread1 = (HANDLE)_beginthreadex(NULL, 0, Read, NULL, 0, NULL);
    hThread2 = (HANDLE)_beginthreadex(NULL, 0, Accu, NULL, 0, NULL);

    WaitForSingleObject(hThread1, INFINITE);
    WaitForSingleObject(hThread2, INFINITE);

    CloseHandle(semOne);
    CloseHandle(semTwo);
    system("pause");
    return 0;
}

// Read 线程：负责读取用户输入
unsigned WINAPI Read(void* arg)
{
    int i;
    for (i = 0; i < 5; i++)
    {
        fputs("Input num: ", stdout);  // 1  5  11
        printf("begin read\n");        // 3  6  12

        // 等待内核对象 semTwo 的信号。如果有信号，继续执行；如果没有信号，等待
        WaitForSingleObject(semTwo, INFINITE);
        printf("beginning read\n");    // 4  10 16
        scanf("%d", &num);
        ReleaseSemaphore(semOne, 1, NULL);  // 释放 semOne，计数变为 1
    }
    return 0;
}

// Accu 线程：负责累加计算
unsigned WINAPI Accu(void* arg)
{
    int sum = 0, i;
    for (i = 0; i < 5; i++)
    {
        printf("begin Accu\n");        // 2  9  15

        // 等待内核对象 semOne 的信号。如果有信号，继续执行；如果没有信号，等待
        WaitForSingleObject(semOne, INFINITE);
        printf("beginning Accu\n");    // 7  13
        sum += num;
        printf("sum = %d\n", sum);     // 8  14
        ReleaseSemaphore(semTwo, 1, NULL);  // 释放 semTwo，计数变为 1
    }
    printf("Result: %d\n", sum);
    return 0;
}
```

### 执行流程分析

**回合 1：启动与第一次执行**

1. **Main** 启动 hThread1 (Read) 和 hThread2 (Accu)
2. **Thread 2 (Accu) 启动**：
   - 打印 "begin Accu\n" (打印 2)
   - 遇到 `WaitForSingleObject(semOne, INFINITE)`，由于 semOne 初始计数为 0，Accu 线程立即阻塞
3. **Thread 1 (Read) 启动**：
   - 打印 "Input num: " (stdout，打印 1)
   - 打印 "begin read\n" (打印 3)
   - 遇到 `WaitForSingleObject(semTwo, INFINITE)`，由于 semTwo 初始计数为 1，Read 线程获取信号量，计数变为 0，开始执行
   - 打印 "beginning read\n" (打印 4)
   - `scanf("%d", &num)` 等待用户输入，比如输入 `num`
   - `ReleaseSemaphore(semOne, 1, NULL)`：释放 semOne，计数变为 1
4. **Accu 线程被唤醒**：由于 semOne 变为 1，Accu 线程从阻塞中唤醒，如果拥有 Mutex 的线程异常终止，Mutex 会进入"已放弃"状态，可以被检测到
   - 打印 "beginning Accu\n" (打印 7)
   - `sum += num;`
   - 打印 "sum = %d\n", sum (打印 8)
   - `ReleaseSemaphore(semTwo, 1, NULL)`：释放 semTwo，计数变为 1

**后续回合：交替执行**

- **Read 回合**：获取 semTwo（计数变为 0），获取 num，释放 semOne（计数变为 1），然后回循
- **Accu 回合**：获取 semOne（计数变为 0），累加 num，释放 semTwo（计数变为 1）

> 这两个线程通过两个信号量实现了"先读入，后累加"的顺序，这种模式允许实现生产者-消费者问题的经典解法。
>
> - **semTwo (Read 许可)**：确保 Read 线程在 Accu 处理完上一个数据后才开始读取
> - **semOne (Accu 许可)**：确保 Accu 线程在 Read 读取完新数据后才开始累加

## 与互斥量的比较

| 特性 | 信号量 (Semaphore) | [[3.4 互斥量\|互斥量 (Mutex)]] |
|------|-------------------|-------------------------------|
| **计数范围** | 0 到 N（可配置最大值） | 0 或 1（二元状态） |
| **典型用途** | 限制并发访问数量（如连接池） | 保护[[3.8 临界区\|临界区]]的互斥访问 |
| **释放者** | 任何线程都可以释放 | 只有持有者可以释放 |
| **所有权** | 无所有权概念 | 有明确的所有权 |

---

## 关键要点

1. **信号量是一个计数器**，用于控制同时访问共享资源的线程数量
2. **P 操作（Wait）**：获取资源，计数器减 1；如果计数器为 0 则阻塞
3. **V 操作（Signal）**：释放资源，计数器加 1；可能唤醒等待的线程
4. **与互斥量的区别**：信号量允许多个线程同时访问，互斥量只允许一个
5. **常用于**：资源池管理、生产者-消费者问题、限流控制
