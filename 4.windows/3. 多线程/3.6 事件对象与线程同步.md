# 事件对象与线程同步

## 概述

事件对象 (Event Object) 是 Windows 提供的一种**内核同步对象**，用于线程间的信号通知和同步。与[[3.4 互斥量|互斥量]]不同，事件对象主要用于**通知机制**，而非资源的互斥访问。

---

## CreateEvent 函数

```cpp
HANDLE CreateEvent(
    LPSECURITY_ATTRIBUTES lpEventAttributes,  // 安全属性
    BOOL bManualReset,                         // 手动/自动重置
    BOOL bInitialState,                        // 初始状态
    LPCSTR lpName                              // 事件名称
);
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `lpEventAttributes` | 安全属性，通常设为 NULL |
| `bManualReset` | TRUE = 手动重置事件，FALSE = 自动重置事件 |
| `bInitialState` | TRUE = 初始为激活状态，FALSE = 初始为非激活状态 |
| `lpName` | 事件名称，NULL 表示匿名事件 |

---

## 事件类型

### 1. 自动重置事件 (Auto-Reset Event)

```cpp
hEvent = CreateEvent(NULL, FALSE, FALSE, NULL);
```

- **特点**：当事件被设置为激活状态后，**只有一个**等待的线程会被唤醒
- **自动重置**：一旦有线程通过 `WaitForSingleObject` 成功等待，事件会自动变回非激活状态
- **适用场景**："独占式"通知，确保只有一个线程响应事件

### 2. 手动重置事件 (Manual-Reset Event)

```cpp
hEvent = CreateEvent(NULL, TRUE, FALSE, NULL);
```

- **特点**：当事件被设置为激活状态后，**所有**等待的线程都会被唤醒
- **需手动重置**：必须显式调用 `ResetEvent` 才能将事件恢复为非激活状态
- **适用场景**："广播式"通知，需要同时唤醒多个线程

---

## 核心函数

| 函数 | 描述 |
|------|------|
| `CreateEvent` | 创建事件对象 |
| `WaitForSingleObject` / `WaitForMultipleObjects` | 等待事件变为激活状态 |
| `SetEvent` | 将事件设置为激活状态（触发信号） |
| `ResetEvent` | 将事件重置为非激活状态（清除信号） |

### 函数详解

#### SetEvent - 触发事件

```cpp
BOOL SetEvent(HANDLE hEvent);
```

- 将事件对象设置为**激活状态 (signaled)**
- 等待该事件的线程将被唤醒

#### ResetEvent - 重置事件

```cpp
BOOL ResetEvent(HANDLE hEvent);
```

- 将事件对象设置为**非激活状态 (non-signaled)**
- 对于手动重置事件，必须调用此函数来清除信号

## 示例代码：字符统计

以下示例展示了使用事件对象进行线程同步的典型场景：

### 代码实现

```cpp
#include <stdio.h>
#include <windows.h>
#include <process.h>

#define STR_LEN 100

unsigned WINAPI NumberOfA(void* arg);
unsigned WINAPI NumberOfOthers(void* arg);

static char str[STR_LEN];
static HANDLE hEvent;

int main(int argc, char* argv[])
{
    HANDLE hThread1, hThread2;

    // 创建手动重置事件，初始为非激活状态
    hEvent = CreateEvent(NULL, TRUE, FALSE, NULL);

    // 创建两个工作线程
    hThread1 = (HANDLE)_beginthreadex(NULL, 0, NumberOfA, NULL, 0, NULL);
    hThread2 = (HANDLE)_beginthreadex(NULL, 0, NumberOfOthers, NULL, 0, NULL);

    // 主线程：获取用户输入
    fputs("Input string: ", stdout);
    fgets(str, STR_LEN, stdin);

    // 触发事件，通知工作线程开始处理
    SetEvent(hEvent);

    // 等待工作线程完成
    WaitForSingleObject(hThread1, INFINITE);
    WaitForSingleObject(hThread2, INFINITE);

    // 重置事件（为下一轮做准备）
    ResetEvent(hEvent);

    CloseHandle(hEvent);
    return 0;
}

// 统计字符 'A' 的数量
unsigned WINAPI NumberOfA(void* arg)
{
    int i, cnt = 0;

    // 等待事件激活
    WaitForSingleObject(hEvent, INFINITE);

    for (i = 0; str[i] != 0; i++)
    {
        if (str[i] == 'A')
            cnt++;
    }

    printf("Number of A = %d\n", cnt);
    return 0;
}

// 统计其他字符的数量
unsigned WINAPI NumberOfOthers(void* arg)
{
    int i, cnt = 0;

    // 等待事件激活
    WaitForSingleObject(hEvent, INFINITE);

    for (i = 0; str[i] != 0; i++)
    {
        if (str[i] != 'A')
            cnt++;
    }

    printf("Number of Others = %d\n", cnt - 1);  // -1 排除换行符
    return 0;
}
```

## 自动重置 vs 手动重置对比

| 特性 | 自动重置事件 | 手动重置事件 |
|------|-------------|-------------|
| 创建方式 | `CreateEvent(NULL, FALSE, ...)` | `CreateEvent(NULL, TRUE, ...)` |
| 唤醒线程数 | 只唤醒**一个**等待线程 | 唤醒**所有**等待线程 |
| 重置方式 | 自动重置（无需调用 ResetEvent） | 必须手动调用 `ResetEvent` |
| 典型场景 | 任务分发、工作队列 | 广播通知、初始化完成信号 |

### 场景示例

**自动重置**：生产者-消费者模型中，每次只唤醒一个消费者

```cpp
// 自动重置事件
hEvent = CreateEvent(NULL, FALSE, FALSE, NULL);

// 生产者
SetEvent(hEvent);  // 只会唤醒一个消费者
```

**手动重置**：多个线程等待初始化完成

```cpp
// 手动重置事件
hEvent = CreateEvent(NULL, TRUE, FALSE, NULL);

// 初始化完成后
SetEvent(hEvent);  // 所有等待线程都被唤醒
```

## 与互斥量的区别

| 特性 | [[3.4 互斥量\|互斥量 (Mutex)]] | 事件对象 (Event) |
|------|------------------------------|-----------------|
| 主要用途 | 互斥访问共享资源 | 线程间信号通知 |
| 所有权 | 有所有权概念（只有拥有者能释放） | 无所有权概念 |
| 典型模式 | 锁定-访问-解锁 | 等待-通知 |
| 适用场景 | 保护临界区 | 线程协调、事件触发 |

---

## 注意事项

1. **选择正确的事件类型**：
   - 需要唤醒单个线程：使用自动重置事件
   - 需要唤醒所有线程：使用手动重置事件

2. **手动重置事件的陷阱**：
   - 如果不调用 `ResetEvent`，事件将持续保持激活状态
   - 后续的 `WaitForSingleObject` 调用会立即返回

3. **避免竞态条件**：
   - 在设置事件前确保共享数据已准备好
   - 考虑使用[[3.4 互斥量|互斥量]]配合事件对象

4. **资源释放**：
   - 使用完毕后调用 `CloseHandle` 关闭事件句柄
