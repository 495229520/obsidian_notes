# WM_COPYDATA 消息通信

## 1. 简介

`WM_COPYDATA` 是 Windows 提供的一种简单的[[4.1 进程简介|进程]]间通信机制，允许一个应用程序通过 `SendMessage` 向另一个应用程序**发送数据**。

### 1.1 核心特点

| 特性 | 描述 |
|------|------|
| **通信方式** | 通过 Windows 消息机制传递数据 |
| **通信方向** | 单向（发送者 → 接收者） |
| **同步性** | 同步通信，`SendMessage` 会阻塞直到接收方处理完消息 |
| **数据大小** | 理论上无限制，但建议不超过几 MB |
| **适用对象** | 必须是拥有窗口的应用程序 |
| **可靠性** | 可靠通信 |

### 1.2 工作原理

`WM_COPYDATA` 的独特之处在于：**系统会自动将数据从发送进程的地址空间复制到接收进程的地址空间**，解决了跨进程内存访问的问题。

```
┌────────────────┐                          ┌────────────────┐
│   发送进程     │                          │   接收进程     │
├────────────────┤                          ├────────────────┤
│                │      WM_COPYDATA         │                │
│ SendMessage() ─┼────────────────────────►│ WindowProc()   │
│                │  (系统自动复制数据)      │                │
│ COPYDATASTRUCT │                          │ COPYDATASTRUCT │
│  + 数据缓冲区  │                          │  + 数据缓冲区  │
└────────────────┘                          └────────────────┘
```

## 2. COPYDATASTRUCT 结构体

`WM_COPYDATA` 消息通过 `COPYDATASTRUCT` 结构体传递数据：

```cpp
typedef struct tagCOPYDATASTRUCT {
    ULONG_PTR dwData;    // 自定义数据标识，可用于区分消息类型
    DWORD     cbData;    // lpData 指向的数据大小（字节）
    PVOID     lpData;    // 指向要传递的数据的指针
} COPYDATASTRUCT, *PCOPYDATASTRUCT;
```

| 字段 | 说明 |
|------|------|
| `dwData` | 用户自定义数据，可用于标识消息类型或传递简单数值 |
| `cbData` | 数据缓冲区大小（字节数） |
| `lpData` | 指向要传递的数据，可以是任意类型（字符串、结构体、数组等） |

---

## 3. 关键 API 函数

| 函数 | 作用 | 说明 |
|------|------|------|
| `FindWindow()` | 查找目标窗口 | 根据窗口类名或标题获取窗口句柄 |
| `SendMessage()` | 发送消息 | 同步发送 WM_COPYDATA 消息 |
| `DefWindowProc()` | 默认消息处理 | 通常在接收方处理完数据后调用 |

### 3.1 SendMessage 参数说明

```cpp
LRESULT SendMessage(
    HWND   hWnd,      // 接收窗口的句柄
    UINT   Msg,       // 消息类型（WM_COPYDATA）
    WPARAM wParam,    // 发送窗口的句柄（可选，用于回复）
    LPARAM lParam     // 指向 COPYDATASTRUCT 的指针
);
```

## 4. 代码示例

### 4.1 发送端：发送字符串

```cpp
#include <windows.h>
#include <stdio.h>

// 定义消息类型标识
#define MY_MSG_STRING 1
#define MY_MSG_ARRAY  2

int main()
{
    printf("[Sender] 发送端启动...\n");

    // ========== 1. 查找接收窗口 ==========
    // 根据窗口标题查找（需要接收端先运行）
    HWND hRecvWnd = FindWindow(NULL, TEXT("ReceiverWindow"));

    if (hRecvWnd == NULL)
    {
        printf("[Sender] 找不到接收窗口！请先启动接收端。\n");
        return -1;
    }
    printf("[Sender] 找到接收窗口: 0x%p\n", hRecvWnd);

    // ========== 2. 准备要发送的数据 ==========
    char szMessage[] = "Hello from WM_COPYDATA!";

    // ========== 3. 填充 COPYDATASTRUCT ==========
    COPYDATASTRUCT cds;
    cds.dwData = MY_MSG_STRING;           // 消息类型标识
    cds.cbData = strlen(szMessage) + 1;   // 数据大小（包含结尾 \0）
    cds.lpData = szMessage;               // 数据指针

    // ========== 4. 发送 WM_COPYDATA 消息 ==========
    printf("[Sender] 正在发送数据: %s\n", szMessage);

    LRESULT result = SendMessage(
        hRecvWnd,                         // 目标窗口
        WM_COPYDATA,                      // 消息类型
        (WPARAM)NULL,                     // 发送窗口句柄（可选）
        (LPARAM)&cds                      // COPYDATASTRUCT 指针
    );

    if (result)
    {
        printf("[Sender] 发送成功！\n");
    }
    else
    {
        printf("[Sender] 发送失败或接收方返回 0\n");
    }

    printf("[Sender] 发送端结束\n");
    system("pause");
    return 0;
}
```

### 4.2 发送端：发送数组/结构体

```cpp
#include <windows.h>
#include <stdio.h>

// 定义要传递的结构体
typedef struct {
    int id;
    char name[32];
    double value;
} MyData;

int main()
{
    HWND hRecvWnd = FindWindow(NULL, TEXT("ReceiverWindow"));
    if (hRecvWnd == NULL)
    {
        printf("[Sender] 找不到接收窗口！\n");
        return -1;
    }

    // ========== 方式1：发送整数数组 ==========
    int nArray[] = { 1, 2, 3, 4, 5 };

    COPYDATASTRUCT cds1;
    cds1.dwData = 2;                              // 类型标识：数组
    cds1.cbData = sizeof(nArray);                 // 数组总大小
    cds1.lpData = nArray;                         // 数组指针

    SendMessage(hRecvWnd, WM_COPYDATA, 0, (LPARAM)&cds1);

    // ========== 方式2：发送结构体 ==========
    MyData data = { 100, "Test Data", 3.14159 };

    COPYDATASTRUCT cds2;
    cds2.dwData = 3;                              // 类型标识：结构体
    cds2.cbData = sizeof(MyData);                 // 结构体大小
    cds2.lpData = &data;                          // 结构体指针

    SendMessage(hRecvWnd, WM_COPYDATA, 0, (LPARAM)&cds2);

    return 0;
}
```

### 4.3 接收端：处理 WM_COPYDATA 消息

```cpp
#include <windows.h>
#include <stdio.h>

// 消息类型标识
#define MY_MSG_STRING 1
#define MY_MSG_ARRAY  2
#define MY_MSG_STRUCT 3

// 窗口过程函数
LRESULT CALLBACK WindowProc(HWND hWnd, UINT uMsg, WPARAM wParam, LPARAM lParam)
{
    switch (uMsg)
    {
    case WM_COPYDATA:
    {
        // ========== 1. 获取 COPYDATASTRUCT 指针 ==========
        PCOPYDATASTRUCT pCds = (PCOPYDATASTRUCT)lParam;

        printf("[Receiver] 收到 WM_COPYDATA 消息\n");
        printf("[Receiver] dwData = %lu, cbData = %lu\n",
               pCds->dwData, pCds->cbData);

        // ========== 2. 根据消息类型处理数据 ==========
        switch (pCds->dwData)
        {
        case MY_MSG_STRING:
        {
            // 处理字符串
            char* szMsg = (char*)pCds->lpData;
            printf("[Receiver] 收到字符串: %s\n", szMsg);

            // 也可以复制数据以便后续使用
            char szBuffer[256];
            strncpy(szBuffer, szMsg, pCds->cbData);
            MessageBoxA(hWnd, szBuffer, "收到消息", MB_OK);
            break;
        }
        case MY_MSG_ARRAY:
        {
            // 处理整数数组
            int* pArray = (int*)pCds->lpData;
            int count = pCds->cbData / sizeof(int);

            printf("[Receiver] 收到数组，元素数量: %d\n", count);
            for (int i = 0; i < count; i++)
            {
                printf("  pArray[%d] = %d\n", i, pArray[i]);
            }
            break;
        }
        case MY_MSG_STRUCT:
        {
            // 处理结构体（需要知道结构体定义）
            // MyData* pData = (MyData*)pCds->lpData;
            printf("[Receiver] 收到结构体数据\n");
            break;
        }
        default:
            printf("[Receiver] 未知消息类型: %lu\n", pCds->dwData);
            break;
        }

        // ========== 3. 返回处理结果 ==========
        return TRUE;  // 返回非零表示已处理
    }

    case WM_DESTROY:
        PostQuitMessage(0);
        return 0;
    }

    return DefWindowProc(hWnd, uMsg, wParam, lParam);
}

int WINAPI WinMain(HINSTANCE hInstance, HINSTANCE hPrevInstance,
                   LPSTR lpCmdLine, int nCmdShow)
{
    // ========== 1. 注册窗口类 ==========
    WNDCLASS wc = { 0 };
    wc.lpfnWndProc = WindowProc;
    wc.hInstance = hInstance;
    wc.lpszClassName = TEXT("MyReceiverClass");
    wc.hbrBackground = (HBRUSH)(COLOR_WINDOW + 1);

    RegisterClass(&wc);

    // ========== 2. 创建窗口 ==========
    HWND hWnd = CreateWindow(
        TEXT("MyReceiverClass"),
        TEXT("ReceiverWindow"),          // 窗口标题（发送端通过此查找）
        WS_OVERLAPPEDWINDOW,
        CW_USEDEFAULT, CW_USEDEFAULT,
        400, 300,
        NULL, NULL, hInstance, NULL
    );

    ShowWindow(hWnd, nCmdShow);
    UpdateWindow(hWnd);

    printf("[Receiver] 接收端已启动，等待消息...\n");

    // ========== 3. 消息循环 ==========
    MSG msg;
    while (GetMessage(&msg, NULL, 0, 0))
    {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    }

    return (int)msg.wParam;
}
```

## 5. 使用注意事项

### 5.1 数据生命周期

| 注意点 | 说明 |
|--------|------|
| **临时数据** | `lpData` 指向的数据在 `WM_COPYDATA` 消息处理期间有效 |
| **需要保存** | 如果需要在消息处理后使用数据，必须复制到自己的缓冲区 |
| **不要保存指针** | 消息处理完成后，`lpData` 指针失效 |

### 5.2 线程安全

```cpp
// 正确做法：复制数据
case WM_COPYDATA:
{
    PCOPYDATASTRUCT pCds = (PCOPYDATASTRUCT)lParam;

    // 分配内存并复制数据
    char* pBuffer = new char[pCds->cbData];
    memcpy(pBuffer, pCds->lpData, pCds->cbData);

    // 现在可以安全地在其他线程使用 pBuffer
    // ...

    delete[] pBuffer;
    return TRUE;
}
```

### 5.3 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `FindWindow` 返回 NULL | 接收窗口未创建或标题不匹配 | 确保接收端先启动，检查窗口标题 |
| 数据乱码 | 字符编码不一致 | 统一使用 ANSI 或 Unicode |
| 发送失败 | 接收窗口未处理 WM_COPYDATA | 确保接收端正确处理该消息 |
| 程序卡死 | `SendMessage` 是同步的 | 接收端应快速处理消息 |

## 6. 与其他 IPC 方式对比

| 方式 | 适用场景 | 是否需要窗口 | 数据大小限制 | 同步性 |
|------|----------|--------------|--------------|--------|
| **WM_COPYDATA** | 简单数据传递 | 需要 | 建议 < 几 MB | 同步 |
| **[[4.4 匿名管道通讯\|匿名管道]]** | 父子进程通信 | 不需要 | 无限制 | 可同步/异步 |
| **[[4.5 命名管道通讯\|命名管道]]** | 双向通信、网络通信 | 不需要 | 无限制 | 可同步/异步 |
| **[[4.3 油槽通信\|油槽]]** | 一对多广播 | 不需要 | 424 字节 | 异步 |
| **[[4.2 剪切板通讯\|剪切板]]** | 用户复制粘贴 | 不需要 | 无限制 | 同步 |
| **共享内存** | 大数据量、高频通信 | 不需要 | 无限制 | 需同步 |

## 8. 关键要点

1. **WM_COPYDATA** 是基于 Windows 消息机制的 IPC 方式
2. 使用 **COPYDATASTRUCT** 结构体传递数据
3. **SendMessage** 是同步的，会阻塞直到接收方处理完毕
4. 系统**自动复制数据**到接收进程的地址空间
5. 接收方需要**复制数据**以便后续使用（数据在消息处理后失效）
6. 适合**简单的数据传递**，不适合大数据量或高频通信
7. 双方都必须是**有窗口的应用程序**
