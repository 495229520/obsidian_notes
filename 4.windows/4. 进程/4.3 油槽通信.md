# 油槽通信 (Mailslot IPC)

## 1. 简介

油槽（Mailslot，又称邮槽）是 Windows 提供的一种[[4.1 进程简介|进程]]间通信机制，允许进程之间进行**单向**的消息传递。

### 1.1 核心特点

| 特性 | 描述 |
|------|------|
| **通信方向** | 单向通信，只有服务端能从油槽中读取消息，客户端只能写入消息 |
| **消息顺序** | 先入先出 (FIFO)，客户端先写入的消息在服务端先被读取 |
| **消息大小** | 一条消息**不能大于 424 字节** |
| **数据格式** | 可以是任意格式 |
| **可靠性** | 不可靠通信（基于 UDP 协议） |

### 1.2 本地通信 vs 网络通信

- **本地通信**：油槽可在本机内进行进程间通信
- **网络通信**：油槽也可以在不同主机间通信，但数据通过网络传输时使用的是**数据报协议 (UDP)**，因此是一种**不可靠的通信**
- **网络通信要求**：客户端必须知道服务端的**主机名或域名**

> 关于更多进程间通信的方式，请参考 [[11.2 进程间通信IPC|进程间通信IPC]]。

## 2. 通信模型

```
┌─────────────┐                      ┌─────────────┐
│   客户端    │                      │   服务端    │
│  (写入方)   │                      │  (读取方)   │
├─────────────┤                      ├─────────────┤
│             │     ┌──────────┐     │             │
│ CreateFile()│────►│          │     │CreateMail-  │
│             │     │  油槽    │────►│  slot()     │
│ WriteFile() │────►│(Mailslot)│     │             │
│             │     │          │────►│ ReadFile()  │
│ CloseHandle │     └──────────┘     │             │
└─────────────┘                      └─────────────┘
     写入                                  读取
```

## 3. 关键 API 函数

### 3.1 服务端函数（读取方）

| 函数 | 作用 | 说明 |
|------|------|------|
| `CreateMailslot()` | 创建/打开油槽 | 设置最大消息大小、超时时间等 |
| `ReadFile()` | 从油槽读取消息 | 读取客户端发送的数据 |

### 3.2 客户端函数（写入方）

| 函数 | 作用 | 说明 |
|------|------|------|
| `CreateFile()` | 打开已存在的油槽 | 以写入模式打开 |
| `WriteFile()` | 向油槽写入消息 | 发送数据到油槽 |
| `CloseHandle()` | 关闭句柄 | 释放资源 |

---

## 4. 油槽命名规则

油槽名称遵循特定的格式：

```
\\.\mailslot\[path]name        // 本地油槽
\\hostname\mailslot\[path]name // 远程油槽
\\*\mailslot\[path]name        // 广播到域内所有计算机
```

| 格式 | 说明 | 示例 |
|------|------|------|
| `\\\\.\\mailslot\\name` | 本地油槽 | `\\\\.\\mailslot\\Mymailslot` |
| `\\\\hostname\\mailslot\\name` | 指定主机的油槽 | `\\\\server1\\mailslot\\Mymailslot` |
| `\\\\*\\mailslot\\name` | 广播到域内所有计算机 | `\\\\*\\mailslot\\Mymailslot` |

## 5. 代码示例

### 5.1 服务端：创建油槽并接收数据

```cpp
void CProcessCommunitDlg::OnBnClickedRecvBtn()
{
    // 1. 创建一个油槽
    LPCTSTR szSlotName = TEXT("\\\\.\\mailslot\\Mymailslot");
    HANDLE hSlot = CreateMailslot(
        szSlotName,
        0,                        // no maximum message size (0 表示不限制)
        MAILSLOT_WAIT_FOREVER,    // no time-out for operations (永久等待)
        NULL);                    // default security

    if (hSlot == INVALID_HANDLE_VALUE)
    {
        TRACE("CreateMailslot failed with %d\n", GetLastError());
        return;
    }

    // 2. 读取数据
    char szBuf[100] = { 0 };
    DWORD dwRead;
    if (!ReadFile(hSlot, szBuf, 100, &dwRead, NULL))
    {
        MessageBox("Read Failed!!");
        return;
    }

    TRACE("####dwRead = %d\n", dwRead);
    MessageBox(szBuf);

    // 3. 关闭句柄
    CloseHandle(hSlot);
}
```

### 5.2 客户端：打开油槽并发送数据

```cpp
void CProcessClientDlg::OnBnClickedSendBtn()
{
    // 1. 创建一个文件句柄（打开已存在的油槽）
    LPCTSTR szSlotName = TEXT("\\\\.\\mailslot\\Mymailslot");
    HANDLE hMailSlot = CreateFile(
        szSlotName,
        FILE_GENERIC_WRITE,       // 写入权限
        FILE_SHARE_READ,          // 共享读取
        NULL,                     // 默认安全属性
        OPEN_EXISTING,            // 打开已存在的油槽
        FILE_ATTRIBUTE_NORMAL,    // 普通文件属性
        NULL);                    // 无模板

    if (hMailSlot == INVALID_HANDLE_VALUE)
    {
        TRACE("CreateFile failed with %d\n", GetLastError());
        return;
    }

    // 2. 写入数据
    char szBuf[100] = "MailSlot Comming";
    DWORD dwWrite;
    if (!WriteFile(hMailSlot, szBuf, strlen(szBuf) + 1, &dwWrite, NULL))
    {
        MessageBox("Write Failed!!");
        return;
    }

    // 3. 关闭句柄
    CloseHandle(hMailSlot);
}
```

## 6. CreateMailslot 函数详解

```cpp
HANDLE CreateMailslot(
    LPCTSTR lpName,           // 油槽名称
    DWORD   nMaxMessageSize,  // 最大消息大小（0 表示不限制，但实际受系统限制）
    DWORD   lReadTimeout,     // 读取超时时间（毫秒）
    LPSECURITY_ATTRIBUTES lpSecurityAttributes  // 安全属性
);
```

| 参数 | 说明 |
|------|------|
| `lpName` | 油槽名称，格式为 `\\.\mailslot\name` |
| `nMaxMessageSize` | 最大消息大小，0 表示使用系统默认（424 字节） |
| `lReadTimeout` | 读取超时时间，`MAILSLOT_WAIT_FOREVER` 表示永久等待 |
| `lpSecurityAttributes` | 安全属性，通常为 NULL |
| **返回值** | 成功返回油槽句柄，失败返回 `INVALID_HANDLE_VALUE` |

## 7. 与其他 IPC 方式对比

| 方式 | 通信方向 | 可靠性 | 消息大小限制 | 网络支持 | 适用场景 |
|------|----------|--------|--------------|----------|----------|
| **油槽** | 单向 | 不可靠 (UDP) | 424 字节 | 支持 | 一对多广播、简单通知 |
| **[[4.2 剪切板通讯\|剪切板]]** | 单向 | 可靠 | 无限制 | 不支持 | 用户复制/粘贴 |
| **[[4.4 匿名管道通讯\|匿名管道]]** | 单向 | 可靠 | 无限制 | 不支持 | 父子进程通信 |
| **[[4.5 命名管道通讯\|命名管道]]** | 双向 | 可靠 | 无限制 | 支持 | 双向数据交换 |
| **[[4.6 VM_COPYDATA\|WM_COPYDATA]]** | 单向 | 可靠 | 无限制 | 不支持 | 窗口间消息传递 |

## 8. 注意事项

1. **单向通信**：油槽只支持单向通信，服务端只能读取，客户端只能写入
2. **消息大小限制**：单条消息不能超过 424 字节
3. **不可靠性**：网络通信基于 UDP，消息可能丢失
4. **先启动服务端**：必须先启动服务端创建油槽，客户端才能连接
5. **广播功能**：油槽支持向域内所有计算机广播消息
6. **资源释放**：使用完毕后必须调用 `CloseHandle()` 关闭句柄

---

## 9. 关键要点

1. **油槽是单向的 IPC 机制**，适合一对多的消息广播
2. **CreateMailslot()** 创建油槽，**CreateFile()** 打开油槽
3. 消息大小限制为 **424 字节**
4. 网络通信基于 **UDP**，是**不可靠**的
5. 油槽名称格式：`\\.\mailslot\name`（本地）或 `\\hostname\mailslot\name`（远程）
