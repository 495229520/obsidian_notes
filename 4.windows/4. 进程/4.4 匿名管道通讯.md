# 匿名管道通讯 (Anonymous Pipe IPC)

## 1. 简介

匿名管道是 Windows 提供的一种[[4.1 进程简介|进程]]间通信机制，主要用于**父子进程**之间的单向数据传递。它通过重定向子进程的标准输入/输出来实现通信。

> 匿名管道的概念与 Linux 中的管道类似，详见 [[11.3 管道通讯|Linux 管道通讯]]。

---

## 2. 相关知识

### 2.1 核心 API 函数

| 函数 | 作用 |
|------|------|
| `CreatePipe()` | 创建匿名管道，返回管道的读句柄和写句柄 |
| `CreateProcess()` | 创建子进程，并设置 `STARTUPINFO` 结构体进行句柄继承 |
| `ReadFile()` | 从管道中读取数据 |
| `WriteFile()` | 向管道中写入数据 |
| `CloseHandle()` | 关闭不再使用的管道句柄 |

### 2.2 通信特点

- **单向通信**：一对句柄只能实现一个方向的单工通信
- **双向通信**：需要创建两个管道（每个方向一个）
- **句柄继承**：子进程通过继承父进程的管道句柄来进行通信

---

## 3. 编程原则

### 3.1 基本流程

```
┌────────────────────────────────────────────────────────────┐
│                        父进程                              │
├────────────────────────────────────────────────────────────┤
│  1. CreatePipe() 创建管道 → 获得 hRead 和 hWrite           │
│  2. 设置 STARTUPINFO，将管道句柄设为子进程的标准输入/输出  │
│  3. CreateProcess() 创建子进程（bInheritHandles = TRUE）   │
│  4. 关闭父进程中不需要的句柄端                             │
│  5. WriteFile() 向管道写入数据 / ReadFile() 从管道读取     │
└────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────┐
│                        子进程                              │
├────────────────────────────────────────────────────────────┤
│  1. GetStdHandle(STD_INPUT_HANDLE) 获取标准输入句柄        │
│  2. ReadFile() 从标准输入读取数据（实际从管道读取）        │
│  或者：                                                    │
│  1. GetStdHandle(STD_OUTPUT_HANDLE) 获取标准输出句柄       │
│  2. WriteFile() 向标准输出写入数据（实际写入管道）         │
└────────────────────────────────────────────────────────────┘
```

### 3.2 关键设置

创建子进程时，`STARTUPINFO` 结构体的关键字段：

| 字段 | 说明 |
|------|------|
| `cb` | 结构体大小，必须设置为 `sizeof(STARTUPINFO)` |
| `dwFlags` | 标志位，设置为 `STARTF_USESTDHANDLES` 表示使用标准句柄 |
| `hStdInput` | 子进程的标准输入句柄（设为管道读端） |
| `hStdOutput` | 子进程的标准输出句柄（设为管道写端） |
| `hStdError` | 子进程的标准错误句柄 |

**重要**：`CreateProcess()` 的 `bInheritHandles` 参数必须设置为 `TRUE`，否则子进程无法继承管道句柄。

## 4. CreatePipe 函数详解

```cpp
BOOL CreatePipe(
    PHANDLE hReadPipe,                        // 输出：读取端句柄
    PHANDLE hWritePipe,                       // 输出：写入端句柄
    LPSECURITY_ATTRIBUTES lpPipeAttributes,   // 安全属性（需设置可继承）
    DWORD nSize                               // 管道缓冲区大小（0 表示默认）
);
```

### 4.1 安全属性设置

为了让子进程能够继承管道句柄，必须正确设置安全属性：

```cpp
SECURITY_ATTRIBUTES sa;
sa.nLength = sizeof(SECURITY_ATTRIBUTES);
sa.lpSecurityDescriptor = NULL;
sa.bInheritHandle = TRUE;  // 关键：允许句柄被子进程继承
```

## 5. 代码示例

### 5.1 父进程：创建管道并与子进程通信

```cpp
#include <windows.h>
#include <stdio.h>
#include <string.h>

int main(int argc, char* argv[])
{
    // ========== 1. 创建管道 ==========
    HANDLE hRead, hWrite;

    // 设置安全属性，允许句柄继承
    SECURITY_ATTRIBUTES sa;
    sa.nLength = sizeof(SECURITY_ATTRIBUTES);
    sa.lpSecurityDescriptor = NULL;
    sa.bInheritHandle = TRUE;  // 允许继承

    // 创建管道
    if (!CreatePipe(&hRead, &hWrite, &sa, 0))
    {
        printf("CreatePipe Failed!\n");
        return -1;
    }

    // ========== 2. 设置 STARTUPINFO ==========
    STARTUPINFO si;
    memset(&si, 0, sizeof(si));
    si.cb = sizeof(STARTUPINFO);
    si.dwFlags = STARTF_USESTDHANDLES;  // 使用标准句柄
    si.hStdInput = hRead;                // 子进程的标准输入 = 管道读端
    si.hStdOutput = GetStdHandle(STD_OUTPUT_HANDLE);
    si.hStdError = GetStdHandle(STD_ERROR_HANDLE);

    // ========== 3. 创建子进程 ==========
    PROCESS_INFORMATION pi;
    memset(&pi, 0, sizeof(pi));

    // 子进程的命令行（子进程可执行文件路径）
    TCHAR szCmdLine[] = TEXT("ChildProcess.exe");

    BOOL bRet = CreateProcess(
        NULL,           // lpApplicationName
        szCmdLine,      // lpCommandLine
        NULL,           // lpProcessAttributes
        NULL,           // lpThreadAttributes
        TRUE,           // bInheritHandles: 必须为 TRUE！
        CREATE_NEW_CONSOLE,  // dwCreationFlags
        NULL,           // lpEnvironment
        NULL,           // lpCurrentDirectory
        &si,            // lpStartupInfo
        &pi             // lpProcessInformation
    );

    if (!bRet)
    {
        printf("CreateProcess Failed! Error: %d\n", GetLastError());
        CloseHandle(hRead);
        CloseHandle(hWrite);
        return -1;
    }

    // ========== 4. 父进程关闭不需要的读端 ==========
    CloseHandle(hRead);

    // ========== 5. 向管道写入数据 ==========
    char szBuffer[] = "Data from Parent Process";
    DWORD dwWritten;

    printf("[Parent] 正在发送数据...\n");
    if (WriteFile(hWrite, szBuffer, strlen(szBuffer) + 1, &dwWritten, NULL))
    {
        printf("[Parent] 发送成功，字节数: %d\n", dwWritten);
    }
    else
    {
        printf("[Parent] 发送失败!\n");
    }

    // ========== 6. 等待子进程结束 ==========
    WaitForSingleObject(pi.hProcess, INFINITE);

    // ========== 7. 清理资源 ==========
    CloseHandle(hWrite);
    CloseHandle(pi.hProcess);
    CloseHandle(pi.hThread);

    printf("[Parent] 通信完成\n");
    return 0;
}
```

### 5.2 子进程：从管道读取数据

```cpp
#include <windows.h>
#include <stdio.h>

int main(int argc, char* argv[])
{
    printf("[Child] 子进程启动\n");

    // ========== 1. 获取标准输入句柄（实际是管道读端）==========
    HANDLE hStdInput = GetStdHandle(STD_INPUT_HANDLE);

    if (hStdInput == INVALID_HANDLE_VALUE)
    {
        printf("[Child] 获取标准输入失败!\n");
        return -1;
    }

    // ========== 2. 从管道读取数据 ==========
    char szBuffer[256] = { 0 };
    DWORD dwRead;

    printf("[Child] 等待接收数据...\n");
    if (ReadFile(hStdInput, szBuffer, sizeof(szBuffer) - 1, &dwRead, NULL))
    {
        printf("[Child] 收到数据: %s\n", szBuffer);
        printf("[Child] 字节数: %d\n", dwRead);
    }
    else
    {
        printf("[Child] 读取失败! Error: %d\n", GetLastError());
    }

    // ========== 3. 处理数据（示例：转换为大写）==========
    for (DWORD i = 0; i < dwRead; i++)
    {
        if (szBuffer[i] >= 'a' && szBuffer[i] <= 'z')
        {
            szBuffer[i] -= 32;  // 转为大写
        }
    }
    printf("[Child] 处理后: %s\n", szBuffer);

    printf("[Child] 子进程结束\n");
    system("pause");
    return 0;
}
```

## 6. 双向管道通信

要实现双向通信，需要创建**两个管道**：

```
          管道1 (父→子)
    父进程 ──────────────► 子进程
    (写入)                 (读取)

          管道2 (子→父)
    父进程 ◄────────────── 子进程
    (读取)                 (写入)
```

### 6.1 双向通信示例（父进程）

```cpp
// 创建两个管道
HANDLE hReadFromChild, hWriteToChild;
HANDLE hReadFromParent, hWriteToParent;

// 管道1: 父 → 子
CreatePipe(&hReadFromParent, &hWriteToChild, &sa, 0);

// 管道2: 子 → 父
CreatePipe(&hReadFromChild, &hWriteToParent, &sa, 0);

// 设置子进程的标准输入输出
si.hStdInput = hReadFromParent;   // 子进程从管道1读取
si.hStdOutput = hWriteToParent;   // 子进程向管道2写入
```

## 7. 与 Linux 管道对比

| 特性 | Windows 匿名管道 | Linux 匿名管道 |
|------|------------------|----------------|
| **创建函数** | `CreatePipe()` | `pipe()` |
| **进程创建** | `CreateProcess()` | `fork()` |
| **句柄继承** | 通过 `STARTUPINFO` 设置 | 自动继承（fork 复制文件描述符） |
| **通信方向** | 单向 | 单向 |
| **读写函数** | `ReadFile()` / `WriteFile()` | `read()` / `write()` |
| **标准句柄重定向** | 手动设置 `hStdInput/hStdOutput` | 使用 `dup2()` |

## 8. 与其他 IPC 方式对比

| 方式 | 通信方向 | 进程关系 | 网络支持 | 适用场景 |
|------|----------|----------|----------|----------|
| **匿名管道** | 单向 | 父子进程 | 不支持 | 父子进程间简单通信 |
| **[[4.5 命名管道通讯\|命名管道]]** | 双向 | 任意进程 | 支持 | 双向可靠通信 |
| **[[4.3 油槽通信\|油槽]]** | 单向 | 任意进程 | 支持 | 一对多广播 |
| **[[4.2 剪切板通讯\|剪切板]]** | 单向 | 任意进程 | 不支持 | 用户交互 |
| **[[4.6 VM_COPYDATA\|WM_COPYDATA]]** | 单向 | 任意进程 | 不支持 | 窗口间消息 |

## 9. 注意事项

1. **句柄继承**：`CreateProcess()` 的 `bInheritHandles` 必须为 `TRUE`
2. **安全属性**：`SECURITY_ATTRIBUTES.bInheritHandle` 必须为 `TRUE`
3. **关闭未使用的句柄**：父进程写入时应关闭读端，避免资源泄漏
4. **阻塞行为**：`ReadFile()` 会阻塞直到有数据可读或写端全部关闭
5. **标准句柄重定向**：通过 `STARTUPINFO` 将管道句柄设为子进程的标准输入/输出
6. **双向通信**：需要创建两个管道

---

## 10. 关键要点

1. **匿名管道**用于父子进程间的单向通信
2. 使用 **CreatePipe()** 创建管道，获得读/写句柄
3. 通过 **STARTUPINFO** 将管道句柄传递给子进程
4. **bInheritHandles = TRUE** 是关键设置
5. 子进程使用 **GetStdHandle()** 获取重定向后的标准输入/输出
6. 实现双向通信需要创建**两个管道**
