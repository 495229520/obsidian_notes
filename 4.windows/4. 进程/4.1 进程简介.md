# Windows 进程简介

## 1. 核心特征

一个进程通常包含以下几个关键组成部分：

### 1.1 进程控制块 (PCB - Process Control Block)

这是操作系统用来**管理进程**的数据结构，它包含了进程的所有状态信息，例如：

| 信息类型 | 示例 |
|----------|------|
| **进程ID (PID)** | 唯一的标识符 |
| **进程状态** | Running, Ready, Blocked, Terminated 等 |
| **程序计数器 (PC)** | 指示下一条要执行的指令地址 |
| **寄存器值** | 保存CPU寄存器的当前值 |
| **内存管理信息** | 页表、段表等 |
| **I/O 状态信息** | 分配给进程的I/O设备、打开的文件列表等 |

### 1.2 程序代码和数据段 (Text/Data Segment)

| 段类型 | 内容描述 |
|--------|----------|
| **代码段 (Text Segment)** | 包含进程要执行的机器指令 |
| **数据段 (Data Segment)** | 包含全局变量和静态变量的初始化值及运行时动态变化的值 |

> 详细的内存布局请参考 [[1.3 进程的虚拟地址空间|进程的虚拟地址空间]]。

### 1.3 用户栈 (User Stack)

- 用于存放局部变量、函数参数、返回地址等，遵循"后进先出"(LIFO) 的原则
- 每次函数调用都会在栈上生成一个[[1.2 函数栈帧建立与销毁原理|栈帧 (Stack Frame)]]

### 1.4 堆 (Heap)

- 用于动态内存分配，如 C 语言中的 malloc()/calloc() 或 C++ 中的 new 操作符所分配的内存
- 详见 [[6.2 动态内存分配与释放|动态内存分配]]

---

## 2. 进程与线程的区别

| 特性 | 进程 (Process) | 线程 (Thread) |
|------|----------------|---------------|
| **定义** | 程序的一次执行实例，是资源分配的单位 | 进程内的执行单元，是 CPU 调度的基本单位 |
| **资源拥有** | 拥有独立的内存地址空间、文件句柄、PCB 等 | 与同进程的其他线程共享内存地址空间、文件句柄等 |
| **切换开销** | 开销大（需要切换整个地址空间和PCB） | 开销小（只切换线程独有的数据） |
| **通信** | 相对复杂（需要使用IPC机制：管道、消息队列、共享内存等） | 简单（直接读写共享内存，但需同步机制） |
| **独立性** | 一个进程崩溃通常不影响其他进程 | 一个线程崩溃可能导致整个进程崩溃 |

> 更多关于线程的内容请参考 [[3.1 多线程简介|Windows多线程简介]]。

## 3. 创建进程：CreateProcess()

CreateProcess() 是 Windows API 中用于创建新进程的核心函数。

### 3.1 初始化结构体

代码定义并初始化了两个关键的 Windows 结构体：

| 结构体 | 作用 | 初始化方式 |
|--------|------|------------|
| **STARTUPINFO strStartupInfo** | 包含新进程主窗口的属性，如大小、位置、显示方式等 | 使用 memset 清零，并设置 cb 字段为结构体大小，这是 Windows API 的标准做法 |
| **PROCESS_INFORMATION szProcessInformation** | 包含新创建进程及其主线程的句柄和ID，这是进程创建成功后 API 返回的信息句柄 | 使用 memset 清零 |

### 3.2 命令行字符串

```cpp
TCHAR szCommandLine[] = L"\"C:\Program Files\Google\Chrome\Application\chrome.exe \"http://www.bilibili.com/\"";
```

- TCHAR 在 Windows 中，为了兼容 ANSI（窄字符）和 Unicode（宽字符），通常使用 TCHAR，由于字符串前的是 L，表明这是一个 Unicode（宽字符）字符串
- **字符串内容**：这是要执行的完整命令行，包含 Chrome 可执行文件路径（用双引号包裹以处理路径中的空格）和要打开的 URL 作为参数

### 3.3 CreateProcess() 函数

```cpp
int bRet = CreateProcess(
    NULL,                      // lpApplicationName: NULL（表示从命令行读取程序名）
    szCommandLine,             // lpCommandLine: 命令行字符串（程序路径 + 参数）
    NULL,                      // lpProcessAttributes: 进程句柄的继承性
    NULL,                      // lpThreadAttributes: 线程句柄的继承性
    FALSE,                     // bInheritHandles: 不继承父进程的句柄
    CREATE_NEW_CONSOLE,        // dwCreationFlags: 创建标志，这里表示为新进程创建一个新的控制台窗口
    NULL,                      // lpEnvironment: 使用父进程的环境块
    NULL,                      // lpCurrentDirectory: 使用父进程的当前目录
    &strStartupInfo,           // lpStartupInfo: 传入启动信息
    &szProcessInformation      // lpProcessInformation: 接收新进程的信息
);
```

### 3.4 完整示例代码

```cpp
#include <windows.h>
#include <stdio.h>

int RunExe()
{
    STARTUPINFO strStartupInfo;
    memset(&strStartupInfo, 0, sizeof(strStartupInfo));
    strStartupInfo.cb = sizeof(strStartupInfo);

    PROCESS_INFORMATION szProcessInformation;
    memset(&szProcessInformation, 0, sizeof(szProcessInformation));

    TCHAR szCommandLine[] = L"\"C:\Program Files\Google\Chrome\Application\chrome.exe\" \"http://www.bilibili.com/\"";

    int bRet = CreateProcess(
        NULL,
        szCommandLine,
        NULL,
        NULL,
        FALSE,
        CREATE_NEW_CONSOLE,
        NULL,
        NULL,
        &strStartupInfo,
        &szProcessInformation
    );

    if (bRet)
    {
        // 创建成功
        printf("Create Success bRet = %d\n", bRet);
        WaitForSingleObject(szProcessInformation.hProcess, 3000);

        printf("Create Success szProcessInformation.hProcess = %d\n",
               szProcessInformation.hProcess);
        printf("Create Success szProcessInformation.hThread = %d\n",
               szProcessInformation.hThread);
        printf("Create Success szProcessInformation.dwProcessId = %d\n",
               szProcessInformation.dwProcessId);
        printf("Create Success szProcessInformation.dwThreadId = %d\n",
               szProcessInformation.dwThreadId);

        // 关闭句柄
        CloseHandle(szProcessInformation.hProcess);
        CloseHandle(szProcessInformation.hThread);
        szProcessInformation.hProcess = NULL;
        szProcessInformation.hThread = NULL;
        szProcessInformation.dwProcessId = 0;
        szProcessInformation.dwThreadId = 0;
    }
    else
    {
        printf("Create Failed bRet = %d\n", bRet);
        printf("errorcode = %d\n", GetLastError());
    }
    return 0;
}

int main()
{
    printf("This is Chrome");
    RunExe();
    system("pause");
    return 0;
}
```

### 3.5 关键要点

1. CreateProcess 成功返回非零值，失败返回 0
2. 创建成功后应使用 WaitForSingleObject 等待进程或设置超时
3. 必须调用 CloseHandle 关闭进程和线程句柄，避免资源泄漏
4. 使用 GetLastError() 获取失败时的错误代码

## 4. Windows 与 Linux 进程创建对比

| 特性 | Windows | Linux |
|------|---------|-------|
| **创建函数** | CreateProcess() | fork() + exec() |
| **创建方式** | 直接创建新进程并加载程序 | 先复制父进程，再替换程序映像 |
| **进程关系** | 相对独立 | 有明显的父子关系 |
| **进程ID获取** | 通过 PROCESS_INFORMATION 结构体 | fork() 返回值 |

> Linux 下的进程创建请参考 [[11.1 进程简介|Linux进程简介]]。

---

## 5. 关键要点

1. **进程**是资源分配的基本单位，拥有独立的[[1.3 进程的虚拟地址空间|虚拟地址空间]]
2. **PCB** 是操作系统管理进程的核心数据结构
3. **线程**比进程轻量，同进程的线程共享资源
4. **CreateProcess()** 是 Windows 创建进程的核心 API
5. 创建进程后必须正确关闭句柄以避免资源泄漏
