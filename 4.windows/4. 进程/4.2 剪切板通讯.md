# 剪切板通讯 (Clipboard IPC)

## 1. 简介

使用剪切板可以实现多个[[4.1 进程简介|进程]]之间的通讯，但是**并不常见，仅供参考**。

剪切板是 Windows 提供的一种简单的进程间通信机制，它允许不同应用程序之间共享数据。虽然主要用于复制/粘贴操作，但也可以用于简单的 IPC 场景。

> 关于更多进程间通信的方式，请参考 [[11.2 进程间通信IPC|进程间通信IPC]]。

---

## 2. 剪切板操作流程

### 2.1 发送数据流程

```
打开剪切板 → 清空剪切板 → 分配内存 → 加锁写入 → 解锁 → 设置数据 → 关闭剪切板
```

### 2.2 接收数据流程

```
打开剪切板 → 检查格式 → 获取数据句柄 → 加锁读取 → 解锁 → 关闭剪切板
```

---

## 3. 关键 API 函数

| 函数 | 作用 | 说明 |
|------|------|------|
| `OpenClipboard()` | 打开剪切板 | 成功返回 TRUE，失败返回 FALSE |
| `EmptyClipboard()` | 清空剪切板 | 清空当前内容，准备写入新数据 |
| `CloseClipboard()` | 关闭剪切板 | 释放剪切板访问权 |
| `GlobalAlloc()` | 分配全局内存 | 分配可在进程间共享的内存 |
| `GlobalLock()` | 锁定内存 | 获取内存指针以进行读写 |
| `GlobalUnlock()` | 解锁内存 | 释放内存锁定 |
| `SetClipboardData()` | 设置剪切板数据 | 将数据放入剪切板 |
| `GetClipboardData()` | 获取剪切板数据 | 从剪切板读取数据 |
| `IsClipboardFormatAvailable()` | 检查格式是否可用 | 检查剪切板中是否有指定格式的数据 |

## 4. 代码示例

### 4.1 进程一：发送数据

```cpp
void CClipBoardDlg::OnBnClickedSendBtn()
{
    // 1. 打开剪切板，成功打开返回TRUE
    if (OpenClipboard())
    {
        // 2. 清空剪切板
        EmptyClipboard();
        char* szSendBuf;
        
        // 3. 获取编辑框的内容
        CStringA strSend;
        GetDlgItemTextA(IDC_EDIT_SEND, strSend);
        
        // 4. 分配一个内存对象，内存对象的句柄就是hClip
        //    GMEM_MOVEABLE: 分配可移动的内存块
        //    +1 是为了存储字符串结尾的 '\0'
        HANDLE hClip = GlobalAlloc(GMEM_MOVEABLE, strSend.GetLength() + 1);
        
        // 5. 剪切板加锁，获取内存指针
        szSendBuf = (char *)GlobalLock(hClip);
        strcpy(szSendBuf, strSend);
        GlobalUnlock(hClip);
        
        // 6. 将数据放入剪切板
        //    CF_TEXT: 表示数据格式为文本
        SetClipboardData(CF_TEXT, hClip);
        
        // 7. 关闭剪切板
        CloseClipboard();
    }
}
```

### 4.2 进程一：接收数据

```cpp
void CClipBoardDlg::OnBnClickedRecvBtn()
{
    // 1. 打开剪切板，成功打开返回TRUE
    if (OpenClipboard())
    {
        // 2. 判断剪切板是否可用（是否有文本数据）
        if (IsClipboardFormatAvailable(CF_TEXT))
        {
            char* szRecvBuf;
            
            // 3. 从剪切板获取数据句柄
            HANDLE hClip = GetClipboardData(CF_TEXT);
            
            // 4. 锁定内存并读取数据
            szRecvBuf = (char*)GlobalLock(hClip);
            SetDlgItemTextA(IDC_EDIT_RECV, szRecvBuf);
            GlobalUnlock(hClip);
        }
        CloseClipboard();
    }
}
```

### 4.3 进程二：接收数据

```cpp
void CClipRecvDlg::OnBnClickedMyRecvBtn()
{
    // 1. 打开剪切板，成功打开返回TRUE
    if (OpenClipboard())
    {
        // 2. 判断剪切板是否可用
        if (IsClipboardFormatAvailable(CF_TEXT))
        {
            char* szRecvBuf;
            
            // 3. 从剪切板索求数据
            HANDLE hClip = GetClipboardData(CF_TEXT);
            szRecvBuf = (char*)GlobalLock(hClip);
            SetDlgItemTextA(IDC_EDIT_MYRECV, szRecvBuf);
            GlobalUnlock(hClip);
        }
        CloseClipboard();
    }
}
```

## 5. 剪切板数据格式

| 格式常量 | 描述 |
|----------|------|
| `CF_TEXT` | ANSI 文本格式 |
| `CF_UNICODETEXT` | Unicode 文本格式 |
| `CF_BITMAP` | 位图格式 |
| `CF_DIB` | 设备无关位图 |
| `CF_HDROP` | 文件列表（拖放） |

---

## 6. 注意事项

1. **互斥访问**：同一时刻只有一个程序可以打开剪切板，其他程序必须等待
2. **内存管理**：使用 `GlobalAlloc` 分配的内存由系统管理，设置到剪切板后不需要手动释放
3. **格式检查**：在读取数据前应使用 `IsClipboardFormatAvailable` 检查格式
4. **及时关闭**：操作完成后必须调用 `CloseClipboard` 释放剪切板
5. **不适合频繁通信**：剪切板主要用于用户交互，不适合高频率的进程间通信

---

## 7. 与其他 IPC 方式对比

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **剪切板** | 简单易用 | 不适合频繁通信，会干扰用户操作 | 用户主动复制/粘贴 |
| **[[4.3 油槽通信\|油槽]]** | 单向广播，简单 | 不可靠，有大小限制 | 一对多广播 |
| **[[4.4 匿名管道通讯\|匿名管道]]** | 可靠，无大小限制 | 仅限父子进程 | 父子进程通信 |
| **[[4.5 命名管道通讯\|命名管道]]** | 双向通信，可靠 | 实现较复杂 | 双向数据交换 |
| **[[4.6 VM_COPYDATA\|WM_COPYDATA]]** | 简单，系统自动复制数据 | 需要窗口，同步阻塞 | 窗口间简单消息 |
| **共享内存** | 最快 | 需要同步机制 | 大数据量传输 |

---

## 8. 关键要点

1. 剪切板是一种简单但不常用的 IPC 方式
2. 使用前必须调用 `OpenClipboard()`，使用后必须调用 `CloseClipboard()`
3. 写入数据前需要 `EmptyClipboard()` 清空
4. 使用 `GlobalAlloc` + `GlobalLock` 进行内存操作
5. 剪切板适合简单的数据共享，不适合频繁或大量的数据传输
