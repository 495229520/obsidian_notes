# 命名管道通讯 (Named Pipe IPC)

## 1. 简介

命名管道（Named Pipe）是 Windows 提供的一种强大的[[4.1 进程简介|进程]]间通信机制，支持**任意进程**之间的**双向通信**，甚至可以跨网络通信。

### 1.1 核心特点

| 特性 | 描述 |
|------|------|
| **通信方向** | 支持双向通信（全双工），也可配置为单向 |
| **通信对象** | 任意进程，不限于父子进程 |
| **网络支持** | 支持跨网络通信 |
| **可靠性** | 可靠通信，数据不会丢失 |
| **通信模式** | 支持字节流模式和消息模式 |
| **命名** | 有唯一名称，任何知道名称的进程都可以连接 |

### 1.2 与匿名管道对比

| 特性 | 命名管道 | [[4.4 匿名管道通讯\|匿名管道]] |
|------|----------|----------|
| **命名** | 有名称 | 无名称 |
| **通信方向** | 双向 | 单向 |
| **进程关系** | 任意进程 | 仅父子进程 |
| **网络通信** | 支持 | 不支持 |
| **使用场景** | 客户端-服务器模式 | 简单的父子通信 |

## 2. 命名管道的工作模型

```
┌────────────────┐                      ┌────────────────┐
│    服务端      │                      │    客户端      │
├────────────────┤                      ├────────────────┤
│                │                      │                │
│ CreateNamed-   │     \\.\pipe\name    │                │
│    Pipe()      │◄────────────────────►│  CreateFile()  │
│                │                      │                │
│ ConnectNamed-  │      等待连接        │                │
│    Pipe()      │◄─────────────────────│                │
│                │                      │                │
│ ReadFile() /   │     双向数据流       │ WriteFile() /  │
│ WriteFile()    │◄────────────────────►│ ReadFile()     │
│                │                      │                │
│ Disconnect-    │                      │                │
│ NamedPipe()    │                      │ CloseHandle()  │
└────────────────┘                      └────────────────┘
```

## 3. 命名规则

命名管道使用特定的命名格式：

```
\\.\pipe\pipename          // 本地管道
\\servername\pipe\pipename // 远程管道（网络）
```

| 格式 | 说明 | 示例 |
|------|------|------|
| `\\\\.\\pipe\\name` | 本地命名管道 | `\\\\.\\pipe\\MyPipe` |
| `\\\\servername\\pipe\\name` | 远程服务器的管道 | `\\\\Server1\\pipe\\MyPipe` |

> **注意**：`pipename` 不区分大小写，但建议保持一致。

## 4. 关键 API 函数

### 4.1 服务端函数

| 函数 | 作用 | 说明 |
|------|------|------|
| `CreateNamedPipe()` | 创建命名管道 | 指定管道名称、访问模式、缓冲区大小等 |
| `ConnectNamedPipe()` | 等待客户端连接 | 阻塞直到有客户端连接 |
| `DisconnectNamedPipe()` | 断开客户端连接 | 准备接受新的客户端 |
| `ReadFile()` | 从管道读取数据 | 读取客户端发送的数据 |
| `WriteFile()` | 向管道写入数据 | 发送数据给客户端 |

### 4.2 客户端函数

| 函数 | 作用 | 说明 |
|------|------|------|
| `CreateFile()` | 打开已存在的命名管道 | 连接到服务端创建的管道 |
| `WaitNamedPipe()` | 等待管道可用 | 当服务端繁忙时等待 |
| `ReadFile()` | 从管道读取数据 | 读取服务端发送的数据 |
| `WriteFile()` | 向管道写入数据 | 发送数据给服务端 |

## 5. CreateNamedPipe 函数详解

```cpp
HANDLE CreateNamedPipe(
    LPCTSTR lpName,                // 管道名称
    DWORD   dwOpenMode,            // 打开模式
    DWORD   dwPipeMode,            // 管道模式
    DWORD   nMaxInstances,         // 最大实例数
    DWORD   nOutBufferSize,        // 输出缓冲区大小
    DWORD   nInBufferSize,         // 输入缓冲区大小
    DWORD   nDefaultTimeOut,       // 默认超时（毫秒）
    LPSECURITY_ATTRIBUTES lpSecurityAttributes  // 安全属性
);
```

### 5.1 打开模式 (dwOpenMode)

| 常量 | 值 | 说明 |
|------|-----|------|
| `PIPE_ACCESS_DUPLEX` | 0x03 | 双向访问（读写） |
| `PIPE_ACCESS_INBOUND` | 0x01 | 仅入站（服务端只读） |
| `PIPE_ACCESS_OUTBOUND` | 0x02 | 仅出站（服务端只写） |

### 5.2 管道模式 (dwPipeMode)

| 常量 | 说明 |
|------|------|
| `PIPE_TYPE_BYTE` | 字节流模式（数据作为字节流写入） |
| `PIPE_TYPE_MESSAGE` | 消息模式（数据作为消息单元写入） |
| `PIPE_READMODE_BYTE` | 以字节流方式读取 |
| `PIPE_READMODE_MESSAGE` | 以消息方式读取 |
| `PIPE_WAIT` | 阻塞模式（默认） |
| `PIPE_NOWAIT` | 非阻塞模式 |

### 5.3 典型配置

```cpp
// 双向字节流管道
HANDLE hPipe = CreateNamedPipe(
    TEXT("\\\\.\\pipe\\MyPipe"),
    PIPE_ACCESS_DUPLEX,                      // 双向
    PIPE_TYPE_BYTE | PIPE_READMODE_BYTE | PIPE_WAIT,  // 字节流，阻塞
    1,                                       // 最大实例数
    1024,                                    // 输出缓冲区
    1024,                                    // 输入缓冲区
    NMPWAIT_USE_DEFAULT_WAIT,               // 默认超时
    NULL                                     // 默认安全属性
);
```

## 6. 代码示例

### 6.1 服务端：创建管道并与客户端通信

```cpp
#include <windows.h>
#include <stdio.h>

#define PIPE_NAME TEXT("\\\\.\\pipe\\MyTestPipe")
#define BUFFER_SIZE 1024

int main()
{
    printf("[Server] 服务端启动...\n");

    // ========== 1. 创建命名管道 ==========
    HANDLE hNamedPipe = CreateNamedPipe(
        PIPE_NAME,
        PIPE_ACCESS_DUPLEX,              // 双向访问
        PIPE_TYPE_BYTE | PIPE_READMODE_BYTE | PIPE_WAIT,
        1,                               // 最大实例数
        BUFFER_SIZE,                     // 输出缓冲区
        BUFFER_SIZE,                     // 输入缓冲区
        NMPWAIT_USE_DEFAULT_WAIT,        // 默认超时
        NULL
    );

    if (hNamedPipe == INVALID_HANDLE_VALUE)
    {
        printf("[Server] CreateNamedPipe 失败，错误码: %d\n", GetLastError());
        return -1;
    }
    printf("[Server] 管道创建成功，等待客户端连接...\n");

    // ========== 2. 等待客户端连接 ==========
    // ConnectNamedPipe 会阻塞，直到有客户端连接
    if (!ConnectNamedPipe(hNamedPipe, NULL))
    {
        // 注意：如果客户端在 ConnectNamedPipe 之前已连接，
        // 会返回 FALSE，但 GetLastError() == ERROR_PIPE_CONNECTED
        if (GetLastError() != ERROR_PIPE_CONNECTED)
        {
            printf("[Server] ConnectNamedPipe 失败，错误码: %d\n", GetLastError());
            CloseHandle(hNamedPipe);
            return -1;
        }
    }
    printf("[Server] 客户端已连接！\n");

    // ========== 3. 从客户端读取数据 ==========
    char szBuffer[BUFFER_SIZE] = { 0 };
    DWORD dwRead;

    if (ReadFile(hNamedPipe, szBuffer, BUFFER_SIZE - 1, &dwRead, NULL))
    {
        printf("[Server] 收到数据: %s\n", szBuffer);
        printf("[Server] 字节数: %d\n", dwRead);
    }
    else
    {
        printf("[Server] ReadFile 失败，错误码: %d\n", GetLastError());
    }

    // ========== 4. 向客户端发送响应 ==========
    char szResponse[] = "This is Named Pipe from Server!";
    DWORD dwWritten;

    if (WriteFile(hNamedPipe, szResponse, strlen(szResponse) + 1, &dwWritten, NULL))
    {
        printf("[Server] 发送响应成功，字节数: %d\n", dwWritten);
    }
    else
    {
        printf("[Server] WriteFile 失败，错误码: %d\n", GetLastError());
    }

    // ========== 5. 断开连接并清理 ==========
    DisconnectNamedPipe(hNamedPipe);
    CloseHandle(hNamedPipe);

    printf("[Server] 服务端结束\n");
    system("pause");
    return 0;
}
```

### 6.2 客户端：连接管道并通信

```cpp
#include <windows.h>
#include <stdio.h>

#define PIPE_NAME TEXT("\\\\.\\pipe\\MyTestPipe")
#define BUFFER_SIZE 1024

int main()
{
    printf("[Client] 客户端启动...\n");

    // ========== 1. 等待管道可用（可选）==========
    // 如果服务端尚未准备好，可以等待
    if (!WaitNamedPipe(PIPE_NAME, NMPWAIT_WAIT_FOREVER))
    {
        printf("[Client] WaitNamedPipe 失败，错误码: %d\n", GetLastError());
        return -1;
    }

    // ========== 2. 打开命名管道 ==========
    HANDLE hNamedPipe = CreateFile(
        PIPE_NAME,
        GENERIC_READ | GENERIC_WRITE,    // 读写权限
        0,                               // 不共享
        NULL,                            // 默认安全属性
        OPEN_EXISTING,                   // 打开已存在的管道
        FILE_ATTRIBUTE_NORMAL,           // 普通属性
        NULL                             // 无模板
    );

    if (hNamedPipe == INVALID_HANDLE_VALUE)
    {
        printf("[Client] CreateFile 失败，错误码: %d\n", GetLastError());
        return -1;
    }
    printf("[Client] 连接成功！\n");

    // ========== 3. 向服务端发送数据 ==========
    char szMessage[] = "Hello from Client!";
    DWORD dwWritten;

    if (WriteFile(hNamedPipe, szMessage, strlen(szMessage) + 1, &dwWritten, NULL))
    {
        printf("[Client] 发送数据成功，字节数: %d\n", dwWritten);
    }
    else
    {
        printf("[Client] WriteFile 失败，错误码: %d\n", GetLastError());
    }

    // ========== 4. 从服务端读取响应 ==========
    char szBuffer[BUFFER_SIZE] = { 0 };
    DWORD dwRead;

    if (ReadFile(hNamedPipe, szBuffer, BUFFER_SIZE - 1, &dwRead, NULL))
    {
        printf("[Client] 收到响应: %s\n", szBuffer);
        printf("[Client] 字节数: %d\n", dwRead);
    }
    else
    {
        printf("[Client] ReadFile 失败，错误码: %d\n", GetLastError());
    }

    // ========== 5. 关闭句柄 ==========
    CloseHandle(hNamedPipe);

    printf("[Client] 客户端结束\n");
    system("pause");
    return 0;
}
```

## 7. 多客户端支持

命名管道可以支持多个客户端同时连接，需要：

1. **设置 nMaxInstances > 1** 或使用 `PIPE_UNLIMITED_INSTANCES`
2. **为每个客户端创建新的管道实例**
3. **使用多线程处理多个客户端**

### 7.1 多客户端服务器示例框架

```cpp
while (TRUE)
{
    // 创建新的管道实例
    HANDLE hPipe = CreateNamedPipe(
        PIPE_NAME,
        PIPE_ACCESS_DUPLEX,
        PIPE_TYPE_BYTE | PIPE_READMODE_BYTE | PIPE_WAIT,
        PIPE_UNLIMITED_INSTANCES,  // 无限实例
        BUFFER_SIZE,
        BUFFER_SIZE,
        0,
        NULL
    );

    // 等待客户端连接
    if (ConnectNamedPipe(hPipe, NULL) || GetLastError() == ERROR_PIPE_CONNECTED)
    {
        // 创建线程处理该客户端
        CreateThread(NULL, 0, ClientHandler, hPipe, 0, NULL);
    }
    else
    {
        CloseHandle(hPipe);
    }
}
```

## 8. 通信模式详解

### 8.1 字节流模式 vs 消息模式

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| **字节流模式** | 数据作为连续字节流传输，无消息边界 | 大量数据传输、文件传输 |
| **消息模式** | 保留消息边界，每次写入是一个独立消息 | 命令/响应式通信 |

### 8.2 消息模式示例

```cpp
// 服务端使用消息模式
HANDLE hPipe = CreateNamedPipe(
    PIPE_NAME,
    PIPE_ACCESS_DUPLEX,
    PIPE_TYPE_MESSAGE | PIPE_READMODE_MESSAGE | PIPE_WAIT,
    1, 1024, 1024, 0, NULL
);

// 客户端连接后设置为消息模式
DWORD dwMode = PIPE_READMODE_MESSAGE;
SetNamedPipeHandleState(hPipe, &dwMode, NULL, NULL);
```

## 9. 与其他 IPC 方式对比

| 方式 | 通信方向 | 进程关系 | 网络支持 | 可靠性 | 适用场景 |
|------|----------|----------|----------|--------|----------|

## 10. 注意事项

### 10.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 客户端连接失败 | 服务端未创建管道 | 使用 `WaitNamedPipe()` 等待 |
| ConnectNamedPipe 返回 FALSE | 客户端已连接 | 检查 `GetLastError() == ERROR_PIPE_CONNECTED` |
| 数据读取不完整 | 缓冲区太小 | 增大缓冲区或循环读取 |
| 无法跨网络通信 | 防火墙阻止 | 开放相应端口 |

### 10.2 最佳实践

1. **始终检查返回值和错误码**
2. **使用 WaitNamedPipe 确保管道可用**
3. **及时关闭不需要的句柄**
4. **多客户端场景使用多线程**
5. **设置合理的超时时间**

---

## 11. 关键要点

1. **命名管道**支持任意进程间的双向通信，可跨网络
2. 管道名称格式：`\\.\pipe\pipename`（本地）或 `\\servername\pipe\pipename`（远程）
3. **服务端**使用 `CreateNamedPipe()` 创建，`ConnectNamedPipe()` 等待连接
4. **客户端**使用 `CreateFile()` 连接管道
5. 支持**字节流模式**和**消息模式**两种通信模式
6. 是 Windows 最常用、最灵活的 IPC 方式之一
