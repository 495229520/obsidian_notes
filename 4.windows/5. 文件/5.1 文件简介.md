# 5.1 文件简介

本节介绍 Windows 环境下的文件操作方式，涵盖 C 语言标准库、C++ 标准库、Windows API 和 MFC 四种不同层次的实现方法。

---

## 1. 文件打开模式

| 模式 | 文件存在 | 文件不存在 | 说明 |
|:---:|:---:|:---:|:---|
| `r` | 打开文件 | 打开失败 | 只读，从文件开头读取 |
| `w` | 清空内容 | 创建新文件 | 只写，会覆盖已有内容 |
| `a` | 追加写入 | 创建新文件 | 追加模式，在文件末尾写入 |
| `r+` | 打开文件 | 打开失败 | 读写，从文件开头开始 |
| `w+` | 清空内容 | 创建新文件 | 读写，清空原有内容 |
| `a+` | 打开文件 | 创建新文件 | 读写，追加模式 |

> **提示**：在 Windows 下，可以添加 `b` 后缀表示二进制模式（如 `rb`、`wb`），避免换行符转换问题。

## 2. C 语言文件操作

使用标准库 `<stdio.h>` 提供的函数。

### 2.1 写文件

```cpp
void CMFCFileDlg::OnBnClickedWriteFile()
{
    // C语言写文件
    FILE* pFile = fopen("1.txt", "w");
    if (pFile == NULL)
    {
        MessageBox("打开失败");
        return;
    }

    char szBuf[1024] = "C语言文件操作";
    int iLen = strlen(szBuf) + 1;
    if (fwrite(szBuf, 1, strlen(szBuf) + 1, pFile) <= 0)
    {
        MessageBox("写入失败");
        return;
    }
    fclose(pFile);
}
```

### 2.2 读文件

```cpp
void CMFCFileDlg::OnBnClickedReadFile()
{
    FILE* pFile = fopen("1.txt", "r");
    if (pFile == NULL)
    {
        MessageBox("打开失败");
        return;
    }

    char szBuf[1024] = { 0 };

    // fseek函数: 设置文件的偏移量
    fseek(pFile, 0, SEEK_END);

    // 得到文件指针的当前位置
    int iFileLen = ftell(pFile);
    fseek(pFile, 0, SEEK_SET);

    // fread按照偏移量当前读文件到缓冲区的大小
    int iLen = fread(szBuf, 1, iFileLen, pFile);
    fclose(pFile);
    MessageBox(szBuf);
}
```

### C 语言文件函数总结

| 函数 | 说明 |
|:---|:---|
| `fopen()` | 打开文件 |
| `fclose()` | 关闭文件 |
| `fread()` | 读取数据 |
| `fwrite()` | 写入数据 |
| `fseek()` | 设置文件位置指针 |
| `ftell()` | 获取当前文件位置 |
| `rewind()` | 将文件位置重置到开头 |
| `feof()` | 检测是否到达文件末尾 |

## 3. C++ 文件操作

使用 `<fstream>` 库提供的流对象。

### 3.1 写文件

```cpp
void CMFCFileDlg::OnBnClickedWriteFile()
{
    // C++ 写文件
    ofstream ofs("2.txt");
    char szBuf[1024] = "C++操作文件";
    ofs.write("C++ 操作文件", strlen(szBuf));
    ofs.close();
}
```

### 3.2 读文件

```cpp
void CMFCFileDlg::OnBnClickedReadFile()
{
    // C++ 读文件
    ifstream ifs("2.txt");
    char szBuf[1024] = "C++操作文件";
    ifs.read(szBuf, 1024);
    ifs.close();
    MessageBox(szBuf);
}
```

### C++ 文件流类

| 类 | 说明 |
|:---|:---|
| `ifstream` | 输入文件流（读文件） |
| `ofstream` | 输出文件流（写文件） |
| `fstream` | 读写文件流 |

## 4. Windows API 文件操作

使用 Win32 API 进行底层文件操作，提供更细粒度的控制。

### 4.1 写文件

```cpp
void CMFCFileDlg::OnBnClickedWriteFile()
{
    // window API 写文件
    DWORD dwWrites;
    char szBuf[1024] = "WIN API 操作文件";

    HANDLE hFile = CreateFile("3.txt", GENERIC_WRITE, NULL, NULL,
                              OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, NULL);
    if (hFile == INVALID_HANDLE_VALUE)
    {
        MessageBox("创建文件对象失败");
        return;
    }

    WriteFile(hFile, szBuf, strlen(szBuf), &dwWrites, NULL);
    TRACE("###dwWrites = %d\n", dwWrites);
    CloseHandle(hFile);
}
```

### 4.2 读文件

```cpp
void CMFCFileDlg::OnBnClickedReadFile()
{
    // window API 读文件
    HANDLE hFile;
    hFile = CreateFile("3.txt", GENERIC_READ, NULL, NULL,
                       OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, NULL);
    if (hFile == INVALID_HANDLE_VALUE)
    {
        TRACE("###errorNum = %d\n", GetLastError());
        MessageBox("创建文件对象失败");
        return;
    }

    // 读文件
    DWORD dwReads;
    char szBuf[1024] = {0};
    ReadFile(hFile, szBuf, 1024, &dwReads, NULL);
    TRACE("###dwReads = %d\n", dwReads);
    CloseHandle(hFile);
    MessageBox(szBuf);
}
```

### CreateFile 参数说明

| 参数 | 说明 |
|:---|:---|
| `lpFileName` | 文件名/路径 |
| `dwDesiredAccess` | 访问权限（`GENERIC_READ`/`GENERIC_WRITE`） |
| `dwShareMode` | 共享模式 |
| `lpSecurityAttributes` | 安全属性 |
| `dwCreationDisposition` | 创建方式（`CREATE_NEW`/`OPEN_EXISTING` 等） |
| `dwFlagsAndAttributes` | 文件属性 |
| `hTemplateFile` | 模板文件句柄 |

## 5. MFC 文件操作

MFC 封装了 Windows API，提供更简洁的 C++ 接口。

### 5.1 写文件

```cpp
void CMFCFileDlg::OnBnClickedWriteFile()
{
    // MFC写文件
    CFile file("4.txt", CFile::modeCreate | CFile::modeWrite);
    char szBuf[1024] = "MFC操作文件";
    file.Write("C++ 操作文件", strlen(szBuf));
    file.Close();
}
```

### CFile 打开模式

| 模式 | 说明 |
|:---|:---|
| `CFile::modeRead` | 只读 |
| `CFile::modeWrite` | 只写 |
| `CFile::modeReadWrite` | 读写 |
| `CFile::modeCreate` | 创建新文件 |
| `CFile::modeNoTruncate` | 不截断已存在的文件 |

## 6. 四种方式对比

| 方式 | 优点 | 缺点 | 适用场景 |
|:---|:---|:---|:---|
| C 标准库 | 跨平台、简单 | 功能有限 | 简单文件读写、跨平台项目 |
| C++ 流 | 面向对象、类型安全 | 性能略低 | C++ 项目、格式化 I/O |
| Win32 API | 功能强大、性能好 | 代码复杂、仅 Windows | 需要高级控制的场景 |
| MFC | 封装好、易使用 | 依赖 MFC 库 | MFC 应用程序 |

---

## 7. 关键要点

1. **C 语言**：使用 `fopen`/`fclose`/`fread`/`fwrite`，跨平台兼容
2. **C++**：使用 `ifstream`/`ofstream`，面向对象风格
3. **Win32 API**：使用 `CreateFile`/`ReadFile`/`WriteFile`，提供底层控制
4. **MFC**：使用 `CFile` 和 `CFileDialog`，简化 Windows 开发
5. **注意**：操作完成后务必关闭文件句柄，避免资源泄漏
