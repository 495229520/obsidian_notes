# 5.2 读写配置文件和注册表

本节介绍 Windows 程序中两种常见的配置数据存储方式：INI 配置文件和 Windows 注册表。

---

## 1. INI 配置文件

INI 文件是一种简单的文本配置文件格式，由节（Section）、键（Key）和值（Value）组成。

### 1.1 INI 文件格式

```ini
[Param1]
QueryInterval=3600
CheckInterval=4000
PopupInterval=3000

[Param2]
ServerIP=192.168.1.100
Port=8080
```

### 1.2 写配置文件

使用 `WritePrivateProfileString()` 函数写入 INI 文件：

```cpp
void CMFCFileDlg::OnBnClickedWriteFile()
{
    // 写配置文件
    char szPath[MAX_PATH] = { 0 };
    GetCurrentDirectory(MAX_PATH, szPath);

    char szMyPath[MAX_PATH] = { 0 };
    sprintf(szMyPath, "%s\\Test.ini", szPath);

    // 参数: 节名, 键名, 键值, 文件路径
    WritePrivateProfileString("Param1", "QueryInterval", "3600", szMyPath);
    WritePrivateProfileString("Param2", "CheckInterval", "4000", szMyPath);
    WritePrivateProfileString("Param3", "PopupInterval", "3000", szMyPath);
}
```

### 1.3 读配置文件

使用 `GetPrivateProfileString()` 函数读取 INI 文件：

```cpp
void CMFCFileDlg::OnBnClickedReadFile()
{
    // 读配置文件
    char szPath[MAX_PATH] = { 0 };
    GetCurrentDirectory(MAX_PATH, szPath);

    char szMyPath[MAX_PATH] = { 0 };
    sprintf(szMyPath, "%s\\Test.ini", szPath);

    char str1[1024] = { 0 };
    char str2[1024] = { 0 };
    char str3[1024] = { 0 };

    // 参数: 节名, 键名, 默认值, 缓冲区, 缓冲区大小, 文件路径
    GetPrivateProfileString("Param1", "QueryInterval", NULL, str1, 1024, szMyPath);
    GetPrivateProfileString("Param2", "CheckInterval", NULL, str2, 1024, szMyPath);
    GetPrivateProfileString("Param3", "PopupInterval", NULL, str3, 1024, szMyPath);

    CString strShow;
    strShow.Format("QueryInterval = %s CheckInterval = %s PopupInterval = %s",
                   str1, str2, str3);
    MessageBox(strShow);
}
```

### 1.4 INI 文件操作函数

| 函数 | 说明 |
|:---|:---|
| `WritePrivateProfileString()` | 写入字符串值 |
| `GetPrivateProfileString()` | 读取字符串值 |
| `WritePrivateProfileInt()` | 写入整数值 |
| `GetPrivateProfileInt()` | 读取整数值 |
| `WritePrivateProfileSection()` | 写入整个节 |
| `GetPrivateProfileSection()` | 读取整个节 |

## 2. Windows 注册表

Windows 注册表是一个层次化的数据库，用于存储系统和应用程序的配置信息。更详细的注册表操作可参见 [[2.5 隐藏进程、修改注册表#注册表操作|注册表操作详解]]。

### 2.1 注册表结构

```
HKEY_LOCAL_MACHINE
├── SOFTWARE
│   └── MYWEIGHT
│       └── amdin
│           └── weight = 80 (REG_DWORD)
```

### 2.2 注册表根键

| 根键 | 说明 |
|:---|:---|
| `HKEY_LOCAL_MACHINE` (HKLM) | 本地计算机配置 |
| `HKEY_CURRENT_USER` (HKCU) | 当前用户配置 |
| `HKEY_CLASSES_ROOT` (HKCR) | 文件关联和 COM 信息 |
| `HKEY_USERS` (HKU) | 所有用户的配置 |
| `HKEY_CURRENT_CONFIG` | 当前硬件配置 |

### 2.3 写注册表

```cpp
void CMFCFileDlg::OnBnClickedWriteFile()
{
    // 写注册表
    HKEY hKey;
    DWORD dwWeight = 80;

    // 创建注册表键
    DWORD dwRet = ::RegCreateKey(HKEY_LOCAL_MACHINE,
                                  "SOFTWARE\\MYWEIGHT\\amdin", &hKey);
    if (dwRet != ERROR_SUCCESS)
    {
        MessageBox("创建注册表失败");
        return;
    }

    // 写注册表值
    dwRet = ::RegSetValueEx(hKey, "weight", NULL, REG_DWORD,
                            (CONST BYTE*)&dwWeight, 4);
    if (dwRet != ERROR_SUCCESS)
    {
        MessageBox("写注册表失败");
        return;
    }

    // 关闭注册表
    ::RegCloseKey(hKey);
}
```

### 2.4 读注册表

```cpp
void CMFCFileDlg::OnBnClickedReadFile()
{
    // 读注册表
    HKEY hKey;
    DWORD dwWeight;
    DWORD dwType;
    DWORD dwSize;

    // 打开注册表键
    DWORD dwRet = ::RegOpenKey(HKEY_LOCAL_MACHINE,
                                "SOFTWARE\\MYWEIGHT\\amdin", &hKey);
    if (dwRet != ERROR_SUCCESS)
    {
        MessageBox("打开注册表失败");
        return;
    }

    // 读注册表值
    dwRet = ::RegQueryValueEx(hKey, "weight", NULL, &dwType,
                              (LPBYTE)&dwWeight, &dwSize);
    if (dwRet != ERROR_SUCCESS)
    {
        MessageBox("读注册表失败");
        return;
    }

    CString strShow;
    strShow.Format("Weight = %d", dwWeight);
    MessageBox(strShow);

    // 关闭注册表
    ::RegCloseKey(hKey);
}
```

### 2.5 注册表值类型

| 类型 | 说明 |
|:---|:---|
| `REG_SZ` | 以 null 结尾的字符串 |
| `REG_DWORD` | 32 位整数 |
| `REG_BINARY` | 二进制数据 |
| `REG_MULTI_SZ` | 多字符串数组 |
| `REG_EXPAND_SZ` | 可展开环境变量的字符串 |
| `REG_QWORD` | 64 位整数 |

### 2.6 注册表操作函数

| 函数 | 说明 |
|:---|:---|
| `RegCreateKey()` / `RegCreateKeyEx()` | 创建键 |
| `RegOpenKey()` / `RegOpenKeyEx()` | 打开键 |
| `RegSetValueEx()` | 设置值 |
| `RegQueryValueEx()` | 查询值 |
| `RegDeleteKey()` | 删除键 |
| `RegDeleteValue()` | 删除值 |
| `RegCloseKey()` | 关闭键句柄 |

## 3. INI 文件 vs 注册表

| 特性 | INI 文件 | 注册表 |
|:---|:---|:---|
| **存储位置** | 文件系统（可自定义路径） | 系统注册表数据库 |
| **可移植性** | 高（可随程序分发） | 低（绑定到系统） |
| **编辑方式** | 可用文本编辑器 | 需要 regedit 或 API |
| **权限控制** | 文件系统权限 | 注册表权限（更细粒度） |
| **性能** | 需要文件 I/O | 内存映射，较快 |
| **安全性** | 相对较低 | 可设置访问控制 |
| **适用场景** | 简单配置、便携程序 | 系统集成、安全敏感数据 |

---

## 4. 关键要点

1. **INI 文件**：使用 `WritePrivateProfileString()` 和 `GetPrivateProfileString()` 操作
2. **注册表**：使用 `RegCreateKey()`、`RegSetValueEx()`、`RegQueryValueEx()` 等函数
3. **注册表权限**：修改 `HKEY_LOCAL_MACHINE` 需要管理员权限
4. **资源释放**：操作完成后务必调用 `RegCloseKey()` 关闭句柄
5. **选择建议**：便携程序优先使用 INI 文件，需要系统集成的应用考虑注册表
