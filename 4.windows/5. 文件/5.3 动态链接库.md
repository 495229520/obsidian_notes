# 5.3 动态链接库

动态链接库（DLL, Dynamic Link Library）是 Windows 系统中实现代码重用和模块化的核心机制。本节详细介绍 DLL 的概念、创建方法和调用方式。
阅读次数: 0
---

## 1. 静态库与动态库

- **静态库**：在[[1.1 程序编译与链接原理#静态链接|编译/链接时]]将库代码完整复制到可执行文件中
- **动态库**：在程序**运行时**才加载库代码到内存

---

## 2. DLL 的优点

DLL 的主要优势在于促进**模块化**、**代码重用**和**内存的高效使用**。

### 2.1 代码重用和共享

多个应用程序可以共享磁盘上的**单个 DLL 副本**。例如，Windows 系统中的许多核心功能（如文件操作、对话框等）都封装在系统 DLL 中，所有程序都可以调用它们。

### 2.2 模块化开发

它允许将大型程序分解成更小、更易于管理的模块。不同团队可以分别开发和更新这些模块。

### 2.3 减少可执行文件大小

由于共享的函数和代码不需要被复制到每个应用程序的可执行文件（.exe）中，因此应用程序的 .exe 文件会更小。

### 2.4 简化部署和更新

当 DLL 中的函数需要更新或修复错误时，只需要替换 DLL 文件本身，**而不需要重新编译或重新链接**所有使用它的应用程序（前提是导出的接口没有改变）。

### 2.5 内存效率

当多个程序同时使用同一个 DLL 时，操作系统只需要将该 DLL 的代码加载到内存中**一次**，供所有进程共享，从而节省了物理内存。

---

## 3. DLL vs 静态库 (LIB) 对比

| 特性 | 动态链接库 (DLL) | 静态链接库 (LIB) |
|:---|:---|:---|
| **文件扩展名** | `.dll` | `.lib`（用于编译/链接时引用） |
| **链接时间** | 运行时（程序启动或运行时加载） | 编译/链接时（代码被复制到 EXE 中） |
| **代码位置** | 在磁盘上保持独立，程序运行时加载 | 被直接嵌入到最终的可执行文件 (.exe) 中 |
| **更新/维护** | 容易，只需替换 DLL 文件 | 困难，需要重新编译和发布整个 .exe |
| **程序大小** | 可执行文件较小 | 可执行文件较大（因为代码被复制了） |
| **共享性** | 可被多个程序共享内存中的副本 | 每个程序都有自己的一份代码副本 |

## 4. 创建 DLL

### 4.1 创建 DLL 项目

1. 启动 Visual Studio，选择"创建新项目"
2. 在模板中搜索并选择"动态链接库 (DLL)"模板（通常是 C++ 下的 Windows 桌面类型）
3. 命名你的项目（例如：MySimpleDLL）并设置好位置

### 4.2 编写 DLL 代码并定义导出接口

DLL 的核心在于将某些函数或类**导出 (Export)**，以便外部的 .exe 程序能够调用它们。

在 Windows C++ 中，最常用的导出方法是使用 Microsoft 特定的关键字：`__declspec(dllexport)`。

```cpp
// dllmain.cpp
extern "C" __declspec(dllexport)
int ave(int a, int b)
{
    return (a + b) / 2;
}

int ave2(int a)
{
    return a / 2;
}
```

> **注意**：
> - `__declspec(dllexport)` 标记函数为导出函数
> - `extern "C"` 禁用 C++ 的[[11.2 编译器基本概念#名称修饰|名称修饰 (Name Mangling)]]，使函数名保持原样

### 4.3 使用 .def 文件定义导出

除了使用 `__declspec(dllexport)`，还可以使用模块定义文件 (.def) 来指定导出函数：

```def
LIBRARY
EXPORTS
    ave
```

编译后会生成 `.dll` 文件和对应的 `.lib` 导入库文件。

## 5. 调用 DLL 的两种方式

### 5.1 隐式链接（静态加载）

在编译时指定要链接的 DLL，程序启动时自动加载。

```cpp
// dllmain.cpp：定义 DLL 应用程序的入口点
#include "pch.h"
#include <Windows.h>

// 入口函数
BOOL APIENTRY DllMain(HMODULE hModule, DWORD ul_reason_for_call,
                      LPVOID lpReserved)
{
    switch (ul_reason_for_call)
    {
    case DLL_PROCESS_ATTACH:  // DLL 被加载
    case DLL_THREAD_ATTACH:   // 线程创建
    case DLL_THREAD_DETACH:   // 线程退出
    case DLL_PROCESS_DETACH:  // DLL 被卸载
        break;
    }
    return TRUE;
}

// C风格，数据导出宏
extern "C" __declspec(dllexport) int ave(int a, int b)
{
    return (a + b) / 2;
}

extern "C" __declspec(dllexport) int ave_int(int a, int b)
{
    return (a + b) / 2;
}
```

**使用隐式链接调用 DLL：**

```cpp
// 方法1: 使用 #pragma comment
#pragma comment(lib, "/import-ave-lan@@MyDll")

// 方法2: 在项目属性中添加库依赖

// 声明导入函数
extern "C" __declspec(dllimport) int ave(int a, int b);

// 调用
int result = ave(10, 20);
```

### 5.2 显式链接（动态加载）

在运行时手动加载 DLL 并获取函数地址。

```cpp
#include <Windows.h>
#include <iostream>
using namespace std;

// 定义函数指针类型
typedef int (*pFunc)(int, int);

int main()
{
    // 加载配置读取类型的字符串
    HMODULE hMod = LoadLibrary("7.dll");
    if (!hMod)
    {
        cout << "加载失败" << endl;
        return -1;
    }

    // 获取函数地址使用函数指针
    pFunc pFunc = (pFunc)GetProcAddress(hMod, "ave");
    if (!pFunc)
    {
        cout << "获取函数地址失败" << endl;
        FreeLibrary(hMod);
        return -1;
    }

    // 调用
    int ret = pFunc(100, 200);
    cout << "结果: " << ret << endl;

    // 释放 DLL
    FreeLibrary(hMod);
    return 0;
}
```

### 5.3 两种链接方式对比

| 特性 | 隐式链接 | 显式链接 |
|:---|:---|:---|
| **加载时机** | 程序启动时自动加载 | 运行时按需加载 |
| **DLL 缺失** | 程序无法启动 | 可以优雅处理错误 |
| **灵活性** | 较低 | 高，可动态选择 DLL |
| **使用复杂度** | 简单 | 需要手动管理 |
| **适用场景** | 必需的核心功能 | 可选插件、热更新 |

## 6. 调用约定与名称修饰

### 6.1 __stdcall 调用约定

在 Windows API 中，`__stdcall` 是常用的调用约定：

```cpp
extern "C" __declspec(dllexport) int __stdcall ave(int a, int b)
{
    return (a + b) / 2;
}
```

使用 `__stdcall` 时，Microsoft Visual C++ 会对导出的函数名进行修饰，添加前缀 `_` 和后缀 `@参数字节数`。

例如：`int __stdcall ave(int a, int b)` 会被修饰为 `_ave@8`。

### 6.2 名称修饰问题

由于 [[11.2 编译器基本概念#名称修饰|Name Mangling]]，不同编译器和调用约定会产生不同的符号名：

| 调用方式 | 导出符号名 |
|:---|:---|
| `extern "C"` (默认 `__cdecl`) | `ave` |
| `extern "C" __stdcall` | `_ave@8` |
| C++ (无 `extern "C"`) | `?ave@@YAHHH@Z` (编译器相关) |

### 6.3 解决方案

使用 `.def` 文件可以精确控制导出的符号名：

```def
LIBRARY MyDLL
EXPORTS
    ave = _ave@8    ; 别名映射
```

或者使用 [[10.1 C和C++联调#extern "C"|extern "C"]] 禁用 C++ 名称修饰。

## 7. DLL 显式调用完整示例

```cpp
#include <Windows.h>
#include <stdio.h>

// 定义函数指针类型，匹配 DLL 中的函数签名
typedef int (*AddFunc)(int, int);

int main()
{
    // 1. 加载 DLL
    HMODULE hDll = LoadLibrary("MyMath.dll");
    if (hDll == NULL)
    {
        printf("LoadLibrary failed: %lu\n", GetLastError());
        return 1;
    }

    // 2. 获取函数地址
    AddFunc pAdd = (AddFunc)GetProcAddress(hDll, "Add");
    if (pAdd == NULL)
    {
        printf("GetProcAddress failed: %lu\n", GetLastError());
        FreeLibrary(hDll);
        return 1;
    }

    // 3. 调用函数
    int result = pAdd(10, 20);
    printf("10 + 20 = %d\n", result);

    // 4. 释放 DLL
    FreeLibrary(hDll);
    return 0;
}
```

---

## 8. 关键要点

1. **DLL 优势**：代码重用、模块化开发、内存共享、便于更新
2. **导出函数**：使用 `__declspec(dllexport)` 或 `.def` 文件
3. **隐式链接**：编译时链接，启动时加载，简单但不灵活
4. **显式链接**：运行时加载（`LoadLibrary` + `GetProcAddress`），灵活但代码复杂
5. **名称修饰**：使用 `extern "C"` 避免 C++ 名称修饰问题，确保跨语言兼容
6. **资源释放**：显式加载的 DLL 需要调用 `FreeLibrary()` 释放
