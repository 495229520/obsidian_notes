# Windows 网络编程基础

## 简介

**Socket（套接字）** 是网络数据传输用的软件设备，是应用层与传输层之间的接口。在 Windows 平台上，网络编程主要通过 **Winsock**（Windows Sockets）API 实现。

> Socket 可以理解为：网络中用于标识通信端点的抽象概念，它封装了网络通信的底层细节。

---

## 面向连接 vs 面向消息

网络通信可分为两种基本模式：**面向连接（TCP）** 和 **面向消息（UDP）**。

### 面向连接（TCP）

面向连接的通信模式要求在数据传输之前，通信双方必须先建立一个专门的"管道"进行传输。

**核心特点：**

| 特性 | 说明 |
|:---|:---|
| **可靠性 (Reliability)** | 保证数据能够无丢失地到达目的地；如果数据在传输过程中丢失或损坏，协议会自动进行重传 |
| **有序性 (Ordered)** | 保证数据按照发送的顺序到达接收方。发送方发送 A、B、C 个数据包，接收方收到的顺序也一定是 A、B、C |
| **建立连接 (Connection Setup)** | 数据传输前有明确的"三次握手"来建立连接；传输结束后有"四次挥手"来断开连接，确保双方都能友好地进行通信 |
| **流式传输 (Stream-Oriented)** | 数据被看作一个连续的字节流，没有明确的消息边界；接收方可以按顺序将它们拼接成一个完整的数据流 |
| **开销较大 (Higher Overhead)** | 因为需要建立连接和维护连接状态，并发送确认、重传等控制信息，所以开销比面向消息更大，速度也相对较慢 |

### 面向消息（UDP）

面向消息的通信模式不需要事先建立连接，发送方直接将数据打包成一个独立的"消息"（也称为数据报，Datagram），然后"尽力而为"（Best-Effort）地发送到目的地。每个消息都是独立路由和处理的。

**核心特点：**

| 特性 | 说明 |
|:---|:---|
| **不可靠性 (Unreliable)** | 不保证数据一定能到达目的地，消息可能会在网络中丢失，也不会自动重传 |
| **无序性 (Unordered)** | 不保证消息按照发送的顺序到达，先发送的消息可能后到达，消息也可能被重复发送 |
| **保留消息边界 (Preserves Message Boundaries)** | 这是与流式传输最大的不同。发送方发送一个大的数据包，接收方就会收到一个大的数据包。例如，发送方发送了一个 100 字节的消息，接收方要么收到一个完整的 100 字节消息，要么就收不到，不会出现只收到 50 字节的情况。这使得数据包理解更简单 |
| **无需建立连接 (Connectionless)** | 发送数据前不需要任何握手过程，直接发送即可 |
| **开销小，速度快 (Low Overhead, Fast)** | 由于没有建立连接和保证可靠性的复杂机制，它的协议头开销小，传输效率高，速度快 |

## TCP 与 UDP 对比

| 特性 | 面向连接(TCP) | 面向消息(UDP) |
|:---|:---|:---|
| **连接** | 必须先建立连接才能传输数据 | 无需建立连接，直接发送 |
| **可靠性** | 可靠，保证数据顺达 | 不可靠，尽力而为，可能丢包 |
| **数据顺序** | 有序，按发送顺序到达 | 无序，到达顺序不确定 |
| **传输模式** | 流模式 (Stream) - 无消息边界 | 数据报模式 (Datagram) - 保留消息边界 |
| **速度/开销** | 速度慢，开销大 | 速度快，开销小 |
| **典型应用** | 网页浏览(HTTP)、文件传输(FTP)、邮件(SMTP) | 视频直播、在线游戏、DNS、VoIP |
| **好比** | 打电话 | 寄信/发短信 |

> 关于 TCP 和 UDP 协议的深入原理，可参考 [[7.5 TCP协议基础]] 和 [[9.1 UDP原理]]。

## IP 地址

可以把 **IP 地址** 想象成你家的街道地址。

### 定义

**IP 地址（Internet Protocol Address）** 是一个分配给连接到网络上的设备（比如你的电脑、手机、服务器等）的数字标签，它是一个**逻辑地址**。

### 作用

它用于唯一地标识互联网上的每一台设备，类似于邮寄包裹时收件人的地址。

### 如何找到目标设备

如果你要发送一个包裹（数据包）到朋友家，你首先需要知道他家的地址。对吧？在网络世界中，IP 地址就扮演了这个角色，它告诉路由器（网络中的邮局）数据包应该被发送到哪台设备。

### 格式

- **IPv4**：常见的有 IPv4（例如：`192.168.1.1`，由四组数字组成）
- **IPv6**：和更新的 IPv6（例如：`2001:db8:85a3:0:0:8a2e:370:7334`）

## 端口号

如果说 IP 地址是"街道地址"，那么 **端口号** 就像你家里的某个特定房间或收件人。

### 定义

**端口号** 是一个 **16 位** 的数字（范围从 0 到 65535），用于标识一台设备上的**特定应用程序或服务**。

### 作用

一台电脑上可能同时运行着浏览器、邮件客户端、游戏等多个程序。当数据包到达这台电脑的 IP 地址后，端口号就告诉操作系统：**这个数据是给哪个应用程序的？**

### 如何找到特定程序

想象一下你家收到了一个包裹，收件人是"客厅里的游戏机"。端口号就是这个"客厅里的游戏机"，它确保数据不会被错误地送达你的邮件程序或浏览器。

### 常见端口号 (Well-known Ports)

| 端口号 | 协议 | 用途 |
|:---:|:---|:---|
| **80** | HTTP | 网页浏览 |
| **443** | HTTPS | 安全的网页浏览 |
| **21** | FTP | 文件传输 |
| **22** | SSH | 安全远程登录 |

---

## IP 地址与端口号的区别

| 概念 | 类比 | 作用 |
|:---|:---|:---|
| **IP 地址** | 街道地址 | 找到互联网上的目标设备（哪台电脑） |
| **端口号** | 房间号/收件人 | 找到目标设备上运行的目标程序（哪个应用） |

---

## 关键要点

1. **Socket** 是网络编程的基础抽象，提供了应用层与传输层之间的接口
2. **TCP** 面向连接、可靠但开销大，适合需要保证数据完整性的场景
3. **UDP** 无连接、快速但不可靠，适合实时性要求高的场景
4. **IP 地址** 定位网络中的设备，**端口号** 定位设备上的应用程序
5. Windows 网络编程使用 **Winsock** API，需要先初始化 `WSAStartup()`
