# UDP 客户端与服务端

## 通信流程概述

UDP 是无连接的协议，不需要建立连接即可直接发送数据。相比 [[2.3 TCP客户端与服务端|TCP]]，UDP 的通信流程更加简洁：

```
         UDP Server                           UDP Client
            │                                     │
      ┌─────┴─────┐                               │
      │ Socket()  │ 创建套接字                     │
      └─────┬─────┘                               │
            │                                     │
      ┌─────┴─────┐                         ┌─────┴─────┐
      │  bind()   │ 绑定本地端口             │ Socket()  │
      └─────┬─────┘                         └─────┬─────┘
            │                                     │
      ┌─────┴─────┐     一直等待直到                │
      │ Recvfrom()│←────客户连接到─────────────────┤
      └─────┬─────┘     数据（请求）         ┌─────┴─────┐
            │                               │ Sendto()  │
            │        处理请求               └─────┬─────┘
      ┌─────┴─────┐                              │      循
      │ sendto()  │─────────────────────────────→│      环
      └─────┬─────┘     数据（应答）         ┌─────┴─────┐
            │                               │ Recvfrom()│
            │  循环                         └─────┬─────┘
      ┌─────┴─────┐                         ┌────┴──────┐
      │  Close()  │                         │  Close()  │
      └───────────┘                         └───────────┘
```

> 关于 [[9.1 UDP原理|UDP 协议的详细原理]]，包括其无连接、不可靠、保留消息边界等特性，请参考 Linux 网络编程笔记。

---

## UDP 与 TCP 的主要区别

| 特性 | TCP | UDP |
|:---|:---|:---|
| **连接** | 需要三次握手建立连接 | 无需连接，直接发送 |
| **函数** | `listen()`, `accept()`, `connect()` | 不需要这些函数 |
| **数据收发** | `send()` / `recv()` | `sendto()` / `recvfrom()` |
| **套接字类型** | `SOCK_STREAM` | `SOCK_DGRAM` |
| **可靠性** | 可靠，保证数据到达 | 不可靠，可能丢包 |

## 客户端代码

```cpp
#include <stdio.h>
#include <stdlib.h>
#include <WinSock2.h>
#include <iostream>
#pragma comment(lib, "ws2_32.lib")

using namespace std;

int main()
{
    cout << "UDP Client" << endl;

    // 1. 初始化套接字库
    WORD wVersion;
    WSADATA wsaData;
    int err;

    wVersion = MAKEWORD(2, 2);
    err = WSAStartup(wVersion, &wsaData);
    if (err != 0)
    {
        return err;
    }

    // 验证版本
    if (LOBYTE(wsaData.wVersion) != 2 || HIBYTE(wsaData.wVersion) != 2)
    {
        WSACleanup();
        return -1;
    }

    // 2. 创建 UDP 套接字
    SOCKET sockCli = socket(AF_INET, SOCK_DGRAM, 0);  // SOCK_DGRAM 表示 UDP
    if (INVALID_SOCKET == sockCli)
    {
        printf("socket_error = %d\n", GetLastError());
        return -1;
    }

    // 3. 填充服务器地址和端口
    SOCKADDR_IN addrSrv;
    addrSrv.sin_addr.S_un.S_addr = inet_addr("127.0.0.1");
    addrSrv.sin_family = AF_INET;
    addrSrv.sin_port = htons(6020);

    int len = sizeof(SOCKADDR_IN);

    // 4. 发送数据
    char sendBuf[] = "hello";
    char recvBuf[] = { 0 };

    // sendto: 向指定地址发送 UDP 数据报
    sendto(sockCli, sendBuf, strlen(sendBuf) + 1, 0, (SOCKADDR*)&addrSrv, len);

    // 5. 接收服务器响应
    recvfrom(sockCli, recvBuf, 100, 0, (SOCKADDR*)&addrSrv, &len);
    cout << "****" << endl;
    cout << recvBuf << endl;

    // 6. 关闭套接字
    closesocket(sockCli);
    system("pause");

    return 0;
}
```

## 服务端代码

```cpp
#include <stdio.h>
#include <stdlib.h>
#include <WinSock2.h>
#include <iostream>
#pragma comment(lib, "ws2_32.lib")

using namespace std;

int main()
{
    cout << "UDP Server" << endl;

    // 1. 初始化套接字库
    WORD wVersion;
    WSADATA wsaData;
    int err;

    wVersion = MAKEWORD(2, 2);
    err = WSAStartup(wVersion, &wsaData);
    if (err != 0)
    {
        return err;
    }

    // 验证版本
    if (LOBYTE(wsaData.wVersion) != 2 || HIBYTE(wsaData.wVersion) != 2)
    {
        WSACleanup();
        return -1;
    }

    // 2. 创建 UDP 套接字
    SOCKET sockSrv = socket(AF_INET, SOCK_DGRAM, 0);  // SOCK_DGRAM 表示 UDP
    if (INVALID_SOCKET == sockSrv)
    {
        printf("socket_error = %d\n", GetLastError());
        return -1;
    }

    // 3. 分配地址和端口
    SOCKADDR_IN addrSrv;
    // 将服务器套接字绑定到所有可用的本地网络接口，监听所有的 IP 地址
    addrSrv.sin_addr.S_un.S_addr = htonl(INADDR_ANY);
    addrSrv.sin_family = AF_INET;
    addrSrv.sin_port = htons(6020);

    if (SOCKET_ERROR == bind(sockSrv, (SOCKADDR*)&addrSrv, sizeof(SOCKADDR_IN)))
    {
        printf("bind errorNo = %d\n", GetLastError());
        return -1;
    }

    // 4. 等待接收数据
    SOCKADDR_IN addCli;  // 用于存储发送方的地址信息
    int len = sizeof(SOCKADDR_IN);

    char recvBuf[100] = { 0 };
    char sendBuf[100] = { 0 };

    while (1)
    {
        // recvfrom: 接收数据，同时获取发送方地址
        recvfrom(sockSrv, recvBuf, 100, 0, (SOCKADDR*)&addCli, &len);
        cout << recvBuf << endl;

        // 5. 发送响应
        sprintf_s(sendBuf, 100, "Ack: %s", recvBuf);
        sendto(sockSrv, sendBuf, strlen(sendBuf) + 1, 0, (SOCKADDR*)&addCli, len);
    }

    // 6. 关闭套接字
    closesocket(sockSrv);
    system("pause");

    return 0;
}
```

## 代码解析

### 关键函数说明

| 函数 | 说明 |
|:---|:---|
| `socket(AF_INET, SOCK_DGRAM, 0)` | 创建 UDP 套接字，`SOCK_DGRAM` 表示数据报套接字 |
| `sendto()` | 发送 UDP 数据报到指定地址，每次发送都需要指定目标地址 |
| `recvfrom()` | 接收 UDP 数据报，同时获取发送方的地址信息 |
| `bind()` | 服务端绑定本地地址和端口，客户端通常不需要显式绑定 |

### 服务端与客户端的区别

| 角色 | 是否需要 bind() | 说明 |
|:---|:---|:---|
| **服务端** | 是 | 必须绑定到固定端口，以便客户端知道向哪里发送数据 |
| **客户端** | 否（通常） | 系统会自动分配一个临时端口 |

## UDP 的特点

### 优点

1. **速度快**：无需建立连接，直接发送数据
2. **开销小**：协议头小，传输效率高
3. **支持广播和多播**：可以一次向多个接收者发送数据

### 缺点

1. **不可靠**：数据可能丢失、重复或乱序
2. **无流量控制**：发送方不关心接收方是否能处理
3. **需要应用层处理可靠性**：如需保证数据完整，需自行实现重传机制

---

## 适用场景

| 场景 | 原因 |
|:---|:---|
| **视频/音频直播** | 实时性要求高，偶尔丢帧可接受 |
| **在线游戏** | 低延迟比可靠性更重要 |
| **DNS 查询** | 简单的请求-响应模式 |
| **物联网传感器数据** | 数据量小，丢失几个包影响不大 |

---

## 关键要点

1. **UDP 流程简化**：`socket()` → `bind()`（服务端） → `sendto()/recvfrom()` → `closesocket()`
2. UDP 使用 `SOCK_DGRAM` 套接字类型
3. `sendto()` 和 `recvfrom()` 每次调用都需要/返回地址信息
4. UDP 保留消息边界，发送多少就收到多少（如果没丢失的话）
5. UDP 适合实时性要求高、可以容忍少量数据丢失的场景
