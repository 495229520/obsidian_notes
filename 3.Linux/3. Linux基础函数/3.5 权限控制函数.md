# 权限控制函数

## 权限表示方式

### 符号表示法

设置权限可以使用符号或数字。建议在脚本中使用数字。

| 权限表示 | 说明             |
| ---- | -------------- |
| `r`  | 读权限 (Read)     |
| `w`  | 写权限 (Write)    |
| `x`  | 执行权限 (Execute) |
| `-`  | 无权限            |

```
r w x r - x r - x
```

### 数字表示法（八进制）

| 权限  | 数值  | 二进制 |
| --- | --- | --- |
| `r` | 4   | 100 |
| `w` | 2   | 010 |
| `x` | 1   | 001 |

权限组合示例：
- `rwx` = 4 + 2 + 1 = 7
- `rw-` = 4 + 2 + 0 = 6
- `r-x` = 4 + 0 + 1 = 5
- `r--` = 4 + 0 + 0 = 4

完整权限示例：
- `755` = `rwxr-xr-x`（所有者完全权限，组和其他用户只读和执行）
- `644` = `rw-r--r--`（所有者读写，组和其他用户只读）

## 用户与组

### 1. 用户类型 (Who)

`chmod()` 函数中，需要指定对哪些用户设置权限：

- **u (User)**：可以被翻译为文件"所有者"，指的是当前文件的拥有者
- **g (Group)**：可以被翻译为"组"或"群组"，指的是与文件拥有者同组的所有用户
- **o (Other)**：可以翻译为"其他"，指的是除所有者和同组用户之外的所有其他用户
- **a (All)**：指 u、g、o 三者，即所有人

### 2. 权限类型 (What)

在 Linux 文件系统中对文件的可访问性有以下三种权限：

| 权限 | 文件 | 目录 |
|------|------|------|
| **读取 (r)** | 可以读取文件内容 | 可以列出目录中的文件 |
| **写入 (w)** | 可以修改文件内容 | 可以在目录中创建/删除文件 |
| **执行 (x)** | 可以执行文件（如脚本或程序） | 可以进入目录 |

### 3. 操作 (Operation)

使用符号法修改权限时的操作符：

- `+`：添加权限
- `-`：移除权限
- `=`：设置精确权限（覆盖原有权限）

## UID 和 GID

### UID（用户标识符）

每个用户都有一个唯一的数字标识符（UID）：

- **UID 0**：超级用户 root
- **UID 1-999**：系统用户（服务账户）
- **UID 1000+**：普通用户

### GID（组标识符）

每个组都有一个唯一的数字标识符（GID）。用户可以属于多个组。

## /etc/passwd 文件

`/etc/passwd` 是一个记录用户账户信息的系统文件。不过这个文件并不真正存储密码，它只是存储用户的基本属性。

每行格式：
```
用户名:x:UID:GID:描述:主目录:登录Shell
```

| 字段 | 说明 |
|------|------|
| 用户名 | 登录名 |
| x | 密码占位符（实际密码在 /etc/shadow） |
| UID | 用户ID |
| GID | 主组ID |
| 描述 | 用户描述信息 |
| 主目录 | 用户主目录路径 |
| 登录Shell | 用户登录时使用的Shell |

示例：
```
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
nobody:x:65534:65534:nobody:/nonexistent:/usr/sbin/nologin
```

## getpwuid 函数

获取用户的详细信息：

```c
#include <sys/types.h>
#include <pwd.h>

struct passwd *getpwuid(uid_t uid);
```

返回的 `passwd` 结构体包含：
- `pw_name`：用户名
- `pw_uid`：用户ID
- `pw_gid`：组ID
- `pw_dir`：主目录
- `pw_shell`：登录Shell

---

## getgrgid 函数

获取组的详细信息：

```c
#include <sys/types.h>
#include <grp.h>

struct group *getgrgid(gid_t gid);
```

返回的 `group` 结构体包含：
- `gr_name`：组名
- `gr_gid`：组ID
- `gr_mem`：组成员列表

## stat 结构体

使用 `stat()` 函数可以获取文件的详细信息，包括权限：

```c
struct stat {
    dev_t     st_dev;      // 设备ID
    ino_t     st_ino;      // inode号
    mode_t    st_mode;     // 文件类型和权限
    nlink_t   st_nlink;    // 硬链接数
    uid_t     st_uid;      // 所有者UID
    gid_t     st_gid;      // 所有者GID
    dev_t     st_rdev;     // 设备ID（特殊文件）
    off_t     st_size;     // 文件大小（字节）
    blksize_t st_blksize;  // 块大小
    blkcnt_t  st_blocks;   // 块数
    time_t    st_atime;    // 最后访问时间
    time_t    st_mtime;    // 最后修改时间
    time_t    st_ctime;    // 最后状态改变时间
};
```

### st_mode 字段

`st_mode` 包含文件类型和权限信息：

**文件类型检测宏：**
- `S_ISREG(m)`：是否为普通文件
- `S_ISDIR(m)`：是否为目录
- `S_ISCHR(m)`：是否为字符设备
- `S_ISBLK(m)`：是否为块设备
- `S_ISFIFO(m)`：是否为FIFO（命名管道）
- `S_ISLNK(m)`：是否为符号链接
- `S_ISSOCK(m)`：是否为套接字

**权限位：**
- `S_IRUSR` / `S_IWUSR` / `S_IXUSR`：所有者权限
- `S_IRGRP` / `S_IWGRP` / `S_IXGRP`：组权限
- `S_IROTH` / `S_IWOTH` / `S_IXOTH`：其他用户权限

## chmod 函数

修改文件权限。这是 [[1.2 cp、rm、chmod、vim、gcc#chmod 命令|chmod 命令]]的编程接口：

```c
#include <sys/stat.h>

int chmod(const char *pathname, mode_t mode);
int fchmod(int fd, mode_t mode);
```

示例：
```c
// 设置权限为 rwxr-xr-x (755)
chmod("myfile.txt", S_IRWXU | S_IRGRP | S_IXGRP | S_IROTH | S_IXOTH);

// 或使用八进制数字
chmod("myfile.txt", 0755);
```

---

## umask 函数

设置文件创建时的权限掩码：

```c
#include <sys/stat.h>

mode_t umask(mode_t mask);
```

`umask` 决定了新创建文件的默认权限。实际权限 = 请求权限 & ~umask

示例：
```c
umask(022);  // 新文件默认权限为 644，新目录默认权限为 755
```

## 示例代码

### 获取文件权限信息

```c
#include <stdio.h>
#include <sys/stat.h>
#include <pwd.h>
#include <grp.h>

int main() {
    struct stat st;

    if (stat("myfile.txt", &st) == 0) {
        // 获取所有者和组信息
        struct passwd *pw = getpwuid(st.st_uid);
        struct group *gr = getgrgid(st.st_gid);

        printf("File: myfile.txt\n");
        printf("Owner: %s (UID: %d)\n", pw->pw_name, st.st_uid);
        printf("Group: %s (GID: %d)\n", gr->gr_name, st.st_gid);
        printf("Permissions: %o\n", st.st_mode & 0777);
        printf("Size: %ld bytes\n", st.st_size);
    }

    return 0;
}
```

## 关键要点

- Linux 使用三组权限（所有者、组、其他）控制文件访问
- 权限可以用符号（rwx）或八进制数字（0-7）表示
- `stat()` 函数获取文件的详细信息，包括权限
- `chmod()` 修改文件权限
- `umask()` 设置文件创建时的默认权限掩码
- UID/GID 是用户和组的数字标识符
- `/etc/passwd` 存储用户账户基本信息
