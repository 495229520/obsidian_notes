# 文件锁

**文件锁**（file locking）是一种用于控制多个进程对同一文件的并发访问的机制。它允许进程在读写文件时，防止其他进程同时修改文件，从而保证数据一致性。

---

## 文件锁的基本概念

### 为什么需要文件锁？

当多个进程同时访问同一个文件时，可能会出现以下问题：

- **数据竞争**：多个进程同时写入，导致数据混乱或损坏
- **读写冲突**：一个进程正在写入时，另一个进程读取到不完整的数据
- **丢失更新**：多个进程同时修改，后写入的覆盖先写入的

文件锁通过协调进程对文件的访问，解决这些并发问题。

### 锁的类型

| 锁类型 | 别名 | 说明 |
|--------|------|------|
| **共享锁** | 读锁（Read Lock） | 允许多个进程同时持有共享锁并读取文件，但阻止其他进程获取排他锁 |
| **排他锁** | 写锁（Exclusive Lock） | 只允许一个进程持有排他锁，阻止其他进程获取任何类型的锁 |

### 锁的兼容性矩阵

| 已持有锁 \ 请求锁 | 共享锁 | 排他锁 |
|------------------|--------|--------|
| **无锁** | 允许 | 允许 |
| **共享锁** | 允许 | 阻塞 |
| **排他锁** | 阻塞 | 阻塞 |

### 建议性锁与强制性锁

| 类型 | 说明 |
|------|------|
| **建议性锁**（Advisory Lock） | 依赖进程自愿遵守锁协议。即使文件被锁定，其他进程仍可直接进行 I/O 操作（如果它不检查锁） |
| **强制性锁**（Mandatory Lock） | 由内核强制执行。任何进程在访问被锁定的文件区域时，都会被内核阻止 |

> **注意**：Linux 默认使用**建议性锁**。强制性锁需要特殊的文件系统挂载选项（`-o mand`）和文件权限设置（set-GID 位开启，group-execute 位关闭）。

## flock() - 整文件锁

`flock()` 是一个简单的文件锁函数，它对**整个文件**进行加锁，不支持锁定文件的特定区域。

### 函数原型

```c
#include <sys/file.h>

int flock(int fd, int operation);
```

### 参数说明

- **fd**：要锁定的[[4.1 打开、读取、写入、关闭#文件描述符（File Descriptor）|文件描述符]]。如果使用 `fopen()` 打开文件，可通过 [[6.2 高级文件操作函数#fileno() - 获取文件描述符|fileno()]] 获取 fd
- **operation**：锁操作类型

### operation 参数

| 值 | 说明 |
|----|------|
| `LOCK_SH` | 获取共享锁（读锁）。多个进程可以同时持有共享锁 |
| `LOCK_EX` | 获取排他锁（写锁）。同一时刻只能有一个进程持有排他锁 |
| `LOCK_UN` | 释放锁 |
| `LOCK_NB` | 非阻塞模式（可与上述值通过 `|` 组合）。如果无法立即获取锁，立即返回错误而不是等待 |

### 返回值

- 成功：返回 0
- 失败：返回 -1，并设置 errno（如 `EWOULDBLOCK` 表示非阻塞模式下锁被占用）

### flock 的特点

1. **整文件锁定**：不支持锁定文件的部分区域
2. **与文件描述符绑定**：锁与打开的文件描述符关联，`close()` 或进程退出会自动释放锁
3. **建议性锁**：其他进程可以忽略锁直接访问文件

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/file.h>

int main() {
    int fd = open("data.txt", O_RDWR | O_CREAT, 0644);
    if (fd == -1) {
        perror("open");
        return 1;
    }

    printf("尝试获取排他锁...\n");

    // 获取排他锁（阻塞方式）
    if (flock(fd, LOCK_EX) == -1) {
        perror("flock");
        close(fd);
        return 1;
    }

    printf("成功获取锁，开始处理文件...\n");

    // 模拟文件处理
    sleep(10);

    // 释放锁
    flock(fd, LOCK_UN);
    printf("锁已释放\n");

    close(fd);
    return 0;
}
```

### 非阻塞模式示例

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/file.h>
#include <errno.h>

int main() {
    int fd = open("data.txt", O_RDWR);
    if (fd == -1) {
        perror("open");
        return 1;
    }

    // 尝试非阻塞方式获取排他锁
    if (flock(fd, LOCK_EX | LOCK_NB) == -1) {
        if (errno == EWOULDBLOCK) {
            printf("文件已被其他进程锁定，无法获取锁\n");
        } else {
            perror("flock");
        }
        close(fd);
        return 1;
    }

    printf("成功获取锁\n");

    // 处理文件...

    flock(fd, LOCK_UN);
    close(fd);
    return 0;
}
```

## fcntl() - 记录锁（区域锁）

`fcntl()` 是一个功能强大的文件控制函数，其中一个重要功能是提供**记录锁**（Record Lock），允许锁定文件的**特定区域**而非整个文件。

### 函数原型

```c
#include <fcntl.h>
#include <unistd.h>

int fcntl(int fd, int cmd, ... /* struct flock *lock */);
```

### 参数说明

- **fd**：文件描述符
- **cmd**：操作命令
- **lock**：指向 `struct flock` 结构的指针（可选，取决于 cmd）

### cmd 命令（锁相关）

| 命令 | 说明 |
|------|------|
| `F_GETLK` | 测试锁。检查是否可以获取指定的锁，如果存在冲突的锁，将冲突锁的信息填入 flock 结构 |
| `F_SETLK` | 设置锁（非阻塞）。尝试获取或释放锁，如果无法立即获取则返回错误 |
| `F_SETLKW` | 设置锁（阻塞）。尝试获取锁，如果无法获取则阻塞等待（W = Wait） |

### struct flock 结构

```c
struct flock {
    short l_type;    // 锁类型：F_RDLCK（读锁）、F_WRLCK（写锁）、F_UNLCK（解锁）
    short l_whence;  // 起始位置基准：SEEK_SET、SEEK_CUR、SEEK_END
    off_t l_start;   // 锁定区域的起始偏移量（相对于 l_whence）
    off_t l_len;     // 锁定区域的长度（字节数），0 表示从 l_start 到文件末尾
    pid_t l_pid;     // 持有锁的进程 ID（仅 F_GETLK 返回时有效）
};
```

### 锁类型

| 类型 | 说明 |
|------|------|
| `F_RDLCK` | 共享锁（读锁）。多个进程可以在同一区域持有读锁 |
| `F_WRLCK` | 排他锁（写锁）。独占指定区域，阻止其他进程的读写锁 |
| `F_UNLCK` | 解锁。释放指定区域的锁 |

### 返回值

- **F_GETLK**：成功返回 0，失败返回 -1
- **F_SETLK/F_SETLKW**：成功返回 0，失败返回 -1

### fcntl 锁的特点

1. **区域锁**：可以锁定文件的任意字节范围
2. **与进程绑定**：锁属于进程，同一进程的多个文件描述符共享锁
3. **自动释放**：进程终止或关闭**任一**指向该文件的描述符时，锁会被释放
4. **可升级/降级**：读锁可升级为写锁，写锁可降级为读锁

### 示例：锁定文件区域

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <string.h>

int main() {
    int fd = open("data.txt", O_RDWR);
    if (fd == -1) {
        perror("open");
        return 1;
    }

    struct flock fl;

    // 设置写锁，锁定前 100 字节
    fl.l_type = F_WRLCK;       // 写锁
    fl.l_whence = SEEK_SET;    // 从文件开头计算
    fl.l_start = 0;            // 起始位置：偏移 0
    fl.l_len = 100;            // 锁定长度：100 字节

    printf("尝试锁定文件前 100 字节...\n");

    // 阻塞方式获取锁
    if (fcntl(fd, F_SETLKW, &fl) == -1) {
        perror("fcntl");
        close(fd);
        return 1;
    }

    printf("成功获取锁，正在处理...\n");

    // 模拟文件处理
    sleep(10);

    // 解锁
    fl.l_type = F_UNLCK;
    if (fcntl(fd, F_SETLK, &fl) == -1) {
        perror("fcntl unlock");
    }

    printf("锁已释放\n");
    close(fd);
    return 0;
}
```

### 示例：检测锁状态

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    int fd = open("data.txt", O_RDWR);
    if (fd == -1) {
        perror("open");
        return 1;
    }

    struct flock fl;

    // 检查前 100 字节是否被锁定
    fl.l_type = F_WRLCK;
    fl.l_whence = SEEK_SET;
    fl.l_start = 0;
    fl.l_len = 100;

    // F_GETLK 不会真正加锁，只是查询
    if (fcntl(fd, F_GETLK, &fl) == -1) {
        perror("fcntl");
        close(fd);
        return 1;
    }

    if (fl.l_type == F_UNLCK) {
        printf("该区域未被锁定，可以安全访问\n");
    } else {
        printf("该区域被进程 %d 锁定\n", fl.l_pid);
        printf("锁类型: %s\n", fl.l_type == F_RDLCK ? "读锁" : "写锁");
    }

    close(fd);
    return 0;
}
```

## flock 与 fcntl 的对比

| 特性 | flock() | fcntl() |
|------|---------|---------|
| 锁定粒度 | 整个文件 | 任意字节范围 |
| 锁类型 | 共享锁、排他锁 | 读锁、写锁 |
| 锁的归属 | 与文件描述符关联 | 与进程关联 |
| 跨 fork() | 子进程继承但不共享 | 子进程继承且共享 |
| NFS 支持 | 不支持 | 支持 |
| 复杂度 | 简单 | 较复杂 |
| 使用场景 | 简单的文件互斥访问 | 数据库等需要精细锁控制的场景 |

## 文件锁的注意事项

### 1. 死锁问题

当多个进程互相等待对方持有的锁时，会发生死锁：

```
进程 A 持有文件 1 的锁，请求文件 2 的锁
进程 B 持有文件 2 的锁，请求文件 1 的锁
→ 双方都在等待，形成死锁
```

**避免方法**：
- 按固定顺序获取锁
- 使用非阻塞模式尝试获取锁，失败时释放已持有的锁并重试
- 设置超时机制

### 2. 建议性锁的局限

Linux 的文件锁默认是建议性的，即使文件被锁定，不遵守锁协议的进程仍可直接读写文件。因此：
- 所有访问共享文件的进程都必须使用锁
- 无法防止恶意或不规范的程序

### 3. 锁与 fork()

- **flock()**：子进程继承父进程的锁，但它们是独立的锁
- **fcntl()**：子进程继承父进程的锁，并且共享同一把锁

### 4. 锁与关闭文件

- **flock()**：关闭文件描述符时释放锁
- **fcntl()**：关闭指向该文件的**任一**文件描述符时，释放该进程在该文件上的所有锁

---

## 关键要点

- **文件锁**用于协调多进程对文件的并发访问，分为共享锁（读锁）和排他锁（写锁）
- **flock()**：简单的整文件锁，适合基本的文件互斥访问场景
- **fcntl()**：功能强大的记录锁，支持锁定文件的任意字节范围，适合需要精细控制的场景
- Linux 默认使用**建议性锁**，依赖进程自愿遵守锁协议
- 使用文件锁时需注意**死锁**问题和锁的**继承行为**
