# Linux 文件 I/O 基本操作

本节介绍 Linux 系统中最基本的文件 I/O 函数：`open`、`read`、`write`、`close`。

---

## 文件描述符（File Descriptor）

**文件描述符**（File Descriptor，简称 fd）是一个非负整数，是 Linux 内核用来标识已打开文件的"句柄"。它是用户空间与内核空间之间进行 I/O 交互的桥梁。

当进程打开一个文件时，内核会：
1. 在内核中创建一个文件表项（file table entry）
2. 返回一个非负整数作为文件描述符给进程

### 标准文件描述符

每个进程启动时，系统会自动打开三个标准的文件描述符：

| 文件描述符 | 宏定义 | 说明 |
|-----------|--------|------|
| 0 | `STDIN_FILENO` | 标准输入（键盘） |
| 1 | `STDOUT_FILENO` | 标准输出（显示器） |
| 2 | `STDERR_FILENO` | 标准错误输出（显示器） |

### 文件描述符的特点

- 文件描述符是**进程级别**的资源，每个进程都有自己的文件描述符表
- 新打开的文件描述符总是使用当前可用的**最小非负整数**
- 文件描述符的值域通常为 0~1023（默认），可通过 `ulimit` 调整
- 文件描述符可以指向普通文件、目录、设备文件、管道、socket 等

## open() - 打开文件

`open()` 函数用于打开一个已有的文件或创建一个新文件。

### 函数原型

```c
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>

int open(const char *pathname, int flags);
int open(const char *pathname, int flags, mode_t mode);
```

### 参数说明

- **pathname**：要打开或创建的文件路径，可以是绝对路径或相对路径
- **flags**：打开文件的方式，由一个或多个标志位通过按位或（`|`）组合而成
- **mode**：仅在创建新文件时有效，指定新文件的访问权限（需要与 `O_CREAT` 配合使用）

### flags 标志位

**必选标志**（三选一）：

| 标志 | 说明 |
|------|------|
| `O_RDONLY` | 只读模式 |
| `O_WRONLY` | 只写模式 |
| `O_RDWR` | 读写模式 |

**可选标志**（按需组合）：

| 标志 | 说明 |
|------|------|
| `O_CREAT` | 如果文件不存在则创建，需要指定 mode 参数 |
| `O_EXCL` | 与 `O_CREAT` 一起使用，如果文件已存在则返回错误 |
| `O_TRUNC` | 如果文件存在且为普通文件，将其长度截断为 0 |
| `O_APPEND` | 追加模式，每次写入前将文件偏移量移动到文件末尾 |
| `O_NONBLOCK` | 非阻塞模式 |
| `O_SYNC` | 同步 I/O，每次 write 都等待数据写入磁盘 |

### mode 权限参数

当使用 `O_CREAT` 创建新文件时，必须指定 mode 参数。常用权限值：

| 权限 | 八进制值 | 说明 |
|------|---------|------|
| `S_IRWXU` | 0700 | 所有者读、写、执行 |
| `S_IRUSR` | 0400 | 所有者读 |
| `S_IWUSR` | 0200 | 所有者写 |
| `S_IXUSR` | 0100 | 所有者执行 |
| `S_IRWXG` | 0070 | 组读、写、执行 |
| `S_IRWXO` | 0007 | 其他人读、写、执行 |

> **注意**：实际文件权限 = mode & ~umask

### 返回值

- 成功：返回新打开的文件描述符（最小可用非负整数）
- 失败：返回 -1，并设置 errno

### 示例代码

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    // 1. 只读方式打开已存在的文件
    int fd = open("test.txt", O_RDONLY);
    if (fd == -1) {
        perror("open error");
        return 1;
    }

    // 2. 创建新文件（如果存在则截断）
    int fd2 = open("output.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
    if (fd2 == -1) {
        perror("open error");
        return 1;
    }

    close(fd);
    close(fd2);
    return 0;
}
```

## read() - 读取文件

`read()` 函数从打开的文件描述符中读取数据。

### 函数原型

```c
#include <unistd.h>

ssize_t read(int fd, void *buf, size_t count);
```

### 参数说明

- **fd**：要读取的文件描述符
- **buf**：存放读取数据的缓冲区指针
- **count**：请求读取的最大字节数

### 返回值

- 成功：返回实际读取的字节数
  - 返回值可能小于 count（如文件剩余内容不足）
  - 返回 0 表示已到达文件末尾（EOF）
- 失败：返回 -1，并设置 errno

### 读取行为

1. 从文件当前偏移量位置开始读取
2. 读取成功后，文件偏移量会向后移动实际读取的字节数
3. 如果请求的字节数大于文件剩余内容，只返回实际可读的字节数

### 示例代码

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    int fd = open("test.txt", O_RDONLY);
    if (fd == -1) {
        perror("open error");
        return 1;
    }

    char buf[1024];
    ssize_t bytes_read;

    // 循环读取，直到文件末尾
    while ((bytes_read = read(fd, buf, sizeof(buf) - 1)) > 0) {
        buf[bytes_read] = '\0';  // 添加字符串结束符
        printf("%s", buf);
    }

    if (bytes_read == -1) {
        perror("read error");
    }

    close(fd);
    return 0;
}
```

## write() - 写入文件

`write()` 函数向打开的文件描述符写入数据。

### 函数原型

```c
#include <unistd.h>

ssize_t write(int fd, const void *buf, size_t count);
```

### 参数说明

- **fd**：要写入的文件描述符
- **buf**：包含待写入数据的缓冲区指针
- **count**：请求写入的字节数

### 返回值

- 成功：返回实际写入的字节数（通常等于 count，除非磁盘满或被信号中断）
- 失败：返回 -1，并设置 errno

### 写入行为

1. 从文件当前偏移量位置开始写入
2. 写入成功后，文件偏移量会向后移动实际写入的字节数
3. 如果以 `O_APPEND` 模式打开，每次写入前会自动将偏移量移到文件末尾
4. 写入操作通常先写入内核缓冲区（page cache），再异步刷新到磁盘

### 示例代码

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <string.h>

int main() {
    int fd = open("output.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
    if (fd == -1) {
        perror("open error");
        return 1;
    }

    const char *msg = "Hello, Linux I/O!\n";
    ssize_t bytes_written = write(fd, msg, strlen(msg));

    if (bytes_written == -1) {
        perror("write error");
    } else {
        printf("成功写入 %zd 字节\n", bytes_written);
    }

    close(fd);
    return 0;
}
```

## close() - 关闭文件

`close()` 函数关闭一个打开的文件描述符，释放相关资源。

### 函数原型

```c
#include <unistd.h>

int close(int fd);
```

### 参数说明

- **fd**：要关闭的文件描述符

### 返回值

- 成功：返回 0
- 失败：返回 -1，并设置 errno

### 关闭行为

1. 释放进程持有的文件描述符
2. 如果这是指向该文件的最后一个描述符，释放内核中的文件表项
3. 如果文件有未写入磁盘的数据，会触发刷新（但不保证立即完成）

> **注意**：进程结束时会自动关闭所有打开的文件描述符，但良好的编程习惯是显式关闭不再使用的文件描述符。

## 综合示例：文件复制

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

#define BUF_SIZE 4096

int main(int argc, char *argv[]) {
    if (argc != 3) {
        fprintf(stderr, "用法: %s <源文件> <目标文件>\n", argv[0]);
        return 1;
    }

    // 打开源文件（只读）
    int src_fd = open(argv[1], O_RDONLY);
    if (src_fd == -1) {
        perror("打开源文件失败");
        return 1;
    }

    // 创建目标文件（写入，不存在则创建，存在则截断）
    int dst_fd = open(argv[2], O_WRONLY | O_CREAT | O_TRUNC, 0644);
    if (dst_fd == -1) {
        perror("创建目标文件失败");
        close(src_fd);
        return 1;
    }

    // 读取并写入
    char buf[BUF_SIZE];
    ssize_t bytes_read, bytes_written;

    while ((bytes_read = read(src_fd, buf, BUF_SIZE)) > 0) {
        bytes_written = write(dst_fd, buf, bytes_read);
        if (bytes_written != bytes_read) {
            perror("写入失败");
            close(src_fd);
            close(dst_fd);
            return 1;
        }
    }

    if (bytes_read == -1) {
        perror("读取失败");
    }

    // 关闭文件
    close(src_fd);
    close(dst_fd);

    printf("文件复制完成\n");
    return 0;
}
```

## 关键要点

- **文件描述符**：非负整数，是内核标识打开文件的句柄，0/1/2 分别对应标准输入/输出/错误
- **open()**：打开或创建文件，返回文件描述符，flags 控制打开模式
- **read()**：从文件描述符读取数据到缓冲区，返回实际读取字节数，0 表示 EOF
- **write()**：将缓冲区数据写入文件描述符，数据先进入内核缓冲区
- **close()**：关闭文件描述符，释放系统资源
