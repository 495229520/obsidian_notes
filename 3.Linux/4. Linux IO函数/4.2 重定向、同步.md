# 文件重定向与同步

本节介绍 Linux 中的文件描述符重定向（`dup`/`dup2`）、文件同步（`fsync`）、文件偏移量操作（`lseek`）以及临时文件创建（`mkstemp`）。

---

## 文件描述符重定向

### dup() - 复制文件描述符

`dup()` 函数用于复制一个现有的[[4.1 打开、读取、写入、关闭#文件描述符（File Descriptor）|文件描述符]]，返回一个新的文件描述符，这两个描述符指向同一个文件表项。

#### 函数原型

```c
#include <unistd.h>

int dup(int oldfd);
```

#### 参数说明

- **oldfd**：要复制的现有文件描述符

#### 返回值

- 成功：返回一个新的文件描述符（当前可用的最小非负整数）
- 失败：返回 -1，并设置 errno

#### 特点

- `dup()` 返回的新描述符与 oldfd 共享同一个文件偏移量
- 访问模式（只读、只写、读写）也与原描述符相同
- 文件状态标志（如 `O_APPEND`）也共享

---

### dup2() - 指定新描述符的复制

`dup2()` 函数与 `dup()` 类似，但允许指定新文件描述符的值。

#### 函数原型

```c
#include <unistd.h>

int dup2(int oldfd, int newfd);
```

#### 参数说明

- **oldfd**：要复制的现有文件描述符
- **newfd**：指定的新文件描述符值。如果 newfd 已经打开，会先将其关闭

#### 返回值

- 成功：返回 newfd
- 失败：返回 -1，并设置 errno

#### dup2 的主要用途

`dup2()` 最常见的用途是**实现输入/输出重定向**，将标准输入/输出/错误重定向到文件。

> **提示**：在标准 I/O 层（`FILE*`），可以使用 [[6.2 高级文件操作函数#freopen() - 重定向文件流|freopen()]] 实现类似的重定向功能。

#### 示例：将标准输出重定向到文件

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    // 1. 以只写方式打开文件
    int fd = open("test.txt", O_RDWR);
    if (fd == -1) {
        perror("open error");
        return 1;
    }

    // 2. 将标准输出重定向到文件
    //    newfd=1 是标准输出，dup2 会先关闭原来的 stdout，再复制 fd 到 1
    dup2(fd, STDOUT_FILENO);  // STDOUT_FILENO = 1

    // 3. 此时 printf 输出会写入文件而非屏幕
    printf("这段文字会被写入 test.txt 文件\n");

    // 4. 刷新缓冲区确保数据写入
    fflush(stdout);

    close(fd);
    return 0;
}
```

#### 重定向原理图

```
重定向前:
┌──────────────────┐
│  fd table        │
├──────────────────┤
│ 0 → stdin        │
│ 1 → stdout (tty) │
│ 2 → stderr       │
│ 3 → file.txt     │
└──────────────────┘

dup2(3, 1) 后:
┌──────────────────┐
│  fd table        │
├──────────────────┤
│ 0 → stdin        │
│ 1 → file.txt     │  ← stdout 现在指向文件
│ 2 → stderr       │
│ 3 → file.txt     │
└──────────────────┘
```

## fsync() - 文件同步

在 Linux 中，`write()` 系统调用将数据写入内核的页缓存（page cache）后就返回，数据实际写入磁盘的时机由内核决定。`fsync()` 函数可以强制将缓冲区的数据刷新到磁盘。

### 为什么需要 fsync？

- **数据持久性**：确保数据真正写入磁盘，防止系统崩溃时数据丢失
- **数据一致性**：在关键操作后确保数据已持久化

### 数据流向

```
数据 ──→ buffer（用户态缓冲区）──→ 硬盘
              ↑
           fsync()
```

![[Pasted image 20251218104418.png]]

`fsync()` 可以将文件的数据和元数据（如修改时间、文件大小等）都刷新到磁盘，确保数据完全落盘后才返回。

### 函数原型

```c
#include <unistd.h>

int fsync(int fd);
```

### 参数说明

- **fd**：一个已经打开的文件描述符

### 返回值

- 成功：返回 0
- 失败：返回 -1，并设置 errno

### 相关函数

| 函数 | 说明 |
|------|------|
| `fsync(fd)` | 同步文件数据和元数据到磁盘 |
| `fdatasync(fd)` | 只同步文件数据，不同步元数据（性能更好） |
| `sync()` | 将所有文件系统的缓冲区刷新到磁盘 |

### 示例代码

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <string.h>

int main() {
    int fd = open("test.txt", O_RDWR);
    if (fd == -1) {
        perror("open error");
        return 1;
    }

    char buf[20];

    // 1. 从文件开头读取内容
    ssize_t bytes_read = read(fd, buf, sizeof(buf));
    buf[bytes_read] = '\0';
    printf("Read from file: %s\n", buf);  // 输出: Hello

    // 2. 将读写位置移到文件末尾
    lseek(fd, 0, SEEK_END);

    // 3. 将读写位置从末尾向后偏移 5 字节
    off_t new_pos = lseek(fd, 5, SEEK_END);
    printf("New position at end: %ld\n", new_pos);  // 输出: 12 (Hello World! 占12个字符)

    // 4. 获取当前位置偏移量
    new_pos = lseek(fd, -5, SEEK_END);
    printf("New position from end: %ld\n", new_pos);  // 输出: 7

    // 5. 从当前位置（位置 7）开始写入
    const char *data = "Linux";
    write(fd, data, 5);  // 写入 "Linux"，覆盖 "orld!"

    // 6. 重置位置到开头，读取验证
    lseek(fd, 0, SEEK_SET);
    bytes_read = read(fd, buf, sizeof(buf));
    buf[bytes_read] = '\0';
    printf("Content after writing: %s\n", buf);  // 输出: Hello WLinux

    // 7. 确保数据写入磁盘
    fsync(fd);

    close(fd);
    return 0;
}
```

## lseek() - 修改读写位置偏移量

`lseek()` 函数用于改变文件的当前读写位置（偏移量）。这个函数不会移动任何数据，只是修改内核中记录的文件偏移量。

### 函数原型

```c
#include <sys/types.h>
#include <unistd.h>

off_t lseek(int fd, off_t offset, int whence);
```

### 参数说明

- **fd**：一个已打开文件的文件描述符
- **offset**：偏移量，通常是一个有符号整数
- **whence**：基准位置，有三个可选值：

| whence | 说明 |
|--------|------|
| `SEEK_SET` | 从文件**开头**计算偏移量 |
| `SEEK_CUR` | 从文件**当前位置**计算偏移量 |
| `SEEK_END` | 从文件**末尾**计算偏移量 |

### 返回值

- 成功：返回新的文件偏移量（从文件开头算起的字节数）
- 失败：返回 -1，并设置 errno

### 常见用法

```c
// 移动到文件开头
lseek(fd, 0, SEEK_SET);

// 获取当前位置
off_t current_pos = lseek(fd, 0, SEEK_CUR);

// 移动到文件末尾
lseek(fd, 0, SEEK_END);

// 获取文件大小
off_t file_size = lseek(fd, 0, SEEK_END);

// 从当前位置向后移动 100 字节
lseek(fd, 100, SEEK_CUR);

// 从当前位置向前移动 50 字节
lseek(fd, -50, SEEK_CUR);
```

### 示例：使用 lseek 定位读写

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    int fd = open("test.txt", O_RDWR);
    if (fd == -1) {
        perror("open error");
        return 1;
    }

    char buf[20];

    // 读取前 5 个字节
    read(fd, buf, 5);
    buf[5] = '\0';
    printf("First 5 bytes: %s\n", buf);

    // 跳过 3 个字节
    lseek(fd, 3, SEEK_CUR);

    // 读取接下来的 5 个字节
    read(fd, buf, 5);
    buf[5] = '\0';
    printf("After skip: %s\n", buf);

    close(fd);
    return 0;
}
```

## mkstemp() - 创建临时文件

`mkstemp()` 函数用于创建一个唯一的临时文件。它比 `tmpnam` 或 `tmpfile` 更安全，因为它同时创建并打开文件，避免了竞态条件。

### 函数原型

```c
#include <stdlib.h>

int mkstemp(char *template);
```

### 参数说明

- **template**：文件名模板字符串，**必须**包含至少 6 个连续的 `X` 字符。`mkstemp` 会将这 6 个 `X` 替换为随机字符以生成唯一文件名

### 返回值

- 成功：返回已创建的临时文件的文件描述符
- 失败：返回 -1，并设置 errno

### 重要注意事项

1. **template 必须可修改**：不能是字符串字面量（如 `"temp_XXXXXX"`），必须是字符数组
2. **文件不会自动删除**：程序结束后临时文件仍然存在，需要手动调用 `unlink()` 删除
3. **文件已打开**：返回的是文件描述符，可直接进行读写操作

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <string.h>

int main() {
    // 模板字符串 - 必须以 XXXXXX 结尾
    char template[] = "/tmp/my_temp_file_XXXXXX";

    // mkstemp 会替换 XXXXXX 为随机字符并创建文件
    int fd = mkstemp(template);
    if (fd == -1) {
        perror("mkstemp error");
        return 1;
    }

    printf("Temporary file created: %s\n", template);

    // 写入一些数据到临时文件
    const char *data = "This is a temporary file.\n";
    write(fd, data, strlen(data));

    // 关闭文件描述符
    close(fd);

    // 验证文件存在 - 读取内容
    FILE *file = fopen(template, "r");
    if (file) {
        char buf[100];
        fgets(buf, sizeof(buf), file);
        printf("Content of temp file: %s", buf);
        fclose(file);
    }

    // 删除临时文件
    unlink(template);
    printf("Temporary file deleted.\n");

    return 0;
}
```

### 输出示例

```
Temporary file created: /tmp/my_temp_file_abc123
Content of temp file: This is a temporary file.
Temporary file deleted.
```

## 关键要点

- **dup()/dup2()**：复制文件描述符，常用于实现 I/O 重定向，新旧描述符共享文件偏移量和状态标志
- **fsync()**：强制将缓冲区数据和元数据刷新到磁盘，确保数据持久性
- **lseek()**：修改文件读写位置偏移量，支持从文件头、当前位置、文件尾三种基准定位
- **mkstemp()**：安全地创建临时文件，自动生成唯一文件名并返回文件描述符
