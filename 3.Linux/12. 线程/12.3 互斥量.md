# 12.3 互斥量

## 1. 临界区与竞态条件

### 1.1 什么是临界区

**临界区 (Critical Section)** 是指访问共享资源的那段代码。在多线程环境中，当多个线程同时访问和修改同一个共享资源时，如果没有适当的保护机制，就会产生**竞态条件 (Race Condition)**，导致数据不一致或程序行为异常。

### 1.2 竞态条件示例

```c
#include <stdio.h>
#include <pthread.h>

int num = 0;  // 共享变量

void *thread_inc(void *arg) {
    for (int i = 0; i < 25000; i++) {
        num++;  // 临界区
    }
    return NULL;
}

void *thread_dec(void *arg) {
    for (int i = 0; i < 25000; i++) {
        num--;  // 临界区
    }
    return NULL;
}

int main(void) {
    pthread_t threads[4];

    pthread_create(&threads[0], NULL, thread_inc, NULL);
    pthread_create(&threads[1], NULL, thread_inc, NULL);
    pthread_create(&threads[2], NULL, thread_dec, NULL);
    pthread_create(&threads[3], NULL, thread_dec, NULL);

    for (int i = 0; i < 4; i++) {
        pthread_join(threads[i], NULL);
    }

    // 期望结果：0，实际结果：随机值
    printf("Result: %d\n", num);
    return 0;
}
```

运行多次，你会发现结果几乎每次都不同，而且很少是期望的 0。

### 1.3 为什么会出现这种情况？

`num++` 和 `num--` 看起来是简单的操作，但在 CPU 层面，它们并不是原子操作。以 `num++` 为例，它实际上分解为：

1. 从内存读取 `num` 的值到寄存器
2. 在寄存器中对值进行 +1
3. 将结果写回内存

如果两个线程同时执行 `num++`：

| 时间 | 线程 A | 线程 B | num 实际值 |
|------|--------|--------|------------|
| T1 | 读取 num = 0 | | 0 |
| T2 | | 读取 num = 0 | 0 |
| T3 | 计算 0 + 1 = 1 | | 0 |
| T4 | | 计算 0 + 1 = 1 | 0 |
| T5 | 写入 num = 1 | | 1 |
| T6 | | 写入 num = 1 | 1 |

两次 `++` 操作后，`num` 应该是 2，但实际只有 1。这就是**数据竞争**。

## 2. 互斥量 (Mutex)

**互斥量 (Mutual Exclusion)**，简称 **Mutex**，是一种同步原语，用于保护临界区，确保同一时刻只有一个线程能够访问共享资源。

### 2.1 互斥量的工作原理

- 互斥量就像一把**锁**
- 线程进入临界区前必须**获取锁** (lock)
- 如果锁已被其他线程持有，当前线程会**阻塞等待**
- 线程离开临界区后必须**释放锁** (unlock)
- 其他等待的线程才有机会获取锁

```
线程 A                    线程 B
   │                         │
   ▼                         ▼
获取锁 ─────────────────► 尝试获取锁（阻塞）
   │                         │
   ▼                         │
执行临界区                    │ 等待中...
   │                         │
   ▼                         │
释放锁 ─────────────────► 获取锁成功
   │                         │
   ▼                         ▼
继续执行                  执行临界区
                             │
                             ▼
                          释放锁
```

## 3. 互斥量相关函数

### 3.1 pthread_mutex_init - 初始化

```c
#include <pthread.h>

int pthread_mutex_init(pthread_mutex_t *mutex, const pthread_mutexattr_t *attr);
```

| 参数 | 说明 |
|------|------|
| `mutex` | 指向要初始化的互斥量 |
| `attr` | 互斥量属性，传入 `NULL` 使用默认属性 |

也可以使用静态初始化：
```c
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
```

### 3.2 pthread_mutex_lock - 加锁

```c
int pthread_mutex_lock(pthread_mutex_t *mutex);
```

- 如果互斥量未被锁定，则锁定它并立即返回
- 如果互斥量已被其他线程锁定，则**阻塞**直到互斥量可用
- **同一线程重复加锁会导致死锁**（使用默认属性时）

### 3.3 pthread_mutex_trylock - 尝试加锁

```c
int pthread_mutex_trylock(pthread_mutex_t *mutex);
```

- 非阻塞版本的 lock
- 如果互斥量可用，则锁定并返回 0
- 如果互斥量已被锁定，立即返回 `EBUSY`，不会阻塞

### 3.4 pthread_mutex_unlock - 解锁

```c
int pthread_mutex_unlock(pthread_mutex_t *mutex);
```

- 释放互斥量
- **必须由持有锁的线程调用**
- 释放后，等待该互斥量的其他线程会被唤醒

### 3.5 pthread_mutex_destroy - 销毁

```c
int pthread_mutex_destroy(pthread_mutex_t *mutex);
```

- 销毁互斥量，释放相关资源
- 销毁前必须确保互斥量**未被锁定**
- 销毁后不能再使用该互斥量

---

## 4. 互斥量的使用模式

标准的互斥量使用模式如下：

```c
pthread_mutex_t mutex;

// 初始化
pthread_mutex_init(&mutex, NULL);

// 使用
pthread_mutex_lock(&mutex);
// === 临界区开始 ===
// 访问共享资源
// === 临界区结束 ===
pthread_mutex_unlock(&mutex);

// 销毁
pthread_mutex_destroy(&mutex);
```

## 5. 完整示例

使用互斥量修复前面的竞态条件问题：

```c
#include <stdio.h>
#include <pthread.h>

int num = 0;
pthread_mutex_t mutex;  // 声明互斥量

void *thread_inc(void *arg) {
    for (int i = 0; i < 25000; i++) {
        pthread_mutex_lock(&mutex);    // 加锁
        num++;                          // 临界区
        pthread_mutex_unlock(&mutex);  // 解锁
    }
    return NULL;
}

void *thread_dec(void *arg) {
    for (int i = 0; i < 25000; i++) {
        pthread_mutex_lock(&mutex);    // 加锁
        num--;                          // 临界区
        pthread_mutex_unlock(&mutex);  // 解锁
    }
    return NULL;
}

int main(void) {
    pthread_t threads[4];

    // 初始化互斥量
    pthread_mutex_init(&mutex, NULL);

    pthread_create(&threads[0], NULL, thread_inc, NULL);
    pthread_create(&threads[1], NULL, thread_inc, NULL);
    pthread_create(&threads[2], NULL, thread_dec, NULL);
    pthread_create(&threads[3], NULL, thread_dec, NULL);

    for (int i = 0; i < 4; i++) {
        pthread_join(threads[i], NULL);
    }

    // 销毁互斥量
    pthread_mutex_destroy(&mutex);

    // 现在结果总是 0
    printf("Result: %d\n", num);
    return 0;
}
```

## 6. 死锁

**死锁 (Deadlock)** 是指两个或多个线程相互等待对方持有的资源，导致所有线程都无法继续执行。

### 6.1 死锁的四个必要条件

| 条件 | 说明 |
|------|------|
| **互斥** | 资源只能被一个线程独占 |
| **持有并等待** | 线程持有资源的同时等待其他资源 |
| **不可剥夺** | 资源只能由持有者主动释放 |
| **循环等待** | 存在线程的循环等待链 |

### 6.2 死锁示例

```c
pthread_mutex_t mutex_a, mutex_b;

void *thread_1(void *arg) {
    pthread_mutex_lock(&mutex_a);
    sleep(1);  // 模拟一些操作
    pthread_mutex_lock(&mutex_b);  // 等待 mutex_b
    // ...
    pthread_mutex_unlock(&mutex_b);
    pthread_mutex_unlock(&mutex_a);
    return NULL;
}

void *thread_2(void *arg) {
    pthread_mutex_lock(&mutex_b);
    sleep(1);  // 模拟一些操作
    pthread_mutex_lock(&mutex_a);  // 等待 mutex_a
    // ...
    pthread_mutex_unlock(&mutex_a);
    pthread_mutex_unlock(&mutex_b);
    return NULL;
}
```

线程 1 持有 mutex_a，等待 mutex_b；线程 2 持有 mutex_b，等待 mutex_a。两个线程互相等待，形成死锁。

### 6.3 避免死锁的方法

| 方法 | 说明 |
|------|------|
| **固定加锁顺序** | 所有线程按相同顺序获取多个锁 |
| **使用 trylock** | 尝试获取锁失败时释放已持有的锁并重试 |
| **避免嵌套锁** | 尽量减少同时持有多个锁 |
| **设置超时** | 使用 `pthread_mutex_timedlock` 设置超时时间 |

**修复示例**（固定加锁顺序）：

```c
void *thread_1(void *arg) {
    pthread_mutex_lock(&mutex_a);  // 先锁 A
    pthread_mutex_lock(&mutex_b);  // 再锁 B
    // ...
    pthread_mutex_unlock(&mutex_b);
    pthread_mutex_unlock(&mutex_a);
    return NULL;
}

void *thread_2(void *arg) {
    pthread_mutex_lock(&mutex_a);  // 也先锁 A
    pthread_mutex_lock(&mutex_b);  // 再锁 B
    // ...
    pthread_mutex_unlock(&mutex_b);
    pthread_mutex_unlock(&mutex_a);
    return NULL;
}
```

## 7. 与 C++ 原子操作的比较

对于简单的计数器等场景，除了使用互斥量，还可以使用 [[15.3 atomic|C++11 的 std::atomic]]。两者的对比：

| 特性 | 互斥量 (Mutex) | 原子操作 (Atomic) |
|------|----------------|-------------------|
| **适用场景** | 保护复杂的临界区 | 保护单个变量的简单操作 |
| **性能** | 较慢（涉及系统调用） | 较快（硬件级别支持） |
| **灵活性** | 高（可保护任意代码段） | 低（只能保护单个变量） |
| **复杂度** | 需要手动加锁/解锁 | 使用简单 |

---

## 8. 关键要点

1. **临界区**是访问共享资源的代码段，需要保护以避免竞态条件
2. **互斥量**是保护临界区的同步机制，确保同一时刻只有一个线程访问
3. **使用模式**：lock → 临界区 → unlock，必须成对使用
4. **死锁**是多线程编程的常见问题，需要通过固定加锁顺序等方法避免
5. 对于简单变量的操作，可以考虑使用[[15.3 atomic|原子操作]]替代互斥量
