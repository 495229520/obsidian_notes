# 12.2 线程的创建与运行

## 1. pthread_create 函数

在 Linux 中使用 **POSIX 线程库 (Pthreads)** 来创建和管理线程。`pthread_create` 是创建新线程的核心函数。

> C++11 提供了更现代的线程接口 [[15.2 thread|std::thread]]，但理解底层的 POSIX 线程对于系统编程非常重要。

### 1.1 函数原型

```c
#include <pthread.h>

int pthread_create(pthread_t *thread, const pthread_attr_t *attr,
                   void *(*start_routine)(void *), void *arg);
```

### 1.2 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `thread` | `pthread_t *` | 输出参数，成功创建后，线程的 ID 会被存储在这个指针指向的位置 |
| `attr` | `const pthread_attr_t *` | 设置线程的属性（如栈大小、调度策略等）。传入 `NULL` 表示使用默认属性，默认属性创建的线程是**可结合的 (joinable)** |
| `start_routine` | `void *(*)(void *)` | 这是一个函数指针，指向线程启动时要执行的函数。该函数的签名必须是：接收一个 `void*` 参数，返回一个 `void*` |
| `arg` | `void *` | 传递给 `start_routine` 函数的参数。如果需要传递多个参数，可以将它们封装到一个结构体中，然后传递结构体的指针 |

### 1.3 返回值

| 返回值 | 说明 |
|--------|------|
| `0` | 成功创建线程 |
| 非零 | 失败，返回错误码 |

### 1.4 基本示例

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

// 线程执行的函数
void *thread_function(void *arg) {
    int id = *(int *)arg;
    for (int i = 0; i < 5; i++) {
        printf("Thread %d: count = %d\n", id, i);
        sleep(1);
    }
    return NULL;
}

int main(void) {
    pthread_t thread_id;
    int thread_arg = 1;

    // 创建线程
    int result = pthread_create(&thread_id, NULL, thread_function, &thread_arg);
    if (result != 0) {
        printf("Error creating thread\n");
        return 1;
    }

    // 主线程继续执行
    printf("Main thread: created thread %lu\n", (unsigned long)thread_id);

    // 等待线程结束
    pthread_join(thread_id, NULL);

    printf("Main thread: thread finished\n");
    return 0;
}
```

### 1.5 创建多线程示例

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

#define NUM_THREADS 5

void *thread_main(void *arg) {
    int id = *(int *)arg;
    printf("Thread %d started\n", id);
    sleep(2);
    printf("Thread %d finished\n", id);
    return NULL;
}

int main(void) {
    pthread_t threads[NUM_THREADS];
    int thread_ids[NUM_THREADS];

    // 创建多个线程
    for (int i = 0; i < NUM_THREADS; i++) {
        thread_ids[i] = i + 1;
        if (pthread_create(&threads[i], NULL, thread_main, &thread_ids[i]) != 0) {
            printf("Failed to create thread %d\n", i);
            return 1;
        }
    }

    // 等待所有线程结束
    for (int i = 0; i < NUM_THREADS; i++) {
        pthread_join(threads[i], NULL);
    }

    printf("All threads completed\n");
    return 0;
}
```

**编译命令**：
```bash
gcc -o thread_demo thread_demo.c -lpthread
```

## 2. pthread_join 函数

`pthread_join` 用于等待指定线程结束，并获取其返回值。

### 2.1 函数原型

```c
int pthread_join(pthread_t thread, void **retval);
```

### 2.2 参数说明

| 参数 | 说明 |
|------|------|
| `thread` | 要等待的线程 ID |
| `retval` | 输出参数，用于存储目标线程的返回值。如果不需要返回值，可以传入 `NULL` |

### 2.3 工作原理

1. 调用 `pthread_join` 的线程会**阻塞**，直到目标线程执行完毕
2. 目标线程结束后，其资源会被回收
3. 如果目标线程已经结束，`pthread_join` 会立即返回

### 2.4 获取线程返回值示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

void *thread_sum(void *arg) {
    int n = *(int *)arg;
    int *result = malloc(sizeof(int));
    *result = 0;

    for (int i = 1; i <= n; i++) {
        *result += i;
    }

    return result;  // 返回计算结果
}

int main(void) {
    pthread_t thread;
    int n = 100;
    void *thread_result;

    pthread_create(&thread, NULL, thread_sum, &n);
    pthread_join(thread, &thread_result);  // 获取返回值

    printf("Sum from 1 to %d = %d\n", n, *(int *)thread_result);

    free(thread_result);  // 释放动态分配的内存
    return 0;
}
```

## 3. 分离线程：pthread_detach

默认创建的线程是**可结合的 (joinable)**，需要调用 `pthread_join` 来回收资源。如果不想等待线程结束，可以将线程设为**分离状态 (detached)**。

### 3.1 函数原型

```c
int pthread_detach(pthread_t thread);
```

### 3.2 特点

- 分离后的线程结束时，系统会**自动回收**其资源
- 分离后**不能再调用** `pthread_join`
- 适用于不需要获取线程返回值的场景

### 3.3 示例

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

void *background_task(void *arg) {
    printf("Background thread running...\n");
    sleep(2);
    printf("Background thread finished\n");
    return NULL;
}

int main(void) {
    pthread_t thread;

    pthread_create(&thread, NULL, background_task, NULL);
    pthread_detach(thread);  // 分离线程

    printf("Main thread continues...\n");
    sleep(3);  // 等待足够时间让后台线程完成

    printf("Main thread finished\n");
    return 0;
}
```

## 4. CPU 与线程亲和性

**线程亲和性 (Thread Affinity)** 是指将线程绑定到特定的 CPU 核心上运行。这可以减少缓存失效，提高性能。

### 4.1 为什么需要设置亲和性

- **缓存优化**：当线程在同一个 CPU 核心上运行时，可以更好地利用 CPU 缓存
- **避免迁移开销**：线程在不同 CPU 核心之间迁移会带来额外开销
- **实时性要求**：某些实时应用需要确保线程在特定核心上运行

### 4.2 相关函数

```c
#include <pthread.h>
#include <sched.h>

// 设置线程的 CPU 亲和性
int pthread_setaffinity_np(pthread_t thread, size_t cpusetsize,
                           const cpu_set_t *cpuset);

// 获取线程的 CPU 亲和性
int pthread_getaffinity_np(pthread_t thread, size_t cpusetsize,
                           cpu_set_t *cpuset);
```

### 4.3 示例

```c
#define _GNU_SOURCE
#include <stdio.h>
#include <pthread.h>
#include <sched.h>
#include <unistd.h>

void *thread_func(void *arg) {
    int cpu = sched_getcpu();
    printf("Thread running on CPU %d\n", cpu);
    return NULL;
}

int main(void) {
    pthread_t thread;
    cpu_set_t cpuset;

    pthread_create(&thread, NULL, thread_func, NULL);

    // 设置线程只在 CPU 0 上运行
    CPU_ZERO(&cpuset);
    CPU_SET(0, &cpuset);
    pthread_setaffinity_np(thread, sizeof(cpu_set_t), &cpuset);

    pthread_join(thread, NULL);
    return 0;
}
```

## 5. 线程安全问题

当多个线程访问共享数据时，如果没有适当的同步机制，可能会导致**数据竞争 (Race Condition)**。

### 5.1 问题示例

```c
#include <stdio.h>
#include <pthread.h>

int counter = 0;  // 共享变量

void *increment(void *arg) {
    for (int i = 0; i < 100000; i++) {
        counter++;  // 非原子操作，存在竞争条件
    }
    return NULL;
}

int main(void) {
    pthread_t t1, t2;

    pthread_create(&t1, NULL, increment, NULL);
    pthread_create(&t2, NULL, increment, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    // 期望值：200000，实际值可能小于 200000
    printf("Counter = %d\n", counter);
    return 0;
}
```

上面的代码存在竞争条件，因为 `counter++` 不是原子操作，它实际上包含三步：
1. 从内存读取 counter 的值
2. 将值加 1
3. 将新值写回内存

两个线程可能同时读取相同的值，导致结果不正确。

### 5.2 解决方案

要解决这个问题，需要使用[[12.3 互斥量|互斥量]]或[[15.3 atomic|原子操作]]来保护共享数据。

---

## 6. 关键要点

1. **pthread_create** 创建新线程，需要指定线程函数和参数
2. **pthread_join** 等待线程结束并回收资源
3. **pthread_detach** 分离线程，让系统自动回收资源
4. **线程亲和性**可以将线程绑定到特定 CPU 核心
5. **多线程访问共享数据时需要同步机制**，否则会产生竞争条件
