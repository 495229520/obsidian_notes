# 非局部跳转

## 简介

在 C 语言中，`goto` 语句只能在同一函数内跳转。而 `setjmp()` 和 `longjmp()` 提供了**非局部跳转**（non-local jump）的能力，可以跨越函数边界，从深层嵌套的函数调用中直接跳转回之前保存的执行点。

这种机制常用于实现异常处理、错误恢复、以及从深层调用栈中快速退出。

---

## 1. setjmp() 和 longjmp() - 基本非局部跳转

### 函数原型

```c
#include <setjmp.h>

int setjmp(jmp_buf env);
void longjmp(jmp_buf env, int val);
```

### 功能

| 函数 | 功能 |
|------|------|
| `setjmp()` | 保存当前执行环境（寄存器、[[1.2 函数栈帧建立与销毁原理\|栈指针]]等）到 `jmp_buf` 中 |
| `longjmp()` | 恢复之前保存的执行环境，跳转回 `setjmp()` 的位置 |

### setjmp() 返回值

| 返回值 | 含义 |
|--------|------|
| `0` | 直接调用 `setjmp()` 时返回 |
| 非零 | 通过 `longjmp()` 跳转回来时返回，值为 `longjmp()` 的第二个参数 |

### jmp_buf 结构

`jmp_buf` 是一个数组类型，用于保存执行环境，包括：
- 程序计数器（PC）
- 栈指针（SP）
- 帧指针（FP/BP）
- 其他必要的寄存器

### 代码示例

```c
#include <stdio.h>
#include <setjmp.h>

jmp_buf jump_buffer;

void second_function(void) {
    printf("In second_function, about to longjmp.\n");
    longjmp(jump_buffer, 42);  // 跳转回 setjmp，返回值为 42
    printf("This will never be printed.\n");
}

void first_function(void) {
    printf("In first_function, calling second_function.\n");
    second_function();
    printf("This will never be printed.\n");
}

int main() {
    int ret;

    ret = setjmp(jump_buffer);

    if (ret == 0) {
        // 第一次调用 setjmp，返回 0
        printf("setjmp returned 0, calling first_function.\n");
        first_function();
    } else {
        // 从 longjmp 跳转回来
        printf("longjmp returned %d, back in main.\n", ret);
    }

    printf("Program continues normally.\n");
    return 0;
}
```

### 运行结果

```
setjmp returned 0, calling first_function.
In first_function, calling second_function.
In second_function, about to longjmp.
longjmp returned 42, back in main.
Program continues normally.
```

### 执行流程图

```
main()
   │
   ├── setjmp() ──────────────────────┐
   │     ↓ (返回 0)                    │
   ├── first_function()               │
   │     │                            │
   │     └── second_function()        │
   │           │                      │
   │           └── longjmp() ─────────┘
   │                                  ↓ (返回 42)
   └── 继续执行 ←─────────────────────
```

## 2. sigsetjmp() 和 siglongjmp() - 保存信号掩码

### 问题背景

`setjmp()` 和 `longjmp()` 不保证保存和恢复信号掩码。在信号处理程序中使用 `longjmp()` 可能导致信号掩码状态不正确。

### 函数原型

```c
#include <setjmp.h>

int sigsetjmp(sigjmp_buf env, int savesigs);
void siglongjmp(sigjmp_buf env, int val);
```

### 参数说明

| 参数 | 含义 |
|------|------|
| `env` | 用于保存执行环境的缓冲区 |
| `savesigs` | 非零值表示保存当前信号掩码 |
| `val` | `siglongjmp()` 的返回值（传递给 `sigsetjmp()`） |

### 代码示例：信号处理中的安全跳转

```c
#include <stdio.h>
#include <signal.h>
#include <setjmp.h>
#include <unistd.h>

static sigjmp_buf jump_buffer;
static volatile sig_atomic_t can_jump = 0;

void signal_handler(int sig) {
    printf("\nCaught SIGINT (%d), jumping back to main...\n", sig);
    if (can_jump) {
        siglongjmp(jump_buffer, 1);
    }
}

int main() {
    // 设置信号处理程序
    signal(SIGINT, signal_handler);

    if (sigsetjmp(jump_buffer, 1) == 0) {
        // 第一次调用，设置跳转点
        can_jump = 1;
        printf("Starting a long-running task. Press Ctrl+C to interrupt.\n");
        
        // 模拟长时间运行的任务
        while (1) {
            printf("Working...\n");
            sleep(1);
        }
    } else {
        // 从信号处理程序跳转回来
        printf("Interrupted! Resuming the task and exiting.\n");
    }

    printf("Program finished.\n");
    return 0;
}
```

### 运行示例

```
Starting a long-running task. Press Ctrl+C to interrupt.
Working...
Working...
Working...
^C
Caught SIGINT (2), jumping back to main...
Interrupted! Resuming the task and exiting.
Program finished.
```

## 3. 注意事项与陷阱

### 3.1 局部变量的值

`longjmp()` 跳转后，自动变量（局部变量）的值是**不确定的**，除非它们被声明为 `volatile`。

```c
#include <stdio.h>
#include <setjmp.h>

jmp_buf env;

void do_jump(void) {
    longjmp(env, 1);
}

int main() {
    int x = 0;  // 可能不正确
    volatile int y = 0;  // 正确

    if (setjmp(env) == 0) {
        x = 10;
        y = 10;
        do_jump();
    } else {
        // x 的值不确定（可能是 0 或 10）
        // y 的值确定是 10
        printf("x = %d (uncertain), y = %d (certain)\n", x, y);
    }

    return 0;
}
```

### 3.2 不要跳转到已返回的函数

如果调用 `setjmp()` 的函数已经返回，再调用 `longjmp()` 会导致**未定义行为**。

```c
// 错误示例 - 不要这样做！
jmp_buf env;

void setup_jump(void) {
    setjmp(env);  // setup_jump 返回后，这个 env 就无效了
}

int main() {
    setup_jump();
    // ...
    longjmp(env, 1);  // 未定义行为！栈帧已被销毁
}
```

### 3.3 资源泄漏

`longjmp()` 跳转时不会执行中间函数的清理代码（如 `free()`、关闭文件等），可能导致资源泄漏。

```c
void process_file(void) {
    FILE *fp = fopen("data.txt", "r");
    if (some_error_condition) {
        longjmp(env, 1);  // fp 不会被关闭，资源泄漏！
    }
    fclose(fp);
}
```

**解决方案**：在 `longjmp()` 跳转目标处进行统一的资源清理。

## 4. 实际应用场景

### 4.1 错误处理（简易异常机制）

```c
#include <stdio.h>
#include <setjmp.h>

jmp_buf error_handler;

#define TRY       if (setjmp(error_handler) == 0)
#define THROW(n)  longjmp(error_handler, n)
#define CATCH     else

void risky_operation(int value) {
    if (value < 0) {
        THROW(1);  // 抛出错误
    }
    printf("Operation succeeded with value: %d\n", value);
}

int main() {
    TRY {
        risky_operation(10);   // 成功
        risky_operation(-5);   // 失败，跳转到 CATCH
        risky_operation(20);   // 不会执行
    }
    CATCH {
        printf("Error caught! Handling error...\n");
    }

    return 0;
}
```

### 4.2 从深层递归中退出

```c
#include <stdio.h>
#include <setjmp.h>

jmp_buf escape;

void deep_recursion(int depth) {
    printf("Depth: %d\n", depth);
    
    if (depth > 100) {
        printf("Too deep! Escaping...\n");
        longjmp(escape, depth);
    }
    
    deep_recursion(depth + 1);
}

int main() {
    int depth = setjmp(escape);
    
    if (depth == 0) {
        deep_recursion(1);
    } else {
        printf("Escaped from depth %d\n", depth);
    }
    
    return 0;
}
```

## 5. 总结对比

| 函数 | 保存信号掩码 | 使用场景 |
|------|-------------|----------|
| `setjmp()` / `longjmp()` | 不保证 | 一般错误处理 |
| `sigsetjmp()` / `siglongjmp()` | 可选（`savesigs` 控制） | 信号处理程序中 |

---

## 关键要点

- **setjmp()** 保存执行环境，**longjmp()** 恢复并跳转
- `setjmp()` 直接调用返回 0，从 `longjmp()` 跳回返回非零值
- 在信号处理程序中使用 **sigsetjmp()/siglongjmp()**
- 使用 **volatile** 关键字保护跳转后需要正确值的变量
- 避免跳转到已返回函数的 `jmp_buf`
- 注意资源泄漏问题，需要在跳转目标处统一清理
- 非局部跳转是 C 语言中实现异常处理的底层机制
