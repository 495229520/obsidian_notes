# 结束进程

## 简介

在 Linux 中，进程可以通过多种方式终止。本节介绍进程终止相关的函数，包括正常退出和异常退出的处理方式。

---

## 1. exit() 和 _exit() - 终止进程

### 函数原型

```c
#include <stdlib.h>
void exit(int status);

#include <unistd.h>
void _exit(int status);
```

### 功能对比

| 函数 | 头文件 | 清理操作 | 特点 |
|------|--------|----------|------|
| `exit()` | `<stdlib.h>` | 执行 | 标准 C 库函数，会执行清理操作 |
| `_exit()` | `<unistd.h>` | **不执行** | 系统调用，立即终止进程 |

### exit() 的清理操作

`exit()` 在终止进程前会执行以下操作：

1. **调用 `atexit()` 注册的函数**：按注册的**逆序**执行
2. **刷新所有打开的输出流**：将缓冲区的数据写入文件
3. **关闭所有打开的流**
4. **删除 `tmpfile()` 创建的临时文件**
5. **调用 `_exit()`**：最终执行系统调用终止进程

### _exit() 的行为

`_exit()` 是系统调用，会：

1. **关闭所有文件描述符**
2. **释放进程占用的内存**
3. **向父进程发送 `SIGCHLD` 信号**
4. **不刷新 stdio 缓冲区**

### 代码示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

void cleanup(void) {
    printf("Cleanup function called.\n");
}

int main() {
    atexit(cleanup);  // 注册清理函数

    printf("This message is in buffer");  // 注意：没有换行符，数据在缓冲区

    // 使用 exit() - 会刷新缓冲区并调用 cleanup
    exit(0);

    // 如果使用 _exit(0)，则：
    // 1. 不会打印 "This message is in buffer"
    // 2. 不会调用 cleanup()

    return 0;  // 不会执行到这里
}
```

### 何时使用 _exit()

- 在 `fork()` 后的子进程中，如果 `exec()` 失败，应使用 `_exit()` 而不是 `exit()`
- 避免子进程执行父进程注册的 `atexit()` 清理函数
- 避免重复刷新共享的缓冲区

```c
pid_t pid = fork();
if (pid == 0) {
    // 子进程
    execlp("nonexistent", "nonexistent", NULL);
    // exec 失败
    perror("exec failed");
    _exit(1);  // 使用 _exit() 而非 exit()
}
```

## 2. atexit() - 注册退出处理函数

### 函数原型

```c
#include <stdlib.h>

int atexit(void (*function)(void));
```

### 功能

注册一个函数，该函数会在程序正常终止时（通过 `exit()` 或从 `main()` 返回）被自动调用。

### 特点

- **最多可注册 32 个函数**（POSIX 规定至少 32 个）
- **逆序执行**：后注册的先执行（LIFO 顺序）
- **同一函数可多次注册**：每注册一次就会执行一次
- **不接收参数，无返回值**

### 返回值

| 返回值 | 含义 |
|--------|------|
| `0` | 成功 |
| 非零 | 失败 |

### 代码示例

```c
#include <stdio.h>
#include <stdlib.h>

void cleanup1(void) {
    printf("Cleanup 1: Releasing resources...\n");
}

void cleanup2(void) {
    printf("Cleanup 2: Closing connections...\n");
}

void cleanup3(void) {
    printf("Cleanup 3: Saving state...\n");
}

int main() {
    // 注册顺序：1, 2, 3
    atexit(cleanup1);
    atexit(cleanup2);
    atexit(cleanup3);

    printf("Main function executing...\n");
    printf("Exiting normally.\n");

    return 0;  // 或 exit(0)
}
```

### 运行结果

```
Main function executing...
Exiting normally.
Cleanup 3: Saving state...
Cleanup 2: Closing connections...
Cleanup 1: Releasing resources...
```

> 注意执行顺序是**逆序**的：3 → 2 → 1

## 3. on_exit() - 扩展的退出处理函数（非标准）

### 函数原型

```c
#include <stdlib.h>

int on_exit(void (*function)(int, void *), void *arg);
```

### 功能

类似 `atexit()`，但注册的函数可以接收两个参数：
- **status**：传递给 `exit()` 的状态码
- **arg**：注册时提供的自定义参数

### 代码示例

```c
#include <stdio.h>
#include <stdlib.h>

void my_exit_handler(int status, void *arg) {
    printf("Exit status: %d, Message: %s\n", status, (char *)arg);
}

int main() {
    on_exit(my_exit_handler, "Goodbye!");

    printf("Program running...\n");

    exit(42);  // 退出状态码为 42
}
```

### 运行结果

```
Program running...
Exit status: 42, Message: Goodbye!
```

### 注意事项

> **警告**：`on_exit()` 是非标准函数（GNU 扩展），可移植性较差。建议优先使用标准的 `atexit()`。

## 4. abort() - 异常终止进程

### 函数原型

```c
#include <stdlib.h>

void abort(void);
```

### 功能

立即异常终止进程，产生 `SIGABRT` 信号。

### 特点

- **不返回**：函数不会返回
- **产生 core dump**：如果系统允许，会生成核心转储文件用于调试
- **不调用 `atexit()` 注册的函数**
- **可能不刷新缓冲区**

### 使用场景

- 检测到不可恢复的错误
- 断言失败（`assert()` 内部使用）
- 程序进入不应该到达的代码路径

### 代码示例

```c
#include <stdio.h>
#include <stdlib.h>

void check_critical_condition(int value) {
    if (value < 0) {
        fprintf(stderr, "Critical error: negative value!\n");
        abort();  // 异常终止
    }
}

int main() {
    check_critical_condition(-1);
    printf("This won't be printed.\n");
    return 0;
}
```

## 5. 进程退出状态

### 退出状态码约定

| 状态码 | 含义 |
|--------|------|
| `0` | 成功 |
| `1` | 一般错误 |
| `2` | 命令行用法错误 |
| `126` | 命令不可执行 |
| `127` | 命令未找到 |
| `128+N` | 被信号 N 终止 |

### 获取退出状态

在父进程中，可以通过 [[5.5 等待函数|wait() 或 waitpid()]] 获取子进程的退出状态。

使用宏来解析状态：

| 宏 | 功能 |
|----|------|
| `WIFEXITED(status)` | 子进程是否正常退出 |
| `WEXITSTATUS(status)` | 获取退出状态码（0-255） |
| `WIFSIGNALED(status)` | 子进程是否被信号终止 |
| `WTERMSIG(status)` | 获取终止信号的编号 |

## 6. 总结对比

| 函数 | 清理操作 | atexit 函数 | 缓冲区刷新 | 使用场景 |
|------|----------|-------------|------------|----------|
| `exit()` | 执行 | 调用 | 刷新 | 正常退出 |
| `_exit()` | 不执行 | 不调用 | 不刷新 | fork 后子进程失败 |
| `abort()` | 不执行 | 不调用 | 不确定 | 异常/致命错误 |
| `return` | 执行 | 调用 | 刷新 | main() 中正常返回 |

---

## 关键要点

- **exit()** 执行完整的清理操作，是正常退出的首选
- **_exit()** 跳过清理操作，用于 fork 子进程的异常退出
- **atexit()** 按 LIFO 顺序执行注册的清理函数
- **abort()** 用于不可恢复的错误，会产生 `SIGABRT` 信号
- 退出状态码 0 表示成功，非零表示失败
