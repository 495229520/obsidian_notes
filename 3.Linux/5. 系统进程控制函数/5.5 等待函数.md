# 等待函数

## 简介

当父进程使用 [[5.1 系统进程控制函数#1. fork() - 创建子进程|fork()]] 创建子进程后，需要一种机制来等待子进程结束并回收其资源。`wait()` 和 `waitpid()` 是用于等待子进程结束并回收其资源的函数。

---

## 1. 执行 Shell 命令

### 函数原型

```c
int system(char *command)
```

### 功能

`system()` 函数会创建一个新的子进程来执行你指定的 Shell 命令。这个过程是阻塞的，也就是说，父进程会一直暂停，直到子进程执行完命令后，`system()` 函数才会返回。

### 作用

它是一个非常方便的函数，用于在 C/C++ 程序中直接调用外部程序或执行 Shell 脚本。它内部封装了 `fork()`、`exec()` 和 `wait()` 等复杂操作，让你能轻松地执行命令。

---

## 2. wait() - 等待任意子进程

### 函数原型

```c
#include <sys/wait.h>

pid_t wait(int *status);
```

### 功能

- **功能**：阻塞等待任意一个子进程结束
- **返回值**：成功返回结束的子进程 PID，失败返回 -1
- **参数**：`status` - 用于存储子进程的退出状态（可为 NULL）

### 特点

1. **阻塞**：如果所有子进程都在运行，父进程会阻塞等待
2. **非阻塞返回**：如果子进程已经结束（成为僵尸进程），立即返回
3. **任意子进程**：等待任意一个子进程，不能指定特定子进程

---

## 3. waitpid() - 等待指定子进程

### 函数原型

```c
#include <sys/wait.h>

pid_t waitpid(pid_t pid, int *status, int options);
```

### 参数说明

#### pid 参数

| pid 值 | 含义 |
|--------|------|
| `> 0` | 等待指定 PID 的子进程 |
| `= 0` | 等待与调用进程同一进程组的任意子进程 |
| `= -1` | 等待任意子进程（同 `wait()`） |
| `< -1` | 等待进程组 ID 等于 `|pid|` 的任意子进程 |

#### options 参数

| 选项 | 含义 |
|------|------|
| `0` | 阻塞等待 |
| `WNOHANG` | 非阻塞模式，如果没有子进程结束立即返回 0 |
| `WUNTRACED` | 也报告已停止（但未终止）的子进程状态 |
| `WCONTINUED` | 也报告已恢复运行的子进程状态 |

### 返回值

| 返回值 | 含义 |
|--------|------|
| `> 0` | 结束的子进程 PID |
| `= 0` | 使用 `WNOHANG` 且没有子进程结束 |
| `= -1` | 错误（如没有子进程） |

---

## 4. 状态解析宏

`wait()` 和 `waitpid()` 通过 `status` 参数返回子进程的退出状态。使用以下宏来解析：

### 正常退出

| 宏 | 功能 |
|----|------|
| `WIFEXITED(status)` | 如果子进程正常退出返回真 |
| `WEXITSTATUS(status)` | 获取子进程的退出状态码（0-255） |

### 信号终止

| 宏 | 功能 |
|----|------|
| `WIFSIGNALED(status)` | 如果子进程被信号终止返回真 |
| `WTERMSIG(status)` | 获取终止子进程的信号编号 |
| `WCOREDUMP(status)` | 如果产生了 core dump 返回真 |

### 停止/继续

| 宏 | 功能 |
|----|------|
| `WIFSTOPPED(status)` | 如果子进程当前被停止返回真 |
| `WSTOPSIG(status)` | 获取导致停止的信号编号 |
| `WIFCONTINUED(status)` | 如果子进程通过 SIGCONT 恢复运行返回真 |

---

## 5. 僵尸进程的产生与危害

### 什么是僵尸进程

当子进程终止后，它的进程描述符仍然保留在系统中，等待父进程读取其退出状态。在这个状态下的进程称为**僵尸进程（Zombie Process）**。

### 僵尸进程的特点

- 虽然它本身无害，但它占用的 PID 无法被系统回收
- 如果一个程序反复创建子进程却不回收它们，就会产生大量的僵尸进程，最终**耗尽系统中的可用 PID**，导致新的进程无法创建

### 僵尸进程的危害

因此，`wait()` 和 `waitpid()` 的核心作用就是**防止僵尸进程的产生，确保进程资源的正确回收**。

## 6. 代码示例

### 6.1 基本 wait() 使用

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid1, pid2;

    printf("Parent process (PID: %d) starts.\n", getpid());

    // 1. fork() 创建第一个子进程
    pid1 = fork();
    if (pid1 == 0) {
        // 这是子进程1
        printf("\tChild 1 (PID: %d) created. It will sleep for 3 seconds.\n", getpid());
        sleep(3);
        printf("\tChild 1 finished sleeping and is exiting.\n");
        exit(0);
    }

    // 2. fork() 创建第二个子进程
    pid2 = fork();
    if (pid2 == 0) {
        // 这是子进程2
        printf("\tChild 2 (PID: %d) created. It will run 'ls -l'.\n", getpid());
        // system() 会在内部等待 ls -l 命令执行完成
        system("ls -l");
        printf("\tChild 2 finished executing 'ls -l' and is exiting.\n");
        exit(0);
    }

    // --- 父进程 ---

    // 3. 父进程使用 waitpid() 等待指定的子进程1
    printf("Parent is waiting for child 1 (PID: %d) to finish.\n", pid1);
    waitpid(pid1, NULL, 0);
    printf("Parent successfully collected child 1's status.\n");

    // 4. 父进程使用 wait() 等待任意一个子进程
    printf("Parent is now waiting for any other child to finish.\n");
    wait(NULL);
    printf("Parent successfully collected another child's status.\n");

    printf("Parent process exits.\n");
    return 0;
}
```

### 运行结果

```

    Child 2 (PID: 28535) created. It will run 'ls -l'.
```

### 6.2 分步解读

1. **父进程创建了两个子进程**：`pid1` 和 `pid2`
2. Child 1 开始休眠，而 Child 2 立即执行 `ls -l` 命令
3. **父进程调用 `waitpid(pid1, NULL, 0)`**，它只关注 `pid1`，因此它会阻塞，直到 Child 1 的 `sleep(3)` 结束并退出
4. 当 Child 1 退出后，父进程继续执行
5. **父进程接着调用 `wait(NULL)`**，这个函数会等待任何剩下的子进程。此时，如果 Child 2 已经退出，`wait()` 会立即返回并回收它的资源。如果 Child 2 还没结束，父进程会等待一个子进程

### 6.3 使用状态解析宏

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // 子进程
        printf("Child (PID: %d) running...\n", getpid());
        sleep(2);
        exit(42);  // 退出状态码 42
    }

    // 父进程
    int status;
    pid_t child_pid = wait(&status);

    printf("Child %d terminated.\n", child_pid);

    if (WIFEXITED(status)) {
        printf("Normal exit with status: %d\n", WEXITSTATUS(status));
    } else if (WIFSIGNALED(status)) {
        printf("Killed by signal: %d\n", WTERMSIG(status));
    }

    return 0;
}
```

### 6.4 非阻塞等待 (WNOHANG)

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // 子进程
        printf("Child process running...\n");
        sleep(5);
        exit(0);
    }

    // 父进程：非阻塞轮询
    int status;
    pid_t result;

    do {
        result = waitpid(pid, &status, WNOHANG);
        if (result == 0) {
            printf("Child still running, doing other work...\n");
            sleep(1);
        }
    } while (result == 0);

    if (result > 0) {
        printf("Child %d finished.\n", result);
    }

    return 0;
}
```

### 运行结果

```
Child process running...
Child still running, doing other work...
Child still running, doing other work...
Child still running, doing other work...
Child still running, doing other work...
Child still running, doing other work...
Child 12346 finished.
```

## 7. wait() 和 waitpid() 对比

| 特性 | `wait()` | `waitpid()` |
|------|----------|-------------|
| 等待目标 | 任意子进程 | 可指定子进程/进程组 |
| 阻塞控制 | 始终阻塞 | 可设置 `WNOHANG` 非阻塞 |
| 停止状态 | 不报告 | 可设置 `WUNTRACED` 报告 |
| 灵活性 | 低 | 高 |

### 等价关系

```c
wait(&status);
// 等价于
waitpid(-1, &status, 0);
```

---

## 关键要点

- **wait()** 阻塞等待任意子进程结束
- **waitpid()** 提供更精细的控制（指定进程、非阻塞等）
- 使用**状态解析宏**分析子进程的退出原因
- **僵尸进程**是已终止但未被回收的进程，必须用 wait/waitpid 回收
- 如果父进程不关心退出状态，可将 `status` 设为 `NULL`
- `WNOHANG` 选项实现非阻塞轮询
