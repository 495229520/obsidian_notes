# 进程组识别码

## 简介

在 Linux 中，进程不仅有自己的 PID（进程 ID），还属于**进程组**（Process Group）和**会话**（Session）。进程组是一组相关进程的集合，便于进行统一的信号发送和作业控制。

---

## 1. 基本概念

### 进程标识符层次结构

```
会话 (Session)
  │
  ├── 进程组 1 (前台)
  │     ├── 进程 A (组长)
  │     ├── 进程 B
  │     └── 进程 C
  │
  └── 进程组 2 (后台)
        ├── 进程 D (组长)
        └── 进程 E
```

### 相关术语

| 术语 | 英文 | 含义 |
|------|------|------|
| PID | Process ID | 进程标识符，每个进程唯一 |
| PPID | Parent Process ID | 父进程的 PID |
| PGID | Process Group ID | 进程组标识符 |
| SID | Session ID | 会话标识符 |
| 进程组长 | Process Group Leader | PGID 等于其 PID 的进程 |
| 会话首进程 | Session Leader | SID 等于其 PID 的进程 |

### 进程组的作用

1. **信号传递**：可以向整个进程组发送信号（如 `kill(-pgid, SIGTERM)`）
2. **作业控制**：Shell 使用进程组管理前台和后台作业
3. **终端控制**：控制终端与前台进程组关联

---

## 2. 获取进程组 ID

### getpgrp() - 获取当前进程的 PGID

```c
#include <unistd.h>

pid_t getpgrp(void);
```

- **功能**：获取调用进程的进程组 ID
- **返回值**：当前进程的 PGID（总是成功）

### getpgid() - 获取指定进程的 PGID

```c
#include <unistd.h>

pid_t getpgid(pid_t pid);
```

- **参数**：`pid` - 目标进程的 PID（0 表示当前进程）
- **返回值**：成功返回 PGID，失败返回 -1

### 代码示例

```c
#include <stdio.h>
#include <unistd.h>

int main() {
    pid_t pid = getpid();
    pid_t ppid = getppid();
    pid_t pgid = getpgrp();
    pid_t pgid2 = getpgid(0);  // 等同于 getpgrp()

    printf("PID:  %d\n", pid);
    printf("PPID: %d\n", ppid);
    printf("PGID: %d (getpgrp)\n", pgid);
    printf("PGID: %d (getpgid)\n", pgid2);

    return 0;
}
```

## 3. 设置进程组 ID

### setpgid() - 设置进程的 PGID

```c
#include <unistd.h>

int setpgid(pid_t pid, pid_t pgid);
```

### 参数说明

| 参数 | 含义 |
|------|------|
| `pid` | 目标进程的 PID（0 表示当前进程） |
| `pgid` | 新的进程组 ID（0 表示使用 `pid` 作为 PGID） |

### 返回值

| 返回值 | 含义 |
|--------|------|
| `0` | 成功 |
| `-1` | 失败，设置 `errno` |

### 常见用法

```c
// 将当前进程设为新进程组的组长
setpgid(0, 0);

// 将子进程加入父进程的进程组
setpgid(child_pid, getpgrp());

// 将进程移动到指定进程组
setpgid(pid, target_pgid);
```

### 限制条件

1. 只能设置**自己或子进程**的 PGID
2. 子进程调用 `exec()` 后，父进程不能再修改其 PGID
3. 新的 PGID 必须属于**同一会话**中的进程组
4. 会话首进程不能改变其 PGID

---

## 4. setpgrp() - 简化版设置

### 函数原型

```c
#include <unistd.h>

int setpgrp(void);  // System V 版本
pid_t setpgrp(void);  // BSD 版本
```

### 功能

等同于 `setpgid(0, 0)`，将当前进程设为新进程组的组长。

> **注意**：由于不同系统的实现差异，建议使用更明确的 `setpgid()`。

## 5. 实际应用示例

### 5.1 创建新进程组

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid;
    pid_t parent_pgid = getpgrp();

    printf("Parent Process: PID %d, PGID %d\n", getpid(), parent_pgid);

    pid = fork();

    if (pid < 0) {
        perror("fork");
        exit(1);
    } else if (pid == 0) {
        // 子进程：创建新的进程组
        setpgid(0, 0);  // 成为新进程组的组长
        printf("Child Process: PID %d, PGID %d (new group)\n",
               getpid(), getpgrp());

        // 再创建一个孙进程
        pid_t grandchild = fork();
        if (grandchild == 0) {
            printf("Grandchild Process: PID %d, PGID %d\n",
                   getpid(), getpgrp());
            exit(0);
        }
        wait(NULL);
        exit(0);
    } else {
        // 父进程
        sleep(1);  // 等待子进程设置完成
        printf("Child's PGID from parent view: %d\n", getpgid(pid));
        wait(NULL);
    }

    return 0;
}
```

### 运行结果

```
Parent Process: PID 12345, PGID 12345
Child Process: PID 12346, PGID 12346 (new group)
Grandchild Process: PID 12347, PGID 12346
Child's PGID from parent view: 12346
```

### 5.2 向进程组发送信号

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <signal.h>
#include <sys/wait.h>

void handler(int sig) {
    printf("Process %d received signal %d\n", getpid(), sig);
}

int main() {
    pid_t child1, child2;
    pid_t child_pgid;

    // 创建第一个子进程
    child1 = fork();
    if (child1 == 0) {
        setpgid(0, 0);  // 创建新进程组
        signal(SIGUSR1, handler);
        printf("Child1 (PID %d, PGID %d) waiting for signal...\n",
               getpid(), getpgrp());
        pause();
        exit(0);
    }

    sleep(1);
    child_pgid = getpgid(child1);

    // 创建第二个子进程，加入第一个子进程的进程组
    child2 = fork();
    if (child2 == 0) {
        setpgid(0, child_pgid);  // 加入 child1 的进程组
        signal(SIGUSR1, handler);
        printf("Child2 (PID %d, PGID %d) waiting for signal...\n",
               getpid(), getpgrp());
        pause();
        exit(0);
    }

    sleep(1);
    printf("Parent: Sending SIGUSR1 to process group %d\n", child_pgid);

    // 向整个进程组发送信号（负数 PGID）
    kill(-child_pgid, SIGUSR1);

    // 等待子进程结束
    wait(NULL);
    wait(NULL);

    printf("Parent: All children finished.\n");
    return 0;
}
```

### 运行结果

```
Child1 (PID 12346, PGID 12346) waiting for signal...
Child2 (PID 12347, PGID 12346) waiting for signal...
Parent: Sending SIGUSR1 to process group 12346
Process 12346 received signal 10
Process 12347 received signal 10
Parent: All children finished.
```

## 6. 进程组与作业控制

### Shell 作业控制示例

```bash
# 启动后台作业（创建新进程组）
$ sleep 100 &
[1] 12345

# 查看作业
$ jobs
[1]+  Running    sleep 100 &

# 将作业切换到前台
$ fg %1
sleep 100

# 按 Ctrl+Z 暂停（发送 SIGTSTP 给前台进程组）
^Z
[1]+  Stopped    sleep 100

# 在后台继续运行
$ bg %1
[1]+ sleep 100 &
```

### 进程组在作业控制中的作用

| 操作 | 信号 | 目标 |
|------|------|------|
| Ctrl+C | SIGINT | 前台进程组 |
| Ctrl+Z | SIGTSTP | 前台进程组 |
| Ctrl+\ | SIGQUIT | 前台进程组 |

## 7. 总结

| 函数 | 功能 |
|------|------|
| `getpgrp()` | 获取当前进程的 PGID |
| `getpgid(pid)` | 获取指定进程的 PGID |
| `setpgid(pid, pgid)` | 设置进程的 PGID |
| `setpgrp()` | 将当前进程设为新进程组组长 |

---

## 关键要点

- **进程组** 是一组相关进程的集合，由 PGID 标识
- **进程组长** 的 PID 等于其 PGID
- 使用 `setpgid(0, 0)` 可使进程成为新进程组的组长
- 向负的 PGID 发送信号（`kill(-pgid, sig)`）会发送给整个进程组
- 进程组是 Shell 作业控制的基础
- 子进程默认继承父进程的 PGID
