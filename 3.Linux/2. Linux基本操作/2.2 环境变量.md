# Linux 环境变量

在 Linux Ubuntu 系统中，**环境变量**（Environment Variables）是一个非常基础且核心的概念。它们是动态命名的值，存储在系统中，并可以影响程序或 shell（命令行）会话的运行方式。

---

## 1. 作用域和分类

| 分类 | 作用域 | 生效时间 | 常用设置文件 |
|------|--------|----------|--------------|
| 临时 | 仅在当前的 Shell 会话中有效 | 立即生效，会话关闭后消失 | 命令行直接 `export` |
| 用户级别 | 仅对当前的登录用户有效 | 用户登录或打开新的 Shell 时加载 | `~/.bashrc` 或 `~/.profile` |
| 系统级别 | 对系统上的所有用户都有效 | 系统启动或所有用户登录时加载 | `/etc/environment` 或 `/etc/profile` |

---

## 2. 常见的核心环境变量

| 变量名 | 作用 | 示例值 |
|--------|------|--------|
| `PATH` | 最重要的变量。它定义了 Shell 查找可执行文件的目录列表。当你输入一个命令（如 `ls` 或 `gcc`）时，Shell 就是在这个列表的路径中按顺序查找该命令 | `/usr/local/sbin:/usr/bin:/bin:...` |
| `HOME` | 当前用户的主目录路径 | `/home/username` |
| `USER` | 当前登录用户的用户名 | `john` |
| `SHELL` | 当前用户正在使用的 Shell 程序路径 | `/bin/bash` 或 `/bin/zsh` |
| `LANG` | 定义了系统和程序的默认语言和字符集设置 | `en_US.UTF-8` |
| `PWD` | 当前工作目录的绝对路径 | `/home/username/Documents` |

---

## 3. 如何查看和设置环境变量

| 操作 | 命令 | 示例 |
|------|------|------|
| 查看所有 | `env` 或 `printenv` | `env` |
| 查看特定变量 | `echo` | `echo $PATH` |
| 设置临时变量 | `export` | `export MY_VAR="test value"` |
| 取消变量 | `unset` | `unset MY_VAR` |

---

## 4. 环境变量与程序的继承性

### 父子进程

当一个程序（父进程，例如 Shell）启动另一个程序（子进程，例如你运行的脚本）时，**子进程会继承父进程的所有环境变量**。

### Shell 变量 vs 环境变量

- **Shell 变量**：直接在 Shell 中设置 `VAR=value` 叫做 Shell 变量，它只在当前 Shell 内部有效，不会传递给子进程
- **环境变量**：使用 `export VAR=value` 声明的变量才成为环境变量，可以传递给所有子进程

---

## 5. 持久化设置

为了让设置的环境变量在下次登录后依然有效，需要将其写入特定的配置文件。

### 推荐方法一：用户配置文件

- **文件**：`~/.bashrc`（或 `~/.zshrc` 如果你使用 Zsh）
- **用途**：针对单个用户，每次启动一个新的 Shell 终端时都会执行。适合设置 PATH、别名、函数等
- **格式**：在文件末尾添加：

```bash
export JAVA_HOME="/usr/lib/jvm/java-11-openjdk-amd64"
export PATH=$PATH:$JAVA_HOME/bin
```

- **生效**：修改后需要执行 `source ~/.bashrc` 或打开新终端

### 推荐方法二：系统级配置文件

- **文件**：`/etc/environment`
- **用途**：最简洁、推荐的系统级设置方法。该文件在系统启动和用户登录时读取，对所有用户生效，并且不会执行 Shell 脚本逻辑，只存储纯粹的变量赋值
- **格式**：变量赋值不需要 `export` 关键字，不需要引号（除非值中有空格）：

```
PATH="/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"
MY_GLOBAL_VAR=GlobalValue
```

- **生效**：修改后通常需要重启，或者通过 `source /etc/environment` 立即生效

### 其他文件（较少用）

- `/etc/profile`：全局配置文件，用户首次登录时执行。适合需要执行脚本逻辑的全局设置
- `~/.profile`：用户登录时执行一次，适合设置一次性的用户环境配置

---

## 关键要点

- **PATH**：最重要的环境变量，决定了命令的搜索路径
- **查看变量**：`echo $变量名` 或 `env` 查看所有
- **临时设置**：`export VAR=value`，仅当前会话有效
- **永久设置**：
  - 用户级：写入 `~/.bashrc`
  - 系统级：写入 `/etc/environment`
- **继承性**：子进程继承父进程的环境变量，但不继承普通 Shell 变量
