# 判断错误

本节介绍 C 标准库中用于检测和处理文件流错误状态的函数，包括 `feof()`、`ferror()` 和 `clearerr()`。

---

## feof() - 检测文件末尾

`feof()` 是一个 C 标准库函数，用于检查文件流是否已到达**文件末尾**（end-of-file）。

### 函数原型

```c
#include <stdio.h>

int feof(FILE *stream);
```

### 参数说明

- **stream**：一个指向已打开文件流的指针

### 返回值

- 如果文件流的 EOF 指示器被设置：返回非零值（真）
- 否则：返回 0（假）

### feof() 的工作方式

理解 `feof()` 的关键在于：它只有在**尝试读取**并且已经到达文件末尾**之后**，才会返回非零值。它不会在还没有读取到 EOF 或读取失败时返回非零值。

### 为什么会误读？

在上面的例子中，当跑读取到文件末尾时，最后一次读取（`fgetc()`）会返回 `EOF`，此时 `feof(fp)` 返回非零值。但在下一次循环中，如果再次调用 `fgetc()`，它仍会返回 `EOF`，也不会再次读取到新的 `\n` 结尾字符。

**正确用法**：正确的方式是检查读取函数的返回值而不是依赖 `feof()` 来判断是否需要继续读取。因为当取函数返回 EOF 或失败时，`feof()` 才会返回非零值（一个特殊情况是，如果 `fgetc()` 返回 EOF，`feof()` 返回 NULL）。

### 错误示例（常见陷阱）

```c
// 错误：可能会多读一次
FILE *fp;
char buffer[10];

fp = fopen("example.txt", "r");
if (fp == NULL) {
    perror("fopen error");
    return 1;
}

// 错误的循环条件
while (!feof(fp)) {  // 这是一个错误的惯例：在读取之前判断 feof
    c = fgetc(fp);
    printf("%c", c);  // 可能会输出 '\n' 结尾字符，也不会读入 \n 结尾之后的 \0
}

fclose(fp);
```

### 正确示例

```c
int main() {
    FILE *fp;
    char buffer[10];

    // 创建一个包含一些内容的文件
    fp = fopen("example.txt", "w");
    if (fp == NULL) {
        perror("fopen error");
        return 1;
    }
    fprintf(fp, "1234567890123456789090");
    fclose(fp);

    // 以只读模式打开并读取
    fp = fopen("example.txt", "r");
    if (fp == NULL) {
        perror("fopen error");
        return 1;
    }

    // 循环读取，每次读取 10 个字符
    // fgets 在读取到 EOF 时会返回 NULL
    while (fgets(buffer, sizeof(buffer), fp) != NULL) {
        printf("Read: %s\n", buffer);
    }

    // 检查循环终止的原因
    if (feof(fp)) {
        printf("\n\nEnd-of-file reached successfully.\n");
    } else if (ferror(fp)) {
        printf("\n\nAn error occurred while reading the file.\n");
    }

    fclose(fp);
    return 0;
}
```

## ferror() - 检测文件流错误

`ferror()` 是一个 C 标准库函数，用于检查文件流在最近一次操作中是否发生了**错误**。

### 函数原型

```c
#include <stdio.h>

int ferror(FILE* stream);
```

### 参数说明

- **stream**：一个指向已打开文件流的指针

### 返回值

- 如果文件流的错误指示器被设置：返回非零值（真）
- 否则：返回 0（假）

### ferror() 的工作方式

文件流内部有一个错误标识符，当在任何文件 I/O 函数（如 `fread`、`fwrite`、`fgetc` 等）在执行过程中遇到错误时，比如磁盘空间不足、权限问题或其他 I/O 错误，这个标识符会被设置。

`ferror()` 的作用就是检查这个标识符。它与 `feof()` 不同，`feof()` 检查是否到达文件末尾，而 `ferror()` 检查是否发生了真正的 I/O 错误。

### 为什么需要 ferror？

由于文件 I/O 操作可能因各种原因失败（例如，磁盘已满、文件被锁、权限不足等），仅凭函数返回值有时难以区分是正常的 EOF 还是真正的错误。`ferror()` 提供了一个明确的方式来检测这些错误情况。

---

## clearerr() - 清除错误状态

`clearerr()` 函数用于清除文件流的错误指示器和 EOF 指示器。

### 函数原型

```c
#include <stdio.h>

void clearerr(FILE *stream);
```

### 参数说明

- **stream**：一个指向已打开文件流的指针

### 返回值

- 无返回值（`void`）

### clearerr() 的作用

调用 `clearerr(fp)` 后：

- `feof(fp)` 将返回 0（假）
- `ferror(fp)` 将返回 0（假）

这允许你在处理完错误或 EOF 后，重置流的状态以继续操作。

---

## 综合示例

以下示例演示了 `feof()`、`ferror()` 和 `clearerr()` 的用法：

```c
#include <stdio.h>

int main() {
    FILE *fp = NULL;
    char buffer[10];

    // 1. 创建一个文件用于测试正常读取
    fp = fopen("normal.txt", "w");
    if (fp == NULL) {
        perror("Error creating normal.txt");
        return 1;
    }
    fputs("12345", fp);
    fclose(fp);

    // 2. 正常读取：循环到达文件末尾时正常退出
    printf("--- Testing normal read ---\n");
    fp = fopen("normal.txt", "r");
    if (fp == NULL) {
        perror("Error opening normal.txt");
        return 1;
    }

    // fgets 在读取到文件末尾时返回 NULL
    while (fgets(buffer, sizeof(buffer), fp) != NULL) {
        printf("Read: %s\n", buffer);
    }

    if (ferror(fp)) {
        printf("No I/O error occurred!\n");
    } else if (feof(fp)) {
        printf("End-of-file reached successfully.\n");
    }

    fclose(fp);

    // 3. 模拟一个 I/O 错误
    // 在只读模式下尝试写入：这会设置错误标志但可能不会产生明显的错误
    printf("\n--- Testing I/O error ---\n");
    fp = fopen("normal.txt", "r");
    if (fp == NULL) {
        perror("Error opening normal.txt");
        return 1;
    }

    // 尝试写入，这将失败并设置错误标志
    fputc('A', fp);

    // 检查错误状态
    if (ferror(fp)) {
        printf("Successfully detected an I/O error!\n");

        // 使用 clearerr 清除错误状态
        clearerr(fp);
        printf("Error status cleared. ferror(fp) returns: %d\n", ferror(fp));
    } else {
        printf("No I/O error detected.\n");
    }

    fclose(fp);

    return 0;
}
```

### 分析说明

1. **第一次测试**中，我们正常地从文件读取直到 EOF。`fgets()` 在读取到文件末尾时返回 NULL，此时 `feof(fp)` 返回非零值，`ferror(fp)` 返回 0（没有错误）

2. **第二次测试**中，我们以只读模式打开文件，然后尝试写入。由于权限不匹配，这个操作会失败并设置错误标志。`ferror(fp)` 会返回非零值，表示发生了 I/O 错误

3. `clearerr(fp)` 清除了错误状态，再次调用 `ferror(fp)` 会返回 0

4. 最后，我们使用 `clearerr(fp)` 来清除错误状态，这会同时清除 EOF 标志和错误标志

---

## 关键要点

- **feof()**：检查是否到达文件末尾，**只有在读取操作实际遇到 EOF 后**才返回真
- **ferror()**：检查文件流是否发生了 I/O 错误
- **clearerr()**：清除文件流的 EOF 标志和错误标志

### 函数对比

| 函数 | 功能 | 返回值 | 使用场景 |
|------|------|--------|---------|
| `feof()` | 检测 EOF | 非零（已到 EOF）或 0 | 判断循环终止原因 |
| `ferror()` | 检测错误 | 非零（有错误）或 0 | 区分 EOF 和真正错误 |
| `clearerr()` | 清除状态 | 无 | 重置流状态以继续操作 |

### 最佳实践

1. **不要用 `feof()` 作为循环条件**：应该检查读取函数的返回值
2. **在循环结束后使用 `feof()` 和 `ferror()`** 来判断终止原因
3. **处理完错误后使用 `clearerr()`** 来重置状态（如果需要继续使用该文件流）

### 典型的文件读取模式

```c
// 推荐的读取模式
while (fgets(buffer, sizeof(buffer), fp) != NULL) {
    // 处理数据
}

// 循环结束后检查原因
if (feof(fp)) {
    // 正常到达文件末尾
} else if (ferror(fp)) {
    // 发生了读取错误
    perror("Read error");
}
```
