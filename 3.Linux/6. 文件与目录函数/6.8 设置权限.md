# 设置权限

本节介绍 Linux 系统中用于修改文件所有者和权限的函数：`chown()` 和 `fchmodat()`。

---

## chown() - 修改文件所有者和组

`chown()` 函数用于修改文件的所有者（owner）和所属组（group）。这涉及到 [[3.5 权限控制函数#UID 和 GID|UID 和 GID]] 的概念。

### 函数原型

```c
#include <unistd.h>

int chown(const char *pathname, uid_t owner, gid_t group);
int fchown(int fd, uid_t owner, gid_t group);
int lchown(const char *pathname, uid_t owner, gid_t group);
```

### 参数说明

- **pathname**：要修改的文件路径名
- **fd**：文件描述符（`fchown` 使用）
- **owner**：新的所有者 [[3.5 权限控制函数#UID（用户标识符）|UID]]，传入 -1 表示不修改
- **group**：新的所属组 [[3.5 权限控制函数#GID（组标识符）|GID]]，传入 -1 表示不修改

### 返回值

- 成功：返回 0
- 失败：返回 -1，并设置 errno

### 三个变体的区别

| 函数 | 说明 |
|------|------|
| `chown()` | 通过路径名修改，如果是符号链接则修改链接指向的文件 |
| `fchown()` | 通过文件描述符修改 |
| `lchown()` | 通过路径名修改，如果是符号链接则修改链接本身 |

### 注意事项

- 只有超级用户（root）可以修改文件的所有者
- 普通用户只能将自己拥有的文件的组改为自己所属的其他组
- 修改所有者后，`SUID` 和 `SGID` 位会被清除（出于安全考虑）

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/stat.h>
#include <pwd.h>
#include <grp.h>

int main() {
    const char *filename = "test_ownership.txt";
    FILE *fp;

    // 1. 创建一个测试文件
    fp = fopen(filename, "w");
    if (fp == NULL) {
        perror("fopen");
        return 1;
    }
    fprintf(fp, "This is a test file.\n");
    fclose(fp);

    // 2. 获取文件当前的所有者信息
    struct stat file_info;
    if (stat(filename, &file_info) == -1) {
        perror("stat");
        return 1;
    }

    printf("File: '%s' created.\n", filename);
    printf("Initial ownership - UID: %d, GID: %d\n",
           file_info.st_uid, file_info.st_gid);

    // 3. 尝试修改文件所有者（需要 root 权限）
    uid_t new_uid = 1000;  // 示例 UID
    gid_t new_gid = 1000;  // 示例 GID

    printf("\nAttempting to change ownership to UID: %d, GID: %d\n",
           new_uid, new_gid);
    printf("Note: This operation usually requires root privileges.\n");

    if (chown(filename, new_uid, new_gid) == -1) {
        perror("chown failed");
    } else {
        printf("Ownership of '%s' changed successfully.\n", filename);

        // 验证修改结果
        stat(filename, &file_info);
        printf("Final ownership - UID: %d, GID: %d\n",
               file_info.st_uid, file_info.st_gid);
    }

    // 4. 只修改组（传入 -1 保持 UID 不变）
    if (chown(filename, -1, new_gid) == -1) {
        perror("chown (group only) failed");
    }

    // 清理
    remove(filename);
    return 0;
}
```

### 编译运行

```bash
gcc -o chown_example chown_example.c
sudo ./chown_example  # 需要 root 权限
```

## fchmodat() - 相对目录修改权限

`fchmodat()` 函数用于相对于目录文件描述符修改文件权限。它是 [[3.5 权限控制函数#chmod 函数|chmod()]] 函数的扩展版本，提供了更灵活的路径解析方式。

### 函数原型

```c
#include <sys/stat.h>
#include <fcntl.h>

int fchmodat(int dirfd, const char *pathname, mode_t mode, int flags);
```

### 参数说明

- **dirfd**：目录文件描述符，或特殊值 `AT_FDCWD`（表示当前工作目录）
- **pathname**：文件路径名
  - 如果是绝对路径，则 `dirfd` 被忽略
  - 如果是相对路径，则相对于 `dirfd` 指向的目录
- **mode**：新的[[3.5 权限控制函数#权限表示方式|权限模式]]
- **flags**：标志位，可以是 0 或 `AT_SYMLINK_NOFOLLOW`

### 返回值

- 成功：返回 0
- 失败：返回 -1，并设置 errno

### dirfd 的使用

| dirfd 值 | pathname 类型 | 行为 |
|----------|---------------|------|
| `AT_FDCWD` | 相对路径 | 相对于当前工作目录解析 |
| 有效目录 fd | 相对路径 | 相对于该目录解析 |
| 任意值 | 绝对路径 | `dirfd` 被忽略 |

### 为什么使用 fchmodat？

1. **避免竞态条件**：在多线程/多进程环境中，使用 `fchmodat()` 可以避免 TOCTOU（time-of-check to time-of-use）竞态条件
2. **相对路径操作**：可以相对于已打开的目录进行操作，而不依赖当前工作目录
3. **原子性操作**：提供更安全的文件操作方式

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#include <errno.h>

int main() {
    const char *filename = "example.txt";
    FILE *fp;

    // 1. 创建一个测试文件
    fp = fopen(filename, "w");
    if (fp == NULL) {
        perror("fopen");
        return 1;
    }
    fprintf(fp, "This is a test file.\n");
    fclose(fp);

    printf("File '%s' created with default permissions.\n", filename);

    // 2. 使用 fchmodat() 修改权限为 0644 (rw-r--r--)
    printf("\nChanging permissions to octal(0644) for all...\n");
    if (fchmodat(AT_FDCWD, filename, 0644, 0) == -1) {
        perror("fchmodat");
        return 1;
    }
    printf("Permissions changed successfully.\n");

    // 3. 尝试以只写模式打开文件（验证权限）
    printf("\nAttempting to open file in write mode...\n");
    fp = fopen(filename, "w");
    if (fp == NULL) {
        printf("Open failed as expected: Permission denied\n");
    } else {
        printf("Open succeeded unexpectedly.\n");
        fclose(fp);
    }
    printf("Permissions restored.\n");

    // 4. 恢复权限为 0666
    printf("\nRestoring write permissions for all (0666)...\n");
    if (fchmodat(AT_FDCWD, filename, 0666, 0) == -1) {
        perror("fchmodat");
        return 1;
    }
    printf("Permissions restored.\n");

    // 5. 使用 unlink() 删除文件
    printf("\nAttempting to delete file '%s' with unlink()...\n", filename);
    if (unlink(filename) == -1) {
        perror("unlink");
        return 1;
    }
    printf("File '%s' deleted successfully.\n", filename);

    return 0;
}
```

### 运行结果

```
File 'example.txt' created with default permissions.

Changing permissions to octal(0644) for all...
Permissions changed successfully.

Attempting to open file in write mode...
Open succeeded unexpectedly.
Permissions restored.

Restoring write permissions for all (0666)...
Permissions restored.

Attempting to delete file 'example.txt' with unlink()...
File 'example.txt' deleted successfully.
```

## chmod() vs fchmodat() 对比

| 特性 | chmod() | fchmodat() |
|------|---------|------------|
| 路径解析 | 相对于当前工作目录 | 可相对于任意目录 fd |
| 符号链接处理 | 跟随链接 | 可通过 flags 控制 |
| 安全性 | 可能存在 TOCTOU 问题 | 更安全 |
| 可移植性 | POSIX 标准 | POSIX.1-2008 |

---

## 关键要点

- **chown()**：修改文件所有者和组，通常需要 root 权限
- **fchown()**：通过文件描述符修改所有者
- **lchown()**：修改符号链接本身的所有者
- **fchmodat()**：相对于目录 fd 修改权限，更安全灵活
- 使用 -1 作为 UID 或 GID 参数表示保持不变
- `AT_FDCWD` 表示当前工作目录
- 修改权限时建议使用八进制数（如 `0644`、`0755`）
