# 写入文件

本节介绍 C 标准库中的文件写入函数，包括 `fputc()`、`fputs()`、`fwrite()` 和 `fflush()`。这些函数与[[6.3 读取文件|读取函数]]相对应，用于将数据写入文件流。

---

## fputc() - 写入单个字符

`fputc()` 函数将一个单字符写入指定的文件流中。

### 函数原型

```c
#include <stdio.h>

int fputc(int c, FILE *fp);
```

### 参数说明

- **c**：要写入的字符，类型为 int
- **fp**：一个指向已打开文件流的指针，数据将被写入这里

### 返回值

- 成功：返回写入的字符
- 失败：返回 `EOF`

### 特点

`fputc()` 是最基本的字符写入函数，适用于逐字符输出场景。

---

## fputs() - 写入字符串

`fputs()` 函数将一个字符串写入指定的文件流。

### 函数原型

```c
#include <stdio.h>

int fputs(const char *str, FILE *stream);
```

### 参数说明

- **str**：一个指向要写入的字符串的指针
- **stream**：一个指向文件流的指针

### 返回值

- 成功：返回非负值
- 失败：返回 `EOF`

### fputs 的特点

`fputs()` 的一个重要特点是，它在写入时会在 str 中的空字符 `\0` 处停止，但它**不会将这个空字符写入文件**，也**不会自动添加换行符**。因此，`fputs()` 只相当于写入标准的 C 语言字符串（以 `\0` 结尾的字符序列）。

---

## fwrite() - 写入数据块（二进制写入）

`fwrite()` 是一个二进制文件写入函数，它用于将指定大小和数量的数据块写入文件。

### 函数原型

```c
#include <stdio.h>

size_t fwrite(const void *ptr, size_t size, size_t n_memb, FILE *stream);
```

### 参数说明

- **ptr**：一个指向要写入数据的指针
- **size**：要写入的每个数据块的字节大小
- **n_memb**：要写入的数据块的数量
- **stream**：一个指向文件流的指针

### 返回值

- 成功：返回实际写入的数据块数量
- 失败或部分写入：返回值小于 n_memb

### fwrite 的特点

`fwrite()` 不会理会数据中的空字符，因为它只是简单地将内存中的一个数据块按字节写入文件。这使得它非常适合写入二进制数据，例如结构体或数组的序列化数据。

### fread 与 fwrite 的对应关系

类似于 [[6.3 读取文件#fread() - 二进制读取|fread()]]，`fwrite()` 通常用于写入结构化数据。通过指定 `fread`，可以写入一个结构体，并配套使用读取功能，并用 `fread` 将其读回内存。

### 示例代码

```c
void lesson43() {
    FILE* pFile = fopen("./test1.txt", "w");
    if (pFile) {
        struct Test data = {
            1000, "liner", 28
        };
        size_t ret = fwrite(&data, 1, sizeof(Test), pFile);
        printf("%s %d: %s ret = %u\n", __FILE__, __LINE__, __FUNCTION__, ret);
        fclose(pFile);
    }

    pFile = fopen("./test1.txt", "rb");
    if (pFile) {
        struct Test read;
        size_t ret = fread(&read, 1, sizeof(Test), pFile);
        printf("%s %d: %s ret = %u\n", __FILE__, __LINE__, __FUNCTION__, ret);
        printf("%s %d: %s ret = %d %s %d\n", __FILE__, __LINE__, __FUNCTION__,
               read.id, read.name, read.age);
        fclose(pFile);
    }
}
```

## fflush() - 刷新缓冲区

`fflush()` 函数用于**立即更新文件流的缓冲区**。

### 函数原型

```c
#include <stdio.h>

int fflush(FILE *stream);
```

### 参数说明

- **stream**：一个指向文件流的指针

### 返回值

- 成功：返回 0
- 失败：返回 `EOF`

### 为什么需要 fflush？

C 标准库中的文件流通常都带有缓冲区。当使用 `fputc()`、`fputs()` 等函数写入数据时，数据会先被存储在缓冲区中。当缓冲区满了、文件被关闭或程序退出时，数据才会真正写入磁盘。

`fflush()` 强制将缓冲区中的数据立即写入磁盘。它的时间开销是不可控的，因为这取决于操作系统将数据写入物理磁盘需要多长时间。

### 使用场景

在执行完部分写入后，程序使用 `fflush()` 强制将缓冲区数据写入磁盘，可以保证这部分数据的持久性。

更多关于缓冲区和 `fflush()` 的内容，参见[[6.2 高级文件操作函数#fflush() - 刷新缓冲区|高级文件操作函数中的 fflush 介绍]]。

## 综合示例

以下示例演示了 `fputc()`、`fputs()`、`fwrite()` 和 `fflush()` 的用法：

```c
#include <stdio.h>
#include <string.h>

int main() {
    FILE *fp;
    char text_line[] = "This is a line written by fputs.\n";
    char a_char = 'A';

    // 假设我们有一个结构体用于演示 fwrite
    typedef struct {
        int id;
        float value;
    } Record;

    Record my_record = {101, 99.9};

    // 以二进制写模式打开文件
    fp = fopen("output.bin", "wb+");
    if (fp == NULL) {
        perror("fopen error");
        return 1;
    }

    // 1. 使用 fputc 写入单个字符
    printf("Writing a single character with fputc...\n");
    fputc(a_char, fp);

    // 2. 使用 fputs 写入字符串
    printf("Writing a string with fputs...\n");
    fputs(text_line, fp);

    // 3. 使用 fwrite 写入结构体（二进制数据）
    printf("Writing a struct with fwrite...\n");
    fwrite(&my_record, sizeof(Record), 1, fp);

    // 4. 使用 fflush 强制将缓冲区数据写入文件
    printf("Forcing buffer flush with fflush...\n");
    fflush(fp);

    // 5. 再次写入一段字符串，并用 fflush 验证
    fputs("Another line\n", fp);
    fflush(fp);

    // 关闭文件
    fclose(fp);

    // 验证文件内容（注意：用文本编辑器打开 output.bin 可能会显示乱码）
    // 因为其中包含了 fwrite 写入的二进制数据
    printf("File 'output.bin' has been created.\n");

    return 0;
}
```

### 运行分析

1. **fputc()**：它首先将字符 'A' 写入文件流的缓冲区
2. **fputs()**：它遍历 text_line 数组中的字符并写入，直到 `\0` 之前的所有字符，但只会写入 `\n` 结尾而不是 `\0`
3. **fwrite()**：它将 my_record 结构体的所有字节（`sizeof(Record)` 大小）原封不动地写入缓冲区。这使得它非常适合写入二进制数据
4. **fflush()**：在执行完前面的写入操作后，`fflush(fp)` 强制操作系统将缓冲区中的所有数据真正写入磁盘。在 fputs("Another line\n", fp) 之后再次调用 fflush()，可以保证这段数据也被立即写入
5. **fclose()**：最后，`fclose()` 在关闭文件时，也会自动执行一次缓冲区刷新操作。这个动作通常是隐式的

---

## 关键要点

- **fputc()**：写入单个字符
- **fputs()**：用于写入字符串端的字符串，非常适合二进制文件
- **fwrite()**：用于写入任意类型的数据块，非常适合二进制文件
- **fflush()**：用于在不关闭文件的情况下，强制将缓冲区中的数据同步到磁盘。这在需要数据安全或需要解释时操作系统刷新数据的场景中非常有用

### 函数对比

| 函数 | 写入单位 | 适用场景 | 特点 |
|------|---------|---------|------|
| `fputc()` | 单个字符 | 逐字符输出 | 简单直接 |
| `fputs()` | 字符串 | 文本文件写入 | 不写入 `\0`，不自动换行 |
| `fwrite()` | 数据块 | 二进制文件、结构体 | 精确控制写入字节数 |
| `fflush()` | - | 强制刷新缓冲区 | 确保数据持久化 |
