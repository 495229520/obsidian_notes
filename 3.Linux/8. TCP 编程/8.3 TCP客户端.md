# TCP 客户端

## 概述

TCP 客户端是网络通信中的"主动方"，它主动向服务器发起连接请求。客户端的编程流程比服务端简单，主要步骤包括：
1. 创建套接字
2. 连接到服务器（触发 [[7.5 TCP协议基础#TCP 三次握手（建立连接）|三次握手]]）
3. 与服务器进行数据交换
4. 关闭连接

---

## 客户端与服务端流程对比

```
        服务端                                客户端
           │                                    │
    ┌──────┴──────┐                            │
    │  socket()   │                     ┌──────┴──────┐
    └──────┬──────┘                     │  socket()   │
           │                            └──────┬──────┘
    ┌──────┴──────┐                            │
    │   bind()    │                            │
    └──────┬──────┘                            │
           │                                    │
    ┌──────┴──────┐                            │
    │  listen()   │                            │
    └──────┬──────┘                            │
           │                            ┌──────┴──────┐
    ┌──────┴──────┐ ←───── 建立连接 ────│  connect()  │
    │  accept()   │      （三次握手）    └──────┬──────┘
    └──────┬──────┘                            │
           │                                    │
    ┌──────┴──────┐                     ┌──────┴──────┐
    │read()/write()│ ←───── 数据 ──────→│write()/read()│
    └──────┬──────┘                     └──────┬──────┘
           │                                    │
    ┌──────┴──────┐                     ┌──────┴──────┐
    │   close()   │ ←───── 断开连接 ────│   close()   │
    └─────────────┘      （四次挥手）    └─────────────┘
```

![[Pasted image 20251218084709.png]]

**关键区别**：
- 服务端需要 `bind()` 绑定地址，客户端通常不需要（系统自动分配）
- 服务端需要 `listen()` 进入监听状态
- 服务端使用 `accept()` 接受连接，客户端使用 `connect()` 发起连接

## 客户端编程流程

```
┌──────────────────┐
│    socket()      │  创建套接字
└────────┬─────────┘
         ↓
┌──────────────────┐
│    connect()     │  连接服务器（触发三次握手）
└────────┬─────────┘
         ↓
┌──────────────────┐
│  write()/read()  │  数据收发
└────────┬─────────┘
         ↓
┌──────────────────┐
│     close()      │  关闭连接
└──────────────────┘
```

| 步骤 | 函数 | 说明 |
|:---:|:---|:---|
| 1 | [[7.6 套接字#socket 函数\|socket()]] | 创建套接字 |
| 2 | connect() | 连接到服务器 |
| 3 | write()/read() | 数据收发 |
| 4 | close() | 关闭连接 |

## connect 函数

`connect()` 函数用于客户端主动向服务器发起连接，它会触发 TCP 的三次握手过程。

### 函数原型

```c
#include <sys/socket.h>

int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```

### 参数说明

| 参数 | 说明 |
|:---|:---|
| `sockfd` | 由 [[7.6 套接字#socket 函数\|socket()]] 创建的套接字描述符 |
| `addr` | 指向服务器地址结构体的指针，包含服务器的 [[7.2 IP地址\|IP地址]] 和 [[7.3 端口\|端口号]] |
| `addrlen` | 地址结构体的大小 |

### 返回值

- **成功**：返回 0，表示连接建立成功
- **失败**：返回 -1，并设置 `errno`

### connect 的阻塞行为

默认情况下，`connect()` 是**阻塞**的：

1. 发送 SYN 报文给服务器
2. 等待服务器返回 SYN+ACK
3. 发送 ACK 完成三次握手
4. 握手成功后返回

如果服务器不可达或拒绝连接，`connect()` 会阻塞一段时间后返回错误。

### 常见错误

| errno | 说明 |
|:---|:---|
| `ECONNREFUSED` | 连接被拒绝（服务器未监听该端口） |
| `ETIMEDOUT` | 连接超时（服务器无响应） |
| `ENETUNREACH` | 网络不可达 |
| `EADDRINUSE` | 地址已被使用 |

## 完整客户端代码示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

#define BUFFER_SIZE 1024

int main() {
    int client;
    struct sockaddr_in server_addr;
    char buffer[BUFFER_SIZE];

    // ========== 1. 创建套接字 ==========
    client = socket(AF_INET, SOCK_STREAM, 0);
    if (client < 0) {
        perror("create socket failed");
        exit(1);
    }

    // ========== 2. 设置服务器地址 ==========
    memset(&server_addr, 0, sizeof(server_addr));
    server_addr.sin_family = AF_INET;
    server_addr.sin_port = htons(8080);                    // 服务器端口
    server_addr.sin_addr.s_addr = inet_addr("127.0.0.1");  // 服务器 IP

    // ========== 3. 连接服务器 ==========
    printf("正在连接服务器 %s:%d...\n", "127.0.0.1", 8080);

    if (connect(client, (struct sockaddr*)&server_addr, sizeof(server_addr)) < 0) {
        perror("connect failed");
        exit(1);
    }

    printf("连接成功！\n");

    // ========== 4. 接收数据 ==========
    ssize_t len = read(client, buffer, BUFFER_SIZE - 1);
    if (len < 0) {
        perror("read failed");
    } else if (len == 0) {
        printf("服务器关闭了连接\n");
    } else {
        buffer[len] = '\0';
        printf("收到服务器消息: %s", buffer);
    }

    // ========== 5. 关闭连接 ==========
    close(client);

    return 0;
}
```

## send 和 recv 函数

除了 `read()` 和 `write()`，TCP 编程中还可以使用 `send()` 和 `recv()` 函数进行数据传输，它们提供了更多的控制选项。

### send 函数

```c
#include <sys/socket.h>

ssize_t send(int sockfd, const void *buf, size_t len, int flags);
```

| 参数 | 说明 |
|:---|:---|
| `sockfd` | 套接字描述符 |
| `buf` | 待发送数据的缓冲区 |
| `len` | 数据长度 |
| `flags` | 发送标志，通常为 0 |

**常用 flags**：

| 标志 | 说明 |
|:---|:---|
| `0` | 默认行为，等同于 `write()` |
| `MSG_DONTWAIT` | 非阻塞发送 |
| `MSG_NOSIGNAL` | 不产生 SIGPIPE 信号 |

### recv 函数

```c
#include <sys/socket.h>

ssize_t recv(int sockfd, void *buf, size_t len, int flags);
```

| 参数 | 说明 |
|:---|:---|
| `sockfd` | 套接字描述符 |
| `buf` | 接收数据的缓冲区 |
| `len` | 缓冲区大小 |
| `flags` | 接收标志，通常为 0 |

**常用 flags**：

| 标志 | 说明 |
|:---|:---|
| `0` | 默认行为，等同于 `read()` |
| `MSG_PEEK` | 查看数据但不从队列中移除 |
| `MSG_WAITALL` | 等待接收完整的请求长度 |
| `MSG_DONTWAIT` | 非阻塞接收 |

### send/recv 与 write/read 的对比

| 函数 | 特点 |
|:---|:---|
| `write()`/`read()` | 通用的文件 I/O 函数，适用于所有文件描述符 |
| `send()`/`recv()` | 专门用于套接字，提供额外的 flags 参数 |

## 阻塞与非阻塞

### 阻塞模式（默认）

在阻塞模式下，I/O 操作会挂起进程直到完成：

- `connect()`：等待三次握手完成
- `read()`/`recv()`：等待数据到达
- `write()`/`send()`：等待发送缓冲区有空间

```c
// 阻塞模式下，read 会等待直到有数据可读
ssize_t n = read(sockfd, buffer, sizeof(buffer));
// 程序在这里暂停，直到：
// 1. 有数据到达
// 2. 连接关闭（返回 0）
// 3. 发生错误（返回 -1）
```

### 非阻塞模式

设置非阻塞模式后，I/O 操作会立即返回：

```c
#include <fcntl.h>

// 设置为非阻塞
int flags = fcntl(sockfd, F_GETFL, 0);
fcntl(sockfd, F_SETFL, flags | O_NONBLOCK);

// 非阻塞 read，如果没有数据立即返回 -1
ssize_t n = read(sockfd, buffer, sizeof(buffer));
if (n < 0 && (errno == EAGAIN || errno == EWOULDBLOCK)) {
    // 没有数据可读，稍后重试
}
```

## 常见的关键知识点

### 1. Socket API 概述

- `socket()`：创建 [[7.6 套接字|套接字]]，用于建立网络通信的端点
- `bind()`：绑定 [[7.2 IP地址|IP地址]] 和 [[7.3 端口|端口]]，服务端必须调用
- `listen()`：将套接字转为 [[7.7 listen和accept函数#listen 函数|监听]] 状态
- `accept()`：[[7.7 listen和accept函数#accept 函数|接受]] 客户端连接
- `connect()`：客户端连接服务器

### 2. TCP/IP 协议栈

- **TCP**：传输层协议，提供可靠的字节流传输
- **UDP**：传输层协议，提供不可靠的数据报传输
- 详见 [[7.5 TCP协议基础]]

### 3. 字节序转换

网络通信统一使用大端字节序（网络字节序）：

```c
htons()  // 主机 → 网络（16位）
htonl()  // 主机 → 网络（32位）
ntohs()  // 网络 → 主机（16位）
ntohl()  // 网络 → 主机（32位）
```

### 4. 地址转换

```c
inet_addr("192.168.1.1")   // 字符串 → 网络字节序 IP
inet_ntoa(addr.sin_addr)   // 网络字节序 IP → 字符串
```

---

## 关键要点

1. **客户端流程**：socket → connect → read/write → close
2. **connect 触发三次握手**：客户端调用 connect 时完成 TCP 连接建立
3. **阻塞 I/O**：默认情况下，connect、read、write 都是阻塞的
4. **send/recv vs write/read**：send/recv 提供更多控制选项（flags 参数）
5. **客户端无需 bind**：系统会自动为客户端分配临时端口
