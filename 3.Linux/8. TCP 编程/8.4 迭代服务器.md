# 迭代服务器

## 概述

**迭代服务器**（Iterative Server）是最简单的服务器模型。它同一时刻只能处理一个客户端连接，处理完当前客户端后才能接受下一个客户端的连接请求。

```
                    迭代服务器处理流程

  客户端A ─────→ ┌─────────────┐
                │   服务器    │ ──→ 处理A（其他客户端必须等待）
  客户端B ─────→ │  (单线程)   │
                │             │ ──→ 处理B
  客户端C ─────→ └─────────────┘
                               ──→ 处理C
```

### 特点

| 特点 | 说明 |
|:---|:---|
| **串行处理** | 一次只能服务一个客户端，其他客户端在队列中等待 |
| **实现简单** | 无需处理并发、同步等问题 |
| **适用场景** | 客户端数量少、请求处理时间短的场景 |
| **缺点** | 并发能力差，客户端可能长时间等待 |

## 服务端代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

void init_listen() {
    // 1. 创建套接字
    int client = socket(AF_INET, SOCK_STREAM, 0);

    // 2. 设置服务器地址结构
    struct sockaddr_in serveraddr;
    memset(&serveraddr, 0, sizeof(serveraddr));
    serveraddr.sin_family = AF_INET;
    serveraddr.sin_port = htons(9527);
    serveraddr.sin_addr.s_addr = inet_addr("127.0.0.1");

    // 3. 绑定地址
    int ret = bind(client, (struct sockaddr*)&serveraddr, sizeof(serveraddr));
    if (ret == -1) {
        std::cout << "bind failed!" << std::endl;
        close(client);
        return;
    }

    // 4. 开始监听
    ret = listen(client, 5);
    if (ret == -1) {
        std::cout << "listen failed!" << std::endl;
        close(client);
        return;
    }

    printf("%s %s, %d ____ %s____, %p\n", __FILE__, __LINE__, __FUNCTION__,
           "server is listening...");

    // 5. 循环接受连接
    char buffer[256];
    while (1) {
        struct sockaddr_in clientaddr;
        socklen_t clientlen = sizeof(clientaddr);

        // accept 阻塞等待客户端连接
        int connfd = accept(client, (struct sockaddr*)&clientaddr, &clientlen);
        if (connfd == -1) {
            std::cout << "accept failed!" << std::endl;
            continue;
        }

        // 打印客户端信息
        printf("client connected: %s:%d\n",
               inet_ntoa(clientaddr.sin_addr),
               ntohs(clientaddr.sin_port));

        // 6. 与客户端通信
        memset(buffer, 0, sizeof(buffer));
        int len = recv(connfd, buffer, sizeof(buffer), 0);
        if (len > 0) {
            printf("recv: %s\n", buffer);
            // 将收到的消息发送回客户端
            send(connfd, buffer, len, 0);
        }

        // 7. 关闭连接
        close(connfd);
        printf("client disconnected\n");
    }

    close(client);
}

int main() {
    init_listen();
    return 0;
}
```

## 客户端代码

```c
void send_msg(int client) {
    // 获取服务器地址结构
    struct sockaddr_in serveraddr;
    memset(&serveraddr, 0, sizeof(serveraddr));
    serveraddr.sin_family = AF_INET;
    serveraddr.sin_port = htons(9527);
    serveraddr.sin_addr.s_addr = inet_addr("127.0.0.1");

    // 连接到服务器
    int ret = connect(client, (struct sockaddr*)&serveraddr, sizeof(serveraddr));
    if (ret == -1) {
        std::cout << "connect failed!" << std::endl;
        return;
    }

    // 发送消息
    char buffer[256] = "hello, this is client";
    send(client, buffer, strlen(buffer), 0);

    // 接收服务器回复
    memset(buffer, 0, sizeof(buffer));
    recv(client, buffer, sizeof(buffer), 0);
    printf("server reply: %s\n", buffer);
}

int send_client() {
    // 创建套接字
    int client = socket(AF_INET, SOCK_STREAM, 0);
    if (client == -1) {
        std::cout << "create socket failed!" << std::endl;
        return -1;
    }

    send_msg(client);
    close(client);
    return 0;
}
```

## 代码结构分析

### 服务端流程

服务端遵循标准的 [[7.7 listen和accept函数|TCP 服务器编程流程]]：

1. **[[7.6 套接字#socket 函数|socket()]]**：创建 TCP [[7.6 套接字|套接字]]
2. **[[7.6 套接字#bind 函数|bind()]]**：绑定服务器地址（IP + [[7.3 端口|端口]]）
3. **[[7.7 listen和accept函数#listen 函数|listen()]]**：开始监听，backlog 设为 5
4. **循环**：
   - **[[7.7 listen和accept函数#accept 函数|accept()]]**：阻塞等待客户端连接（[[7.5 TCP协议基础#TCP 三次握手（建立连接）|三次握手]]完成后返回）
   - **recv()**：接收客户端数据
   - **send()**：发送响应
   - **close()**：关闭已连接套接字

### 关键点

- **迭代特性**：`accept()` 返回后，服务器处理当前客户端，处理完成后才再次调用 `accept()` 接受下一个连接
- **阻塞行为**：`accept()` 和 `recv()` 都是阻塞调用
- **单个客户端**：同一时刻只服务一个客户端

### 客户端流程

1. **socket()**：创建 TCP 套接字
2. **connect()**：连接到服务器（触发[[7.5 TCP协议基础#TCP 三次握手（建立连接）|三次握手]]）
3. **send()**：发送数据
4. **recv()**：接收服务器响应
5. **close()**：关闭连接

## 详细运行流程（Step-by-Step）

### 1. 服务器启动

```
服务器                              内核
   │                                 │
   │  socket(AF_INET, SOCK_STREAM, 0)│
   │ ────────────────────────────────→│ 创建套接字，返回 fd
   │                                 │
   │  bind(fd, addr, len)            │
   │ ────────────────────────────────→│ 绑定地址到套接字
   │                                 │
   │  listen(fd, 5)                  │
   │ ────────────────────────────────→│ 套接字转为监听状态
   │                                 │ 创建未完成/已完成连接队列
   │                                 │
   │  accept(fd, ...)                │
   │ ────────────────────────────────→│ 阻塞，等待连接
   │   [阻塞中...]                    │
```

### 2. 客户端连接

```
客户端                   内核                    服务器
   │                      │                        │
   │  socket()            │                        │
   │ ─────────────────────→                        │
   │                      │                        │
   │  connect(server_addr)│                        │
   │ ─────────────────────→                        │
   │                      │── SYN ─────────────────→│
   │                      │←─ SYN+ACK ─────────────│
   │                      │── ACK ─────────────────→│
   │                      │                        │ 连接放入已完成队列
   │  connect() 返回      │                        │ accept() 返回 connfd
```

### 3. 数据传输

```
客户端                                          服务器
   │                                              │
   │  send("hello, this is client")               │
   │ ─────────────────────────────────────────────→│
   │                                              │ recv() 返回数据
   │                                              │ 打印: "recv: hello, this is client"
   │                                              │
   │←─────────────────────────────────────────────│ send() 回传数据
   │  recv() 返回 "hello, this is client"         │
```

### 4. 连接关闭

服务器调用 `close(connfd)` 触发 [[7.5 TCP协议基础#TCP 四次挥手（断开连接）|四次挥手]]，然后重新进入 `accept()` 等待下一个客户端。

## buffer 的传递机制解析

在 TCP 通信中，数据的传输涉及多层缓冲区：

```
       应用层                    内核层                    网络
┌─────────────────┐      ┌─────────────────┐
│  用户空间 buffer │      │  发送缓冲区       │
│  char buf[256]  │ ───→ │ (内核维护)        │ ───→  网络传输
└─────────────────┘      └─────────────────┘
                         ┌─────────────────┐
                         │  接收缓冲区       │
        ←───────────────│ (内核维护)         │ ←───  网络传输
                         └─────────────────┘
```

### 发送过程

1. 用户调用 `send(connfd, buffer, len, 0)`
2. 数据从用户空间 `buffer` **复制**到内核的发送缓冲区
3. `send()` 返回（不等待对方收到）
4. 内核负责将数据发送给对端

### 接收过程

1. 网络数据到达，内核将其放入接收缓冲区
2. 用户调用 `recv(connfd, buffer, sizeof(buffer), 0)`
3. 数据从内核接收缓冲区**复制**到用户空间 `buffer`
4. `recv()` 返回实际读取的字节数

### 注意事项

- `send()` 返回成功只表示数据已复制到内核缓冲区，不保证对方已收到
- `recv()` 可能返回比请求更少的字节数（TCP 是字节流协议）
- 缓冲区满时，`send()` 可能阻塞；缓冲区空时，`recv()` 会阻塞

## 关键要点

1. **迭代服务器的特点**：
   - 同一时刻只处理一个客户端
   - 实现简单，但并发能力差

2. **与并发服务器的对比**：
   - 迭代服务器：串行处理，适合简单场景
   - 并发服务器：使用 [[5.1 系统进程控制函数#1. fork() - 创建子进程|fork()]] 或多线程并行处理多个客户端

3. **TCP 编程流程**：
   - 服务端：socket → bind → listen → accept → recv/send → close
   - 客户端：socket → connect → send/recv → close

4. **阻塞点**：
   - `accept()` 等待新连接
   - `recv()` 等待数据到达
