# TCP 服务端

## 概述

TCP 服务端是网络通信中的"被动方"，它需要先启动并等待客户端的连接请求。服务端的核心职责是：
1. 绑定到指定的 [[7.2 IP地址|IP地址]] 和 [[7.3 端口|端口]]
2. 监听客户端的连接请求
3. 接受连接并与客户端进行数据交换

---

## 服务端编程流程

TCP 服务端的编程遵循固定的步骤：

```
┌──────────────────┐
│    socket()      │  创建套接字
└────────┬─────────┘
         ↓
┌──────────────────┐
│     bind()       │  绑定地址和端口
└────────┬─────────┘
         ↓
┌──────────────────┐
│    listen()      │  开始监听
└────────┬─────────┘
         ↓
┌──────────────────┐
│    accept()      │  接受客户端连接（阻塞等待）
└────────┬─────────┘
         ↓
┌──────────────────┐
│  read()/write()  │  数据收发
└────────┬─────────┘
         ↓
┌──────────────────┐
│     close()      │  关闭连接
└──────────────────┘
```

| 步骤 | 函数 | 说明 |
|:---:|:---|:---|
| 1 | [[7.6 套接字#socket 函数\|socket()]] | 创建套接字，返回文件描述符 |
| 2 | [[7.6 套接字#bind 函数\|bind()]] | 将套接字与 IP 地址和端口绑定 |
| 3 | [[7.7 listen和accept函数#listen 函数\|listen()]] | 将套接字设置为监听状态 |
| 4 | [[7.7 listen和accept函数#accept 函数\|accept()]] | 从已完成连接队列中取出一个连接 |
| 5 | read()/write() | 与客户端进行数据传输 |
| 6 | close() | 关闭套接字，释放资源 |

## 完整服务端代码示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

int main() {
    int server, client;
    struct sockaddr_in server_addr, client_addr;
    socklen_t client_len;
    char message[] = "Hello World!\n";

    // ========== 1. 创建套接字 ==========
    server = socket(AF_INET, SOCK_STREAM, 0);
    if (server < 0) {
        perror("create socket failed");
        exit(1);
    }

    // ========== 2. 绑定地址和端口 ==========
    memset(&server_addr, 0, sizeof(server_addr));
    server_addr.sin_family = AF_INET;                    // IPv4
    server_addr.sin_port = htons(8080);                  // 端口 8080
    server_addr.sin_addr.s_addr = inet_addr("127.0.0.1"); // 本地地址

    if (bind(server, (struct sockaddr*)&server_addr, sizeof(server_addr)) < 0) {
        perror("bindAddress failed");
        exit(1);
    }

    // ========== 3. 开始监听 ==========
    if (listen(server, 128) < 0) {
        perror("listen failed");
        exit(1);
    }

    printf("服务器已启动，等待客户端连接...\n");

    // ========== 4. 接受客户端连接 ==========
    client_len = sizeof(client_addr);
    client = accept(server, (struct sockaddr*)&client_addr, &client_len);
    if (client < 0) {
        perror("accept failed");
        exit(1);
    }

    printf("客户端已连接: %s:%d\n",
           inet_ntoa(client_addr.sin_addr),
           ntohs(client_addr.sin_port));

    // ========== 5. 发送数据 ==========
    ssize_t len = write(client, message, strlen(message));
    if (len < 0) {
        perror("write failed");
    } else {
        printf("发送了 %zd 字节数据\n", len);
    }

    // ========== 6. 关闭连接 ==========
    close(client);
    close(server);

    return 0;
}
```

## 关键函数详解

### 1. socket() - 创建套接字

```c
int socket(int domain, int type, int protocol);
```

- `domain`：地址族，使用 `AF_INET` 表示 IPv4
- `type`：套接字类型，使用 `SOCK_STREAM` 表示 TCP
- `protocol`：通常设为 0，让系统自动选择协议

详细说明参见 [[7.6 套接字#socket 函数]]。

### 2. bind() - 绑定地址

```c
int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```

将套接字与指定的 IP 地址和端口绑定。服务器必须调用此函数，否则客户端无法知道连接到哪里。

**注意事项**：
- 端口号需要使用 `htons()` 转换为网络字节序
- IP 地址需要使用 `inet_addr()` 或 `htonl()` 转换
- 使用 `INADDR_ANY`（即 0.0.0.0）可以监听所有网卡

详细说明参见 [[7.6 套接字#bind 函数]]。

### 3. listen() - 开始监听

```c
int listen(int sockfd, int backlog);
```

- `backlog`：等待连接队列的最大长度
- 调用后，套接字从主动模式变为被动模式（监听模式）

详细说明参见 [[7.7 listen和accept函数#listen 函数]]。

### 4. accept() - 接受连接

```c
int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
```

`accept()` 是一个**阻塞函数**：
- 如果已完成连接队列中有等待的连接，立即返回
- 如果队列为空，会阻塞等待直到有新连接到来

**返回值**：返回一个**新的套接字描述符**，用于与客户端通信。

详细说明参见 [[7.7 listen和accept函数#accept 函数]]。

## 两种套接字的区别

服务端程序中存在两种不同用途的套接字：

| 类型 | 描述 | 用途 |
|:---|:---|:---|
| **监听套接字** | `socket()` 创建，经过 `bind()` 和 `listen()` | 只用于监听新连接，不传输数据 |
| **已连接套接字** | `accept()` 返回 | 与特定客户端进行数据传输 |

```
                            ┌─────────────────────────┐
       新客户端 ──────────→  │     监听套接字           │
                            │   (server socket)       │
                            └───────────┬─────────────┘
                                        │ accept()
                                        ↓
                            ┌─────────────────────────┐
       已连接客户端 ←────→   │    已连接套接字          │
                            │  (connected socket)     │
                            └─────────────────────────┘
```

**生命周期**：
- 监听套接字：随服务器启动创建，服务器关闭时销毁
- 已连接套接字：每次 `accept()` 创建一个，通信结束后关闭

---

## accept 函数的阻塞行为

`accept()` 默认是阻塞的，这意味着：

1. **有等待连接时**：立即返回新的套接字描述符
2. **无等待连接时**：进程被挂起，直到有客户端连接

```c
// accept 会在这里阻塞，等待客户端连接
client = accept(server, (struct sockaddr*)&client_addr, &client_len);
// 只有当客户端连接后，程序才会继续执行
```

### 非阻塞模式

可以通过设置套接字为非阻塞模式来改变这一行为：

```c
#include <fcntl.h>

// 设置为非阻塞
int flags = fcntl(server, F_GETFL, 0);
fcntl(server, F_SETFL, flags | O_NONBLOCK);
```

非阻塞模式下，如果没有等待的连接，`accept()` 会立即返回 -1，并设置 `errno` 为 `EAGAIN` 或 `EWOULDBLOCK`。

## 常见问题

### 1. "Address already in use" 错误

当服务器重启时，可能会遇到这个错误。原因是之前的连接可能还处于 [[10.2 TIME_WAIT状态|TIME_WAIT]] 状态。

**解决方案**：设置 [[10.1 套接字选项#SO_REUSEADDR|地址复用]] 选项

```c
int opt = 1;
setsockopt(server, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
```

详细原理和代码示例参见 [[10.2 TIME_WAIT状态#SO_REUSEADDR 解决方案]]。

### 2. 端口选择建议

- 开发测试：使用 8080、9000 等常见的开发端口
- 生产环境：根据服务类型选择合适的端口
- 避免使用 [[7.3 端口#1. 知名端口（Well-known Ports）：0-1023|知名端口]]（0-1023），除非有 root 权限

---

## 关键要点

1. **服务端流程**：socket → bind → listen → accept → read/write → close
2. **两种套接字**：监听套接字用于接收新连接，已连接套接字用于数据传输
3. **accept 阻塞**：默认情况下 accept 会阻塞等待客户端连接
4. **地址复用**：使用 `SO_REUSEADDR` 避免重启时的地址冲突问题
