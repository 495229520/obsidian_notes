# 回声服务器（Echo Server）

## 概述

**回声服务器**（Echo Server）是一种经典的网络编程练习模型：客户端发送什么数据，服务器就原样返回什么数据。这就像山谷中的回声一样，因此得名。

```
客户端                             服务器
   │                                  │
   │  "Hello, World!"                 │
   │ ─────────────────────────────────→│
   │                                  │
   │  "Hello, World!"                 │
   │←─────────────────────────────────│
```

回声服务器虽然简单，但它是学习网络编程的重要基础，可以帮助理解 TCP 通信的核心概念。

## TCP 字节流特性与数据接收问题

### 问题：一次性发送 1200 字节会发生什么？

假设客户端一次性发送 1200 字节的数据，服务器调用 `recv()` 时能保证一次性收到完整的 1200 字节吗？

**答案是：不一定！**

### 原因分析

TCP 是**字节流**协议（参见 [[7.5 TCP协议基础#TCP 的核心特性|TCP 核心特性]]），具有以下特点：

1. **无消息边界**：TCP 不保留应用层的消息边界，数据被视为连续的字节流
2. **可能分片**：发送的数据可能被拆分成多个 TCP 段传输
3. **接收缓冲区**：`recv()` 返回的是当前接收缓冲区中**已有**的数据，可能少于请求的字节数

```
发送端                              接收端

send(1200字节)
       │
       │──── TCP段1 (500字节) ─────────→│ recv() 可能只返回 500 字节
       │                               │
       │──── TCP段2 (400字节) ─────────→│ 再次 recv() 返回 400 字节
       │                               │
       │──── TCP段3 (300字节) ─────────→│ 再次 recv() 返回 300 字节
```

### 解决方案

要确保接收完整的数据，需要**循环读取**直到收够预期的字节数：

```c
// 循环读取直到收够 total_len 字节
int read_len = 0;
while (read_len < total_len) {
    int len = recv(client, buffer + read_len, total_len - read_len, 0);
    if (len <= 0) {
        break;  // 出错或连接关闭
    }
    read_len += len;
}
```

## 完整代码实现

### 服务端代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

void run_client(int connfd) {
    // 1. 创建服务器地址结构（这里用于记录信息）
    int client = socket(AF_INET, SOCK_STREAM, 0);
    struct sockaddr_in serveraddr;
    memset(&serveraddr, 0, sizeof(serveraddr));
    serveraddr.sin_family = AF_INET;
    serveraddr.sin_port = htons(9527);
    serveraddr.sin_addr.s_addr = inet_addr("127.0.0.1");

    int ret = connect(client, (struct sockaddr*)&serveraddr, sizeof(serveraddr));
    if (ret == -1) {
        printf("%s %d: %s: connect failed!\n", __FILE__, __LINE__, __FUNCTION__);
        return;
    }

    // 2. 发送数据给服务器
    char buffer[1024];
    memset(buffer, 0, sizeof(buffer));
    sprintf(buffer, "%s", "hello, this is client message!");
    send(client, buffer, strlen(buffer), 0);

    // 3. 接收服务器回复（循环读取）
    memset(buffer, 0, sizeof(buffer));

    // 注意：这里演示循环读取的方式
    // len 表示还需要读取的数据长度
    // read_len 表示本次读取后的累计长度
    int len = sizeof(buffer);
    int read_len = 0;

    while (1) {
        len = recv(client, buffer + read_len, len - read_len, 0);
        if (len <= 0) {
            printf("recv failed or connection closed\n");
            break;
        }
        read_len += len;

        // 检查是否收到完整消息（这里以收到数据为准）
        if (read_len > 0) {
            printf("recv from server: %s\n", buffer);
            break;
        }
    }

    close(client);
}

void srv_conn() {
    // 1. 创建套接字
    int client = socket(AF_INET, SOCK_STREAM, 0);

    // 2. 设置服务器地址
    struct sockaddr_in serveraddr;
    memset(&serveraddr, 0, sizeof(serveraddr));
    serveraddr.sin_family = AF_INET;
    serveraddr.sin_port = htons(9527);
    serveraddr.sin_addr.s_addr = inet_addr("127.0.0.1");

    // 3. 绑定地址
    int ret = bind(client, (struct sockaddr*)&serveraddr, sizeof(serveraddr));
    if (ret == -1) {
        printf("%s %d %s: bind failed!\n", __FILE__, __LINE__, __FUNCTION__);
        close(client);
        return;
    }

    // 4. 开始监听
    ret = listen(client, 5);
    if (ret == -1) {
        printf("listen failed!\n");
        close(client);
        return;
    }

    printf("Server is listening on port 9527...\n");

    // 5. 接受连接并处理
    char buffer[1024];
    while (1) {
        struct sockaddr_in clientaddr;
        socklen_t clientlen = sizeof(clientaddr);

        int connfd = accept(client, (struct sockaddr*)&clientaddr, &clientlen);
        if (connfd == -1) {
            printf("accept failed!\n");
            continue;
        }

        printf("Client connected: %s:%d\n",
               inet_ntoa(clientaddr.sin_addr),
               ntohs(clientaddr.sin_port));

        // 6. 循环接收并回传数据（回声）
        memset(buffer, 0, sizeof(buffer));
        int read_len = 0;

        while (1) {
            int len = recv(connfd, buffer + read_len, sizeof(buffer) - read_len, 0);
            if (len <= 0) {
                printf("Client disconnected\n");
                break;
            }

            read_len += len;
            printf("recv: %s (total %d bytes)\n", buffer, read_len);

            // 回声：将收到的数据原样发送回去
            send(connfd, buffer, read_len, 0);

            // 重置缓冲区，准备接收下一条消息
            memset(buffer, 0, sizeof(buffer));
            read_len = 0;
        }

        close(connfd);
    }

    close(client);
}

int init_listen() {
    srv_conn();
    return 0;
}
```

### 客户端代码

```c
void send_msg(int client) {
    struct sockaddr_in serveraddr;
    memset(&serveraddr, 0, sizeof(serveraddr));
    serveraddr.sin_family = AF_INET;
    serveraddr.sin_port = htons(9527);
    serveraddr.sin_addr.s_addr = inet_addr("127.0.0.1");

    int ret = connect(client, (struct sockaddr*)&serveraddr, sizeof(serveraddr));
    if (ret == -1) {
        printf("connect failed!\n");
        return;
    }

    // 发送消息
    char buffer[1024] = "Hello, Echo Server!";
    send(client, buffer, strlen(buffer), 0);

    // 接收回声
    memset(buffer, 0, sizeof(buffer));
    int len = recv(client, buffer, sizeof(buffer), 0);
    if (len > 0) {
        printf("Echo received: %s\n", buffer);
    }
}

int send_client() {
    int client = socket(AF_INET, SOCK_STREAM, 0);
    if (client == -1) {
        printf("create socket failed!\n");
        return -1;
    }

    send_msg(client);
    close(client);
    return 0;
}
```

## 详细执行流程

### 1. 服务器启动阶段

```
服务器进程
    │
    ├── socket() ──→ 创建监听套接字 (fd=3)
    │
    ├── bind() ──→ 绑定到 127.0.0.1:9527
    │
    ├── listen() ──→ 开始监听，backlog=5
    │
    └── accept() ──→ [阻塞等待客户端连接...]
```

### 2. 客户端连接阶段

```
客户端                                          服务器
   │                                               │
   │  socket() 创建套接字                           │
   │                                               │
   │  connect(127.0.0.1:9527)                      │
   │ ──────────────────────────────────────────────→│
   │              三次握手                          │ accept() 返回 connfd
   │                                               │
   │  send("Hello, Echo Server!")                  │
   │ ──────────────────────────────────────────────→│ recv() 接收数据
   │                                               │ 打印: "recv: Hello, Echo Server!"
   │                                               │
   │  recv()                                       │ send() 回传数据
   │←──────────────────────────────────────────────│
   │  打印: "Echo received: Hello, Echo Server!"   │
```

### 3. 数据收发的字节流特性

由于 TCP 是字节流协议，在实际场景中：

```
发送端调用:  send(buffer, 1200, 0)

可能的网络传输情况:

     TCP段1          TCP段2          TCP段3
  [───500B───]   [───400B───]   [───300B───]
       │               │               │
       ↓               ↓               ↓
  ┌────────────────────────────────────────┐
  │          接收端 TCP 接收缓冲区          │
  │  [500B] → [500B + 400B] → [1200B]      │
  └────────────────────────────────────────┘
       │
       ↓
  recv() 返回缓冲区中当前可用的数据
  - 第一次 recv() 可能返回 500
  - 第二次 recv() 可能返回 400
  - 第三次 recv() 可能返回 300
```

### 4. 循环读取的必要性

```c
// 错误示例：假设一次 recv 能读完所有数据
char buffer[1200];
int len = recv(client, buffer, 1200, 0);
// len 可能 < 1200！

// 正确示例：循环读取直到收够
char buffer[1200];
int total_read = 0;
while (total_read < 1200) {
    int len = recv(client, buffer + total_read, 1200 - total_read, 0);
    if (len <= 0) break;
    total_read += len;
}
```

## 回声服务器的变体

### 1. 使用 [[5.1 系统进程控制函数#1. fork() - 创建子进程|fork()]] 的并发版本

```c
while (1) {
    int connfd = accept(listen_fd, ...);

    pid_t pid = fork();
    if (pid == 0) {
        // 子进程处理客户端
        close(listen_fd);  // 子进程不需要监听套接字
        handle_client(connfd);
        exit(0);
    } else {
        // 父进程继续监听
        close(connfd);  // 父进程不需要已连接套接字
    }
}
```

### 2. 设置超时的版本

```c
// 设置接收超时
struct timeval timeout;
timeout.tv_sec = 5;   // 5 秒超时
timeout.tv_usec = 0;
setsockopt(connfd, SOL_SOCKET, SO_RCVTIMEO, &timeout, sizeof(timeout));
```

## 关键要点

1. **回声服务器原理**：
   - 接收客户端数据
   - 原样返回给客户端
   - 简单但能验证网络通信的正确性

2. **TCP 字节流特性**（参见 [[7.5 TCP协议基础#TCP 的核心特性|TCP 核心特性]]）：
   - 无消息边界
   - `recv()` 可能返回部分数据
   - 必须循环读取确保数据完整

3. **循环读取模式**：
   ```c
   while (read_len < expected_len) {
       int len = recv(fd, buf + read_len, expected_len - read_len, 0);
       if (len <= 0) break;
       read_len += len;
   }
   ```

4. **与 [[8.4 迭代服务器]] 的关系**：
   - 回声服务器是迭代服务器的一种实现
   - 核心区别在于业务逻辑：回声服务器原样返回数据
