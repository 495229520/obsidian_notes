# poll 模型

## 1. poll 函数概述

`poll()` 是 [[13.1 IO复用简介|I/O 复用]] 的另一种实现，它是对 [[13.2 select模型|select]] 的改进，解决了 select 的一些限制问题。`poll()` 源自 System V Unix，在 POSIX 标准中也有定义。

---

## 2. poll 与 select 的对比

| 特性 | select | poll |
|:---|:---|:---|
| fd 数量限制 | 固定 1024（FD_SETSIZE） | 无固定限制（取决于系统资源） |
| fd 集合表示 | 位图 fd_set | 动态数组 pollfd |
| 事件与返回分离 | 否（fd_set 被修改） | 是（events 和 revents 分开） |
| 可监控事件类型 | 读/写/异常 | 更丰富的事件类型 |
| 跨平台性 | 更好（Windows 支持） | 较好（Windows 不原生支持） |

---

## 3. poll 函数原型

```c
#include <poll.h>

int poll(struct pollfd *fds, nfds_t nfds, int timeout);
```

### 3.1 参数说明

| 参数 | 说明 |
|:---|:---|
| `fds` | 指向 `pollfd` 结构体数组的指针 |
| `nfds` | 数组中元素的数量 |
| `timeout` | 超时时间（毫秒） |

### 3.2 返回值

| 返回值 | 含义 |
|:---|:---|
| `> 0` | 就绪的文件描述符数量 |
| `0` | 超时，没有任何 fd 就绪 |
| `-1` | 出错，设置 `errno` |

### 3.3 timeout 参数

| 值 | 效果 |
|:---|:---|
| `-1` | 永久阻塞，直到有 fd 就绪 |
| `0` | 立即返回（非阻塞检查） |
| `> 0` | 等待指定的毫秒数 |

---

## 4. pollfd 结构体

```c
struct pollfd {
    int   fd;       // 要监控的文件描述符
    short events;   // 请求监控的事件（输入参数）
    short revents;  // 实际发生的事件（输出参数）
};
```

### 4.1 字段说明

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `fd` | int | 要监控的[[4.1 打开、读取、写入、关闭#文件描述符（File Descriptor）|文件描述符]]，设为负数则忽略该项 |
| `events` | short | 请求监控的事件位掩码 |
| `revents` | short | 内核返回的实际就绪事件（`poll()` 填充） |

### 4.2 events/revents 可用事件

| 事件常量 | 说明 | events | revents |
|:---|:---|:---|:---|
| `POLLIN` | 有数据可读 | ✓ | ✓ |
| `POLLPRI` | 有紧急数据可读（如 TCP 带外数据） | ✓ | ✓ |
| `POLLOUT` | 可以写数据（不会阻塞） | ✓ | ✓ |
| `POLLRDHUP` | 对端关闭连接或关闭写端（Linux 2.6.17+） | ✓ | ✓ |
| `POLLERR` | 发生错误 | - | ✓ |
| `POLLHUP` | 挂起（如管道写端关闭） | - | ✓ |
| `POLLNVAL` | fd 无效（未打开） | - | ✓ |

> **注意**：`POLLERR`、`POLLHUP`、`POLLNVAL` 只能在 `revents` 中返回，不能在 `events` 中设置。

### 4.3 常用事件组合

```c
// 监控可读事件
fds[i].events = POLLIN;

// 监控可读和可写事件
fds[i].events = POLLIN | POLLOUT;

// 监控所有读相关事件
fds[i].events = POLLIN | POLLPRI | POLLRDHUP;
```

## 5. poll 工作流程

```
1. 初始化 pollfd 数组
   ┌─────────────────────────────────────┐
   │ fds[0].fd = listen_fd;              │
   │ fds[0].events = POLLIN;             │
   │ fds[1].fd = client1_fd;             │
   │ fds[1].events = POLLIN;             │
   │ ...                                 │
   └─────────────────┬───────────────────┘
                     ↓
2. 调用 poll()
   ┌─────────────────────────────────────┐
   │ ret = poll(fds, nfds, timeout);     │
   │                                     │
   │ 内核遍历所有 fd，检查状态            │
   │ 将结果写入 revents                  │
   └─────────────────┬───────────────────┘
                     ↓
3. 返回后检查就绪的 fd
   ┌─────────────────────────────────────┐
   │ for (i = 0; i < nfds; i++) {        │
   │     if (fds[i].revents & POLLIN) {  │
   │         // 处理可读事件             │
   │     }                               │
   │     if (fds[i].revents & POLLOUT) { │
   │         // 处理可写事件             │
   │     }                               │
   │ }                                   │
   └─────────────────────────────────────┘
```

> **优势**：与 select 不同，poll 的 `events` 字段不会被修改，无需每次重新设置。

## 6. 完整示例代码

### 6.1 使用 poll 的 TCP 服务器

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <poll.h>
#include <netinet/in.h>
#include <arpa/inet.h>

#define PORT 8080
#define MAX_CLIENTS 100
#define BUFFER_SIZE 1024

int main() {
    int listen_fd, conn_fd;
    struct pollfd fds[MAX_CLIENTS + 1];  // +1 用于监听套接字
    int nfds = 1;  // 当前监控的 fd 数量
    char buffer[BUFFER_SIZE];
    struct sockaddr_in server_addr, client_addr;
    socklen_t addr_len = sizeof(client_addr);

    // 1. 创建监听套接字
    listen_fd = socket(AF_INET, SOCK_STREAM, 0);
    if (listen_fd < 0) {
        perror("socket");
        exit(EXIT_FAILURE);
    }

    // 设置地址复用
    int opt = 1;
    setsockopt(listen_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

    // 2. 绑定地址
    memset(&server_addr, 0, sizeof(server_addr));
    server_addr.sin_family = AF_INET;
    server_addr.sin_addr.s_addr = INADDR_ANY;
    server_addr.sin_port = htons(PORT);

    if (bind(listen_fd, (struct sockaddr*)&server_addr,
             sizeof(server_addr)) < 0) {
        perror("bind");
        exit(EXIT_FAILURE);
    }

    // 3. 开始监听
    if (listen(listen_fd, 5) < 0) {
        perror("listen");
        exit(EXIT_FAILURE);
    }

    // 4. 初始化 pollfd 数组
    // 第一个元素用于监听套接字
    fds[0].fd = listen_fd;
    fds[0].events = POLLIN;

    // 初始化其他位置
    for (int i = 1; i <= MAX_CLIENTS; i++) {
        fds[i].fd = -1;  // -1 表示未使用
    }

    printf("服务器启动，监听端口 %d...\n", PORT);

    // 5. 主循环
    while (1) {
        // 调用 poll
        int ret = poll(fds, nfds, -1);  // -1 表示永久等待

        if (ret < 0) {
            perror("poll");
            break;
        }

        // 6. 检查监听套接字
        if (fds[0].revents & POLLIN) {
            conn_fd = accept(listen_fd,
                            (struct sockaddr*)&client_addr,
                            &addr_len);
            if (conn_fd < 0) {
                perror("accept");
                continue;
            }

            printf("新连接: %s:%d, fd=%d\n",
                   inet_ntoa(client_addr.sin_addr),
                   ntohs(client_addr.sin_port),
                   conn_fd);

            // 添加新客户端到 pollfd 数组
            int added = 0;
            for (int i = 1; i <= MAX_CLIENTS; i++) {
                if (fds[i].fd < 0) {
                    fds[i].fd = conn_fd;
                    fds[i].events = POLLIN;
                    if (i >= nfds) {
                        nfds = i + 1;
                    }
                    added = 1;
                    break;
                }
            }

            if (!added) {
                printf("连接数已满，拒绝连接\n");
                close(conn_fd);
            }
        }

        // 7. 检查所有客户端套接字
        for (int i = 1; i < nfds; i++) {
            if (fds[i].fd < 0) {
                continue;
            }

            // 检查可读事件
            if (fds[i].revents & POLLIN) {
                ssize_t bytes_read = read(fds[i].fd, buffer, BUFFER_SIZE - 1);

                if (bytes_read <= 0) {
                    // 客户端断开连接
                    printf("客户端断开: fd=%d\n", fds[i].fd);
                    close(fds[i].fd);
                    fds[i].fd = -1;
                } else {
                    // 回显数据
                    buffer[bytes_read] = '\0';
                    printf("收到数据 [fd=%d]: %s", fds[i].fd, buffer);
                    write(fds[i].fd, buffer, bytes_read);
                }
            }

            // 检查错误事件
            if (fds[i].revents & (POLLERR | POLLHUP | POLLNVAL)) {
                printf("fd=%d 发生错误，关闭连接\n", fds[i].fd);
                close(fds[i].fd);
                fds[i].fd = -1;
            }
        }
    }

    close(listen_fd);
    return 0;
}
```

## 7. poll 的优缺点

### 7.1 优点

| 优点 | 说明 |
|:---|:---|
| **无 fd 数量限制** | 理论上可监控任意数量的 fd（受系统资源限制） |
| **events/revents 分离** | 无需每次重新设置监控事件 |
| **更丰富的事件类型** | 支持更多事件（如 `POLLRDHUP`） |
| **接口更清晰** | pollfd 结构更直观易用 |

### 7.2 缺点

| 缺点 | 说明 |
|:---|:---|
| **线性扫描** | 每次调用仍需遍历所有 fd，O(n) 复杂度 |
| **数据拷贝** | 每次调用需要将 pollfd 数组拷贝到内核空间 |
| **无法直接获取就绪 fd** | 仍需遍历整个数组找出就绪的 fd |
| **跨平台性稍差** | Windows 不原生支持 |

### 7.3 性能特点

```
poll 的时间复杂度：

1. 用户态 → 内核态拷贝
   拷贝 nfds 个 pollfd 结构体
   开销：O(n)

2. 内核遍历所有 fd
   检查每个 fd 的状态
   时间复杂度：O(n)

3. 用户态再次遍历
   找出 revents 非零的 fd
   时间复杂度：O(n)

总体时间复杂度：O(n)
与 select 相同，但无 fd 数量限制
```

## 8. poll 与 select 的选择建议

| 场景 | 建议 |
|:---|:---|
| 需要跨平台（包括 Windows） | 选择 select |
| fd 数量可能超过 1024 | 选择 poll |
| Linux 高并发服务 | 选择 [[13.4 epoll模型\|epoll]] |
| 需要丰富的事件类型 | 选择 poll |

---

## 9. 关键要点

1. **poll 是 select 的改进版**，解决了 fd 数量限制问题
2. **pollfd 结构体**：`fd`（文件描述符）、`events`（请求事件）、`revents`（返回事件）
3. **events 不会被修改**，无需每次重新设置
4. **常用事件**：`POLLIN`（可读）、`POLLOUT`（可写）、`POLLERR`（错误）
5. **时间复杂度仍为 O(n)**，大量连接时性能不佳
