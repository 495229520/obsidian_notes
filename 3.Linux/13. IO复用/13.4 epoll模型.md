# epoll 模型

## 1. epoll 概述

`epoll` 是 Linux 特有的 [[13.1 IO复用简介|I/O 复用]] 机制，从 Linux 2.6 内核开始引入。相比 [[13.2 select模型|select]] 和 [[13.3 poll模型|poll]]，epoll 在处理大量并发连接时具有显著的性能优势，是 **Linux 高性能网络编程的首选方案**。

### 1.1 epoll 的核心改进

| 问题 | select/poll | epoll |
|:---|:---|:---|
| fd 数量限制 | select 限制 1024 | 无固定限制 |
| 数据拷贝 | 每次调用都要拷贝 | 使用 mmap 共享内存 |
| 事件检测 | O(n) 遍历所有 fd | O(1) 通过回调机制 |
| 返回就绪 fd | 需遍历查找 | 直接返回就绪列表 |

---

## 2. epoll 的三个核心函数

epoll 使用三个系统调用来完成 I/O 复用：

```c
#include <sys/epoll.h>

int epoll_create(int size);           // 创建 epoll 实例
int epoll_ctl(int epfd, int op, int fd, struct epoll_event *event);  // 管理监控列表
int epoll_wait(int epfd, struct epoll_event *events, int maxevents, int timeout);  // 等待事件
```

---

## 3. epoll_create - 创建 epoll 实例

```c
int epoll_create(int size);
int epoll_create1(int flags);  // Linux 2.6.27+
```

### 3.1 参数说明

| 参数 | 说明 |
|:---|:---|
| `size` | 提示内核预期监控的 fd 数量（Linux 2.6.8 后被忽略，但必须 > 0） |
| `flags` | `epoll_create1` 的标志，如 `EPOLL_CLOEXEC` |

### 3.2 返回值

- **成功**：返回 epoll 实例的[[4.1 打开、读取、写入、关闭#文件描述符（File Descriptor）|文件描述符]]（epfd）
- **失败**：返回 -1，设置 `errno`

### 3.3 示例

```c
int epfd = epoll_create(1);  // size 参数现在被忽略，传入任意正数即可
if (epfd < 0) {
    perror("epoll_create");
    exit(EXIT_FAILURE);
}
```

> **注意**：使用完毕后需要 `close(epfd)` 关闭 epoll 实例。

---

## 4. epoll_ctl - 管理监控列表

```c
int epoll_ctl(int epfd, int op, int fd, struct epoll_event *event);
```

### 4.1 参数说明

| 参数 | 说明 |
|:---|:---|
| `epfd` | epoll 实例的文件描述符（由 `epoll_create` 返回） |
| `op` | 操作类型 |
| `fd` | 要操作的目标文件描述符 |
| `event` | 指向 `epoll_event` 结构体的指针 |

### 4.2 操作类型 (op)

| 操作 | 说明 |
|:---|:---|
| `EPOLL_CTL_ADD` | 向 epoll 实例添加新的 fd |
| `EPOLL_CTL_MOD` | 修改已注册 fd 的监控事件 |
| `EPOLL_CTL_DEL` | 从 epoll 实例删除 fd |

### 4.3 返回值

- **成功**：返回 0
- **失败**：返回 -1，设置 `errno`

---

## 5. epoll_event 结构体

```c
struct epoll_event {
    uint32_t     events;    // 监控的事件类型
    epoll_data_t data;      // 用户数据
};

typedef union epoll_data {
    void    *ptr;     // 用户自定义指针
    int      fd;      // 文件描述符
    uint32_t u32;     // 32位整数
    uint64_t u64;     // 64位整数
} epoll_data_t;
```

### 5.1 events 可用事件

| 事件 | 说明 |
|:---|:---|
| `EPOLLIN` | 有数据可读 |
| `EPOLLOUT` | 可以写数据 |
| `EPOLLPRI` | 有紧急数据可读 |
| `EPOLLERR` | 发生错误 |
| `EPOLLHUP` | 挂起（对端关闭） |
| `EPOLLRDHUP` | 对端关闭连接或关闭写端（Linux 2.6.17+） |
| `EPOLLET` | 设置为**边缘触发**模式（Edge Triggered） |
| `EPOLLONESHOT` | 只监控一次事件，触发后需要重新注册 |

### 5.2 示例：注册监控事件

```c
struct epoll_event ev;
ev.events = EPOLLIN;        // 监控可读事件
ev.data.fd = sockfd;        // 保存文件描述符

// 添加到 epoll
if (epoll_ctl(epfd, EPOLL_CTL_ADD, sockfd, &ev) < 0) {
    perror("epoll_ctl");
    exit(EXIT_FAILURE);
}
```

## 6. epoll_wait - 等待事件

```c
int epoll_wait(int epfd, struct epoll_event *events,
               int maxevents, int timeout);
```

### 6.1 参数说明

| 参数 | 说明 |
|:---|:---|
| `epfd` | epoll 实例的文件描述符 |
| `events` | 用于接收就绪事件的数组 |
| `maxevents` | events 数组的最大容量 |
| `timeout` | 超时时间（毫秒） |

### 6.2 timeout 参数

| 值 | 效果 |
|:---|:---|
| `-1` | 永久阻塞，直到有事件就绪 |
| `0` | 立即返回（非阻塞检查） |
| `> 0` | 等待指定的毫秒数 |

### 6.3 返回值

| 返回值 | 含义 |
|:---|:---|
| `> 0` | 就绪的事件数量 |
| `0` | 超时，没有事件就绪 |
| `-1` | 出错，设置 `errno` |

### 6.4 示例

```c
#define MAX_EVENTS 64
struct epoll_event events[MAX_EVENTS];

int nready = epoll_wait(epfd, events, MAX_EVENTS, -1);
if (nready < 0) {
    perror("epoll_wait");
    exit(EXIT_FAILURE);
}

// 只需遍历就绪的事件，而不是所有监控的 fd
for (int i = 0; i < nready; i++) {
    int fd = events[i].data.fd;
    if (events[i].events & EPOLLIN) {
        // 处理可读事件
    }
}
```

## 7. 触发模式

epoll 支持两种事件触发模式：

### 7.1 水平触发（Level Triggered, LT）

**默认模式**，与 select/poll 的行为一致。

```
特点：
- 只要 fd 上有未处理的数据，就会一直触发事件
- 可以不一次性读取所有数据
- 编程简单，不容易出错

工作方式：
┌──────────────────────────────────────┐
│  缓冲区有数据 → epoll_wait 返回        │
│  未读完      → 下次 epoll_wait 还返回  │
│  读完        → 不再返回                │
└──────────────────────────────────────┘
```

### 7.2 边缘触发（Edge Triggered, ET）

**高性能模式**，需要设置 `EPOLLET` 标志。

```
特点：
- 只有状态发生变化时才触发事件（如从无数据变为有数据）
- 必须一次性读取所有数据，否则可能丢失事件
- 通常需要配合非阻塞 I/O 使用
- 减少 epoll_wait 调用次数，性能更高

工作方式：
┌──────────────────────────────────────┐
│  新数据到达 → epoll_wait 返回          │
│  未读完    → 下次 epoll_wait 不返回！   │
│  新数据到达 → epoll_wait 返回          │
└──────────────────────────────────────┘
```

### 7.3 ET 模式的正确使用

```c
// 1. 设置非阻塞模式
int flags = fcntl(fd, F_GETFL, 0);
fcntl(fd, F_SETFL, flags | O_NONBLOCK);

// 2. 注册 ET 模式
struct epoll_event ev;
ev.events = EPOLLIN | EPOLLET;  // 边缘触发
ev.data.fd = fd;
epoll_ctl(epfd, EPOLL_CTL_ADD, fd, &ev);

// 3. 循环读取直到 EAGAIN
while (1) {
    ssize_t n = read(fd, buf, sizeof(buf));
    if (n < 0) {
        if (errno == EAGAIN || errno == EWOULDBLOCK) {
            break;  // 数据已读完
        }
        // 其他错误处理
    } else if (n == 0) {
        // 连接关闭
        break;
    }
    // 处理读取的数据
}
```

### 7.4 LT vs ET 对比

| 特性 | LT（水平触发） | ET（边缘触发） |
|:---|:---|:---|
| 默认模式 | ✓ | ✗ |
| 编程难度 | 简单 | 较复杂 |
| 性能 | 较低 | 更高 |
| 必须非阻塞 | 否 | 是 |
| 必须循环读 | 否 | 是 |
| 事件丢失风险 | 无 | 有（处理不当时） |

## 8. 完整示例代码

### 8.1 使用 epoll 的 TCP 服务器

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <fcntl.h>
#include <errno.h>
#include <sys/socket.h>
#include <sys/epoll.h>
#include <netinet/in.h>
#include <arpa/inet.h>

#define PORT 8080
#define MAX_EVENTS 1024
#define BUFFER_SIZE 4096

// 设置非阻塞模式
void set_nonblocking(int fd) {
    int flags = fcntl(fd, F_GETFL, 0);
    fcntl(fd, F_SETFL, flags | O_NONBLOCK);
}

int main() {
    int listen_fd, conn_fd, epfd;
    struct epoll_event ev, events[MAX_EVENTS];
    struct sockaddr_in server_addr, client_addr;
    socklen_t addr_len = sizeof(client_addr);
    char buffer[BUFFER_SIZE];

    // 1. 创建监听套接字
    listen_fd = socket(AF_INET, SOCK_STREAM, 0);
    if (listen_fd < 0) {
        perror("socket");
        exit(EXIT_FAILURE);
    }

    // 设置地址复用
    int opt = 1;
    setsockopt(listen_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

    // 设置非阻塞
    set_nonblocking(listen_fd);

    // 2. 绑定地址
    memset(&server_addr, 0, sizeof(server_addr));
    server_addr.sin_family = AF_INET;
    server_addr.sin_addr.s_addr = INADDR_ANY;
    server_addr.sin_port = htons(PORT);

    if (bind(listen_fd, (struct sockaddr*)&server_addr,
             sizeof(server_addr)) < 0) {
        perror("bind");
        exit(EXIT_FAILURE);
    }

    // 3. 开始监听
    if (listen(listen_fd, SOMAXCONN) < 0) {
        perror("listen");
        exit(EXIT_FAILURE);
    }

    // 4. 创建 epoll 实例
    epfd = epoll_create1(0);
    if (epfd < 0) {
        perror("epoll_create1");
        exit(EXIT_FAILURE);
    }

    // 5. 将监听套接字添加到 epoll
    ev.events = EPOLLIN;
    ev.data.fd = listen_fd;
    if (epoll_ctl(epfd, EPOLL_CTL_ADD, listen_fd, &ev) < 0) {
        perror("epoll_ctl");
        exit(EXIT_FAILURE);
    }

    printf("服务器启动，监听端口 %d...\n", PORT);

    // 6. 事件循环
    while (1) {
        int nready = epoll_wait(epfd, events, MAX_EVENTS, -1);

        if (nready < 0) {
            if (errno == EINTR) continue;  // 被信号中断
            perror("epoll_wait");
            break;
        }

        // 7. 处理就绪事件
        for (int i = 0; i < nready; i++) {
            int fd = events[i].data.fd;

            // 处理新连接
            if (fd == listen_fd) {
                while (1) {
                    conn_fd = accept(listen_fd,
                                    (struct sockaddr*)&client_addr,
                                    &addr_len);
                    if (conn_fd < 0) {
                        if (errno == EAGAIN || errno == EWOULDBLOCK) {
                            break;  // 所有连接已处理
                        }
                        perror("accept");
                        break;
                    }

                    printf("新连接: %s:%d, fd=%d\n",
                           inet_ntoa(client_addr.sin_addr),
                           ntohs(client_addr.sin_port),
                           conn_fd);

                    // 设置非阻塞并添加到 epoll
                    set_nonblocking(conn_fd);
                    ev.events = EPOLLIN | EPOLLET;  // 使用 ET 模式
                    ev.data.fd = conn_fd;
                    epoll_ctl(epfd, EPOLL_CTL_ADD, conn_fd, &ev);
                }
            }
            // 处理客户端数据
            else {
                if (events[i].events & EPOLLIN) {
                    // ET 模式：循环读取直到 EAGAIN
                    while (1) {
                        ssize_t n = read(fd, buffer, BUFFER_SIZE - 1);

                        if (n < 0) {
                            if (errno == EAGAIN || errno == EWOULDBLOCK) {
                                break;  // 数据已读完
                            }
                            perror("read");
                            close(fd);
                            break;
                        } else if (n == 0) {
                            // 客户端关闭连接
                            printf("客户端断开: fd=%d\n", fd);
                            close(fd);
                            break;
                        }

                        // 回显数据
                        buffer[n] = '\0';
                        printf("收到数据 [fd=%d]: %s", fd, buffer);
                        write(fd, buffer, n);
                    }
                }

                // 处理错误事件
                if (events[i].events & (EPOLLERR | EPOLLHUP)) {
                    printf("fd=%d 发生错误，关闭连接\n", fd);
                    close(fd);
                }
            }
        }
    }

    close(listen_fd);
    close(epfd);
    return 0;
}
```

## 9. epoll 的优缺点

### 9.1 优点

| 优点 | 说明 |
|:---|:---|
| **高效的事件通知** | 使用回调机制，O(1) 复杂度获取就绪 fd |
| **无 fd 数量限制** | 只受系统资源限制 |
| **内存映射** | 使用 mmap 共享内存，减少数据拷贝 |
| **只返回就绪 fd** | 无需遍历所有监控的 fd |
| **支持 ET 模式** | 边缘触发可进一步提高性能 |

### 9.2 缺点

| 缺点 | 说明 |
|:---|:---|
| **仅 Linux 支持** | 不跨平台 |
| **ET 模式复杂** | 需要非阻塞 I/O 和循环读写 |
| **API 较复杂** | 需要三个系统调用配合使用 |

## 10. 三种 I/O 复用机制对比

^92837a

| 特性 | select | poll | epoll |
|:---|:---|:---|:---|
| 最大 fd 数 | 1024 | 无限制 | 无限制 |
| fd 拷贝 | 每次调用 | 每次调用 | 仅 epoll_ctl |
| 时间复杂度 | O(n) | O(n) | O(1) |
| 就绪 fd 返回 | 遍历查找 | 遍历查找 | 直接返回 |
| 触发模式 | LT | LT | LT/ET |
| 跨平台 | ✓ | ✓ | ✗ (Linux only) |

---

## 11. 关键要点

1. **epoll 三大函数**：`epoll_create`、`epoll_ctl`、`epoll_wait`
2. **高性能原理**：
   - 红黑树管理监控的 fd（添加/删除 O(log n)）
   - 就绪链表存储触发的事件（获取就绪 O(1)）
   - mmap 共享内存减少数据拷贝
3. **触发模式**：
   - LT（默认）：有数据就触发，编程简单
   - ET：状态变化才触发，性能更高但需要非阻塞+循环读
4. **适用场景**：Linux 平台的高并发服务器（如 Nginx、Redis）
