# Makefile 基础

## 1. 什么是 Makefile

**Makefile** 是一个用于自动化编译和构建项目的脚本文件。它定义了源文件之间的依赖关系，以及如何将源代码编译成可执行文件。`make` 命令会读取 Makefile 并执行相应的构建任务。

### 1.1 为什么需要 Makefile

| 场景 | 手动编译 | 使用 Makefile |
|------|----------|---------------|
| 单文件 | `gcc main.c -o main` | 可以，但没必要 |
| 多文件 | 需要记住所有文件和依赖 | 只需 `make` |
| 增量编译 | 每次全部重新编译 | 只编译修改过的文件 |
| 团队协作 | 每人编译方式可能不同 | 统一构建流程 |

### 1.2 Makefile 的核心优势

- **自动化**：一条 `make` 命令完成所有编译工作
- **增量编译**：只重新编译修改过的文件，节省时间
- **依赖管理**：自动处理文件间的依赖关系
- **可移植性**：在不同机器上使用相同的构建流程

## 2. Makefile 基本语法

### 2.1 规则（Rule）

Makefile 的核心是**规则**，每条规则定义了如何生成一个目标文件：

```makefile
目标(target): 依赖(prerequisites)
	命令(recipe)
```

**注意**：命令前必须是 **Tab 键**，不能是空格！

```makefile
# 示例：编译 main.c 生成 main
main: main.c
	gcc main.c -o main
```

### 2.2 规则的执行逻辑

```
                    检查目标文件
                         │
           ┌─────────────┴─────────────┐
           │                           │
       目标不存在                    目标存在
           │                           │
           ▼                           ▼
       执行命令                   检查依赖文件时间戳
                                       │
                         ┌─────────────┴─────────────┐
                         │                           │
                    依赖更新                      依赖未更新
                         │                           │
                         ▼                           ▼
                    执行命令                    跳过（无需重新编译）
```

## 3. 第一个 Makefile 示例

### 3.1 项目结构

```
project/
├── main.c
├── utils.c
├── utils.h
└── Makefile
```

### 3.2 源代码

**main.c**：
```c
#include <stdio.h>
#include "utils.h"

int main() {
    printf("Result: %d\n", add(3, 5));
    return 0;
}
```

**utils.h**：
```c
#ifndef UTILS_H
#define UTILS_H

int add(int a, int b);

#endif
```

**utils.c**：
```c
#include "utils.h"

int add(int a, int b) {
    return a + b;
}
```

### 3.3 Makefile

```makefile
# 最终目标：生成可执行文件 app
app: main.o utils.o
	gcc main.o utils.o -o app

# 编译 main.c 生成 main.o
main.o: main.c utils.h
	gcc -c main.c -o main.o

# 编译 utils.c 生成 utils.o
utils.o: utils.c utils.h
	gcc -c utils.c -o utils.o

# 清理编译产物
clean:
	rm -f *.o app
```

### 3.4 执行过程

```bash
$ make
gcc -c main.c -o main.o
gcc -c utils.c -o utils.o
gcc main.o utils.o -o app

$ make
make: 'app' is up to date.    # 无需重新编译

$ touch utils.c               # 修改 utils.c 的时间戳
$ make
gcc -c utils.c -o utils.o     # 只重新编译 utils.c
gcc main.o utils.o -o app

$ make clean
rm -f *.o app
```

## 4. 变量

### 4.1 定义和使用变量

```makefile
# 定义变量
CC = gcc
CFLAGS = -Wall -g
TARGET = app
OBJS = main.o utils.o

# 使用变量：$(变量名) 或 ${变量名}
$(TARGET): $(OBJS)
	$(CC) $(OBJS) -o $(TARGET)

main.o: main.c utils.h
	$(CC) $(CFLAGS) -c main.c -o main.o

utils.o: utils.c utils.h
	$(CC) $(CFLAGS) -c utils.c -o utils.o

clean:
	rm -f $(OBJS) $(TARGET)
```

### 4.2 常用预定义变量

| 变量 | 含义 | 默认值 |
|------|------|--------|
| `CC` | C 编译器 | `cc` |
| `CXX` | C++ 编译器 | `g++` |
| `CFLAGS` | C 编译选项 | 空 |
| `CXXFLAGS` | C++ 编译选项 | 空 |
| `LDFLAGS` | 链接选项 | 空 |
| `LDLIBS` | 链接的库 | 空 |

### 4.3 变量赋值方式

| 符号 | 含义 | 说明 |
|------|------|------|
| `=` | 递归赋值 | 变量的值在使用时展开 |
| `:=` | 简单赋值 | 变量的值在定义时立即展开 |
| `?=` | 条件赋值 | 只有变量未定义时才赋值 |
| `+=` | 追加赋值 | 在现有值后面追加 |

```makefile
# 递归赋值示例
A = $(B)
B = hello
# 此时 $(A) 为 "hello"

# 简单赋值示例
A := $(B)
B := hello
# 此时 $(A) 为空，因为赋值时 B 还未定义

# 条件赋值示例
CC ?= gcc       # 如果 CC 未定义，则设为 gcc

# 追加赋值示例
CFLAGS = -Wall
CFLAGS += -g    # CFLAGS = "-Wall -g"
```

## 5. 伪目标（.PHONY）

### 5.1 什么是伪目标

伪目标不是真实的文件，而是一个标签，用于执行特定的命令。

```makefile
.PHONY: clean all install

clean:
	rm -f *.o app

all: app

install: app
	cp app /usr/local/bin/
```

### 5.2 为什么需要 .PHONY

如果当前目录下存在名为 `clean` 的文件，`make clean` 会认为目标已存在而不执行命令。使用 `.PHONY` 声明后，make 会始终执行该目标的命令。

```bash
$ touch clean           # 创建名为 clean 的文件
$ make clean
make: 'clean' is up to date.   # 不会执行！

# 使用 .PHONY 后
$ make clean
rm -f *.o app                  # 正常执行
```

## 6. 常用目标约定

| 目标 | 用途 |
|------|------|
| `all` | 默认目标，编译整个项目 |
| `clean` | 清理编译产物（.o 文件、可执行文件） |
| `install` | 安装程序到系统目录 |
| `uninstall` | 卸载程序 |
| `dist` | 打包发布 |
| `test` | 运行测试 |

```makefile
.PHONY: all clean install test

# 默认目标
all: app

app: main.o utils.o
	$(CC) $^ -o $@

clean:
	rm -f *.o app

install: app
	install -m 755 app /usr/local/bin/

test: app
	./app
```

## 7. 注释

Makefile 中使用 `#` 进行注释：

```makefile
# 这是一行注释
CC = gcc  # 这是行尾注释

# 多行注释需要每行都加 #
# 第一行
# 第二行
```

## 8. 多目标和多规则

### 8.1 多目标

一条规则可以有多个目标：

```makefile
# main.o 和 utils.o 使用相同的编译选项
main.o utils.o: %.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@
```

### 8.2 多规则

一个目标可以出现在多条规则中，依赖会被合并：

```makefile
app: main.o
app: utils.o
app:
	$(CC) main.o utils.o -o app

# 等价于
app: main.o utils.o
	$(CC) main.o utils.o -o app
```

## 9. 完整示例

```makefile
# ============================================
# 项目配置
# ============================================
CC      = gcc
CFLAGS  = -Wall -Wextra -g
LDFLAGS =
TARGET  = myapp
SRCS    = main.c utils.c helper.c
OBJS    = $(SRCS:.c=.o)

# ============================================
# 伪目标声明
# ============================================
.PHONY: all clean rebuild

# ============================================
# 默认目标
# ============================================
all: $(TARGET)

# ============================================
# 链接目标文件生成可执行文件
# ============================================
$(TARGET): $(OBJS)
	$(CC) $(LDFLAGS) $^ -o $@

# ============================================
# 编译源文件生成目标文件
# ============================================
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

# ============================================
# 清理
# ============================================
clean:
	rm -f $(OBJS) $(TARGET)

# ============================================
# 重新编译
# ============================================
rebuild: clean all
```

---

## 10. 关键要点

1. **规则三要素**：目标、依赖、命令
2. **命令缩进**：必须使用 Tab，不能用空格
3. **增量编译**：make 根据时间戳决定是否重新编译
4. **变量**：使用 `$(VAR)` 引用，提高可维护性
5. **伪目标**：使用 `.PHONY` 声明非文件目标
6. **默认目标**：make 默认执行第一个目标
