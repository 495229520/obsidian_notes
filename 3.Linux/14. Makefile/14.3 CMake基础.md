# CMake 基础

## 1. 什么是 CMake

**CMake** 是一个跨平台的构建系统生成器。它不直接构建项目，而是根据 `CMakeLists.txt` 配置文件生成特定平台的构建文件（如 Makefile、Visual Studio 项目文件等）。

### 1.1 CMake vs Makefile

| 特性 | Makefile | CMake |
|------|----------|-------|
| 跨平台 | 需要手动适配不同平台 | 自动生成各平台构建文件 |
| 语法 | 复杂，容易出错 | 相对简洁，更易维护 |
| IDE 支持 | 较弱 | 可生成 IDE 项目文件 |
| 依赖管理 | 手动处理 | 内置依赖查找机制 |
| 学习曲线 | 中等 | 中等 |

### 1.2 CMake 工作流程

```
   CMakeLists.txt          平台构建文件              最终产物
        │                       │                      │
        │     cmake             │     make/ninja       │
        │     ─────►            │     ─────────►       │
        ▼                       ▼                      ▼
┌─────────────┐         ┌─────────────┐        ┌─────────────┐
│ CMake 配置   │   ──►   │  Makefile   │   ──►  │ 可执行文件   │
│             │         │ 或 .sln    │        │ 或库文件     │
└─────────────┘         └─────────────┘        └─────────────┘
```

## 2. 安装和基本使用

### 2.1 安装 CMake

```bash
# Ubuntu/Debian
sudo apt install cmake

# CentOS/RHEL
sudo yum install cmake

# macOS
brew install cmake

# 验证安装
cmake --version
```

### 2.2 构建步骤

```bash
# 1. 创建构建目录（推荐外部构建）
mkdir build && cd build

# 2. 生成构建文件
cmake ..

# 3. 执行构建
make
# 或使用 cmake 统一接口
cmake --build .

# 4. 安装（可选）
sudo make install
# 或
sudo cmake --install .
```

### 2.3 外部构建 vs 内部构建

```
# 内部构建（不推荐）          # 外部构建（推荐）
project/                     project/
├── CMakeLists.txt           ├── CMakeLists.txt
├── main.c                   ├── main.c
├── CMakeCache.txt  ← 混乱    └── build/
├── CMakeFiles/              　　 ├── CMakeCache.txt
└── Makefile                 　　 ├── CMakeFiles/
                             　　 └── Makefile
```

**外部构建优势**：
- 保持源码目录干净
- 可以同时有多个构建目录（Debug/Release）
- 清理只需删除构建目录

## 3. 第一个 CMake 项目

### 3.1 项目结构

```
hello/
├── CMakeLists.txt
└── main.c
```

### 3.2 源代码

**main.c**：
```c
#include <stdio.h>

int main() {
    printf("Hello, CMake!\n");
    return 0;
}
```

### 3.3 CMakeLists.txt

```cmake
# 指定 CMake 最低版本
cmake_minimum_required(VERSION 3.10)

# 定义项目名称和语言
project(Hello C)

# 添加可执行文件
add_executable(hello main.c)
```

### 3.4 构建和运行

```bash
mkdir build && cd build
cmake ..
make
./hello
# 输出: Hello, CMake!
```

## 4. CMake 基本命令

### 4.1 cmake_minimum_required

指定所需的最低 CMake 版本：

```cmake
cmake_minimum_required(VERSION 3.10)

# 指定版本范围（CMake 3.12+）
cmake_minimum_required(VERSION 3.10...3.25)
```

### 4.2 project

定义项目信息：

```cmake
# 基本用法
project(MyProject)

# 完整用法
project(MyProject
    VERSION 1.0.0
    DESCRIPTION "My awesome project"
    HOMEPAGE_URL "https://example.com"
    LANGUAGES C CXX
)

# 访问项目变量
message("Project: ${PROJECT_NAME}")
message("Version: ${PROJECT_VERSION}")
```

### 4.3 add_executable

创建可执行文件目标：

```cmake
# 单个源文件
add_executable(app main.c)

# 多个源文件
add_executable(app
    main.c
    utils.c
    helper.c
)

# 使用变量
set(SOURCES main.c utils.c helper.c)
add_executable(app ${SOURCES})
```

### 4.4 add_library

创建库文件：

```cmake
# 静态库
add_library(mylib STATIC utils.c helper.c)

# 动态库（共享库）
add_library(mylib SHARED utils.c helper.c)

# 让 CMake 根据 BUILD_SHARED_LIBS 变量决定
add_library(mylib utils.c helper.c)
```

### 4.5 target_link_libraries

链接库到目标：

```cmake
# 链接自定义库
add_library(utils STATIC utils.c)
add_executable(app main.c)
target_link_libraries(app utils)

# 链接系统库
target_link_libraries(app pthread m)

# 现代 CMake 推荐方式
target_link_libraries(app PRIVATE utils)
target_link_libraries(app PUBLIC somelib)
```

**链接类型**：

| 类型 | 说明 |
|------|------|
| `PRIVATE` | 只有当前目标使用该库 |
| `PUBLIC` | 当前目标和依赖它的目标都使用该库 |
| `INTERFACE` | 只有依赖当前目标的目标使用该库 |

## 5. 变量和缓存

### 5.1 普通变量

```cmake
# 设置变量
set(MY_VAR "Hello")
set(SOURCE_FILES main.c utils.c)

# 使用变量
message("MY_VAR = ${MY_VAR}")
add_executable(app ${SOURCE_FILES})

# 列表操作
set(MY_LIST a b c)
list(APPEND MY_LIST d e)        # MY_LIST = a;b;c;d;e
list(LENGTH MY_LIST LEN)        # LEN = 5
list(GET MY_LIST 0 FIRST)       # FIRST = a
```

### 5.2 缓存变量

缓存变量存储在 `CMakeCache.txt` 中，可通过命令行或 GUI 修改：

```cmake
# 定义缓存变量
set(MY_OPTION "default" CACHE STRING "Description")
set(ENABLE_TESTS ON CACHE BOOL "Enable tests")

# 命令行设置
# cmake -DMY_OPTION="custom" ..
```

### 5.3 option 命令

定义布尔选项：

```cmake
option(BUILD_TESTS "Build test programs" ON)
option(BUILD_SHARED_LIBS "Build shared libraries" OFF)

if(BUILD_TESTS)
    add_subdirectory(tests)
endif()
```

### 5.4 常用预定义变量

| 变量 | 说明 |
|------|------|
| `PROJECT_NAME` | 项目名称 |
| `PROJECT_SOURCE_DIR` | 项目源码根目录 |
| `PROJECT_BINARY_DIR` | 项目构建目录 |
| `CMAKE_SOURCE_DIR` | 顶层源码目录 |
| `CMAKE_BINARY_DIR` | 顶层构建目录 |
| `CMAKE_CURRENT_SOURCE_DIR` | 当前 CMakeLists.txt 所在目录 |
| `CMAKE_CURRENT_BINARY_DIR` | 当前构建目录 |
| `CMAKE_C_COMPILER` | C 编译器路径 |
| `CMAKE_CXX_COMPILER` | C++ 编译器路径 |
| `CMAKE_BUILD_TYPE` | 构建类型 (Debug/Release) |

## 6. 编译选项

### 6.1 设置编译标志

```cmake
# 方法 1：直接设置（全局）
set(CMAKE_C_FLAGS "${CMAKE_C_FLAGS} -Wall -Wextra")
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -Wall -Wextra")

# 方法 2：针对特定目标（推荐）
target_compile_options(app PRIVATE -Wall -Wextra)

# 方法 3：添加编译定义
target_compile_definitions(app PRIVATE DEBUG=1 VERSION="1.0")
```

### 6.2 构建类型

```cmake
# 设置默认构建类型
if(NOT CMAKE_BUILD_TYPE)
    set(CMAKE_BUILD_TYPE Release)
endif()

# 根据构建类型设置选项
if(CMAKE_BUILD_TYPE STREQUAL "Debug")
    target_compile_definitions(app PRIVATE DEBUG_MODE)
endif()
```

构建类型及其默认标志：

| 类型 | C 标志 | 说明 |
|------|--------|------|
| `Debug` | `-g` | 调试符号 |
| `Release` | `-O3 -DNDEBUG` | 优化发布 |
| `RelWithDebInfo` | `-O2 -g -DNDEBUG` | 带调试的优化 |
| `MinSizeRel` | `-Os -DNDEBUG` | 最小体积 |

```bash
# 命令行指定构建类型
cmake -DCMAKE_BUILD_TYPE=Debug ..
```

### 6.3 设置 C/C++ 标准

```cmake
# 方法 1：全局设置
set(CMAKE_C_STANDARD 11)
set(CMAKE_C_STANDARD_REQUIRED ON)
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 方法 2：针对特定目标
target_compile_features(app PRIVATE c_std_11)
target_compile_features(app PRIVATE cxx_std_17)
```

## 7. 头文件和包含路径

### 7.1 添加头文件搜索路径

```cmake
# 全局设置（影响所有目标）
include_directories(${PROJECT_SOURCE_DIR}/include)

# 针对特定目标（推荐）
target_include_directories(app PRIVATE
    ${PROJECT_SOURCE_DIR}/include
    ${PROJECT_SOURCE_DIR}/src
)

# 系统头文件路径（不产生警告）
target_include_directories(app SYSTEM PRIVATE
    /usr/local/include
)
```

### 7.2 示例项目结构

```
project/
├── CMakeLists.txt
├── include/
│   └── utils.h
└── src/
    ├── main.c
    └── utils.c
```

```cmake
cmake_minimum_required(VERSION 3.10)
project(MyProject C)

# 添加头文件路径
include_directories(${PROJECT_SOURCE_DIR}/include)

# 收集源文件
file(GLOB SOURCES "src/*.c")

# 创建可执行文件
add_executable(app ${SOURCES})
```

## 8. 条件判断和循环

### 8.1 if 条件

```cmake
if(expression)
    # ...
elseif(expression)
    # ...
else()
    # ...
endif()
```

**常用条件表达式**：

```cmake
# 变量判断
if(DEFINED MY_VAR)          # 变量已定义
if(MY_VAR)                  # 变量为真（非空、非0、非OFF等）
if(NOT MY_VAR)              # 变量为假

# 比较
if(MY_VAR STREQUAL "hello")     # 字符串相等
if(MY_VAR VERSION_GREATER 1.0)  # 版本比较
if(MY_VAR GREATER 10)           # 数值比较

# 文件/目录
if(EXISTS ${FILE_PATH})         # 文件或目录存在
if(IS_DIRECTORY ${PATH})        # 是目录
if(IS_ABSOLUTE ${PATH})         # 是绝对路径

# 目标判断
if(TARGET mylib)                # 目标已定义

# 逻辑组合
if(A AND B)
if(A OR B)
if(NOT A)
```

### 8.2 foreach 循环

```cmake
# 遍历列表
set(ITEMS a b c d)
foreach(ITEM ${ITEMS})
    message("Item: ${ITEM}")
endforeach()

# 遍历范围
foreach(i RANGE 1 5)
    message("Number: ${i}")
endforeach()

# 遍历源文件
file(GLOB SOURCES "src/*.c")
foreach(SRC ${SOURCES})
    message("Source: ${SRC}")
endforeach()
```

### 8.3 while 循环

```cmake
set(COUNT 0)
while(COUNT LESS 5)
    message("Count: ${COUNT}")
    math(EXPR COUNT "${COUNT} + 1")
endwhile()
```

## 9. 函数和宏

### 9.1 function 函数

```cmake
# 定义函数
function(print_info TARGET)
    message("Target: ${TARGET}")
    message("Arguments: ${ARGN}")      # 额外参数
    message("Arg count: ${ARGC}")      # 参数数量
endfunction()

# 调用函数
print_info(app extra1 extra2)
```

### 9.2 macro 宏

```cmake
# 定义宏
macro(set_target_properties_common TARGET)
    target_compile_options(${TARGET} PRIVATE -Wall)
    target_compile_definitions(${TARGET} PRIVATE VERSION="1.0")
endmacro()

# 调用宏
add_executable(app main.c)
set_target_properties_common(app)
```

**function vs macro**：

| 特性 | function | macro |
|------|----------|-------|
| 变量作用域 | 有自己的作用域 | 共享调用者作用域 |
| 参数传递 | 值传递 | 文本替换 |
| 推荐使用 | 大多数情况 | 需要修改调用者变量时 |

## 10. 完整示例

### 10.1 项目结构

```
myproject/
├── CMakeLists.txt
├── include/
│   └── utils.h
├── src/
│   ├── main.c
│   └── utils.c
└── build/
```

### 10.2 CMakeLists.txt

```cmake
# ============================================
#              基本配置
# ============================================
cmake_minimum_required(VERSION 3.10)

project(MyProject
    VERSION 1.0.0
    DESCRIPTION "A sample CMake project"
    LANGUAGES C
)

# ============================================
#              选项
# ============================================
option(BUILD_SHARED_LIBS "Build shared library" OFF)
option(ENABLE_WARNINGS "Enable compiler warnings" ON)

# ============================================
#              编译设置
# ============================================
set(CMAKE_C_STANDARD 11)
set(CMAKE_C_STANDARD_REQUIRED ON)

# 默认构建类型
if(NOT CMAKE_BUILD_TYPE)
    set(CMAKE_BUILD_TYPE Release)
endif()

# ============================================
#              源文件
# ============================================
set(SOURCES
    src/main.c
    src/utils.c
)

set(HEADERS
    include/utils.h
)

# ============================================
#              目标
# ============================================
add_executable(${PROJECT_NAME} ${SOURCES})

# 头文件路径
target_include_directories(${PROJECT_NAME} PRIVATE
    ${PROJECT_SOURCE_DIR}/include
)

# 编译选项
if(ENABLE_WARNINGS)
    target_compile_options(${PROJECT_NAME} PRIVATE
        -Wall -Wextra -Wpedantic
    )
endif()

# 编译定义
target_compile_definitions(${PROJECT_NAME} PRIVATE
    VERSION="${PROJECT_VERSION}"
    $<$<CONFIG:Debug>:DEBUG_MODE>
)

# ============================================
#              安装
# ============================================
install(TARGETS ${PROJECT_NAME}
    RUNTIME DESTINATION bin
)

install(FILES ${HEADERS}
    DESTINATION include
)

# ============================================
#              信息输出
# ============================================
message(STATUS "")
message(STATUS "=== ${PROJECT_NAME} v${PROJECT_VERSION} ===")
message(STATUS "Build type: ${CMAKE_BUILD_TYPE}")
message(STATUS "C Compiler: ${CMAKE_C_COMPILER}")
message(STATUS "Install prefix: ${CMAKE_INSTALL_PREFIX}")
message(STATUS "")
```

### 10.3 构建命令

```bash
# Debug 构建
mkdir build-debug && cd build-debug
cmake -DCMAKE_BUILD_TYPE=Debug ..
make

# Release 构建
mkdir build-release && cd build-release
cmake -DCMAKE_BUILD_TYPE=Release ..
make

# 安装
sudo cmake --install build-release --prefix /usr/local
```

## 11. 关键要点

1. **外部构建**：始终使用独立的 build 目录
2. **cmake_minimum_required**：指定最低版本要求
3. **project**：定义项目信息
4. **add_executable/add_library**：创建目标
5. **target_xxx**：现代 CMake 推荐使用 target_* 命令
6. **CMAKE_BUILD_TYPE**：控制 Debug/Release 构建
7. **生成器表达式**：`$<...>` 用于条件编译
