# 套接字（Socket）

## 简介

套接字（Socket）是网络编程的核心接口，它为应用程序提供了一种**与网络协议无关**的通信方式。简单来说，Socket 就是操作系统提供给程序员的一个"网络插座"，通过这个接口，应用程序可以方便地进行网络数据收发，而无需关心底层协议的复杂细节。

Socket 在 [[7.4 协议|TCP/IP 协议栈]] 中扮演着**抽象层**的角色，它位于应用层和传输层之间，向上为应用程序提供统一的编程接口，向下调用 TCP 或 UDP 等传输层协议。

## socket 函数

`socket()` 函数用于创建一个套接字，它是网络编程的第一步。

### 函数原型

```c
#include <sys/types.h>
#include <sys/socket.h>

int socket(int domain, int type, int protocol);
```

### 参数说明

| 参数 | 说明 |
|:---|:---|
| `domain` | 地址族（Address Family），指定通信使用的协议族 |
| `type` | 套接字类型，指定通信的语义 |
| `protocol` | 协议编号，通常设为 0 让系统自动选择 |

### 返回值

- **成功**：返回一个非负整数，即套接字文件描述符（socket fd）
- **失败**：返回 -1，并设置 `errno`

### domain（地址族）常用值

| 地址族（Address Family） | 含义 |
|:---|:---|
| `AF_INET` | IPv4 网络协议中使用的地址族 |
| `AF_INET6` | IPv6 网络协议中使用的地址族 |
| `AF_LOCAL` / `AF_UNIX` | 本地通信中采用的 UNIX 协议地址族 |

### type（套接字类型）常用值

| 类型 | 说明 |
|:---|:---|
| `SOCK_STREAM` | 面向连接的流式套接字，使用 [[7.4 协议#TCP 协议\|TCP 协议]] |
| `SOCK_DGRAM` | 无连接的数据报套接字，使用 UDP 协议 |
| `SOCK_RAW` | 原始套接字，可访问底层协议 |

### 示例代码

```c
#include <stdio.h>
#include <sys/socket.h>

int main() {
    // 创建一个 TCP 套接字
    int sock = socket(AF_INET, SOCK_STREAM, 0);
    if (sock < 0) {
        perror("socket");
        return -1;
    }

    printf("Socket 创建成功，fd = %d\n", sock);
    return 0;
}
```

## 网络编程相关数据类型

在进行网络编程时，需要使用一些特定的数据类型来保证跨平台的兼容性。

### 基本数据类型

| 数据类型名称 | 数据类型说明 | 声明的头文件 |
|:---|:---|:---|
| `int8_t` | signed 8-bit int | `sys/types.h` |
| `uint8_t` | unsigned 8-bit int (unsigned char) | `sys/types.h` |
| `int16_t` | signed 16-bit int | `sys/types.h` |
| `uint16_t` | unsigned 16-bit int (unsigned short) | `sys/types.h` |
| `int32_t` | signed 32-bit int | `sys/types.h` |
| `uint32_t` | unsigned 32-bit int (unsigned long) | `sys/types.h` |

### 套接字专用数据类型

| 数据类型名称 | 数据类型说明 | 声明的头文件 |
|:---|:---|:---|
| `sa_family_t` | 地址族（address family） | `sys/socket.h` |
| `socklen_t` | 长度（length of struct） | `sys/socket.h` |
| `in_addr_t` | [[7.2 IP地址\|IP地址]]，声明为 `uint32_t` | `netinet/in.h` |
| `in_port_t` | [[7.3 端口\|端口号]]，声明为 `uint16_t` | `netinet/in.h` |

## 地址结构体

### 通用地址结构体 sockaddr

`sockaddr` 是一个通用的地址结构体，用于在各种 socket 函数中传递地址信息。

```c
struct sockaddr {
    sa_family_t  sa_family;    // 地址族
    char         sa_data[14];  // 地址数据
};
```

### IPv4 地址结构体 sockaddr_in

在实际的 IPv4 网络编程中，我们通常使用 `sockaddr_in` 结构体，它更容易使用：

```c
struct sockaddr_in {
    sa_family_t    sin_family;  // 地址族，固定为 AF_INET
    in_port_t      sin_port;    // 端口号（网络字节序）
    struct in_addr sin_addr;    // IP 地址
    char           sin_zero[8]; // 填充字节，使结构体大小与 sockaddr 相同
};

struct in_addr {
    in_addr_t s_addr;           // 32位 IPv4 地址（网络字节序）
};
```

> **注意**：`sin_port` 和 `sin_addr` 必须是**网络字节序**（大端序），需要使用 `htons()`、`htonl()` 等函数进行转换。

### 字节序转换函数

由于不同计算机的字节序可能不同，网络传输统一使用**大端字节序**（网络字节序）：

```c
#include <arpa/inet.h>

uint16_t htons(uint16_t hostshort);  // 主机字节序 → 网络字节序（16位）
uint32_t htonl(uint32_t hostlong);   // 主机字节序 → 网络字节序（32位）
uint16_t ntohs(uint16_t netshort);   // 网络字节序 → 主机字节序（16位）
uint32_t ntohl(uint32_t netlong);    // 网络字节序 → 主机字节序（32位）
```

- `h` = host（主机）
- `n` = network（网络）
- `s` = short（16位）
- `l` = long（32位）

## bind 函数

`bind()` 函数用于将套接字与指定的 IP 地址和 [[7.3 端口|端口号]] 绑定。对于服务器程序，这是必须的步骤，因为客户端需要知道服务器的地址才能连接。

### 函数原型

```c
#include <sys/socket.h>

int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```

### 参数说明

| 参数 | 说明 |
|:---|:---|
| `sockfd` | 套接字文件描述符，由 `socket()` 函数返回 |
| `addr` | 指向地址结构体的指针，包含要绑定的 IP 和端口 |
| `addrlen` | 地址结构体的大小 |

### 返回值

- **成功**：返回 0
- **失败**：返回 -1，并设置 `errno`

### 示例代码

```c
#include <stdio.h>
#include <string.h>
#include <sys/socket.h>
#include <netinet/in.h>

int main() {
    // 1. 创建套接字
    int sockfd = socket(AF_INET, SOCK_STREAM, 0);
    if (sockfd < 0) {
        perror("socket");
        return -1;
    }

    // 2. 设置地址结构体
    struct sockaddr_in addr;
    memset(&addr, 0, sizeof(addr));         // 清零
    addr.sin_family = AF_INET;              // IPv4
    addr.sin_port = htons(8080);            // 端口 8080（转换为网络字节序）
    addr.sin_addr.s_addr = htonl(INADDR_ANY); // 监听所有网卡

    // 3. 绑定地址
    if (bind(sockfd, (struct sockaddr*)&addr, sizeof(addr)) < 0) {
        perror("bind");
        return -1;
    }

    printf("绑定成功！监听端口 8080\n");
    return 0;
}
```

> **说明**：`INADDR_ANY` 表示绑定到本机所有可用的网络接口，这样服务器可以接受来自任何网卡的连接请求。

## 关键要点

1. **Socket 的本质**：Socket 是应用层与传输层之间的抽象接口，提供统一的网络编程 API

2. **创建套接字三要素**：
   - 地址族（domain）：决定使用哪种协议族
   - 类型（type）：决定通信语义（流式/数据报）
   - 协议（protocol）：通常为 0，自动选择

3. **服务器端流程**：
   - `socket()` → 创建套接字
   - `bind()` → 绑定地址
   - `listen()` → 开始监听（参见 [[7.7 listen和accept函数]]）
   - `accept()` → 接受连接（参见 [[7.7 listen和accept函数]]）

4. **字节序注意事项**：端口号和 IP 地址在网络传输时必须使用网络字节序
