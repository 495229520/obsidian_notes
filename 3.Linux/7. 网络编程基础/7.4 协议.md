# 协议

## 简介

协议在计算机网络中可以理解为一套**规则或约定**，它定义了通信双方如何进行数据交换。就像人与人交流需要遵循语言、语法和礼仪一样，计算机之间要实现有效沟通，也必须共同遵守相同的协议。

如果把网络通信比作寄信，协议就是信封上填写地址、邮编的格式，以及邮局分拣和投递的流程。没有这套统一的规则，信息就无法正确传达。

> **协议就是一种网络式交互中数据格式和交互流程的约定**
>
> 通过协议，我们可以与远程的设备进行数据交互，请求或者完成对方的服务。
>
> **协议就是计算机中的语言。**

## TCP 与 UDP 对比

| 特性 | [[7.5 TCP协议基础|TCP]]（传输控制协议） | [[9.1 UDP原理|UDP]]（用户数据报协议） |
|:---|:---|:---|
| **可靠性** | 高。它会确保数据完整、按序地到达 | 低。它只负责发送数据，不保证对方是否收到 |
| **连接性** | 面向连接。通信前需建立连接，通信后需断开 | 无连接。直接发送数据，不需建立连接 |
| **速度** | 慢。因为需要建立连接和数据确认，开销较大 | 快。传输效率高，实时性强 |
| **开销** | 大。有较多的协议头信息和控制机制 | 小。协议头非常简单 |
| **应用场景** | 对数据完整性要求高的场景，如网页浏览（HTTP）、文件传输（FTP）、电子邮件（SMTP） | 对实时性要求高、少量数据丢失可接受的场景，如在线视频、网络电话（VoIP）、网络游戏 |

## HTTP 协议

HTTP（HyperText Transfer Protocol，超文本传输协议）是应用层协议，用于 Web 浏览器与服务器之间的通信。

### HTTP 请求头示例

```http
GET /search/detail?ct=503316480&z=0&ipn=d HTTP/1.1
Host: pic.baidu.com
Connection: keep-alive
Cache-Control: max-age=0
Upgrade-Insecure-Requests: 1
User-Agent: Mozilla/5.0 (Windows NT 10.0; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/70.0.3538.25 Safari/537.36 Core/1.70.3766.400 QQBrowser/10.6.4163.400
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,*/*;q=0.8
Accept-Encoding: gzip, deflate, br
Accept-Language: zh-CN,zh;q=0.9
```

### HTTP 请求头字段说明

| 字段 | 说明 |
|:---|:---|
| `Host` | 请求的目标主机名 |
| `Connection` | 连接管理（keep-alive 表示保持连接） |
| `Cache-Control` | 缓存控制策略 |
| `User-Agent` | 客户端（浏览器）信息 |
| `Accept` | 客户端可接受的内容类型 |
| `Accept-Encoding` | 客户端可接受的压缩编码 |
| `Accept-Language` | 客户端可接受的语言 |

## SSH 协议

SSH（Secure Shell，安全外壳协议）是一种加密的网络传输协议，用于在不安全的网络中安全地访问远程计算机。

### SSH 包头结构示例

```c
struct sshhead
{
    unsigned int tlen;       // 总长度
    unsigned char plen;      // 填充长度
    unsigned char msgcode;   // 消息类型代码
    sshhead(){tlen=6;}
};
```

## 关键要点

1. **协议的本质**：协议就是一种网络式交互中数据格式和交互流程的约定

2. **选择协议的依据**：
   - 需要可靠传输 → [[7.5 TCP协议基础|TCP]]
   - 需要实时性、允许丢包 → [[9.1 UDP原理|UDP]]（编程实践参见 [[9.2 UDP服务端代码|UDP服务端]] 和 [[9.3 UDP客户端代码|UDP客户端]]）

3. **常见协议与 [[7.3 端口|端口]] 对应**：
   - HTTP: 80
   - HTTPS: 443
   - SSH: 22
   - FTP: 20/21
   - DNS: 53
   - SMTP: 25

4. **协议分层**：参见 [[7.1 基础知识|网络模型分层]]
