# IP 地址

## 简介

32位网络地址由四个字节组成：`255*255*255*255`（即 `0000 0000 0000 0000`）

**IP 地址的特殊规则**：
1. 以 0 开头的地址都是不可用的
2. 以 0 结尾的地址表示的是网段，而不是具体的地址
3. 224 - 239 开头的地址是组播地址，不可用于点对点的传输，组播类似于 TCP/UDP 上面的广播
4. 240 - 255 开头的地址是实验用地址，一般不做服务器或设备，一般不能登录
5. 127.0.0.1 保留的是回环网络的地址
6. 0.0.0.0 保留的，一般用于服务器监听，表示全网段监听所有接口
7. 组播可以极大的节约带宽，发送者有限
8. 缺点：容易产生广播风暴

## 广播风暴

广播容易引起广播风暴，主要原因为广播的机制是"一发多收"，它将数据包发送给网络中的所有设备，当网络中的设备数量增加，或者网络配置不当，这种机制就会引发广播风暴，最终导致广播风暴。

### 1. 广播的重复转发

在二层网络中，交换机是根据 MAC 地址表来转发数据。如果交换机收到一个广播数据包，它会将其从**所有其他端口**转发出去。如果网络中存在环路（即数据包可以通过不止一条路径返回到原来的设备），一个广播数据包就会在网络中无限循环转发，大量的广播数据包会导致 CPU 利用率骤升，影响甚至正常工作，甚至可能导致设备瘫痪。

### 2. ARP 请求泛滥

ARP（地址解析协议）是广播风暴的一个常见源头。当一个设备需要与另一个设备通信，但不知道对方的 MAC 地址，它会发送一个 ARP 广播请求来询问。网络中的所有设备都会接受这个请求。如果设备频繁地发送 ARP 请求（例如在网络地址补丁请求变化或受到 ARP 攻击欺骗攻击时），大量的 ARP 广播就会充斥整个网络，导致正常数据传输受影响。

### 3. 设备处理广播包的负担

网络中的每个设备（如计算机、服务器、路由器等）收到一个广播数据包后，都需要对其进行处理，即使这个数据包不是发给自己的。当广播数据包的数量显著增加时，所有设备的 CPU 会忙于处理这些无关的数据包，导致 CPU 利用率骤升，影响甚至正常工作，甚至可能导致设备瘫痪。

### 4. 网络中存在环路

这是引发广播风暴最直接和最严重的原因。当网络存在物理或逻辑上的环路，一个广播包会像病毒一样在环路中无限复制和传播，直到网络完全瘫痪。

> **解决方案**：可以使用 STP（生成树协议）/ STP/RSTP 来阻止环路的形成，一个广播包会像病毒一样在环路中无限复制和传播，直到网络完全瘫痪。

## 地址类别

IP 地址根据网络号和主机号的位数分为 A、B、C、D、E 五类：

![[5e27766f-94d0-4c63-aeda-9f1b90f9f129.png]]
### A 类地址

```
┌───────────────────────────────────────────────────┐
│ 0 │      Network (7bit)     │      Host (24bit)   │
└───────────────────────────────────────────────────┘
```

- **范围**：1.0.0.0 到 126.255.255.255
- **第一个八位字节**：以二进制 0 开头
- **结构**：网络ID（8位）+ 主机ID（24位）
- **特点**：
  - 网络数少：2^7 - 2 = 126 个网络
  - 主机数多：每个网络可以容纳 2^24 - 2 个主机，约1600万台
- **用途**：主要分配给大型组织和国家级网络

### B 类地址

```
┌───────────────────────────────────────────────────┐
│ 1 │ 0 │   Network (14bit)     │    Host (16bit)   │
└───────────────────────────────────────────────────┘
```

- **范围**：128.0.0.0 到 191.255.255.255
- **第一个八位字节**：以二进制 10 开头
- **结构**：网络ID（16位）+ 主机ID（16位）
- **特点**：
  - 网络数适中：2^14 个网络
  - 主机数适中：每个网络可以容纳 2^16 - 2 个主机，约6.5万台
- **用途**：分配给中型到大型组织

### C 类地址

```
┌───────────────────────────────────────────────────┐
│ 1 │ 1 │ 0 │    Network (21bit)    │  Host (8bit)  │
└───────────────────────────────────────────────────┘
```

- **范围**：192.0.0.0 到 223.255.255.255
- **第一个八位字节**：以二进制 110 开头
- **结构**：网络ID（24位）+ 主机ID（8位）
- **特点**：
  - 网络数多：2^21 个网络
  - 主机数少：每个网络只能容纳 2^8 - 2 个主机，即254台
- **用途**：分配给小型网络，如家庭、小型公司或学校

### D 类地址

```
┌───────────────────────────────────────────────────┐
│ 1 │ 1 │ 1 │ 0 │         组播地址                   │
└───────────────────────────────────────────────────┘
```

- **范围**：224.0.0.0 到 239.255.255.255
- **第一个八位字节**：以二进制 1110 开头
- **结构**：无网络ID和主机ID的概念
- **用途**：**组播地址**（Multicast），它不是分配给单个设备的，而是用于同时向一组设备发送数据。例如，视频会议和在线流媒体就常用到这类地址。

### E 类地址

```
┌───────────────────────────────────────────────────┐
│ 1 │ 1 │ 1 │ 1 │ 0 │         保留                  │
└───────────────────────────────────────────────────┘
```

- **范围**：240.0.0.0 到 255.255.255.255
- **第一个八位字节**：以二进制 11110 开头
- **用途**：**保留地址**，用于未来研究和实验，不对外分配使用

## 特殊地址段

在上述分类中，还有一些特殊的保留地址，不能被随意使用：

| 地址/范围 | 说明 |
|:---|:---|
| `127.0.0.0/8` | **回环地址**，用于本机测试，最常用的是 127.0.0.1 |
| `0.0.0.0` | **未指定地址**，通常用于路由或表示服务器监听所有接口 |
| **私有地址** | 在家庭和企业内部网络中使用，不能在公共互联网上直接路由 |

### 私有地址范围

| 类别 | 私有地址范围 |
|:---:|:---|
| A 类私有地址 | 10.0.0.0/8 |
| B 类私有地址 | 172.16.0.0/12 |
| C 类私有地址 | 192.168.0.0/16 |
