# 网络编程基础知识

## OSI 参考模型（七层）

OSI（Open Systems Interconnection，开放系统互连）参考模型将网络通信划分为七个层次，每一层都有特定的功能。

### 数据封装与解封装过程

**发送端**：各个层都有自己的头部信息后跟着 DATA，逐层向下添加数据，发送给接收端。

**接收端**：去除自己对应层的头部信息再处理数据 DATA，一层一层向上递减（共七层）。
![[3b81f678-dd17-4659-bc97-1eda8390492a.png]]
### 各层功能说明

| 层次  |  名称   | 主要功能                | 典型协议/设备              |
| :-: | :---: | :------------------ | :------------------- |
|  7  |  应用层  | 为应用程序提供网络服务接口       | HTTP, FTP, SMTP, DNS |
|  6  |  表示层  | 数据格式转换、加密解密、压缩解压    | SSL/TLS, JPEG, ASCII |
|  5  |  会话层  | 建立、管理和终止会话连接        | NetBIOS, RPC         |
|  4  |  传输层  | 端到端的可靠/不可靠数据传输      | [[7.5 TCP协议基础|TCP]], UDP             |
|  3  |  网络层  | 路由选择、逻辑寻址（[[7.2 IP地址|IP地址]]）     | IP, ICMP, IGMP       |
|  2  | 数据链路层 | 帧的封装与解封、物理寻址（MAC地址） | Ethernet, PPP        |
|  1  |  物理层  | 比特流传输、物理介质规范        | 网线、光纤、中继器            |

## TCP/IP 参考模型

TCP/IP 模型是互联网实际使用的协议栈，它将 OSI 的七层简化为四层。

### 四层结构

| TCP/IP 层次 | 对应 OSI 层 | 主要协议 |
|:---:|:---:|:---|
| 应用层 | 应用层 + 表示层 + 会话层 | HTTP, SMTP, DNS, RTP, FTP |
| 传输层 | 传输层 | [[7.5 TCP协议基础|TCP]], UDP |
| 网际层 | 网络层 | IP, RARP, IGMP, ICMP |
| 网络接口层 | 数据链路层 + 物理层 | 以太网, 令牌环, 帧中继, ATM |

## 五层参考模型

五层模型是教学中常用的简化模型，结合了 OSI 和 TCP/IP 的特点。

### 五层结构与数据单元

| 层次 | 名称 | 数据单元 | 设备 |
|:---:|:---:|:---:|:---:|
| 5 | 应用层 | 报文 | - |
| 4 | 传输层 | 段 | - |
| 3 | 网络层 | 数据报 | 路由器 |
| 2 | 数据链路层 | 帧 | 交换机 |
| 1 | 物理层 | 比特流 | 路由器 |

## 关键要点

1. **分层设计的好处**：
   - 各层独立，便于标准化
   - 灵活性好，各层可独立升级
   - 便于学习和理解

2. **封装与解封装**：
   - 发送时从应用层向下逐层添加头部
   - 接收时从物理层向上逐层去除头部

3. **实际应用**：
   - 现实网络主要使用 TCP/IP 四层模型
   - OSI 七层模型主要用于理论学习和标准制定
   - 网络编程通过 [[7.6 套接字|Socket]] 接口与协议栈交互
