# listen 和 accept 函数

## 概述

在 TCP 服务器编程中，`listen()` 和 `accept()` 是两个核心函数。它们与 [[7.6 套接字#socket 函数|socket()]] 和 [[7.6 套接字#bind 函数|bind()]] 一起，构成了服务器端网络编程的基本流程：

```
socket() → bind() → listen() → accept() → 数据收发 → close()
```

## listen 函数

`listen()` 函数将一个主动套接字转换为**被动套接字**（监听套接字），表示内核应接受指向该套接字的连接请求。

### 函数原型

```c
#include <sys/socket.h>

int listen(int sockfd, int backlog);
```

### 参数说明

| 参数 | 说明 |
|:---|:---|
| `sockfd` | 由 [[7.6 套接字#socket 函数\|socket()]] 创建并经过 [[7.6 套接字#bind 函数\|bind()]] 绑定的套接字描述符 |
| `backlog` | 等待连接队列的最大长度 |

### 返回值

- **成功**：返回 0
- **失败**：返回 -1，并设置 `errno`

### backlog 参数详解

`backlog` 参数指定了**已完成连接队列**（completed connection queue）的最大长度。这涉及到 TCP 三次握手的过程：

```
┌──────────────────────────────────────────────────────────────┐
│                      服务器监听套接字                          │
│  ┌────────────────────┐    ┌────────────────────┐           │
│  │   未完成连接队列     │    │   已完成连接队列     │           │
│  │ (SYN_RCVD 状态)    │ →  │ (ESTABLISHED 状态) │ → accept()│
│  │                    │    │                    │           │
│  │  收到 SYN，发送     │    │  三次握手完成，      │           │
│  │  SYN+ACK，等待 ACK  │    │  等待 accept 取走   │           │
│  └────────────────────┘    └────────────────────┘           │
└──────────────────────────────────────────────────────────────┘
```

**两个队列的作用**：
1. **未完成连接队列**：存放已收到 SYN 但三次握手未完成的连接（SYN_RCVD 状态）
2. **已完成连接队列**：存放三次握手已完成，等待 `accept()` 取走的连接（ESTABLISHED 状态）

> **注意**：`backlog` 的实际含义在不同系统上可能略有不同。在 Linux 2.2+ 中，它指定的是**已完成连接队列**的最大长度。

### backlog 取值建议

- 对于高并发服务器，建议设置较大的值（如 128、256 或更高）
- 如果 `backlog` 设置过小，在高并发时可能导致客户端连接被拒绝
- 可以通过 `/proc/sys/net/core/somaxconn` 查看系统允许的最大值

## accept 函数

`accept()` 函数从**已完成连接队列**中取出一个已建立的连接。如果队列为空，`accept()` 会阻塞等待新连接的到来。

### 函数原型

```c
#include <sys/socket.h>

int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
```

### 参数说明

| 参数 | 说明 |
|:---|:---|
| `sockfd` | 监听套接字描述符（经过 `listen()` 的套接字） |
| `addr` | 输出参数，用于存放客户端的地址信息（可以为 NULL） |
| `addrlen` | 输入输出参数，输入时为 addr 缓冲区大小，输出时为实际地址长度 |

### 返回值

- **成功**：返回一个**新的套接字描述符**（已连接套接字），用于与客户端通信
- **失败**：返回 -1，并设置 `errno`

### 两种套接字的区别

`accept()` 返回后，服务器会同时拥有两种套接字：

| 套接字类型 | 描述符 | 作用 |
|:---|:---|:---|
| **监听套接字** | `sockfd`（listen 的参数） | 持续监听新的连接请求，生命周期与服务器相同 |
| **已连接套接字** | `accept()` 的返回值 | 与特定客户端通信，通信结束后关闭 |

```
                           ┌──────────────────┐
     客户端 A ─────────────→│ 已连接套接字 A    │
                           └──────────────────┘
                                    ↑
监听套接字 ──────────→ accept() ────┤
                                    ↓
                           ┌──────────────────┐
     客户端 B ─────────────→│ 已连接套接字 B    │
                           └──────────────────┘
```

### accept 的阻塞行为

默认情况下，`accept()` 是**阻塞**的：

- 如果已完成连接队列中有等待的连接，`accept()` 立即返回
- 如果队列为空，`accept()` 会阻塞进程，直到有新连接到来

可以通过设置套接字为非阻塞模式来改变这一行为。

## 完整示例：TCP 服务器

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

#define PORT 8080
#define BACKLOG 10
#define BUFFER_SIZE 1024

int main() {
    int listen_fd, conn_fd;
    struct sockaddr_in server_addr, client_addr;
    socklen_t client_len;
    char buffer[BUFFER_SIZE];

    // 1. 创建套接字
    listen_fd = socket(AF_INET, SOCK_STREAM, 0);
    if (listen_fd < 0) {
        perror("socket");
        exit(EXIT_FAILURE);
    }

    // 设置地址复用（避免 "Address already in use" 错误）
    int opt = 1;
    setsockopt(listen_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

    // 2. 绑定地址
    memset(&server_addr, 0, sizeof(server_addr));
    server_addr.sin_family = AF_INET;
    server_addr.sin_port = htons(PORT);
    server_addr.sin_addr.s_addr = htonl(INADDR_ANY);

    if (bind(listen_fd, (struct sockaddr*)&server_addr, sizeof(server_addr)) < 0) {
        perror("bind");
        exit(EXIT_FAILURE);
    }

    // 3. 开始监听
    if (listen(listen_fd, BACKLOG) < 0) {
        perror("listen");
        exit(EXIT_FAILURE);
    }

    printf("服务器启动，监听端口 %d...\n", PORT);

    // 4. 循环接受连接
    while (1) {
        client_len = sizeof(client_addr);

        // accept 会阻塞，直到有客户端连接
        conn_fd = accept(listen_fd, (struct sockaddr*)&client_addr, &client_len);
        if (conn_fd < 0) {
            perror("accept");
            continue;
        }

        // 打印客户端信息
        printf("客户端已连接: %s:%d\n",
               inet_ntoa(client_addr.sin_addr),
               ntohs(client_addr.sin_port));

        // 5. 与客户端通信
        ssize_t n = read(conn_fd, buffer, BUFFER_SIZE - 1);
        if (n > 0) {
            buffer[n] = '\0';
            printf("收到消息: %s\n", buffer);

            // 回复客户端
            const char *response = "Hello from server!";
            write(conn_fd, response, strlen(response));
        }

        // 6. 关闭已连接套接字
        close(conn_fd);
        printf("客户端断开连接\n\n");
    }

    // 关闭监听套接字（实际上不会执行到这里）
    close(listen_fd);
    return 0;
}
```

## TCP 连接流程图

```
       客户端                                    服务器
          │                                        │
          │                              socket()  │
          │                              bind()    │
          │                              listen()  │
          │                                        │
          │          ─────── SYN ───────→          │
          │                              [SYN_RCVD]│ 放入未完成队列
          │          ←──── SYN+ACK ─────          │
   [ESTABLISHED]                                   │
          │          ─────── ACK ───────→          │
          │                              [ESTABLISHED] 移到已完成队列
          │                              accept()  │ 从已完成队列取出
          │                                        │
          │          ←────── 数据 ──────→          │ 使用已连接套接字通信
          │                                        │
          │          ─────── FIN ───────→          │
          │          ←──── FIN+ACK ─────          │
          │                              close()   │
```

## 关键要点

1. **listen 的作用**：
   - 将套接字从主动模式转为被动模式
   - 创建连接队列，准备接受客户端连接
   - `backlog` 决定了队列长度，影响并发处理能力

2. **accept 的作用**：
   - 从已完成连接队列中取出一个连接
   - 返回新的套接字用于与客户端通信
   - 原监听套接字继续监听新连接

3. **两种套接字**：
   - 监听套接字：只用于接收连接请求，不传输数据
   - 已连接套接字：用于与特定客户端的数据传输

4. **服务器基本流程**：
   - `socket()` → `bind()` → `listen()` → `accept()` → `read()/write()` → `close()`

5. **并发服务器考虑**：
   - 使用 `fork()` 创建子进程处理每个连接（参见 [[11.8 基于多任务的多进程并发服务器]]）
   - 或使用多线程
   - 或使用 I/O 多路复用（select/poll/epoll）
