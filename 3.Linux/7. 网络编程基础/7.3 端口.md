# 端口

## 简介

端口（Port）是网络通信中一个非常重要的概念，它是一个**逻辑上的接口**，用来区分一台计算机上不同的网络服务或应用程序。你可以把一个 [[7.2 IP地址|IP地址]] 想象成一栋公寓大楼的地址，而端口号就是这栋大楼里的具体房间号。

---

## 作用

端口的主要作用就是让操作系统知道，当一个数据包到达时，应该把它交给哪个应用程序去处理。

举个简单的例子：你的电脑上可能同时运行着多个网络应用，比如：
- 一个浏览器正在访问网站
- 一个邮件客户端正在收发邮件
- 一个游戏程序正在和服务器通信

当一个数据包从互联网到达你的电脑时，它上面有两个关键信息：**目标 IP 地址**（你电脑的地址）和**目标端口号**。操作系统首先根据 IP 地址将数据包送到你的电脑，然后根据端口号将这个数据包精确地投递给对应的应用程序。

- 端口 80（或443）的数据包会交给浏览器
- 端口 25（或110）的数据包会交给邮件客户端
- 特定游戏端口的数据包会交给游戏程序

如果没有端口，所有的数据包都会涌向同一个地方，操作系统就无法区分它们属于哪个应用，网络通信将彻底混乱。

---

## 分类

端口号是一个16位的数字，范围从 0 到 65535。它们通常分为三类：

### 1. 知名端口（Well-known Ports）：0-1023

这些端口号是保留给一些最常用的网络服务的。它们由互联网号码分配机构（IANA）严格管理，以确保全球的通信一致性。

**常见的知名端口**：

| 端口号 | 服务 | 说明 |
|:---:|:---|:---|
| 20/21 | FTP | 文件传输协议 |
| 22 | SSH | 安全外壳协议 |
| 23 | Telnet | 远程登录协议 |
| 25 | SMTP | 简单邮件传输协议 |
| 53 | DNS | 域名系统 |
| 80 | HTTP | 超文本传输协议 |
| 110 | POP3 | 邮局协议版本3 |
| 143 | IMAP | 互联网消息访问协议 |
| 443 | HTTPS | 加密的 HTTP |

### 2. 注册端口（Registered Ports）：1024-49151

这些端口号通常分配给一些非标准但知名的应用程序和服务，比如各种数据库、游戏服务器等。开发者可以向 IANA 申请注册自己的端口号，以避免冲突。

**常见的注册端口**：

| 端口号 | 服务 | 说明 |
|:---:|:---|:---|
| 3306 | MySQL | MySQL数据库 |
| 5432 | PostgreSQL | PostgreSQL数据库 |
| 6379 | Redis | Redis缓存数据库 |
| 8080 | HTTP Proxy | HTTP代理/备用Web服务 |
| 27017 | MongoDB | MongoDB数据库 |

### 3. 动态/私有端口（Dynamic/Private Ports）：49152-65535

这些端口号通常不会被预先分配给特定服务。当一个客户端应用程序（比如你的浏览器）发起一个网络请求时，操作系统会随机地从这个范围分配一个端口号作为**源端口**，以便服务器能够将响应数据正确地发回给它。

---

## 关于服务器端口的建议

> **尽量不要将动态端口设置为服务器的监听端口**

**原因**：服务器作为服务提供者，通常绑定到**知名端口**（0-1023）或**注册端口**（1024-49151）。这些端口是固定的，不会被操作系统用作动态端口。

如果你让服务器监听一个动态端口，那么它就可能与正在运行的**客户端程序使用的临时端口**发生冲突。例如：
- 你有一个 Web 服务器监听在端口 50000
- 而一个浏览器刚好也临时使用了 50000 作为它的源端口

这会导致端口绑定失败，服务器无法启动。

---

## 查看端口使用情况

在 Linux 系统中，可以使用以下命令查看端口使用情况：

```bash
# 查看所有监听端口
netstat -tlnp

# 查看特定端口
netstat -tlnp | grep 8080

# 使用 ss 命令（更现代）
ss -tlnp

# 查看端口被哪个进程占用
lsof -i :80
```
