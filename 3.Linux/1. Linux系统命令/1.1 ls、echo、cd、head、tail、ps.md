# Linux 基础命令

本节介绍 Linux 系统中最常用的基础命令：`ls`、`echo`、`cd`、`head`、`tail`、`ps`。

---

## ls 命令

`ls` 命令是 Linux 中最常用和最基本的命令之一，它的全称是 *list segments*。这个命令的作用是**列出指定目录中的文件和子目录**。

基本上，它就是你在终端里查看一个文件夹里有什么内容的工具。

### ls 命令的基本用法

- `ls`：直接输入 `ls` 会列出当前工作目录下的所有文件和子目录
- `ls /路径/到/目录`：如果你想查看其他目录的内容，只需在 `ls` 后面加上该目录的路径。例如，`ls /home/user/Documents` 会列出 Documents 目录里的内容

### ls 命令的常用选项（Options）

`ls` 命令有很多选项可以用来改变它的行为和输出格式。在命令中，选项通常以一个或两个破折号开头（`-` 或 `--`）。

以下是一些最常用的选项：

| 选项 | 说明 |
|------|------|
| `-l` | 长格式（long format）。以长格式显示文件和目录的详细信息，包括文件权限、所有者、所属组、文件大小、创建或修改日期等 |
| `-a` | 全部（all）。显示所有文件，包括隐藏文件。在 Linux 中，以点（`.`）开头的文件和目录是隐藏的 |
| `-h` | 人性化（human-readable）。通常与 `-l` 结合使用，它会将文件大小以人类可读的格式显示，比如 KB、MB 或 GB，而不是字节 |
| `-t` | 按时间排序（sort by time）。按文件的修改时间排序，最近修改的会显示在最前面 |

### 示例

```bash
# 以长格式列出当前目录中的文件和目录
ls -l

# 列出当前目录中的所有文件和目录，包括隐藏的
ls -a

# 以长格式显示文件，并且将文件大小以人性化方式显示
ls -lh

# 按修改时间排序（从最新到最旧），并以长格式显示所有文件（包括隐藏文件）
ls -alt

# 给每个文件名和目录名都自动加上双引号（处理含空格或特殊字符时特别有用）
ls -Q

# 列出当前目录及其所有子目录中的所有文件和文件夹（递归显示）
ls -R
```

### ls -l 输出解析

```
drwxr-xr-x 1 feng feng  512 Jun 12 2019 Desktop
-rw-r--r-- 1 root root 1005 Sep 11 2019 server.csr
```

输出各列含义：
- **权限**：第一列（如 `drwxr-xr-x`）
- **创建者**：所有者用户名
- **文件大小**：以字节为单位
- **创建时间**：最后修改时间
- **文件名**：文件或目录名称

## echo 命令

在 Linux 中，`echo` 命令是一个非常基础且常用的命令，它的主要作用是在终端上**打印文本或变量值**。你可以把它想象成一个"回声"程序，你给它什么，它就把它原封不动地在屏幕上显示出来。

### echo 的基本用法

echo 最简单的用法就是打印一段文字。

```bash
echo "Hello, World!"
# 输出: Hello, World!
```

### 打印变量

echo 的另一个重要功能是**打印环境变量或用户定义的变量**。你需要使用 `$` 符号来引用变量。

```bash
# 打印当前用户的主目录路径（环境变量）
echo $HOME

# 打印你自定义的变量
my_var="Linux is great"
echo $my_var
```

### 打印命令结果

```bash
# 打印当前用户的主目录路径
echo `$HOME`
```

### 重定向输出

echo 经常与[[4.2 重定向、同步#文件描述符重定向|重定向]]结合使用，将文本内容写入文件，而不是仅仅显示在屏幕上。

```bash
# >: 将输出内容覆盖写入文件。如果文件不存在，则会创建它
echo "This is a new line." > my_file.txt
# 执行后，my_file.txt 文件里的所有内容都会被这行文字替换

# >>: 将输出内容追加到文件末尾，而不会覆盖原内容
echo "This is another line." >> my_file.txt
# 执行后，这行文字会添加到 my_file.txt 的末尾
```

### 常用选项

echo 还有一些常用的选项，可以控制它的行为：

| 选项 | 说明 |
|------|------|
| `-n` | 不换行。默认情况下，echo 在输出后会自动换行 |
| `-e` | 启用转义字符。这允许你使用一些特殊的字符序列，比如 `\n`（换行）和 `\t`（制表符） |

```bash
# 不换行输出
echo -n "This is a test. "
echo "No newline was added."
# 执行后，两行文字会显示在同一行

# 使用转义字符
echo -e "First line.\nSecond line with a tab\t"
# 输出会换行并包含一个制表符
```

总而言之，echo 是一个功能强大且灵活的命令，无论是简单的文本显示，还是复杂的脚本编程，它都是不可或缺的工具。

## cd 命令

`cd` 命令，全称是 *change directory*，它的作用就是**切换或改变你当前所在的目录**。在 Linux 终端中，你每次执行命令时，都会在一个特定的目录里，这个目录就是你的"当前工作目录"（current working directory）。`cd` 命令就是让你在文件系统中自由地"移动"到其他目录。

你可以把它想象成在文件管理器里双击一个文件夹，只不过是在命令行中完成。

### cd 的多种用法

#### 1. 切换到绝对路径

**绝对路径**是从文件系统的根目录（`/`）开始的完整路径。无论你当前在哪里，都可以通过绝对路径直接跳到目标目录。

```bash
# 假设你想进入 /usr/local/bin 目录
cd /usr/local/bin
# 执行后，你就会进入 /usr/local/bin
```

#### 2. 切换到相对路径

**相对路径**是从你当前所在目录开始的路径。它不需要从根目录开始。

```bash
# 假设你现在在 /home/user 目录下，你想进入 Documents 目录
cd Documents
# 执行后，你就会进入 /home/user/Documents
```

#### 3. 返回上一级目录

使用两个点 `..` 来表示当前目录的上一级目录。这在需要返回上层目录时非常方便。

```bash
# 假设你现在在 /home/user/Documents 目录下，你想返回到 /home/user
cd ..
# 执行后，你就会回到 /home/user

# 你可以连续使用 .. 来跳跃多级目录
# 从 /home/user/Documents/notes 目录直接返回到 /home/user
cd ../..
```

#### 4. 返回用户主目录（Home Directory）

有两种非常快速的方法可以回到你的主目录（通常是 `/home/你的用户名`）：

```bash
# 使用波浪号 ~
cd ~

# 不带任何参数
cd
```

#### 5. 返回上次所在的目录

使用一个破折号 `-`，可以让你快速返回到你上一次所在的目录。

```bash
# 假设你从 /home/user 切换到了 /etc/nginx。现在你想回到 /home/user，而不想输入完整的路径
cd -
# 执行后，你就会从 /etc/nginx 返回到 /home/user
```

## head 命令

`head` 命令是一个非常实用的文本处理工具，它的作用是**显示文件的开头部分**。默认情况下，它会打印文件的前10行内容。这在你想快速预览一个大型文件，或者只查看文件最开头几行内容时非常有用，可以避免加载整个文件。

### head 的多种用法

#### 1. 显示文件的前10行（默认行为）

这是 head 命令最基本的用法。你只需要在命令后面加上文件名即可。

```bash
# 假设有一个名为 access.log 的日志文件，你想快速查看它的开头
head access.log
# 执行后，它会显示 access.log 的前10行内容
```

#### 2. 显示指定行数

如果你想看的文件开头部分多于或少于10行，可以使用 `-n` 选项来指定具体行数。

```bash
# 显示 access.log 的前20行
head -n 20 access.log

# 只显示 access.log 的前5行
head -n 5 access.log
```

#### 3. 显示指定字节数

除了按行显示，你还可以使用 `-c` 选项来按字节显示文件的开头部分。

```bash
# 显示 access.log 的前500个字节
head -c 500 access.log
```

#### 4. 结合管道符（Pipe）使用

head 命令通常与其他命令结合使用，通过管道符 `|` 将前一个命令的输出作为 head 的输入。这在处理流数据时非常方便。

```bash
# 假设你想查看 ls -l 命令输出的前10行
ls -l | head

# 结合 cat 命令和管道符，查看多个文件的开头部分
cat file1.txt file2.txt | head -n 5
# 这个命令会先连接 file1.txt 和 file2.txt 的内容，然后将总输出的前5行显示出来
```

简而言之，head 命令是一个轻量级且高效的工具，尤其适用于快速预览大型文本文件的开头，节省时间和系统资源。

## tail 命令

`tail` 命令是一个非常常用的文本处理工具，它的作用是**显示文件的末尾部分**。默认情况下，它会打印文件的最后10行内容。与 head 命令相反，tail 在处理大型日志文件或实时监控文件内容变化时特别有用。

### tail 的多种用法

#### 1. 显示文件的最后10行（默认行为）

这是 tail 命令最基本的用法。你只需要在命令后面加上文件名即可。

```bash
# 假设有一个名为 error.log 的日志文件，你想快速查看最新的错误信息
tail error.log
# 执行后，它会显示 error.log 的最后10行内容
```

#### 2. 显示指定行数

如果你想看的文件末尾部分多于或少于10行，可以使用 `-n` 选项来指定具体行数。

```bash
# 显示 error.log 的最后20行
tail -n 20 error.log

# 只显示 error.log 的最后3行
tail -n 3 error.log
```

#### 3. 实时监控文件（tail -f）

这是 tail 最强大和最常用的功能之一。`tail -f`（f 代表 follow，跟踪）命令会**实时跟踪文件的内容变化**。只要有新的内容被写入文件，它就会立即显示在终端上。这对于监控实时的日志文件（比如 Web 服务器日志或应用程序日志）非常有用。

```bash
# 实时监控 nginx 的访问日志
tail -f /var/log/nginx/access.log
# 执行后，tail 命令会持续运行，任何新的访问请求都会立即显示在你的终端屏幕上
# 你可以使用 Ctrl + C 来退出
```

#### 4. 显示指定字节数

除了按行显示，你还可以使用 `-c` 选项来按字节显示文件的末尾部分。

```bash
# 显示 error.log 的最后100个字节
tail -c 100 error.log
```

#### 5. 结合管道符（Pipe）使用

tail 命令也经常与其他命令结合使用，通过管道符 `|` 将前一个命令的输出作为 tail 的输入。

```bash
# 假设你想查看 ls -l 命令输出的最后5行
ls -l | tail -n 5
# 这个命令会先列出当前目录中的文件和目录的详细信息，然后只显示最后5行内容
```

总之，tail 是一个必不可少的工具，尤其是在系统管理和故障排查中，它能让你高效地查看最新的日志和文件更新。

## ps 命令

`ps` 命令，全称是 *process status*，它的作用是**显示当前系统上正在运行的进程**（processes）。每个正在执行的程序或命令，在 Linux 系统中都被称为一个进程。ps 命令就像一个"快照"，它会抓取你在运行命令那一刻，所有正在运行的进程信息，并将其显示出来。

这个命令是系统管理员和开发者进行系统监控、故障排查和资源管理时的必备工具。

> 所有的进程都是 init 进程的子进程或者孙子进程

### ps 的多种用法

#### 1. 查看当前用户的进程（默认行为）

这是 ps 命令最基本的用法。不加任何选项时，它会显示与当前终端会话相关的进程。

```bash
ps
```

输出示例：
```
  PID TTY          TIME CMD
 1818 pts/0    00:00:00 bash
 2345 pts/0    00:00:00 ps
```

输出各列含义：
- **PID**：进程ID（Process ID），每个进程的唯一标识符
- **TTY**：进程运行的终端设备
- **TIME**：进程运行的累计CPU时间
- **CMD**：启动进程的命令名
- **PPID**：父进程ID
- **PRI**：进程优先级，可以为正也可以为负
- **VSZ**：虚拟内存的大小
- **RSS**：实际内存的大小

#### 2. 查看所有正在运行的进程（ps aux）

这是最常用也是最重要的 ps 命令组合之一。它会显示系统上所有用户的所有进程。

- `a`：显示所有用户的进程（包括其他用户的）
- `u`：以用户为中心显示进程，提供更多详细信息，如进程所有者、CPU 和内存使用情况等
- `x`：显示所有没有控制终端的进程

```bash
ps aux
```

输出示例（部分）：
```
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.0 170560  9448 ?        Ss   Aug24   0:02 /sbin/init
root       123  0.0  0.0 123456  1234 ?        S    Aug24   0:00 [kworker/u2:0]
user      1818  0.0  0.1 123456 12345 pts/0    S    Aug24   0:00 /bin/bash
```

#### 3. 查看特定进程（与 grep 结合）

通常，你不会关心所有进程，而只会关心某个特定的进程。这时，ps 命令通常与管道符 `|` 和 grep 命令结合使用，来过滤出你想要的进程。

```bash
# 假设你想查找所有与 nginx 相关的进程
ps aux | grep nginx
```

输出示例：
```
root      1234  0.0  0.1 123456  1234 ?        Ss   Aug24   0:00 nginx: master proc
www-data  1235  0.0  0.1 123456  1234 ?        S    Aug24   0:00 nginx: worker proc
user      3456  0.0  0.0 123456  1234 pts/0    S+   Aug24   0:00 grep nginx
```

> **注意**：结果中最后一行 `grep nginx` 是 grep 命令本身，因为它也是一个正在运行的进程。

#### 4. 以树状结构显示进程（ps -ef --forest）

如果你想了解进程之间的父子关系，可以使用 `--forest` 选项以树状结构来显示进程。

- `e`：显示所有进程（与 a 类似）
- `f`：以"完整格式"显示，提供更多信息
- `--forest`：以树状结构显示父子关系

```bash
ps -ef --forest
```

输出示例：
```
UID        PID  PPID  C STIME TTY          TIME CMD
root         1     0  0 12:00 ?        00:00:01 /sbin/init
root       123     1  0 12:00 ?        00:00:00  \_ /usr/lib/systemd/systemd --user
user      1818   123  0 12:05 pts/0    00:00:00      \_ -bash
user      2345  1818  0 12:06 pts/0    00:00:00          \_ ps -ef --forest
```

这清楚地显示了 `/sbin/init` 是父进程，而 `/usr/lib/systemd/systemd` 是其子进程，以此类推。

ps 命令是 Linux 系统管理的核心工具之一，通过它你可以了解系统正在做什么，并找出任何异常的进程。

## 关键要点

- `ls`：列出目录内容，`-l` 显示详情，`-a` 显示隐藏文件
- `echo`：打印文本和变量，支持重定向输出到文件
- `cd`：切换目录，`~` 回主目录，`-` 回上次目录，`..` 返回上级
- `head`：显示文件开头，默认10行，`-n` 指定行数
- `tail`：显示文件末尾，`-f` 实时跟踪文件变化
- `ps`：查看进程状态，`ps aux` 显示所有进程，结合 `grep` 过滤特定进程
