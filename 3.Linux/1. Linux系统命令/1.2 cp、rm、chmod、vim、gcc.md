# Linux 文件管理与开发命令

本节介绍 Linux 系统中常用的文件管理命令（`cp`、`rm`、`chmod`）和开发工具（`vim`、`gcc`）。

---

## cp 命令

`cp` 命令，全称是 *copy*，它的作用是**复制文件或目录**。它是 Linux 系统中最常用和最基本的命令之一。你可以用它来创建文件的副本，或者将文件和目录从一个位置复制到另一个位置。

### 基本语法

```bash
cp [参数] 源文件/文件夹 目标文件/文件夹
```

### 常用选项

| 选项 | 说明 |
|------|------|
| `-a` | 带属性复制。此选项通常在复制目录时使用，它保留链接、文件属性，并复制目录下的所有内容。其作用等于 `dpR` 参数组合 |
| `-d` | 复制时保留链接。这里说的链接相当于 Windows 系统中的快捷方式 |
| `-f` | 强制覆盖。覆盖已经存在的目标文件并且不给出提示。Ubuntu 默认是强制覆盖的 |
| `-i` | 覆盖提示。在覆盖目录之前给出提示，要求用户确认是否覆盖，输入 `y` 时目标文件被覆盖 |
| `-p` | 带权限的复制。除了复制的内容外，还将修改时间和访问权限也复制到新文件中 |
| `-r` | 带文件夹的复制。若给出的源文件是一个目录文件，此时将复制该目录下所有的子目录和文件 |
| `-l` | 创建链接。不复制文件，只是生成链接文件，但是只在部分系统有效 |

### 案例：cp 的多种用法

#### 1. 复制文件到指定目录

这是 cp 最基本的用法，你需要指定源文件（要复制的文件）和目标目录。

```bash
# 假设你有一个名为 report.txt 的文件，你想把它复制到 ~/Documents 目录中
cp report.txt ~/Documents/
# 执行后，report.txt 的一个副本就会出现在 ~/Documents/ 目录下
```

#### 2. 复制文件并重命名

如果你在复制文件的同时想给它一个新的名字，可以在目标路径中指定新的文件名。

```bash
# 假设你想将 notes.txt 文件复制到 ~/backups/ 目录，并命名为 notes_backup.txt
cp notes.txt ~/backups/notes_backup.txt
# 执行后，~/backups/ 目录下会有一个名为 notes_backup.txt 的文件，其内容与 notes.txt 相同
```

#### 3. 复制多个文件到指定目录

cp 命令可以同时复制多个文件，你只需在命令中列出所有要复制的文件，然后指定一个目标目录。

```bash
# 假设你想把 file1.txt、file2.txt 和 image.jpg 复制到 ~/project/ 目录中
cp file1.txt file2.txt image.jpg ~/project/
# 执行后，这三个文件都会被复制到 ~/project/ 目录下
```

#### 4. 复制目录（递归复制）

**注意**：cp 命令默认不能直接复制目录。你需要使用 `-r` 或 `-R` 选项（代表 recursive，递归）来复制整个目录及其所有内容（包括子目录和文件）。

```bash
# 假设你想把名为 my_project 的整个目录复制到 ~/projects_archive/
cp -r my_project ~/projects_archive/
# 执行后，my_project 目录及其所有内容都会被完整地复制到 ~/projects_archive/ 目录下
```

#### 5. 交互式复制（-i 选项）

cp 的 `-i` 选项（代表 interactive，交互式）会在复制过程中，如果目标位置已经存在同名文件，会提示你是否覆盖。这可以防止意外覆盖重要文件。

```bash
# 假设 ~/Documents 目录下已经有一个 report.txt 文件
cp -i report.txt ~/Documents/
# 执行后，你可能会看到类似以下提示：
# cp: overwrite '/home/user/Documents/report.txt'?
# 你可以输入 y (yes) 来确认覆盖，或输入 n (no) 来取消
```

#### 6. 强制覆盖（-f 选项）

与 `-i` 相反，`-f` 选项（代表 force，强制）会**无条件覆盖**目标文件，而不会给出任何提示。

```bash
cp -f report.txt ~/Documents/
# 执行后，~/Documents/report.txt 将会被直接覆盖，没有任何警告
```

cp 命令是日常文件管理中不可或缺的工具，熟练掌握这些用法能让你更高效地进行文件操作。

## rm 命令

`rm` 命令，全称是 *remove*，它的作用是**删除文件或目录**。它是 Linux 系统中一个非常重要，但同时也很危险的命令，因为被 rm 删除的文件通常无法轻易恢复。

### 常用选项

| 选项 | 说明 |
|------|------|
| `-i` | 删除确认。每个被删除的文件都需要确认 |
| `-f` | 强制删除。无需确认就会删除指定的文件 |
| `-r` | 目录删除。删除指定文件夹下的所有文件和子文件夹。参数可以组合：`-ir` 删除文件夹下面的每一个文件都会询问一次 |

### 案例：rm 的多种用法

#### 1. 删除文件

这是 rm 命令最基本的用法。你只需要在命令后面加上要删除的文件名即可。

```bash
# 假设你想删除一个名为 old_file.txt 的文件
rm old_file.txt
# 执行后，old_file.txt 将会被直接删除
```

#### 2. 删除多个文件

你可以一次性删除多个文件，只需在命令中列出它们，用空格隔开。

```bash
# 假设你想同时删除 file1.txt、file2.txt 和 image.jpg
rm file1.txt file2.txt image.jpg
# 执行后，这三个文件都会被删除
```

#### 3. 交互式删除（-i 选项）

为了防止意外删除重要文件，rm 提供了 `-i` 选项（代表 interactive，交互式）。这个选项会在删除每个文件之前都提示你进行确认。

```bash
# 假设你想删除 important_doc.docx
rm -i important_doc.docx
# 执行后，你可能会看到类似以下提示：
# rm: remove regular file 'important_doc.docx'?
# 你可以输入 y (yes) 来确认删除，或输入 n (no) 来取消
```

这是一个很好的习惯，可以避免误操作。

#### 4. 强制删除（-f 选项）

rm 的 `-f` 选项（代表 force，强制）会**无条件地删除文件**，即使文件是只读的，也不会给出任何提示或警告。这个选项非常危险，使用时必须格外小心。

```bash
# 假设你想强制删除一个只读文件 readonly.txt
rm -f readonly.txt
# 执行后，readonly.txt 将会被直接删除，没有任何提示
```

#### 5. 删除目录（递归删除）

**注意**：rm 命令默认不能直接删除目录。你需要使用 `-r` 或 `-R` 选项（代表 recursive，递归）来删除整个目录及其所有内容（包括子目录和文件）。

```bash
# 假设你想删除名为 my_old_project 的整个目录
rm -r my_old_project
# 执行后，my_old_project 目录及其内部的所有文件和子目录都会被删除
```

为了安全起见，通常会结合 `-i` 和 `-r` 一起使用：

```bash
rm -ri my_old_project
# 这个命令会递归地进入目录，并在删除每个文件或子目录前都进行提示确认，极大地降低了误删的风险
```

### rm -rf / 的警告

在 Linux 世界里，`rm -rf /` 是一个非常臭名昭著的命令，它会**强制递归地删除你的文件系统的根目录**。这意味着，它会尝试删除系统上的所有文件。一旦执行，你的系统将几乎无法恢复，并且会立即崩溃。

**请永远不要在生产环境或任何重要系统上执行 `rm -rf /`。**

## chmod 命令

在 Linux 中，`chmod` 命令是 *change mode* 的缩写，它的作用是**改变文件或目录的访问权限**。如果需要在 C/C++ 程序中修改文件权限，可以使用 [[3.5 权限控制函数|chmod() 系统调用函数]]。

在 Linux 系统中，每个文件和目录都有三组基本的权限：

- **读取（read, r）**：允许查看文件内容或列出目录中的文件
- **写入（write, w）**：允许修改、编辑或删除文件内容；对于目录，允许创建、删除或重命名文件
- **执行（execute, x）**：允许运行文件（如果是可执行程序）或进入目录

这些权限分别应用于三类用户：
- **文件所有者（owner, u）**：创建文件的用户
- **文件所有者所在的组（group, g）**：文件所有者所属的用户组
- **其他用户（others, o）**：既不是文件所有者，也不属于该用户组的其他用户
- **所有用户（all, a）**：指以上三者的总和

### chmod 命令的两种常用模式

chmod 命令有两种主要的模式来设置权限：符号模式和数字模式。

#### 1. 符号模式（Symbolic Mode）

这种模式使用字母和符号来设置权限，语法相对直观。

**语法**：`chmod [ugoa][+-=][rwx] 文件/目录`

- `[ugoa]`：指定要修改的用户类型
- `[+-=]`：指定操作类型
  - `+`：添加权限
  - `-`：移除权限
  - `=`：设置权限（会覆盖掉之前的权限）
- `[rwx]`：指定要设置的权限

**案例**：

```bash
# 给所有用户添加文件的执行权限
chmod a+x my_script.sh

# 从所有者那里移除文件的写入权限
chmod u-w important_file.txt

# 将文件的所有者和组权限设置为读写，其他用户没有任何权限
chmod ug=rw,o= my_config.conf
```

#### 2. 数字模式（Octal Mode）

这种模式使用数字来表示权限，每个数字代表一组权限。这种方法更紧凑，常用于脚本中。

**权限的数字对应关系**：
- `r`（读取）= 4（二进制：0100）
- `w`（写入）= 2（二进制：0010）
- `x`（执行）= 1（二进制：0001）
- 没有权限 = 0（二进制：0000）

通过将这些数字相加，可以得到一个权限组合：
- `rwx` = 4 + 2 + 1 = 7
- `rw-` = 4 + 2 + 0 = 6
- `r-x` = 4 + 0 + 1 = 5
- `r--` = 4 + 0 + 0 = 4

数字模式使用一个三位数的数字来设置权限，这三个数字依次代表**所有者、组、其他用户**。

**语法**：`chmod [xyz] 文件/目录`
- `x`：所有者权限
- `y`：组权限
- `z`：其他用户权限

**案例**：

```bash
# 设置文件所有者有读写执行权限，组和其他用户只有读和执行权限
# 所有者权限：rwx = 7
# 组权限：r-x = 5
# 其他权限：r-x = 5
chmod 755 my_script.sh
# 这个权限组合 755 是最常见的，通常用于可执行脚本或程序

# 设置文件所有者有读写权限，组和其他用户只有读权限
# 所有者权限：rw- = 6
# 组权限：r-- = 4
# 其他权限：r-- = 4
chmod 644 my_file.txt
# 这个权限组合 644 是最常见的，通常用于普通文件
```

理解 chmod 命令对于 Linux 系统管理至关重要，它能帮助你控制谁可以访问和操作你的文件。

## vim 命令

vim 是 Linux 中最强大和最流行的文本编辑器之一。它是 vi 编辑器的增强版本。

### 打开文件

在终端中输入 `vim` 命令，后面跟上文件名：

```bash
vim my_file.txt
```

如果文件不存在，vim 会创建一个新文件。

### 主要模式

vim 有三种主要的工作模式：

| 模式 | 说明 |
|------|------|
| **命令模式** | 这是你打开 vim 时的默认模式。在这个模式下，你不能直接输入文本，而是可以执行各种命令，比如复制、粘贴、删除行，或者在文件中移动光标。**按 `u` 撤销，按 `uu` 撤销一行，按 `Ctrl+r` 返回撤销，`dd` 删除一行** |
| **插入模式** | 在这个模式下，你才能像在其他编辑器中一样输入文本。要进入这个模式，你需要从命令模式**按下 `i`、`a` 或 `o` 等键** |
| **末行模式** | 在这个模式下，你可以在屏幕底部输入以 `:` 开头的命令，比如保存、退出、搜索或替换。要进入这个模式，你需要从命令模式**按下 `:` 键** |

### 退出保存

这是初学者最容易困惑的地方。在 vim 中，你不能直接关闭终端来退出。

1. 首先，确保你处于**命令模式**（如果不是，按 `Esc` 键）
2. 然后，按下 `:` 进入末行模式
3. 最后，输入以下命令之一：

| 命令 | 说明 |
|------|------|
| `:w` | 保存文件，但不要退出 |
| `:q` | 退出文件，如果文件没有改动 |
| `:wq` | **保存并退出**（write 和 quit） |
| `:q!` | **强制退出**，放弃所有未保存的更改 |

## gcc 命令

gcc 是 GNU Compiler Collection（GNU 编译器集合）的缩写，它是 Linux 系统中最常用的 C/C++ 编译器。

### 基本语法

```bash
gcc [选项] <源文件> [-o <输出文件>]
```

- `gcc`：是调用 GCC 编译器的命令
- `[选项]`：用于控制编译过程的参数，比如指定优化级别、链接库、定义宏等
- `<源文件>`：需要编译的源代码文件，可以是 `.c`、`.cpp` 等
- `[-o <输出文件>]`：这是一个可选选项，用于指定编译后生成的可执行文件的名称。如果不指定，GCC 会默认生成一个名为 `a.out` 的文件

### gcc 编译过程

理解 GCC 的工作原理有助于更好地使用它。编译过程通常分为四个阶段：**预处理、编译、汇编**和**链接**。

1. **预处理**：处理源代码文件中的预处理指令，如 `#include` 和 `#define`，它会展开宏定义，并包含头文件内容，生成一个 `.i`（C 语言）或 `.ii`（C++ 语言）的文件
2. **编译**：将预处理后的文件翻译成汇编语言，生成一个 `.s` 文件
3. **汇编**：将汇编语言代码转换成机器语言，生成一个可重定位的**目标文件**（`.o` 文件）。这个文件是二进制格式，但还不能直接运行
4. **链接**：将所有目标文件以及所需的库文件（如标准库）合并在一起，生成最终的**可执行文件**

### 常见选项

#### 1. 控制编译阶段的选项

| 选项 | 说明 |
|------|------|
| `-E` | 只进行预处理。例如：`gcc -E test.c -o test.i` |
| `-S` | 只进行编译，生成汇编代码。例如：`gcc -S test.c` |
| `-c` | 只进行编译和汇编，生成目标文件。例如：`gcc -c test.c -o test.o` |

#### 2. 警告和错误选项

| 选项 | 说明 |
|------|------|
| `-Wall` | 启用所有常见的警告。这是一个非常重要的选项，建议在所有项目中使用，它可以帮助你发现潜在的 bug |
| `-Wextra` | 启用更多额外的警告 |
| `-Werror` | 将所有警告视为错误。如果出现警告，编译将失败 |
| `-w` | 关闭所有警告 |

#### 3. 优化选项

| 选项 | 说明 |
|------|------|
| `-O0` | 不进行任何优化，这是默认设置，编译速度最快 |
| `-O1` | 进行基本的优化 |
| `-O2` | 进行中等程度的优化，这是生产环境中最常用的优化级别 |
| `-O3` | 进行最高级别的优化，可能会增加编译时间，但通常会生成更快的代码 |
| `-Os` | 优化以减小可执行文件的大小 |

#### 4. 链接选项

| 选项 | 说明 |
|------|------|
| `-l<库名>` | 链接特定的库。例如，要使用数学函数，需要链接数学库：`gcc main.c -lm`。**注意**：库名是去掉 `lib` 前缀和 `.a` 或 `.so` 后缀的部分 |
| `-L<库路径>` | 指定额外的库文件搜索路径。例如：`gcc main.c -L/usr/local/lib -lmylib` |

#### 5. 头文件和宏定义选项

| 选项 | 说明 |
|------|------|
| `-I<头文件路径>` | 指定额外的头文件搜索路径。例如：`gcc main.c -I/usr/local/include` |
| `-D<宏定义>` | 在编译时定义宏。例如：`gcc main.c -DDEBUG`，这等同于在代码中写 `#define DEBUG` |
| `-U<宏定义>` | 取消宏定义 |

### 案例1：简单编译

```c
#include <stdio.h>

int main() {
    printf("Hello, GCC!\n");
    return 0;
}
```

**编译并运行**：

```bash
# 1. 最简单的编译
gcc hello.c
# 这会生成一个默认的可执行文件 a.out，你可以运行它：
./a.out

# 2. 指定输出文件名
gcc hello.c -o hello
# 这会生成一个名为 hello 的可执行文件
./hello
```

### 案例2：多文件编译

当项目包含多个源文件时，可以分步编译：

**func.h**（头文件）：
```c
#ifndef FUNC_H
#define FUNC_H

void say_hello();

#endif
```

**func.c**（函数实现）：
```c
#include <stdio.h>
#include "func.h"

void say_hello() {
    printf("Hello from the function!\n");
}
```

**main.c**（主程序）：
```c
#include "func.h"

int main() {
    say_hello();
    return 0;
}
```

**分步编译过程**：

```bash
# 1. 生成目标文件
gcc -c main.c -o main.o
gcc -c func.c -o func.o
# 现在你会看到两个目标文件：main.o 和 func.o

# 2. 链接目标文件
gcc main.o func.o -o myapp
# 这会将两个目标文件链接起来，生成最终的可执行文件 myapp

# 3. 运行程序
./myapp
```

## 关键要点

- `cp`：复制文件或目录，`-r` 递归复制目录，`-i` 交互式确认
- `rm`：删除文件或目录，`-r` 递归删除目录，`-f` 强制删除，**谨慎使用**
- `chmod`：修改文件权限，`755` 常用于可执行文件，`644` 常用于普通文件
- `vim`：强大的文本编辑器，三种模式（命令、插入、末行），`:wq` 保存退出
- `gcc`：C/C++ 编译器，`-o` 指定输出，`-Wall` 开启警告，`-c` 只编译不链接
