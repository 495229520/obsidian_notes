# 11.1 进程简介

## 1. 什么是进程

> **进程是资源分配的基本单位**：每个进程都有自己独立的[[1.3 进程的虚拟地址空间|地址空间]]、文件描述符、内存和数据段，这意味着一个进程无法直接访问另一个进程的内存。
>
> **进程是占用内存空间的正在运行的程序**。

---

## 2. 进程的生命周期

一个进程在其生命周期中，通常会经历以下几种状态：

```
                    时间片用完
                  ┌───────────┐
                  │           ▼
              ┌───────┐   ┌───────┐
    创建 ────►│ 就绪态 │   │ 运行态 │────► 终止
              └───────┘   └───────┘
                  ▲           │
                  │           │ 等待I/O或资源
                  │       ┌───────┐
                  └───────│ 阻塞态 │
                I/O完成   └───────┘
```

| 状态 | 英文 | 描述 |
|------|------|------|
| **运行态** | Running | 进程正在 CPU 上执行，或正在等待被 CPU 调度 |
| **就绪态** | Ready | 进程已经具备了所有运行条件，只等待 CPU 分配时间片 |
| **阻塞态** | Blocked | 进程因等待某事件的发生而暂停执行（如等待 I/O 操作完成、等待从设备读取数据、等待接收信号等） |

### 2.1 CPU 调度

假如有 2 个就绪态的 CPU 和双核 CPU，则有 4 个核 CPU，意思是 1 个 CPU 中可能包含多个运算器核心。每个核只能运行一个进程。

进程经历多次状态的切换：
- **就绪 → 运行**：被调度程序选中
- **运行 → 就绪**：时间片用完
- **运行 → 阻塞**：等待 I/O 或资源
- **阻塞 → 就绪**：等待的事件发生

---

## 3. 进程 ID（PID）

**进程 ID**，通常简称为 **PID**，是操作系统为每个正在运行的进程分配的一个**唯一**的数字标识符。

- 可以把 PID 想象成每个进程的"身份证号"，当你启动一个程序时，操作系统将会给它分配一个 PID，这个号是**独一无二**的，直到你把程序关闭后，该号才会被回收
- 1 号分配给操作系统启动的第一个程序（用于初始化操作系统），每个进程，包括 init 进程无法获得 0 值

### 3.1 进程相关概念

| 概念 | 说明 |
|------|------|
| **进程管理和调度** | 内核使用 PID 来跟踪和管理系统中的所有进程。当需要对某个进程进行操作时（比如暂停、继续或终止），操作系统会使用 PID 来精确地识别目标进程 |
| **进程间通信** | 在进行[[11.2 进程间通信IPC|进程间通信 (IPC)]] 时，PID 是一个重要概念。进程可以通过另一个进程的 PID 来发送信号或建立通信 |
| **查找和监控进程** | 系统管理员和开发者经常使用 PID 来监控和调度进程。例如，可以使用 `ps` 或 `top` 命令来查看当前有哪些正在运行的进程及其 PID，然后使用 `kill` 命令根据 PID 来终止进程 |

## 4. 创建进程：fork() 函数

### 4.1 fork() 的作用

`fork()` 函数用于创建进程的进程副本，`fork()` 函数的本质是复制。

- 当你使用 `exec()` 系列函数（如 `execl()`、`execv()` 等）创建进程时，新进程会加载并执行一个**全新的程序文件**，比如执行 `ls` 命令时就是这样情况
- 但 `fork()` 的行为完全不同，它会创建一个与调用 `fork()` 的原始进程**几乎一模一样的进程副本**。这个新创建的进程被称为**子进程 (child process)**，而调用 `fork()` 的原始进程被称为**父进程 (parent process)**

### 4.2 子进程继承父进程的资源

子进程会继承父进程的几乎所有资源：

| 继承项 | 说明 |
|--------|------|
| **内存内数据** | 在调用 `fork()` 的那一刻，子进程获得父进程内存数据的副本 |
| **文件描述符** | 所有打开的文件描述符 |
| **程序计数器** | 子进程的程序计数器（即将执行的下一条指令的地址）与父进程相同，即从 `fork()` 之后的位置继续执行 |
| **环境变量** | 复制父进程的环境变量 |

### 4.3 fork() 返回值

```c
pid_t pid = fork();
```

| 返回值 | 含义 |
|--------|------|
| `> 0` | 在父进程中，返回子进程的 PID |
| `== 0` | 在子进程中，返回 0 |
| `< 0` | 创建失败，返回 -1 |

### 4.4 代码示例

```c
#include <stdio.h>
#include <unistd.h>

int gval = 10;

int main(void) {
    pid_t pid;
    int lval = 25;      // lval 复制为 25
    gval++;             // gval 复制为 11
    lval += 5;          // lval 复制为 30

    pid = fork();
    if (pid == 0) {     // 子进程
        gval++;         // gval 变为 12
        lval++;         // lval 变为 31
    } else {            // 父进程
        gval--;         // gval 变为 10
        lval--;         // lval 变为 29
    }

    if (pid == 0) {
        printf("子进程: gval=%d, lval=%d\n", gval, lval);
        // 输出: 子进程: gval=12, lval=31
    } else {
        printf("父进程: gval=%d, lval=%d\n", gval, lval);
        // 输出: 父进程: gval=10, lval=29
    }

    return 0;
}
```

**关键点**：`fork()` 之后，父子进程拥有**独立的内存空间**，对变量的修改互不影响。

> **注意**：子进程终止后，父进程需要调用 `wait()` 或 `waitpid()` 来回收子进程资源，否则子进程会变成[[11.7 僵尸进程|僵尸进程]]。

## 5. 进程与线程

> **[[12.1 线程简介|线程]]是 CPU 调度的基本单位**：一个进程可以包含多个线程，所有线程共享进程的资源（如[[1.3 进程的虚拟地址空间|地址空间]]、文件描述符等）。线程之间的通信比进程间通信更简单，开销更小。

| 对比项 | 进程 | 线程 |
|--------|------|------|
| **资源** | 独立的地址空间和资源 | 共享进程的地址空间和资源 |
| **创建开销** | 较大 | 较小 |
| **通信方式** | 需要 IPC 机制 | 可直接访问共享内存 |
| **隔离性** | 强（一个进程崩溃不影响其他进程） | 弱（一个线程崩溃可能导致整个进程崩溃） |

---

## 6. 关键要点

1. **进程是资源分配的基本单位**，每个进程有独立的[[1.3 进程的虚拟地址空间|虚拟地址空间]]
2. **进程的三种状态**：运行态、就绪态、阻塞态
3. **PID** 是进程的唯一标识符，用于进程管理和[[11.2 进程间通信IPC|进程间通信]]
4. **fork()** 创建子进程，子进程继承父进程的资源副本
5. **线程**比进程轻量，同一进程的线程共享资源
