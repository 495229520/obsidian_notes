# 11.7 僵尸进程

## 1. 什么是僵尸进程

当一个子进程终止时，它的进程描述符不会立即被释放，而是会保留一部分信息（如进程 ID、终止状态、资源使用统计等），等待父进程通过 `wait()` 或 `waitpid()` 来获取。在父进程获取这些信息之前，子进程就处于**僵尸状态（Zombie）**。

### 1.1 为什么会产生僵尸进程

根据 [[11.1 进程简介#4. 创建进程：fork() 函数|fork()]] 函数的调用特性：

- 向创建子进程的父进程返回子进程的 PID
- 向子进程返回 0

**父进程会持有子进程的 PID**，因此可以通过子进程的 PID 来销毁子进程。换言之，如果父进程不主动销毁子进程，子进程就会一直存在。

```
┌─────────────────────────────────────────────────────────────┐
│                        进程状态转换                           │
│                                                             │
│   运行态 ──────→ 终止 ──────→ 僵尸态 ──────→ 彻底消失         │
│              (子进程退出)   (等待父进程回收)  (父进程调用wait) │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 僵尸进程的危害

- 僵尸进程虽然不占用 CPU 和内存资源，但会占用**进程号（PID）**
- 系统的 PID 数量是有限的，大量僵尸进程会耗尽可用的 PID
- 导致无法创建新进程

## 2. 销毁僵尸进程：wait() 函数

父进程可以使用 `wait()` 函数来等待子进程终止，并回收其资源。

### 2.1 函数原型

```c
#include <sys/wait.h>

pid_t wait(int *status);
```

### 2.2 参数与返回值

| 项目 | 说明 |
|:---|:---|
| `status` | 输出参数，用于保存子进程的终止状态信息（可以为 NULL） |
| **返回值** | 成功时返回终止的子进程 PID；失败时返回 -1 |

### 2.3 status 状态解析宏

| 宏 | 说明 |
|:---|:---|
| `WIFEXITED(status)` | 子进程正常终止时返回真（true） |
| `WEXITSTATUS(status)` | 返回子进程的退出值（需要先用 `WIFEXITED` 判断） |
| `WIFSIGNALED(status)` | 子进程被信号终止时返回真 |
| `WTERMSIG(status)` | 返回导致子进程终止的信号编号 |
| `WIFSTOPPED(status)` | 子进程被暂停时返回真 |
| `WSTOPSIG(status)` | 返回导致子进程暂停的信号编号 |

### 2.4 wait() 的阻塞特性

调用 `wait()` 函数时：
- 如果**有子进程已终止**：函数立即返回，回收该子进程资源
- 如果**没有已终止的子进程**：函数会**阻塞**，直到有子进程终止

### 2.5 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main(void) {
    int status;
    pid_t pid = fork();

    if (pid == 0) {
        // 子进程
        printf("子进程运行中...\n");
        sleep(3);
        return 10;  // 返回值 10，低 8 位返回给操作系统
    } else {
        // 父进程
        printf("父进程等待子进程终止...\n");
        pid_t child_pid = wait(&status);  // 阻塞等待

        if (WIFEXITED(status)) {
            printf("子进程 %d 正常终止，返回值: %d\n",
                   child_pid, WEXITSTATUS(status));
        }
    }
    return 0;
}
```

**输出示例**：
```
父进程等待子进程终止...
子进程运行中...
(等待 3 秒)
子进程 12345 正常终止，返回值: 10
```

## 3. 非阻塞等待：waitpid() 函数

`waitpid()` 提供了比 `wait()` 更灵活的控制，支持非阻塞模式和指定等待特定子进程。

### 3.1 函数原型

```c
#include <sys/wait.h>

pid_t waitpid(pid_t pid, int *status, int options);
```

### 3.2 参数说明

| 参数 | 说明 |
|:---|:---|
| `pid` | 指定要等待的子进程 |
| `status` | 同 `wait()` 的 status 参数 |
| `options` | 控制选项，可以实现非阻塞等待 |

### 3.3 pid 参数取值

| 值 | 含义 |
|:---|:---|
| `> 0` | 等待进程 ID 等于 pid 的子进程 |
| `== -1` | 等待任意子进程（等同于 `wait()`） |
| `== 0` | 等待与调用进程同一进程组的任意子进程 |
| `< -1` | 等待进程组 ID 等于 pid 绝对值的任意子进程 |

### 3.4 options 选项

| 选项 | 说明 |
|:---|:---|
| `0` | 阻塞等待（默认行为） |
| `WNOHANG` | 非阻塞模式：如果没有已终止的子进程，立即返回 0 |
| `WUNTRACED` | 同时报告暂停状态的子进程 |

### 3.5 示例：非阻塞等待

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main(void) {
    int status;
    pid_t pid = fork();

    if (pid == 0) {
        // 子进程
        sleep(5);
        return 20;
    } else {
        // 父进程：非阻塞轮询
        while (1) {
            pid_t result = waitpid(pid, &status, WNOHANG);

            if (result == -1) {
                perror("waitpid");
                break;
            } else if (result == 0) {
                // 子进程还在运行
                printf("子进程还在运行，父进程可以做其他事...\n");
                sleep(1);
            } else {
                // 子进程已终止
                if (WIFEXITED(status)) {
                    printf("子进程 %d 终止，返回值: %d\n",
                           result, WEXITSTATUS(status));
                }
                break;
            }
        }
    }
    return 0;
}
```

## 4. 信号处理方式：signal() 函数

除了主动调用 `wait()`/`waitpid()`，还可以通过**注册信号处理函数**来自动回收僵尸进程。

### 4.1 SIGCHLD 信号

当子进程终止时，内核会向父进程发送 `SIGCHLD` 信号。父进程可以注册一个信号处理函数，在收到信号时调用 `wait()` 回收子进程。

### 4.2 signal() 函数原型

```c
#include <signal.h>

typedef void (*sighandler_t)(int);

sighandler_t signal(int signum, sighandler_t handler);
```

**函数原型的完整形式**：
```c
void (*signal(int signo, void (*func)(int)))(int);
```

### 4.3 参数说明

| 参数 | 说明 |
|:---|:---|
| `signum` | 信号编号（如 `SIGCHLD`、`SIGINT`、`SIGALRM` 等） |
| `handler` | 信号处理函数，或特殊值 `SIG_IGN`（忽略）、`SIG_DFL`（默认行为） |

### 4.4 返回值

- **成功**：返回之前的信号处理函数指针
- **失败**：返回 `SIG_ERR`

### 4.5 常用信号

| 信号 | 说明 |
|:---|:---|
| `SIGCHLD` | 子进程状态改变（终止或暂停）时发送给父进程 |
| `SIGALRM` | 由 `alarm()` 函数设置的定时器到期时触发 |
| `SIGINT` | 用户按下 `Ctrl+C` 时发送 |
| `SIGTERM` | 终止请求信号 |

### 4.6 示例：使用信号自动回收僵尸进程

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <signal.h>
#include <sys/wait.h>

void sigchld_handler(int signo) {
    int status;
    pid_t pid;

    // 使用循环处理可能同时终止的多个子进程
    while ((pid = waitpid(-1, &status, WNOHANG)) > 0) {
        printf("子进程 %d 已回收\n", pid);
    }
}

int main(void) {
    // 注册 SIGCHLD 信号处理函数
    signal(SIGCHLD, sigchld_handler);

    pid_t pid = fork();

    if (pid == 0) {
        // 子进程
        printf("子进程 %d 运行中...\n", getpid());
        sleep(2);
        printf("子进程退出\n");
        exit(0);
    } else {
        // 父进程继续执行其他任务
        printf("父进程继续工作...\n");
        for (int i = 0; i < 5; i++) {
            printf("父进程工作中... %d\n", i);
            sleep(1);
        }
    }

    return 0;
}
```

## 5. alarm() 定时器函数

`alarm()` 函数用于设置一个定时器，在指定秒数后向进程发送 `SIGALRM` 信号。

### 5.1 函数原型

```c
#include <unistd.h>

unsigned int alarm(unsigned int seconds);
```

### 5.2 参数与返回值

| 项目 | 说明 |
|:---|:---|
| `seconds` | 定时器时间（秒）。设为 0 可取消已设置的定时器 |
| **返回值** | 返回之前设置的定时器剩余秒数，如果之前没有设置则返回 0 |

### 5.3 特点

- 每个进程只能有一个 `alarm` 定时器
- 再次调用 `alarm()` 会重置定时器
- `SIGALRM` 的默认处理是终止进程，通常需要注册自定义处理函数

### 5.4 示例代码

```c
#include <stdio.h>
#include <unistd.h>
#include <signal.h>

void timeout_handler(int signo) {
    printf("定时器触发！\n");
    alarm(2);  // 重新设置定时器，实现周期性触发
}

int main(void) {
    signal(SIGALRM, timeout_handler);
    alarm(2);  // 2 秒后触发

    while (1) {
        printf("等待中...\n");
        sleep(1);
    }

    return 0;
}
```

---

## 6. 关键要点

1. **僵尸进程**：子进程终止后，父进程未调用 `wait()` 回收，子进程进入僵尸态
2. **wait()**：阻塞等待任意子进程终止并回收
3. **waitpid()**：更灵活，支持非阻塞模式（`WNOHANG`）和指定子进程
4. **signal()**：注册信号处理函数，可用于自动回收僵尸进程
5. **SIGCHLD**：子进程状态改变时内核发送给父进程的信号
6. **alarm()**：设置定时器，到期时发送 `SIGALRM` 信号
