# 11.4 共享内存 (Shared Memory)

## 1. System V IPC 简介

**System V IPC** 是 Unix/Linux 系统中一组经典的[[11.2 进程间通信IPC|进程间通信]]机制，主要包括三种：

| 机制 | 说明 | 主要用途 |
|------|------|----------|
| **共享内存** | 多个进程共享同一块物理内存 | 大量数据的高速交换 |
| **[[11.5 信号量\|信号量]]** | 用于进程同步的计数器 | 控制对共享资源的访问 |
| **[[11.6 消息队列\|消息队列]]** | 内核中的消息链表 | 异步消息传递 |

### 1.1 IPC 键值 (key)

System V IPC 使用**键值 (key)** 来标识 IPC 对象：

- 不同进程可以通过相同的 key 来访问同一个 IPC 对象
- `IPC_PRIVATE`（值为 0）创建私有的 IPC 对象，只有亲缘关系的进程能访问
- 通常使用 `ftok()` 函数生成唯一的 key

```c
#include <sys/types.h>
#include <sys/ipc.h>

key_t ftok(const char *pathname, int proj_id);
```

- `pathname`：一个存在的文件路径
- `proj_id`：项目 ID（只使用低 8 位）
- 返回值：成功返回 key，失败返回 -1

## 2. 共享内存原理

共享内存是**最快的 IPC 方式**，因为数据不需要在进程之间复制——多个进程直接访问同一块物理内存。

### 2.1 工作原理

```
         进程 A                      进程 B
    ┌──────────────┐            ┌──────────────┐
    │  用户空间     │            │  用户空间     │
    │              │            │              │
    │  ┌────────┐  │            │  ┌────────┐  │
    │  │ 虚拟   │  │            │  │ 虚拟   │  │
    │  │ 地址   │  │            │  │ 地址   │  │
    │  └────┬───┘  │            │  └────┬───┘  │
    └───────┼──────┘            └───────┼──────┘
            │                          │
            │      页表映射            │
            ▼                          ▼
    ┌───────────────────────────────────────────┐
    │              共享内存区域                   │
    │         （内核空间的物理内存）              │
    └───────────────────────────────────────────┘
```

1. 进程 A 调用 `shmget()` 创建一块共享内存
2. 进程 A 调用 `shmat()` 将共享内存映射到自己的[[1.3 进程的虚拟地址空间|虚拟地址空间]]
3. 进程 B 通过相同的 key 获取同一块共享内存
4. 进程 B 也将其映射到自己的地址空间
5. 两个进程可以像访问普通内存一样读写这块区域

### 2.2 优缺点

| 优点 | 缺点 |
|------|------|
| **速度最快**：无需数据复制 | **需要同步机制**：必须配合[[11.5 信号量\|信号量]]等避免竞态条件 |
| **大容量**：适合传输大量数据 | **生命周期管理**：需要手动删除 |
| **灵活**：可以存储任意数据结构 | **安全性**：需要设置适当的权限 |

## 3. 相关函数

### 3.1 shmget() - 创建/获取共享内存

```c
#include <sys/ipc.h>
#include <sys/shm.h>

int shmget(key_t key, size_t size, int shmflg);
```

| 参数 | 说明 |
|------|------|
| `key` | IPC 键值，用于标识共享内存。使用 `IPC_PRIVATE` 创建新的私有共享内存 |
| `size` | 共享内存的大小（字节）。实际分配会向上取整到页大小的倍数 |
| `shmflg` | 标志位，包括权限和选项 |
| 返回值 | 成功返回共享内存标识符 (shmid)，失败返回 -1 |

**常用标志位**：

| 标志 | 说明 |
|------|------|
| `IPC_CREAT` | 如果不存在则创建 |
| `IPC_EXCL` | 与 `IPC_CREAT` 一起使用，如果已存在则失败 |
| `0666` | 权限位（类似文件权限） |

**示例**：

```c
// 创建或获取共享内存
int shmid = shmget(key, 1024, IPC_CREAT | 0666);

// 创建新的共享内存（如果已存在则失败）
int shmid = shmget(key, 1024, IPC_CREAT | IPC_EXCL | 0666);

// 获取已存在的共享内存
int shmid = shmget(key, 0, 0);
```

### 3.2 shmat() - 映射共享内存

```c
void *shmat(int shmid, const void *shmaddr, int shmflg);
```

| 参数 | 说明 |
|------|------|
| `shmid` | `shmget()` 返回的共享内存标识符 |
| `shmaddr` | 指定映射地址，通常设为 `NULL` 让系统自动选择 |
| `shmflg` | 标志位 |
| 返回值 | 成功返回映射后的地址，失败返回 `(void *)-1` |

**常用标志位**：

| 标志 | 说明 |
|------|------|
| `0` | 默认（可读写） |
| `SHM_RDONLY` | 只读模式 |
| `SHM_RND` | 地址向下取整到页边界 |

**示例**：

```c
// 映射共享内存（系统自动选择地址）
char *ptr = (char *)shmat(shmid, NULL, 0);
if (ptr == (void *)-1) {
    perror("shmat failed");
    exit(1);
}
```

### 3.3 shmdt() - 解除映射

```c
int shmdt(const void *shmaddr);
```

| 参数 | 说明 |
|------|------|
| `shmaddr` | `shmat()` 返回的地址 |
| 返回值 | 成功返回 0，失败返回 -1 |

> **注意**：`shmdt()` 只是解除当前进程与共享内存的映射关系，并不会删除共享内存。共享内存会一直存在直到被显式删除或系统重启。

### 3.4 shmctl() - 控制共享内存

```c
int shmctl(int shmid, int cmd, struct shmid_ds *buf);
```

| 参数 | 说明 |
|------|------|
| `shmid` | 共享内存标识符 |
| `cmd` | 控制命令 |
| `buf` | `shmid_ds` 结构体指针，用于获取/设置信息 |
| 返回值 | 成功返回 0（或其他值），失败返回 -1 |

**常用命令**：

| 命令 | 说明 |
|------|------|
| `IPC_STAT` | 获取共享内存状态信息到 `buf` |
| `IPC_SET` | 设置共享内存属性（从 `buf` 读取） |
| `IPC_RMID` | 标记删除共享内存（当连接数为 0 时真正删除） |

**删除共享内存**：

```c
// 删除共享内存
shmctl(shmid, IPC_RMID, NULL);
```

## 4. 代码示例

### 4.1 写入进程 (shm_write.c)

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <sys/ipc.h>
#include <sys/shm.h>

#define SHM_SIZE 1024

int main(void) {
    key_t key;
    int shmid;
    char *shmptr;

    // 1. 生成 key
    key = ftok(".", 'a');
    if (key == -1) {
        perror("ftok failed");
        exit(1);
    }

    // 2. 创建共享内存
    shmid = shmget(key, SHM_SIZE, IPC_CREAT | 0666);
    if (shmid == -1) {
        perror("shmget failed");
        exit(1);
    }
    printf("共享内存创建成功，shmid = %d\n", shmid);

    // 3. 映射共享内存
    shmptr = (char *)shmat(shmid, NULL, 0);
    if (shmptr == (void *)-1) {
        perror("shmat failed");
        exit(1);
    }

    // 4. 写入数据
    strcpy(shmptr, "Hello from writer process!");
    printf("数据已写入: %s\n", shmptr);

    // 5. 解除映射（保留共享内存供其他进程读取）
    shmdt(shmptr);

    return 0;
}
```

### 4.2 读取进程 (shm_read.c)

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/ipc.h>
#include <sys/shm.h>

#define SHM_SIZE 1024

int main(void) {
    key_t key;
    int shmid;
    char *shmptr;

    // 1. 生成相同的 key
    key = ftok(".", 'a');
    if (key == -1) {
        perror("ftok failed");
        exit(1);
    }

    // 2. 获取已存在的共享内存
    shmid = shmget(key, SHM_SIZE, 0666);
    if (shmid == -1) {
        perror("shmget failed");
        exit(1);
    }
    printf("获取共享内存成功，shmid = %d\n", shmid);

    // 3. 映射共享内存
    shmptr = (char *)shmat(shmid, NULL, 0);
    if (shmptr == (void *)-1) {
        perror("shmat failed");
        exit(1);
    }

    // 4. 读取数据
    printf("读取到的数据: %s\n", shmptr);

    // 5. 解除映射
    shmdt(shmptr);

    // 6. 删除共享内存
    shmctl(shmid, IPC_RMID, NULL);
    printf("共享内存已删除\n");

    return 0;
}
```

### 4.3 编译和运行

```bash
# 编译
gcc -o shm_write shm_write.c
gcc -o shm_read shm_read.c

# 运行（先写后读）
./shm_write
./shm_read
```

## 5. 查看和管理共享内存

### 5.1 命令行工具

```bash
# 查看所有 System V IPC 对象
ipcs

# 只查看共享内存
ipcs -m

# 删除指定的共享内存
ipcrm -m <shmid>

# 删除所有共享内存（谨慎使用）
ipcrm -a
```

### 5.2 ipcs 输出示例

```
------ Shared Memory Segments --------
key        shmid      owner      perms      bytes      nattch     status
0x61012345 32769      user       666        1024       0
```

| 字段 | 说明 |
|------|------|
| `key` | IPC 键值 |
| `shmid` | 共享内存标识符 |
| `owner` | 创建者 |
| `perms` | 权限 |
| `bytes` | 大小（字节） |
| `nattch` | 当前连接的进程数 |

## 6. 共享内存与同步

共享内存本身**不提供同步机制**，多个进程同时读写可能导致数据不一致。通常需要配合以下机制：

| 同步方式 | 说明 |
|----------|------|
| **[[11.5 信号量\|信号量]]** | System V 信号量或 POSIX 信号量 |
| **互斥锁** | pthread_mutex（需要 `PTHREAD_PROCESS_SHARED`） |
| **文件锁** | `flock()` 或 `fcntl()` |

详细的同步机制请参考 [[11.5 信号量]]。

---

## 7. 关键要点

1. **共享内存是最快的 IPC 方式**：数据直接在内存中共享，无需复制
2. **使用流程**：`ftok()` → `shmget()` → `shmat()` → 读写 → `shmdt()` → `shmctl(IPC_RMID)`
3. **需要同步机制**：必须配合[[11.5 信号量|信号量]]等避免竞态条件
4. **生命周期**：共享内存会持续存在直到被删除或系统重启
5. **key 的作用**：不同进程通过相同的 key 访问同一块共享内存
