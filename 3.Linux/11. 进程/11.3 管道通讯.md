# 11.3 管道通讯

## 1. 管道简介

管道是一种最基础的[[11.2 进程间通信IPC|进程间通信 (IPC)]]方式，用于在具有亲缘关系的进程（通常是父子进程）之间传递数据。

### 1.1 管道的特点

```
              管道
    ┌────────────────────────┐
    │  ⚙️ ──► ⚙️ ──► ⚙️ ──► │
    │                        │
    │◄── ◄── ◄── ◄── ◄── ◄── │
    └────────────────────────┘
     进程A                进程B
     (写入端)             (读取端)
```

- **半双工通信**：为了实现双向通信，需要创建两个管道（每个方向一个）
- **单向数据流**：数据从写端流向读端
- **字节流**：数据以字节为单位传输，没有消息边界

## 2. 匿名管道 (Anonymous Pipe)

### 2.1 基本概念

匿名管道用于有亲缘关系的进程之间通信，利用了 [[11.1 进程简介|fork()]] 创建子进程时会复制文件描述符的特性。

```c
#include <unistd.h>

int pipe(int fd[2]);
```

| 参数 | 说明 |
|------|------|
| `fd[0]` | 管道的**读取端**（read end） |
| `fd[1]` | 管道的**写入端**（write end） |
| 返回值 | 成功返回 0，失败返回 -1 |

### 2.2 工作原理

```
    父进程创建管道
         │
         ▼
    ┌─────────┐
    │ fd[0]=3 │  读取端
    │ fd[1]=4 │  写入端
    └─────────┘
         │
         │ fork()
         ▼
    ┌─────────────────────────────┐
    │                             │
    ▼                             ▼
 父进程                         子进程
┌─────────┐                   ┌─────────┐
│ fd[0]=3 │                   │ fd[0]=3 │
│ fd[1]=4 │                   │ fd[1]=4 │
└─────────┘                   └─────────┘
    │                             │
    │ 关闭 fd[0]                  │ 关闭 fd[1]
    ▼                             ▼
 父进程                         子进程
┌─────────┐                   ┌─────────┐
│ fd[1]=4 │ ──── 数据流 ────► │ fd[0]=3 │
│ (写入)  │                   │ (读取)  │
└─────────┘                   └─────────┘
```

### 2.3 代码示例

```c
#include <stdio.h>
#include <unistd.h>
#include <string.h>
#include <sys/wait.h>

#define BUF_SIZE 30

int main(void) {
    int fds[2];
    char str[] = "hello, I am sub process!";
    char buf[BUF_SIZE];
    pid_t pid;

    // 创建管道
    pipe(fds);
    pid = fork();

    if (pid == 0) {
        // 子进程：写入数据
        write(fds[1], str, sizeof(str));
    } else {
        // 父进程：读取数据
        read(fds[0], buf, BUF_SIZE);
        printf("父进程收到: %s\n", buf);
    }

    return 0;
}
```

### 2.4 双向管道通信

要实现双向通信，需要创建**两个管道**：

```c
#include <stdio.h>
#include <unistd.h>
#include <string.h>

#define BUF_SIZE 30

int main(void) {
    int fds1[2], fds2[2];
    char str1[] = "你好，我是子进程!";
    char str2[] = "你好，我是父进程!";
    char buf[BUF_SIZE];
    pid_t pid;

    // 创建两个管道
    pipe(fds1);  // 子进程 → 父进程
    pipe(fds2);  // 父进程 → 子进程

    pid = fork();

    if (pid == 0) {
        // 子进程
        write(fds1[1], str1, sizeof(str1));     // 向管道1写入
        read(fds2[0], buf, BUF_SIZE);           // 从管道2读取
        printf("子进程收到: %s\n", buf);
    } else {
        // 父进程
        read(fds1[0], buf, BUF_SIZE);           // 从管道1读取
        printf("父进程收到: %s\n", buf);
        write(fds2[1], str2, sizeof(str2));     // 向管道2写入
    }

    return 0;
}
```

**双向管道结构**：

```
          fds1[1]              fds1[0]
    子进程 ──────────► 管道1 ────────► 父进程

          fds2[0]              fds2[1]
    子进程 ◄──────────  管道2 ◄──────── 父进程
```

## 3. 命名管道 (Named Pipe / FIFO)

### 3.1 基本概念

命名管道（FIFO）克服了匿名管道的局限性，可以在**不相关的进程**之间进行通信。命名管道在文件系统中有一个对应的路径，进程可以通过打开这个路径来进行读写操作。

```c
#include <sys/types.h>
#include <sys/stat.h>

int mkfifo(const char *pathname, mode_t mode);
```

| 参数 | 说明 |
|------|------|
| `pathname` | FIFO 文件的路径名 |
| `mode` | 文件权限（如 `0666`） |
| 返回值 | 成功返回 0，失败返回 -1 |

### 3.2 创建 FIFO

```bash
# 命令行创建
mkfifo /tmp/myfifo

# 或使用 mknod 命令
mknod /tmp/myfifo p
```

### 3.3 代码示例

**写入端 (writer.c)**：

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/stat.h>

#define FIFO_PATH "/tmp/myfifo"

int main(void) {
    int fd;
    char buffer[100];

    // 创建 FIFO（如果不存在）
    mkfifo(FIFO_PATH, 0666);

    // 以写模式打开 FIFO
    fd = open(FIFO_PATH, O_WRONLY);

    printf("输入要发送的消息: ");
    fgets(buffer, sizeof(buffer), stdin);

    write(fd, buffer, sizeof(buffer));
    printf("消息已发送!\n");

    close(fd);
    return 0;
}
```

**读取端 (reader.c)**：

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

#define FIFO_PATH "/tmp/myfifo"

int main(void) {
    int fd;
    char buffer[100];

    // 以读模式打开 FIFO
    fd = open(FIFO_PATH, O_RDONLY);

    read(fd, buffer, sizeof(buffer));
    printf("收到消息: %s", buffer);

    close(fd);
    return 0;
}
```

## 4. 管道符号 `|`

在 Shell 中，管道符号 `|` 用于将一个命令的标准输出连接到另一个命令的标准输入：

```bash
# 将 ls 的输出传递给 grep
ls -la | grep ".txt"

# 多级管道
cat file.txt | grep "error" | wc -l
```

**工作原理**：Shell 会为每个 `|` 创建一个匿名管道，并通过 [[11.1 进程简介|fork()]] 创建子进程来执行各个命令。

---

## 5. 关键要点

1. **管道是半双工的**：数据只能单向流动
2. **匿名管道**只能在有亲缘关系的进程间使用
3. **命名管道（FIFO）**可以在任意进程间使用，有文件系统路径
4. **双向通信**需要创建两个管道
5. **管道符 `|`** 在 Shell 中实现命令间的数据传递

---

## 6. 匿名管道 vs 命名管道

| 特性 | 匿名管道 | 命名管道 (FIFO) |
|------|----------|-----------------|
| 进程关系 | 必须有亲缘关系（父子进程） | 任意进程 |
| 文件系统 | 不存在于文件系统 | 有文件路径 |
| 创建方式 | `pipe()` | `mkfifo()` |
| 持久性 | 随进程终止而销毁 | 可以持久存在 |
| 使用场景 | 简单的父子进程通信 | 不相关进程间通信 |
