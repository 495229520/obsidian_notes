# 11.5 信号量 (Semaphore)

## 1. 什么是信号量

**信号量 (Semaphore)** 是一种用于[[11.2 进程间通信IPC|进程间同步]]的机制。与[[11.4 共享内存|共享内存]]和[[11.6 消息队列|消息队列]]不同，信号量**不用于传递数据**，而是用于**控制对共享资源的访问**。

> 信号量本质上是一个**非负整数计数器**，用于协调多个进程对共享资源的访问，从而避免竞态条件 (Race Condition)。

### 1.1 信号量的核心操作

信号量支持两种原子操作：

| 操作 | 别名 | 描述 |
|------|------|------|
| **P 操作** | wait / down / 申请 | 如果信号量 > 0，则减 1；否则阻塞等待 |
| **V 操作** | signal / up / 释放 | 信号量加 1，唤醒等待的进程 |

```
P 操作 (申请资源):
    if (semaphore > 0) {
        semaphore--;
        // 继续执行
    } else {
        // 阻塞等待
    }

V 操作 (释放资源):
    semaphore++;
    // 唤醒等待队列中的进程
```

## 2. 信号量的类型

### 2.1 二进制信号量 (Binary Semaphore)

二进制信号量的值只能是 **0** 或 **1**，功能类似于**互斥锁 (Mutex)**。

| 值 | 含义 |
|----|------|
| 1 | 资源可用（未被占用） |
| 0 | 资源不可用（已被占用） |

**使用场景**：保护临界区，确保同一时刻只有一个进程访问共享资源。

```c
// 伪代码示例
sem_wait(&sem);      // P 操作：获取锁
// 临界区代码
sem_post(&sem);      // V 操作：释放锁
```

### 2.2 计数信号量 (Counting Semaphore)

计数信号量的值可以是**任意非负整数**，表示可用资源的数量。

| 值 | 含义 |
|----|------|
| n > 0 | 有 n 个资源可用 |
| 0 | 没有资源可用，新请求将阻塞 |

**使用场景**：控制对有限资源池的访问（如数据库连接池、缓冲区槽位）。

```c
// 生产者-消费者模型中的缓冲区管理
sem_t empty_slots;   // 空槽位数量
sem_t filled_slots;  // 已填充槽位数量

// 生产者
sem_wait(&empty_slots);   // 等待空槽位
// 生产数据
sem_post(&filled_slots);  // 增加已填充数量

// 消费者
sem_wait(&filled_slots);  // 等待已填充槽位
// 消费数据
sem_post(&empty_slots);   // 增加空槽位数量
```

### 2.3 互斥量 (Mutex) vs 二进制信号量

| 特性 | 互斥量 (Mutex) | 二进制信号量 |
|------|---------------|-------------|
| **所有权** | 有，只有加锁者能解锁 | 无，任何进程都能操作 |
| **用途** | 保护临界区 | 同步和互斥 |
| **递归** | 可以支持递归锁 | 通常不支持 |
| **优先级继承** | 通常支持 | 不支持 |

## 3. System V 信号量

System V 信号量是一组（**信号量集**）而不是单个信号量，可以在一次操作中对多个信号量进行原子操作。

### 3.1 semget() - 创建/获取信号量集

```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/sem.h>

int semget(key_t key, int nsems, int semflg);
```

| 参数 | 说明 |
|------|------|
| `key` | IPC 键值，与[[11.4 共享内存\|共享内存]]使用相同的 `ftok()` 生成 |
| `nsems` | 信号量集中信号量的数量 |
| `semflg` | 标志位（`IPC_CREAT`、权限等） |
| 返回值 | 成功返回信号量集标识符 (semid)，失败返回 -1 |

**示例**：

```c
// 创建包含 1 个信号量的信号量集
int semid = semget(key, 1, IPC_CREAT | 0666);

// 获取已存在的信号量集
int semid = semget(key, 0, 0);
```

### 3.2 semop() - 信号量操作

```c
int semop(int semid, struct sembuf *sops, size_t nsops);
```

| 参数 | 说明 |
|------|------|
| `semid` | 信号量集标识符 |
| `sops` | `sembuf` 结构体数组，描述要执行的操作 |
| `nsops` | 操作的数量 |
| 返回值 | 成功返回 0，失败返回 -1 |

**sembuf 结构体**：

```c
struct sembuf {
    unsigned short sem_num;  // 信号量在集合中的索引（从 0 开始）
    short          sem_op;   // 操作值
    short          sem_flg;  // 操作标志
};
```

| sem_op 值 | 操作类型 | 说明 |
|-----------|----------|------|
| `> 0` | V 操作 | 信号量值增加 sem_op |
| `< 0` | P 操作 | 信号量值减少 \|sem_op\|，若不够则阻塞 |
| `== 0` | 等待零 | 阻塞直到信号量值变为 0 |

**常用标志 (sem_flg)**：

| 标志 | 说明 |
|------|------|
| `0` | 默认行为（阻塞） |
| `IPC_NOWAIT` | 非阻塞操作，资源不足时立即返回错误 |
| `SEM_UNDO` | 进程退出时自动撤销操作（防止死锁） |

**封装 P/V 操作**：

```c
// P 操作（申请资源）
void sem_p(int semid, int sem_num) {
    struct sembuf op;
    op.sem_num = sem_num;
    op.sem_op = -1;          // P 操作：减 1
    op.sem_flg = SEM_UNDO;   // 进程退出时自动释放
    semop(semid, &op, 1);
}

// V 操作（释放资源）
void sem_v(int semid, int sem_num) {
    struct sembuf op;
    op.sem_num = sem_num;
    op.sem_op = 1;           // V 操作：加 1
    op.sem_flg = SEM_UNDO;
    semop(semid, &op, 1);
}
```

### 3.3 semctl() - 信号量控制

```c
int semctl(int semid, int semnum, int cmd, ...);
```

| 参数 | 说明 |
|------|------|
| `semid` | 信号量集标识符 |
| `semnum` | 信号量索引（对于某些命令） |
| `cmd` | 控制命令 |
| `...` | 可选参数（`union semun`） |

**常用命令**：

| 命令 | 说明 |
|------|------|
| `SETVAL` | 设置单个信号量的值 |
| `GETVAL` | 获取单个信号量的值 |
| `SETALL` | 设置所有信号量的值 |
| `GETALL` | 获取所有信号量的值 |
| `IPC_RMID` | 删除信号量集 |
| `IPC_STAT` | 获取信号量集状态 |

**union semun**（需要自己定义）：

```c
union semun {
    int              val;    // SETVAL 使用
    struct semid_ds *buf;    // IPC_STAT/IPC_SET 使用
    unsigned short  *array;  // SETALL/GETALL 使用
};
```

**初始化和删除示例**：

```c
// 初始化信号量值为 1
union semun arg;
arg.val = 1;
semctl(semid, 0, SETVAL, arg);

// 获取信号量值
int val = semctl(semid, 0, GETVAL);

// 删除信号量集
semctl(semid, 0, IPC_RMID);
```

## 4. 代码示例：生产者-消费者模型

以下是使用信号量实现的生产者-消费者模型，用于保护[[11.4 共享内存|共享内存]]的访问。

### 4.1 公共头文件 (common.h)

```c
#ifndef COMMON_H
#define COMMON_H

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/ipc.h>
#include <sys/shm.h>
#include <sys/sem.h>

#define SHM_SIZE 256

// 信号量操作封装
union semun {
    int val;
    struct semid_ds *buf;
    unsigned short *array;
};

// P 操作
void sem_p(int semid) {
    struct sembuf op = {0, -1, SEM_UNDO};
    if (semop(semid, &op, 1) == -1) {
        perror("sem_p failed");
        exit(1);
    }
}

// V 操作
void sem_v(int semid) {
    struct sembuf op = {0, 1, SEM_UNDO};
    if (semop(semid, &op, 1) == -1) {
        perror("sem_v failed");
        exit(1);
    }
}

#endif
```

### 4.2 生产者 (producer.c)

```c
#include "common.h"

int main(void) {
    key_t key = ftok(".", 'a');

    // 创建共享内存
    int shmid = shmget(key, SHM_SIZE, IPC_CREAT | 0666);
    char *shmptr = (char *)shmat(shmid, NULL, 0);

    // 创建信号量（初始值为 1，表示资源可用）
    int semid = semget(key, 1, IPC_CREAT | 0666);
    union semun arg;
    arg.val = 1;
    semctl(semid, 0, SETVAL, arg);

    printf("Producer: Waiting to write...\n");

    // P 操作：获取访问权
    sem_p(semid);

    // 写入数据到共享内存
    strcpy(shmptr, "Hello from producer!");
    printf("Producer: Data written: %s\n", shmptr);

    // 模拟处理时间
    sleep(3);

    // V 操作：释放访问权
    sem_v(semid);
    printf("Producer: Released lock\n");

    shmdt(shmptr);
    return 0;
}
```

### 4.3 消费者 (consumer.c)

```c
#include "common.h"

int main(void) {
    key_t key = ftok(".", 'a');

    // 获取共享内存
    int shmid = shmget(key, SHM_SIZE, 0666);
    if (shmid == -1) {
        perror("shmget failed");
        exit(1);
    }
    char *shmptr = (char *)shmat(shmid, NULL, 0);

    // 获取信号量
    int semid = semget(key, 1, 0666);
    if (semid == -1) {
        perror("semget failed");
        exit(1);
    }

    printf("Consumer: Waiting to read...\n");

    // P 操作：获取访问权
    sem_p(semid);

    // 读取数据
    printf("Consumer: Data read: %s\n", shmptr);

    // V 操作：释放访问权
    sem_v(semid);
    printf("Consumer: Released lock\n");

    // 清理资源
    shmdt(shmptr);
    shmctl(shmid, IPC_RMID, NULL);
    semctl(semid, 0, IPC_RMID);
    printf("Consumer: Resources cleaned up\n");

    return 0;
}
```

### 4.4 编译和运行

```bash
# 编译
gcc -o producer producer.c
gcc -o consumer consumer.c

# 在两个终端中分别运行
# 终端 1
./producer

# 终端 2（在 producer 运行期间启动）
./consumer
```

**预期输出**：

```
# 终端 1 (producer)
Producer: Waiting to write...
Producer: Data written: Hello from producer!
Producer: Released lock

# 终端 2 (consumer) - 会等待 producer 释放锁后才能读取
Consumer: Waiting to read...
Consumer: Data read: Hello from producer!
Consumer: Released lock
Consumer: Resources cleaned up
```

## 5. POSIX 信号量

除了 System V 信号量，还有更现代的 **POSIX 信号量**，使用更简单。

### 5.1 有名信号量 (Named Semaphore)

```c
#include <semaphore.h>
#include <fcntl.h>

// 创建/打开
sem_t *sem = sem_open("/mysem", O_CREAT, 0666, 1);

// P 操作
sem_wait(sem);

// V 操作
sem_post(sem);

// 关闭
sem_close(sem);

// 删除
sem_unlink("/mysem");
```

### 5.2 无名信号量 (Unnamed Semaphore)

```c
#include <semaphore.h>

sem_t sem;

// 初始化（第二个参数: 0=线程间, 非0=进程间）
sem_init(&sem, 0, 1);

// P 操作
sem_wait(&sem);

// V 操作
sem_post(&sem);

// 销毁
sem_destroy(&sem);
```

### 5.3 System V vs POSIX 信号量

| 特性 | System V | POSIX |
|------|----------|-------|
| **复杂度** | 复杂（信号量集） | 简单 |
| **可移植性** | 广泛支持 | 需要 POSIX 支持 |
| **接口** | semget/semop/semctl | sem_open/sem_wait/sem_post |
| **原子操作** | 支持多个信号量原子操作 | 单个信号量操作 |


## 7. 关键要点

1. **信号量用于同步，不传递数据**：与[[11.4 共享内存|共享内存]]配合使用
2. **P 操作申请资源，V 操作释放资源**：这两个操作都是原子的
3. **二进制信号量类似互斥锁**：值只能是 0 或 1
4. **计数信号量管理资源池**：值表示可用资源数量
5. **使用 `SEM_UNDO` 防止死锁**：进程异常退出时自动释放
6. **POSIX 信号量更简单**：推荐在新项目中使用
