# 11.8 基于多任务的多进程并发服务器

## 1. 并发服务器概述

在 [[7.7 listen和accept函数#完整示例：TCP 服务器|基本 TCP 服务器]] 中，服务器一次只能处理一个客户端连接。当一个客户端正在被服务时，其他客户端必须等待。这种**迭代式服务器**在实际应用中效率很低。

**并发服务器**可以同时处理多个客户端连接，常见的实现方式有：

| 方式 | 特点 |
|:---|:---|
| **多进程** | 每个客户端由独立子进程处理，进程间隔离性好 |
| **多线程** | 线程比进程轻量，共享地址空间，但需要同步 |
| **[[13.1 IO复用简介\|I/O 多路复用]]** | 单进程/线程处理多连接（select/poll/epoll） |

本节介绍基于 [[11.1 进程简介#4. 创建进程：fork() 函数|fork()]] 的多进程并发服务器。

## 2. 多进程服务器工作原理

### 2.1 基本流程

```
                           服务器
                              │
                        socket(), bind()
                        listen()
                              │
                              ▼
                    ┌─────────────────┐
                    │  accept() 阻塞  │◄──────────────────┐
                    └────────┬────────┘                   │
                             │ 有客户端连接                 │
                             ▼                            │
                    ┌─────────────────┐                   │
                    │     fork()      │                   │
                    └────────┬────────┘                   │
              ┌──────────────┴──────────────┐             │
              │                             │             │
         父进程                          子进程             │
              │                             │             │
     关闭 conn_fd                    关闭 listen_fd        │
              │                             │             │
              └─────────────────┐     处理客户端请求        │
                                │           │             │
                                │     close(conn_fd)      │
                                │     exit(0)             │
                                │                         │
                                └─────────────────────────┘
```

### 2.2 关键设计要点

1. **fork() 后文件描述符的继承**
   - 子进程会继承父进程的所有打开的文件描述符
   - 父子进程都持有 `listen_fd` 和 `conn_fd` 的引用

2. **关闭不需要的文件描述符**
   - **父进程**：关闭 `conn_fd`（已连接套接字），继续监听新连接
   - **子进程**：关闭 `listen_fd`（监听套接字），专注处理当前客户端

3. **避免[[11.7 僵尸进程|僵尸进程]]**
   - 子进程处理完毕退出后，父进程需要回收
   - 使用 `signal(SIGCHLD, handler)` 注册信号处理函数自动回收

## 3. 服务器端代码实现

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <signal.h>
#include <sys/socket.h>
#include <sys/wait.h>
#include <arpa/inet.h>

#define BUFFER_SIZE 1024

// SIGCHLD 信号处理函数，回收僵尸进程
void sigchld_handler(int signo) {
    int status;
    pid_t pid;
    // 循环回收所有已终止的子进程
    while ((pid = waitpid(-1, &status, WNOHANG)) > 0) {
        printf("子进程 %d 已回收\n", pid);
    }
}

int main(int argc, char *argv[]) {
    if (argc != 2) {
        printf("Usage: %s <port>\n", argv[0]);
        return -1;
    }

    // 注册 SIGCHLD 信号处理函数
    signal(SIGCHLD, sigchld_handler);

    // 1. 创建监听套接字
    int listen_fd = socket(AF_INET, SOCK_STREAM, 0);
    if (listen_fd < 0) {
        perror("socket");
        return -1;
    }

    // 设置地址复用
    int opt = 1;
    setsockopt(listen_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

    // 2. 绑定地址
    struct sockaddr_in server_addr;
    memset(&server_addr, 0, sizeof(server_addr));
    server_addr.sin_family = AF_INET;
    server_addr.sin_port = htons(atoi(argv[1]));
    server_addr.sin_addr.s_addr = htonl(INADDR_ANY);

    if (bind(listen_fd, (struct sockaddr*)&server_addr,
             sizeof(server_addr)) < 0) {
        perror("bind");
        return -1;
    }

    // 3. 开始监听
    if (listen(listen_fd, 5) < 0) {
        perror("listen");
        return -1;
    }

    printf("服务器启动，监听端口 %s...\n", argv[1]);

    // 4. 循环接受连接
    while (1) {
        struct sockaddr_in client_addr;
        socklen_t client_len = sizeof(client_addr);

        // accept 可能被 SIGCHLD 信号中断
        int conn_fd = accept(listen_fd,
                             (struct sockaddr*)&client_addr,
                             &client_len);
        if (conn_fd < 0) {
            // 被信号中断，继续等待
            continue;
        }

        printf("客户端已连接: %s:%d\n",
               inet_ntoa(client_addr.sin_addr),
               ntohs(client_addr.sin_port));

        // 5. 创建子进程处理客户端
        pid_t pid = fork();

        if (pid < 0) {
            perror("fork");
            close(conn_fd);
            continue;
        } else if (pid == 0) {
            // ===== 子进程 =====
            close(listen_fd);  // 子进程不需要监听套接字

            char buffer[BUFFER_SIZE];
            ssize_t n;

            // 循环读取客户端数据
            while ((n = read(conn_fd, buffer, BUFFER_SIZE - 1)) > 0) {
                buffer[n] = '\0';
                printf("[PID %d] 收到: %s", getpid(), buffer);

                // 回显数据给客户端
                write(conn_fd, buffer, n);
            }

            printf("[PID %d] 客户端断开连接\n", getpid());
            close(conn_fd);
            exit(0);  // 子进程退出
        } else {
            // ===== 父进程 =====
            close(conn_fd);  // 父进程不需要已连接套接字
            // 继续循环接受新连接
        }
    }

    close(listen_fd);
    return 0;
}
```

## 4. 客户端代码实现

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <arpa/inet.h>

#define BUFFER_SIZE 1024

int main(int argc, char *argv[]) {
    if (argc != 3) {
        printf("Usage: %s <IP> <port>\n", argv[0]);
        return -1;
    }

    // 1. 创建套接字
    int sock_fd = socket(AF_INET, SOCK_STREAM, 0);
    if (sock_fd < 0) {
        perror("socket");
        return -1;
    }

    // 2. 设置服务器地址
    struct sockaddr_in server_addr;
    memset(&server_addr, 0, sizeof(server_addr));
    server_addr.sin_family = AF_INET;
    server_addr.sin_port = htons(atoi(argv[2]));

    if (inet_pton(AF_INET, argv[1], &server_addr.sin_addr) <= 0) {
        perror("inet_pton");
        return -1;
    }

    // 3. 连接服务器
    if (connect(sock_fd, (struct sockaddr*)&server_addr,
                sizeof(server_addr)) < 0) {
        perror("connect");
        return -1;
    }

    printf("已连接到服务器 %s:%s\n", argv[1], argv[2]);

    // 4. 与服务器通信
    char send_buf[BUFFER_SIZE];
    char recv_buf[BUFFER_SIZE];

    while (1) {
        printf("输入消息 (quit 退出): ");
        fgets(send_buf, BUFFER_SIZE, stdin);

        if (strncmp(send_buf, "quit", 4) == 0) {
            break;
        }

        // 发送数据
        write(sock_fd, send_buf, strlen(send_buf));

        // 接收回显
        ssize_t n = read(sock_fd, recv_buf, BUFFER_SIZE - 1);
        if (n > 0) {
            recv_buf[n] = '\0';
            printf("服务器回显: %s", recv_buf);
        } else if (n == 0) {
            printf("服务器断开连接\n");
            break;
        }
    }

    close(sock_fd);
    return 0;
}
```

## 5. 代码注释与分析

### 5.1 为什么要关闭不需要的文件描述符

```
fork() 之前:
┌─────────────────────────────────┐
│         父进程                   │
│  listen_fd ──→ 监听套接字        │
│  conn_fd   ──→ 已连接套接字      │
└─────────────────────────────────┘

fork() 之后:
┌─────────────────────────────────┐   ┌─────────────────────────────────┐
│         父进程                   │   │         子进程                   │
│  listen_fd ──→ 监听套接字 ←───────────── listen_fd                    │
│  conn_fd   ──→ 已连接套接字 ←───────────── conn_fd                     │
└─────────────────────────────────┘   └─────────────────────────────────┘
                                       (继承了父进程的文件描述符)

正确做法:
┌─────────────────────────────────┐   ┌─────────────────────────────────┐
│         父进程                   │   │         子进程                   │
│  listen_fd ──→ 监听套接字        │   │  listen_fd ──→ (已关闭)         │
│  conn_fd   ──→ (已关闭)         │   │  conn_fd   ──→ 已连接套接字      │
└─────────────────────────────────┘   └─────────────────────────────────┘
```

- 父进程关闭 `conn_fd`：避免资源泄漏，该连接由子进程负责
- 子进程关闭 `listen_fd`：子进程不需要接受新连接
- 文件描述符有**引用计数**，只有所有引用都关闭后，资源才会释放

### 5.2 僵尸进程处理

子进程退出后，如果父进程不调用 [[11.7 僵尸进程#2. 销毁僵尸进程：wait() 函数|wait()]] 回收，子进程会变成僵尸进程。

本例使用 [[11.7 僵尸进程#4. 信号处理方式：signal() 函数|signal()]] 注册 `SIGCHLD` 信号处理函数：

```c
signal(SIGCHLD, sigchld_handler);
```

当子进程终止时，内核自动向父进程发送 `SIGCHLD` 信号，触发处理函数调用 `waitpid()` 回收。

## 6. 多进程服务器的优缺点

### 6.1 优点

| 优点 | 说明 |
|:---|:---|
| **隔离性好** | 每个客户端由独立进程处理，一个进程崩溃不影响其他 |
| **编程简单** | 不需要考虑线程同步问题 |
| **稳定性高** | 进程间内存独立，不会互相干扰 |

### 6.2 缺点

| 缺点 | 说明 |
|:---|:---|
| **资源开销大** | 每个进程占用独立的内存空间 |
| **创建成本高** | `fork()` 需要复制进程地址空间 |
| **进程间通信复杂** | 如果需要共享数据，必须使用 [[11.2 进程间通信IPC\|IPC 机制]] |
| **扩展性有限** | 大量并发时，进程数量会成为瓶颈 |

### 6.3 适用场景

- 连接数量不多（几十到几百）
- 每个连接需要大量计算或 I/O 操作
- 需要高度隔离和稳定性的场景

---

## 7. 关键要点

1. **多进程服务器流程**：`accept()` 后 `fork()`，子进程处理客户端
2. **文件描述符管理**：父子进程各自关闭不需要的文件描述符
3. **僵尸进程处理**：使用 `signal(SIGCHLD, handler)` 自动回收
4. **优点**：隔离性好、稳定、编程简单
5. **缺点**：资源开销大、扩展性有限
6. **替代方案**：多线程、I/O 多路复用（适合高并发场景）
