# 套接字选项

## 概述

套接字选项用于获取和设置 [[7.6 套接字|套接字]] 的各种属性，包括超时时间、缓冲区大小、地址复用等。通过合理配置套接字选项，可以优化网络通信的性能和行为。

![[29a32e4b-9989-4a83-b179-b1a024fcd483.png]]

---

## 常用套接字选项

### SOL_SOCKET 级别选项

| 选项 | 说明 | 数据类型 | 描述 |
|:---|:---|:---|:---|
| `SO_REUSEADDR` | 地址复用 | int | 允许重用本地地址，解决 [[10.2 TIME_WAIT状态\|TIME_WAIT]] 导致的"Address already in use"问题 |
| `SO_BROADCAST` | 广播 | int | 允许发送广播数据报 |
| `SO_SNDBUF` | 发送缓冲区 | int | 设置发送缓冲区大小 |
| `SO_RCVBUF` | 接收缓冲区 | int | 设置接收缓冲区大小 |
| `SO_KEEPALIVE` | 保活机制 | int | 启用 TCP 保活机制，定期检测连接是否存活 |
| `SO_DONTROUTE` | 不路由 | int | 绕过路由表直接发送 |
| `SO_LINGER` | 延迟关闭 | struct linger | 控制 close() 的行为 |
| `SO_OOBINLINE` | 带外数据内联 | int | 将带外数据放入普通数据流 |
| `SO_SNDTIMEO` | 发送超时 | struct timeval | 设置发送操作的超时时间 |
| `SO_RCVTIMEO` | 接收超时 | struct timeval | 设置接收操作的超时时间 |

### IPPROTO_IP 级别选项

| 选项 | 说明 | 数据类型 |
|:---|:---|:---|
| `IP_MULTICAST_TTL` | 多播TTL | int |
| `IP_MULTICAST_IF` | 多播接口 | struct in_addr |
| `IP_ADD_MEMBERSHIP` | 加入多播组 | struct ip_mreq |
| `IP_DROP_MEMBERSHIP` | 离开多播组 | struct ip_mreq |

### IPPROTO_TCP 级别选项

| 选项 | 说明 | 数据类型 |
|:---|:---|:---|
| `TCP_KEEPALIVE` | TCP 保活 | int |
| `TCP_NODELAY` | 禁用 [[10.3 Nagle算法\|Nagle 算法]] | int |
| `TCP_MAXSEG` | TCP 最大段大小 | int |

---

## 核心选项详解

### SO_SNDBUF / SO_RCVBUF

设置 [[7.6 套接字|套接字]] 的发送和接收缓冲区大小。

**注意事项**：
- TCP 中，当设置缓冲区大小时，内核会将其翻倍（用于流量控制）
- 缓冲区大小会影响 TCP 的滑动窗口大小
- 设置过小可能导致数据传输效率降低

### SO_REUSEADDR

允许地址复用，主要用于解决服务器重启时的 "Address already in use" 错误。

**应用场景**：
- 服务器程序重启时，端口可能处于 [[10.2 TIME_WAIT状态|TIME_WAIT]] 状态
- 设置此选项后，即使端口处于 TIME_WAIT 状态也可以绑定

详细用法参见 [[10.2 TIME_WAIT状态#SO_REUSEADDR 解决方案]]。

### SO_BROADCAST

允许发送广播数据报，仅对 [[9.1 UDP原理|UDP]] 套接字有效。

**注意**：只能用于 `SOCK_STREAM` 或 `SOCK_DGRAM` 类型的套接字。

### SO_TYPE

获取套接字类型，返回值：
- `SOCK_STREAM`：流式套接字（[[7.5 TCP协议基础|TCP]]）
- `SOCK_DGRAM`：数据报套接字（[[9.1 UDP原理|UDP]]）

## getsockopt 函数

`getsockopt()` 用于获取套接字选项的当前值。

### 函数原型

```c
#include <sys/socket.h>

int getsockopt(int sock, int level, int optname, void *optval, socklen_t *optlen);
```

### 参数说明

| 参数 | 说明 |
|:---|:---|
| `sock` | 套接字文件描述符，由 [[7.6 套接字#socket 函数\|socket()]] 函数返回 |
| `level` | 选项所在的协议层 |
| `optname` | 要获取的选项名称 |
| `optval` | 输出参数，保存获取到的选项值 |
| `optlen` | 输入/输出参数，指定 optval 缓冲区的大小，返回时包含实际数据大小 |

### level 参数取值

| 值 | 说明 |
|:---|:---|
| `SOL_SOCKET` | 套接字级别选项 |
| `IPPROTO_IP` | IP 协议级别选项 |
| `IPPROTO_TCP` | TCP 协议级别选项 |

### 返回值

- **成功**：返回 0
- **失败**：返回 -1，并设置 `errno`

### 使用示例

```c
#include <stdio.h>
#include <sys/socket.h>

int main() {
    int sock = socket(PF_INET, SOCK_STREAM, 0);

    // 获取套接字类型
    int sock_type;
    socklen_t optlen = sizeof(sock_type);

    if (getsockopt(sock, SOL_SOCKET, SO_TYPE, &sock_type, &optlen) == 0) {
        printf("套接字类型: %d\n", sock_type);  // SOCK_STREAM = 1
    }

    // 获取发送缓冲区大小
    int sndbuf;
    optlen = sizeof(sndbuf);

    if (getsockopt(sock, SOL_SOCKET, SO_SNDBUF, &sndbuf, &optlen) == 0) {
        printf("发送缓冲区大小: %d 字节\n", sndbuf);
    }

    // 获取接收缓冲区大小
    int rcvbuf;
    optlen = sizeof(rcvbuf);

    if (getsockopt(sock, SOL_SOCKET, SO_RCVBUF, &rcvbuf, &optlen) == 0) {
        printf("接收缓冲区大小: %d 字节\n", rcvbuf);
    }

    close(sock);
    return 0;
}
```

## setsockopt 函数

`setsockopt()` 用于设置套接字选项的值。

### 函数原型

```c
#include <sys/socket.h>

int setsockopt(int sock, int level, int optname, const void *optval, socklen_t optlen);
```

### 参数说明

| 参数 | 说明 |
|:---|:---|
| `sock` | 套接字文件描述符 |
| `level` | 选项所在的协议层（SOL_SOCKET、IPPROTO_IP、IPPROTO_TCP 等） |
| `optname` | 要设置的选项名称 |
| `optval` | 输入参数，指向要设置的选项值 |
| `optlen` | optval 指向数据的大小 |

### 返回值

- **成功**：返回 0
- **失败**：返回 -1，并设置 `errno`

### 使用示例

```c
#include <stdio.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <netinet/tcp.h>

int main() {
    int sock = socket(PF_INET, SOCK_STREAM, 0);

    // 设置地址复用（解决 TIME_WAIT 问题）
    int optval = 1;
    if (setsockopt(sock, SOL_SOCKET, SO_REUSEADDR, &optval, sizeof(optval)) < 0) {
        perror("setsockopt SO_REUSEADDR failed");
    }

    // 设置发送缓冲区大小
    int sndbuf = 65536;  // 64KB
    if (setsockopt(sock, SOL_SOCKET, SO_SNDBUF, &sndbuf, sizeof(sndbuf)) < 0) {
        perror("setsockopt SO_SNDBUF failed");
    }

    // 禁用 Nagle 算法
    int nodelay = 1;
    if (setsockopt(sock, IPPROTO_TCP, TCP_NODELAY, &nodelay, sizeof(nodelay)) < 0) {
        perror("setsockopt TCP_NODELAY failed");
    }

    // 设置接收超时（5秒）
    struct timeval timeout;
    timeout.tv_sec = 5;
    timeout.tv_usec = 0;
    if (setsockopt(sock, SOL_SOCKET, SO_RCVTIMEO, &timeout, sizeof(timeout)) < 0) {
        perror("setsockopt SO_RCVTIMEO failed");
    }

    close(sock);
    return 0;
}
```

## 常见应用场景

### 1. 解决服务器重启时的端口占用问题

```c
int sock = socket(PF_INET, SOCK_STREAM, 0);

// 在 bind() 之前设置 SO_REUSEADDR
int optval = 1;
setsockopt(sock, SOL_SOCKET, SO_REUSEADDR, &optval, sizeof(optval));

// 然后再 bind()
bind(sock, (struct sockaddr*)&addr, sizeof(addr));
```

详见 [[10.2 TIME_WAIT状态]]。

### 2. 禁用 Nagle 算法以降低延迟

```c
int nodelay = 1;
setsockopt(sock, IPPROTO_TCP, TCP_NODELAY, &nodelay, sizeof(nodelay));
```

详见 [[10.3 Nagle算法]]。

### 3. 设置超时避免阻塞

```c
struct timeval timeout;
timeout.tv_sec = 10;   // 10秒超时
timeout.tv_usec = 0;

// 设置接收超时
setsockopt(sock, SOL_SOCKET, SO_RCVTIMEO, &timeout, sizeof(timeout));

// 设置发送超时
setsockopt(sock, SOL_SOCKET, SO_SNDTIMEO, &timeout, sizeof(timeout));
```

---

## 关键要点

1. **getsockopt**：获取套接字选项值，`optlen` 是输入/输出参数
2. **setsockopt**：设置套接字选项值，需在 [[7.6 套接字#bind 函数|bind()]] 之前调用某些选项
3. **SO_REUSEADDR**：解决 [[10.2 TIME_WAIT状态|TIME_WAIT]] 导致的端口占用问题
4. **TCP_NODELAY**：禁用 [[10.3 Nagle算法|Nagle 算法]]，降低小数据包的传输延迟
5. **缓冲区设置**：影响 [[7.5 TCP协议基础|TCP]] 滑动窗口和传输效率
