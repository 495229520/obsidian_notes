# 215. 数组中的第K个最大元素

## 题目描述

给定整数数组 `nums` 和整数 `k`，返回数组中第 `k` 个最大的元素。

注意：题目要找的是**排序后的第 `k` 个最大元素**，不是第 `k` 个不同的元素。也就是说，重复数字也要正常参与排名。

### 示例

```text
输入：nums = [3,2,1,5,6,4], k = 2
输出：5
```

```text
输入：nums = [3,2,3,1,2,4,5,5,6], k = 4
输出：4
```

---

## 核心转换

如果把数组从小到大排序：

```text
第 1 大  -> 下标 n - 1
第 2 大  -> 下标 n - 2
第 k 大  -> 下标 n - k
```

所以本题的关键下标是：

```cpp
int target = nums.size() - k;
```

只要能让 `nums[target]` 变成升序排序后应该位于这个位置的元素，就能返回答案。

---

## 写法一：nth_element

### 核心思想

`nth_element(first, nth, last)` 是 C++ STL 里的部分排序算法。

它不会把整个区间完全排好序，而是只保证三件事：

- `nth` 位置上的元素，等于整个区间完全排序后应该放在这里的元素
- `nth` 左边的元素都不大于 `*nth`
- `nth` 右边的元素都不小于 `*nth`

也就是说，`nth_element` 做的是“快速定位某个排名”，而不是完整排序。

![[5_1_215_1.svg|760]]

### 代码实现

```cpp
class Solution {
public:
    int findKthLargest(vector<int>& nums, int k) {
        int target = nums.size() - k;
        nth_element(nums.begin(), nums.begin() + target, nums.end());
        return nums[target];
    }
};
```

### 代码分析

- `target = nums.size() - k`：把“第 `k` 大”转换成“升序下标 `target`”
- `nums.begin() + target`：告诉 `nth_element` 要定位哪个位置
- `return nums[target]`：定位完成后，这个位置就是答案

> [!warning]
> `nth_element` 会改变 `nums` 中元素的排列顺序。如果后续还需要原数组顺序，需要先拷贝一份再操作。

### 为什么比排序更合适

完整排序会把所有元素都排好：

```cpp
sort(nums.begin(), nums.end());
return nums[nums.size() - k];
```

但本题只需要一个位置的元素，不需要整个数组有序。

因此 `nth_element` 能省掉很多不必要的排序工作：

| 方法 | 时间复杂度 | 是否完整排序 |
|------|------------|--------------|
| `sort` | `O(n log n)` | 是 |
| `nth_element` | 平均 `O(n)` | 否 |

## 写法二：小根堆

### 核心思想

用一个大小最多为 `k` 的小根堆，始终保存当前见过的**最大的 `k` 个数**。

小根堆的特点是：堆顶是堆中最小的元素。

当堆里保存的是最大的 `k` 个数时：

- 堆顶就是这 `k` 个数里最小的
- 也就是整个数组里的第 `k` 大元素

![[5_1_215_2.svg|760]]

### 处理流程

遍历 `nums` 中的每个元素 `x`：

1. 把 `x` 放入小根堆
2. 如果堆大小超过 `k`，弹出堆顶
3. 遍历结束后，堆中剩下的就是最大的 `k` 个数
4. 堆顶就是第 `k` 大元素

### 代码实现

```cpp
class Solution {
public:
    int findKthLargest(vector<int>& nums, int k) {
        priority_queue<int, vector<int>, greater<int>> heap;

        for (int x : nums) {
            heap.push(x);

            if (heap.size() > k) {
                heap.pop();
            }
        }

        return heap.top();
    }
};
```

### 代码分析

- `priority_queue<int>` 默认是大根堆
- `priority_queue<int, vector<int>, greater<int>>` 表示小根堆
- `heap.size() > k` 时弹出堆顶，相当于把当前最大的 `k + 1` 个候选里最小的那个删掉
- 最后 `heap.top()` 就是最大的 `k` 个数中最小的那个，即第 `k` 大

> [!tip]
> 小根堆写法不需要修改原数组，而且适合处理数据流：数字一个个到来时，也可以实时维护第 `k` 大。

---

## 两种方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 是否修改原数组 | 适合场景 |
|------|------------|------------|----------------|----------|
| `nth_element` | 平均 `O(n)` | `O(1)` | 会修改 | 只处理一个数组，追求速度 |
| 小根堆 | `O(n log k)` | `O(k)` | 不修改 | 数据流、不能改变原数组、`k` 较小 |

---

## 易错点

1. 第 `k` 大对应升序下标 `nums.size() - k`，不是 `k - 1`
2. `nth_element` 只保证目标位置正确，不保证左右两边内部有序
3. `priority_queue` 默认是大根堆，小根堆需要写 `greater<int>`
4. 小根堆大小超过 `k` 时要立刻 `pop`，否则堆顶不一定是第 `k` 大
5. 题目不是找第 `k` 个不同元素，重复值要参与排名

---

## 复杂度

### nth_element

- 时间复杂度：平均 `O(n)`
- 空间复杂度：`O(1)`

### 小根堆

- 时间复杂度：`O(n log k)`
- 空间复杂度：`O(k)`
