# 25. K 个一组翻转链表

## 题目简介

给定一个单链表的头节点 `head`，每 `k` 个节点作为一组进行翻转，并返回修改后的链表。

如果最后剩余的节点不足 `k` 个，就保持原来的顺序。

要求：

- 必须真实改变节点之间的 `next` 指针
- 不能只交换节点内部的 `val`

示例：

```text
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]
```

```text
输入：head = [1,2,3,4,5], k = 3
输出：[3,2,1,4,5]
```

---

## 核心思路

这题可以看成是 [[206. 反转链表]] 的进阶版：

- 普通反转链表：把整条链表翻转
- 本题：每次只翻转长度为 `k` 的一小段

关键不是“怎么翻转”，而是每翻转完一组后，要把这一组重新接回原链表。

因此需要维护三段关系：

```text
已处理部分 -> 当前待翻转组 -> 后续未处理部分
```

为了统一处理头节点所在的第一组，通常使用虚拟头节点 `dummy`：

```cpp
ListNode dummy(0);
dummy.next = head;
```

这样即使第一组翻转后头节点发生变化，也可以通过 `dummy.next` 返回最终答案。

读图时重点看四个边界指针：

- `groupPrev`：站在当前组前面，负责把上一段接到翻转后的新组头
- `kth`：当前组的最后一个节点，翻转后会成为这一组的新头
- `groupNext`：当前组后面的第一个节点，既是翻转循环的停止边界，也是翻转后组尾要接回的位置
- `oldGroupHead`：翻转前的组头，翻转后会成为组尾，也是下一轮的 `groupPrev`

以第一张 `k = 2` 图为例，第 3 步最关键：

```cpp
ListNode* prev = groupNext;
ListNode* curr = groupPrev->next;
```

`prev` 不是初始化为 `nullptr`，而是初始化为 `groupNext`。这样翻转组内节点时，原来的组头 `1` 最终会指向 `3`，也就是后续链表入口。

第 6 步完成两件事：

```cpp
groupPrev->next = kth;
groupPrev = oldGroupHead;
```

第一句把上一段接到翻转后的新组头 `2`，第二句把 `groupPrev` 移动到当前组尾 `1`，下一轮就可以继续处理 `[3,4]`。

## 指针含义

代码中的几个核心指针：

| 指针 | 含义 |
|------|------|
| `dummy` | 虚拟头节点，方便处理第一组翻转 |
| `groupPrev` | 当前组前一个节点，也就是已处理部分的尾节点 |
| `kth` | 当前组的第 `k` 个节点 |
| `groupNext` | 当前组后面的第一个节点 |
| `prev` | 翻转过程中当前节点应该指向的前驱 |
| `curr` | 当前正在翻转的节点 |
| `oldGroupHead` | 翻转前的组头，翻转后会变成组尾 |

每一轮循环开始时，结构可以理解为：

```text
groupPrev -> 第1个节点 -> 第2个节点 -> ... -> 第k个节点 -> groupNext
```

翻转完成后应变成：

```text
groupPrev -> 第k个节点 -> ... -> 第2个节点 -> 第1个节点 -> groupNext
```

---

## 每轮处理流程

### 1. 找到当前组的第 k 个节点

```cpp
ListNode* kth = groupPrev;

for (int i = 0; i < k; i++) {
    kth = kth->next;

    if (kth == nullptr) {
        return dummy.next;
    }
}
```

这里从 `groupPrev` 往后走 `k` 步。

如果中途遇到 `nullptr`，说明剩余节点不足 `k` 个，不能翻转，直接返回当前链表。

### 2. 记录当前组后面的节点

```cpp
ListNode* groupNext = kth->next;
```

`groupNext` 是当前组之后的第一个节点。

它有两个作用：

- 作为本组翻转循环的结束边界
- 作为翻转后本组尾节点要连接的后续节点

### 3. 翻转当前 k 个节点

```cpp
ListNode* prev = groupNext;
ListNode* curr = groupPrev->next;

while (curr != groupNext) {
    ListNode* temp = curr->next;
    curr->next = prev;
    prev = curr;
    curr = temp;
}
```

这里和普通链表反转很像，但有一个关键细节：

```cpp
ListNode* prev = groupNext;
```

普通反转链表通常让 `prev = nullptr`，表示翻转后的尾节点指向空。

但本题只是翻转一组，翻转后的组尾应该接回 `groupNext`，所以一开始就让 `prev` 指向 `groupNext`。

这样当原组头变成新组尾时，它会自然指向后续未处理部分。

### 4. 把翻转后的组接回链表

翻转前：

```text
groupPrev -> oldGroupHead -> ... -> kth -> groupNext
```

翻转后，`kth` 会变成这一组的新头，`oldGroupHead` 会变成这一组的新尾。

```cpp
ListNode* oldGroupHead = groupPrev->next;

groupPrev->next = kth;
groupPrev = oldGroupHead;
```

含义：

- `oldGroupHead`：先保存旧组头，因为它翻转后会成为组尾
- `groupPrev->next = kth`：把上一段接到翻转后的新组头
- `groupPrev = oldGroupHead`：移动到当前组尾，准备处理下一组

---

## 示例推演

以链表 `1 -> 2 -> 3 -> 4 -> 5`，`k = 2` 为例。

初始：

```text
dummy -> 1 -> 2 -> 3 -> 4 -> 5
groupPrev = dummy
```

第一组：

```text
groupPrev = dummy
kth = 2
groupNext = 3
```

翻转 `1 -> 2` 后：

```text
dummy -> 2 -> 1 -> 3 -> 4 -> 5
groupPrev = 1
```

第二组：

```text
groupPrev = 1
kth = 4
groupNext = 5
```

翻转 `3 -> 4` 后：

```text
dummy -> 2 -> 1 -> 4 -> 3 -> 5
groupPrev = 3
```

最后剩下节点 `5` 不足 `2` 个，直接返回：

```text
2 -> 1 -> 4 -> 3 -> 5
```

## 代码实现

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* reverseKGroup(ListNode* head, int k) {
        if (head == nullptr || head->next == nullptr) {
            return head;
        }

        ListNode dummy(0);
        dummy.next = head;

        ListNode* groupPrev = &dummy;

        while (true) {
            ListNode* kth = groupPrev;

            for (int i = 0; i < k; i++) {
                kth = kth->next;

                if (kth == nullptr) {
                    return dummy.next;
                }
            }

            ListNode* groupNext = kth->next;

            ListNode* prev = groupNext;
            ListNode* curr = groupPrev->next;

            while (curr != groupNext) {
                ListNode* temp = curr->next;
                curr->next = prev;
                prev = curr;
                curr = temp;
            }

            ListNode* oldGroupHead = groupPrev->next;

            groupPrev->next = kth;
            groupPrev = oldGroupHead;
        }
    }
};
```

---

## 为什么 `groupPrev->next = kth` 是安全的

在翻转之前，`kth` 是当前组的最后一个节点。

在翻转之后，当前组的指针方向全部反过来，所以 `kth` 会变成当前组的新头节点。

例如：

```text
翻转前：
groupPrev -> 1 -> 2 -> groupNext
              ^         ^
              |         |
        oldGroupHead   kth
```

翻转后：

```text
groupPrev    2 -> 1 -> groupNext
             ^
             |
            kth
```

所以只需要：

```cpp
groupPrev->next = kth;
```

就能把已处理部分接到翻转后的新头。

---

## 易错点

1. 找 `kth` 时要从 `groupPrev` 开始走 `k` 步，而不是从 `groupPrev->next` 开始走 `k` 步。
2. 如果不足 `k` 个节点，必须直接返回，不能翻转剩余节点。
3. `groupNext` 必须在翻转前保存，否则翻转后会丢失后续链表的入口。
4. 翻转本组时，`prev` 应该初始化为 `groupNext`，不是 `nullptr`。
5. 翻转完成后，要先保存 `oldGroupHead`，再修改 `groupPrev->next`。
6. 下一轮的 `groupPrev` 应该移动到当前组翻转后的尾节点，也就是 `oldGroupHead`。

---

## 复杂度

每个节点只会被访问和修改常数次。

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

---

## 总结

本题的核心是把链表分成一组一组处理：

```text
找够 k 个节点 -> 保存 groupNext -> 局部翻转 -> 接回原链表 -> 移动 groupPrev
```

只要记住：

- `groupPrev` 永远站在当前组前面
- `kth` 翻转后会变成当前组新头
- `oldGroupHead` 翻转后会变成当前组新尾
- `groupNext` 是后续链表入口，也是本组翻转的结束边界

这道题就能稳定写出来。
