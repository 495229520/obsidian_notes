# 53. 最大子数组和

## 动态规划简介

本题是线性动态规划的经典入门题。它的难点不在代码，而在状态定义：我们不是直接问“整个数组的最大子数组和是多少”，而是先问一个更容易递推的问题：

```text
以当前位置 i 结尾的最大连续子数组和是多少？
```

这个状态天然带着“连续”限制。因为既然子数组必须以 `i` 结尾，那么到达 `nums[i]` 时只有两种选择：从 `nums[i]` 重新开始，或者把 `nums[i]` 接到前一个位置结尾的最优连续段后面。

### 核心思想

如果前面的连续和会拖累当前元素，就丢掉前面，从当前元素重新开始；否则继续接上当前元素。

> [!tip]
> 当题目要求数组中的**连续子数组**，并且目标是最大值、最小值、最优和这类结果时，可以优先考虑“以当前位置结尾”的线性 DP 状态。

### 基本模版

```cpp
int cur = nums[0];
int ans = nums[0];

for (int i = 1; i < nums.size(); ++i) {
    cur = max(nums[i], cur + nums[i]);
    ans = max(ans, cur);
}
```

如果写成完整 DP 数组，就是：

```cpp
dp[i] = max(nums[i], dp[i - 1] + nums[i]);
```

其中 `dp[i]` 表示“必须以 `i` 结尾的最大连续子数组和”。你的代码把 `dp[i - 1]` 压缩成了变量 `cur`。

## 题目描述

给定一个整数数组 `nums`，找出一个和最大的连续子数组，并返回这个最大和。

子数组必须是连续的一段，并且至少包含一个元素。

### 示例

```text
输入：nums = [-2,1,-3,4,-1,2,1,-5,4]
输出：6
解释：连续子数组 [4,-1,2,1] 的和最大，为 6。
```

```text
输入：nums = [1]
输出：1
```

```text
输入：nums = [5,4,-1,7,8]
输出：23
```

> [!important]
> 初始化时要用 `nums[0]`，不能用 `0`。因为数组可能全是负数，例如 `[-3,-2,-5]` 的答案应该是 `-2`，不是 `0`。

## 解题思路

这道题的关键是区分两个变量：

- `cur`：必须以当前位置 `i` 结尾的最大连续子数组和
- `ans`：遍历到当前位置为止，见过的最大连续子数组和

每走到一个新元素 `nums[i]`，都做一次选择：

```cpp
cur = max(nums[i], cur + nums[i]);
```

这行代码的含义是：

- `nums[i]`：前面那段不要了，从当前元素重新开始
- `cur + nums[i]`：保留前面那段，把当前元素接上去

然后用新的 `cur` 更新全局答案：

```cpp
ans = max(ans, cur);
```

### 步骤分解

1. 用第一个元素初始化 `cur` 和 `ans`。
2. 从第二个元素开始遍历数组。
3. 对每个位置判断：重新开始一段，还是接到前一段后面。
4. 每次得到新的 `cur` 后，用它更新 `ans`。
5. 遍历结束后，`ans` 就是最大连续子数组和。

## 示例推演

以 `nums = [-2,1,-3,4,-1,2,1,-5,4]` 为例：

| i | nums[i] | cur 的选择 | cur | ans |
|---|---------|------------|-----|-----|
| 0 | -2 | 初始化 | -2 | -2 |
| 1 | 1 | `max(1, -2 + 1)` | 1 | 1 |
| 2 | -3 | `max(-3, 1 - 3)` | -2 | 1 |
| 3 | 4 | `max(4, -2 + 4)` | 4 | 4 |
| 4 | -1 | `max(-1, 4 - 1)` | 3 | 4 |
| 5 | 2 | `max(2, 3 + 2)` | 5 | 5 |
| 6 | 1 | `max(1, 5 + 1)` | 6 | 6 |
| 7 | -5 | `max(-5, 6 - 5)` | 1 | 6 |
| 8 | 4 | `max(4, 1 + 4)` | 5 | 6 |

最终答案是 `6`。

## 代码实现

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int cur = nums[0];
        int ans = nums[0];

        for (int i = 1; i < nums.size(); i++) {
            cur = max(nums[i], cur + nums[i]);
            ans = max(ans, cur);
        }

        return ans;
    }
};
```

### 代码分析

```cpp
int cur = nums[0];
int ans = nums[0];
```

`cur` 和 `ans` 都从第一个元素开始，保证全负数数组也能得到正确答案。

```cpp
cur = max(nums[i], cur + nums[i]);
```

这一行是状态转移。它回答的问题是：以 `i` 结尾的最大连续子数组，到底是从 `i` 重新开始，还是从前面的连续段延续过来。

```cpp
ans = max(ans, cur);
```

`cur` 只代表“以当前位置结尾”的最优值，不一定是全局最优。所以每一步都要用 `ans` 记录历史最大值。

## 为什么不是滑动窗口

这题虽然也是连续子数组问题，但不能直接用常见的双指针滑动窗口。因为数组里有负数，窗口扩大后和可能变小，窗口缩小后和也可能变大，没有一个稳定的“窗口不合法就收缩”的判断条件。

动态规划的优势在于：它只关心“前面的连续和对当前元素有没有帮助”。如果没有帮助，就从当前元素重新开始。

## 易错点

1. `cur` 不是全局最大值，而是“必须以当前位置结尾”的最大和。
2. `ans` 才是最终答案，需要每一轮更新。
3. 初始化不能写成 `0`，否则全负数数组会出错。
4. 子数组要求连续，不能跳过中间的负数；例如 `[4,-1,2,1]` 中的 `-1` 必须保留。
5. `cur + nums[i]` 使用的是更新前的 `cur`，表示前一个位置结尾的最优连续段。

> [!warning]
> 如果你发现自己想把所有正数都加起来，就已经偏离题意了。最大子数组要求连续，不能只挑正数。

## 复杂度

- **时间复杂度**：`O(n)` — 只遍历数组一次。
- **空间复杂度**：`O(1)` — 只使用 `cur` 和 `ans` 两个变量。
