# 206. 反转链表

## 翻转链表简介

链表翻转指的是：把单链表中每个节点的 `next` 指针方向反过来。

原链表：

```text
1 -> 2 -> 3 -> 4 -> 5 -> nullptr
```

翻转后：

```text
5 -> 4 -> 3 -> 2 -> 1 -> nullptr
```

### 核心问题

单链表只能通过 `next` 指针向后走，所以翻转时最容易丢失后续节点。

因此无论是常规迭代法还是递归法，本质都要解决两个问题：

- 保存当前节点的下一个节点，避免链表断掉后找不到后面部分
- 修改当前节点的 `next` 指向，让它指回前一个节点

> [!tip]
> 链表题里，一旦要修改 `next` 指针，先想清楚“断开之前要不要保存后继节点”。

---

## 题目描述

给你单链表的头节点 `head`，请你反转链表，并返回反转后的链表。

### 示例

```text
输入：head = [1,2,3,4,5]
输出：[5,4,3,2,1]
```

```text
输入：head = [1,2]
输出：[2,1]
```

```text
输入：head = []
输出：[]
```

---

## 常规法：三指针迭代

### 思路

用三个指针维护翻转过程：

- `pre`：已经翻转好的部分的头节点
- `head`：当前正在处理的节点
- `last`：当前节点原本的下一个节点，用来防止链表断开后丢失后续节点

每次循环做四件事：

1. `last = head->next`：先保存后继节点
2. `head->next = pre`：让当前节点指向前一个节点
3. `pre = head`：当前节点变成新的已翻转头节点
4. `head = last`：继续处理原链表中的下一个节点

### 过程示意

```text
初始：
pre = nullptr
head = 1 -> 2 -> 3 -> nullptr

处理 1：
nullptr <- 1    2 -> 3 -> nullptr
pre = 1
head = 2

处理 2：
nullptr <- 1 <- 2    3 -> nullptr
pre = 2
head = 3

处理 3：
nullptr <- 1 <- 2 <- 3
pre = 3
head = nullptr
```

当 `head == nullptr` 时，说明原链表已经处理完，`pre` 就是翻转后的新头节点。

### 代码实现

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* pre = nullptr;
        ListNode* last = nullptr;

        while (head != nullptr) {
            last = head->next;
            head->next = pre;
            pre = head;
            head = last;
        }

        return pre;
    }
};
```

### 关键点

- `last = head->next` 必须放在修改 `head->next` 之前
- 循环结束后返回的是 `pre`，不是 `head`
- 空链表时循环不会进入，直接返回 `nullptr`

## 递归法

### 思路

递归法可以理解为：先把后面的链表翻转好，再把当前节点接到最后。

对于链表：

```text
1 -> 2 -> 3 -> 4 -> 5
```

递归先处理：

```text
2 -> 3 -> 4 -> 5
```

假设递归已经把它翻转成：

```text
5 -> 4 -> 3 -> 2
```

此时当前节点 `1` 还指向 `2`：

```text
1 -> 2
```

要把 `1` 接到尾部，只需要：

```cpp
head->next->next = head;
head->next = nullptr;
```

含义：

- `head->next->next = head`：让 `2` 指回 `1`
- `head->next = nullptr`：断开 `1 -> 2`，避免形成环

### 递归出口

```cpp
if (head == nullptr || head->next == nullptr) {
    return head;
}
```

含义：

- 空链表不需要反转
- 只有一个节点时，它本身就是反转后的头节点

### 代码实现

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        if (head == nullptr || head->next == nullptr) {
            return head;
        }

        ListNode* newHead = reverseList(head->next);

        head->next->next = head;
        head->next = nullptr;

        return newHead;
    }
};
```

### 递归法的理解重点

递归函数 `reverseList(head)` 的语义是：

> 把以 `head` 为头节点的链表整体翻转，并返回翻转后的新头节点。

所以执行 `ListNode* newHead = reverseList(head->next);` 后，可以相信 `head` 后面的链表已经翻转完成。当前层只需要把 `head` 接到尾部即可。

---

## 两种写法对比

| 方法 | 思想 | 优点 | 注意点 |
|------|------|------|--------|
| 常规迭代法 | 用指针逐个改变 `next` 方向 | 空间复杂度低，最常用 | 先保存后继节点 |
| 递归法 | 先翻转后半段，再回溯处理当前节点 | 代码短，适合理解递归 | 注意断开原指向，避免成环 |

---

## 复杂度

两种方法都会遍历每个节点一次：

- 时间复杂度：`O(n)`
- 迭代法空间复杂度：`O(1)`
- 递归法空间复杂度：`O(n)`，来自递归调用栈

---

## 易错点

1. 修改 `head->next` 之前，必须先保存 `head->next`
2. 迭代法最后返回 `pre`
3. 递归法必须写 `head->next = nullptr`，否则可能形成环
4. 递归出口要覆盖空链表和单节点链表
5. 翻转的是节点之间的指针，不是交换节点中的值
