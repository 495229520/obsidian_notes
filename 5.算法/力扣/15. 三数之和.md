# 15. 三数之和

## 题目描述

给定一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]`，满足：

- `i != j`
- `i != k`
- `j != k`
- `nums[i] + nums[j] + nums[k] == 0`

返回所有和为 `0` 且不重复的三元组。

> [!warning]
> 答案中不能包含重复三元组。三元组内部顺序不重要，结果列表的顺序也不重要。

---

## 示例

```text
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
```

```text
输入：nums = [0,1,1]
输出：[]
```

```text
输入：nums = [0,0,0]
输出：[[0,0,0]]
```

---

## 核心思路

三数之和可以转化成“固定一个数，再在剩余区间里找两数之和”。

如果直接三层循环，时间复杂度是 `O(n^3)`。更合适的做法是：

1. 先对数组排序
2. 枚举第一个数 `nums[i]`
3. 在 `i + 1` 到 `n - 1` 区间内使用双指针 `left`、`right`
4. 根据三数之和的大小移动指针
5. 通过跳过重复值避免重复答案

---

## 为什么排序后可以用双指针

排序后，数组从小到大排列。

固定 `nums[i]` 后，问题变成在右侧区间寻找：

```text
nums[left] + nums[right] == -nums[i]
```

因为区间已经有序：

- 如果 `sum < 0`，说明当前和太小，需要让和变大，所以 `left++`
- 如果 `sum > 0`，说明当前和太大，需要让和变小，所以 `right--`
- 如果 `sum == 0`，说明找到一个答案，同时移动两个指针继续寻找

这就是双指针能够把内层查找从 `O(n^2)` 降到 `O(n)` 的原因。

---

## 算法流程

设排序后的数组为 `nums`：

1. 如果 `nums.size() < 3`，直接返回空结果
2. 对 `nums` 升序排序
3. 从左到右枚举 `i`
4. 如果 `nums[i] > 0`，可以提前结束
5. 如果 `i > 0 && nums[i] == nums[i - 1]`，跳过重复的第一个数
6. 初始化：
   - `left = i + 1`
   - `right = nums.size() - 1`
7. 当 `left < right` 时计算三数之和：
   - `sum < 0`：`left++`
   - `sum > 0`：`right--`
   - `sum == 0`：加入答案，然后跳过重复的 `left` 和 `right`

> [!tip]
> `nums[i] > 0` 时可以提前结束，因为排序后 `nums[i]` 以及它右侧的数都大于等于 `nums[i]`，三数之和不可能再等于 `0`。

---

## 去重逻辑

本题最容易错在去重，去重分为两个层次。

### 1. 固定数 `i` 的去重

如果当前固定的数和上一次相同，就会产生重复三元组：

```cpp
if (i > 0 && nums[i] == nums[i - 1]) {
    continue;
}
```

例如排序后数组中有两个 `-1`，如果两个 `-1` 都作为第一个数枚举，就可能重复得到 `[-1, 0, 1]`。

### 2. 命中答案后的双指针去重

找到一个答案后，`left` 和 `right` 指向的值也可能重复：

```cpp
while (left < right && nums[left] == nums[left + 1]) {
    ++left;
}
while (left < right && nums[right] == nums[right - 1]) {
    --right;
}
```

跳过重复值后，再执行：

```cpp
++left;
--right;
```

## 代码实现

这版代码不再使用 `set` 做结果去重，而是在排序后的扫描过程中主动跳过重复值，因此结果天然不重复。

> [!note]
> 最初版本里用 `set<vector<int>>` 去重也能表达思路，但命中 `sum == 0` 后如果不移动 `left/right`，循环会卡住；而且 `set` 会额外带来插入排序开销。下面这版直接在双指针流程中去重，更符合本题的标准写法。

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>> res;
        sort(nums.begin(), nums.end());

        int n = nums.size();

        for (int i = 0; i < n; i++) {
            if (nums[i] > 0) {
                break;
            }

            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int left = i + 1;
            int right = n - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum > 0) {
                    right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    res.push_back({nums[i], nums[left], nums[right]});

                    while (left < right && nums[left] == nums[left + 1]) {
                        left++;
                    }

                    while (left < right && nums[right] == nums[right - 1]) {
                        right--;
                    }

                    left++;
                    right--;
                }
            }
        }

        return res;
    }
};
```

## 示例过程

以 `nums = [-1,0,1,2,-1,-4]` 为例。

排序后：

```text
[-4, -1, -1, 0, 1, 2]
```

### 固定 `i = 0`

```text
nums[i] = -4
left = 1, right = 5
```

此时无论如何移动，都找不到和为 `0` 的组合。

### 固定 `i = 1`

```text
nums[i] = -1
left = 2, right = 5
sum = -1 + (-1) + 2 = 0
```

得到答案：

```text
[-1, -1, 2]
```

继续移动指针后，还可以得到：

```text
[-1, 0, 1]
```

### 固定 `i = 2`

```text
nums[i] = -1
```

因为 `nums[2] == nums[1]`，跳过，避免重复答案。

## 复杂度分析

| 项目 | 复杂度 | 原因 |
|------|--------|------|
| 时间复杂度 | `O(n^2)` | 排序是 `O(n log n)`，主循环加双指针是 `O(n^2)` |
| 空间复杂度 | `O(log n)` 到 `O(n)` | 取决于排序实现的额外栈空间；不计答案数组 |

---

## 易错点

1. 不能用三层循环暴力做，否则容易超时
2. 必须先排序，否则双指针移动没有单调性依据
3. `i` 的去重条件必须是 `i > 0 && nums[i] == nums[i - 1]`
4. 找到答案后，要先跳过重复的 `left/right`，再同时移动两个指针
5. `nums[i] > 0` 后可以直接 `break`，不是 `continue`
6. 返回结果不能包含重复三元组

---

## 关键记忆

> [!important]
> 三数之和的本质是：排序后固定一个数，把问题降维成有序区间里的两数之和。
