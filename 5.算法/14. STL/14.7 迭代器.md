# 迭代器

> 本笔记与 [[12.1 迭代器基础概念]]、[[12.2 vector容器的迭代器实现]]、[[12.3 迭代器的失效问题]] 相关。

## 简介

### 迭代器是什么？

在 C++ 中，迭代器是一种对象，它允许程序员遍历容器（如数组或链表）中的元素，而无需了解容器的内部结构。**迭代器提供了一种方法来按顺序访问容器中的每个元素，通常用于循环和算法中对容器的元素进行操作。**

```cpp
vector<int> a;

// 传统下标访问
for (int i = 0; i < a.size(); i++)
    a[i] = i;

// 使用迭代器访问
for (vector<int>::iterator it = a.begin(); it != a.end(); ++it)
    *it;

// 反向迭代器
for (vector<int>::iterator it = a.rbegin(); it != a.rend(); ++it)
    *it;
```

### 基本操作

```cpp
a.begin();   // 指向第一个元素
a.end();     // 指向最后一个元素的下一个位置
a.rbegin();  // 反向头迭代器
a.rend();    // 反向尾迭代器
```

### 为什么需要迭代器？

因为不是所有的容器都是线性排列的，对于非线性的数据结构就需要迭代器，比如二叉树的中序遍历。

---

## 迭代器分类

迭代器主要分为五种：

| 类型 | 说明 | 支持的操作 |
|------|------|------------|
| **输入迭代器**（Input Iterator） | 只能向前移动，用于读取元素 | `++`, `*`, `==`, `!=` |
| **输出迭代器**（Output Iterator） | 只能向前移动，用于写入元素 | `++`, `*` |
| **前向迭代器**（Forward Iterator） | 可以向前移动，用于读写元素，并且可以用于多次遍历 | `++`, `*`, `==`, `!=` |
| **双向迭代器**（Bidirectional Iterator） | 可以向前和向后移动，用于读写元素 | `++`, `--`, `*`, `==`, `!=` |
| **随机访问迭代器**（Random Access Iterator） | 可以直接访问任何位置的元素，支持复杂的指针算术操作 | `++`, `--`, `+n`, `-n`, `[]`, `<`, `>` |

迭代器的使用使得算法与容器的具体实现解耦，提高了代码的复用性和灵活性。例如，可以使用相同的迭代器接口对 `std::vector` 和 `std::list` 进行操作，尽管它们内部的数据结构完全不同。

---

## 常见方法

| 方法 | 说明 |
|------|------|
| `.begin()` | 头迭代器 |
| `.end()` | 尾迭代器 |
| `.rbegin()` | 反向头迭代器 |
| `.rend()` | 反向尾迭代器 |
| 迭代器 + 整型 | 将迭代器向后移动 |
| 迭代器 - 整型 | 将迭代器向前移动 |
| 迭代器 ++ | 将迭代器向后移动 1 位 |
| 迭代器 -- | 将迭代器向前移动 1 位 |
| 迭代器 - 迭代器 | 两个迭代器的距离 |
| `prev(it)` | 返回 it 的前一个迭代器 |
| `next(it)` | 返回 it 的后一个迭代器 |

### 迭代器运算示例

```cpp
// 对于 vector 而言，迭代器减去迭代器得到的是整型
int o = a.end() - a.begin();
```

### 注意事项

- `.end()` 和 `.rend()` 指向的位置是没有意义的值
- 因此遍历的时候是 `it != vector.end()`

## 迭代器删除操作警惕

对迭代器进行删除操作的时候需要警惕！

```cpp
vector<int> a1{ 1, 2, 3, 4 };
for (auto it = a1.begin(); it != a1.end(); ++it)
    if (*it == 2 || *it == 3)
        a1.erase();  // 错误：删除后迭代器失效

for (auto it = a1.begin(); it != a1.end(); ++it)
    if (*it == 4)
        a1.erase();  // 如果删除了 4 的迭代器，那么循环会自动加一，此时后面就没有迭代器了，会进行报错
```

### 正确的删除方式

```cpp
vector<int> a1{ 1, 2, 3, 4 };
for (auto it = a1.begin(); it != a1.end(); /* 不在这里 ++it */)
{
    if (*it == 2 || *it == 3)
    {
        it = a1.erase(it);  // erase 返回下一个有效迭代器
    }
    else
    {
        ++it;
    }
}
```

## vector 与迭代器

在 C++ 中，`vector` 是一个序列容器，它封装了动态大小的数组。`vector` 可以动态地增长和缩小，这意味着你可以在程序执行过程中改变它的大小。

使用 `push_back()` 方法可以向 vector 里面添加元素。

### 迭代器的定义

在 C++ 中，迭代器是一种对象，它允许程序员遍历容器（如数组或链表）中的元素，而无需了解容器的内部结构。**迭代器提供了一种方法来按顺序访问容器中的每个元素，通常用于循环和算法中对容器的元素进行操作。**

### 代码示例

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> vec;  // 创建一个 int 类型的 vector
    for (int i = 0; i < 10; ++i) {
        vec.push_back(i);  // 向 vector 添加元素
    }

    // 使用迭代器遍历 vector
    for (std::vector<int>::iterator it = vec.begin(); it != vec.end(); ++it) {
        std::cout << *it << ' ';  // 输出当前迭代器指向的元素
    }

    return 0;
}
```

在这个例子中，我们创建了一个 `std::vector<int>` 类型的 vec，并向其中添加了 10 个整数。然后，我们声明了一个 `std::vector<int>::iterator` 类型的迭代器 it，并使用它来遍历 vec 中的所有元素。

## auto 关键字

在 C++ 中，`auto` 关键字用于自动类型推导，它可以让编译器根据变量的初始化表达式来推断变量的类型。这意味着使用 `auto` 时，你不需要显式地指定变量的类型，编译器会自动为你完成这一工作。

这意味着你最好不要使用 `auto` 进行变量的声明，这样会出现一些错误。

你可以使用 `auto` 代替迭代器的作用，因为这里 v 已经被声明为了 vector 类型：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // 使用 auto 进行迭代器类型推导
    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << ' ';
    }

    // 使用 auto 进行变量类型推导
    auto num = 42;  // num 被推导为 int 类型

    return 0;
}
```

## 关键要点

1. **迭代器是通用的**：所有 STL 容器都支持迭代器
2. **迭代器失效**：插入和删除操作可能导致迭代器失效
3. **范围 for 循环**：底层使用迭代器实现
4. **const 迭代器**：`cbegin()` 和 `cend()` 返回常量迭代器，不能修改元素
