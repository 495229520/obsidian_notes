# 14.9 函数对象

## 概念

**函数对象**（Function Object），也称为**仿函数**（Functor），是一个可以像函数一样被调用的对象。它本质上是一个重载了 `operator()` 的类的实例，因此也叫做**函数调用运算符重载**。

> 函数对象把函数调用运算符 `()` 进行 [[11. 运算符重载|运算符重载]]，使得该对象可以像函数一样被调用。

### 函数对象的优点

| 序号 | 优点 |
|------|------|
| 1 | 通过函数对象调用 `operator()`，可以省略函数的调用开销，比通过函数指针调用函数（无法 inline 内联调用）效率更高 |
| 2 | 因为函数对象是用类生成的，所以可以添加相关的成员变量，用来记录函数对象使用时更多的信息 |

## 函数对象 vs 函数指针

### 使用函数指针的方式

```cpp
template<typename T>
bool mygreater(T a, T b)
{
    return a > b;
}

template<typename T>
bool myless(T a, T b)
{
    return a < b;
}

// compare 是 C++ 库函数模版
template<typename T, typename Compare>
bool compare(T a, T b, Compare comp)
{
    return comp(a, b);
}

int main()
{
    // 通过函数指针调用函数，是没有办法内联的，效率较低
    cout << compare(10, 20, mygreater<int>) << endl;
    cout << compare(10, 20, myless<int>) << endl;
    return 0;
}
```

### 使用内联函数指针（仍无法优化）

通过函数指针调用函数是**无法进行内联优化**的，因为编译器在编译时无法确定函数指针指向哪个函数：

```cpp
// 使用 inline 的函数指针解决方案（仍然无效）
template<typename T>
inline bool mygreater(T a, T b)
{
    return a > b;
}

template<typename T>
inline bool myless(T a, T b)
{
    return a < b;
}
```

即使加了 `inline`，通过函数指针调用时仍**无法内联**，因为 `inline` 只是建议，而函数指针是间接调用。

## 使用函数对象解决

### C++ 库函数对象解决方案

```cpp
template<typename T>
class mygreater
{
public:
    bool operator()(T a, T b)  // 二元函数对象
    {
        return a > b;
    }
};

template<typename T>
class myless
{
public:
    bool operator()(T a, T b)  // 二元函数对象
    {
        return a < b;
    }
};

// compare 是 C++ 库函数模版
template<typename T, typename Compare>
bool compare(T a, T b, Compare comp)
{
    // 调用函数对象的 operator()，是有办法内联的
    return comp(a, b);
}

int main()
{
    // 通过函数对象调用函数，是没有办法内联的，效率较低
    cout << compare(10, 20, mygreater<int>()) << endl;
    cout << compare(10, 20, myless<int>()) << endl;
    return 0;
}
```

> 这里的 `mygreater<int>()` 是一个对象，通过对象调用 `operator()` 的函数类型在编译时是确定的，所以**可以内联**。

## 案例：优先级队列与函数对象

```cpp
// 默认大根堆
priority_queue<int> que1;  // 底层是 vector
for (int i = 0; i < 10; ++i)
{
    que1.push(rand() % 100);
}

while (!que1.empty())
{
    cout << que1.top() << " ";
    que1.pop();
}
// 78 69 67 64 62 58 41 34 24 0
cout << endl;

// 小根堆
// using 是用来创建别名，命名为 MinHeap
// int：元素的类型，vector<int>：底层容器的类型，greater<int>：比较器，使用"小于"关系来排序
using MinHeap = priority_queue<int, vector<int>, greater<int>>;
MinHeap que2;  // 底层使用 vector
for (int i = 0; i < 10; ++i)
{
    que2.push(rand() % 100);
}

while (!que2.empty())
{
    cout << que2.top() << " ";
    que2.pop();
}
// 5 27 27 36 42 45 61 81 91 95
cout << endl;

// 传入函数对象 greater<int>
set<int, greater<int>> set1;
for (int i = 0; i < 10; ++i)
{
    set1.insert(rand() % 100);
}

for (int v : set1)
{
    cout << v << " ";
}
cout << endl;
```

> `std::greater<int>` 这是一个类模板，它作为模板参数传递给容器（如 `priority_queue` 或 `set`）在内部自动创建相应的对象。

## STL 中的函数对象

STL 提供了一系列预定义的函数对象，位于 `<functional>` 头文件中：

### 算术运算函数对象

| 函数对象 | 作用 |
|----------|------|
| `plus<T>` | 加法 |
| `minus<T>` | 减法 |
| `multiplies<T>` | 乘法 |
| `divides<T>` | 除法 |
| `modulus<T>` | 取模 |
| `negate<T>` | 取反 |

### 关系运算函数对象

| 函数对象 | 作用 |
|----------|------|
| `equal_to<T>` | 等于 |
| `not_equal_to<T>` | 不等于 |
| `greater<T>` | 大于 |
| `less<T>` | 小于（默认） |
| `greater_equal<T>` | 大于等于 |
| `less_equal<T>` | 小于等于 |

### 逻辑运算函数对象

| 函数对象 | 作用 |
|----------|------|
| `logical_and<T>` | 逻辑与 |
| `logical_or<T>` | 逻辑或 |
| `logical_not<T>` | 逻辑非 |

---

## 关键要点

1. **函数对象** = 重载了 `operator()` 的类的对象
2. **优势**：可内联优化，可携带状态
3. **vs 函数指针**：函数指针无法内联，函数对象可以
4. **STL 中的使用**：`greater<T>`、`less<T>` 等用于排序和比较
