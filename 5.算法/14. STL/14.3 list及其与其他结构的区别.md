# list 链表容器

## 简介

- **全称**：链表容器
- **底层数据结构**：双向循环列表

```
双向循环链表结构：

     ┌──────────────────────────────────────────┐
     │                                          │
     ↓                                          │
  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐     │
  │     │←──→│     │←──→│     │←──→│     │─────┘
  └─────┘    └─────┘    └─────┘    └─────┘
     ↑                                │
     └────────────────────────────────┘
```

## 常见方法

```cpp
list<int> mylist;

// 增加
mylist.push_back(20);       // 从末尾添加元素 O(1)
mylist.push_front(20);      // 从首部添加元素 O(1)
mylist.insert(it, 20);      // 从 it 迭代器指向的位置添加元素 O(n)

// 删除
mylist.pop_back();          // 从末尾删除元素 O(1)
mylist.pop_front();         // 从首部删除元素 O(1)
mylist.erase(it);           // 从 it 指向的位置删除元素 O(n)

// 查询搜索
// iterator（连续的 insert 和 erase 一定要考虑迭代器失效的问题）
```

## 区别

### deque 和 list 比 vector 容器多出来的增加删除函数接口

- `push_front` 和 `pop_front`

### vector、deque、list 三方对比

| 容器 | 特点 |
|------|------|
| **vector** | 动态数组，内存是连续的，并且以二倍的方式进行扩容<br>`vector<int> vec; 0-1-2-4-8...` |
| **deque** | 动态开辟的二维数组空间，第二维是固定长度的数组空间，扩容的时候，第一维的数组进行二倍扩容<br>**面经问题**：deque 底层内存是否是连续的？<br>不是，deque 每一个二维数组都是独立 new 出来的，单独的二维数组是连续的，但是各个二维数组的内存空间不连续 |
| **list** | 底层数据结构的区别：一个是一维的，一个是二维的<br>前中后插入删除元素的时间复杂度：<br>- **前面插入**：`vector` O(n)，`deque` O(n)，**末尾插入** `vector` O(1)，`deque` O(1)<br>- 对于内存的使用效率：`vector` 需要的内存空间必须是连续的，`deque` 可以分块进行数据存储，不需要内存空间必须是连续的<br>- 在中间进行 insert 或者 erase 和 deque 的效率区别？虽然二者的时间复杂度都是 O(n)，但是由于 deque 的第二维内存空间不是连续的，所以在 deque 中间进行元素的 insert 或者 erase 的时候，元素移动的效率比 vector 要慢 |

### vector 和 deque 的区别

1. **底层数据结构的区别**：一个是一维的，一个是二维的
2. **前中后插入删除元素的时间复杂度**：

| 操作 | 前面插入 | 中间插入 | 末尾插入 |
|------|----------|----------|----------|
| vector | O(n) | O(n) | O(1) |
| deque | O(1) | O(1) | O(1) |

3. **对于内存的使用效率**：
   - `vector` 需要的内存空间必须是连续的
   - `deque` 可以分块进行数据存储，不需要内存空间必须是连续的

4. **在中间进行 insert 或者 erase 和 deque 的效率区别**：
   - 虽然二者的时间复杂度都是 O(n)
   - 但是由于 deque 的第二维内存空间不是连续的
   - 所以在 deque 中间进行元素的 insert 或者 erase 的时候，元素移动的效率比 vector 要慢

```
deque 中间删除示意图：

          ┌─────────────────┐
          │   预留空行       │
          ├─────────────────┤ first
          │   ○ ○ ○ ○ ○     │ ←── 需要移动
          ├─────────────────┤
          │   ○ ○ X ○ ○     │ ←── 删除中间元素
          │       ↑         │
          │   需要跨块移动   │
          ├─────────────────┤ last
          │   预留空行       │
          └─────────────────┘
```

### vector 和 list 的区别

| 对比项 | vector | list |
|--------|--------|------|
| **底层结构** | 一个动态开辟的数组 | 一个双向循环链表 |
| **增加删除时间复杂度** | 数组增加删除 O(n)，查询 O(n)，随机访问 O(1) | 链表增加删除 O(1)，查询 O(n)，链表没有随机访问 |

---

## 关键要点

1. **list 适合频繁插入删除**：由于是链表结构，在任意位置插入删除都是 O(1)（找到位置后）
2. **list 不支持随机访问**：不能使用 `[]` 下标访问，只能通过迭代器遍历
3. **list 内存不连续**：每个节点独立分配，可能导致缓存不友好
4. **迭代器稳定性**：list 的迭代器在插入删除后不会失效（除了被删除的元素）
