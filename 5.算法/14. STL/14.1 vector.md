# vector 容器

> 本笔记是对 [[4.2 vector数组]] 的深入补充，主要讲解 vector 的底层实现原理和高级用法。

## 简介

`vector` 是**向量容器**，底层数据结构是**动态数组**（Dynamic Array）。它允许你在程序执行过程中动态地改变数组的大小，与静态数组不同，不需要在编译时确定大小。

```cpp
#include <vector>
using namespace std;
```

### vector 的特点

- 动态开辟的数组，每次以**原来空间大小的 2 倍**进行扩容（MSVC 实现为 1.5 倍）
- 扩容时会分配新的连续内存空间，将原有数据复制过去，释放旧内存
- 支持随机访问，底层内存连续

## 初始化方式

```cpp
vector<int> vec;                    // 空 vector
vector<int> vec(10);                // 10 个元素，默认初始化为 0
vector<int> vec(10, 5);             // 10 个元素，全部初始化为 5
vector<int> vec{1, 2, 3, 4, 5};     // 列表初始化
vector<int> vec = {1, 2, 3, 4, 5};  // 等价写法

// 拷贝构造
vector<int> vec2(vec);
vector<int> vec3 = vec;

// 从迭代器范围构造
vector<int> vec4(vec.begin(), vec.end());
```

## 常用操作

### 增加元素

| 操作 | 说明 | 时间复杂度 |
|------|------|-----------|
| `push_back(20)` | 末尾添加元素 | 均摊 O(1) |
| `insert(it, 20)` | 在迭代器指向的位置添加一个元素 | O(n) |
| `vec.insert(vec.begin(), 20)` | 导致容器扩容，需要更新迭代器 | O(n) |

### 删除元素

| 操作 | 说明 | 时间复杂度 |
|------|------|-----------|
| `pop_back()` | 末尾删除元素 | O(1) |
| `erase(it)` | 删除迭代器指向的元素 | O(n) |

### 查询

| 操作 | 说明 | 时间复杂度 |
|------|------|-----------|
| `operator[]` | 下标随机访问 `vec[5]` | O(1) |
| `iterator` | 迭代器遍历 | - |
| `find_for_each` | 范围遍历 | O(n) |
| `foreach` | 底层也是通过 iterator 实现 | O(n) |

### 使用迭代器

```cpp
// 正向迭代
for (vector<int>::iterator it = vec.begin(); it != vec.end(); ++it) {
    cout << *it << " ";
}

// 反向迭代
for (vector<int>::iterator it = vec.rbegin(); it != vec.rend(); ++it) {
    cout << *it << " ";
}

// 使用 auto 简化（C++11）
for (auto it = vec.begin(); it != vec.end(); ++it) {
    cout << *it << " ";
}
```

- `begin()` 返回指向 vector 第一个元素的迭代器
- `end()` 返回指向 vector 最后一个元素之后位置的迭代器
- **注意**：对容器进行连续插入或删除操作（insert/erase），需要更新迭代器，因为首次 insert 或 erase 完成后迭代器就失效了

### 常用方法

| 方法 | 说明 |
|------|------|
| `size()` | 获取 vector 的长度（元素个数） |
| `empty()` | 判断 vector 是否为空 |
| `reserve(20)` | 为 vector 预留空间（只改变 capacity，不改变 size） |
| `resize(20)` | 重新分配的内存空间，用于容器扩充（改变 size） |
| `swap` | 两个容器进行元素交换 |

## 代码示例

```cpp
int main()
{
    // 只让容器底层开辟指定大小的内存空间，并不会添加新的元素
    // 此时容器是空的
    vec.reserve(20);    // 为 vector 预留空间
    // 重新分配 20 的内存空间，用于容器扩充
    vec.resize(20);

    cout << vec.empty() << endl;     // 1（reserve 后为空）
    cout << vec.size() << endl;      // 0

    for (int i = 0; i < 20; ++i)
    {
        vec.push_back(rand() % 100 + 1);
    }

    cout << vec.empty() << endl;     // 0
    cout << vec.size() << endl;      // 20

    // vector 的 operator[] 运算符重载函数
    int size = vec.size();
    for (int i = 0; i < size; ++i)
    {
        cout << vec[i] << " ";
    }

    // 把 vec 容器中所有的偶数全部删除
    auto it2 = vec.begin();
    while (it2 != vec.end())
    {
        if (*it2 % 2 == 0)
        {
            it2 = vec.erase(it2);  // 更新迭代器
        }
        else
        {
            ++it2;
        }
    }

    // 通过迭代器遍历 vector 容器
    auto it1 = vec.begin();
    for (; it1 != vec.end(); ++it1)
    {
        cout << *it1 << " ";
    }
    cout << endl;

    // 给 vector 容器中所有的奇数前面都添加一个小于奇数 1 的偶数
    for (it1 = vec.begin(); it1 != vec.end(); ++it1)
    {
        if (*it1 % 2 != 0)
        {
            it1 = vec.insert(it1, *it1 - 1);
            ++it1;  // 跳过刚插入的元素
        }
    }

    return 0;
}
```

## vector 底层机制

### 三个核心指针

vector 内部维护三个指针：

```cpp
template<class T>
class vector {
    T* _first;   // 指向数组起始位置
    T* _last;    // 指向有效元素的后继位置
    T* _end;     // 指向数组空间的后继位置
};
```

```
内存布局：
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │ 5 │   │   │   │
└───┴───┴───┴───┴───┴───┴───┴───┘
↑                   ↑           ↑
_first            _last       _end
```

- `size()` = `_last - _first`（有效元素个数）
- `capacity()` = `_end - _first`（容量大小）

### 扩容机制详解

当 `push_back` 导致 `size > capacity` 时：

1. 分配新内存（通常为原容量的 2 倍，MSVC 为 1.5 倍）
2. 将原有元素拷贝到新内存
3. 释放原内存
4. 更新 `_first`、`_last`、`_end` 指针

```cpp
vector<int> vec;
// vec: 0-1-2-4-8... 扩容序列
```

### size 与 capacity 的区别

| 属性 | 说明 |
|------|------|
| `size` | 当前有效元素的个数 |
| `capacity` | 当前分配的内存能容纳的元素个数 |

```cpp
vector<int> v;
v.reserve(100);  // capacity = 100, size = 0
v.resize(50);    // capacity = 100, size = 50
```

## 迭代器失效问题

> 详见 [[12.3 迭代器的失效问题]]

### 什么时候迭代器会失效？

1. **插入元素**：当容器需要扩容时，所有迭代器失效
2. **删除元素**：被删除元素及其之后的迭代器失效

### 解决方法

对迭代器进行更新操作：

```cpp
// 删除时更新迭代器
it = vec.erase(it);

// 插入时更新迭代器
it = vec.insert(it, value);
```

## 性能优化建议

### 1. 使用 reserve 预分配内存

```cpp
vector<int> vec;
vec.reserve(1000);  // 预分配空间，避免多次扩容
for (int i = 0; i < 1000; ++i) {
    vec.push_back(i);
}
```

### 2. 使用 emplace_back 代替 push_back

`emplace_back` 直接在容器内原地构造对象，避免临时对象的创建和拷贝：

```cpp
struct Point {
    int x, y;
    Point(int x, int y) : x(x), y(y) {}
};

vector<Point> points;
points.push_back(Point(1, 2));   // 构造临时对象 + 移动/拷贝
points.emplace_back(1, 2);       // 直接原地构造
```

### 3. 释放多余内存

```cpp
// 方法1：shrink_to_fit (C++11)
vec.shrink_to_fit();

// 方法2：swap with temporary (C++11之前)
vector<int>(vec).swap(vec);

// 方法3：swap with copy
vector<int> v2(vec);
v2.swap(vec);
```

## 一个简略的例子

```cpp
// 直接类型 size=sizeof(int) 即 4c
vector<int> v;
// v:_first   _last   _end
// 初始状态：v._first = nullptr, v._last = nullptr, v._end = nullptr

for (int i = 0; i < 20; ++i) {
    v.push_back(rand() % 100);
}
// 扩容过程：1 -> 2 -> 4 -> 8 -> 16 -> 32
```

观察发现，为什么 6~7 能过 vector 类初始化成员变量的扩容，但改成使用下标初始化就不行：

```cpp
// 错误示例
vector<int> v;
for (int i = 0; i < 100; i++) {
    v[i] = i;  // 未分配内存，行为未定义！
}

// 正确做法
vector<int> v(100);  // 或 v.resize(100);
for (int i = 0; i < 100; i++) {
    v[i] = i;
}
```

## 关键要点

1. **摊还时间复杂度**（Amortized Time Complexity）为 O(1)
   - 虽然单次扩容操作是 O(n)，但由于扩容次数随元素增加而指数级减少
   - 从长远来看，每次 `push_back` 的平均时间仍是 O(1)

2. **内存连续**：vector 的内存是连续的，支持 O(1) 随机访问

3. **迭代器失效**：插入和删除操作可能导致迭代器失效，需要更新

4. **扩容开销**：频繁扩容会影响性能，建议使用 `reserve` 预分配
