# set 集合容器

## 简介

**集合三要素**：

| 要素 | 解释 | set | multiset | unordered_set |
|------|------|-----|----------|---------------|
| **确定性** | 一个元素要么在集合中，要么不在 | ✓ | ✓ | ✓ |
| **互异性** | 一个元素仅可以在集合中出现一次 | ✓ | ✗ | ✓ |
| **有序性** | 集合中的元素是没有顺序的 | ✓ | ✓ | ✗ |

```cpp
set<int> st;
st.insert(1);
st.insert(2);
st.erase(2);
if (st.count(2))
{
    cout << "yes" << endl;
}
cout << st.size() << endl;
st.clear();
cout << st.empty() << endl;
```

## 常用方法

### set 基于红黑树实现的关联容器

`set` 是一个基于[[14.14 红黑树]]实现的关联容器，它的主要特点是：
- **唯一性**：set 中的所有元素都是唯一的，插入重复元素会被忽略
- **有序性**：元素在插入时会自动按升序排列（默认），你不能直接修改 set 中的元素，因为这会破坏排序
- **高效的查找、插入和删除**：由于其基于红黑树的实现，这些操作的时间复杂度均为 **O(logn)**，其中 n 是元素的数量
- **存储动态数组**：

```cpp
vector<int> intersection(vector<int>& nums1, vector<int>& nums2) {
    unordered_set<int> nums_set(nums1.begin(), nums1.end());
}
```

### 操作方法表

| 作用 | 用法 | 示例 |
|------|------|------|
| 插入元素 | `.insert(元素)` | `set.insert(1);` |
| 删除元素 | `.erase(元素)` | `set.erase(2);` |
| 查找元素 | `.find(元素)` | `auto it = set.find(1);` |
| 判断元素是否存在<br>找到返回 1，否则 0 | `.count(元素)` | `set.count(3);` |
| 判断大小 | `.size()` | `set.size()` |
| 清空 | `.clear()` | `set.clear()` |
| 判断是否为空 | `.empty()` | `set.empty()` |

### 遍历

```cpp
// 遍历，使用迭代器
for (set<int>::iterator it = st.begin(); it != st.end(); ++it)
    cout << *it << endl;

// 底层也是使用的迭代器
for (auto& ele : st)
    cout << ele << endl;
```

### 适用情景

- **元素去重**：`[1,1,3,2,4,4]` → `[1,2,3,4]`
- **维护顺序**：`[1,5,3,7,9]` → `[1,3,5,7,9]`
- **元素是否出现过**：元素大小 [-10^18, 10^18]，元素数量 10^6，vis 数组无法实现，通过 set 可以完成，`set.count(x)`

### 注意事项

1. set 可以遍历，但是仅可以使用迭代器进行遍历，不存在下标
2. 元素只读，set 迭代器取到的元素是只读的，不可以修改值
3. 不可以通过迭代器相减计算下标

## 底层原理

- `std::set` 的底层实现是一棵**红黑树**（Red-Black Tree）
- 红黑树是一种**自平衡的二叉搜索树**（Self-Balancing Binary Search Tree）
- 正是由于红黑树的这两个特性，保证了 `std::set` 的元素是有序的，并且插入、删除、查找等操作的时间复杂度都能维持在 **O(logn)**（自平衡特性）

### insert 步骤

当你调用 `set.insert(value)` 时，主要发生以下几个步骤：

1. **查找插入位置**：从红黑树的根节点开始，按照二叉搜索树的规则（小于往左子树，大于往右子树）向下查找合适的插入位置，这个过程的时间复杂度是 O(logn)

2. **检查元素是否存在**：在查找过程中，如果发现树中已经存在一个与 value 相等的元素，则 insert 方法会返回 false，并且不会插入新元素

3. **插入新节点**：如果找到了合适的空位置，就在这里创建一个新的红黑树节点来存储 value，新插入的节点默认是红色

4. **自我平衡（Recoloring）和旋转（Rotation）**：插入新的红色节点后，可能会破坏红黑树的五个基本性质。此时，红黑树需要进行一系列的修正操作，这个修正过程最多需要 2 次旋转和少量的颜色翻转，其时间复杂度是 O(logn)

因此，整个插入操作的总时间复杂度是 O(logn)。

## 红黑树

### 什么是红黑树？它的五个基本性质是什么？

**红黑树五个基本性质**（必须牢记）：

1. **性质 1**：每个节点要么是红色，要么是黑色（Node is Red or Black）
2. **性质 2**：根节点必须是黑色（Root is Black）
3. **性质 3**：所有叶子节点（NIL 节点，即空指针）都是黑色的（All leaves are Black）
4. **性质 4**：如果一个节点是红色的，那么它的两个子节点必须是黑色的（No two adjacent Red nodes）。这条性质是关键，它限制了树的路径上不能有连续的红色节点
5. **性质 5**：从任意一个节点到其所有后代叶子节点的路径上，所含的黑色节点的数量必须相同（Same number of Black nodes on every path from a given node to any of its descendant NIL nodes）。这条性质被称为"**黑高（Black-Height）**"守恒，是确保树近似平衡的核心

### 为什么 std::set 的插入和查找效率是 O(logn)？

**回答**：因为它的底层是红黑树。二叉搜索树的查找过程类似于二分查找，每次比较都可以将搜索范围减半。所以理想情况下的时间复杂度是 O(logn)。但普通的二叉搜索树在极端情况下（例如，插入一个有序序列）会退化成一个链表，导致复杂度变为 O(n)。而红黑树通过一系列的自平衡规则（后面会详细解释），能在任何情况下都维持着近乎平衡的高度差异性能，其最高的高度差异性能比是 2:1，因而复杂度也是 O(logn)。从而保证了高效的高度差异性能比的是常量，其插入复杂度也是 O(logn)。

## unordered_set

`unordered_set` 是**无序集合**，底层使用**哈希表**实现。

```cpp
set<int> st;
st.insert(1);
st.insert(2);
st.erase(2);
if (st.count(2))
{
    cout << "yes" << endl;
}
cout << st.size() << endl;
st.clear();
cout << st.empty() << endl;
```

### 特点

- 元素无序存储
- 查找、插入、删除的平均时间复杂度为 **O(1)**
- 最坏情况下为 O(n)（哈希冲突严重时）

## 有序集合存储自定义类型

### 简介

1. 存放元素不可重复
2. 存放元素从小到大进行排列
3. 存放自定义类型时，需要重载比较运算符，使得编译器能够确认各元素大小

### 案例

```cpp
class Student
{
public:
    Student(int id, string name)
        : _id(id), _name(name) { }

    bool operator<(const Student& stu) const
    {
        return _id < stu._id;
    }

private:
    int _id;
    string _name;
    friend ostream& operator<<(ostream& out, const Student& stu);
};

ostream& operator<<(ostream& out, const Student& stu)
{
    out << "id:" << stu._id << " name:" << stu._name << endl;
    return out;
}

int main()
{
    // 如果在有序关联容器中存储自定义类型，那么编译器会无法进行大小比较
    // 此时需要在自定义的类型中重写一个大小比较方法
    set<Student> set1;

    set1.insert(Student(1000, "张三"));
    set1.insert(Student(1020, "李四"));

    // 使用迭代器进行编译
    for (auto it = set1.begin(); it != set1.end(); ++it)
    {
        cout << *it << endl;
    }
}
```

## 关键要点

| 容器 | 底层结构 | 有序性 | 元素唯一性 | 查找复杂度 |
|------|----------|--------|------------|------------|
| `set` | [[14.14 红黑树]] | 有序 | 唯一 | O(logn) |
| `multiset` | [[14.14 红黑树]] | 有序 | 可重复 | O(logn) |
| `unordered_set` | 哈希表 | 无序 | 唯一 | 平均 O(1) |
