# queue 队列容器

## queue 简介

`queue` 是一种**先进先出**（FIFO: First-In, First-Out）的数据结构。你可以把它想象成银行的排队队伍：最先排队的人，总是最先得到服务并离开队伍。

### 核心特性

- **先进先出 (FIFO)**：这是 queue 最核心的特性。第一个进入队列的元素，将是第一个离开队列的元素
- **容器适配器**：与 [[14.8 容器适配器|stack]] 类似，queue 本身也不是独立的容器，而是对其他基础容器（如 deque 或 list）的封装。它提供了一个受限的接口，只允许你进行 FIFO 操作。默认情况下，queue 基于 `deque` 实现
- **受限的操作**：queue 只提供以下几种基本操作，以确保其 FIFO 特性：
  - `push()`：将一个元素添加到队列的**尾部**
  - `pop()`：移除队列**头部**的元素
  - `front()`：访问队列**头部**的元素，但不移除它
  - `back()`：访问队列**尾部**的元素，但不移除它
  - `empty()`：检查队列是否为空
  - `size()`：返回队列中元素的数量

### 常用方法

```cpp
queue<int> que;
que.push(1);      // 入队
que.front();      // 访问队头元素
que.back();       // 访问队尾元素
que.pop();        // 出队（移除队头）
que.empty();      // 判断是否为空
que.size();       // 返回元素个数
```

### 注意事项

1. queue **不可以访问内部元素**，只能访问队头和队尾
2. 有些时候可以把 queue 当做 stack 使用（只使用一端操作）

### 适用场景

持续维护元素的有序性：每次向队列插入大小不定的元素，或者每次从队列里取出大小最小/最大的元素，元素数量、插入操作数量 k：
- 使用普通队列维护，每次插入后进行快速排序：k·n·logn
- 使用优先队列维护：k·logn

## deque 双端队列

### 简介

- **全称**：双端队列容器
- **底层数据结构**：动态开辟的二维数组，一维数组从 2 开始，以 2 倍的方式进行扩容
- 每次扩容后原来的二维数组从新的第一维数组的下标 `oldsize / 2` 开始存放，上下都预留相同的空行
- 方便支持 deque 的首尾元素增加

### 常见方法

```cpp
deque<int> deq;

// 增加
deq.push_back(20);       // 从末尾添加元素 O(1)
deq.push_front(20);      // 从首部添加元素 O(1)
deq.insert(it, 20);      // 从 it 迭代器指向的位置添加元素 O(n)

// 删除
deq.pop_back();          // 从末尾删除元素 O(1)
deq.pop_front();         // 从首部删除元素 O(1)
deq.erase(it);           // 从 it 指向的位置删除元素 O(n)

// 查询搜索
// iterator（连续的 insert 和 erase 一定要考虑迭代器失效的问题）
```

## priority_queue 优先级队列

`priority_queue` 是一种特殊的队列，它不是按照先进先出（FIFO）的原则排列元素，而是按照元素的**优先级**排序。

### 核心特性

- 最大（或最小）优先级的元素总是在队列的最前面，可以最先被访问和移除
- 通常用堆（Heap）数据结构来实现，这使得插入和移除操作都能在 O(log n) 的时间复杂度内完成
- **重要操作**：
  - `push()`：添加一个元素，优先队列会自动将其放置到正确的位置，以保持堆的性质
  - `pop()`：移除优先级最高的元素（默认是最大的元素）
  - `top()`：访问优先级最高的元素，但不移除它
  - `empty()`：检查队列是否为空
  - `size()`：返回队列中元素的数量

### 底层实现

`priority_queue` 底层依赖 vector，底层默认是大根堆。

底层默认把数据组成一个大根堆结构，通常在一个内存的连续数组上构建一个大根堆或者小根堆。因为堆中每一个节点跟它左右孩子之间的关系是通过下标实现的：
- 左孩子：`2 * i + 1`
- 右孩子：`2 * i + 2`
- 父节点：`(i - 1) / 2`

### 基本操作

| 容器 | 操作 |
|------|------|
| stack | `push`: 入栈 `pop`: 出栈 `top`: 查看栈顶元素 `empty`: 判断栈空 `size`: 返回元素个数 |
| queue | `push`: 入队 `pop`: 出队 `front`: 查看队头元素 `empty`: 判断队空 `size`: 返回元素个数<br>stack 底层依赖 deque，原理同上 |
| priority_queue | `push`: 入队 `pop`: 出队 `top`: 查看队头元素 `empty`: 判断队空 `size`: 返回元素个数<br>priority_queue 底层依赖 vector，底层默认是大根堆 |

### 大根堆（Max-Heap）

大根堆（Max-Heap）中，父节点的值总是大于或等于其子节点的值。因此，堆的根节点（即 `top()` 返回的元素）总是堆中最大的元素。

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    // A. 实现自定义比较器
    // // 1.1 按照默认的规则大堆（Max-Heap）
    // struct cmp {
    //     // 规则：当 A.priority < B.priority 时，返回 true (A 优先级低)
    //     bool operator()(const Task& A, const Task& B) const {
    //         return A.priority < B.priority;  // 告诉优先队列
    //     }
    // };

    // // 1.2 实现比较器自定义小堆（Min-Heap）
    // // 规则：当 A.priority > B.priority 时，返回 true
    // struct cmp {
    //     bool operator()(const Task& A, const Task& B) const {
    //         return A.priority > B.priority;  // 优先级高的反而排后面
    //     }
    // };

    // 默认大顶堆
    priority_queue<int> max_heap;
    max_heap.push(rand() % 100 + 1);

    // 使用 greater 创建小顶堆
    priority_queue<int, vector<int>, greater<int>> min_heap;

    // 1.2 定义空容器，最高优先级 priority 最大的 Item
    priority_queue<int> max_heap_top1;
    max_heap_top1.push(10);  // 优先队列大顶堆
    // or max_heap_top.priority <= 1 or <= std::endl;

    // 测试 Task E（Priority: 90)
    // 输出: Task E（Priority: 90)
    // 输出: Task E（Priority: 90)

    // 使用 less 创建大顶堆
    priority_queue<string, vector<string>, less<string>> max_heap2;
    max_heap2.push("Task A");
    max_heap2.push("Task X");
    max_heap2.push("Task C");
    cout << max_heap2.top() << endl;  // Task X (字典序最大)

    return 0;
}
```

### 使用 Lambda 表达式自定义比较器

```cpp
#include <iostream>
#include <queue>
#include <functional>  // 需要包含 <functional>

struct Task {
    string name;
    int priority;
};

int main()
{
    // 2. 使用 Lambda 表达式
    // Min Heap 用于找最小的元素
    auto min_heap_lambda = [](const Task& A, const Task& B) {
        return A.priority > B.priority;  // 注意这里是 >
    };

    // Lambda min_heap_lambda 的底层 decltype 取类型生成 Lambda 匿名函数种类，Lambda 实际上 '>' 号值函数
    // 中的的尖括号描述种类，'>' 号括号的每个尖括号传个参数 Lambda 函数地址/引用传函数指针
    priority_queue<Task, vector<Task>, decltype(min_heap_lambda)>
        lambda_min_heap_top(min_heap_lambda);  // 需要 decltype 获取 Lambda 匿名类型

    lambda_min_heap_top.push({"Task A", 10});
    lambda_min_heap_top.push({"Task B", 5});
    cout << "Top: " << lambda_min_heap_top.top().name << endl;
    // 输出: Top: Task B (Priority: 5)
}
```

### 使用注意

1. 对于自定义类型，需要提供比较器
2. `greater<T>` 创建小顶堆，`less<T>` 创建大顶堆（默认）
3. 底层使用堆结构，插入和删除都是 O(log n)

## 关键要点

| 容器 | 特点 | 底层结构 | 适用场景 |
|------|------|----------|----------|
| queue | FIFO 先进先出 | deque | 任务队列、BFS |
| deque | 双端操作 O(1) | 分段连续数组 | 需要两端操作的场景 |
| priority_queue | 按优先级排序 | 堆（基于 vector） | 任务调度、Dijkstra 算法 |
