# 动态规划（Dynamic Programming）

## 什么是动态规划

动态规划（DP）是一种通过**把原问题分解为重叠的子问题**来求解的算法思想。

它的核心理念是：如果一个问题的最优解可以由其子问题的最优解推导出来，那么我们可以自底向上逐步构建答案，而不是反复重新计算。

> [!important] 动态规划 vs 分治
> - **分治**：子问题之间相互独立（如归并排序），递归即可
> - **动态规划**：子问题之间有重叠（如斐波那契），需要存储中间结果避免重复计算

---

## 适用条件

一个问题能用 DP 解决，通常需要满足两个性质：

### 1. 最优子结构

原问题的最优解包含子问题的最优解。

例如：到达第 `n` 阶楼梯的最优方案，一定包含到达第 `n-1` 阶或第 `n-2` 阶的最优方案。

### 2. 重叠子问题

在递归过程中，相同的子问题会被反复计算。

以斐波那契数列 `fib(5)` 为例，暴力递归的调用树如下：

![暴力递归的重复子问题|1160](dp_fibonacci_tree.svg)

`fib(3)` 被计算了 2 次，`fib(2)` 被计算了 3 次。随着 `n` 增大，重复计算呈指数级增长。

#### 为什么递归会产生重复子问题？

斐波那契数列的递推关系是：

```text
fib(n) = fib(n - 1) + fib(n - 2)
```

这意味着每次计算 `fib(n)`，都会继续拆成两个更小的子问题。问题在于：这两个分支并不是完全独立的。

例如计算 `fib(5)`：

```text
fib(5)
= fib(4) + fib(3)
= (fib(3) + fib(2)) + fib(3)
```

可以看到，`fib(3)` 既会出现在 `fib(5)` 的右分支中，也会出现在 `fib(4)` 的计算过程中。

继续展开：

```text
fib(4) = fib(3) + fib(2)
fib(3) = fib(2) + fib(1)
```

所以 `fib(2)` 也会被多条路径反复访问。

本质原因是：**递归树中存在多条不同的调用路径，最终会到达同一个子问题**。如果没有缓存，普通递归不会记住之前算过的结果，每次遇到 `fib(3)`、`fib(2)` 都会从头再算一遍。

因此，暴力递归的节点数量很多，但真正不同的子问题其实只有：

```text
fib(0), fib(1), fib(2), ..., fib(n)
```

也就是 `n + 1` 个。动态规划正是利用这一点，把每个子问题只计算一次，从而把时间复杂度从指数级优化到线性级。

> [!tip]
> 当你发现递归解法中存在大量重复调用时，就是使用动态规划的信号。

## 两种实现方式

![自顶向下 vs 自底向上|1284](dp_topdown_vs_bottomup.svg)

### 自顶向下：记忆化搜索（Memoization）

在递归的基础上加一个缓存表（备忘录），遇到算过的子问题直接查表返回。

```cpp
std::vector<int> memo(n + 1, -1);

int fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];

    memo[n] = fib(n - 1) + fib(n - 2);
    return memo[n];
}
```

**特点**：

- 写法接近递归，容易从暴力解改造
- 只计算实际需要的子问题（惰性求值）
- 有递归栈开销，极深递归可能栈溢出

### 自底向上：递推（Tabulation）

直接用循环从最小子问题开始，逐步填表推算到目标问题。

```cpp
int fib(int n) {
    if (n <= 1) return n;

    std::vector<int> dp(n + 1);
    dp[0] = 0;
    dp[1] = 1;

    for (int i = 2; i <= n; ++i) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }

    return dp[n];
}
```

**特点**：

- 没有递归开销，效率更高
- 力扣题解中最常用的写法
- 需要想清楚遍历顺序

### 如何选择

| 场景 | 推荐方式 |
|------|---------|
| 从暴力递归出发，快速优化 | 记忆化搜索 |
| 子问题数量确定，追求效率 | 递推 |
| 状态空间稀疏，很多子问题用不到 | 记忆化搜索 |
| 面试/竞赛，追求代码简洁 | 递推 |

## 解题五步法

![动态规划解题五步法|1318](dp_solving_steps.svg)

拿到一道 DP 题，按照以下五步来思考：

### 第一步：定义状态

明确 `dp[i]` 或 `dp[i][j]` 的含义。

这是最关键的一步。状态定义错误，后面的一切都会崩塌。

**常见状态定义模式**：

| 模式 | 示例 |
|------|------|
| `dp[i]` = 以第 `i` 个元素结尾的最优解 | 最长递增子序列 |
| `dp[i]` = 前 `i` 个元素的最优解 | 打家劫舍 |
| `dp[i][j]` = 区间 `[i, j]` 的最优解 | 最长回文子串 |
| `dp[i][j]` = 前 `i` 个物品、容量为 `j` 的最优解 | 0-1 背包 |

### 第二步：推导状态转移方程

找到 `dp[i]` 和更小规模子问题之间的关系。

状态转移方程就是把"当前状态"用"已知状态"表达出来的递推公式。

```text
dp[i] = f(dp[i-1], dp[i-2], ...)
```

> [!tip] 技巧
> 思考"最后一步"：到达状态 `i` 的最后一个决策是什么？从这个角度出发往往能找到转移方程。

### 第三步：确定初始化

递推的起点，也就是 base case。

通常是最简单的、不需要转移就能确定的状态。例如：

- `dp[0] = 0`：空集合/零元素的结果
- `dp[1] = 1`：只有一个元素的结果

> [!warning] 常见错误
> 初始化不正确是 DP 题 WA 的高频原因。特别注意 `dp[0]` 到底应该是 0 还是 1，以及数组大小是否多开了一位。

### 第四步：确定遍历顺序

保证在计算 `dp[i]` 时，它所依赖的子问题已经被计算过。

- 如果 `dp[i]` 依赖 `dp[i-1]`：正序遍历
- 如果 `dp[i]` 依赖 `dp[i+1]`：逆序遍历
- 二维 DP：需要同时考虑行和列的遍历方向

### 第五步：手动推演验证

用一个小规模样例手动填表，确认：

- 状态转移是否正确
- 初始化是否合理
- 最终答案在 `dp` 数组的哪个位置

## 入门例题：爬楼梯（LeetCode 70）

### 题目

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶。有多少种不同的方法可以爬到楼顶？

### 五步法解题

**第一步：定义状态**

`dp[i]` = 到达第 `i` 阶楼梯的方法数

**第二步：状态转移**

到达第 `i` 阶，只有两种可能：

- 从第 `i-1` 阶爬 1 步上来
- 从第 `i-2` 阶爬 2 步上来

所以：`dp[i] = dp[i-1] + dp[i-2]`

**第三步：初始化**

- `dp[0] = 1`：站在地面，算一种方法
- `dp[1] = 1`：到第 1 阶只有 1 种走法

**第四步：遍历顺序**

`dp[i]` 依赖 `dp[i-1]` 和 `dp[i-2]`，所以正序遍历。

**第五步：推演验证**

![爬楼梯状态转移|1210](dp_climbing_stairs.svg)

```text
dp[0] = 1
dp[1] = 1
dp[2] = dp[1] + dp[0] = 2
dp[3] = dp[2] + dp[1] = 3
dp[4] = dp[3] + dp[2] = 5
dp[5] = dp[4] + dp[3] = 8
```

和预期一致。

### 代码实现

```cpp
class Solution {
public:
    int climbStairs(int n) {
        if (n <= 2) return n;

        std::vector<int> dp(n + 1);
        dp[0] = 1;
        dp[1] = 1;

        for (int i = 2; i <= n; ++i) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }

        return dp[n];
    }
};
```

### 空间优化

由于 `dp[i]` 只依赖前两个值，可以用两个变量代替整个数组：

```cpp
class Solution {
public:
    int climbStairs(int n) {
        if (n <= 2) return n;

        int prev2 = 1, prev1 = 1;

        for (int i = 2; i <= n; ++i) {
            int curr = prev1 + prev2;
            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }
};
```

空间复杂度从 `O(n)` 降到 `O(1)`。

## 经典模型：0-1 背包

### 问题描述

有 `n` 个物品，每个物品有重量 `w[i]` 和价值 `v[i]`。背包容量为 `W`。每个物品只能选或不选（0-1），求能装下的最大价值。

### 五步法解题

**第一步：定义状态**

`dp[i][j]` = 考虑前 `i` 个物品，背包容量为 `j` 时的最大价值

**第二步：状态转移**

对于第 `i` 个物品，只有两种选择：

- **不选**：`dp[i][j] = dp[i-1][j]`
- **选**（前提是 `j >= w[i]`）：`dp[i][j] = dp[i-1][j - w[i]] + v[i]`

取最大值：

```text
dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i])
```

**第三步：初始化**

- `dp[0][j] = 0`：没有物品可选，价值为 0
- `dp[i][0] = 0`：容量为 0，装不下任何物品

**第四步：遍历顺序**

外层遍历物品（`i` 从 1 到 n），内层遍历容量（`j` 从 0 到 W）。

**第五步：推演验证**

![0-1 背包 DP 表格填充|1117](dp_knapsack_table.svg)

### 代码实现

```cpp
int knapsack(std::vector<int>& w, std::vector<int>& v, int W) {
    int n = w.size();
    std::vector<std::vector<int>> dp(n + 1, std::vector<int>(W + 1, 0));

    for (int i = 1; i <= n; ++i) {
        for (int j = 0; j <= W; ++j) {
            dp[i][j] = dp[i - 1][j];

            if (j >= w[i - 1]) {
                dp[i][j] = std::max(dp[i][j], dp[i - 1][j - w[i - 1]] + v[i - 1]);
            }
        }
    }

    return dp[n][W];
}
```

### 空间优化：滚动数组

观察转移方程，`dp[i][j]` 只依赖 `dp[i-1][...]`，即上一行。可以压缩为一维数组：

```cpp
int knapsack(std::vector<int>& w, std::vector<int>& v, int W) {
    int n = w.size();
    std::vector<int> dp(W + 1, 0);

    for (int i = 0; i < n; ++i) {
        for (int j = W; j >= w[i]; --j) {
            dp[j] = std::max(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    return dp[W];
}
```

> [!warning] 逆序遍历
> 一维优化时内层**必须逆序**遍历容量。如果正序遍历，`dp[j - w[i]]` 可能已经被本轮更新过，相当于物品被重复选取（变成完全背包）。

## 常见 DP 分类

### 线性 DP

状态沿一个维度线性递推。

| 题目 | 状态定义 | 转移方程 |
|------|---------|---------|
| 爬楼梯 | `dp[i]` = 到第 i 阶的方法数 | `dp[i] = dp[i-1] + dp[i-2]` |
| 打家劫舍 | `dp[i]` = 前 i 家能偷的最大金额 | `dp[i] = max(dp[i-1], dp[i-2] + nums[i])` |
| 最大子数组和 | `dp[i]` = 以 i 结尾的最大和 | `dp[i] = max(nums[i], dp[i-1] + nums[i])` |

### 背包 DP

在有限容量中做选择。

| 类型 | 物品使用 | 内层遍历 |
|------|---------|---------|
| 0-1 背包 | 每个只能选一次 | 逆序 |
| 完全背包 | 每个可选无限次 | 正序 |
| 多重背包 | 每个有数量限制 | 二进制优化 |

### 区间 DP

状态定义在区间 `[i, j]` 上。

```text
dp[i][j] = f(dp[i][k], dp[k+1][j])  对所有 i <= k < j
```

典型题目：最长回文子串、矩阵链乘法、戳气球。

### 树形 DP

在树结构上做 DP，通常用后序遍历（先处理子节点）。

典型题目：二叉树的最大路径和、打家劫舍 III。

### 状态压缩 DP

用二进制位表示状态集合，适用于元素数量较小（通常 <= 20）的问题。

典型题目：旅行商问题、最短 Hamilton 路径。

## 空间优化技巧

### 滚动数组

当 `dp[i]` 只依赖 `dp[i-1]` 时，只需要保留两行：

```cpp
std::vector<std::vector<int>> dp(2, std::vector<int>(m, 0));

for (int i = 1; i <= n; ++i) {
    for (int j = 0; j < m; ++j) {
        dp[i & 1][j] = f(dp[(i - 1) & 1][j], ...);
    }
}
```

### 一维压缩

进一步压缩为一维数组，但要注意遍历顺序：

- 依赖正上方和左上方 → **逆序**遍历（0-1 背包）
- 依赖正上方和正左方 → **正序**遍历（完全背包）

### 变量代替数组

如果只依赖前 1~2 个状态，直接用变量：

```cpp
int prev2 = base0, prev1 = base1;

for (int i = 2; i <= n; ++i) {
    int curr = f(prev1, prev2);
    prev2 = prev1;
    prev1 = curr;
}
```

## DP 常见错误与调试

### 1. 状态定义模糊

`dp[i]` 到底是"以 i 结尾"还是"前 i 个"？两种定义会导致完全不同的转移方程。

### 2. 初始化遗漏

忘记初始化 `dp[0]` 或者 `dp[1]`，或者数组大小没有多开一位。

### 3. 遍历顺序错误

一维背包正序/逆序搞反，导致物品被重复选取或者使用了未计算的状态。

### 4. 返回值位置错误

有时答案不在 `dp[n]`，而是需要遍历整个 `dp` 数组取最大值（比如最长递增子序列）。

### 调试方法

把 `dp` 数组打印出来，对照手动推演的结果逐行检查：

```cpp
for (int i = 0; i <= n; ++i) {
    std::cout << "dp[" << i << "] = " << dp[i] << "\n";
}
```

## 复杂度分析

| 因素 | 复杂度 |
|------|--------|
| 时间 | `O(状态数量 × 每个状态的转移代价)` |
| 空间 | `O(状态数量)`，可通过滚动数组/变量优化 |

以 0-1 背包为例：

- 状态数量：`n × W`
- 每个状态转移代价：`O(1)`
- 时间复杂度：`O(nW)`
- 空间复杂度：优化后 `O(W)`

## 总结

动态规划的本质是**用空间换时间**，通过存储子问题的结果来避免重复计算。

核心记忆点：

1. **适用条件**：最优子结构 + 重叠子问题
2. **两种实现**：自顶向下（记忆化）、自底向上（递推）
3. **解题五步法**：定义状态 → 状态转移 → 初始化 → 遍历顺序 → 推演验证
4. **空间优化**：滚动数组 → 一维压缩 → 变量代替
5. **调试方法**：打印 dp 数组，对照手动推演

> [!quote]
> "那些忘记过去的人注定要重蹈覆辙。" —— 这就是动态规划的哲学：记住已经解决的问题，不要重复劳动。
