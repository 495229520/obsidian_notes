# 14.16 常见排序算法与 STL 排序

## 排序的本质

排序解决的是：把一组元素按照某种规则重新排列。

在 C++ 实战中，优先使用标准库：

- 全量排序：`std::sort`
- 稳定排序：`std::stable_sort`
- 只要前 `k` 个：`std::partial_sort`
- 只要第 `k` 小 / 第 `k` 大：`std::nth_element`
- 动态维护最大 / 最小值：`std::priority_queue` 或堆算法

手写快排、归并、堆排序、计数排序，主要用于：

1. 理解底层思想
2. 面试推导边界
3. 特殊场景下控制复杂度、稳定性或空间

> [!tip]
> 选择排序算法时，不要只看时间复杂度，还要看稳定性、额外空间、数据范围、是否接近有序、是否只需要局部排名。

---

## 常见排序算法对比

| 算法 | 核心思想 | 平均时间 | 最坏时间 | 额外空间 | 是否稳定 | 适合场景 |
|------|----------|----------|----------|----------|----------|----------|
| 快速排序 | 选基准值，分成小于 / 大于基准的两边 | `O(nlogn)` | `O(n²)` | `O(logn)` | 不稳定 | 通用比较排序、理解分治 |
| 归并排序 | 拆分数组，再合并有序区间 | `O(nlogn)` | `O(nlogn)` | `O(n)` | 稳定 | 需要稳定性、链表排序 |
| 堆排序 | 建大根堆，最大值反复进入有序区 | `O(nlogn)` | `O(nlogn)` | `O(1)` | 不稳定 | 空间敏感、理解堆结构 |
| 计数排序 | 统计每个值出现次数，再按值域回填 | `O(n + k)` | `O(n + k)` | `O(k)` | 可稳定 | 整数且值域较小 |
| `std::sort` | 标准库排序，通常采用 introsort 思想 | `O(nlogn)` | `O(nlogn)` | 实现相关 | 不稳定 | 工程中最常用 |
| `std::stable_sort` | 标准库稳定排序，通常基于归并思想 | `O(nlogn)` | 通常 `O(nlogn)` | 通常 `O(n)` | 稳定 | 相等元素要保留原顺序 |

这里的 `k` 表示值域大小：`maxValue - minValue + 1`。

## 快速排序

### 核心思想

快速排序的核心操作是 **partition（分区）**：

1. 选择一个基准值 `pivot`
2. 把小于 `pivot` 的元素放到左边
3. 把大于 `pivot` 的元素放到右边
4. 对左右区间继续递归排序

![[5_14_16_1.svg|900]]

快排不是每一轮都把数组整体排好，而是让当前区间变得更“有方向”：小的往左，大的往右。

### 适用场景

- 想理解分治和 partition 思想
- 面试中需要手写排序
- 数据在内存中，且不要求稳定性
- 工程代码中通常直接使用 `std::sort`

### 代码模板：Hoare 分区法

```cpp
#include <vector>
#include <utility>

void quickSort(std::vector<int>& nums, int left, int right) {
    if (left >= right) return;

    int i = left - 1;
    int j = right + 1;
    int pivot = nums[left + (right - left) / 2];

    while (i < j) {
        do ++i; while (nums[i] < pivot);
        do --j; while (nums[j] > pivot);
        if (i < j) std::swap(nums[i], nums[j]);
    }

    quickSort(nums, left, j);
    quickSort(nums, j + 1, right);
}

std::vector<int> sortArray(std::vector<int> nums) {
    if (!nums.empty()) {
        quickSort(nums, 0, static_cast<int>(nums.size()) - 1);
    }
    return nums;
}
```

### 关键边界

这份模板使用的是 **Hoare 分区法**，它和“把 `pivot` 放到最终位置”的写法不同。

| 点 | 含义 |
|----|------|
| `pivot` | 只是基准值，不代表最终下标 |
| `j` | 分区结束后的分割边界 |
| 左递归 | `[left, j]` |
| 右递归 | `[j + 1, right]` |

分区结束后能保证：

```text
[left, j]       整体 <= pivot
[j + 1, right]  整体 >= pivot
```

注意：这只表示元素被分到正确的一侧，不表示两侧内部已经有序。

### 为什么递归左边是 `[left, j]`

一次 partition 后可能得到：

```text
left     j  j+1        right
[3, 1, 2 | 5, 9, 8, 7]
```

此时：

- `j` 左边是“小的一侧”
- `j + 1` 右边是“大的一侧”
- 后续只需要分别处理两个区间

所以递归写成：

```cpp
quickSort(nums, left, j);
quickSort(nums, j + 1, right);
```

如果写成 `quickSort(nums, left, j - 1)`，可能漏掉 `j` 位置；如果写成 `quickSort(nums, j, right)`，又容易让边界不收缩。

### 指针初始化与比较条件

```cpp
int i = left - 1;
int j = right + 1;
```

原因是后面使用 `do...while`：

```cpp
do ++i; while (nums[i] < pivot);
do --j; while (nums[j] > pivot);
```

所以第一次移动后：

- `i` 正好来到 `left`
- `j` 正好来到 `right`

比较条件必须使用 `<` 和 `>`，不要写成 `<=` 和 `>=`。

| 写法 | 结果 |
|------|------|
| `nums[i] < pivot` | `i` 停在第一个 `>= pivot` 的位置 |
| `nums[j] > pivot` | `j` 停在第一个 `<= pivot` 的位置 |

如果跳过等于 `pivot` 的元素，在 `[5, 5, 5]` 这类重复值数组中，指针可能一路越界。

### 为什么循环条件是 `i < j`

```cpp
while (i < j) {
    do ++i; while (nums[i] < pivot);
    do --j; while (nums[j] > pivot);
    if (i < j) std::swap(nums[i], nums[j]);
}
```

- `i < j`：左右指针尚未相遇，可能还有元素需要交换
- `i >= j`：指针相遇或交叉，分区结束

> [!important] Hoare 模板记忆方式
> 返回边界是 `j`，递归区间是 `[left, j]` 和 `[j + 1, right]`；外层循环只在 `i < j` 时继续。

### 特点

- 平均速度快，常数小
- 原地排序，额外空间主要来自递归栈
- 不稳定
- `pivot` 选择很差时可能退化成 `O(n²)`

> [!warning]
> 学习时可以手写快排；工程代码中优先使用 `std::sort`，标准库已经处理了很多边界和性能问题。

## 归并排序

### 核心思想

归并排序是典型的 **分治算法**：

1. 把数组递归拆成左右两半
2. 拆到单个元素时，单元素天然有序
3. 回溯时把两个有序区间合并成更大的有序区间

![[5_14_16_2.svg|903]]

归并排序的关键不是“交换”，而是“合并两个有序序列”。

### 适用场景

- 需要稳定排序
- 需要稳定的 `O(nlogn)` 最坏复杂度
- 链表排序
- 外部排序：数据太大，不能一次全部放进内存

### 代码模板

```cpp
#include <vector>

void mergeSort(std::vector<int>& nums, int left, int right) {
    if (left >= right) return;

    int mid = left + (right - left) / 2;
    mergeSort(nums, left, mid);
    mergeSort(nums, mid + 1, right);

    std::vector<int> temp;
    int i = left;
    int j = mid + 1;

    while (i <= mid && j <= right) {
        if (nums[i] <= nums[j]) temp.push_back(nums[i++]);
        else temp.push_back(nums[j++]);
    }

    while (i <= mid) temp.push_back(nums[i++]);
    while (j <= right) temp.push_back(nums[j++]);

    for (int k = 0; k < static_cast<int>(temp.size()); ++k) {
        nums[left + k] = temp[k];
    }
}
```

### 关键步骤

| 步骤 | 代码 | 含义 |
|------|------|------|
| 递归终止 | `if (left >= right) return;` | 空区间或单元素天然有序 |
| 取中点 | `left + (right - left) / 2` | 避免 `left + right` 溢出 |
| 拆左边 | `[left, mid]` | 排好左半区间 |
| 拆右边 | `[mid + 1, right]` | 排好右半区间 |
| 合并 | `temp.push_back(...)` | 把两个有序区间合成一个有序区间 |
| 拷回 | `nums[left + k] = temp[k]` | 覆盖原数组对应区间 |

### 为什么 `<=` 能保证稳定

合并时写的是：

```cpp
if (nums[i] <= nums[j])
```

当左右两边当前元素相等时，优先取左边元素。左边元素在原数组中本来更靠前，因此相等元素的原始相对顺序不会改变。

这就是归并排序能做到稳定排序的关键。

### 为什么要处理剩余元素

主循环结束时，说明左半边或右半边至少有一边已经用完：

```cpp
while (i <= mid) temp.push_back(nums[i++]);
while (j <= right) temp.push_back(nums[j++]);
```

剩下的那一边本来已经有序，直接追加即可。

例如：

```text
左半边：[1, 4, 9]
右半边：[2, 3]

比较后先得到：[1, 2, 3]
右半边用完，追加左半边剩余：[4, 9]
最终结果：[1, 2, 3, 4, 9]
```

> [!important] 归并排序的核心
> 递归负责让左右两半各自有序，合并负责把两个有序区间变成一个更大的有序区间。

### 特点

- 最好、平均、最坏时间复杂度都是 `O(nlogn)`
- 稳定排序
- 需要 `O(n)` 额外空间
- 链表排序常用归并，因为链表合并不需要大量移动元素

## 堆排序

### 核心思想

堆排序利用 **大根堆** 的性质：堆顶永远是当前最大值。

1. 把数组调整成大根堆
2. 交换堆顶和当前堆尾
3. 缩小堆的范围
4. 对剩余堆重新调整
5. 重复直到数组有序

![[5_14_16_3.svg|900]]

### 适用场景

- 希望最坏复杂度稳定在 `O(nlogn)`
- 额外空间要求严格，希望原地排序
- 想理解 `priority_queue`、`make_heap`、`pop_heap` 的底层思想

### 使用 STL 写堆排序

```cpp
#include <algorithm>
#include <vector>

void heapSort(std::vector<int>& nums) {
    std::make_heap(nums.begin(), nums.end());
    std::sort_heap(nums.begin(), nums.end());
}
```

`std::make_heap` 默认建大根堆，`std::sort_heap` 会反复把堆顶最大值放到末尾，最终得到升序结果。

### 手动弹出最大值

```cpp
#include <algorithm>
#include <vector>

std::vector<int> nums{4, 1, 7, 3, 9, 2};

std::make_heap(nums.begin(), nums.end());
std::pop_heap(nums.begin(), nums.end());

int maxValue = nums.back();
nums.pop_back();
```

执行 `pop_heap` 后：

- 最大值被交换到区间末尾
- `[begin, end - 1)` 仍然保持堆结构
- 真正删除最大值，需要再执行 `pop_back`

### 特点

- 最坏时间复杂度也是 `O(nlogn)`
- 原地排序，额外空间 `O(1)`
- 不稳定
- 缓存局部性通常不如快排，实际速度未必比 `std::sort` 快

## 计数排序

### 核心思想

计数排序不是比较排序。它利用元素值可以映射到数组下标这一点：

1. 扫描原数组，统计每个值出现了几次
2. 按值从小到大遍历计数数组
3. 每个值出现几次，就回填几次

![[5_14_16_4.svg|900]]

例如：

```text
原数组：[4, 2, 2, 8, 3, 3, 1]
计数：  1 出现 1 次，2 出现 2 次，3 出现 2 次，4 出现 1 次，8 出现 1 次
结果：  [1, 2, 2, 3, 3, 4, 8]
```

### 适用场景

计数排序适合同时满足这些条件的场景：

- 元素是整数，或可以离散映射为整数下标
- 值域比较小
- 需要线性时间排序
- 可以接受 `O(k)` 额外空间

不适合：

- 值域特别大，例如只有 3 个数却分布在 `[-1e9, 1e9]`
- 元素不能直接映射到有限整数下标
- 需要通用比较规则的对象排序

### 简单版代码：非负整数

适用于 `nums[i] >= 0` 且最大值不大的情况。

```cpp
#include <algorithm>
#include <vector>

std::vector<int> countingSortNonNegative(const std::vector<int>& nums) {
    if (nums.empty()) return {};

    int maxValue = *std::max_element(nums.begin(), nums.end());
    std::vector<int> count(maxValue + 1, 0);

    for (int value : nums) {
        ++count[value];
    }

    std::vector<int> result;
    result.reserve(nums.size());

    for (int value = 0; value <= maxValue; ++value) {
        for (int times = 0; times < count[value]; ++times) {
            result.push_back(value);
        }
    }

    return result;
}
```

### 支持负数：使用偏移量

如果数组中有负数，可以用最小值作为偏移量：

```cpp
#include <algorithm>
#include <vector>

std::vector<int> countingSort(const std::vector<int>& nums) {
    if (nums.empty()) return {};

    auto [minIt, maxIt] = std::minmax_element(nums.begin(), nums.end());
    int minValue = *minIt;
    int maxValue = *maxIt;
    int range = maxValue - minValue + 1;

    std::vector<int> count(range, 0);

    for (int value : nums) {
        ++count[value - minValue];
    }

    std::vector<int> result;
    result.reserve(nums.size());

    for (int index = 0; index < range; ++index) {
        int value = index + minValue;
        for (int times = 0; times < count[index]; ++times) {
            result.push_back(value);
        }
    }

    return result;
}
```

### 稳定版思路

如果排序的是对象，并且要保持相同 key 的原始顺序，需要使用“前缀和 + 逆序回填”。

步骤：

1. `count[x]` 记录 key 为 `x` 的元素数量
2. 对 `count` 做前缀和，让 `count[x]` 表示 key `<= x` 的元素总数
3. 从右向左扫描原数组
4. 根据 `count[key] - 1` 找到元素最终位置
5. 放入结果数组后，让 `count[key]--`

为什么从右向左？

- 相同 key 的元素，后出现的先放到后面
- 先出现的后处理，最终会落在更靠前的位置
- 因此可以保持原始相对顺序

### 复杂度

| 项 | 复杂度 |
|----|--------|
| 时间复杂度 | `O(n + k)` |
| 空间复杂度 | `O(k)` |
| `n` | 元素数量 |
| `k` | 值域大小：`maxValue - minValue + 1` |

> [!warning]
> 计数排序快的前提是 `k` 不大。如果值域远大于数据量，计数数组会浪费大量空间，反而不如 `std::sort`。

## STL 中现成的排序工具

### `std::sort`

`std::sort` 是最常用的排序函数，定义在 `<algorithm>` 中。
基础用法也可以回看 [[14.11 算法]]，这里更关注排序工具该怎么选。

```cpp
#include <algorithm>
#include <functional>
#include <vector>

std::vector<int> nums{5, 1, 9, 3, 7};

std::sort(nums.begin(), nums.end());
std::sort(nums.begin(), nums.end(), std::greater<int>{});
```

`std::sort` 要求传入**随机访问迭代器**，适合：

- `std::vector`
- `std::array`
- `std::deque`
- 普通数组

不适合 `std::list`，因为链表不支持随机访问。`std::list` 应该使用自己的成员函数：

```cpp
#include <list>

std::list<int> nums{5, 1, 9, 3, 7};
nums.sort();
```

相关容器可以回看 [[14.1 vector]]、[[14.3 list及其与其他结构的区别]]。

### 自定义排序规则

```cpp
#include <algorithm>
#include <string>
#include <vector>

struct Student {
    std::string name;
    int score;
};

std::vector<Student> students{
    {"Tom", 90},
    {"Alice", 95},
    {"Bob", 90}
};

std::sort(students.begin(), students.end(),
    [](const Student& left, const Student& right) {
        if (left.score != right.score) return left.score > right.score;
        return left.name < right.name;
    });
```

> [!warning] 比较函数必须是严格弱序
> 比较函数表达的是“谁应该排在前面”，不能写成 `a <= b`。对于相等元素，比较函数应该返回 `false`。

### `std::stable_sort`

如果相等元素的原始顺序有意义，就用稳定排序。

```cpp
std::stable_sort(students.begin(), students.end(),
    [](const Student& left, const Student& right) {
        return left.score > right.score;
    });
```

例如学生成绩相同，希望保持原始报名顺序，这时 `std::stable_sort` 比 `std::sort` 更合适。

### `std::partial_sort`

如果只需要“前 `k` 个最小值，并且这 `k` 个内部有序”，使用 `std::partial_sort`。

```cpp
#include <algorithm>
#include <vector>

std::vector<int> nums{9, 1, 7, 3, 5, 2, 8};
int k = 3;

std::partial_sort(nums.begin(), nums.begin() + k, nums.end());
```

结果形态：

```text
前 3 个元素是最小的 3 个，并且已经升序：1 2 3 ...
后面的元素不保证有序。
```

### `std::nth_element`

如果只关心第 `k` 小或第 `k` 大，不要求左右两边内部有序，使用 `std::nth_element`。

```cpp
#include <algorithm>
#include <vector>

std::vector<int> nums{9, 1, 7, 3, 5, 2, 8};
int k = 3;

std::nth_element(nums.begin(), nums.begin() + k - 1, nums.end());
int kthSmallest = nums[k - 1];
```

求第 `k` 大可以配合 `std::greater<int>{}`：

```cpp
std::nth_element(nums.begin(), nums.begin() + k - 1, nums.end(), std::greater<int>{});
int kthLargest = nums[k - 1];
```

这类问题也可以联系 [[215. 数组中的第K个最大元素]]。

### 堆相关算法

STL 的堆算法适合“动态取最大值 / 最小值”的场景。

| 函数 | 作用 |
|------|------|
| `make_heap(first, last)` | 把区间建成堆 |
| `push_heap(first, last)` | 新元素已放在末尾，将其上浮到正确位置 |
| `pop_heap(first, last)` | 把堆顶放到末尾，并调整剩余堆 |
| `sort_heap(first, last)` | 对堆区间排序 |

如果只是不断插入元素、取最大值，更常用的是 [[14.8 容器适配器]] 中的 `priority_queue`。

## 怎么选

| 需求 | 推荐 |
|------|------|
| 普通数组或 `vector` 全量排序 | `std::sort` |
| 相同元素要保持原始顺序 | `std::stable_sort` |
| 只要前 `k` 个，而且前 `k` 个要有序 | `std::partial_sort` |
| 只要第 `k` 小 / 第 `k` 大 | `std::nth_element` |
| 数据持续插入，并且经常取最大 / 最小 | `priority_queue` 或堆算法 |
| 整数排序，值域很小 | 计数排序 |
| 要稳定且最坏复杂度稳定 | 归并排序 |
| 要原地排序且最坏复杂度稳定 | 堆排序 |
| 学习排序思想、准备面试 | 手写快排、归并、堆排序、计数排序 |

一句话总结：

1. **工程中优先 `std::sort`**
2. **要稳定性用 `std::stable_sort`**
3. **只要局部排名用 `partial_sort` 或 `nth_element`**
4. **要动态维护最大 / 最小值用堆**
5. **整数且值域小，可以考虑计数排序**

## 关键要点

- 快排靠 partition 分区，平均快，但不稳定，最坏可能 `O(n²)`。
- 归并靠“先拆后合”，稳定且最坏 `O(nlogn)`，但需要 `O(n)` 空间。
- 堆排序靠大根堆让最大值不断进入有序区，原地但不稳定。
- 计数排序不是比较排序，适合整数小值域，复杂度是 `O(n + k)`。
- STL 排序工具已经覆盖大多数工程场景，优先使用标准库。
