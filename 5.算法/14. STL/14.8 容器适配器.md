# 容器适配器

## 简介

**标准容器 - 容器适配器**

如何理解这个适配器？

1. 适配器底层没有自己的数据结构，它是另外一个容器的封装，它的方法全部由底层依赖的容器来实现
2. 没有实现自己的迭代器

## 模版定义

```cpp
template<typename T, typename Container = deque<T>>
class Stack
{
public:
    // 入栈
    void push(const T& val) { con.push_back(val); }
    // 出栈
    void pop() { con.pop_back(); }
    // 返回栈顶元素
    T top() const { return con.back(); }
private:
    Container con;
};
```

---

## 基本操作

### stack 栈

| 操作 | 说明 |
|------|------|
| `push` | 入栈 |
| `pop` | 出栈 |
| `top` | 查看栈顶元素 |
| `empty` | 判断栈空 |
| `size` | 返回元素个数 |

**stack 底层依赖 deque**，因为：
1. vector 初始内存使用效率太低了，0-1-2-4-8；deque 直接 4096 / sizeof(T)
2. 对于 queue 来说，需要支持尾部插入，头部删除，O(1)。如果 queue 依赖 vector，其出队效率很低
3. vector 需要大片的连续内存，而 deque 只需要分段的连续内存，deque 对于内存的利用率更好

### queue 队列

| 操作 | 说明 |
|------|------|
| `push` | 入队 |
| `pop` | 出队 |
| `front` | 查看队头元素 |
| `empty` | 判断队空 |
| `size` | 返回元素个数 |

**stack 底层依赖 deque**，原理同上

### priority_queue 优先级队列

| 操作 | 说明 |
|------|------|
| `push` | 入队 |
| `pop` | 出队 |
| `top` | 查看队头元素 |
| `empty` | 判断队空 |
| `size` | 返回元素个数 |

**priority_queue 底层依赖 vector**，底层默认是**大根堆**

底层默认把数据组成一个大根堆结构，通常在一个内存的连续数组上构建一个大根堆或者小根堆。因为堆中每一个节点跟它左右孩子之间的关系是通过下标实现的：
- 左孩子：`2 * i + 1`
- 右孩子：`2 * i + 2`
- 父节点：`(i - 1) / 2`

## 代码案例

```cpp
// 优先级队列
priority_queue<int> pque;
for (int i = 0; i < 20; ++i)
{
    pque.push(rand() % 100 + 1);
}
cout << pque.size() << endl;

while (!pque.empty())
{
    cout << pque.top() << " ";
    pque.pop();
}

cout << endl;
cout << "----------------" << endl;

// 普通队列
queue<int> que;
for (int i = 0; i < 20; ++i)
{
    que.push(rand() % 100 + 1);
}
cout << que.size() << endl;

while (!que.empty())
{
    cout << que.front() << " ";
    que.pop();
}

cout << endl;
cout << "----------------" << endl;

// 栈容器适配器常用的方法：
// push：入栈   pop：出栈   top：查看栈顶元素   empty：判断栈空   size：返回元素个数
stack<int> s1;
for (int i = 0; i < 20; ++i)
{
    s1.push(rand() % 100 + 1);
}
cout << s1.size() << endl;

while (!s1.empty())
{
    cout << s1.top() << " ";
    s1.pop();
}
```

## 容器适配器对比

| 适配器 | 底层容器 | 特点 | 应用场景 |
|--------|----------|------|----------|
| `stack` | deque | LIFO 后进先出 | 函数调用栈、表达式求值、括号匹配 |
| `queue` | deque | FIFO 先进先出 | 任务队列、BFS 广度优先搜索 |
| `priority_queue` | vector | 按优先级排序（堆） | 任务调度、Dijkstra 算法、事件驱动模拟 |

---

## 关键要点

1. **容器适配器不是容器**：它们是对底层容器的封装，提供特定的接口
2. **没有迭代器**：容器适配器不支持迭代器遍历
3. **底层容器可配置**：可以指定底层使用的容器类型

```cpp
// 使用 vector 作为 stack 的底层容器
stack<int, vector<int>> s;

// 使用 list 作为 queue 的底层容器
queue<int, list<int>> q;
```
