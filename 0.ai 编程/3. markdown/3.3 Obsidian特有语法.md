# Obsidian 特有语法

Obsidian 是一款基于 Markdown 的知识管理和笔记软件，它在标准 Markdown 基础上扩展了许多实用功能。本笔记介绍 Obsidian 特有的语法和功能。

> **前置知识**：建议先阅读 [[3.1 Markdown基础语法]] 和 [[3.2 Markdown高级语法]]。

---

## 1. 内部链接 (Wiki-Links)

Obsidian 最核心的功能是双向链接，使用双方括号 `[[]]` 创建内部链接。

### 1.1 基本链接

```markdown
链接到笔记：[[笔记名称]]

链接到笔记并显示别名：[[笔记名称|显示的文本]]

示例：
- [[3.1 Markdown基础语法]]
- [[3.1 Markdown基础语法|基础语法入门]]
```

### 1.2 链接到标题

```markdown
链接到特定标题：[[笔记名称#标题名称]]

链接到当前笔记的标题：[[#标题名称]]

示例：
- [[3.2 Markdown高级语法#4. 流程图与图表 (Diagrams)]]
- [[#1. 内部链接 (Wiki-Links)]]
```

### 1.3 链接到块

Obsidian 支持链接到特定的段落或块：

```markdown
链接到块：[[笔记名称#^block-id]]

在目标位置添加块 ID：
这是一个重要的段落。 ^important-block

然后可以这样引用：
参见 [[笔记名称#^important-block]]
```

### 1.4 链接语法对比

| 语法 | 效果 | 说明 |
|------|------|------|
| `[[笔记]]` | 链接到笔记 | 最基本的链接方式 |
| `[[笔记\|别名]]` | 显示别名 | 自定义显示文本 |
| `[[笔记#标题]]` | 链接到标题 | 精确定位 |
| `[[笔记#^id]]` | 链接到块 | 最精确定位 |
| `[[#标题]]` | 当前笔记标题 | 页内跳转 |

## 2. 嵌入内容 (Embeds)

使用 `![[]]` 语法可以将其他内容嵌入到当前笔记中。

### 2.1 嵌入笔记

```markdown
嵌入整个笔记：
![[笔记名称]]

嵌入笔记的特定标题下的内容：
![[笔记名称#标题]]

嵌入特定块：
![[笔记名称#^block-id]]
```

### 2.2 嵌入图片

```markdown
基本嵌入：
![[image.png]]

调整图片大小（宽度）：
![[image.png|300]]

调整图片大小（宽x高）：
![[image.png|300x200]]
```

### 2.3 嵌入其他文件

```markdown
嵌入 PDF（可指定页码）：
![[document.pdf]]
![[document.pdf#page=5]]

嵌入音频：
![[audio.mp3]]

嵌入视频：
![[video.mp4]]
```

## 3. 标签 (Tags)

### 3.1 基本标签

```markdown
在笔记中添加标签：#标签名

嵌套标签：#parent/child/grandchild

示例：
#编程 #编程/C++ #编程/Python
```

### 3.2 标签使用建议

| 标签类型 | 示例 | 用途 |
|----------|------|------|
| 主题标签 | `#网络编程` | 内容分类 |
| 状态标签 | `#待完成` `#已完成` | 进度追踪 |
| 类型标签 | `#概念` `#教程` `#参考` | 笔记类型 |
| 嵌套标签 | `#项目/个人/学习` | 层级分类 |

## 4. Frontmatter (YAML)

在笔记开头使用 YAML 格式添加元数据：

```yaml
---
title: 笔记标题
date: 2024-01-15
tags:
  - 编程
  - C++
aliases:
  - 别名1
  - 别名2
cssclass: custom-class
---
```

### 常用 Frontmatter 字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 笔记标题 | `title: "我的笔记"` |
| `date` | 创建日期 | `date: 2024-01-15` |
| `tags` | 标签列表 | `tags: [tag1, tag2]` |
| `aliases` | 笔记别名（可用于链接） | `aliases: [别名]` |
| `cssclass` | 自定义 CSS 类 | `cssclass: wide-page` |
| `publish` | 是否发布 | `publish: true` |

## 5. Callouts (提示框)

Obsidian 支持多种样式的提示框：

### 5.1 基本语法

```markdown
> [!note]
> 这是一个普通提示。

> [!tip] 自定义标题
> 这是一个带自定义标题的提示。

> [!warning]- 可折叠的警告
> 点击标题可以展开/折叠此内容。
```

### 5.2 Callout 类型

| 类型 | 别名 | 用途 |
|------|------|------|
| `note` | - | 普通提示 |
| `tip` | `hint`, `important` | 提示/建议 |
| `info` | - | 信息说明 |
| `warning` | `caution`, `attention` | 警告 |
| `danger` | `error` | 危险/错误 |
| `success` | `check`, `done` | 成功/完成 |
| `question` | `help`, `faq` | 问题/疑问 |
| `quote` | `cite` | 引用 |
| `example` | - | 示例 |
| `bug` | - | Bug 说明 |
| `abstract` | `summary`, `tldr` | 摘要 |

### 5.3 使用示例

> [!tip] 编程建议
> 编写代码时要注意边界条件的处理。

> [!warning] 注意
> 使用指针前一定要检查是否为空。

> [!example] 代码示例
> ```cpp
> int* ptr = nullptr;
> if (ptr != nullptr) {
>     // 安全使用指针
> }
> ```

## 6. 查询与 Dataview

### 6.1 内置搜索

Obsidian 的搜索支持多种运算符：

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `tag:` | 搜索标签 | `tag:#编程` |
| `path:` | 搜索路径 | `path:C++基础` |
| `file:` | 搜索文件名 | `file:指针` |
| `line:` | 搜索特定行内容 | `line:(TODO)` |
| `section:` | 搜索标题 | `section:总结` |

### 6.2 Dataview 插件（需安装）

Dataview 是一个强大的数据查询插件：

````markdown
```dataview
TABLE file.ctime as "创建时间", file.mtime as "修改时间"
FROM "C++基础"
WHERE contains(file.name, "指针")
SORT file.mtime DESC
```
````

常用 Dataview 查询类型：
- `TABLE`：表格形式
- `LIST`：列表形式
- `TASK`：任务列表

## 7. 模板 (Templates)

### 7.1 模板变量

在模板文件中可以使用以下变量：

| 变量 | 说明 | 示例输出 |
|------|------|----------|
| `{{title}}` | 笔记标题 | `我的笔记` |
| `{{date}}` | 当前日期 | `2024-01-15` |
| `{{time}}` | 当前时间 | `14:30` |
| `{{date:格式}}` | 自定义日期格式 | `{{date:YYYY-MM-DD}}` |

### 7.2 模板示例

```markdown
---
date: {{date:YYYY-MM-DD}}
tags: []
---

# {{title}}

## 概述



## 主要内容



## 总结



---
创建时间: {{date:YYYY-MM-DD HH:mm}}
```

## 总结

| 功能 | 语法 | 说明 |
|------|------|------|
| 内部链接 | `[[笔记]]` | 双向链接核心功能 |
| 嵌入 | `![[内容]]` | 嵌入笔记/图片/文件 |
| 标签 | `#标签` | 支持嵌套标签 |
| Frontmatter | `---yaml---` | 笔记元数据 |
| Callouts | `> [!type]` | 彩色提示框 |
| 高亮 | `==文本==` | 文本高亮 |
| 注释 | `%%注释%%` | 隐藏内容 |
| 块引用 | `^block-id` | 精确引用 |

> **最佳实践**：
> - 善用双向链接建立知识网络
> - 使用标签和 Frontmatter 组织笔记
> - 利用 Callouts 突出重要信息
> - 使用模板提高效率
