# GitHub SSH 多账号推送

当一台电脑同时使用多个 GitHub 账号时，建议为每个账号配置独立的 SSH key 和 Host alias，避免推送时选错身份。

## 基本思路

- 每个账号使用一把独立的 `ed25519` key
- 在 `~/.ssh/config` 中用不同的 `Host` 区分账号
- 仓库的 `origin` 使用对应的 Host alias
- 使用 `ssh -T HostAlias` 验证当前认证到哪个 GitHub 账号

## 示例配置

```sshconfig
Host github-hosendovebelva-boop
  HostName github.com
  User git
  IdentityFile ~/.ssh/hosendovebelva_boop_ed25519
  IdentitiesOnly yes
  AddKeysToAgent yes
  UseKeychain yes
```

仓库 remote 可以写成：

```bash
git remote set-url origin git@github-hosendovebelva-boop:495229520/obsidian_notes.git
```

验证身份：

```bash
ssh -T github-hosendovebelva-boop
```

如果输出中出现 `Hi hosendovebelva-boop!`，说明当前 Host alias 已经使用目标账号认证。

## 注意事项

- `user.name` 和 `user.email` 决定提交作者。
- SSH key 决定 GitHub 识别到的推送账号。
- 如果仓库属于另一个账号，需要目标推送账号拥有该仓库的写权限。
