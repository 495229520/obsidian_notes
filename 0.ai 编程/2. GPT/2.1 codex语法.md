# OpenAI Codex CLI 常用命令

Codex CLI 是 OpenAI 推出的命令行 AI 编程助手，专注于代码生成、审查和优化任务。

---

## 基础配置命令

| 命令 | 说明 |
|------|------|
| `/model` | 选择模型 |
| `/approvals` | 提供给 AI 权限，设置哪些操作不需要用户确认 |

---

## 代码审查 `/review`

审查当前代码，让 Codex 自动检查当前文件夹中的代码。

**输出内容包括：**
- bug（代码缺陷）
- 风险点（潜在问题）
- 性能问题
- 风格建议
- 更优写法

> **提示**：此命令非常适合重构前的代码审查，帮助发现潜在问题。

---

## 会话管理

| 命令 | 说明 |
|------|------|
| `/new` | 强制启动新的独立话题（清空当前上下文） |
| `/resume` | 恢复之前的对话 |

---

## 项目初始化 `/init`

创建一个 `AGENTS.md` 文件，用于自动生成一份「如何让 Codex 工作」的说明文档。

**包括：**
- 项目描述
- 文件结构
- 编码风格要求
- 任务原则

> **说明**：`AGENTS.md` 类似于 Claude Code 的 `CLAUDE.md`，是项目级别的 AI 配置文件。

---

## 上下文与历史管理

| 命令 | 说明 |
|------|------|
| `/compact` | 压缩会话历史（释放上下文空间） |
| `/undo` | 撤回一次编辑 |

---

## 代码编辑 `/edit`

用途：让 Codex 直接修改选中文件。

**示例：**
```bash
/edit 把这个函数优化到 O(n log n)
```

> **提示**：可以指定具体的优化目标，如时间复杂度、空间复杂度等。

## 关键概念补充

### AGENTS.md 文件
`AGENTS.md` 是 Codex CLI 的项目配置文件，作用类似于 [[1.1 claude code语法#CLAUDE.md 文件|Claude Code 的 CLAUDE.md]]。它帮助 AI 理解：
- 项目的整体架构
- 代码风格偏好
- 测试和构建流程
- 特定的编码约定

### 代码审查最佳实践
使用 `/review` 命令时，建议：
1. 在提交代码前进行审查
2. 关注安全相关的风险点提示
3. 根据性能建议优化热点代码
4. 保持代码风格的一致性

### 时间复杂度优化
使用 `/edit` 进行算法优化时，常见的复杂度目标：
- **O(1)**：常数时间，如哈希表查找
- **O(log n)**：对数时间，如二分查找
- **O(n)**：线性时间，如数组遍历
- **O(n log n)**：如高效排序算法
- **O(n²)**：如简单嵌套循环

| 上下文压缩 | `/compact` | `/compact` |
| 会话恢复 | `resume` | `/resume` |
| 文档查询 | 对话式 | `/doc` |
| 代码解释 | 对话式 | `/explain` |
| 测试生成 | 对话式 | `/test` |
