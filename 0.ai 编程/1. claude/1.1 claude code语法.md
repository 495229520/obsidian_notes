# Claude Code 常用命令

Claude Code 是 Anthropic 推出的官方命令行 AI 编程助手，支持代码生成、重构、调试等任务。

配置文件：C:\Users\Administrator\AppData\Roaming\Claude\claude_desktop_config.json

---

## 基础斜杠命令

| 命令              | 说明                                  |
| --------------- | ----------------------------------- |
| `/help`         | 查看所有可用命令和帮助信息 |
| `/init`         | 分析项目结构，初始化 `CLAUDE.md` 文件，自己还可以继续补充 |
| `/add-dir`      | 把另外一个目录添加为工作目录                      |
| `/clear`        | 清除历史记录和上下文                          |
| `/compact`      | 压缩上下文信息（可附加焦点指令，如 `/compact 保留数据库相关内容`） |
| `/diff`         | 打开交互式 diff 查看器，展示 Claude 对文件做的所有修改 |
| `/copy`         | 复制 Claude 的上一条回复到剪贴板（多个代码块时可选择） |
| `/cost`         | 查看 token 使用量和费用统计 |
| `/model`        | 查看或切换 AI 模型 |
| `/config`       | 进入配置管理 |
| `/doctor`       | 诊断 Claude Code 安装问题 |
| `/hooks`        | 管理钩子配置 |
| `/bug`          | 向 Anthropic 报告 Bug |
| `/exit`         | 退出 Claude Code 会话 |
| `/login`        | 登录 Anthropic 账户 |
| `/logout`       | 退出登录 |
| `/status`       | 查看当前会话状态 |
| `/review`       | 审查 Pull Request |
| `/pr-comments`  | 获取 GitHub PR 上的评论 |
| `/terminal-setup`| 配置终端集成 |
| `/vim`          | 切换 Vim 编辑模式 |

## 上下文与会话管理

| 命令        | 说明                                                        |
| --------- | --------------------------------------------------------- |
| `/context` | 查看 token 使用量。注意每次对话都会增加上下文长度，可能会导致关键信息的缺失，因此需要 `/compact` |
| `/rewind`  | 回到同一个窗口之前的对话状态，相当于每次对话都是一次存档点，可以将代码还原到之前的状态               |
| `/resume`  | 切换聊天上下文（恢复之前的对话） |

## CLI 命令行参数

### 基础参数

| 参数 | 说明 |
|------|------|
| `-p, --print` | 单次回答模式，发送查询后直接退出，不进入交互模式 |
| `-c, --continue` | 继续上一次的对话会话 |
| `-r, --resume` | 恢复一个暂停的会话 |
| `--model` | 指定使用的模型 |
| `--version` | 显示版本号 |
| `-h, --help` | 显示帮助信息 |
| `-w, --worktree` | 创建隔离的 git worktree 并在其中启动会话 |

### 输出与控制

| 参数 | 说明 |
|------|------|
| `--output-format` | 输出格式：`json`、`stream-json`、`text`（默认） |
| `--max-turns` | 限制最大交互轮数 |
| `--verbose` | 详细输出模式 |
| `--effort` | 控制任务完成的努力程度 |

### 权限与安全

| 参数 | 说明 |
|------|------|
| `--allowedTools` | 白名单：只允许使用指定的工具 |
| `--disallowedTools` | 黑名单：禁止使用指定的工具 |
| `--dangerously-skip-permissions` | 绕过所有权限检查 **（危险！谨慎使用）** |

### 系统提示词定制

| 参数 | 说明 |
|------|------|
| `--system-prompt` | 自定义系统提示词 |
| `--system-prompt-file` | 从文件加载系统提示词 |
| `--append-system-prompt` | 追加系统提示词 |
| `--append-system-prompt-file` | 从文件追加系统提示词 |

### 其他

| 参数 | 说明 |
|------|------|
| `--add-dir` | 添加目录到上下文 |
| `--session-id` | 指定会话 ID |
| `--no-cache` | 禁用缓存 |
| `--no-telemetry` | 禁用遥测数据收集 |

### 使用示例

```bash
# 单次提问
claude -p "你的问题或指令"

# 继续上次对话
claude -c

# 指定模型
claude --model claude-opus-4-6

# 管道输入（将文件内容传给 Claude）
cat main.cpp | claude -p "解释这段代码"

# JSON 输出（适合脚本集成）
claude -p "列出所有函数" --output-format json
```

## Hooks 钩子系统

Hooks 允许在 Claude Code 的工具执行生命周期中插入自定义脚本，实现自动化工作流。

### 钩子事件类型

| 事件类型 | 触发时机 | 用途 |
|----------|----------|------|
| `PreToolUse` | 工具执行**之前** | 检查并可选择**阻止**操作 |
| `PostToolUse` | 工具执行**之后** | 对完成的操作做后处理（格式化、日志等） |
| `UserPromptSubmit` | 用户提交提示词时 | 验证或修改用户输入 |
| `Notification` | 系统通知时 | 记录日志或触发通知 |
| `Stop` | Claude Code 停止时 | 清理和收尾任务 |
| `SubagentStop` | 子代理结束时 | 处理子代理完成后的后续操作 |
| `PreCompact` | 上下文压缩之前 | 在压缩前执行特定操作 |

### 配置格式

钩子在 `settings.json` 中配置，基本结构如下：

```json
{
  "hooks": {
    "事件类型": [
      {
        "matcher": "工具名称",
        "hooks": [
          {
            "type": "command",
            "command": "要执行的命令"
          }
        ]
      }
    ]
  }
}
```

### Matcher 匹配器

Matcher 决定哪些工具触发钩子：

| Matcher | 匹配目标 |
|---------|----------|
| `Bash` | Shell 命令执行 |
| `Write` | 文件创建 |
| `Edit` | 文件编辑 |
| `Read` | 文件读取 |
| `Glob` | 文件搜索 |
| `Grep` | 内容搜索 |
| `WebSearch` | 网页搜索 |
| `WebFetch` | 网页获取 |
| `mcp__服务名__工具名` | MCP 工具（如 `mcp__drawio__add-rectangle`） |
| `*` | 通配符，匹配所有工具 |

### 环境变量

钩子脚本可以使用以下环境变量获取上下文信息：

| 变量 | 可用事件 | 说明 |
|------|----------|------|
| `CLAUDE_TOOL_NAME` | PreToolUse / PostToolUse | 工具名称（如 `"Write"`、`"Bash"`） |
| `CLAUDE_TOOL_INPUT` | PreToolUse / PostToolUse | JSON 格式的工具输入参数 |
| `CLAUDE_FILE_PATHS` | Write / Edit 相关 | 空格分隔的文件路径列表 |
| `CLAUDE_COMMAND` | Bash 相关 | 正在执行的 Bash 命令 |
| `CLAUDE_SESSION_ID` | 所有事件 | 当前会话 ID |
| `CLAUDE_PROJECT_ROOT` | 所有事件 | 项目根目录 |

### 退出码含义

| 退出码 | 含义 | 行为 |
|--------|------|------|
| `0` | 成功/放行 | PreToolUse：允许工具执行；PostToolUse：正常继续 |
| `1` | 脚本错误 | 记录错误日志，但工具仍会执行 |
| `2` | **阻止执行** | **仅 PreToolUse 有效**：阻止工具执行，stderr 输出作为错误提示显示给用户 |

> **关键**：只有 `PreToolUse` 的 `exit 2` 能阻止操作。`PostToolUse` 中工具已经执行完毕，无法阻止。

### 配置文件位置与优先级

| 位置 | 作用域 | 说明 |
|------|--------|------|
| `~/.claude/settings.json` | 全局（所有项目） | 用户级别的通用钩子 |
| `.claude/settings.json` | 项目级（团队共享） | 提交到版本控制，团队共用 |
| `.claude/settings.local.json` | 项目级（个人） | 本地覆盖，加入 `.gitignore` |

### 实用案例

#### 案例 1：写文件后自动格式化（Prettier）

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write $CLAUDE_FILE_PATHS 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

> Claude 每次创建/修改文件后，自动用 Prettier 格式化。`|| true` 确保 Prettier 未安装时不报错。

#### 案例 2：阻止危险的 Bash 命令

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'COMMAND=$(echo \"$CLAUDE_TOOL_INPUT\" | jq -r \".command\"); if [[ \"$COMMAND\" =~ ^rm\\ -rf\\  ]]; then echo \"Error: rm -rf is blocked\" >&2; exit 2; fi; exit 0'"
          }
        ]
      }
    ]
  }
}
```

> 在 Bash 工具执行前检查命令，如果包含 `rm -rf` 则用 `exit 2` 阻止执行。

#### 案例 3：禁止写入敏感文件

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'FILE=$(echo \"$CLAUDE_TOOL_INPUT\" | jq -r \".path\"); if [[ \"$FILE\" =~ \\.env$ ]]; then echo \"Error: 禁止写入 .env 文件\" >&2; exit 2; fi; exit 0'"
          }
        ]
      }
    ]
  }
}
```

> 阻止 Claude 写入 `.env` 等敏感配置文件。

#### 案例 4：Python 文件自动格式化（Black + isort）

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'for FILE in $CLAUDE_FILE_PATHS; do if [[ \"$FILE\" =~ \\.py$ ]]; then black \"$FILE\" 2>/dev/null || true; isort \"$FILE\" 2>/dev/null || true; fi; done'"
          }
        ]
      }
    ]
  }
}
```

#### 案例 5：记录所有命令执行日志

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[$(date +'%Y-%m-%d %H:%M:%S')] $(echo $CLAUDE_TOOL_INPUT | jq -r '.command')\" >> ~/.claude/command-log.txt"
          }
        ]
      }
    ]
  }
}
```

> 审计用途：将 Claude 执行的每条命令记录到日志文件。

#### 案例 6：Git Commit 消息验证

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'CMD=$(echo \"$CLAUDE_TOOL_INPUT\" | jq -r \".command\"); if [[ \"$CMD\" =~ ^git\\ commit ]]; then MSG=$(echo \"$CMD\" | grep -oP \"(?<=-m ['\\\"]).*?(?=['\\\"])\"); if [[ ${#MSG} -lt 10 ]]; then echo \"Error: Commit 消息至少 10 个字符\" >&2; exit 2; fi; fi; exit 0'"
          }
        ]
      }
    ]
  }
}
```

### Hooks 最佳实践

1. **保持脚本轻量** — 钩子会阻塞工具执行，尽量快速完成
2. **PostToolUse 用 `|| true`** — 避免后处理脚本失败影响流程
3. **用 `jq` 解析 JSON** — `CLAUDE_TOOL_INPUT` 是 JSON 格式，用 `jq` 提取字段
4. **有意义的错误信息** — `exit 2` 阻止时，stderr 输出会显示给用户，写清楚原因
5. **团队共享格式化钩子** — 将 `.claude/settings.json` 提交到版本控制

## 特殊指令

### `#` 指令
键给 Claude 一个指令，它会自动将其纳入相关的 `CLAUDE.md` 中：
- 添加记忆，每次对话都会添加记忆
- 告诉 Claude 项目加了什么东西

### `!` 指令
把对话窗口切换为命令行模式，执行临时的命令行命令。AI 也可以知道已经安装过什么东西。

## 记忆系统

| 类型 | 说明 | 位置 |
|------|------|------|
| 全局记忆 | 对所有的项目生效，用户级别的 | `~/.claude/CLAUDE.md` |
| 局部记忆 | 对当前项目生效，项目级别 | 项目目录中的 `CLAUDE.md` |

> **最佳实践**：在项目根目录创建 `CLAUDE.md` 文件，记录项目结构、编码规范和常用命令，Claude 会自动读取这些信息来更好地理解项目上下文。

## 关键概念补充

### CLAUDE.md 文件
`CLAUDE.md` 是 Claude Code 的项目配置文件，作用类似于 [[2.1 codex语法#AGENTS.md 文件|Codex CLI 的 AGENTS.md]]。它包含：
- 项目结构描述
- 编码风格指南
- 常用命令和脚本
- 项目特定的上下文信息

### MCP (Model Context Protocol)
MCP 是一种协议，允许 Claude Code 连接外部服务和工具，扩展其能力范围。通过 `mcp` 命令可以管理这些服务器连接。

### Token 与上下文
- **Token**：AI 模型处理文本的基本单位
- **上下文窗口**：模型能够"记住"的信息量有限
- 当上下文过长时，使用 `/compact` 压缩以保留关键信息
