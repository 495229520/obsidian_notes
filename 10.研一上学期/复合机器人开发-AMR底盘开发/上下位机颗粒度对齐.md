# 上下位机颗粒度对齐

基于 `src/mfms_server/client_api/include/mfms_server/CommunicationInterface.h`、`src/com_interfaces/msg/SeerCtrlState.msg`、`src/com_interfaces/srv/SeerCtrlCmdInterface.srv` 和 `MFMS_BASE_04171715.sql` 的对照整理如下。

## 接口对照

| 顶层接口 | 现状 | 下位机现有支撑 | 缺失项 | 结论 |
| --- | --- | --- | --- | --- |
| `sendAGVState(const SeerCtrlState::SharedPtr)` | 已打通 | `SeerCtrlState.msg` 已提供 `name`、`err_code`、`err_msg`、`pose_x`、`pose_y`、`angle`、`confidence`、`current_station`、`last_station`、`vel_x`、`vel_y`、`vel_ang` | 无 | 可支撑 |
| `connectRobot(const QString& name)` | 已可用于 AGV | 中台可根据 `device_state` 找到 AGV，并直接订阅 `/{device_id}_state` | 下位机没有单独 AGV“代理连接”必要协议 | 可支撑 |
| `disconnectRobot()` | 已可用于 AGV | 中台本地取消订阅即可 | 无需下位机额外能力 | 可支撑 |
| `getStations()` | 已打通 | `SeerCtrlCmdInterface` 响应里有 `station_list`、`current_station` | `srv` 本身没有正式定义“查询站点”对应的 `id` 语义，现在依赖命令码约定 | 可支撑，但依赖约定 |
| `exeToStation(QString& stationName)` | 已打通 | `SeerCtrlCmdInterface` 请求里有 `station`，响应里有 `err_code`、`msg`、`task_status`、`task_type`、`target_station`、`current_station` | 同样依赖 `id` 命令码约定 | 可支撑，但依赖约定 |
| `agvMoveForward(double, double)` | 未实现 | 无 | `srv` 请求里没有 `vx` / `vy` / `vw` / `dist` / `angle` / `mode` / `duration` 这类手动运动字段 | 当前协议不足，无法正确实现 |
| `agvMoveBackward(double, double)` | 未实现 | 无 | 同上 | 当前协议不足，无法正确实现 |
| `agvTurnLeft(double, double)` | 未实现 | 无 | 同上 | 当前协议不足，无法正确实现 |
| `agvTurnRight(double, double)` | 未实现 | 无 | 同上 | 当前协议不足，无法正确实现 |
| `getPaths()` | 未实现 | 无 | `srv` 没有“路径名列表”查询能力；数据库也没有独立路径表 | 当前协议和数据库都不足 |
| `exeToPath(QString& stationName)` | 未实现 | 无 | `srv` 只有 `station` / `station_list`，没有“按路径名执行”能力；数据库也没有路径名到站点序列映射 | 当前协议和数据库都不足 |
| `addPath(const QList<QString>& stationList)` | 未实现 | 无 | `srv` 没有路径写入能力；`MFMS_BASE_04171715.sql` 也没有可用路径持久化模型 | 当前协议和数据库都不足 |

## 关键结论

| 类别 | 结论 |
| --- | --- |
| 状态类接口 | 下位机 `msg` 足够，已经能支撑 |
| 站点类接口 | 下位机 `srv` 基本够，已经能打通，但命令码语义目前是约定性的 |
| 手动运动类接口 | 下位机 `srv` 不够，缺速度、距离、角速度、角度等控制语义 |
| 路径管理类接口 | 下位机 `srv` 不够，`MFMS_BASE_04171715.sql` 也没有路径模型，双重不足 |

## 如果后面要把剩余接口做全，最少需要下位机补什么

| 顶层能力 | 下位机最少补充 |
| --- | --- |
| 手动前后 / 转向 | 在 `SeerCtrlCmdInterface.srv` 或新增 AGV 控制 `srv` 中补 `vx` / `vy` / `vw` / `dist` / `angle` / `mode` / `duration` 之类字段 |
| 查询路径 | 补“路径名列表”查询协议，或在中台-客户端数据库中建立路径表 |
| 按路径执行 | 补“路径名 -> 站点序列”解析能力，或允许直接传路径名给下位机 |
| 新增路径 | 补路径持久化接口，或者把路径管理明确下放到中台-客户端数据库 |
