# VaR概念与尾部风险

> **对应培养方案：** 第5周第31天——VaR 概念（简化版）+ 其不足之处

---

## 核心概念

### 什么是VaR（Value at Risk）？

**定义**：VaR（风险价值）是一个统计度量，回答一个简单的问题：**在正常市场条件下，给定的置信水平和时间期限内，我最多可能亏多少钱？**

例如："95%置信水平下的1天VaR为2万元"意味着：在正常情况下，你有95%的把握，一天内的损失不会超过2万元。

> **菲利普·乔瑞（Philippe Jorion，VaR权威）**："VaR用一个数字概括了整个投资组合的市场风险。"
> — 《风险价值》

> **艾伦·格林斯潘（前美联储主席）**："VaR是过去十年金融风险管理中最重要的进步之一。"

#### 生活中的例子

**例子1：天气预报与VaR**

想象气象预报说："明天有5%的概率会下大雨"。

- 这意味着95%的概率不会下大雨
- 你可能选择不带伞出门
- 但如果真的下了那5%概率的大雨，你会被淋湿

VaR就像是投资的"天气预报"：
- "95% VaR = 2万"意味着"有5%的概率你会损失超过2万"
- 这5%的情况发生时，损失可能是3万、5万甚至更多——VaR没有告诉你

**例子2：汽车保险的免赔额**

你的车险有2000元的免赔额：
- 正常小剐蹭损失在2000元以内，你自己承担
- VaR就像是说："95%的情况下，我的损失在某个金额以内"
- 但那5%的大事故会发生什么？可能是10万的损失——远超免赔额

**例子3：赌场的最大下注额**

赌场设定单次最大下注额为10万：
- 这相当于赌场的"VaR控制"——限制单次最大可能损失
- 但如果有人连赢10把？赌场可能损失100万
- VaR告诉你"单次最多亏多少"，但没说极端情况会有多糟

### VaR的三个要素

$$\text{VaR} = f(\text{置信水平}, \text{时间期限}, \text{投资组合})$$

| 要素 | 含义 | 常见选择 | 投资者含义 |
|------|------|----------|------------|
| **置信水平** | 你有多大把握损失不超过这个数 | 95%、99% | 置信水平越高，VaR数值越大 |
| **时间期限** | 损失可能发生的时间窗口 | 1天、1周、1月 | 期限越长，VaR越大 |
| **投资组合** | 你持有的资产和仓位 | — | 波动性越大，VaR越大 |

#### 置信水平的直觉

| 置信水平 | 含义 | 多久可能突破一次？ |
|----------|------|-------------------|
| 90% | 10%概率超过VaR | 约每10天一次 |
| 95% | 5%概率超过VaR | 约每20天一次（每月一次） |
| 99% | 1%概率超过VaR | 约每100天一次（每半年一次） |
| 99.9% | 0.1%概率超过VaR | 约每1000天一次（每4年一次） |

> **纳西姆·塔勒布**："VaR就像安全带——它能保护你免受轻微碰撞，但在真正的致命车祸中可能毫无用处。"
> — 《黑天鹅》

### VaR的计算方法（简化版）

#### 方法1：历史模拟法（最直观）

**步骤**：
1. 收集过去N天的每日收益率数据
2. 将收益率从小到大排序
3. 找到第5%分位数（对于95%置信水平）
4. 这个数值乘以你的投资金额就是VaR

**案例：计算沪深300的1天95% VaR**

假设你收集了沪深300过去250天的每日收益率，排序后：

| 排名（从小到大） | 日收益率 | 说明 |
|-----------------|----------|------|
| 第1位（最差） | -5.2% | |
| 第2位 | -4.8% | |
| ... | ... | |
| 第12位 | -2.1% | |
| **第13位** | **-1.9%** | **5%分位数（250×5%≈13）** |
| 第14位 | -1.8% | |
| ... | ... | |

**结论**：95%置信水平下的1天VaR约为1.9%。

如果你持有100万元沪深300指数基金：
$$\text{VaR} = 100万 \times 1.9\% = 1.9万元$$

**解读**：在正常市场条件下，你有95%的把握，明天的损失不会超过1.9万元。

#### 方法2：参数法（假设正态分布）

$$\text{VaR} = \mu - z_\alpha \times \sigma$$

其中：
- $\mu$ = 预期收益率（短期可假设为0）
- $z_\alpha$ = 置信水平对应的标准正态分位数
- $\sigma$ = 日收益率标准差（日波动率）

| 置信水平 | $z_\alpha$ |
|----------|-----------|
| 90% | 1.28 |
| 95% | 1.65 |
| 99% | 2.33 |

**简化计算**：
$$\text{日VaR}_{95\%} \approx 1.65 \times \text{日波动率} \times \text{投资金额}$$

**案例**：
- 沪深300日波动率约1.2%
- 投资金额100万元
- 日VaR(95%) ≈ 1.65 × 1.2% × 100万 = **1.98万元**

#### 方法3：时间换算

$$\text{VaR}_T = \text{VaR}_1 \times \sqrt{T}$$

| 时间期限 | 换算系数 | 举例（日VaR=2万） |
|----------|----------|------------------|
| 1天 | 1 | 2万 |
| 1周（5天） | √5 ≈ 2.24 | 4.48万 |
| 1月（21天） | √21 ≈ 4.58 | 9.16万 |
| 1年（252天） | √252 ≈ 15.87 | 31.7万 |

**重要提醒**：时间换算基于收益率独立同分布假设，实际市场中波动性会聚集，因此长期VaR通常被低估。

### VaR的致命缺陷

> **沃伦·巴菲特**："VaR是'在晴天帮你算出你可能被淋湿多少'的工具——但真正的暴风雨来临时，它完全失灵了。"

#### 缺陷1：尾部风险盲区

VaR只告诉你"95%概率内的最大损失"，但没说那5%会发生什么。

**案例：2008年金融危机**

| 日期 | 标普500单日跌幅 | 正态分布下的概率 | 实际发生 |
|------|----------------|-----------------|----------|
| 2008年10月15日 | -9.03% | 几乎不可能（<0.001%） | 发生了 |
| 2008年10月9日 | -7.62% | 极端罕见 | 发生了 |
| 2008年9月29日 | -8.81% | 极端罕见 | 发生了 |

**问题**：在2008年10月，这种"百年一遇"的事件一个月内发生了多次。

> **纳西姆·塔勒布**："VaR让交易员在99%的日子里睡得很好，然后在剩下的1%日子里把公司搞破产。"

#### 缺陷2：不满足次可加性

次可加性意味着：分散投资的风险应该 ≤ 单独投资风险之和。

**反直觉的例子**：

| 资产 | 单独VaR | 组合后VaR |
|------|---------|----------|
| A | 100万 | - |
| B | 100万 | - |
| A+B（各50%） | - | **可能>100万** |

这意味着VaR可能"惩罚"分散投资——这显然违背常识。

#### 缺陷3：正态分布假设

金融市场的收益率分布是"肥尾"的——极端事件发生的频率远高于正态分布预测。

| 事件 | 正态分布预测频率 | 实际观察频率 |
|------|-----------------|-------------|
| 日跌幅超过3% | 约0.1%（每3年1次） | 实际更频繁 |
| 日跌幅超过5% | 几乎不可能 | 每2-3年发生 |
| 日跌幅超过10% | 天文数字概率 | 历史上多次发生 |

**经典案例：LTCM（长期资本管理公司）**

| 事件节点 | 发生了什么 |
|----------|------------|
| **背景** | 两位诺贝尔奖得主创立的对冲基金，使用VaR管理风险 |
| **他们的模型** | 根据VaR，他们的策略"几乎不可能"亏损超过某个水平 |
| **1998年8月** | 俄罗斯债务违约，"不可能"发生的极端事件真的发生了 |
| **结果** | 46亿美元资本几乎全部亏光，需要美联储协调救助 |

> **迈伦·斯科尔斯（LTCM创始人之一）**事后承认："我们的模型忽略了市场在极端情况下的行为。"

#### 缺陷4：后视镜驾驶

历史VaR基于过去数据，但未来可能完全不同：
- 波动率可能突然飙升
- 相关性可能突然改变（危机时资产同涨同跌）
- "没发生过"不等于"不会发生"

## 投资者视角

### 为什么投资者仍然要理解VaR？

1. **风险可视化**：VaR把模糊的"风险"变成具体的数字，便于决策
2. **组合比较**：可以比较不同投资组合的风险水平
3. **仓位规划**：基于VaR决定该配置多少资金
4. **合规要求**：机构投资者通常需要报告VaR

### VaR的正确使用方式

| 使用方式 | 正确 ✓ | 错误 ✗ |
|----------|--------|--------|
| **作为下限** | VaR是"至少可能亏这么多" | 认为"最多只会亏这么多" |
| **结合场景分析** | VaR + 压力测试一起用 | 只看VaR数字 |
| **保守估计** | 使用更高置信水平（99%） | 用90%显得风险小 |
| **频繁更新** | 根据市场情况调整 | 用一年前的VaR |

### 从VaR到CVaR（条件VaR）

**CVaR（Expected Shortfall）**：当损失超过VaR时，平均会亏多少？

$$\text{CVaR}_\alpha = E[L | L > \text{VaR}_\alpha]$$

| 指标 | 含义 | 举例 |
|------|------|------|
| VaR(95%) | 95%概率内的最大损失 | 2万元 |
| CVaR(95%) | 当损失超过VaR时的平均损失 | 3.5万元 |

**投资者偏好**：CVaR比VaR更保守，更能反映尾部风险。如果你只能看一个数字，看CVaR更安全。

### 个人投资者的VaR应用框架

| 步骤 | 内容 | 工具 |
|------|------|------|
| 1. 估算日波动率 | 查询ETF的历史波动率 | 券商数据/财经网站 |
| 2. 计算日VaR | VaR ≈ 1.65 × 波动率 × 金额 | Excel/计算器 |
| 3. 换算周VaR | 日VaR × √5 | 用于每周检查 |
| 4. 设定警戒线 | 周VaR > 心理承受能力 → 减仓 | 你的风险偏好 |
| 5. 压力测试 | 如果跌幅是VaR的2-3倍呢？ | 情景分析 |

## 关键要点

1. **VaR是下限不是上限**：95% VaR告诉你"95%概率内最多亏这么多"，但那5%的情况可能亏更多——远远更多

2. **尾部风险是真正的杀手**：2008年、2020年的"百年一遇"事件比模型预测的频繁得多。VaR在最需要它的时候最不可靠

3. **VaR需要压力测试补充**：永远问自己："如果损失是VaR的2倍、3倍、5倍，我还能活下来吗？"

4. **历史不代表未来**：基于过去100天数据计算的VaR，无法预测下一个黑天鹅事件

5. **CVaR比VaR更保守**：如果只看一个风险指标，CVaR（条件VaR）更能反映极端情况

## 延伸思考

### 为什么专业机构也会被VaR误导？

> **查理·芒格**："在金融领域，历史不会简单重复，但会押韵。"

| 案例 | VaR给他们的错觉 | 实际发生了什么 |
|------|-----------------|----------------|
| **LTCM（1998）** | 模型显示亏损超过20%"几乎不可能" | 一个月内亏损90%以上 |
| **贝尔斯登（2008）** | VaR显示风险可控 | 两周内倒闭 |
| **雷曼兄弟（2008）** | 每日VaR报告显示"绿灯" | 破产前几天还在发VaR报告 |

### VaR的心理陷阱

| 陷阱 | 表现 | 后果 |
|------|------|------|
| **虚假安全感** | "VaR只有2%，很安全" | 忽视尾部风险 |
| **精确的错觉** | "VaR是1.87%，很精确" | 过度相信模型 |
| **后视镜驾驶** | "过去没出过事" | 未来可能不同 |
| **合规即安全** | "我们符合VaR限制" | 监管≠真正的风险管理 |

**投资者警示**：VaR是有用的风险度量工具，但它不是风险管理的全部。永远记住：**最危险的情况是你最没准备的情况**。VaR告诉你"晴天"的风险，但你需要为"暴风雨"做好准备——这就是明天压力测试的主题。
