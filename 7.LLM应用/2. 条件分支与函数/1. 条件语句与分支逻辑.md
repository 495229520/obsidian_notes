## 核心概念

### 什么是条件语句？

**定义**：条件语句让程序能够根据某个条件是否成立，选择执行不同的代码路径。

> **PY4E**："条件语句给了我们检查条件并改变程序行为的能力。"

#### 为什么重要

- **问题**：程序需要处理多种情况（及格/不及格、VIP/普通用户、工作日/周末）
- **解决**：条件语句让程序能"思考"，根据情况做出不同响应

#### 直观理解

**类比**：条件语句就像人生的岔路口

```
           ┌─ 成绩 >= 60 ──→ "及格"
           │
你的成绩 ──┤
           │
           └─ 成绩 < 60 ───→ "不及格"
```

---

### 比较运算符

在写条件之前，需要掌握比较运算符：

| 运算符 | 含义 | 示例 | 结果 |
|--------|------|------|------|
| `==` | 等于 | `5 == 5` | `True` |
| `!=` | 不等于 | `5 != 3` | `True` |
| `>` | 大于 | `5 > 3` | `True` |
| `<` | 小于 | `5 < 3` | `False` |
| `>=` | 大于等于 | `5 >= 5` | `True` |
| `<=` | 小于等于 | `5 <= 3` | `False` |

**关键点**：
- `==` 是比较，`=` 是赋值，千万别搞混！
- 比较运算的结果是布尔值：`True` 或 `False`

---

### 布尔运算符

多个条件可以组合：

| 运算符 | 含义 | 示例 | 结果 |
|--------|------|------|------|
| `and` | 且（都满足） | `True and False` | `False` |
| `or` | 或（满足一个） | `True or False` | `True` |
| `not` | 非（取反） | `not True` | `False` |

```python
age = 25
# 同时满足两个条件
if age >= 18 and age <= 60:
    print("劳动年龄人口")

# 满足其中一个条件
if age < 18 or age > 60:
    print("非劳动年龄人口")
```

### if-elif-else 结构

```python
if 条件1:
    # 条件1为True时执行
    代码块1
elif 条件2:
    # 条件1为False，条件2为True时执行
    代码块2
elif 条件3:
    # 前面都为False，条件3为True时执行
    代码块3
else:
    # 所有条件都为False时执行
    代码块4
```

**关键点**：
- 缩进很重要！Python 用缩进区分代码块（通常4个空格）
- `elif` 是 "else if" 的缩写
- `else` 是可选的，用于处理"其他所有情况"
- 条件从上到下依次判断，一旦满足就不再检查后面的

## 代码实践

### 脚本1：grade.py - 成绩评级

**目标**：根据分数输出等级（A/B/C/D/F）

```python
# grade.py - 成绩评级系统

print("=== 成绩评级系统 ===")

# 获取成绩
score_str = input("请输入你的成绩(0-100): ")
score = float(score_str)

# 评级逻辑
if score >= 90:
    grade = "A"
    comment = "优秀！继续保持！"
elif score >= 80:
    grade = "B"
    comment = "良好，还有提升空间"
elif score >= 70:
    grade = "C"
    comment = "中等，需要更努力"
elif score >= 60:
    grade = "D"
    comment = "及格，要加油了"
else:
    grade = "F"
    comment = "不及格，需要重修"

# 输出结果
print(f"\n你的成绩: {score}")
print(f"等级: {grade}")
print(f"评语: {comment}")
```

**关键点**：
- 条件从高到低排列，避免重复判断
- `score >= 90` 会先被检查，如果满足就不会检查 `score >= 80`
- 使用变量存储结果，最后统一输出

**运行示例**：

```
=== 成绩评级系统 ===
请输入你的成绩(0-100): 85

你的成绩: 85.0
等级: B
评语: 良好，还有提升空间
```

### 脚本2：leap_year.py - 闰年判断

**目标**：判断某年是否为闰年

**闰年规则**：
1. 能被4整除 且 不能被100整除 → 闰年
2. 能被400整除 → 闰年
3. 其他 → 平年

```python
# leap_year.py - 闰年判断器

print("=== 闰年判断器 ===")

year_str = input("请输入年份: ")
year = int(year_str)

# 闰年判断逻辑
# 方法1：使用嵌套条件
if year % 400 == 0:
    is_leap = True
elif year % 100 == 0:
    is_leap = False
elif year % 4 == 0:
    is_leap = True
else:
    is_leap = False

# 方法2（更简洁）：使用布尔表达式
# is_leap = (year % 400 == 0) or (year % 4 == 0 and year % 100 != 0)

# 输出结果
if is_leap:
    print(f"{year} 年是闰年，有 366 天")
else:
    print(f"{year} 年是平年，有 365 天")

# 额外信息
if is_leap:
    print("2月有29天")
else:
    print("2月有28天")
```

**关键点**：
- `%` 是取余运算符，`year % 4 == 0` 表示能被4整除
- 判断顺序很重要：先判断400，再判断100，最后判断4
- 布尔变量 `is_leap` 存储判断结果，方便后续使用

**运行示例**：

```
=== 闰年判断器 ===
请输入年份: 2024
2024 年是闰年，有 366 天
2月有29天
```

**验证表格**：

| 年份 | 能被400整除 | 能被100整除 | 能被4整除 | 结果 |
|------|-------------|-------------|-----------|------|
| 2000 | ✓ | ✓ | ✓ | 闰年 |
| 1900 | ✗ | ✓ | ✓ | 平年 |
| 2024 | ✗ | ✗ | ✓ | 闰年 |
| 2023 | ✗ | ✗ | ✗ | 平年 |

### 脚本3：billing.py - 分段计费

**目标**：实现阶梯电价计费

**计费规则**（示例）：
- 0-100度：0.5元/度
- 101-200度：0.8元/度
- 201度以上：1.2元/度

```python
# billing.py - 阶梯电价计算器

print("=== 阶梯电价计算器 ===")

usage_str = input("请输入本月用电量(度): ")
usage = float(usage_str)

# 定义价格（方便修改）
PRICE_TIER1 = 0.5   # 第一档：0-100度
PRICE_TIER2 = 0.8   # 第二档：101-200度
PRICE_TIER3 = 1.2   # 第三档：201度以上

LIMIT_TIER1 = 100
LIMIT_TIER2 = 200

# 分段计算
if usage <= 0:
    total = 0
    detail = "无用电"
elif usage <= LIMIT_TIER1:
    # 只在第一档
    total = usage * PRICE_TIER1
    detail = f"{usage}度 × {PRICE_TIER1}元 = {total}元"
elif usage <= LIMIT_TIER2:
    # 跨第一档和第二档
    tier1_cost = LIMIT_TIER1 * PRICE_TIER1
    tier2_cost = (usage - LIMIT_TIER1) * PRICE_TIER2
    total = tier1_cost + tier2_cost
    detail = (f"第一档: {LIMIT_TIER1}度 × {PRICE_TIER1}元 = {tier1_cost}元\n"
              f"第二档: {usage - LIMIT_TIER1}度 × {PRICE_TIER2}元 = {tier2_cost}元")
else:
    # 跨三个档位
    tier1_cost = LIMIT_TIER1 * PRICE_TIER1
    tier2_cost = (LIMIT_TIER2 - LIMIT_TIER1) * PRICE_TIER2
    tier3_cost = (usage - LIMIT_TIER2) * PRICE_TIER3
    total = tier1_cost + tier2_cost + tier3_cost
    detail = (f"第一档: {LIMIT_TIER1}度 × {PRICE_TIER1}元 = {tier1_cost}元\n"
              f"第二档: {LIMIT_TIER2 - LIMIT_TIER1}度 × {PRICE_TIER2}元 = {tier2_cost}元\n"
              f"第三档: {usage - LIMIT_TIER2}度 × {PRICE_TIER3}元 = {tier3_cost}元")

# 输出结果
print(f"\n用电量: {usage} 度")
print(f"\n费用明细:")
print(detail)
print(f"\n总费用: {total:.2f} 元")
```

**关键点**：
- 用大写变量名定义常量（`PRICE_TIER1`），方便修改和理解
- 分段计费需要累加每一档的费用
- `:.2f` 格式化保留两位小数

**运行示例**：

```
=== 阶梯电价计算器 ===
请输入本月用电量(度): 250

用电量: 250.0 度

费用明细:
第一档: 100度 × 0.5元 = 50.0元
第二档: 100度 × 0.8元 = 80.0元
第三档: 50.0度 × 1.2元 = 60.0元

总费用: 190.00 元
```

## 踩坑记录

### 问题1：`=` 和 `==` 搞混

**症状**：

```python
if score = 90:  # SyntaxError!
    print("满分")
```

**原因**：`=` 是赋值，`==` 才是比较

**解决方案**：

```python
if score == 90:  # 正确：比较
    print("满分")
```

---

### 问题2：缩进错误

**症状**：

```python
if score >= 60:
print("及格")  # IndentationError!
```

**原因**：`if` 后面的代码块必须缩进

**解决方案**：

```python
if score >= 60:
    print("及格")  # 正确：缩进4个空格
```

---

### 问题3：条件顺序错误

**症状**：

```python
# 错误的顺序
if score >= 60:
    grade = "D"
elif score >= 90:  # 永远不会执行到这里！
    grade = "A"
```

**原因**：90分也满足 `>= 60`，会被第一个条件捕获

**解决方案**：

```python
# 正确的顺序：从严到宽
if score >= 90:
    grade = "A"
elif score >= 60:
    grade = "D"
```

---

### 问题4：浮点数比较陷阱

**症状**：

```python
if 0.1 + 0.2 == 0.3:  # False！
    print("相等")
```

**原因**：浮点数有精度问题，`0.1 + 0.2 = 0.30000000000000004`

**解决方案**：

```python
# 使用近似比较
if abs(0.1 + 0.2 - 0.3) < 0.0001:
    print("近似相等")
```

## 最佳实践

| 推荐做法 | 避免做法 | 原因 |
|----------|----------|------|
| 条件从严到宽排列 | 条件从宽到严 | 避免逻辑错误 |
| 用变量存储条件结果 | 重复计算条件 | 代码更清晰 |
| 用 `elif` 处理互斥条件 | 用多个独立 `if` | 效率更高 |
| 常量用大写命名 | 硬编码数字 | 方便维护修改 |

## 关键要点

1. **条件语句结构**：`if` → `elif`（可选多个）→ `else`（可选）
2. **比较运算符**：`==` `!=` `>` `<` `>=` `<=`
3. **布尔运算符**：`and`（且）、`or`（或）、`not`（非）
4. **缩进很重要**：Python 用缩进区分代码块
5. **条件顺序**：从严格到宽松排列，避免逻辑错误
