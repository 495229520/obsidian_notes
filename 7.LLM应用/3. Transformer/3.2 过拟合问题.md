## 一、过拟合（Overfitting）

### 1.1 是什么

**过拟合**是指模型在训练集上表现优异，但在未见过的测试集上表现糟糕的现象。

| 指标 | 过拟合时的表现 |
|------|--------------|
| 训练损失 | 很低 |
| 验证/测试损失 | 很高 |
| 训练精度 | 接近100% |
| 测试精度 | 明显低于训练精度 |

直观理解：学生死记硬背了所有练习题的答案，考试遇到新题就不会做了。模型"记住"了训练数据的噪声和细节，而没有学到真正的规律。

### 1.2 为什么会发生：偏差-方差权衡

过拟合的数学本质可以用**偏差-方差分解（Bias-Variance Decomposition）**来解释。

对于一个预测函数 $\hat{f}(x)$，其在某个输入 $x$ 上的期望泛化误差可以分解为：

$$E\left[(y - \hat{f}(x))^2\right] = \underbrace{\left(\text{Bias}[\hat{f}(x)]\right)^2}_{\text{偏差}^2} + \underbrace{\text{Var}[\hat{f}(x)]}_{\text{方差}} + \underbrace{\sigma^2}_{\text{不可约噪声}}$$

其中：

- **偏差（Bias）**：模型预测的期望值与真实值之间的差距。偏差高意味着模型太简单，系统性地偏离真实规律（欠拟合）。

$$\text{Bias}[\hat{f}(x)] = E[\hat{f}(x)] - f(x)$$

- **方差（Variance）**：模型预测值在不同训练集上的波动程度。方差高意味着模型太复杂，对训练数据的微小变化过度敏感（过拟合）。

$$\text{Var}[\hat{f}(x)] = E\left[(\hat{f}(x) - E[\hat{f}(x)])^2\right]$$

- **不可约噪声（$\sigma^2$）**：数据本身的固有噪声，无论模型多好都无法消除。

**权衡关系**：模型复杂度增加时，偏差下降但方差上升；复杂度降低时反之。最佳模型应该在两者之间找到平衡点。

```
        误差
         ^
         |  \          / 方差
         |   \        /
         |    \      /
         |     \    /          总误差
         |      \  / --------/
         |  偏差  \/  最优点 /
         |       /\        /
         |      /  \------/
         |     /
         +----------------------------> 模型复杂度
        欠拟合           过拟合
```

### 1.3 过拟合的常见原因

| 原因 | 说明 |
|------|------|
| 训练数据不足 | 数据量远小于模型参数量 |
| 模型过于复杂 | 参数数量远超数据所蕴含的信息量 |
| 训练时间过长 | 模型有足够时间"记住"噪声 |
| 缺乏正则化 | 没有对模型复杂度进行约束 |
| 特征噪声大 | 输入数据包含大量无关特征 |

## 二、防止过拟合的策略

### 2.1 数据增强（Data Augmentation）

**是什么**：在不获取新数据的情况下，通过对现有数据施加合理变换来人工扩大训练集。

**为什么有效**：更多样化的数据迫使模型学习更鲁棒的特征，而非记忆特定样本的细节。

**怎么用**：

```python
import torchvision.transforms as T

# 图像数据增强示例
train_transform = T.Compose([
    T.RandomHorizontalFlip(p=0.5),      # 50%概率水平翻转
    T.RandomRotation(degrees=15),        # 随机旋转+-15度
    T.RandomResizedCrop(224, scale=(0.8, 1.0)),  # 随机裁剪缩放
    T.ColorJitter(brightness=0.2, contrast=0.2), # 颜色抖动
    T.ToTensor(),
    T.Normalize(mean=[0.485, 0.456, 0.406],
                std=[0.229, 0.224, 0.225]),
])
```

不同领域的数据增强策略：

| 领域 | 常用增强方法 |
|------|------------|
| 图像 | 旋转、翻转、裁剪、颜色变换、Mixup、CutMix |
| 文本 | 同义词替换、随机插入/删除、反向翻译（Back Translation） |
| 音频 | 时间扭曲、音量缩放、添加背景噪声、频谱增强 |

### 2.2 简化模型

**是什么**：降低模型容量（参数数量），使其不足以记住训练集的所有细节。

**为什么有效**：根据偏差-方差权衡，降低复杂度能减小方差，以适度增加偏差为代价换取更好的泛化。

**怎么用**：减少层数、减少每层神经元数量、使用更简单的架构。缺点是如果过度简化，会导致欠拟合。

### 2.3 正则化（Regularization）

正则化的核心思想：在优化目标中增加一个**惩罚项**，约束模型参数不要过大。

#### L2 正则化（Ridge / 权重衰减）

**是什么**：在损失函数中添加所有权重平方和的惩罚项。

**数学形式**：

$$L_{\text{total}} = L_{\text{data}} + \frac{\lambda}{2m}\sum_{l=1}^{L}\|W^{(l)}\|_F^2$$

其中 $\|W^{(l)}\|_F^2 = \sum_i \sum_j (W_{ij}^{(l)})^2$ 是 Frobenius 范数的平方，$\lambda$ 是正则化强度超参数，$m$ 是样本数。

**对梯度的影响**：

$$\frac{\partial L_{\text{total}}}{\partial W^{(l)}} = \frac{\partial L_{\text{data}}}{\partial W^{(l)}} + \frac{\lambda}{m} W^{(l)}$$

权重更新规则变为：

$$W^{(l)} \leftarrow W^{(l)} - \alpha\left(\frac{\partial L_{\text{data}}}{\partial W^{(l)}} + \frac{\lambda}{m} W^{(l)}\right) = \left(1 - \frac{\alpha\lambda}{m}\right)W^{(l)} - \alpha\frac{\partial L_{\text{data}}}{\partial W^{(l)}}$$

注意前面的系数 $\left(1 - \frac{\alpha\lambda}{m}\right) < 1$，这意味着**每次更新都会将权重缩小一点**，所以 L2 正则化也叫"权重衰减（Weight Decay）"。

**为什么有效**：大权重意味着模型对输入的微小变化非常敏感（高方差）。L2 惩罚迫使权重保持较小值，使决策边界更平滑。

#### L1 正则化（Lasso）

**是什么**：在损失函数中添加所有权重绝对值之和的惩罚项。

**数学形式**：

$$L_{\text{total}} = L_{\text{data}} + \frac{\lambda}{m}\sum_{l=1}^{L}\|W^{(l)}\|_1$$

其中 $\|W^{(l)}\|_1 = \sum_i \sum_j |W_{ij}^{(l)}|$。

#### L1 vs L2 深入对比

| 特性 | L1 正则化 | L2 正则化 |
|------|----------|----------|
| 惩罚项 | $\lambda\sum\|w_i\|$ | $\frac{\lambda}{2}\sum w_i^2$ |
| 梯度 | $\lambda \cdot \text{sign}(w)$（常数大小） | $\lambda \cdot w$（与权重成比例） |
| 对大权重 | 恒定惩罚力度 | 强惩罚（二次增长） |
| 对小权重 | 恒定惩罚力度 | 弱惩罚（接近零） |
| 产生的权重 | **稀疏**（很多恰好为0） | **均匀小**（趋近于0但不等于0） |
| 几何约束 | 菱形（L1球） | 圆形（L2球） |
| 典型用途 | 特征选择、压缩模型 | 通用正则化 |

**L1 为什么产生稀疏权重？关键在于梯度的性质：**

L2 的梯度 $\frac{\partial}{\partial w}(\frac{1}{2}w^2) = w$：当 $w$ 很小时（比如 $w=0.001$），梯度也很小（$0.001$），推动力微弱，$w$ 很难被精确推到 0。

L1 的梯度 $\frac{\partial}{\partial w}(|w|) = \text{sign}(w)$：无论 $w$ 多小，只要 $w \neq 0$，梯度大小恒为 1。这个恒定的推动力会持续将 $w$ 推向 0，直到精确等于 0。

从几何角度看：L1 的约束区域是菱形，等高线与菱形的交点更可能落在坐标轴上（即某些 $w_i = 0$）；L2 的约束区域是圆形，交点通常不在坐标轴上。

**PyTorch 中使用正则化**：

```python
import torch
import torch.nn as nn

model = nn.Linear(10, 1)

# 方式1：L2正则化 —— 直接在优化器中设置 weight_decay
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)

# 方式2：L1正则化 —— 手动添加到损失函数
def l1_loss(model, lambda_l1=1e-5):
    l1_norm = sum(p.abs().sum() for p in model.parameters())
    return lambda_l1 * l1_norm

# 训练时
loss = criterion(output, target) + l1_loss(model)
```

### 2.4 Dropout

**是什么**：在训练过程中，每个前向传播随机将一部分神经元的输出置为 0（"关闭"它们），丢弃率为 $p$。

**数学表述**：

训练时，对第 $l$ 层的激活值 $a_i^{(l)}$：

$$\tilde{a}_i^{(l)} = \frac{r_i \cdot a_i^{(l)}}{1 - p}, \quad r_i \sim \text{Bernoulli}(1-p)$$

其中 $r_i$ 是一个伯努利随机变量，以概率 $p$ 为 0（关闭），以概率 $(1-p)$ 为 1（保留）。

除以 $(1-p)$ 是**反演缩放（Inverted Dropout）**的关键：保持输出期望值不变，这样测试时不需要做任何额外处理。

验证：$E[\tilde{a}_i] = \frac{(1-p) \cdot a_i}{1-p} = a_i$，期望不变。

**为什么有效**：

1. **隐式集成学习**：每次前向传播使用不同的子网络（$n$ 个神经元有 $2^n$ 种子网络），相当于训练了指数级数量的模型并取平均。
2. **打破共适应**：神经元无法依赖特定其他神经元的存在，被迫学习更独立、更鲁棒的特征。
3. **等效正则化**：数学上可以证明 Dropout 等价于对权重施加了一种自适应 L2 正则化。

**怎么用**：

```python
class MyNetwork(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.dropout = nn.Dropout(p=0.5)   # 训练时随机关闭50%的神经元
        self.fc2 = nn.Linear(256, 10)

    def forward(self, x):
        x = torch.relu(self.fc1(x))
        x = self.dropout(x)  # 训练时随机丢弃，测试时自动关闭
        x = self.fc2(x)
        return x

# 关键：训练时 model.train()，测试时 model.eval()
model.train()   # Dropout 生效
model.eval()    # Dropout 关闭，使用全部神经元
```

### 2.5 早停（Early Stopping）

**是什么**：在训练过程中持续监控验证集损失，当验证损失不再下降（甚至开始上升）时提前终止训练。

**为什么有效**：随着训练进行，模型先学到通用规律（验证损失下降），后开始记忆噪声（验证损失上升）。早停恰好在过拟合发生前截断训练。

**怎么用**：

```python
best_val_loss = float('inf')
patience = 10          # 容忍多少个 epoch 不改善
wait = 0
best_state = None

for epoch in range(max_epochs):
    train_one_epoch(model, train_loader, optimizer)
    val_loss = evaluate(model, val_loader)

    if val_loss < best_val_loss:
        best_val_loss = val_loss
        wait = 0
        best_state = model.state_dict().copy()  # 保存最佳模型
    else:
        wait += 1
        if wait >= patience:
            print(f"Early stopping at epoch {epoch}")
            model.load_state_dict(best_state)   # 恢复最佳模型
            break
```

## 三、激活函数与梯度流动

在讨论梯度消失/爆炸之前，我们需要深入理解激活函数的导数性质，因为它是梯度在网络中传播的关键因素。

### 3.1 Sigmoid 函数的导数推导

**Sigmoid 函数**：

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

**导数推导**（利用商法则）：

$$\sigma'(z) = \frac{d}{dz}\left(\frac{1}{1+e^{-z}}\right) = \frac{0 \cdot (1+e^{-z}) - 1 \cdot (-e^{-z})}{(1+e^{-z})^2} = \frac{e^{-z}}{(1+e^{-z})^2}$$

进一步化简（这是一个优美的结果）：

$$\sigma'(z) = \frac{1}{1+e^{-z}} \cdot \frac{e^{-z}}{1+e^{-z}} = \sigma(z) \cdot \frac{e^{-z}}{1+e^{-z}}$$

注意 $1 - \sigma(z) = 1 - \frac{1}{1+e^{-z}} = \frac{e^{-z}}{1+e^{-z}}$，因此：

$$\boxed{\sigma'(z) = \sigma(z)(1 - \sigma(z))}$$

**导数的取值范围分析**：

由于 $\sigma(z) \in (0, 1)$，设 $s = \sigma(z)$，则 $\sigma'(z) = s(1-s)$。

这是一个关于 $s$ 的二次函数，在 $s = 0.5$（即 $z = 0$）时取最大值：

$$\sigma'(z)_{\max} = 0.5 \times 0.5 = 0.25$$

当 $|z|$ 很大时，$\sigma(z)$ 趋近 0 或 1，导数 $\sigma'(z) \to 0$。

**结论**：Sigmoid 的导数始终 $\leq 0.25$，且在大部分输入区间内远小于 0.25。这就是梯度消失的"罪魁祸首"。

### 3.2 ReLU 函数

**定义**：

$$\text{ReLU}(z) = \max(0, z)$$

**导数**：

$$\text{ReLU}'(z) = \begin{cases} 1 & z > 0 \\ 0 & z < 0 \\ \text{未定义} & z = 0 \end{cases}$$

（实践中，$z=0$ 处通常取 0 或 1，不影响训练。）

### 3.3 ReLU 为什么能缓解梯度消失

**核心区别**：

| 性质 | Sigmoid | ReLU |
|------|---------|------|
| 正区间导数 | $\sigma(z)(1-\sigma(z)) \leq 0.25$ | 恒等于 1 |
| 连乘效应（10层） | $0.25^{10} \approx 10^{-6}$ | $1^{10} = 1$ |
| 饱和区域 | $z \to \pm\infty$ 时梯度趋近 0 | $z > 0$ 时梯度恒为 1 |
| 计算复杂度 | 需要指数运算 | 仅需比较运算 |

当使用 Sigmoid 时，反向传播每经过一层，梯度至少缩小为原来的 $\frac{1}{4}$。而使用 ReLU 时，对于被激活的神经元（$z > 0$），梯度可以**无衰减地**向后传播。

**ReLU 的问题**：**Dead ReLU（神经元死亡）** —— 如果某个神经元的输入始终为负值，其梯度永远为 0，参数将无法更新。

**改进变体**：

$$\text{Leaky ReLU}(z) = \begin{cases} z & z > 0 \\ \alpha z & z \leq 0 \end{cases} \quad (\alpha = 0.01 \text{ 等小常数})$$

Leaky ReLU 在负区间也保留了微小梯度 $\alpha$，避免神经元死亡。

## 四、梯度消失与梯度爆炸：完整数学推导

### 4.1 问题的数学根源

考虑一个 $L$ 层的全连接网络，每层的前向传播为：

$$z^{(l)} = W^{(l)} a^{(l-1)} + b^{(l)}, \quad a^{(l)} = f(z^{(l)})$$

其中 $f$ 是激活函数，$a^{(0)} = x$ 是输入。

根据链式法则，损失 $\mathcal{L}$ 对第 $l$ 层权重的梯度为：

$$\frac{\partial \mathcal{L}}{\partial W^{(l)}} = \frac{\partial \mathcal{L}}{\partial a^{(L)}} \cdot \prod_{k=l+1}^{L}\left(\frac{\partial a^{(k)}}{\partial z^{(k)}} \cdot \frac{\partial z^{(k)}}{\partial a^{(k-1)}}\right) \cdot \frac{\partial a^{(l)}}{\partial z^{(l)}} \cdot \frac{\partial z^{(l)}}{\partial W^{(l)}}$$

其中：

$$\frac{\partial a^{(k)}}{\partial z^{(k)}} = \text{diag}(f'(z^{(k)})), \quad \frac{\partial z^{(k)}}{\partial a^{(k-1)}} = W^{(k)}$$

因此梯度中包含一个**连乘项**：

$$\prod_{k=l+1}^{L} \text{diag}(f'(z^{(k)})) \cdot W^{(k)}$$

这个连乘的 $(L - l)$ 个因子决定了梯度的命运。

### 4.2 梯度消失：具体数字推演

**场景**：10 层网络，使用 Sigmoid 激活函数。

Sigmoid 导数最大值为 0.25。假设每层的 $\sigma'(z^{(k)})$ 平均为 0.2，权重 $W^{(k)}$ 的范数约为 1。则每层的梯度缩放因子约为：

$$\|f'(z^{(k)}) \cdot W^{(k)}\| \approx 0.2 \times 1 = 0.2$$

从第 10 层反向传播到第 1 层，梯度经过 9 层缩放：

| 传播到第几层 | 梯度缩放倍数 | 绝对大小（假设初始梯度为1） |
|-------------|-------------|------------------------|
| 第10层（输出层） | $1$ | $1$ |
| 第9层 | $0.2^1$ | $0.2$ |
| 第8层 | $0.2^2$ | $0.04$ |
| 第7层 | $0.2^3$ | $0.008$ |
| 第6层 | $0.2^4$ | $0.0016$ |
| 第5层 | $0.2^5$ | $3.2 \times 10^{-4}$ |
| 第4层 | $0.2^6$ | $6.4 \times 10^{-5}$ |
| 第3层 | $0.2^7$ | $1.28 \times 10^{-5}$ |
| 第2层 | $0.2^8$ | $2.56 \times 10^{-6}$ |
| 第1层 | $0.2^9$ | $5.12 \times 10^{-7}$ |

第1层的梯度仅为输出层的 **五千万分之一**。以学习率 $\alpha = 0.01$ 计算，权重更新量为 $\Delta W^{(1)} = 0.01 \times 5.12 \times 10^{-7} \approx 5 \times 10^{-9}$，几乎为零。

**后果**：浅层参数无法有效学习，深层网络表现反而不如浅层网络（网络退化问题）。

### 4.3 梯度爆炸：具体数字推演

**场景**：10 层网络，权重初始化过大，每层权重范数约为 2.0。

即使使用 ReLU（正区间导数为 1），每层的梯度缩放因子为：

$$\|f'(z^{(k)}) \cdot W^{(k)}\| \approx 1 \times 2.0 = 2.0$$

| 传播到第几层 | 梯度缩放倍数 | 绝对大小 |
|-------------|-------------|---------|
| 第10层 | $1$ | $1$ |
| 第9层 | $2^1$ | $2$ |
| 第8层 | $2^2$ | $4$ |
| 第7层 | $2^3$ | $8$ |
| ... | ... | ... |
| 第1层 | $2^9$ | $512$ |

梯度为 512，权重更新 $\Delta W = 0.01 \times 512 = 5.12$。如果原始权重量级在 $[-1, 1]$，一步更新就完全改变了权重，下一步可能更大，导致权重发散为 NaN/Inf。

### 4.4 核心结论

梯度问题的本质是**连乘效应**：

$$\text{梯度} \propto \prod_{k=l+1}^{L} \|\text{diag}(f'(z^{(k)})) \cdot W^{(k)}\|$$

- 若每个因子 $< 1$：连乘 $\to 0$（**梯度消失**）
- 若每个因子 $> 1$：连乘 $\to \infty$（**梯度爆炸**）
- 若每个因子 $\approx 1$：梯度稳定传播（**理想状态**）

**所有解决方案的目标，都是让每层的梯度缩放因子尽可能接近 1。**

## 五、梯度问题的解决方案

### 5.1 权重初始化

#### 5.1.1 为什么随机初始化很重要

如果所有权重初始化为相同值（比如全 0），那么同一层的所有神经元计算结果相同，梯度也相同，更新后仍然相同。这叫**对称性问题**，网络永远无法学到不同的特征。所以必须随机初始化来打破对称性。

但随机初始化的**方差**至关重要：太大会导致梯度爆炸，太小会导致梯度消失。

#### 5.1.2 Xavier 初始化：详细推导

**目标**：找到权重的初始方差 $\text{Var}(W)$，使得每层输出的方差与输入的方差相等。

**推导过程**：

考虑一层线性变换（暂不考虑激活函数）：

$$z = \sum_{j=1}^{n_{\text{in}}} w_j x_j$$

假设：
- $w_j$ 和 $x_j$ 相互独立
- $E[w_j] = 0$，$E[x_j] = 0$
- 所有 $w_j$ 同分布，所有 $x_j$ 同分布

计算 $z$ 的方差：

$$\text{Var}(z) = \text{Var}\left(\sum_{j=1}^{n_{\text{in}}} w_j x_j\right)$$

由于各项独立：

$$\text{Var}(z) = \sum_{j=1}^{n_{\text{in}}} \text{Var}(w_j x_j)$$

对于两个独立的零均值随机变量 $A, B$，有：

$$\text{Var}(AB) = E[A^2B^2] - (E[AB])^2 = E[A^2]E[B^2] - 0 = \text{Var}(A) \cdot \text{Var}(B)$$

（最后一步用到了 $E[A] = 0$ 所以 $E[A^2] = \text{Var}(A)$，$B$ 同理。）

因此：

$$\text{Var}(z) = \sum_{j=1}^{n_{\text{in}}} \text{Var}(w_j) \cdot \text{Var}(x_j) = n_{\text{in}} \cdot \text{Var}(w) \cdot \text{Var}(x)$$

要使输出方差等于输入方差 $\text{Var}(z) = \text{Var}(x)$，需要：

$$n_{\text{in}} \cdot \text{Var}(w) = 1 \quad \Longrightarrow \quad \boxed{\text{Var}(w) = \frac{1}{n_{\text{in}}}}$$

这是**前向传播**的要求。类似地，从**反向传播**的角度分析（希望梯度的方差也保持稳定），可以得到：

$$\text{Var}(w) = \frac{1}{n_{\text{out}}}$$

Xavier 初始化取两者的折中：

$$\boxed{\text{Var}(w) = \frac{2}{n_{\text{in}} + n_{\text{out}}}}$$

**实际使用**：

- 正态分布形式：$W \sim \mathcal{N}\left(0, \frac{2}{n_{\text{in}} + n_{\text{out}}}\right)$

- 均匀分布形式：$W \sim U[-a, a]$，其中 $\text{Var}(U[-a,a]) = \frac{a^2}{3}$，所以 $\frac{a^2}{3} = \frac{2}{n_{\text{in}} + n_{\text{out}}}$，得到：

$$a = \sqrt{\frac{6}{n_{\text{in}} + n_{\text{out}}}}$$

#### 5.1.3 He 初始化：为 ReLU 修正

Xavier 的推导假设激活函数是线性的（或在零点附近近似线性，如 Tanh）。但 ReLU 会将约一半的值置零，这意味着有效的输出方差会减半：

$$\text{Var}(a) = \text{Var}(\text{ReLU}(z)) \approx \frac{1}{2}\text{Var}(z)$$

（直觉：$z$ 有约一半的概率为负被 ReLU 置零，输出方差减半。）

要补偿这个因子 $\frac{1}{2}$，需要将权重方差翻倍：

$$\text{Var}(w) = \frac{2}{n_{\text{in}}}$$

这就是 **He 初始化**（Kaiming 初始化）。

**PyTorch 代码**：

```python
import torch.nn as nn

# Xavier 初始化（适用于 Sigmoid/Tanh）
nn.init.xavier_uniform_(layer.weight)    # 均匀分布版本
nn.init.xavier_normal_(layer.weight)     # 正态分布版本

# He 初始化（适用于 ReLU）
nn.init.kaiming_uniform_(layer.weight, nonlinearity='relu')
nn.init.kaiming_normal_(layer.weight, nonlinearity='relu')
```

### 5.2 批量归一化（Batch Normalization）

#### 是什么

Batch Normalization（BN）对每一层的输入进行标准化，使其均值为 0、方差为 1，然后通过可学习参数进行缩放和平移。

#### 算法步骤

对一个 mini-batch $\mathcal{B} = \{z_1, z_2, ..., z_m\}$（某一层的未激活输出）：

**Step 1 - 计算 batch 统计量**：

$$\mu_{\mathcal{B}} = \frac{1}{m}\sum_{i=1}^{m} z_i, \quad \sigma_{\mathcal{B}}^2 = \frac{1}{m}\sum_{i=1}^{m}(z_i - \mu_{\mathcal{B}})^2$$

**Step 2 - 标准化**：

$$\hat{z}_i = \frac{z_i - \mu_{\mathcal{B}}}{\sqrt{\sigma_{\mathcal{B}}^2 + \epsilon}}$$

其中 $\epsilon$ 是极小常数（如 $10^{-5}$），防止除以零。

**Step 3 - 缩放和平移（仿射变换）**：

$$y_i = \gamma \hat{z}_i + \beta$$

其中 $\gamma$（缩放）和 $\beta$（平移）是**可学习的参数**。

#### 为什么需要 $\gamma$ 和 $\beta$？

这是一个非常关键的问题。

如果只做 Step 2 的标准化，每层的输出都被强制成均值 0、方差 1 的分布。这会**限制网络的表达能力**：

1. **恒等映射的丧失**：假设某一层已经学到了最优分布，其均值不为 0、方差不为 1。强制标准化会破坏这个已学到的好分布，网络被迫重新学习。

2. **激活函数的有效区间被限制**：以 Sigmoid 为例，标准化后 $\hat{z} \sim \mathcal{N}(0, 1)$，大部分值落在 $[-2, 2]$，这恰好是 Sigmoid 的近似线性区间。网络失去了利用 Sigmoid 非线性部分的能力。

引入 $\gamma$ 和 $\beta$ 后，网络可以学习**恢复**到标准化之前的任意分布。特别地，当 $\gamma = \sqrt{\sigma_{\mathcal{B}}^2 + \epsilon}$，$\beta = \mu_{\mathcal{B}}$ 时，BN 层等价于恒等变换：

$$y_i = \gamma \hat{z}_i + \beta = \sqrt{\sigma_{\mathcal{B}}^2+\epsilon} \cdot \frac{z_i - \mu_{\mathcal{B}}}{\sqrt{\sigma_{\mathcal{B}}^2+\epsilon}} + \mu_{\mathcal{B}} = z_i$$

这意味着 $\gamma$ 和 $\beta$ 给了网络一个"逃生通道"：如果标准化有害，网络可以学会撤销它。但在实践中，BN 通常确实有助于训练，所以 $\gamma$ 和 $\beta$ 会学到不同于恒等映射的值。

#### 训练 vs 推理

| 阶段 | $\mu$ 和 $\sigma^2$ 的来源 |
|------|--------------------------|
| 训练 | 当前 mini-batch 的统计量 |
| 推理 | 训练期间累积的**移动平均** |

推理时使用移动平均的原因：推理时可能只有单个样本，无法计算 batch 统计量；且推理结果需要确定性（不因 batch 组成而变）。

#### 为什么有效

1. **稳定梯度**：标准化使每层输入分布稳定，梯度不会因分布漂移而剧烈变化。
2. **允许更大学习率**：分布稳定后，模型对学习率的敏感性降低。
3. **弱正则化效果**：batch 统计量本身有噪声（因采样随机性），等价于注入了噪声正则化。
4. **减少对初始化的依赖**：即使初始化不完美，BN 也能将分布拉回正常范围。

### 5.3 梯度裁剪（Gradient Clipping）

**是什么**：当梯度范数超过设定阈值时，按比例缩小梯度向量，保持方向不变但限制大小。

**为什么有效**：直接阻止梯度爆炸，是 RNN/LSTM 训练中的标配手段。

**数学表述**：

$$\boldsymbol{g} \leftarrow \begin{cases} \boldsymbol{g} & \|\boldsymbol{g}\| \leq \tau \\ \tau \cdot \frac{\boldsymbol{g}}{\|\boldsymbol{g}\|} & \|\boldsymbol{g}\| > \tau \end{cases}$$

其中 $\tau$ 是阈值（如 1.0 或 5.0），$\boldsymbol{g}$ 是梯度向量。

**怎么用**：

```python
# PyTorch 一行搞定
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

### 5.4 残差连接（Residual Connection）

**是什么**：在网络中引入"跳跃连接"，让输入可以绕过若干层直接加到输出上。

$$\text{输出} = F(x) + x$$

其中 $F(x)$ 是残差块中几层网络的变换。网络只需学习残差 $F(x) = \text{输出} - x$，而不是完整的映射。

**为什么能解决梯度消失**：

$$\frac{\partial \mathcal{L}}{\partial x} = \frac{\partial \mathcal{L}}{\partial (F(x)+x)} \cdot \left(\frac{\partial F(x)}{\partial x} + 1\right)$$

即使 $\frac{\partial F(x)}{\partial x}$ 很小（趋近 0），加上恒等项 $+1$ 后，梯度至少为 $\frac{\partial \mathcal{L}}{\partial (F(x)+x)} \cdot 1$，保证了梯度可以无衰减地流回浅层。这就是 ResNet 能训练 152 层甚至 1000+ 层网络的秘密。

## 六、Mini-batch 梯度下降

### 6.1 三种梯度下降的对比

根据每次参数更新所使用的样本数量，梯度下降分为三种：

| 方法 | 每次更新用的样本数 | 梯度计算 |
|------|------------------|---------|
| 批量梯度下降（Batch GD） | 全部 $N$ 个样本 | $\nabla L = \frac{1}{N}\sum_{i=1}^{N}\nabla L_i$ |
| 随机梯度下降（SGD） | 1 个样本 | $\nabla L = \nabla L_i$ |
| **Mini-batch GD** | $B$ 个样本（如 32, 64, 128） | $\nabla L = \frac{1}{B}\sum_{i=1}^{B}\nabla L_i$ |

### 6.2 为什么用 Mini-batch

**Batch GD 的问题**：

- 需要把整个数据集载入内存，对大数据集不可行（ImageNet 有 1400 万张图）
- 每个 epoch 只更新一次参数，收敛极慢
- 梯度精确但太"稳定"，容易陷入尖锐的局部最小值

**纯 SGD（batch size = 1）的问题**：

- 梯度噪声太大，更新方向不稳定，loss 曲线剧烈震荡
- 无法利用 GPU 的并行计算能力（每次只处理一个样本）

**Mini-batch 的优势**：

1. **内存可控**：一次只载入 $B$ 个样本
2. **GPU 并行友好**：矩阵运算在 $B$ 个样本上并行执行
3. **适度的梯度噪声**：梯度估计比 SGD 更稳定，同时保留一定随机性帮助逃离局部最小值
4. **每个 epoch 更新 $N/B$ 次**：比 Batch GD 快得多

### 6.3 Batch Size 的影响

| Batch Size | 梯度估计 | 训练速度 | 泛化能力 | GPU利用率 |
|-----------|---------|---------|---------|----------|
| 很小（8-16） | 噪声大 | 慢（更新频繁但每步慢） | 通常更好（噪声有正则化效果） | 低 |
| 中等（32-128） | 适中 | 最佳平衡 | 良好 | 高 |
| 很大（512+） | 精确 | 每步快但收敛需更多epoch | 可能更差（尖锐最小值） | 最高 |

**实践经验**：通常从 32 或 64 开始尝试。大 batch size 训练时往往需要配合**学习率热身（warmup）**和**线性缩放规则**（batch size 翻倍则学习率也翻倍）。

**PyTorch 中的 Mini-batch**：

```python
from torch.utils.data import DataLoader, TensorDataset

# 创建数据加载器
dataset = TensorDataset(X_train, y_train)
train_loader = DataLoader(
    dataset,
    batch_size=64,       # mini-batch 大小
    shuffle=True,        # 每个 epoch 随机打乱顺序
    drop_last=True,      # 丢弃最后一个不完整的 batch
)

# 训练循环
for epoch in range(num_epochs):
    for batch_x, batch_y in train_loader:  # 每次迭代取一个 mini-batch
        output = model(batch_x)
        loss = criterion(output, batch_y)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()    # 每个 mini-batch 更新一次参数
```

## 七、优化器演化：从 SGD 到 Adam

所有优化器的出发点都是基本的梯度下降：$W \leftarrow W - \alpha \nabla \mathcal{L}$。但这个朴素版本存在诸多问题，后续的优化器逐步解决了这些问题。

### 7.1 SGD（Stochastic Gradient Descent）

**是什么**：最基础的优化器，直接沿梯度方向更新参数。

$$W_{t+1} = W_t - \alpha \nabla \mathcal{L}(W_t)$$

**问题**：
- 在"峡谷"型损失面上震荡严重（某些方向梯度大、某些方向梯度小）
- 在平坦区域移动缓慢
- 容易陷入鞍点（梯度为零但不是最小值）

### 7.2 SGD with Momentum（动量法）

**是什么**：引入"速度"变量，累积历史梯度方向，让更新具有惯性。

**物理直觉**：想象一个小球在损失曲面上滚动。没有动量时，小球在每个点都只看当前坡度决定方向；有动量后，小球会保持之前的运动趋势，在一致方向上加速，在震荡方向上因正负抵消而减速。

**公式**：

$$v_t = \beta v_{t-1} + (1 - \beta) \nabla \mathcal{L}(W_t)$$

$$W_{t+1} = W_t - \alpha v_t$$

其中：
- $v_t$ 是速度向量（梯度的指数加权移动平均）
- $\beta \in [0.9, 0.99]$ 是动量系数。$\beta = 0.9$ 表示大约平均最近 $\frac{1}{1-\beta} = 10$ 步的梯度。

**为什么有效**：
- 一致方向上的梯度被累积放大，加速收敛
- 来回震荡方向上的梯度正负抵消，减少震荡
- 惯性帮助穿越局部极小值和鞍点

### 7.3 RMSProp（Root Mean Square Propagation）

**是什么**：为每个参数维护一个梯度平方的移动平均，用它来自适应地调整每个参数的学习率。

**核心思想**：梯度大的参数用小学习率，梯度小的参数用大学习率，实现自适应调节。

**公式**：

$$s_t = \rho \, s_{t-1} + (1 - \rho)(\nabla \mathcal{L})^2$$

$$W_{t+1} = W_t - \frac{\alpha}{\sqrt{s_t + \epsilon}} \nabla \mathcal{L}$$

其中：
- $s_t$ 是梯度平方的指数加权移动平均（二阶矩估计）
- $\rho \approx 0.9$ 是衰减率
- $\epsilon \approx 10^{-8}$ 防止除以零

**为什么有效**：

$$\frac{\alpha}{\sqrt{s_t + \epsilon}}$$

这个自适应学习率会自动调节：
- 当历史梯度一直很大时，$s_t$ 大，有效学习率 $\frac{\alpha}{\sqrt{s_t}}$ 小 --> 放慢脚步
- 当历史梯度一直很小时，$s_t$ 小，有效学习率 $\frac{\alpha}{\sqrt{s_t}}$ 大 --> 加快脚步

### 7.4 Adam（Adaptive Moment Estimation）

**是什么**：结合 Momentum（一阶矩）和 RMSProp（二阶矩）的优点，并增加了偏差修正。

**完整算法**（初始化 $m_0 = 0, v_0 = 0, t = 0$）：

$$t = t + 1$$

**一阶矩估计（动量）**：

$$m_t = \beta_1 m_{t-1} + (1 - \beta_1) \nabla \mathcal{L}$$

**二阶矩估计（自适应学习率）**：

$$v_t = \beta_2 v_{t-1} + (1 - \beta_2) (\nabla \mathcal{L})^2$$

**偏差修正**：

$$\hat{m}_t = \frac{m_t}{1 - \beta_1^t}, \quad \hat{v}_t = \frac{v_t}{1 - \beta_2^t}$$

**参数更新**：

$$W_{t+1} = W_t - \alpha \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon}$$

默认超参数：$\beta_1 = 0.9, \; \beta_2 = 0.999, \; \alpha = 0.001, \; \epsilon = 10^{-8}$。

#### 偏差修正为什么必要？（含纠正的计算示例）

初始化时 $m_0 = 0$，这导致早期的 $m_t$ 系统性地偏向零。

**第 1 步**（$t=1$），设真实梯度为 $g$：

$$m_1 = \beta_1 \cdot m_0 + (1 - \beta_1) \cdot g = 0.9 \times 0 + 0.1 \times g = 0.1g$$

$m_1 = 0.1g$，仅为真实梯度的 10%。这是严重的低估。

**偏差修正**：

$$\hat{m}_1 = \frac{m_1}{1 - \beta_1^1} = \frac{0.1g}{1 - 0.9} = \frac{0.1g}{0.1} = g$$

修正后恢复为完整的梯度估计 $g$。

**第 2 步**（$t=2$），假设梯度仍为 $g$：

$$m_2 = 0.9 \times 0.1g + 0.1 \times g = 0.09g + 0.1g = 0.19g$$

$$\hat{m}_2 = \frac{0.19g}{1 - 0.9^2} = \frac{0.19g}{1 - 0.81} = \frac{0.19g}{0.19} = g$$

修正后仍为 $g$，符合预期。

**后期**（$t$ 很大时），$\beta_1^t \to 0$，修正因子 $\frac{1}{1-\beta_1^t} \to 1$，修正自动消失。

对 $v_t$ 的偏差修正原理完全相同，只是用 $\beta_2$ 替代 $\beta_1$。

#### 优化器对比总结

| 特性 | SGD | Momentum | RMSProp | Adam |
|-----|-----|----------|---------|------|
| 梯度累积（一阶矩） | 无 | 有 | 无 | 有 |
| 自适应学习率（二阶矩） | 无 | 无 | 有 | 有 |
| 偏差修正 | 无 | 无 | 无 | 有 |
| 超参数数量 | 1（$\alpha$） | 2（$\alpha, \beta$） | 2（$\alpha, \rho$） | 3（$\alpha, \beta_1, \beta_2$） |
| 对稀疏梯度的处理 | 差 | 一般 | 好 | 好 |
| 推荐场景 | 精调/研究对比 | GAN训练 | RNN/NLP | **通用首选** |

**实践建议**：
- 默认使用 Adam，学习率 $\alpha = 10^{-3}$ 或 $3 \times 10^{-4}$
- $\beta_1, \beta_2$ 极少需要调整，保持默认即可
- 如果需要更好的泛化，考虑 **AdamW**（Adam with decoupled weight decay）

## 八、完整训练框架示例

将本笔记所有技术整合到一个 PyTorch 训练框架中：

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

# ==================== 模型定义 ====================
class RegularizedNet(nn.Module):
    """整合 BN + Dropout + He初始化 的网络"""

    def __init__(self, input_dim=784, hidden_dim=256, num_classes=10, dropout_p=0.3):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.BatchNorm1d(hidden_dim),     # 批量归一化
            nn.ReLU(),                       # ReLU激活
            nn.Dropout(dropout_p),           # Dropout

            nn.Linear(hidden_dim, hidden_dim),
            nn.BatchNorm1d(hidden_dim),
            nn.ReLU(),
            nn.Dropout(dropout_p),

            nn.Linear(hidden_dim, num_classes),  # 输出层不加BN/Dropout
        )
        self._init_weights()

    def _init_weights(self):
        """He初始化（配合ReLU）"""
        for m in self.modules():
            if isinstance(m, nn.Linear):
                nn.init.kaiming_normal_(m.weight, nonlinearity='relu')
                nn.init.zeros_(m.bias)

    def forward(self, x):
        return self.net(x)


# ==================== 训练配置 ====================
model = RegularizedNet()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)  # L2正则化
criterion = nn.CrossEntropyLoss()
max_epochs = 100
patience = 10

# 假设已有数据
# train_loader = DataLoader(train_dataset, batch_size=64, shuffle=True)
# val_loader = DataLoader(val_dataset, batch_size=256, shuffle=False)

# ==================== 训练循环 ====================
best_val_loss = float('inf')
wait = 0

for epoch in range(max_epochs):
    # --- 训练阶段 ---
    model.train()                  # 启用 Dropout 和 BN 的训练模式
    for batch_x, batch_y in train_loader:
        logits = model(batch_x)
        loss = criterion(logits, batch_y)

        optimizer.zero_grad()
        loss.backward()
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)  # 梯度裁剪
        optimizer.step()

    # --- 验证阶段 ---
    model.eval()                   # 关闭 Dropout，BN使用移动平均
    val_loss = 0.0
    with torch.no_grad():
        for batch_x, batch_y in val_loader:
            logits = model(batch_x)
            val_loss += criterion(logits, batch_y).item()
    val_loss /= len(val_loader)

    # --- 早停 ---
    if val_loss < best_val_loss:
        best_val_loss = val_loss
        wait = 0
        torch.save(model.state_dict(), 'best_model.pt')
    else:
        wait += 1
        if wait >= patience:
            print(f"Early stopping at epoch {epoch}")
            model.load_state_dict(torch.load('best_model.pt'))
            break

    print(f"Epoch {epoch}: val_loss={val_loss:.4f}, patience={wait}/{patience}")
```

## 九、关键要点总结

| 主题 | 核心结论 |
|------|---------|
| **过拟合的本质** | 偏差-方差权衡，模型复杂度超过数据信息量 |
| **梯度消失** | 激活函数导数 $< 1$ 的连乘效应：$0.25^{L} \to 0$ |
| **梯度爆炸** | 权重范数 $> 1$ 的连乘效应：$c^{L} \to \infty$ |
| **ReLU 的优势** | 正区间导数恒为 1，打断梯度衰减链 |
| **Xavier 初始化** | $\text{Var}(w) = \frac{2}{n_{\text{in}}+n_{\text{out}}}$，保持各层方差一致 |
| **He 初始化** | $\text{Var}(w) = \frac{2}{n_{\text{in}}}$，补偿 ReLU 的半区置零 |
| **BN 中 $\gamma, \beta$** | 恢复网络表达能力，使标准化可逆 |
| **Mini-batch** | 兼顾 GPU 效率和梯度噪声的正则化效果 |
| **Adam 偏差修正** | 补偿零初始化的低估：$\hat{m}_t = m_t / (1-\beta_1^t)$ |

**防过拟合策略速查**：

```
训练 loss 不下降     --> 检查学习率、权重初始化、网络是否太浅
验证 loss >> 训练 loss --> 加 Dropout、L2正则化、数据增强
权重变成 NaN        --> 梯度裁剪、降低学习率、检查初始化
收敛太慢            --> 换用 Adam、调大学习率、加 BN
```
