# 3.3 从矩阵到 CNN

> 从全连接层的参数爆炸问题出发，理解卷积神经网络（CNN）如何用局部感知和权重共享解决图像识别中的效率与空间感知问题。

---

## 1. 全连接层的问题

### 1.1 全连接层结构回顾

在 [[3.1 神经网络]] 中，每一层的计算是：

$$z^{(l)} = W^{(l)} \cdot a^{(l-1)} + b^{(l)}$$

$$a^{(l)} = \sigma\!\left(z^{(l)}\right)$$

其中：
- $W^{(l)}$ 是第 $l$ 层的权重矩阵
- $\mathbf{b}^{(l)}$ 是偏置向量
- $\sigma$ 是激活函数（如 ReLU, Sigmoid）

**全连接层**：该层的每一个神经元都与前一层的**所有**神经元相连。

![[Pasted image 20260216221613.png]]

![[Pasted image 20260216221638.png]]

![[Pasted image 20260216221729.png]]

那么如何区分层有多少呢？神经元的层没有体现在公式中，用上标 $l$ 表示在第几层：

$$W^{(l)},\quad b^{(l)},\quad z^{(l)},\quad a^{(l)} \quad \Longleftarrow \text{第 } l \text{ 层的量}$$

![[Pasted image 20260216221841.png]]

什么是全连接层？就是该层的每一个神经元都跟前一层的神经元相连，这个就是全连接层。

![[Pasted image 20260216221950.png]]

### 1.2 参数爆炸问题

但是这样引出了一个问题：对于这样一个 30×30 像素的图片，假如它的下一层有 1000 个神经元，那么全连接层的总参数量就达到了 90 万，而且没法保留像素之间的空间关系。

![[Pasted image 20260216222247.png]]

**参数量计算**：

$$\text{参数量} = \underbrace{(30 \times 30)}_{\text{输入像素}} \times \underbrace{1000}_{\text{神经元数}} = 900{,}000$$

对于更实际的场景，以 224×224 的 RGB 图片为例：

$$\text{参数量} = (224 \times 224 \times 3) \times 1000 = 150{,}528{,}000 \approx 1.5 \text{ 亿}$$

这带来了两个核心问题：

| 问题 | 说明 |
|------|------|
| **参数爆炸** | 训练慢、内存占用大、易过拟合 |
| **空间信息丢失** | 图像展开为一维向量时，像素的空间位置关系被破坏 |

## 2. 卷积运算原理

### 2.1 什么是卷积？

此时加上一个固定的矩阵，两个矩阵进行一个矩阵运算，对应位置的值相乘再相加。这样遍历图像的每一个像素的，得出的结果再组成一个灰度照片，这个运算过程就是卷积运算，固定的矩阵就是卷积核。

![[Pasted image 20260216222247.png]]

**卷积的直觉**：用一个小窗口（卷积核）在图像上**滑动扫描**，每次只看局部区域。

通过卷积核可以将图片进行模糊、锐化、轮廓处理。在深度学习领域卷积核是未知的，他是被训练出来的一组值，把全连接层替换为卷积层，这样可以大大的减少权重参数的数量，还可以捕捉到空间的局部特征。

### 2.2 卷积运算过程（详解）

假设输入图像为 $5 \times 5$，卷积核为 $3 \times 3$：

```
输入图像 (5×5):             卷积核 (3×3):
┌───┬───┬───┬───┬───┐       ┌───┬───┬───┐
│ 1 │ 2 │ 3 │ 0 │ 1 │       │ 1 │ 0 │-1 │
├───┼───┼───┼───┼───┤       ├───┼───┼───┤
│ 4 │ 5 │ 6 │ 1 │ 0 │       │ 1 │ 0 │-1 │
├───┼───┼───┼───┼───┤       ├───┼───┼───┤
│ 7 │ 8 │ 9 │ 2 │ 1 │       │ 1 │ 0 │-1 │
├───┼───┼───┼───┼───┤       └───┴───┴───┘
│ 0 │ 1 │ 2 │ 3 │ 0 │
├───┼───┼───┼───┼───┤       ← 这个卷积核可以检测
│ 1 │ 0 │ 1 │ 4 │ 1 │         垂直边缘特征
└───┴───┴───┴───┴───┘
```

**第一步**：卷积核放在左上角，逐元素相乘再求和：

$$\text{输出}_{(0,0)} = 1\times1 + 2\times0 + 3\times(-1) + 4\times1 + 5\times0 + 6\times(-1) + 7\times1 + 8\times0 + 9\times(-1) = -6$$

**第二步**：卷积核向右滑动一格，重复计算...

**输出特征图 (Feature Map)** 大小为 $3 \times 3$（步长=1，无padding）：

$$\text{输出尺寸} = \left\lfloor \frac{N + 2P - K}{S} \right\rfloor + 1$$

其中 $N$ = 输入尺寸，$K$ = 卷积核尺寸，$S$ = 步长（stride），$P$ = 填充大小（padding，无填充时 $P=0$）

### 2.3 卷积数学公式

在公式上就是**全连接层的矩阵乘法变成了卷积运算**：

**全连接层的前向传播**：

$$z^{(l)}_j = \sum_{i} W^{(l)}_{ji} \cdot a^{(l-1)}_i + b^{(l)}_j$$

- 第 $j$ 个神经元对前一层**所有输出**求和
- 每个输出神经元有**不同的权重** $W_{ji}$

**卷积层的前向传播**：

$$z^{(l)}_{k,i,j} = \sum_{c=0}^{C-1}\sum_{m=0}^{K-1} \sum_{n=0}^{K-1} W^{(l)}_{k,c,m,n} \cdot a^{(l-1)}_{c,\,i+m,\,j+n} + b^{(l)}_k$$

- 只对**局部** $K \times K$ 区域求和（局部感知）
- 同一输出通道 $k$ 的所有位置 $(i,j)$ **共用同一组** $W_k$（权重共享）；不同输出通道使用不同卷积核

![[Pasted image 20260216223148.png]]

![[Pasted image 20260216223249.png]]

**卷积层的前向传播公式详解**：

$$z^{(l)}_{k,i,j} = \sum_{c=0}^{C-1}\sum_{m=0}^{K-1} \sum_{n=0}^{K-1} W^{(l)}_{k,c,m,n} \cdot a^{(l-1)}_{c,\,i+m,\,j+n} + b^{(l)}_k$$

其中：
- $k$：输出通道索引（第 $k$ 个卷积核产生第 $k$ 个特征图）
- $c$：输入通道索引，$C$ 为输入通道总数
- $W^{(l)}_{k,c,m,n}$：第 $l$ 层第 $k$ 个卷积核在输入通道 $c$ 上第 $(m,n)$ 个权重（**可训练参数**）
- $a^{(l-1)}_{c,\,i+m,\,j+n}$：上一层通道 $c$ 在 $(i+m, j+n)$ 位置的激活值
- $K$：卷积核的空间大小（如 $K=3$ 表示 3×3 卷积核）
- $b^{(l)}_k$：第 $k$ 个卷积核的偏置项

**与全连接层对比**：全连接层用矩阵乘法 $W \cdot x$，卷积层用滑动窗口的点积，本质是**带有权重共享的稀疏矩阵乘法**。

### 2.4 卷积核的物理意义

**举例：不同卷积核的作用**

```
模糊卷积核（均值滤波）:    边缘检测卷积核:
┌─────┬─────┬─────┐       ┌────┬────┬────┐
│ 1/9 │ 1/9 │ 1/9 │       │ -1 │ -1 │ -1 │
├─────┼─────┼─────┤       ├────┼────┼────┤
│ 1/9 │ 1/9 │ 1/9 │       │ -1 │  8 │ -1 │
├─────┼─────┼─────┤       ├────┼────┼────┤
│ 1/9 │ 1/9 │ 1/9 │       │ -1 │ -1 │ -1 │
└─────┴─────┴─────┘       └────┴────┴────┘
  → 模糊图片                 → 提取轮廓边缘
```

在 CNN 中，这些卷积核的值**不是人工设定的**，而是通过反向传播**自动学习**出来的。

## 3. CNN 的参数效率

### 3.1 参数量对比

**全连接层**（30×30 图片 → 1000 神经元）：

$$\text{参数量} = 900 \times 1000 = 900{,}000$$

**卷积层**（相同场景，3×3 卷积核，32 个卷积核）：

$$\text{参数量} = \underbrace{3 \times 3}_{\text{卷积核大小}} \times \underbrace{32}_{\text{卷积核数量}} + \underbrace{32}_{\text{偏置}} = 288 + 32 = 320$$

> **关键洞察**：参数量从 90 万降到了 320，减少了 **2800 倍**，核心原因是**权重共享**。

### 3.2 两个核心思想

| 思想 | 含义 | 效果 |
|------|------|------|
| **局部感知**（Local Receptive Field）| 每个神经元只看图像的一个局部区域 | 保留空间位置信息 |
| **权重共享**（Weight Sharing）| 同一个卷积核扫描整张图时，权重不变 | 大幅减少参数量 |

**权重共享的直觉**：检测"猫耳朵"的特征应该无论出现在图像的哪个位置都能被识别，所以用同一个卷积核扫描整张图是合理的。

## 4. 特征图（Feature Map）

### 4.1 什么是特征图？

一个卷积核对输入图像进行完整扫描后，得到的输出矩阵称为**特征图（Feature Map）**。

```
输入图像 (H×W×C)         卷积核1 → 特征图1 (边缘)
                     →   卷积核2 → 特征图2 (纹理)   →  输出 (H'×W'×N)
                         卷积核N → 特征图N (形状)
```

- 每个卷积核学习检测一种特定的局部特征
- $N$ 个卷积核 → 输出 $N$ 个特征图（即输出的**深度/通道数** = $N$）

### 4.2 多通道卷积（RGB 图片）

彩色图片有 3 个通道（R、G、B），卷积核也需要有对应的深度：

$$\text{3×3 卷积核 (RGB)} = 3 \times 3 \times 3 = 27 \text{ 个参数}$$

**多通道卷积过程**：

```
输入: H×W×3 (RGB)
      ┌─────────────┐
      │    R 通道    │  ×  卷积核的 R 切片 (3×3)
      ├─────────────┤
      │    G 通道    │  ×  卷积核的 G 切片 (3×3)  → 求和 → 一个输出特征图
      ├─────────────┤
      │    B 通道    │  ×  卷积核的 B 切片 (3×3)
      └─────────────┘
```

## 5. 步长（Stride）与填充（Padding）

### 5.1 步长

卷积核每次滑动的距离：

| 步长 S | 效果 | 输出尺寸 |
|--------|------|---------|
| S=1（默认）| 密集扫描，输出尺寸接近输入 | $\left\lfloor\frac{N+2P-K}{1}\right\rfloor+1$ |
| S=2 | 跳步扫描，输出尺寸约减半 | $\left\lfloor\frac{N+2P-K}{2}\right\rfloor+1$ |

### 5.2 填充

在图像边缘补充 0（Zero Padding），避免边缘信息丢失：

| Padding | 说明 | 输出尺寸 |
|---------|------|---------|
| `valid`（无填充）| 输出尺寸 < 输入 | $N - K + 1$ |
| `same`（有填充）| 输出尺寸 = 输入 | $N$（步长=1时）|

## 6. 池化层（Pooling Layer）

池化层是对卷积层进行降维的。

![[Pasted image 20260216223312.png]]

### 6.1 池化的目的

池化是对卷积层进行**空间降维**：

| 目标        | 说明              |
| --------- | --------------- |
| **降低计算量** | 特征图尺寸减小，后续层参数更少 |
| **增强鲁棒性** | 对输入的小位移不敏感（卷积本身具有平移等变性，池化进一步提供一定程度的平移不变性）|
| **扩大感受野** | 后续卷积层能感知更大的图像范围 |
|           |                 |

### 6.2 最大池化 vs 平均池化

**最大池化（Max Pooling）**：取窗口内的最大值，保留最显著的特征。

```
输入 (4×4):                    最大池化 (2×2, 步长2):
┌────┬────┬────┬────┐          ┌────┬────┐
│  1 │  3 │  2 │  4 │          │  6 │  4 │   取每个2×2块
├────┼────┼────┼────┤   →      ├────┼────┤
│  5 │  6 │  1 │  2 │          │  2 │  3 │
├────┼────┼────┼────┤          └────┴────┘
│  1 │  2 │  3 │  0 │
├────┼────┼────┼────┤
│  0 │  1 │  0 │  3 │
└────┴───-┴────┴────┘
的最大值
```

**平均池化（Average Pooling）**：取窗口内的平均值，保留整体激活水平。

## 7. 完整 CNN 架构

卷积层和池化层可以有多个。

### 7.1 典型 CNN 结构

```
输入图像
  │
  ↓
[卷积层 Conv1] → 激活函数 ReLU → [池化层 Pool1]
  │              提取低层特征       降维
  ↓
[卷积层 Conv2] → 激活函数 ReLU → [池化层 Pool2]
  │              提取高层特征       降维
  ↓
[展平 Flatten]  ← 将三维特征图展为一维向量
  │
  ↓
[全连接层 FC]  ← 综合特征，进行分类
  │
  ↓
[Softmax 输出]  ← 得到各类别的概率
```

### 7.2 层级特征提取

CNN 的层次越深，提取的特征越抽象：

```
第1层卷积:  学习边缘、颜色等底层特征
第2层卷积:  学习纹理、形状等中层特征
第3层卷积:  学习眼睛、轮子等高层特征（对象的局部）
全连接层:   综合所有特征，判断"这是一只猫"
```

## 8. CNN 与 Transformer 的关系

CNN 是 Transformer 出现之前**图像领域的主导架构**。理解 CNN 的局限性有助于理解 ViT 在大数据、大模型场景下的优势（两者目前仍并行使用）：

| 特性 | CNN | Transformer（ViT）|
|------|-----|-----------------|
| 感受野 | 局部（由卷积核大小决定）| **全局**（所有 patch 互相注意）|
| 归纳偏置 | 强（平移等变性，配合池化后具有一定平移不变性）| 弱（需要更多数据）|
| 参数效率 | 高（权重共享）| 相对低（但可扩展）|
| 长距离依赖 | 需要很深的网络才能捕捉 | **天然支持** |

## 关键要点总结

1. **全连接层的两大问题**：参数爆炸（无法扩展到高分辨率图片）+ 空间信息丢失
2. **卷积 = 局部感知 + 权重共享**：用同一个小卷积核扫描整张图
3. **卷积核是训练出来的**：通过反向传播自动学习检测什么特征
4. **特征图的通道数 = 卷积核的数量**：每个卷积核负责检测一种特征
5. **池化层**：空间降维，提升平移鲁棒性，扩大感受野
6. **CNN 的层次性**：低层→边缘，高层→语义
