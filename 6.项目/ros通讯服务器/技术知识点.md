## 项目架构概览

本项目是一个**生产级的分布式机器人通讯服务器**，采用分层架构设计：

```
┌─────────────────────────────────────────────────────┐
│              应用处理层 (ServerHandler)              │
│                  主业务逻辑分派                      │
├─────────────────────────────────────────────────────┤
│              业务服务层 (Services)                   │
│  ┌──────────────┬──────────────┬────────────────┐  │
│  │ DB Bridge    │  ROS Bridge  │  File Manager  │  │
│  │ 数据库工作池  │  ROS节点桥接  │  文件上传管理   │  │
│  └──────────────┴──────────────┴────────────────┘  │
├─────────────────────────────────────────────────────┤
│              网络反应器层 (Reactor)                  │
│       Epoll事件驱动 | 会话管理 | 超时处理            │
├─────────────────────────────────────────────────────┤
│              协议处理层 (Protocol)                   │
│       帧序列化 | 状态机解析 | CRC32校验              │
└─────────────────────────────────────────────────────┘
```

### 3. 原子操作

项目使用 `std::atomic` 实现无锁操作：

**相关代码 (`queue.hpp:96-104`)**：
```cpp
bool push(const T& item) {
    std::size_t head = head_.load(std::memory_order_relaxed);
    std::size_t next = (head + 1) % Capacity;
    if (next == tail_.load(std::memory_order_acquire)) {
        return false;  // 队列满
    }
    buffer_[head] = item;
    head_.store(next, std::memory_order_release);
    return true;
}
```

## 设计模式应用

### 1. Reactor 模式

项目核心架构采用 Reactor 模式：

```
┌─────────────────────────────────────────┐
│           Reactor (事件分发器)           │
│  ┌─────────────────────────────────┐   │
│  │        Epoll 事件循环            │   │
│  │   监听 → 接收 → 分发 → 处理      │   │
│  └─────────────────────────────────┘   │
│              ↓ 事件                     │
│  ┌─────────────────────────────────┐   │
│  │    IReactorHandler (处理器)      │   │
│  │    onFrame / onDisconnect       │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

这符合《Linux多线程服务端编程》中的 "One Loop Per Thread" 模型。

## 协议设计

### 1. 二进制协议帧结构

项目设计了紧凑的 24 字节协议头：

```
偏移   字段            大小    说明
0      Magic           4B      0x4D464D53 ("MFMS")
4      Version         2B      协议版本 (0x0001)
6      MsgType         2B      消息类型
8      Flags           4B      标志位
12     RequestId       4B      请求ID（用于请求-响应匹配）
16     PayloadLen      4B      载荷长度（最大32MB）
20     HeaderCRC32     4B      头部CRC校验
```

**标志位设计 (`frame.hpp:44-48`)**：
```cpp
enum class FrameFlags : std::uint32_t {
    NONE            = 0,
    REQUIRES_ACK    = 1 << 0,  // 需要确认
    COMPRESSED      = 1 << 1,  // 已压缩
    HAS_CRC         = 1 << 2,  // 有CRC校验
};
```
