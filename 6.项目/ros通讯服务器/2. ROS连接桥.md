# ROS 连接桥 (MfmsRosBridge)

> MfmsRosBridge 是连接 ROS 2 Topic 与 Qt 前端的桥接服务，负责从数据库查询在线设备、订阅设备状态 Topic、将实时数据通过 Qt 信号推送给 GUI。

---

## 1. 设计背景

### 1.1 问题分析

在混合机器人控制系统中，Qt GUI 需要实时显示机器人状态（位姿、IO、错误码等），但面临以下挑战：

| 挑战 | 说明 |
|------|------|
| 通信框架差异 | ROS 2 使用发布订阅模型，Qt 使用信号槽机制 |
| 线程模型不同 | ROS 回调在 executor 线程执行，Qt GUI 在主线程 |
| 设备类型多样 | 法奥机器人、华数机器人、Seer AGV 使用不同消息类型 |
| 动态设备发现 | 设备在运行时上下线，需要动态订阅 |

### 1.2 设计目标

```
┌──────────────────────────────────────────────────────────────┐
│                        Qt GUI 主线程                          │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐       │
│  │ 设备选择框   │    │ 状态面板    │    │ 位姿显示    │       │
│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘       │
│         │                  │                  │              │
│         │    Qt 信号       │                  │              │

│  ┌──────────────────────────────────────────────────┐        │
```

## 2. 核心数据结构

### 2.1 设备类型枚举

系统支持多种异构设备，通过枚举区分：

```cpp
enum class DeviceType {
    Unknown = 0,
    FrRobot,      // 法奥机器人 (FANUC 兼容)
    HsRobot,      // 华数机器人
    SeerAgv,      // 仙工 AGV
    HkCamera,     // 海康相机
    YoloCamera,   // YOLO 相机
    Kossi         // Kossi 控制器
};
```

**设计思路**：使用枚举而非字符串，编译期类型检查更安全，switch 语句可以检查是否覆盖所有情况。

### 2.2 设备类型推断

设备类型可能存储在数据库的 JSON 字段中，也可能需要从设备名称推断：

```cpp
inline DeviceType inferDeviceType(const QString& name, const QString& typeName) {
    // 优先根据类型名称判断（来自数据库 info.type）
    if (typeName.contains("法奥") || typeName.contains("FANUC")) {
        return DeviceType::FrRobot;
    }
    if (typeName.contains("华数")) {
        return DeviceType::HsRobot;
    }
    if (typeName.contains("AGV") || typeName.contains("Seer")) {
        return DeviceType::SeerAgv;
    }

    // 根据名称前缀判断（约定：Fr1, Hs1, Seer_Agv 等）
    if (name.startsWith("Fr", Qt::CaseInsensitive)) {
        return DeviceType::FrRobot;
    }
    if (name.startsWith("Hs", Qt::CaseInsensitive)) {
        return DeviceType::HsRobot;
    }
    // ...

    return DeviceType::Unknown;
}
```

**关键点**：
- 双重判断策略：先查类型名（准确），再查名称前缀（兼容）
- 使用 `Qt::CaseInsensitive` 忽略大小写
- 返回 `Unknown` 而非抛异常，允许优雅降级

### 2.3 在线设备信息 (OnlineDevice)

封装从数据库查询到的设备信息：

```cpp
struct OnlineDevice {
    QString id;              // 设备ID (主键，如 "rbafra0001")
    QString name;            // 设备名称 (用于Topic: /{name}_state)
    DeviceType type;         // 设备类型
    QString typeName;        // 类型名称原始字符串
    QString state;           // 当前状态 ("online" 或 "connected")
    QString ip;              // IP 地址
    int groupId = 0;         // 设备组 ID

    // 判断设备是否可用
    bool isAvailable() const {
        return state == "online" || state == "connected";
    }

    // 生成 ROS Topic 名称
    QString topicName() const {
        return QString("/%1_state").arg(name);  // 如 "/Fr1_state"
    }

    // 生成下拉框显示文本
    QString displayText() const {
        QString stateText = (state == "connected") ? "已连接" : "在线";
        return QString("%1 (%2) - %3").arg(name, ip, stateText);
    }

    // 从 JSON 解析 info 字段
    void parseInfoJson(const QJsonObject& info);
};
```

**设计要点**：
- `topicName()` 封装 Topic 命名规则，调用方无需关心细节
- `displayText()` 提供 UI 友好的显示格式
- `parseInfoJson()` 解析数据库中的 JSON 字段

### 2.4 IO 状态结构 (IOState)

机器人 IO 状态使用位域紧凑存储：

```cpp
struct IOState {
    uint8_t digitalOutputH = 0;     // 数字输出高8位 (DO15-DO8)
    uint8_t digitalOutputL = 0;     // 数字输出低8位 (DO7-DO0)
    uint8_t digitalInputH = 0;      // 数字输入高8位 (DI15-DI8)
    uint8_t digitalInputL = 0;      // 数字输入低8位 (DI7-DI0)
    std::array<uint16_t, 2> analogInput = {0, 0};   // 模拟输入 AI
    std::array<uint16_t, 2> analogOutput = {0, 0};  // 模拟输出 AO
    uint8_t toolDigitalOutput = 0;  // 工具端 DO
    uint8_t toolDigitalInput = 0;   // 工具端 DI

    // 检查指定数字输出位
    bool isDigitalOutputHigh(int bit) const {
        if (bit < 8) {
            return (digitalOutputL >> bit) & 1;
        } else {
            return (digitalOutputH >> (bit - 8)) & 1;
        }
    }

    // 检查指定数字输入位
    bool isDigitalInputHigh(int bit) const {
        if (bit < 8) {
            return (digitalInputL >> bit) & 1;
        } else {
            return (digitalInputH >> (bit - 8)) & 1;
        }
    }
};
```

**位操作说明**：

```
digitalOutputL: [DO7][DO6][DO5][DO4][DO3][DO2][DO1][DO0]
                  ↑
                bit=7                               bit=0

digitalOutputH: [DO15][DO14][DO13][DO12][DO11][DO10][DO9][DO8]
                  ↑
                bit=15                                    bit=8
```

使用位移和掩码操作提取单个位的值，避免为每个 IO 点使用单独的 bool 变量。

### 2.5 机器人实时状态 (RobotRealtimeStatus)

统一所有机器人类型的状态数据，这是桥接服务的核心输出：

```cpp
struct RobotRealtimeStatus {
    // === 标识信息 ===
    QString deviceId;            // 关联数据库的设备ID
    int robotIndex = 0;          // 机器人索引（多机器人场景）
    QString robotName;           // 机器人名称 "Fr1_publisher"
    QString robotType;           // 机器人类型 "法奥机器人"
    QString softVersion;         // 软件版本 "v3.8.1"
    QString robotIp;             // IP 地址

    // === 任务状态 ===
    int currentTaskIndex = 0;    // 当前任务索引
    int currentStepIndex = 0;    // 当前步骤索引
    int programState = 0;        // 程序状态：0=停止, 1=运行, 2=暂停
    bool motionDone = true;      // 运动完成标志

    // === 错误信息 ===
    int errorCode = 0;           // 错误码
    int mainFaultCode = 0;       // 主故障码
    int subFaultCode = 0;        // 子故障码

    // === 运行状态 ===
    int robotMode = 0;           // 模式：0=手动, 1=自动
    double speed = 0.0;          // 当前速度

    // === 位姿数据 ===
    std::array<double, 6> jointPos = {0};  // 关节角度 J1-J6 (度)
    std::array<double, 6> cartPos = {0};   // 笛卡尔 X,Y,Z,RX,RY,RZ (mm/度)

    // === IO 状态 ===
    IOState io;

    // === 安全状态 ===
    bool emergencyStop = false;  // 急停标志

    // === 时间戳 ===
    qint64 timestamp = 0;        // 接收时间戳 (ms)

    // === 辅助方法 ===
    bool hasError() const { return errorCode != 0 || mainFaultCode != 0; }
    QString modeString() const { return robotMode == 1 ? "自动" : "手动"; }

    QString jointPosString() const {
        return QString("J1=%1, J2=%2, J3=%3, J4=%4, J5=%5, J6=%6")
            .arg(jointPos[0], 0, 'f', 2)
            .arg(jointPos[1], 0, 'f', 2)
            // ...
    }

    void updateTimestamp() {
        timestamp = QDateTime::currentMSecsSinceEpoch();
    }
};
```

**设计思路**：
- **统一抽象**：不同机器人的状态映射到统一结构，UI 层无需关心具体机器人类型
- **使用 `std::array`**：固定大小，内存连续，比 `QVector` 更高效
- **辅助方法**：`hasError()`, `modeString()` 等封装常用判断逻辑

### 3.2 生命周期管理

#### 启动服务

```cpp
bool MfmsRosBridge::start() {
    QMutexLocker locker(&mutex_);

    if (running_) {
        qWarning() << "[MfmsRosBridge] 服务已在运行中";
        return true;
    }

    // 1. 检查 ROS 节点
    if (!rosNode_) {
        emit errorOccurred(RosBridgeError::ROS_NODE_NULL, "ROS 节点未初始化");
        return false;
    }

    // 2. 初始化数据库连接
    if (!initDatabase()) {
        qWarning() << "[MfmsRosBridge] 数据库连接失败";
        // 不阻止启动，允许纯 ROS 模式运行
    }

    // 3. 创建 ROS spin 定时器
    spinTimer_ = new QTimer(this);
    connect(spinTimer_, &QTimer::timeout, this, &MfmsRosBridge::spinRos);
    spinTimer_->start(1000 / spinRateHz_);  // 默认 100Hz → 10ms

    running_ = true;
    return true;
}
```

**关键设计**：
- **互斥锁保护**：使用 `QMutexLocker` 确保线程安全
- **优雅降级**：数据库连接失败不阻止启动，支持纯 ROS 模式
- **定时器驱动**：使用 `QTimer` 定时调用 `rclcpp::spin_some()`

#### ROS Spin 机制

ROS 2 的回调需要通过 `spin` 触发，但 GUI 程序不能阻塞在 `spin()` 中：

```cpp
void MfmsRosBridge::spinRos() {
    if (rosNode_ && running_) {
        rclcpp::spin_some(rosNode_);  // 非阻塞，处理当前可用的回调
    }
}
```

**为什么用定时器而非单独线程**：
- 避免多线程复杂性
- 回调天然在 Qt 主线程执行，无需跨线程同步
- `spin_some()` 是非阻塞的，不会卡住 GUI

### 3.3 设备查询

从数据库查询在线设备列表：

```cpp
void MfmsRosBridge::refreshOnlineDevices() {
    if (!dbConnected_) {
        emit onlineDevicesUpdated(QList<OnlineDevice>());
        return;
    }

    QList<OnlineDevice> devices;
    QSqlQuery query(db_);

    // 联合查询 device_state 和 device 表
    query.prepare(R"(
        SELECT
            ds.id,
            ds.state,
            ds.info,
            COALESCE(d.group_id, 0) as group_id
        FROM device_state ds
        LEFT JOIN device d ON ds.id = d.id
        WHERE ds.state IN ('online', 'connected')
        ORDER BY d.group_id, ds.id
    )");

    if (!query.exec()) {
        emit errorOccurred(RosBridgeError::DB_QUERY_FAILED,
                           query.lastError().text());
        return;
    }

    while (query.next()) {
        OnlineDevice dev;
        dev.id = query.value("id").toString();
        dev.state = query.value("state").toString();
        dev.groupId = query.value("group_id").toInt();

        // 解析 info JSON 字段
        QString infoStr = query.value("info").toString();
        if (!infoStr.isEmpty()) {
            QJsonDocument doc = QJsonDocument::fromJson(infoStr.toUtf8());
            if (doc.isObject()) {
                dev.parseInfoJson(doc.object());
            }
        }

        // 推断设备类型
        if (dev.type == DeviceType::Unknown) {
            dev.type = inferDeviceType(dev.name, dev.typeName);
        }

        devices.append(dev);
    }

    emit onlineDevicesUpdated(devices);
}
```

**查询逻辑说明**：
- **LEFT JOIN**：即使 device 表没有对应记录也返回（兼容性）
- **COALESCE**：空值时使用默认值 0
- **ORDER BY**：按组 ID 和设备 ID 排序，方便 UI 展示

### 3.4 Topic 订阅

#### 订阅流程

```cpp
void MfmsRosBridge::subscribeDevice(const OnlineDevice& device) {
    QMutexLocker locker(&mutex_);

    if (!running_ || !rosNode_) {
        emit subscribeResult(device.id, false, "服务未启动");
        return;
    }

    // 如果已订阅相同设备，直接返回
    if (isSubscribed_ && currentDevice_.id == device.id) {
        emit subscribeResult(device.id, true, QString());
        return;
    }

    // 取消之前的订阅
    doUnsubscribe();

    try {
        subscribeByType(device);  // 根据类型创建订阅
        currentDevice_ = device;
        isSubscribed_ = true;
        emit subscribeResult(device.id, true, QString());
    } catch (const std::exception& e) {
        emit subscribeResult(device.id, false, e.what());
    }
}
```

#### 类型特定订阅

根据设备类型创建对应的订阅器：

```cpp
void MfmsRosBridge::subscribeByType(const OnlineDevice& device) {
    QString topicName = device.topicName();

    switch (device.type) {
        case DeviceType::FrRobot: {
            auto sub = rosNode_->create_subscription<com_interfaces::msg::FrRobotState>(
                topicName.toStdString(),
                rclcpp::SensorDataQoS(),  // 适用于高频传感器数据
                [this](const std::shared_ptr<com_interfaces::msg::FrRobotState> msg) {
                    this->onFrRobotState(msg);
                }
            );
            subscription_ = sub;  // 存储为基类指针
            break;
        }

        case DeviceType::HsRobot: {
            auto sub = rosNode_->create_subscription<com_interfaces::msg::HsRobotState>(
                topicName.toStdString(),
                rclcpp::SensorDataQoS(),
                [this](const std::shared_ptr<com_interfaces::msg::HsRobotState> msg) {
                    this->onHsRobotState(msg);
                }
            );
            subscription_ = sub;
            break;
        }

        case DeviceType::SeerAgv: {
            auto sub = rosNode_->create_subscription<com_interfaces::msg::SeerAgvState>(
                topicName.toStdString(),
                rclcpp::SensorDataQoS(),
                [this](const std::shared_ptr<com_interfaces::msg::SeerAgvState> msg) {
                    this->onSeerAgvState(msg);
                }
            );
            subscription_ = sub;
            break;
        }

        default:
            throw std::runtime_error("Unsupported device type");
    }
}
```

**关键点**：
- **SensorDataQoS**：ROS 2 预定义的传感器数据 QoS 配置，best-effort、深度较小
- **Lambda 捕获**：捕获 `this` 指针以调用成员函数
- **基类指针存储**：使用 `SubscriptionBase` 基类统一存储不同类型订阅器

### 3.5 消息转换

将 ROS 消息转换为统一的 `RobotRealtimeStatus` 结构：

#### 法奥机器人消息转换

```cpp
RobotRealtimeStatus MfmsRosBridge::convertFrMessage(
    const std::shared_ptr<com_interfaces::msg::FrRobotState>& msg)
{
    RobotRealtimeStatus status;

    // 标识信息
    status.robotIndex = msg->robot_index;
    status.robotName = QString::fromStdString(msg->robot_name);
    status.robotType = QString::fromStdString(msg->robot_type);
    status.softVersion = QString::fromStdString(msg->robot_soft_index);
    status.robotIp = QString::fromStdString(msg->robot_ip);

    // 任务状态
    status.currentTaskIndex = msg->curtask_index;
    status.currentStepIndex = msg->curstep_index;
    status.programState = msg->program_state;
    status.motionDone = (msg->robot_motion_done != 0);

    // 错误信息
    status.errorCode = msg->robot_err_code;
    status.mainFaultCode = msg->main_code;
    status.subFaultCode = msg->sub_code;

    // 位姿数据
    for (size_t i = 0; i < 6 && i < msg->jt_cur_pos.size(); ++i) {
        status.jointPos[i] = msg->jt_cur_pos[i];
    }
    for (size_t i = 0; i < 6 && i < msg->tl_cur_pos.size(); ++i) {
        status.cartPos[i] = msg->tl_cur_pos[i];
    }

    // IO 状态
    status.io.digitalOutputH = msg->cl_dgt_output_h;
    status.io.digitalOutputL = msg->cl_dgt_output_l;
    status.io.digitalInputH = msg->cl_dgt_input_h;
    status.io.digitalInputL = msg->cl_dgt_input_l;

    status.emergencyStop = (msg->emergency_stop != 0);
    status.updateTimestamp();

    return status;
}
```

#### AGV 消息转换

AGV 没有关节数据，需要特殊映射：

```cpp
void MfmsRosBridge::onSeerAgvState(
    const std::shared_ptr<com_interfaces::msg::SeerAgvState> msg)
{
    RobotRealtimeStatus status;
    status.deviceId = currentDevice_.id;
    status.robotName = QString::fromStdString(msg->vehicle_name);
    status.robotType = "仙工AGV";

    // 将 AGV 位姿映射到 cartPos
    status.cartPos[0] = msg->pose_x * 1000;  // m → mm
    status.cartPos[1] = msg->pose_y * 1000;
    status.cartPos[2] = 0;                    // Z (AGV 无高度)
    status.cartPos[5] = msg->angle * 180.0 / M_PI;  // rad → deg

    // 速度计算
    status.speed = std::sqrt(msg->vel_x * msg->vel_x + msg->vel_y * msg->vel_y);

    // blocked 映射到急停
    status.emergencyStop = msg->blocked;

    // confidence 存储到 jointPos[0]
    status.jointPos[0] = msg->confidence;

    status.updateTimestamp();
    emit robotStatusUpdated(status);
}
```

**映射策略**：
- **单位转换**：米→毫米，弧度→度
- **字段复用**：`jointPos[0]` 存储定位可信度
- **状态映射**：`blocked` 映射为急停状态

## 4. Qt 元类型注册

Qt 信号槽传递自定义类型时，需要注册元类型：

```cpp
// 在头文件末尾声明
Q_DECLARE_METATYPE(mfms::DeviceType)
Q_DECLARE_METATYPE(mfms::OnlineDevice)
Q_DECLARE_METATYPE(mfms::RobotRealtimeStatus)
Q_DECLARE_METATYPE(QList<mfms::OnlineDevice>)

// 在 .cpp 文件中注册（静态初始化）
static bool registerMetaTypes() {
    qRegisterMetaType<DeviceType>("mfms::DeviceType");
    qRegisterMetaType<OnlineDevice>("mfms::OnlineDevice");
    qRegisterMetaType<RobotRealtimeStatus>("mfms::RobotRealtimeStatus");
    qRegisterMetaType<QList<OnlineDevice>>("QList<mfms::OnlineDevice>");
    return true;
}
static bool _metatypes_registered = registerMetaTypes();
```

**为什么需要注册**：
- 跨线程信号槽使用 `Qt::QueuedConnection`
- 队列连接需要序列化/反序列化对象
- 未注册的类型无法在队列中传递

## 5. 使用示例

### 5.1 基本使用

```cpp
// 1. 创建 ROS 节点
auto node = std::make_shared<rclcpp::Node>("qt_app_node");

// 2. 创建桥接服务
auto bridge = new MfmsRosBridge(node, this);

// 3. 连接信号
connect(bridge, &MfmsRosBridge::onlineDevicesUpdated,
        this, &MainWindow::onDevicesUpdated);

connect(bridge, &MfmsRosBridge::robotStatusUpdated,
        this, &MainWindow::onStatusUpdated);

connect(bridge, &MfmsRosBridge::errorOccurred,
        this, [](int code, const QString& msg) {
            qWarning() << "Error:" << code << msg;
        });

// 4. 启动服务
bridge->start();

// 5. 刷新设备列表（触发下拉框更新）
bridge->refreshOnlineDevices();

// 6. 用户选择设备后订阅
void MainWindow::onDeviceSelected(int index) {
    OnlineDevice device = deviceList_[index];
    bridge->subscribeDevice(device);
}

// 7. 处理实时状态更新
void MainWindow::onStatusUpdated(const RobotRealtimeStatus& status) {
    // 更新 UI
    ui->labelJointPos->setText(status.jointPosString());
    ui->labelCartPos->setText(status.cartPosString());
    ui->labelSpeed->setText(QString::number(status.speed, 'f', 1));

    // 错误状态
    if (status.hasError()) {
        ui->labelError->setText(QString("Error: %1").arg(status.errorCode));
    }
}
```

## 7. 线程安全分析

| 操作 | 线程 | 保护机制 |
|------|------|----------|
| `start()/stop()` | GUI 主线程 | `QMutex` |
| `subscribeDevice()` | GUI 主线程 | `QMutex` |
| `spinRos()` | GUI 主线程（定时器） | 无需（单线程） |
| ROS 回调 | GUI 主线程（spin_some） | 无需（单线程） |
| 信号发射 | GUI 主线程 | 无需 |

**设计优势**：通过 `QTimer` + `spin_some()` 使所有操作在 GUI 主线程执行，避免跨线程同步。

## 8. 关键设计模式总结

### 8.1 桥接模式

RosBridge 作为 ROS 2 与 Qt 之间的桥梁：
- 输入：ROS Topic (FrRobotState, HsRobotState, SeerAgvState)
- 输出：Qt 信号 (robotStatusUpdated)
- 转换：消息类型 → 统一结构

### 8.2 策略模式

根据设备类型选择不同的订阅策略：

```
DeviceType::FrRobot  → create_subscription<FrRobotState>
DeviceType::HsRobot  → create_subscription<HsRobotState>
DeviceType::SeerAgv  → create_subscription<SeerAgvState>
```

### 8.3 观察者模式

Qt 信号槽机制本质是观察者模式：
- Subject：MfmsRosBridge
- Observers：MainWindow 等 UI 组件
- Event：robotStatusUpdated 信号
