# 6.9-Bug-01 线程懒启动的消息队列竞态

> 这次提交修复的第一类问题：网络线程在 `SendPacket()` 里"用到才启动"，带来两层隐患——一层是语法 bug（`=` 写成了 `==`），另一层是更隐蔽的竞态：线程刚创建，消息队列还没建好，就已经有消息投进来了。

---

## 两个问题叠在一起

旧版本的 `SendPacket()` 长这样：

```cpp
bool CClientSocket::SendPacket(HWND hWnd, const CPacket& pack, bool isAutoClosed, WPARAM wParam)
{
    // ❌ 语法 bug：= 是赋值，不是比较
    if (m_hThread = INVALID_HANDLE_VALUE)
    {
        m_hThread = (HANDLE)_beginthreadex(NULL, 0, &CClientSocket::threadEntry, this, 0, &m_nThreadID);
    }
    ...
    return PostThreadMessage(m_nThreadID, WM_SEND_PACK, ...);
}
```

**第一层问题——语法 bug**：

`if (m_hThread = INVALID_HANDLE_VALUE)` 不是比较，是赋值。
执行过程：
1. `m_hThread` 被赋值为 `INVALID_HANDLE_VALUE`（即 `(HANDLE)-1`，非零值）
2. `if` 对赋值结果求布尔值 → **永远为 `true`**
3. 每次调用 `SendPacket` 都会执行 `_beginthreadex`，重新创建一个线程

结果是：每发一次包，系统里就多一条 `threadFunc2` 在跑，多个线程争同一个消息队列和 socket。

> 正确写法应该是 `if (m_hThread == INVALID_HANDLE_VALUE)`。
> 这类 `=` vs `==` 的误写可以用编译器警告 `/W4` 在编译阶段捕获。

**第二层问题——竞态**：

即便把 `=` 改成 `==`，"在 `SendPacket` 里懒启动"这个结构本身也不安全：

```
调用 SendPacket()
  → _beginthreadex 启动网络线程
  → 立刻 PostThreadMessage(m_nThreadID, WM_SEND_PACK, ...)
                ↑
      此时线程可能还没跑到 GetMessage()
      Win32 消息队列尚未创建
      PostThreadMessage 投递的消息可能直接丢失
```

Win32 线程的消息队列是**懒创建**的，只有线程第一次调用 `GetMessage()` 或 `PeekMessage()` 时，系统才会为这个线程分配消息队列。在这之前，`PostThreadMessage` 会失败（返回 FALSE，`GetLastError` 返回 `ERROR_INVALID_THREAD_ID`）。

这个竞态在多数情况下不会每次都触发，所以很难在测试时稳定复现，但在慢机器或高负载时会暴露出来。

---

## 这次提交的修法

把线程创建从 `SendPacket()` 挪到 `CClientSocket` 构造函数，并用事件做"启动握手"：

```cpp
// 构造函数里
m_eventInvoke = CreateEvent(NULL, TRUE, FALSE, NULL);
m_hThread = (HANDLE)_beginthreadex(NULL, 0, &CClientSocket::threadEntry, this, 0, &m_nThreadID);
WaitForSingleObject(m_eventInvoke, 100);  // 等线程进消息泵
CloseHandle(m_eventInvoke);

// threadFunc2 里
void CClientSocket::threadFunc2()
{
    SetEvent(m_eventInvoke);  // 告诉主线程"我准备好了"
    while (::GetMessage(&msg, NULL, 0, 0))
    { ... }
}
```

握手完成之后，`SendPacket()` 被调用时，线程的消息队列早就在那里了，不存在任何竞态。

## 经验总结

| 问题 | 教训 |
|------|------|
| `=` 写成 `==` | 条件判断里避免赋值表达式；开 `/W4` 警告；习惯用 `== INVALID_HANDLE_VALUE` 而非 `= ` |
| 懒启动线程 + 立刻发消息 | 线程句柄就绪 ≠ 消息队列就绪，需要用事件或其他同步机制确认 |
| 发包函数兼管线程生命周期 | 职责混入导致竞态更难定位，应单独管理线程生命周期 |
