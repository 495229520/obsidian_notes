# 6.0 第6章热力图总览

> [!note]
> 这篇笔记用于汇总 `6.网络与多线程问题` 目录下所有主笔记的最近修改活跃度。
> 需要启用 `Dataview` 和 `heatmap-tracker` 插件，并在阅读视图中查看。

```dataviewjs
const basePath = "6.项目/远控系统/6.网络与多线程问题";
const pages = dv.pages(`"${basePath}"`)
    .where(p => p.heatmap_tracker === true)
    .where(p => !p.file.path.includes("/Bug目录/"))
    .where(p => p.file.name != "6.0 第6章热力图总览");

const countByDay = new Map();
const entries = [];

for (const page of pages) {
    const date = page.file.mday?.toFormat("yyyy-MM-dd");
    if (!date) continue;

    const weight = Number(page.heatmap_weight ?? 1);
    countByDay.set(date, (countByDay.get(date) ?? 0) + weight);

    entries.push({
        date,
        intensity: weight,
        content: `${page.file.name}\n最后修改：${date}`,
        filePath: page.file.path
    });
}

renderHeatmapTracker(this.container, {
    heatmapTitle: "第6章笔记活动热力图",
    heatmapSubtitle: "按笔记最后修改日期聚合",
    entries,
    separateMonths: false,
    intensityScaleStart: 1,
    intensityScaleEnd: 3
});

dv.header(2, "按日期汇总");
dv.table(
    ["日期", "活跃笔记数"],
    Array.from(countByDay.entries())
        .sort((a, b) => b[0].localeCompare(a[0]))
        .map(([date, count]) => [date, count])
);

dv.header(2, "纳入统计的笔记");
dv.table(
    ["笔记", "最后修改时间", "分组", "权重"],
    pages
        .sort(p => p.file.mday, "desc")
        .map(p => [
            p.file.link,
            p.file.mday?.toFormat("yyyy-MM-dd HH:mm"),
            p.heatmap_group ?? "",
            p.heatmap_weight ?? 1
        ])
);
```

## 使用说明

- 热力图中的每个格子代表某一天。
- 格子颜色越深，表示这一天落在该日期上的第 6 章笔记越多。
- 点击热力图中的条目提示，可定位到对应笔记。
- 如果后面你还想把 `Bug目录` 也算进去，只需要删掉代码里的排除条件。
