# 2.4-2.6 总结：文件管理功能实现

> 本篇总结 [[2.4 获取磁盘分区信息]]、[[2.5 获取指定文件目录下的文件和文件夹]]、[[2.6 文件打开与下载]] 三篇笔记的设计思路、代码结构与演进关系。

---

## 三篇笔记的演进关系

```
2.4 CPacket 打包构造 + 磁盘枚举（MakeDriverInfo）+ 命令分发框架
    ↓ 能力：可响应命令1，获取磁盘列表
2.5 FILEINFO 结构体 + 目录遍历（MakeDirectoryInfo）+ 流式传输
    ↓ 能力：可响应命令2，遍历指定目录
2.6 文件打开（RunFile）+ 文件下载（DownloadFile）+ 分块传输
    ↓ 能力：可响应命令3/4，执行文件和下载文件
```

从 [[2.3-2.1 总结]] 的网络协议基础出发，2.4-2.6 三节在 CPacket + CServerSocket 之上逐步构建出完整的**文件管理业务逻辑**：磁盘枚举 → 目录浏览 → 文件操作。

---

## 命令体系总览

| 命令码 | 功能 | 实现函数 | 请求数据 | 响应数据 | 笔记 |
|:---:|:---|:---|:---|:---|:---|
| 1 | 获取磁盘分区 | `MakeDriverInfo()` | 无 | 盘符列表 `"C,D,E"` | 2.4 |
| 2 | 获取目录列表 | `MakeDirectoryInfo()` | 目录路径 | FILEINFO 结构体流 | 2.5 |
| 3 | 打开/运行文件 | `RunFile()` | 文件路径 | 空包（确认执行） | 2.6 |
| 4 | 下载文件 | `DownloadFile()` | 文件路径 | 文件大小 + 数据块流 + 空包 | 2.6 |

## 各节新增代码与功能

### 2.4 — 磁盘枚举与打包构造

**新增内容**：

| 新增 | 位置 | 功能 |
|------|------|------|
| `CPacket(WORD, BYTE*, size_t)` | ServerSocket.h | 打包构造函数（发送方向） |
| `Data()` / `Size()` | ServerSocket.h | 序列化为字节流 / 获取总长度 |
| `MakeDriverInfo()` | RemoteCtrl.cpp | 枚举磁盘分区，打包发送 |
| `Dump()` | RemoteCtrl.cpp | 十六进制调试输出 |
| `switch(nCmd)` 框架 | RemoteCtrl.cpp | 命令分发入口 |

**核心 API**：`_chdrive(int drive)` — 尝试切换驱动器，成功=存在，失败=不存在。

**CPacket 双向构造**：

| 构造函数 | 方向 | 用途 |
|----------|------|------|
| `CPacket(BYTE*, size_t&)` | 接收 | 从字节流解析包（2.3 已有） |
| `CPacket(WORD, BYTE*, size_t)` | 发送 | 将数据封装成包（2.4 新增） |

### 2.5 — 目录遍历与流式传输

**新增内容**：

| 新增 | 位置 | 功能 |
|------|------|------|
| `FILEINFO` 结构体 | RemoteCtrl.cpp | 文件信息载体（264 字节固定大小） |
| `MakeDirectoryInfo()` | RemoteCtrl.cpp | 遍历目录，逐条发送文件信息 |
| `GetFilePath()` | ServerSocket.h | 从数据包中提取路径字符串 |

**FILEINFO 字段**：

| 字段            | 大小   | 作用           |
| ------------- | ---- | ------------ |
| `IsInvalid`   | 4B   | 路径是否有效（错误反馈） |
| `IsDirectory` | 4B   | 文件 or 目录     |
| `HasNext`     | 4B   | 流式传输结束标志     |
| `szFileName`  | 256B | 文件名          |

**核心 API**：`_findfirst` / `_findnext` / `_findclose` — Windows C 运行时文件遍历。

### 2.6 — 文件打开与下载

**新增内容**：

| 新增 | 位置 | 功能 |
|------|------|------|
| `RunFile()` | RemoteCtrl.cpp | 调用 ShellExecuteA 打开文件 |
| `DownloadFile()` | RemoteCtrl.cpp | 分块读取文件并逐包发送 |
| CPacket 空包支持 | ServerSocket.h | `nSize=0` 时不执行 memcpy |
| `GetFilePath()` 扩展 | ServerSocket.h | 支持命令 2-4（原仅命令 2） |

**核心 API**：

| 函数 | 用途 |
|------|------|
| `ShellExecuteA()` | 根据文件扩展名自动选择程序打开 |
| `fopen_s` / `fread` / `fclose` | C 安全文件 IO |
| `_ftelli64()` | 64 位文件大小（支持 >4GB） |

## 三种传输模式对比

2.4-2.6 展示了三种不同的数据传输模式：

| 模式 | 命令 | 特点 | 示例 |
|------|------|------|------|
| **单包** | 1, 3 | 一个请求对应一个响应包 | 磁盘列表 `"C,D"` |
| **流式** | 2 | 多个同构包 + 结束标志 | FILEINFO 逐条发送 |
| **分块** | 4 | 头包(元信息) + 数据块流 + 空包 | 文件大小 → 1024B 块 → 空包 |

```
命令1/3 单包模式：  请求 → [响应]
命令2   流式模式：  请求 → [数据1] → [数据2] → ... → [HasNext=FALSE]
命令4   分块模式：  请求 → [文件大小] → [块1] → [块2] → ... → [空包]
```

### 从 2.3 到 2.6 的改动

**CPacket 改动**：

| 版本 | 改动 | 原因 |

## 调用关系（2.6 完成时）

```
main()
    ├→ pserver->InitSocket()       → bind(9527) + listen(1)
    ├→ pserver->AcceptClient()     → accept（阻塞等待）
    └→ pserver->DealCommand()      → recv + CPacket 解析 → 返回 sCmd
            ↓
        switch (sCmd)
            ├→ case 1: MakeDriverInfo()
            │           ├→ _chdrive(1..26) 枚举磁盘
            │           ├→ CPacket(1, "C,D", 3) 打包
            │           └→ Send(pack)
            │
            ├→ case 2: MakeDirectoryInfo()
            │           ├→ GetFilePath() 提取路径
            │           ├→ _chdir() 切换目录
            │           ├→ _findfirst/_findnext 遍历
            │           ├→ 逐条 CPacket(2, FILEINFO) + Send
            │           └→ 发送 HasNext=FALSE 结束标志
            │
            ├→ case 3: RunFile()
            │           ├→ GetFilePath() 提取路径
            │           ├→ ShellExecuteA() 打开文件
            │           └→ Send(CPacket(3, NULL, 0)) 确认
            │
            └→ case 4: DownloadFile()
                        ├→ GetFilePath() 提取路径
                        ├→ fopen_s() 打开文件
                        ├→ _ftelli64() 获取大小 → Send 大小包
                        ├→ fread 循环 → 逐块 Send 数据包
                        ├→ fclose()
                        └→ Send(CPacket(4, NULL, 0)) 结束标志
```

## 已知 Bug 汇总

### Bug 1：DownloadFile 循环终止条件错误（2.6, `d7401da`）

```cpp
// ❌ 原代码
do {
    rlen = fread(buffer, 1, 1024, pFile);
    CPacket pack(4, (BYTE*)buffer, rlen);
    CServerSocket::getInstance()->Send(pack);
} while (rlen > 1024);  // fread 最多返回 1024，此条件永远为假

// ✅ 应为
} while (rlen == 1024);  // 读满说明还有数据，未读满说明到文件末尾
```

**影响**：只发送文件的前 1024 字节，后续数据丢失。

### Bug 2：MakeDirectoryInfo 未关闭搜索句柄（2.5, `b733fc3`）

```cpp
int hfind = _findfirst("*", &fdata);  // 打开句柄
// ... 遍历 ...
// ❌ 缺少 _findclose(hfind)，句柄泄漏
```

### Bug 3：MakeDirectoryInfo 结束标志未发送（2.5, `b733fc3`）

```cpp
FILEINFO finfo;
finfo.HasNext = FALSE;
// ❌ 构造了结束标志但未调用 Send()，控制端无法知道传输结束
```

### Bug 4：_findfirst 返回类型截断（2.5, `b733fc3`）

```cpp
int hfind = _findfirst("*", &fdata);
// ❌ 64 位系统下 _findfirst 返回 intptr_t，int 可能截断
// ✅ 应为 intptr_t hfind = ...
```

### Bug 5：DealCommand 内存泄漏（延续自 2.3）

```cpp
char* buffer = new char[BUFFER_SIZE];
// ...
return m_packet.sCmd;  // ❌ 未 delete[] buffer
```

## 安全风险总结

| 风险 | 涉及命令 | 说明 |
|------|----------|------|
| **路径穿越** | 2, 3, 4 | 恶意路径如 `..\..\Windows\System32` 可访问系统文件 |
| **任意文件执行** | 3 | ShellExecuteA 无确认机制，可运行任意程序 |
| **敏感文件下载** | 4 | 无白名单限制，可下载 SAM、注册表等系统文件 |
| **无认证** | 全部 | 任何能连接 9527 端口的客户端都能执行所有命令 |

## 改进建议汇总

| 类别 | 当前状态 | 建议 |
|------|----------|------|
| 磁盘枚举 | `_chdrive()` 有副作用 | 改用 `GetLogicalDrives()` |
| 资源管理 | 手动 fclose / 缺少 _findclose | RAII 封装或使用 C++ 文件流 |
| 文件大小 | 无限制 | 添加最大文件大小检查 |
| 缓冲区 | 1024 字节 | 增大至 64KB 减少包数量 |
| 路径安全 | 无校验 | 白名单 + 禁止 `..` |
| Unicode | 仅 ANSI | 使用 `_wfindfirst` + UTF-8 传输 |
| 错误处理 | 部分命令无反馈 | 统一错误响应格式 |

## 与前一阶段的对比

| 维度 | 2.1-2.3 | 2.4-2.6 |
|------|---------|---------|
| **关注点** | 网络基础设施（Winsock、单例、协议） | 业务功能（磁盘、目录、文件） |
