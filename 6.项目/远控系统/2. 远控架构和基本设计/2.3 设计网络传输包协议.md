> 本节在 [[2.2 网络编程架构设计]] 单例模式的基础上，设计自定义的**应用层协议**，解决 TCP 流式传输的**粘包/拆包**问题。

---

## 为什么需要自定义协议？

### TCP 流式传输的问题

TCP 是**面向字节流**的协议，不保留消息边界：

```
发送端发送:  [命令A: 100字节] [命令B: 50字节] [命令C: 80字节]
接收端收到:  [150字节] [80字节]  ← 消息边界丢失！
```

| 现象 | 说明 |
|------|------|
| **粘包** | 多个小包合并成一个大包 |
| **拆包** | 一个大包被拆分成多个小包 |
| **混合** | 前一个包的尾部 + 后一个包的头部 |

### 解决方案：应用层协议

在数据前添加**协议头**，告诉接收方：
1. 这是一个完整的数据包（魔数标识）
2. 数据包有多长（长度字段）
3. 如何验证数据完整性（校验和）

## 协议包结构设计

### 包格式定义

```
┌──────────┬──────────┬──────────┬────────────────┬──────────┐
│  sHead   │ nLength  │   sCmd   │    strData     │   sSum   │
│  包头     │  包长度   │   命令   │     数据       │  校验和   │
├──────────┼──────────┼──────────┼────────────────┼──────────┤
│  2 Bytes │  4 Bytes │  2 Bytes │   N Bytes      │  2 Bytes │
│  0xFEFF  │  4 + N   │  命令码   │   可变长度      │  累加和   │
└──────────┴──────────┴──────────┴────────────────┴──────────┘
                      ↑←────────────── nLength ─────────────→↑
```

### 字段说明

|    字段     |   类型   | 大小   | 说明                                 |
| :-------: | :----: | ---- | ---------------------------------- |
|  `sHead`  |  WORD  | 2 字节 | 包头魔数，固定为 `0xFEFF`，用于定位包起始位置        |
| `nLength` | DWORD  | 4 字节 | 包体长度 = `sCmd` + `strData` + `sSum` |
|  `sCmd`   |  WORD  | 2 字节 | 控制命令类型（如：1=查看目录, 2=下载文件）           |
| `strData` | string | N 字节 | 实际数据，长度 = `nLength - 4`            |
|  `sSum`   |  WORD  | 2 字节 | 校验和，`strData` 所有字节的累加和             |

### 长度计算关系

```cpp
// 总包长度 = 包头 + 长度字段 + 包体
total_size = 2 + 4 + nLength;

// 数据长度 = 包体长度 - 命令 - 校验
data_size = nLength - 2 - 2 = nLength - 4;
```

> [!tip] 设计要点
> - **包头魔数**：用于在字节流中定位包起始，选择不常见的值（如 0xFEFF）
> - **长度在前**：先读取长度，再决定读取多少数据
> - **校验在后**：校验和放在最后，便于流式计算

## CPacket 类实现（CClientSocket.h）

### 类定义

```cpp
#pragma once
#include "pch.h"
#include "framework.h"

/**
 * @brief 网络数据包封装类
 *
 * 负责将原始字节流解析为结构化的数据包，
 * 处理粘包/拆包问题，并提供校验和验证。
 */
 
    // ==================== 构造函数 ====================

    /**
     * @brief 默认构造函数
     * 初始化所有字段为0/空
     */
    CPacket() : sHead(0), nLength(0), sCmd(0), sSum(0) {}

    /**
     * @brief 拷贝构造函数
     * @param pack 源数据包
     */
    CPacket(const CPacket& pack)
    {
        sHead = pack.sHead;
        nLength = pack.nLength;
        sCmd = pack.sCmd;
        strData = pack.strData;
        sSum = pack.sSum;
    }

    /**
     * @brief 从字节流解析构造
     * @param pData  原始字节流指针
     * @param nSize  [in/out] 输入：可用数据长度；输出：已消费的字节数（0表示解析失败）
     *
     * 核心功能：
     * 1. 在字节流中搜索包头 0xFEFF
     * 2. 解析各字段
     * 3. 验证校验和
     * 4. 返回已消费的字节数，供调用者移除已处理数据
     */
    CPacket(const BYTE* pData, size_t& nSize)
    {
        size_t i = 0;

        // ===== 第一步：搜索包头 =====
        // 在字节流中寻找魔数 0xFEFF
        for (; i < nSize; i++)
        {
            // 检查当前位置是否为包头
            // 注意：0xFEFF 在小端存储中为 FF FE
            // BYTE* → 1 字节 | WORD* → 2 字节 | DWORD* → 4 字节
            if (*(WORD*)(pData + i) == 0xFEFF)
            {
                sHead = *(WORD*)(pData + i);
                i += 2;  // 跳过包头
                break;
            }
        }

        // ===== 第二步：检查数据是否足够 =====
        // 最小包结构：包头(2) + 长度(4) + 命令(2) + 校验(2) = 10字节
        // i 当前指向 nLength 字段，需要确保后面还有 4+2+2=8 字节
        if (i + 4 + 2 + 2 > nSize)
        {
            nSize = 0;  // 数据不完整，通知调用者继续接收
            return;
        }
		
		// DWORD* → 4 字节
        // ===== 第三步：解析长度字段 =====
        nLength = *(DWORD*)(pData + i);
        i += 4;
		
		// WORD* → 2 字节
        // ===== 第四步：解析命令字段 =====
        sCmd = *(WORD*)(pData + i);
        i += 2;

        // ===== 第五步：检查完整包是否已到达 =====
        // nLength 包含：sCmd(已读) + strData + sSum
        // 剩余需要：strData + sSum = nLength - 2
        // 但 i 已经跳过了 sCmd，所以检查 i + (nLength - 2) <= nSize
        // 原代码：if (nLength + i > nSize) 有误
        if (i + nLength - 2 > nSize)  // 修正：需要减去已读取的 sCmd
        {
            nSize = 0;  // 包不完整
            return;
        }

        // ===== 第六步：读取数据载荷 =====
        // 数据长度 = nLength - sCmd(2) - sSum(2) = nLength - 4
        if (nLength > 4)
        {
	        // 预分配 std::string 的空间，大小为数据长度
            strData.resize(nLength - 4);
            // i 指向 strData 的起始位置,将数据拷贝到 strData 的内部缓冲区
            memcpy((void*)strData.c_str(), pData + i, nLength - 4);
            i += nLength - 4;
        }
		
		// sSum: 发送端计算的校验和
        // ===== 第七步：读取校验和 =====
        sSum = *(WORD*)(pData + i);
        i += 2;

		// sum: 接收端验证和
        // ===== 第八步：验证校验和 =====
        WORD sum = 0;
        for (size_t j = 0; j < strData.size(); j++)  // 注意：遍历 strData
        {
	        // 提取低8位 | 确保只保留一个字节的值 (0~255)
            sum += BYTE(strData[j]) & 0xFF;  // 字节累加
        }

        // 校验通过
        if (sum == sSum)
        {
            nSize = i;  // 返回已消费的字节数
            return;
        }

        // 校验失败
        nSize = 0;
    }

    ~CPacket() {}

    /**
     * @brief 赋值运算符
     */
    CPacket& operator=(const CPacket& pack)
    {
        if (this != &pack)
        {
            sHead = pack.sHead;
            nLength = pack.nLength;
            sCmd = pack.sCmd;
            strData = pack.strData;
            sSum = pack.sSum;
        }
        return *this;
    }
};
```

## 解析流程详解

### 状态机视角

```
            ┌─────────────────────────────────────────────────────────┐
            │                      字节流                              │
            │  [垃圾][FF][FE][len...][cmd][data...][sum][下一个包...]   │
            └─────────────────────────────────────────────────────────┘
                     ↑
               搜索包头 0xFEFF
                     │
                     ↓
            ┌─────────────────┐
            │ 检查数据是否足够  │ ──否──→ nSize=0, 等待更多数据
            └────────┬────────┘
                    是│
                     ↓
            ┌─────────────────┐
            │   解析 nLength   │
            └────────┬────────┘
                     │
                     ↓
            ┌─────────────────┐
            │ 检查包是否完整    │ ──否──→ nSize=0, 等待更多数据
            └────────┬────────┘
                    是│
                     ↓
            ┌─────────────────┐
            │  解析 cmd, data  │
            └────────┬────────┘
                     │
                     ↓
            ┌─────────────────┐
            │   校验和验证     │ ──失败──→ nSize=0, 丢弃/重试
            └────────┬────────┘
                   通过│
                     ↓
            ┌─────────────────┐
            │ nSize = 已消费量  │ ──→ 返回，处理下一个包
            └─────────────────┘
```

### 粘包处理示例

```cpp
// 假设接收到 150 字节，包含两个完整包
// 包A: 60字节, 包B: 50字节, 剩余40字节是包C的一部分

buffer: [包A 60B][包B 50B][包C部分 40B]
         ↑
         index = 0

// 第一次解析
CPacket packA(buffer, len);  // len=150
// 返回 len=60, 表示包A占用60字节
// packA 已填充数据

// 移除已处理数据
memmove(buffer, buffer + 60, 150 - 60);
index = 90;  // buffer 现在有 90 字节

// 第二次解析
CPacket packB(buffer, len);  // len=90
// 返回 len=50
// packB 已填充数据

// 移除已处理数据
memmove(buffer, buffer + 50, 90 - 50);
index = 40;  // buffer 现在有 40 字节

// 第三次解析
CPacket packC(buffer, len);  // len=40
// 包C不完整，返回 len=0
// 需要继续 recv() 接收更多数据
```

## 整合到 CServerSocket

### DealCommand 方法

```cpp
#define BUFFER_SIZE 4096

/**
 * @brief 处理客户端命令
 * @return 命令类型 sCmd，-1 表示连接断开或错误
 *
 * 核心逻辑：
 * 1. 循环接收数据到缓冲区
 * 2. 尝试解析出完整数据包
 * 3. 移除已处理数据（处理粘包）
 * 4. 返回命令类型供上层处理
 */
int DealCommand()
{
    // 检查客户端连接是否有效
    if (m_client == -1)
        return -1;

    // 接收缓冲区 - 使用堆内存避免栈溢出
    char* buffer = new char[BUFFER_SIZE];
    memset(buffer, 0, BUFFER_SIZE);

    size_t index = 0;  // 缓冲区中有效数据的长度

    while (true)
    {
        // ===== 接收数据 =====
        // 从 buffer + index 开始写入，最多写 BUFFER_SIZE - index 字节
        size_t len = recv(m_client, buffer + index, BUFFER_SIZE - index, 0);

        if (len <= 0)
        {
            delete[] buffer;  // 释放内存
            return -1;        // 连接断开或错误
        }

        // 更新有效数据长度
        index += len;
        len = index;  // len 现在表示可解析的数据量

        // ===== 尝试解析数据包 =====
        m_packet = CPacket((BYTE*)buffer, len);

        // len > 0 表示成功解析出一个完整包
        if (len > 0)
        {
            // ===== 处理粘包：移除已解析数据 =====
            // 将未处理的数据移动到缓冲区开头
            memmove(buffer, buffer + len, BUFFER_SIZE - len);
            index -= len;  // 更新有效数据长度

            delete[] buffer;
            return m_packet.sCmd;  // 返回命令类型
        }

        // len == 0 表示数据不完整，继续接收
        // 循环继续...
    }

    delete[] buffer;
    return -1;
}
```

## 原始代码问题分析

### Bug 1：重复读取 sCmd

```cpp
// ❌ 原代码（第 40-48 行）
sCmd = *(WORD*)(pData + i);
i += 2;
// ...检查完整性...
sCmd = *(WORD*)(pData + i);  // 重复读取！此时 i 已指向 strData
i += 2;

// ✅ 修正：删除第二次读取
sCmd = *(WORD*)(pData + i);
i += 2;
// 检查完整性后直接读取 strData
```

### Bug 2：校验和计算的循环错误

```cpp
// ❌ 原代码（第 59-62 行）
for (size_t j = 0; i < strData.size(); j++)  // 条件应该是 j < strData.size()
{
    sum += BYTE(strData[i]) & 0xFF;  // 索引应该是 j，不是 i
}

// ✅ 修正
for (size_t j = 0; j < strData.size(); j++)
{
    sum += BYTE(strData[j]) & 0xFF;
}
```

> [!warning] 这个 Bug 的后果
> - 循环条件 `i < strData.size()` 中 `i` 是包解析位置，不是循环变量
> - 导致校验和计算结果完全错误
> - 几乎所有包都会校验失败

### Bug 3：完整性检查逻辑问题

原代码的检查位置和条件不匹配。**检查条件取决于检查发生在读取 sCmd 之前还是之后**：

| 检查位置 | 此时 i 指向 | 剩余需读取 | 正确条件 |
|---------|------------|-----------|---------|
| 读 sCmd **之前** | sCmd | `nLength` | `i + nLength > nSize` |
| 读 sCmd **之后** | strData | `nLength - 2` | `i + nLength - 2 > nSize` |

```cpp
// ✅ 方案 A：先检查，再读 sCmd
nLength = *(DWORD*)(pData + i);
i += 4;
if (i + nLength > nSize)       // sCmd 还没读，需要完整的 nLength
{
    nSize = 0;
    return;
}
sCmd = *(WORD*)(pData + i);
i += 2;

// ✅ 方案 B：先读 sCmd，再检查（本笔记采用此方案）
nLength = *(DWORD*)(pData + i);
i += 4;
sCmd = *(WORD*)(pData + i);
i += 2;
if (i + nLength - 2 > nSize)   // sCmd 已读，只需 nLength - 2
{
    nSize = 0;
    return;
}
```

> [!warning] 常见错误
> 如果用方案 A 的位置（先检查）但用方案 B 的条件（`-2`），会少检查 2 字节，可能导致越界读取。

## 最佳实践建议

### 1. 使用 Modern C++ 特性

```cpp
// ✅ 推荐：使用智能指针管理缓冲区
std::unique_ptr<char[]> buffer = std::make_unique<char[]>(BUFFER_SIZE);

// ✅ 推荐：使用 vector 作为动态缓冲区
std::vector<char> buffer(BUFFER_SIZE);
```

### 2. 添加序列化方法

```cpp
class CPacket
{
public:
    /**
     * @brief 将数据包序列化为字节流（用于发送）
     * @return 序列化后的字节流
     */
    std::string serialize() const
    {
        std::string result;
        result.resize(2 + 4 + nLength);

        size_t i = 0;
        *(WORD*)(result.data() + i) = sHead;     i += 2;
        *(DWORD*)(result.data() + i) = nLength;  i += 4;
        *(WORD*)(result.data() + i) = sCmd;      i += 2;
        memcpy(result.data() + i, strData.data(), strData.size()); i += strData.size();
        *(WORD*)(result.data() + i) = sSum;

        return result;
    }
};
```

### 3. 封装构造函数用于创建包

```cpp
/**
 * @brief 创建发送用数据包
 * @param cmd  命令类型
 * @param data 数据载荷
 */
CPacket(WORD cmd, const std::string& data)
    : sHead(0xFEFF), sCmd(cmd), strData(data)
{
    // 计算校验和
    sSum = 0;
    for (char c : strData)
    {
        sSum += static_cast<BYTE>(c);
    }

    // 计算长度
    nLength = 2 + strData.size() + 2;  // sCmd + strData + sSum
}
```

## 字节序注意事项

> [!warning] 网络字节序
> 当前实现假设通信双方都是 **小端序 (Little-Endian)** 机器（如 x86/x64）。
> 如果需要跨平台，应使用 `htons()`/`ntohs()` 等函数转换字节序。

```cpp
// ✅ 跨平台安全写法
sHead = ntohs(*(WORD*)(pData + i));      // 网络序转主机序
nLength = ntohl(*(DWORD*)(pData + i));   // 网络序转主机序
```
