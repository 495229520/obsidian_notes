# 2.4 获取磁盘分区信息

> 本节在 [[2.3 设计网络传输包协议]] 的基础上，实现**磁盘分区枚举**功能。CPacket、CServerSocket 等已在 2.3 中解释，此处不再重复。

---

## 本节新增功能

### 功能目标

远控系统需要获取被控端的磁盘信息，作为文件管理功能的基础：

```
控制端请求(cmd=1) → 被控端枚举磁盘 → 打包发送 → 控制端显示 C:, D:, E: ...
```

## 新增代码解析

### MakeDriverInfo() - 磁盘枚举函数

```cpp
#include <direct.h>  // _chdrive()

// 驱动器编号: 1=>A 2=>B 3=>C ... 26=>Z
int MakeDriverInfo()
{
    std::string result;
    for (int i = 1; i <= 26; i++)
    {
        if (_chdrive(i) == 0)  // 尝试切换到该驱动器
        {
            if (result.size() > 0)
                result += ',';
            result += 'A' + i - 1;  // 数字转盘符: 1→A, 3→C
        }
    }
    CPacket pack(1, (BYTE*)result.c_str(), result.size());  // 命令码=1
    Dump((BYTE*)pack.Data(), pack.Size());
    return 0;
}
```

#### 核心API: `_chdrive(int drive)`

| 属性 | 说明 |
|------|------|
| 头文件 | `<direct.h>` (Windows专用) |
| 参数 | 驱动器编号：1=A, 2=B, 3=C, ... |
| 返回值 | 成功返回0，失败返回-1 |
| 原理 | 尝试切换当前工作目录到指定驱动器 |

**工作流程**：
```
_chdrive(3) → 尝试 cd C:\ → 成功=C盘存在，失败=C盘不存在/不可访问
```

> [!tip] 盘符映射技巧
> `'A' + i - 1` 利用 ASCII 连续性：`'A'=65, 'B'=66, 'C'=67...`
> 当 i=3 时：`'A' + 3 - 1 = 65 + 2 = 67 = 'C'`

### Dump() - 调试辅助函数

```cpp
void Dump(BYTE* pData, size_t nSize)
{
    std::string strOut;
    for (size_t i = 0; i < nSize; i++)  //`BYTE` = 1 字节，i 代表第 i 个字节
    {
        char buf[8] = "";
        if (i > 0 && (i % 16 == 0))
            strOut += '\n';
	        snprintf(buf, sizeof(buf), "%02X", pData[i] & 0xFF);   //输出 2 位大写十六进制
        strOut += buf;
    }
    strOut += '\n';
    OutputDebugStringA(strOut.c_str());  // 输出到VS调试窗口
}
```

**作用**：将字节流以十六进制格式输出，便于调试协议包内容。

### CPacket 新增构造函数

2.3 中只展示了**解析构造函数**（从字节流解析），本节新增了**打包构造函数**（用于发送）：

```cpp
CPacket(WORD nCmd, const BYTE* pData, size_t nSize)
{
    sHead = 0xFEFF;
    nLength = nSize + 4;  // data + sCmd(2) + sSum(2)
    sCmd = nCmd;
    strData.resize(nSize);
    memcpy((void*)strData.c_str(), pData, nSize);

    // 计算校验和
    sSum = 0;
    for (size_t j = 0; j < strData.size(); j++)
    {
        sSum += BYTE(strData[j]) & 0xFF;
    }
}
```

| 构造函数                           | 用途        | 方向  |
| ------------------------------ | --------- | --- |
| `CPacket(BYTE*, size_t&)`      | 解析收到的数据包  | 接收  |
| `CPacket(WORD, BYTE*, size_t)` | 构建待发送的数据包 | 发送  |

## 数据包验证

### 输出解析

```
FFFE070000000100432C44B300
```

| 字节          | 值      | 含义             |
| ----------- | ------ | -------------- |
| FF FE       | 0xFEFF | sHead (包头魔数)   |
| 07 00 00 00 | 7      | nLength (包体长度) |
| 01 00       | 1      | sCmd (命令：获取磁盘) |
| 43 2C 44    | "C,D"  | strData (磁盘列表) |
| B3 00       | 179    | sSum (校验和)     |

### 验证计算

```
nLength = sCmd(2) + strData(3) + sSum(2) = 7 ✓

校验和 = 'C' + ',' + 'D' = 0x43 + 0x2C + 0x44 = 179 (0xB3) ✓

总包长 = sHead(2) + nLength(4) + 包体(7) = 13 字节 ✓
```

## 注意事项

> [!warning] `_chdrive()` 的局限性
> 1. **副作用**：会改变当前工作目录
> 2. **权限问题**：无访问权限的驱动器会返回失败
> 3. **可移动设备**：U盘/光驱插拔状态可能不准确
> 4. **网络驱动器**：映射的网络驱动器可能响应慢

### 更健壮的替代方案

```cpp
// ✅ 推荐：使用 GetLogicalDrives()
DWORD drives = GetLogicalDrives();  // 返回位掩码
for (int i = 0; i < 26; i++)
{
    if (drives & (1 << i))  // 检查第i位
    {
        char letter = 'A' + i;
        // 驱动器存在
    }
}
```

| 方法 | 优点 | 缺点 |
|------|------|------|
| `_chdrive()` | 简单直接 | 有副作用，慢 |
| `GetLogicalDrives()` | 无副作用，快 | 仅返回存在性 |
| `GetDriveType()` | 可获取类型(固定/可移动/网络) | 需额外调用 |

## 完整代码

> [!note] 代码说明
> 以下代码与 [[2.3 设计网络传输包协议]] 中的 CPacket、CServerSocket 配合使用。
> CPacket 和 CServerSocket 的详细解释见 2.3。

### ServerSocket.h

```cpp
#pragma once
#include "pch.h"
#include "framework.h"

// 关闭结构体字节对齐，确保 CPacket 各字段在内存中紧密排列
// 不对齐则编译器会插入 padding，导致网络包格式错乱
#pragma pack(push)
#pragma pack(1)

/**
 * CPacket - 网络通信数据包
 *
 * 包结构（共 6 + nLength 字节）：
 * ┌──────────┬──────────┬────────┬──────────┬────────┐
 * │ sHead(2) │nLength(4)│ sCmd(2)│ strData  │ sSum(2)│
 * └──────────┴──────────┴────────┴──────────┴────────┘
 *
 * 有两个方向的构造函数：
 * - 打包构造 CPacket(nCmd, pData, nSize)：发送前封装数据
 * - 解析构造 CPacket(pData, nSize)：接收后从字节流还原
 */
class CPacket
{
public:
    // 默认构造函数：创建空包，各字段归零
    CPacket() :sHead(0), nLength(0), sCmd(0), sSum(0) {}

    /**
     * [NEW] 打包构造函数 - 用于构建待发送的数据包
     *
     * @param nCmd   命令码 (如: 1=获取磁盘, 2=获取目录)
     * @param pData  数据载荷指针
     * @param nSize  数据载荷长度（字节数）
     *
     * 与解析构造函数的区别：
     * - 解析构造函数: 从字节流中提取包 (接收方向)
     * - 打包构造函数: 将数据封装成包 (发送方向)
     */
    CPacket(WORD nCmd, const BYTE* pData, size_t nSize)
    {
        sHead = 0xFEFF;           // 包头魔数，用于接收端定位包的起始位置
        nLength = nSize + 4;       // 包体长度 = 数据长度 + sCmd(2字节) + sSum(2字节)
        sCmd = nCmd;               // 设置命令码
        strData.resize(nSize);     // 预分配数据空间
        memcpy((void*)strData.c_str(), pData, nSize);  // 将载荷数据拷贝进包

        // 计算校验和：累加所有数据字节，接收端可用同样方式验证完整性
        sSum = 0;
        for (size_t j = 0; j < strData.size(); j++)
        {
            sSum += BYTE(strData[j]) & 0xFF;  // & 0xFF 防止 char 符号扩展污染结果
        }
    }

    // 拷贝构造函数：逐字段复制，支持值语义
    CPacket(const CPacket& pack)
    {
        sHead = pack.sHead;
        nLength = pack.nLength;
        sCmd = pack.sCmd;
        strData = pack.strData;
        sSum = pack.sSum;
    }

    /**
     * 解析构造函数 - 从原始字节流中解析出一个完整的数据包
     *
     * @param pData  接收缓冲区指针
     * @param nSize  [in/out] 传入缓冲区已用字节数；
     *               解析成功后输出本次消耗的字节数，供调用方移动缓冲区；
     *               解析失败则置 0（数据不足或校验和错误）
     *
     * 处理粘包：逐字节扫描魔数 0xFEFF，找到包头后再读取其余字段
     */
    CPacket(const BYTE* pData, size_t& nSize)
    {
        // === 第一步：扫描包头魔数 0xFEFF ===
        // 网络数据可能粘包，需要逐字节找到包的起点
        size_t i = 0;
        for (; i < nSize; i++)
        {
            if (*(WORD*)(pData + i) == 0xFEFF)  // 找到魔数
            {
                sHead = *(WORD*)(pData + i);
                i += 2;  // 跳过 sHead(2字节)，指向 nLength
                break;
            }
        }

        // === 第二步：检查剩余数据是否足够读取包头其余字段 ===
        // 至少还需要: nLength(4) + sCmd(2) + sSum(2) = 8 字节
        if (i + 4 + 2 + 2 > nSize)
        {
            nSize = 0;  // 数据不足，等待下次接收
            return;
        }

        // === 第三步：读取 nLength，并验证包体数据是否已全部到达 ===
        nLength = *(DWORD*)(pData + i);  // 读取包体长度
        i += 4;                           // 跳过 nLength(4字节)，指向 sCmd
        if (nLength + i > nSize)          // 包体尚未完整接收
        {
            nSize = 0;
            return;
        }

        // === 第四步：读取命令码 ===
        sCmd = *(WORD*)(pData + i);
        i += 2;  // 跳过 sCmd(2字节)，指向 strData

        // === 第五步：读取数据载荷（nLength > 4 说明有实际数据）===
        // nLength = sCmd(2) + strData(?) + sSum(2)，所以 strData 长度 = nLength - 4
        if (nLength > 4)
        {
            strData.resize(nLength - 2 - 2);  // 分配 strData 空间
            memcpy((void*)strData.c_str(), pData + i, nLength - 4);
            i += nLength - 4;  // 跳过 strData，指向 sSum
        }

        // === 第六步：读取校验和，并与重新计算的值对比 ===
        sSum = *(WORD*)(pData + i);
        i += 2;  // 跳过 sSum，i 现在指向本包末尾（下一包起点）
        WORD sum = 0;
        for (size_t j = 0; j < strData.size(); j++)
        {
            sum += BYTE(strData[j]) & 0xFF;
        }
        if (sum == sSum)
        {
            nSize = i;  // 解析成功：返回本包消耗的字节数
            return;
        }
        nSize = 0;  // 校验和不符：丢弃此包
    }

    ~CPacket() {}

    // 赋值运算符：防止自赋值，逐字段拷贝
    CPacket& operator=(const CPacket& pack)
    {
        if (this != &pack)
        {
            sHead = pack.sHead;
            nLength = pack.nLength;
            sCmd = pack.sCmd;
            strData = pack.strData;
            sSum = pack.sSum;
        }
        return *this;
    }

    // 返回整个数据包的总字节数 = sHead(2) + nLength(4) + 包体(nLength)
    int Size() { return nLength + 6; }

    /**
     * 将包序列化为连续字节流，供 send() 直接发送
     *
     * 布局：[sHead 2B][nLength 4B][sCmd 2B][strData NB][sSum 2B]
     * 使用成员 strOut 作为内部缓冲区，避免每次动态分配
     */
    const char* Data()
    {
        strOut.resize(nLength + 6);          // 分配总长度的缓冲区
        BYTE* pData = (BYTE*)strOut.c_str();
        *(WORD*)pData = sHead;               // 写入包头魔数
        pData += 2;
        *(DWORD*)(pData) = nLength;          // 写入包体长度
        pData += 4;
        *(WORD*)pData = sCmd;                // 写入命令码
        pData += 2;
        memcpy(pData, strData.c_str(), strData.size());  // 写入数据载荷
        pData += strData.size();
        *(WORD*)pData = sSum;                // 写入校验和
        return strOut.c_str();
    }

public:
    WORD   sHead;     // 包头魔数，固定为 0xFEFF，用于定位包起始
    DWORD  nLength;   // 包体长度 = sCmd(2) + strData(N) + sSum(2)
    WORD   sCmd;      // 命令码，定义操作类型（1=获取磁盘, 2=目录, ...）
    std::string strData;  // 数据载荷，内容由具体命令决定
    WORD   sSum;      // 校验和，strData 所有字节累加，用于验证完整性
    std::string strOut;   // 序列化缓冲区，Data() 方法写入此处后返回指针
};

// 恢复默认字节对齐，避免影响后续其他结构体
#pragma pack(pop)

/**
 * CServerSocket - 被控端 TCP 服务器（单例）
 *
 * 职责：监听控制端连接，接收命令包，发送响应包
 * 单例保证全程只有一个监听 socket，避免端口冲突
 * 生命周期由内部嵌套类 CHelper 的静态实例管理
 */
class CServerSocket
{
public:
    // 获取单例实例（懒汉式，首次调用时创建）
    static CServerSocket* getInstance() {
        if (m_instance == NULL)
        {
            m_instance = new CServerSocket();
        }
        return m_instance;
    }

    // 绑定端口并开始监听，返回 false 表示失败
    bool InitSocket()
    {
        if (m_sock == -1)
            return false;
        sockaddr_in serv_adr;
        memset(&serv_adr, 0, sizeof(serv_adr));
        serv_adr.sin_family = AF_INET;           // IPv4
        serv_adr.sin_addr.s_addr = INADDR_ANY;   // 监听所有网卡
        serv_adr.sin_port = htons(9527);          // 端口号（htons 转为网络字节序）

        if (bind(m_sock, reinterpret_cast<sockaddr*>(&serv_adr), sizeof(serv_adr)) == -1)
            return false;
        if (listen(m_sock, 1) == -1)  // 最大等待队列长度为 1（只服务一个控制端）
            return false;
        return true;
    }

    // 阻塞等待控制端连接，成功后将客户端 socket 存入 m_client
    bool AcceptClient()
    {
        sockaddr_in client_adr;
        int cli_sz = sizeof(client_adr);
        m_client = accept(m_sock, (sockaddr*)&client_adr, &cli_sz);
        if (m_client == -1)
            return false;
        return true;
    }

#define BUFFER_SIZE 4096
    /**
     * 接收并解析一条命令，返回命令码
     *
     * 循环 recv 直到凑齐一个完整包。
     * 解析成功后将缓冲区已消耗部分移走（memmove），
     * 剩余数据留给下次调用，从而处理粘包。
     *
     * @return 命令码 sCmd，-1 表示连接断开或解析失败
     */
    int DealCommand()
    {
        if (m_client == -1)
            return -1;
        char* buffer = new char[BUFFER_SIZE];
        memset(buffer, 0, BUFFER_SIZE);
        size_t index = 0;  // 缓冲区已用字节数
        while (true)
        {
            // 从 buffer + index 处继续接收，避免覆盖之前未处理的数据
            size_t len = recv(m_client, buffer + index, BUFFER_SIZE - index, 0);
            if (len <= 0)
                return -1;  // 连接断开或错误
            index += len;
            len = index;    // len 作为 [in/out]：传入已有数据量，解析后返回消耗量
            m_packet = CPacket((BYTE*)buffer, len);
            if (len > 0)    // len > 0 说明解析成功，len 为本次消耗字节数
            {
                // 将已解析部分移出缓冲区，保留未解析的残余数据
                memmove(buffer, buffer + len, BUFFER_SIZE - len);
                index -= len;
                return m_packet.sCmd;  // 返回命令码供外层 switch 分发
            }
        }
        return -1;
    }

    // 发送原始字节数据
    bool Send(const char* pData, size_t nSize)
    {
        if (m_client == -1)
            return false;
        return send(m_client, pData, nSize, 0) > 0;
    }

    // 发送 CPacket 对象（自动序列化为字节流）
    bool Send(CPacket& pack)
    {
        if (m_client == -1)
            return false;
        return send(m_client, pack.Data(), pack.Size(), 0) > 0;
    }

private:
    SOCKET m_client;   // 已连接的控制端 socket
    SOCKET m_sock;     // 监听 socket
    CPacket m_packet;  // 复用的包对象，存储最近一次解析结果

    // 禁止赋值和拷贝，强制单例语义
    CServerSocket& operator=(const CServerSocket& ss) {}
    CServerSocket(const CServerSocket& ss)
    {
        m_sock = ss.m_sock;
        m_client = ss.m_client;
    }

    // 私有构造函数：初始化 Winsock 并创建监听 socket
    CServerSocket() {
        m_client = INVALID_SOCKET;
        if (InitSockEnv() == FALSE)
        {
            MessageBox(NULL, _T("无法初始化套接字环境,请检查网络设置！"), _T("初始化错误！"), MB_OK | MB_ICONERROR);
            exit(0);
        }
        m_sock = socket(PF_INET, SOCK_STREAM, 0);  // 创建 TCP socket
    }

    // 析构函数：释放 socket 资源并清理 Winsock
    ~CServerSocket()
    {
        closesocket(m_sock);
        WSACleanup();
    }

    // 初始化 Winsock 环境（必须在任何 socket 操作之前调用）
    BOOL InitSockEnv()
    {
        WSADATA data;
        // MAKEWORD(1,1): 请求 Winsock 1.1 版本
        if (WSAStartup(MAKEWORD(1, 1), &data) != 0)
        {
            return FALSE;
        }
        return TRUE;
    }

    // 销毁单例实例，由 CHelper 析构时调用
    static void releaseInstance()
    {
        if (m_instance != NULL)
        {
            CServerSocket* tmp = m_instance;
            m_instance = NULL;  // 先置空再删除，防止多线程下重入
            delete tmp;
        }
    }

    static CServerSocket* m_instance;  // 单例指针

    /**
     * CHelper - 生命周期守卫（RAII 惯用法）
     *
     * 利用静态成员的构造/析构时机：
     * - 程序启动时 CHelper 构造 → 触发 getInstance() → 创建单例
     * - 程序退出时 CHelper 析构 → 触发 releaseInstance() → 销毁单例
     * 这样无需手动管理单例的创建与销毁
     */
    class CHelper
    {
    public:
        CHelper() { CServerSocket::getInstance(); }
        ~CHelper() { CServerSocket::releaseInstance(); }
    };
    static CHelper m_helper;  // 静态成员，程序生命周期内唯一
};

extern CServerSocket server;
```

### ServerSocket.cpp

```cpp
#include "pch.h"
#include "ServerSocket.h"

// 静态成员类外定义（C++ 规则：静态成员必须在类外提供定义）
// m_instance 初始为 nullptr，等待 getInstance() 首次调用时赋值
CServerSocket* CServerSocket::m_instance = NULL;

// m_helper 的定义触发 CHelper 构造函数，进而调用 getInstance() 创建单例
```

```cpp
/**
 * [NEW] Dump - 十六进制调试输出函数
 *
 * @brief  将字节数组以十六进制格式输出到 VS 调试窗口
 * @param  pData  待输出的字节数组指针
 * @param  nSize  字节数组长度
 *
 * 功能说明：
 * - 每个字节输出为2位十六进制 (如 0x0A → "0A")
 * - 每16字节换一行，便于阅读
 * - 输出到 OutputDebugString，可在 VS 的"输出"窗口查看
 *
 * 用途：调试网络协议包，验证包结构是否正确
 */
void Dump(BYTE* pData, size_t nSize)
{
    std::string strOut;
    for (size_t i = 0; i < nSize; i++)
    {
        char buf[8] = "";
        if (i > 0 && (i % 16 == 0))  // 每16字节换行
            strOut += '\n';
        // %02X: 输出2位十六进制，不足补0
        // & 0xFF: 确保只取低8位，避免符号扩展
        snprintf(buf, sizeof(buf), "%02X", pData[i] & 0xFF);
        strOut += buf;
    }
    strOut += '\n';
    OutputDebugStringA(strOut.c_str());  // 输出到VS调试窗口
}

/**
 * [NEW] MakeDriverInfo - 磁盘分区枚举函数
 *
 * @brief  枚举系统中所有可用的磁盘驱动器，打包成数据包
 * @return 0 表示成功
 *
 * 功能说明：
 * - 遍历 A-Z 共26个可能的驱动器
 * - 使用 _chdrive() 测试驱动器是否存在/可访问
 * - 将存在的驱动器盘符拼接成字符串 (如 "C,D,E")
 * - 封装成 CPacket 数据包 (命令码=1)
 *
 * 数据流向：
 * 枚举磁盘 → 构建字符串 "C,D" → CPacket打包 → 调试输出/网络发送
 */
int MakeDriverInfo()
{
    std::string result;  // 存储磁盘盘符列表

    // 遍历所有可能的驱动器 (1=A, 2=B, ..., 26=Z)
    for (int i = 1; i <= 26; i++)
    {
        // _chdrive(i): 尝试切换到第i个驱动器
        // 返回0表示成功(驱动器存在)，-1表示失败
        if (_chdrive(i) == 0)
        {
            // 非首个盘符前加逗号分隔
            if (result.size() > 0)
                result += ',';
            // 数字转盘符: 1→'A', 2→'B', 3→'C'
            // 原理: 'A'的ASCII码是65，i=1时: 65+1-1=65='A'
            result += 'A' + i - 1;
        }
    }

    // 构建数据包: 命令码=1(获取磁盘), 数据=盘符列表
    CPacket pack(1, (BYTE*)result.c_str(), result.size());

    // 调试输出: 以十六进制显示整个数据包
    Dump((BYTE*)pack.Data(), pack.Size());

    // TODO: 实际应用中应发送给控制端
    // pServer->Send(pack);

    return 0;
}

int main()
{
    int nRetCode = 0;
    HMODULE hModule = ::GetModuleHandle(nullptr);

    if (hModule != nullptr)
    {
        // MFC 初始化
        if (!AfxWinInit(hModule, nullptr, ::GetCommandLine(), 0))
        {
            wprintf(L"错误: MFC 初始化失败\n");
            nRetCode = 1;
        }
        else
        {
            // [NEW] 命令分发框架
            // 实际应用中 nCmd 来自网络接收的数据包
            int nCmd = 1;
            switch (nCmd)
            {
            case 1:  // 命令1: 获取磁盘分区信息
                MakeDriverInfo();
                break;
            // case 2: TODO - 获取目录列表
            // case 3: TODO - 下载文件
            // case 4: TODO - 上传文件
            }
        }
    }
    else
    {
        wprintf(L"错误: GetModuleHandle 失败\n");
        nRetCode = 1;
    }

    return nRetCode;
}
```
