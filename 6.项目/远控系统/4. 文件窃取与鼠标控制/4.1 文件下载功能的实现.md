# 4.1 文件下载功能的实现

> 实现从被控端下载文件到控制端的完整流程，包括 UI 交互、网络传输和文件写入。

---

## 功能概述

| 项目 | 说明 |
|------|------|
| **功能** | 从被控端指定路径下载文件到控制端本地 |
| **命令码** | `sCmd = 4` |
| **触发方式** | 右键菜单 → 下载文件 |
| **涉及组件** | RemoteClientDlg (控制端)、RemoteCtrl (被控端) |

---

## 设计背景

### 问题分析

用户在文件列表中选中一个文件后，需要能够将其下载到本地。这涉及：
1. **UI 交互**：获取用户选中的文件，弹出保存对话框
2. **路径拼接**：将树控件的目录路径与文件名组合成完整路径
3. **分块传输**：大文件需要分块发送，避免内存溢出
4. **进度追踪**：需要知道文件总大小以判断传输完成

### 设计目标

1. 用户选中文件后右键菜单触发下载
2. 先发送文件大小，再分块发送文件内容
3. 控制端根据文件大小判断接收完成

---

## 架构设计

### 整体流程

```
┌──────────────────────────────────────────────────────────────────────┐
│                           控制端 (Client)                             │
├──────────────────────────────────────────────────────────────────────┤
│  1. 右键菜单点击 "下载文件"                                            │
│           ↓                                                          │
│  2. OnDownloadFile() 获取选中文件名                                    │
│           ↓                                                          │
│  3. CFileDialog 弹出保存对话框，用户选择本地保存路径                     │
│           ↓                                                          │
│  4. GetPath() 获取树控件当前目录路径，拼接完整远程路径                   │
│           ↓                                                          │
│  5. SendCommandPacket(4, path) 发送下载命令                           │
│           ↓                                                          │
│  6. 接收第一个包：文件大小 (8 字节 long long)                          │
│           ↓                                                          │
│  7. 循环调用 DealCommand() 接收数据包                                  │
│           ↓                                                          │
│  8. fwrite() 写入本地文件，累计字节数                                   │
│           ↓                                                          │
│  9. 累计字节数 >= 文件大小时，传输完成                                  │
└──────────────────────────────────────────────────────────────────────┘
                              ↑ 网络 ↓
┌──────────────────────────────────────────────────────────────────────┐
│                           被控端 (Server)                             │
├──────────────────────────────────────────────────────────────────────┤
│  1. DealCommand() 收到 sCmd=4                                        │
│           ↓                                                          │
│  2. DownloadFile() 被调用                                             │
│           ↓                                                          │
│  3. GetFilePath() 从数据包解析远程文件路径                             │
│           ↓                                                          │
│  4. fopen_s() 打开文件，失败则发送长度=0                               │
│           ↓                                                          │
│  5. _ftelli64() 获取文件大小，发送 head 包 (8 字节)                    │
│           ↓                                                          │
│  6. 循环 fread() 每次读 1024 字节                                     │
│           ↓                                                          │
│  7. 封装成 CPacket 发送给控制端                                        │
│           ↓                                                          │
│  8. 读取不足 1024 字节时结束循环                                       │
└──────────────────────────────────────────────────────────────────────┘
```

### 数据包结构

| 阶段 | 方向 | 内容 |
|------|------|------|
| 请求 | Client → Server | `CPacket(4, 文件路径, 路径长度)` |
| 响应1 | Server → Client | `CPacket(4, 文件大小, 8)` - **head 包** |
| 响应2..N | Server → Client | `CPacket(4, 文件内容, ≤1024)` - **数据包** |

> 📎 CPacket 协议格式详见 [[2.3 设计网络传输包协议]]

## 核心实现

### 控制端：OnDownloadFile()

> 📁 `RemoteClient/RemoteClientDlg.cpp` : OnDownloadFile (行 348-402)

**技术栈**：
- `CListCtrl::GetSelectionMark()` - 获取列表选中项索引
- `CFileDialog` - MFC 文件保存对话框
- `CTreeCtrl::GetSelectedItem()` - 获取树控件选中项
- `fopen() / fwrite() / fclose()` - C 标准文件操作

**设计思路**：
1. 从列表获取文件名，从树控件获取目录路径
2. 弹出保存对话框让用户选择本地路径
3. 发送下载命令后，先接收文件大小，再循环接收内容

```cpp
void CRemoteClientDlg::OnDownloadFile()
{
    // ===== 1. 获取列表中选中的文件名 =====
    // GetSelectionMark: 返回当前选中项的索引，无选中返回 -1
    int nListSelected = m_List.GetSelectionMark();
    // GetItemText: 获取指定行、指定列的文本，第二参数 0 表示第一列（文件名）
    CString strFile = m_List.GetItemText(nListSelected, 0);

    // ===== 2. 弹出文件保存对话框 =====
    // CFileDialog 参数说明：
    //   FALSE - 保存对话框（TRUE 为打开对话框）
    //   "*" - 默认扩展名
    //   strFile - 默认文件名（使用远程文件名）
    //   OFN_HIDEREADONLY - 隐藏"只读"复选框
    //   OFN_OVERWRITEPROMPT - 覆盖已有文件时提示确认
    CFileDialog dlg(FALSE, "*",
        strFile, OFN_HIDEREADONLY | OFN_OVERWRITEPROMPT,
        NULL, this);

    if (dlg.DoModal() == IDOK)
    {
        // ===== 3. 创建本地文件 =====
        // GetPathName: 获取用户选择的完整保存路径
        // "wb+" - 二进制写入模式，文件不存在则创建
        FILE* pFile = fopen(dlg.GetPathName(), "wb+");
        if (pFile == NULL)
        {
            AfxMessageBox(_T("本地没有权限保存该文件，文件无法创建！"));
            return;
        }

        // ===== 4. 拼接远程文件完整路径 =====
        // 回顾：m_List 文件列表、m_Tree 目录树
        // 从选中的树节点递归向上遍历到根节点，拼出完整的目录路径。
        HTREEITEM hSelected = m_Tree.GetSelectedItem();
        strFile = GetPath(hSelected) + strFile;  // 目录 + 文件名
        TRACE("%s\r\n", LPCSTR(strFile));

        // ===== 5. 发送下载命令 =====
        // sCmd=4 表示下载文件命令
        // 参数为远程文件的完整路径
        int ret = SendCommandPacket(4, false, (BYTE*)(LPCSTR)strFile, strFile.GetLength());
        if (ret < 0)
        {
            AfxMessageBox("执行下载命令失败！");
            TRACE("执行下载失败：ret = %d\r\n", ret);
            return;
        }

        // ===== 6. 接收文件大小（head 包） =====
        // 被控端先发送 8 字节的文件大小
        CClientSocket* pClient = CClientSocket::getInstance();
        // 将数据区的前 8 字节解释为 long long
        long long nLenght = *(long long*)pClient->GetPacket().strData.c_str();
        if (nLenght == 0)
        {
            AfxMessageBox("文件长度为零或者无法读取文件！！！");
            return;
        }

        // ===== 7. 循环接收文件内容 =====
        long long nCount = 0;  // 已接收字节数
        while (nCount < nLenght)
        {
            // DealCommand: 接收并解析一个数据包
            ret = pClient->DealCommand();
            if (ret < 0)
            {
                AfxMessageBox("传输失败！！");
                TRACE("传输失败：ret = %d\r\n", ret);
                break;
            }
            // 将数据包内容写入本地文件
            fwrite(pClient->GetPacket().strData.c_str(),
                   1,
                   pClient->GetPacket().strData.size(),
                   pFile);
            nCount += pClient->GetPacket().strData.size();
        }

        // ===== 8. 清理资源 =====
        fclose(pFile);
        pClient->CloseSocket();
    }
}
```

**关键点解析**：

| 要点 | 说明 |
|------|------|
| 文件大小传输 | 用 `long long` (8 字节) 支持大于 4GB 的文件 |
| 累计计数 | `nCount` 累加每个包的大小，与 `nLenght` 比较判断完成 |
| 路径拼接 | `GetPath()` 返回目录路径，需要加上文件名 |

### 被控端：DownloadFile()

> 📁 `RemoteCtrl/RemoteCtrl.cpp` : DownloadFile (行 124-168)

**技术栈**：
- `fopen_s()` - 安全版文件打开
- `_ftelli64()` - 获取 64 位文件大小
- `fread()` - 读取文件内容
- `fseek()` - 移动文件指针

**设计思路**：
1. 从数据包解析文件路径
2. 先发送文件大小，让客户端知道总量
3. 每次读 1024 字节发送，直到读取不足 1024 字节

```cpp
int DownloadFile()
{
    // ===== 1. 获取文件路径 =====
    std::string strPath;
    CServerSocket::getInstance()->GetFilePath(strPath);
    TRACE("DownloadFile path: [%s] len=%d\r\n", strPath.c_str(), strPath.size());

    long long data = 0;  // 文件大小，初始化为 0
    FILE* pFile = NULL;

    // ===== 2. 打开文件 =====
    // fopen_s: 安全版 fopen，返回错误码而非 NULL
    //   err=0 表示成功
    //   "rb" - 二进制只读模式
    errno_t err = fopen_s(&pFile, strPath.c_str(), "rb");
    if (err != 0)
    {
        TRACE("fopen_s failed: err=%d path=[%s]\r\n", err, strPath.c_str());
        // 文件打开失败，发送大小=0 告知客户端
        CPacket pack(4, (BYTE*)&data, 8);
        CServerSocket::getInstance()->Send(pack);
        return -1;
    }

    if (pFile != NULL)
    {
        // ===== 3. 获取文件大小 =====
        fseek(pFile, 0, SEEK_END);        // 移动到文件末尾
        data = _ftelli64(pFile);           // 获取当前位置（即文件大小）

        // ===== 4. 发送 head 包（文件大小） =====
        CPacket head(4, (BYTE*)&data, 8);  // 8 字节 = sizeof(long long)
        CServerSocket::getInstance()->Send(head);  // ⚠️ 关键：必须发送！

        // ===== 5. 循环读取并发送文件内容 =====
        fseek(pFile, 0, SEEK_SET);         // 移动回文件开头
        char buffer[1024] = "";
        size_t rlen = 0;
        do {
            // fread: 每次最多读取 1024 字节
            rlen = fread(buffer, 1, 1024, pFile);
            CPacket pack(4, (BYTE*)buffer, rlen);
            CServerSocket::getInstance()->Send(pack);
        // ⚠️ 关键：rlen >= 1024 表示可能还有数据
        // 当 rlen < 1024 时，说明已到文件末尾
        } while (rlen >= 1024);

        fclose(pFile);
    }

    // 发送空包表示结束（可选）
    CPacket pack(4, NULL, 0);
    CServerSocket::getInstance()->Send(pack);
    return 0;
}
```

## Bug 修复记录

### Bug 1：树控件选中状态未同步

> 📁 `RemoteClient/RemoteClientDlg.cpp` : LoadFileInfo (行 244-248)

**问题描述**：
`LoadFileInfo()` 使用 `HitTest()` 获取点击位置的树节点，但未设置该节点为选中状态。导致后续 `OnDownloadFile()` 调用 `GetSelectedItem()` 时返回的是**旧的选中项**或 `NULL`。

**现象**：下载的文件路径错误，或提示文件不存在。

```cpp
// ❌ 原代码：只通过 HitTest 获取节点，未设置选中状态
HTREEITEM hTreeSelected = m_Tree.HitTest(ptMouse, 0);
// ... 后续代码使用 hTreeSelected

// ✅ 修复：显式设置选中状态
HTREEITEM hTreeSelected = m_Tree.HitTest(ptMouse, 0);
if (hTreeSelected == NULL)
    return;
if (m_Tree.GetChildItem(hTreeSelected) == NULL)
    return;
m_Tree.SelectItem(hTreeSelected);  // ⬅️ 新增：同步选中状态
```

**原理**：
- `HitTest()` 只是查询点击位置对应的节点，不改变选中状态
- `GetSelectedItem()` 返回当前选中的节点（高亮显示的那个）
- 两者可能不一致，需要用 `SelectItem()` 同步

---

### Bug 2：head 包未发送

> 📁 `RemoteCtrl/RemoteCtrl.cpp` : DownloadFile (行 147-149)

**问题描述**：
创建了 `CPacket head` 用于发送文件大小，但**忘记调用 Send()**。

**现象**：客户端收到的第一个包直接是文件内容，将其误解析为文件大小，导致下载的文件为空或数据错乱。

```cpp
// ❌ 原代码：创建了 head 包但忘记发送
CPacket head(4, (BYTE*)&data, 8);
// 缺少 Send 调用
fseek(pFile, 0, SEEK_SET);

// ✅ 修复：添加 Send 调用
CPacket head(4, (BYTE*)&data, 8);
CServerSocket::getInstance()->Send(head);  // ⬅️ 新增
fseek(pFile, 0, SEEK_SET);
```

---

### Bug 3：循环条件错误导致只传输一次

> 📁 `RemoteCtrl/RemoteCtrl.cpp` : DownloadFile (行 157-160)

**问题描述**：
原代码使用 `while (rlen > 1024)` 作为循环条件，但 `fread(..., 1024, ...)` 最多返回 1024，所以 `rlen` 永远不会大于 1024，导致**循环只执行一次**。

**现象**：只下载文件的前 1024 字节，大文件严重截断。

```cpp
// ❌ 原代码：条件永远不满足
do {
    rlen = fread(buffer, 1, 1024, pFile);  // rlen 最大 = 1024
    CPacket pack(4, (BYTE*)buffer, rlen);
    CServerSocket::getInstance()->Send(pack);
} while (rlen > 1024);  // 永远为 false！

// ✅ 修复：改为 >= 判断
do {
    rlen = fread(buffer, 1, 1024, pFile);
    CPacket pack(4, (BYTE*)buffer, rlen);
    CServerSocket::getInstance()->Send(pack);
} while (rlen >= 1024);  // 读满 1024 时继续，不足时结束
```

**循环条件分析**：

| rlen 值 | 含义 | 应继续循环？ |
|---------|------|-------------|
| 1024 | 读满缓冲区，可能还有更多数据 | ✅ 是 |
| 1~1023 | 未读满，说明已到文件末尾 | ❌ 否 |
| 0 | 已在文件末尾，无数据可读 | ❌ 否 |

## 易错点与调试

> [!warning] 常见错误

### 1. 路径拼接问题

```cpp
// ❌ 错误：直接使用文件名，缺少目录路径
CString strFile = m_List.GetItemText(nListSelected, 0);
SendCommandPacket(4, false, (BYTE*)(LPCSTR)strFile, strFile.GetLength());
// 发送的是 "test.txt" 而不是 "C:\Users\test.txt"

// ✅ 正确：拼接完整路径
HTREEITEM hSelected = m_Tree.GetSelectedItem();
strFile = GetPath(hSelected) + strFile;  // "C:\Users\" + "test.txt"
```

### 2. 文件大小解析错误

```cpp
// ❌ 错误：直接转换指针类型
long long nLength = (long long)pClient->GetPacket().strData.c_str();
// 这是把地址转成 long long，不是读取内容！

// ✅ 正确：解引用指针
long long nLength = *(long long*)pClient->GetPacket().strData.c_str();
// 将字符串首地址解释为 long long* 再解引用
```

### 3. 调试技巧

在关键位置添加 TRACE 输出：

```cpp
// 输出文件路径
```
