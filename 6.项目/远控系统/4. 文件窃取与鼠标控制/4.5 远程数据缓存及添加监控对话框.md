## 设计背景

在 [[4.4 远程桌面显示功能设计与数据接收发送]] 中，我们实现了屏幕数据接收线程的基本框架，但留下了一个关键问题：**如何将网络接收的 PNG 二进制数据转换为可显示的图像？**

本节解决三个核心问题：
1. **PNG 数据解码**：网络传输的是 PNG 压缩数据，需要解码为 CImage 对象
2. **缓存同步机制**：生产者线程写入缓存，消费者对话框读取缓存
3. **监控对话框**：独立窗口显示远程桌面画面

---

## 核心实现分析

### 1. PNG 数据到 CImage 的转换

#### 问题：CImage::Load() 不接受内存指针

MFC 的 `CImage::Load()` 方法只支持两种输入：
- 文件路径：`Load(LPCTSTR pszFileName)`
- IStream 接口：`Load(IStream* pStream)`

网络接收的是内存中的 `BYTE*` 数据，无法直接使用。

#### 解决方案：内存流桥接

```
PNG 二进制数据 (BYTE*)
        ↓
    GlobalAlloc() 分配内存
        ↓
    CreateStreamOnHGlobal() 创建 IStream
        ↓
    IStream::Write() 写入数据
        ↓
    IStream::Seek() 重置读取位置
        ↓
    CImage::Load(IStream*) 解码 PNG
```

#### 实现代码（RemoteClientDlg.cpp）

```cpp
void CRemoteClientDlg::threadWatchData()
{
    CClientSocket* pClient = CClientSocket::getInstance();

    while (_isWatching)
    {
        // 1. 发送屏幕请求命令
        int ret = SendCommandPacket(6, false);
        if (ret == 6)
        {
            // 2. 检查缓存状态（生产者逻辑）
            if (m_isFull == false)
            {
                // 3. 获取 PNG 数据
                BYTE* pData = (BYTE*)pClient->GetPacket().strData.c_str();
                size_t nSize = pClient->GetPacket().strData.size();

                // 4. 分配全局内存
                HGLOBAL hMem = GlobalAlloc(GMEM_MOVEABLE, 0);
                if (hMem == NULL)
                {
                    TRACE("内存不足");
                    Sleep(1);
                    continue;
                }

                // 5. 创建内存流
                IStream* pStream = NULL;
                HRESULT hRet = CreateStreamOnHGlobal(hMem, TRUE, &pStream);
                if (hRet == S_OK)
                {
                    // 6. 写入 PNG 数据
                    ULONG length = 0;
                    pStream->Write(pData, nSize, &length);

                    // 7. 重置流位置到开头
                    LARGE_INTEGER bg = { 0 };
                    pStream->Seek(bg, STREAM_SEEK_SET, NULL);

                    // 8. CImage 从流加载 PNG
                    m_image.Load(pStream);

                    // 9. 标记缓存已满
                    m_isFull = true;
                }
            }
        }
    }
}
```

### 2. COM IStream 接口详解

#### GlobalAlloc 参数

| 标志 | 含义 |
|------|------|
| `GMEM_MOVEABLE` | 允许系统移动内存块（与 IStream 配合必须使用） |
| `GMEM_FIXED` | 固定内存位置（不能用于 CreateStreamOnHGlobal） |

#### CreateStreamOnHGlobal 原型

```cpp
HRESULT CreateStreamOnHGlobal(
    HGLOBAL hGlobal,      // 内存句柄，传 NULL 则自动分配
    BOOL fDeleteOnRelease, // TRUE = 释放 Stream 时自动释放内存
    IStream** ppstm        // 输出的 IStream 指针
);
```

#### IStream::Seek 参数

```cpp
HRESULT Seek(
    LARGE_INTEGER dlibMove,  // 移动偏移量
    DWORD dwOrigin,          // 起始位置
    ULARGE_INTEGER* plibNew  // 新位置（可为 NULL）
);

// dwOrigin 取值：
// STREAM_SEEK_SET (0) - 从流开头计算
// STREAM_SEEK_CUR (1) - 从当前位置计算
// STREAM_SEEK_END (2) - 从流末尾计算
```

### 3. 缓存访问接口设计

#### RemoteClientDlg.h 新增成员

```cpp
class CRemoteClientDlg : public CDialogEx
{
public:
    // 消费者接口：检查缓存是否有数据
    bool isFull() const
    {
        return m_isFull;
    }

    // 消费者接口：获取缓存的图像引用
    CImage& GetImage()
    {
        return m_image;
    }

private:
    CImage m_image;  // 图像缓存
    bool m_isFull;   // 缓存状态标志
};
```

#### 设计考量

| 方法 | 返回类型 | 原因 |
|------|----------|------|
| `isFull()` | `bool` + `const` | 只读查询，不修改状态 |
| `GetImage()` | `CImage&` | 返回引用避免拷贝大对象 |

## 易错点与注意事项

### 1. IStream 资源泄漏

**当前代码问题**：未调用 `pStream->Release()` 释放 COM 对象。

```cpp
// 修正方案
if (hRet == S_OK)
{
    ULONG length = 0;
    pStream->Write(pData, nSize, &length);

    LARGE_INTEGER bg = { 0 };
    pStream->Seek(bg, STREAM_SEEK_SET, NULL);

    m_image.Load(pStream);
    m_isFull = true;

    // 必须释放 IStream
    pStream->Release();  // ← 添加这行
}
```

### 2. 线程生命周期管理

**问题**：对话框关闭后，线程可能仍在运行并访问已销毁的成员。

**解决方案**：
- 需要 `_isWatching` 标志控制线程退出
- 对话框关闭时设置 `_isWatching = false`
- 等待线程安全退出

### 3. 缓存竞态条件

**问题**：`m_isFull` 标志的读写缺乏同步。

**现状分析**：
- 生产者：`if (m_isFull == false) { ... m_isFull = true; }`
- 消费者：`if (pParent->isFull()) { ... }`

**风险**：bool 读写在 x86 上是原子的，但严格来说应使用 `std::atomic<bool>`。
