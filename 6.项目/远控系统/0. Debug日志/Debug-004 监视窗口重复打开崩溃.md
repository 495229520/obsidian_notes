# Debug-004：监视窗口重复打开崩溃

> **日期**：2026-01-27
> **影响范围**：远程桌面监控功能
> **严重程度**：🔴 崩溃
> **状态**：✅ 已修复
> **详细笔记**：[[4.11 远程控制 Bug 修复与线程生命周期管理]]

---

## 现象

关闭远程桌面监视窗口后再次打开，程序崩溃或图像显示异常。

## 调试过程

1. **复现**：打开监视窗口 → 关闭 → 再次打开 → 崩溃
2. **调试器捕获**：崩溃发生在 `CImage::Load()` 或线程访问已销毁对象时
3. **分析线程状态**：发现关闭对话框后，后台线程仍在运行（`for(;;)` 无限循环无退出机制）
4. **分析资源状态**：`CImage` 对象重复 `Load()` 未先 `Destroy()`，导致 GDI 资源泄漏

## 根因

三个问题叠加：
1. **线程无退出机制**：`for(;;)` 无限循环，对话框关闭后线程仍在运行
2. **线程访问已销毁对象**：线程通过父指针访问已关闭的对话框
3. **CImage 资源泄漏**：`Load()` 前未 `Destroy()` 旧图像

## 修复

1. 新增 `bool m_isClosed` 标志，循环改为 `while (!m_isClosed)`
2. 对话框关闭后设置 `m_isClosed = true`，`WaitForSingleObject(hThread, 500)` 等待线程退出
3. `Load()` 前检查 `if ((HBITMAP)m_image != NULL) m_image.Destroy()`

## 调试经验

| 经验 | 说明 |
|------|------|
| **线程生命周期管理** | 线程必须有明确的退出机制，不能用 `for(;;)` 无限循环 |
| **先停线程再销毁资源** | 设置退出标志 → 等待线程结束 → 再销毁共享资源 |
| **GDI 资源必须配对** | `Create/Load` 必须配对 `Destroy`，否则泄漏 |
