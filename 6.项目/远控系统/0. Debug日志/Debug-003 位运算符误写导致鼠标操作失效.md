# Debug-003：位运算符误写导致鼠标操作失效

> **日期**：2026-01-27
> **影响范围**：Cmd 5（鼠标事件）
> **严重程度**：🔴 功能完全失效
> **状态**：✅ 已修复
> **详细笔记**：[[4.11 远程控制 Bug 修复与线程生命周期管理]]

---

## 现象

远程鼠标的双击、按下、弹起操作完全无效，只有单击正常。

## 调试过程

1. **控制端排查**：断点确认 `MOUSEEV` 结构体数据正确，`nAction` 值正确
2. **网络排查**：抓包确认数据包正确发送和接收
3. **被控端排查**：在 `MouseEvent()` 中加 `TRACE("nFlags=%08X", nFlags)` 发现 nFlags 值异常——动作标志位没有被设置

## 根因

```cpp
case 1: nFlags != 0x20;  // ← 比较运算符！结果被丢弃
case 1: nFlags |= 0x20;  // ← 应该是按位或赋值
```

`!=` 是比较运算符，返回 bool 后被丢弃；`|=` 才是按位或赋值。编译器不报错，因为语法合法。

## 修复

将 `!=` 改为 `|=`（3 处：双击、按下、弹起）。

## 调试经验

| 经验 | 说明 |
|------|------|
| **"编译通过≠逻辑正确"** | `!=` 和 `|=` 只差一个字符，编译都通过，但语义完全不同 |
| **开启高警告级别** | `/W4` 会警告"表达式结果未使用"，能捕获此类 Bug |
| **加 TRACE 看中间值** | 在关键变量赋值后打印，快速发现值不符合预期 |
