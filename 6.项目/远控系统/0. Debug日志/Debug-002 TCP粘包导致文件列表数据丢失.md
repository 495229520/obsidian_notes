# Debug-002：TCP 粘包导致文件列表数据丢失

> **日期**：2026-01-19
> **影响范围**：Cmd 2（获取目录列表）
> **严重程度**：🟡 数据丢失
> **状态**：✅ 已修复
> **详细笔记**：[[4.2 文件和盘符显示不全bug修复]]

---

## 现象

服务端成功遍历 27 个文件并发送，客户端只收到 3 个。

## 调试过程

1. **两端加日志对比**：服务端 `TRACE("server:count = %d", count)` 打印 27，客户端 `TRACE("Count = %d", Count)` 打印 3
2. **确认发送端正常**：服务端日志显示 27 个文件全部发送
3. **定位接收端问题**：客户端 `DealCommand()` 每次调用都 `memset(buffer, 0, BUFFER_SIZE)` 清零 + `index` 是局部变量

## 根因

TCP 是流式协议，`recv()` 一次可能收到多个包的数据。但 `DealCommand()` 每次调用都：
1. `memset` 清零缓冲区 → 上次剩余数据丢失
2. `size_t index = 0` 局部变量 → 偏移量无法跨调用保持

## 修复

- `index` 改为 `static` 变量
- 删除 `memset` 清零

## 调试经验

| 经验 | 说明 |
|------|------|
| **两端日志对比** | 发送端和接收端各加计数日志，快速定位丢失发生在哪一侧 |
| **TCP 流式思维** | `recv` 一次收到的数据量不等于 `send` 一次发送的数据量 |
