# Debug-006：文件下载只传输 1024 字节

> **日期**：2026-01-19
> **影响范围**：Cmd 4（文件下载）
> **严重程度**：🔴 数据截断
> **状态**：✅ 已修复
> **详细笔记**：[[4.1 文件下载功能的实现]]

---

## 现象

下载文件时只收到前 1024 字节，文件被截断。

## 调试过程

1. **对比文件大小**：源文件 50KB，下载后只有 1024 字节
2. **服务端加日志**：`fread` 循环只执行了一次就退出
3. **检查循环条件**：`while (rlen > 1024)` — `fread(..., 1024, ...)` 最多返回 1024，永远不会 > 1024

## 根因

```cpp
do {
    rlen = fread(buffer, 1, 1024, pFile);
    // ...
} while (rlen > 1024);  // ← 永远为 false！fread 最多返回 1024
```

## 修复

```cpp
} while (rlen >= 1024);  // ← 等于 1024 说明可能还有数据，继续读
```

## 调试经验

| 经验 | 说明 |
|------|------|
| **边界条件检查** | `>` vs `>=` 一个字符的差异导致循环只执行一次 |
| **理解 API 返回值** | `fread` 返回实际读取的元素数，最多等于请求数，不会超过 |
