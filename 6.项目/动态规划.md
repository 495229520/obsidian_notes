# 动态规划

动态规划（Dynamic Programming，简称 DP）是一种把大问题拆成子问题，并把子问题答案保存起来避免重复计算的算法思想。

它最适合处理这类问题：当前选择会影响后续结果，暴力递归会反复计算相同子问题，而子问题之间又存在稳定的递推关系。

## 核心思想

动态规划不是某一种固定代码模板，而是一种“填表”思维：

1. 明确每个状态表示什么。
2. 找到状态之间如何递推。
3. 初始化最小规模问题。
4. 按正确顺序填表。
5. 从表中取出最终答案。

> [!tip] 判断能否使用动态规划
> 如果一个问题可以拆成多个规模更小的同类问题，并且这些子问题会被重复使用，就可以考虑动态规划。

## 两个关键性质

### 最优子结构

一个问题的最优解可以由子问题的最优解推出。

例如求最少硬币数时，若最后选了一枚硬币 `coin`，那么剩余金额 `amount - coin` 也必须用最少硬币组成，否则整体就不是最优解。

### 重叠子问题

不同递归分支会重复求解相同子问题。

例如斐波那契数列中：

```text
fib(5)
├── fib(4)
│   ├── fib(3)
│   └── fib(2)
└── fib(3)
```

`fib(3)` 被重复计算。动态规划会把 `fib(3)` 的结果保存下来，下次直接读取。

## 解题步骤

### 1. 定义状态

状态是 DP 表中每个位置的含义。

常见写法：

```text
dp[i] 表示考虑前 i 个元素时的答案
dp[i][j] 表示考虑到第 i 行第 j 列时的答案
dp[i][j] 表示使用前 i 个物品、容量为 j 时的最大价值
```

状态定义要足够清楚，否则后面的转移方程会很难写。

### 2. 写出状态转移方程

状态转移方程描述“当前状态由哪些旧状态推出”。

例如爬楼梯：

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

含义是：到达第 `i` 阶，可以从第 `i - 1` 阶跨一步，也可以从第 `i - 2` 阶跨两步。

### 3. 初始化边界

边界是最小规模问题的答案。

例如爬楼梯：

```text
dp[1] = 1
dp[2] = 2
```

如果边界错了，后面的状态都会跟着错。

### 4. 确定遍历顺序

遍历顺序必须保证：计算当前状态时，它依赖的旧状态已经算出来。

常见顺序：

| DP 类型 | 常见遍历顺序 | 依赖关系 |
|---|---|---|
| 一维 DP | 从左到右 | `dp[i]` 依赖 `dp[i - 1]` |
| 二维 DP | 从上到下、从左到右 | `dp[i][j]` 依赖上方或左方 |
| 背包 DP | 根据是否允许重复选择决定正序或倒序 | 防止物品被重复使用 |

### 5. 返回答案

答案可能是某个固定状态，也可能是整张表中的最大值或最小值。

例如：

```text
返回 dp[n]
返回 dp[m][n]
返回 max(dp[i])
```

## 经典例子：爬楼梯

题意：每次可以爬 1 或 2 阶，问爬到第 `n` 阶有多少种方法。

### 状态定义

```text
dp[i] 表示爬到第 i 阶的方法数
```

### 状态转移

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

### C++ 写法

```cpp
#include <vector>

int climbStairs(int n) {
    if (n <= 2) {
        return n;
    }

    std::vector<int> dp(n + 1, 0);
    dp[1] = 1;
    dp[2] = 2;

    for (int i = 3; i <= n; ++i) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }

    return dp[n];
}
```

时间复杂度是 $O(n)$，空间复杂度是 $O(n)$。

因为当前状态只依赖前两个状态，也可以压缩空间：

```cpp
int climbStairs(int n) {
    if (n <= 2) {
        return n;
    }

    int prev2 = 1;
    int prev1 = 2;

    for (int i = 3; i <= n; ++i) {
        int cur = prev1 + prev2;
        prev2 = prev1;
        prev1 = cur;
    }

    return prev1;
}
```

空间复杂度降为 $O(1)$。

## 常见 DP 类型

| 类型 | 状态设计 | 典型题目 |
|---|---|---|
| 线性 DP | `dp[i]` 表示前 `i` 个位置的答案 | 爬楼梯、打家劫舍、最大子数组和 |
| 二维 DP | `dp[i][j]` 表示两个维度共同决定的状态 | 最小路径和、编辑距离、最长公共子序列 |
| 背包 DP | `dp[i][j]` 或 `dp[j]` 表示容量限制下的最优值 | 01 背包、完全背包、分割等和子集 |
| 区间 DP | `dp[l][r]` 表示区间 `[l, r]` 的答案 | 戳气球、石子合并 |
| 树形 DP | `dp[node]` 表示以某个节点为根的子树答案 | 二叉树打家劫舍 |

## 常见错误

> [!warning] 常见坑
> - 状态定义不清楚，只凭感觉写递推式。
> - 边界初始化漏掉 `0`、空串、第一行或第一列。
> - 遍历顺序错误，导致使用了还没计算出的状态。
> - 空间压缩后仍按原来的方向遍历，破坏旧状态。

## 记忆口诀

```text
一看状态，二写转移；
三定边界，四排顺序；
最后取值，检查复杂度。
```

动态规划题的难点通常不在代码，而在“状态如何定义”和“当前状态依赖哪些旧状态”。做题时先把这两点写清楚，再开始写代码。
